你已经在本对话中附加了 BEDC 论文 PDF (papers/bedc/main.pdf)。基于全文研究, 提议 5-10 个值得作为新 BOARD target 的命题候选。

每个候选必须满足:
1. 题目精确, 一句话能讲清楚要证什么
2. implication 形式 ("在 setup S 下, P(x) 推出 Q(x)") 或类似可形式化陈述
3. **当前论文未证**: 你判断这条结果在 PDF 里没有作为 theorem / lemma / proposition / corollary 出现 (允许重要 obstruction / counterexample 命题)
4. **落地到 BEDC 现有 chapter** 之一: core / proof_obligations / concrete_hardening / concrete_instances / hardening / formalization / proof_sprint / proof_standing
5. **真正可做的尺寸**: 一个 oracle / codex 可以在 1-3 轮内闭合的命题。不是十年项目, 不是博士论文级别的全新理论
6. **优先发表价值**: 主结构定理 / 分类定理 / 刚性定理 / 二分定理 / 等价性定理 / 不可实现性定理 / 预算下界定理 / 多尺度归纳定理 / 局部到全局失败障碍定理 等等. 不要琐碎引理.
7. **不要重复**: 已有 BOARD 候选见下方, 不要提语义等价的同义改写
8. **不要混轴**: 不要提 `closurestatus` / `\leanchecked` / Lean
   verification / chapter retirement 作为 target. B 只生产 paper theorem /
   lemma / proposition / definition 正文块; 验证轴由别的管线处理.
9. **不要参数搬运**: 不要提抽象 carrier/classifier A 与 B 等价后字段逐项
   搬运的 theorem. 候选必须是具体 BEDC 对象上的 closure / inversion /
   coverage / determinacy / uniqueness / obstruction / bridge.
10. **落点安全**: `local_inputs` 应指向具体正文文件. 如果章节文件基本只是
   `\input{{...}}` hub, 请选择 child file. 避免选择已经接近 800 行的文件;
   760 行以上只提能落在现有 child file 的较小 prerequisite.
11. **逻辑结构继承**: Russell 给出的五条读法纪律已经进入论文边界:
    relation-first / description-expansion / contextual-class-reading /
    stratification / witnessed-descent. 在此基础上, 选题时优先继承可落地的
    逻辑学结构, 而不是泛泛引用历史:
    - Gentzen / natural deduction: 找 introduction 与 elimination 不成对的
      BEDC 接口; 候选应是具体对象的 intro-elim soundness / inversion /
      normalization-style theorem.
    - Tarski / hierarchy discipline: 找 object-language 与 meta-language
      混用处; 候选应是 no-self-classifier / no-internal-truth /
      level-separation obstruction.
    - Kripke / possible-world semantics: 找 stability across histories /
      refinements / continuations; 候选应是 monotonicity, persistence, 或
      failure-under-unwitnessed-descent.
    - Sequent calculus / cut elimination: 找 bridge 或 package composition
      是否可去中间项; 候选应是 cut-admissibility-style composition,
      factorization, 或 minimal-certificate theorem.
    - Categorical / fibrational logic: 找 forgetful functor, reindexing,
      pullback, displayed-certificate, 或 fiberwise classifier; 候选应落成
      concrete BEDC forget / lift / reindex / base-change theorem.
    - Proof-theoretic ordinal / well-founded recursion: 找 pipeline 或
      certificate descent 的终止度量; 候选应是 ranking, no-loop, 或
      finite-descent theorem.
    只有当这些结构能转成本论文现有对象上的精确命题时才使用; 不要输出
    “增加哲学说明”或“引用某逻辑学家”的候选.
12. **证明压缩纪律**: 每个候选优先选择能暴露以下至少两类结构的题目:
    - axiom_budget / strength_level: 最弱有限见证、schedule、rate/modulus、
      finite cover、countable construction 或 choice-like surface.
    - cut_rank / elimination_plan: bridge 或中间 carrier 是否可消去.
    - witness_extractor / existence_mode: 存在性是否有可执行见证、bounded
      search、compactness extraction, 或必须标为 nonconstructive boundary.
    - equality_kind / interpretation_kind: equality、equivalence、bisimulation、
      mirror、interpretation、embedding 不得混用.
    - resource_trace / dependency_trace: history、observation、oracle output、
      certificate 的复制和实际依赖必须可追踪.
    - focusing_phase / oracle_mode: 定义展开、normalization、obligation split、
      schema check、paper dedup 是本地确定性工作, 不作为 oracle 发现候选.
    如果候选只是要求 oracle 做确定性展开或格式检查, 丢弃.
13. **Pre-TasteGate 章节准入**: 你可以提出现有章节里的 lemma /
    obligation / ledger row；但如果你认为候选必须成为 `new_chapter`,
    必须先给出可审计的章节级入口面:
    - `carrier_surface`: 新对象的 BEDC-native carrier, 不是外部来源路径或 JSON
    - `classifier_surface`: 如何判定/分类该对象
    - `nontrivial_witness_plan`: 至少两个不同实例或非空且非退化的见证计划
    - `field_faithful_plan`: carrier 字段投影为何不丢失对象判定
    - `structural_atomicity`: 为什么它不是已有 sibling carrier 的换名/搬运
    - `falsifiable_prediction`: 写入后会产生什么可检查的失败/成功预测
    - `independence_witness`: 它不只是已有章节改名的独立性证据
    - `elimination_plan`: 它以后如何被消去、下降、或被下游消费
    缺任一项时不要选 `new_chapter`; 改成
    `existing_chapter_lemma`, `existing_chapter_obligation`,
    `existing_chapter_ledger_row`, 或 `conjecture_fallback`.
14. **Logic packet schema**: 每个候选都必须暴露最小逻辑资源面, 这样
    board intake 可以做确定性 gate:
    - `axiom_budget` / `strength_level`: 只能用
      `B0_finite_witness`, `B1_primitive_recursive_schedule`,
      `B2_rate_or_modulus`, `B3_finite_cover_or_compactness`,
      `B4_countable_construction`, `B5_impredicative_or_choice_like`.
    - `budget_reason`: 说明为什么这是最弱可见层级.
    - 存在性 claim 要有 `existence_mode` 和 witness extractor; 非构造边界
      要显式写 `classical_noncomputable`.
    - bridge / transport / composition 要有 `cut_rank` 和 `elimination_plan`.
    - equality / equivalence / isomorphism / mirror 要有 `equality_kind`.
    - limit / completion / compactness / continuity / Cauchy 要有
      `rate_modulus_surface`.
    - cross-system 或 external movement 要有 `interpretation_kind`, 且不能把
      外部路径写进 paper body.

## 选题策略

优先以下类型 (高发表价值, 真闭合):
- 一组定理因为论文里某个 setup field 多次提到但从未独立成定理
- 已有 A→B 类定理但缺 B→A / 双向版本
- 已有结构定理但缺其极值 / 分类版本
- 已有正例但缺反例 / 障碍定理
- 一个 chapter 已有定义集但缺 representation / soundness theorem 配套
- 多个 chapter 之间有 prerequisite 链但中间环节缺 transitive 闭合定理

避免以下:
- 简单的术语换皮 (\sim_R 改成 \sim_M 之类)
- 已经被 \leanchecked 标记的目标
- `closurestatus` / Lean verification / marker-only 工作
- 抽象参数搬运 / source-equivalence 续集 / 字段逐项 repackaging
- hub 文件或接近行数上限的直接 append 目标
- 跨 BEDC 的纯外部数学 (number theory / general categorical extensions)
- 大到一组定理才能闭合的"研究方向"

quality > quantity. 5 个真 open 强 candidate 远好于 10 个似是而非.

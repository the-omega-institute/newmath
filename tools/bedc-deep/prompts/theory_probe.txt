A genuine open target must satisfy ALL:
1. Concrete proposition expressible as a single implication
   "P(x) implies Q(x) under setup S" — not a survey, not a research direction,
   not "we should investigate".
2. NOT already stated as `\begin{{theorem|lemma|proposition|corollary}}` with
   `\label{{...}}` anywhere under `papers/bedc/parts/`.
3. NOT already covered by `\leanchecked` / `\leanvariant` / `\leansorryd` /
   `\leandef` / `\leanstmt` macros.
4. Within an existing chapter directory.
5. Has clear local context — you cite which existing definitions / theorems
   it builds on, by file:line.
6. NOT a parameter-transport echo: reject claims whose only content is
   "if carrier/classifier A is equivalent to carrier/classifier B, then
   every field/property transports". Valid targets are concrete closure,
   inversion, coverage, determinacy, uniqueness, or obstruction claims about
   an existing BEDC object.
7. Targets a concrete body file. If the natural landing file is a hub
   mostly containing `\input{{...}}` lines, pick a child file instead. Avoid

Audit method (execute thoroughly — many tool calls expected, take your time):

1. Map paper structure. List `papers/bedc/parts/` directory tree. Read
   `papers/bedc/main.tex` to understand chapter ordering. Note which chapters
   are large vs small, which have many theorems vs none.

2. Inventory coverage. Use Grep:
   - All `\label{{...}}` values across parts/ — these are existing claims.
   - All `\leanchecked|leanvariant|leandef|leanstmt|leansorryd{{X}}` markers —
     these are claims tied to Lean.

3. Find structural gap candidates by category:
   - **Definition-only chapters** — `\begin{{definition}}` blocks exist but
     `grep -c '\\begin{{theorem|lemma|proposition}}' file.tex` returns 0.
     Each definition's natural property becomes a candidate.
   - **Composition gaps** — chapter has theorem about source→mid and
     theorem about mid→target, but no transitive composition theorem.
   - **One-side without dual** — A→B exists but not B→A.
   - **Concrete instance missing** — framework defines a TypeClass with
     multiple instances, one natural instance is absent.
   - **Stability-obligation-without-theorem** — definition lists "stability
     obligations" as named fields but never lifts them to standalone
     theorems with proofs.

   for the concept name. If you find ANY matching theorem environment, the
   candidate is NOT open — drop it. If you find only a remark or definition,
   it may still be open (proceed).

5. Filter lane-mismatches. Drop marker-only, `closurestatus`, Lean
   verification, parameter-echo, hub-file, and near-line-cap candidates.

6. Build a private candidate pool before emitting. Consider at least 5

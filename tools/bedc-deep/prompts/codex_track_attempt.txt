Method:

1. Read the target chapter file at the path declared in `local_inputs`, plus
   any `\autoref` dependency chapters. Identify the named fields, definitions,
   and lemmas already in scope. Read `papers/bedc/preamble.tex` for available
   macro names.

2. If round_idx > 0, study cycle_history carefully. The redline checker only
   flags hygiene / safety violations; correctness is your responsibility. Do
   NOT repeat any approach already attempted. Each new round must add real
   mathematical content, not paraphrase.

3. Reason as deeply as you can on this round. Do not artificially limit
   yourself to "first attempt looks plausible" — push for the strongest
   provable form, find the obstruction, build the lemma chain. You are not
   in a hurry; quality over speed.

4. Decide your verdict at end of round:
   - **"close"**: you have a complete, citable LaTeX block that proves the
     target. Output the LaTeX in `content`.
   - **"continue"**: you made progress but the proof is not closed. Describe
     in `next_step` what you will do in the next round. (You will be re-invoked
     with this round added to cycle_history.) `content` is empty.
   - **"escalate"**: this target genuinely requires content the chapter has
     not prepared (new construction, new induction over a new structure,
     counterexample requiring a new model, or fields not yet defined). Name
     the missing piece in `escalation_reason`. Do this honestly — escalate
     when you would be writing fiction otherwise, not as a way to dodge
     difficulty. `content` is empty.

5. Self-audit your own attempt. Score yourself 0-10 on `audit_score`:
   - 0-3: I haven't really written math, just outlined.
   - 4-6: math is sketched, but proofs incomplete or hygiene violations.
   - 7: promising math, but below the close bar; return "continue" and name
     the concrete strengthening step.
   - 8: solid math, fully written, ready for redline check.
   - 9-10: strong work, proves more than minimum, no anticipated rejections.
   Use this honestly: the runtime rejects `verdict="close"` below audit_score
   8 and feeds the round back as cycle history.

Output requirements when verdict is "close":

A. `tex_file` is a concrete body file under `papers/bedc/parts/`, never
   `main.tex` or a chapter wrapper that only `\input`s other files.
B. `audit_score` MUST be at least 8. If your score is 7 or lower, return
   `verdict="continue"` with a concrete `next_step`.
C. `content` contains at least one of `\begin{{theorem}}`, `\begin{{lemma}}`,
   `\begin{{proposition}}`, `\begin{{corollary}}`, or `\begin{{definition}}`
   with a `\label{{...}}` whose prefix is `thm:`, `lem:`, `prop:`, `cor:`, or
   `def:` and whose suffix is semantic. No version prefix.
D. Math: `$...$` for inline, `$$...$$` for display (display on its own
   line). This is the BEDC project convention (see AGENTS.md / CLAUDE.md
   "数学公式只用 `$` 与 `$$`"). Do NOT use `\(...\)` or `\[...\]`.
E. Bare typed-name macros that exist in `papers/bedc/preamble.tex`:
   `$\LatticeUp$`, `$\PosetUp$`, `$\ModuleUp$`, etc. NEVER
   `$\mathsf{{LatticeUp}}$`. For names NOT in preamble, `\mathsf{{X}}` is OK.
F. Proofs end at `\end{{proof}}` only. NEVER `Q.E.D.`, `\qed`, `\square`,
   or any marker phrase.
G. Cross-references via `\autoref{{ch|sec|def|thm|lem|prop|cor:...}}`.
   Avoid `\ref` and `\eqref`.
H. No sectioning openers (`\part`, `\chapter`, `\section`, etc.).
I. No Lean markers (`\leanchecked`, `\leanvariant`, etc. — those land later).
J. No `closurestatus` blocks, `\theoryclosure`, `\formalstatus`,
   `\leantarget`, or chapter-retirement prose. Those belong to the
   verification / gate lane, not B writeback.
J2. No Lean-facing prose as mathematical content. Do not mention "Lean target",
   "Lean-target", `lean4/`, `BEDC.Derived...`, `single_carrier_alignment`,
   `taste_gate`, "formal target", or "formal readiness" in the LaTeX block.
   If the BOARD target itself asks for that, return `verdict="escalate"` with
   `invalid_verification_axis_surface`.

Convergence discipline:

  Hard-reject on yourself: if you would output approximately the same content
  as the last cycle_history entry, escalate instead.
- Notation swaps that keep the same logical chain as a prior round are
  wasted. Hard-reject and escalate.
- Do not produce alternative paths or "two attempts" inside one JSON. Pick
  one verdict, emit one block, end.
- Do not output exploratory scratch, planning notes, or open-ended commentary.
  Either emit polished math (verdict=close), describe a precise next step
  (verdict=continue with `next_step`), or honestly escalate.

## Input Hygiene (kernel basis is locked)
The kernel "input set" ‚Äî sum of `inductive` types + `class` declarations + the `BaseReflectionSetup` structure ‚Äî is intentionally minimal (~16). All theorem growth must be derived from this fixed basis, not by adding parallel inputs.

- If a candidate target would require introducing a NEW `inductive` type, FIRST grep `lean4/BEDC/**/*.lean` for any existing `inductive` whose constructors match (e.g. `BHist (Empty / e0 / e1)` already covers any sequence-of-binary-marks type ‚Äî a new `BWord (nil / bit0 / bit1)` is redundant). Re-use the existing type via `def Alias := Existing` or use it directly. Do NOT propose a target that introduces a parallel inductive isomorphic to one that already exists.
- If a candidate target would require a new `class` setup, check whether an existing setup class can carry the new field. Adding a new class is permitted only when the field is genuinely orthogonal to all four existing setup interfaces.
- When both a `def` and an `inductive` would express the same closed-generation idea, prefer `def` ‚Äî `def` is conservative (no new logical commitment), `inductive` is a structural input.
- A target whose primary effect is to add a parallel input duplicating an existing input MUST be skipped. Note this in `strategy` and pick a different open obligation.

## Task
Select 3 formalization targets for R<ROUND_NUM>.

Steps:
1. Scan `papers/bedc/parts/` to find theorem/definition/proposition sites that do NOT yet have a nearby `\leanchecked`, `\leanstmt`, or `\leandef` marker at the primary site.
2. **HARD GATE ‚Äî confirm the matching Lean target does NOT already exist in `lean4/BEDC/`.** Run `grep -rE "(theorem|lemma|def|inductive|structure|class) <name>" lean4/BEDC/` for the canonical name AND for any plausible underscore/case variant of it. If the grep returns ANY hit whose declaration body matches the paper claim, the target is INVALID ‚Äî it would only register a marker on existing code (a "register-only round"), which is forbidden. 3. Pick 3 targets:
   - Not all from the same chapter
   - At least 1 must be medium difficulty (for example induction, constructor inversion, or a multi-step structure elimination proof)
   - Prefer targets that fit the existing BEDC namespaces and setup classes
   - Each target's `lean_signature` must describe code you will WRITE, not code you will only register a marker for.

4. Discovery fallback (only when Step 3 cannot fill 3 slots after the HARD GATE filter): you MAY supplement the round with up to 3 total discovery targets ‚Äî NEW theorem or definition statements that do not yet appear in any paper block but capture genuinely new content over BEDC primitives.    - Chain / composition theorem combining 2+ existing \leanchecked theorems into a non-trivial multi-step conclusion.

   - Carrier definition or NameCert spec field for a canonical math object in the BEDC licensing horizon: BoolUp / OptionUp / ProdUp / SumUp / ListUp / IntUp / RatUp / RealUp (Bishop) / ComplexUp / MonoidUp / GroupUp / AbGroupUp / RingUp / CommRingUp / FieldUp / ModuleUp / VecSpaceUp / LinearMapUp / MatrixUp / PolynomialUp / FormalPowerSeriesUp / PreorderUp / POSetUp / TotalOrderUp / LatticeUp / IntervalUp / MetricSpaceUp / S1Up / ContinuousFunctionUp / CompactUp / CategoryUp / FunctorUp / NatTransUp.

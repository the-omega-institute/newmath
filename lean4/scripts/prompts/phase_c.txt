## Anti-hollow semantic discipline (HARD GATE)

The `NameCert.mk () () () () ()` pattern is forbidden. Specifically:

1. NO `Unit`-only proofs. Any new `def` / `theorem` / `lemma` whose body is
   `()`, `Unit.unit`, `True.intro`, `trivial`, or a sequence of these for
   each constructor argument is INVALID. Such proofs say nothing about the
   BEDC kernel and are forbidden by the current prompt discipline.

2. NO `MinimalNameCertSetup` in license proofs. New `NameCert` constructions
   must inhabit the `SemanticNameCert` shape, which forces the carrier to be
   a predicate over `BHist` (or another BEDC inductive) and the classifier
   to be a real equivalence relation (typically restriction of `hsame`).

3. Proofs that construct a `NameCert.mk` from non-Unit witnesses tied to
   the BEDC kernel remain valid, but new work MUST use the semantic track.

4. Step 5.5 HARD GATE is supplemented: if `git diff --cached lean4/BEDC/`
   contains any line matching `:= ()$`, `:= Unit$`, `:= Unit.unit$`,
   `NameCert.mk \(\)`, `NameCert.mk Unit\.unit`, `(NameCert.mk \(\) \(\) \(\) \(\) \(\))`,
   or similar trivial constructions, the round is INVALID. Discard the
   paper-only changes and the trivial Lean changes; pick a target that
   produces real semantic content.

If you cannot find a target that satisfies these constraints, output
`{"targets": []}` from Phase B and let gate_check fail the round.
Vacuous certificates are worse than empty rounds.

## Script-backed Phase D gates

Phase D rejects the commit even if Lean and LaTeX compile when any of the following occurs:

1. Register-only round: paper markers changed but no new declaration was added under `lean4/BEDC/`.
2. Stale marker target: a new `\leanchecked`, `\leanstmt`, or `\leandef` marker points to a Lean declaration that was not introduced in this round.
3. New `\leanvariant`: automated rounds must not add variants. If your target only yields a wrapper, projection, weakened form, pair-bundle, or forwarding lemma for an already checked theorem, abandon it and implement a genuinely new statement.
4. Hollow construction: added Lean lines using `Unit`, `()`, `Unit.unit`, `True.intro`, `trivial`, `NameCert.mk ()`, `NameCert.mk Unit.unit`, `MinimalNameCertSetup`, or `import Init.Data.Unit` are rejected.
5. No BEDC touchpoint: every new declaration must mention a concrete BEDC kernel/setup object in its body, such as `BMark`, `BHist`, `msame`, `hsame`, `Ext`, `Cont`, `ProbeBundle`, `SigRel`, `sameSig`, `Pkg`, `InGap`, `NameCert`, `SemanticNameCert`, `BaseReflectionSetup`, `AskSetup`, `PackageSetup`, `DomainSetup`, `NameCertSetup`, or a `BEDC.` object.
6. Parameter echo: new theorem bodies that mainly return assumptions named like `le_refl`, `le_trans`, `carrier_transport`, `lower_transport`, `upper_transport`, or `normalized` are rejected.
7. Carrier/spec-only growth: new `Carrier`, `SourceSpec`, `PatternSpec`, `ClassifierSpec`, `LedgerPolicy`, `Lower`, `Upper`, or `Normalized` definitions are rejected unless the same round adds a theorem whose statement uses those concrete definitions.
8. Duplicate theorem inflation (tree-wide). A new theorem is rejected if any existing theorem under `lean4/BEDC/` has a conclusion equivalent to the new one modulo: binder renaming, hypothesis reordering, qualified-name rewriting (`BEDC.FKernel.Hist.hsame` vs `hsame` after `open`), or definitional unfolding the same file already proves (e.g. `PreorderClassifierSpec h k := hsame h k` makes `... → PreorderClassifierSpec h k` and `... → hsame h k` the same conclusion). Two new theorems in the same round whose conclusions match each other under these rules are also rejected. The cleanup that produced v2.5 deleted six theorems accumulated this way across rounds (`PreorderPrefixLE_target_carrier` / `_result_carrier` / `_preserves_carrier`; `RatCarrier_component_hsame_transport` / `_hsame_transport`; `SumHistoryClassifier_left_right_absurd` / `_mixed_tags_absurd`; `cont_assoc_proof_standing` / `_up_to_hsame_spine`). Do not reintroduce that pattern.

9. Alias `def` rejection. A `def` whose body is `Existing` (single-token alias) or whose body re-bundles fields already exposed by an iff-equivalent predicate the same file already proves is rejected. Concrete forbidden shapes: `def Foo ... := RatCarrier ...`, `def Bar h := UnaryHistory h`, `def Baz h k := hsame h k` next to an existing classifier of the same shape. Use the existing predicate directly via `open`.

## No nursery-shell growth (HARD GATE)

Do not implement targets that only make a horizon module look larger. A round
must make BEDC theory more closed, composable, or concrete.

The following implementation shapes are INVALID even if they compile and pass
the current anti-hollow checks:

1. Parameter echo: a theorem that accepts semantic laws such as `le_refl`,
   `le_trans`, `carrier_transport`, `lower_transport`, `upper_transport`, or
   `normalized` and returns those same laws unchanged or simply bundles them.
   If the proof is mostly `exact le_refl`, `exact le_trans`, or constructor
   packaging of input assumptions, abandon the target.

2. Abstract-shell classifier: adding `Carrier : BHist → Prop`,
   `lower upper : BHist → Prop`, `Le : BHist → BHist → Prop`, or
   `normalized : ... → Prop` without also proving a concrete closure theorem
   about a BEDC-defined predicate. Do not fabricate general Prop interfaces just
   to manufacture a `NameCert`.

3. Isolated derived definitions: adding only carrier/spec definitions for
   `RatUp`, `IntervalUp`, `PreorderUp`, or another horizon object. If you add a
   new derived carrier/spec, the same round must also prove at least one theorem
   that uses that concrete definition: transport, closure, constructor
   inversion, determinacy, composition, or a certificate whose object-specific
   content is not supplied entirely as parameters.

4. Unclosed certificates: a `NameCert` theorem over arbitrary uninterpreted
   predicates is invalid unless the theorem also proves an object-specific BEDC
   property from concrete constructors/relations in the file.

Valid examples:
- `IntCarrier` transport across `hsame` because it uses the concrete carrier
  fields and `unary_transport`.
- List classifier transitivity because it performs induction over list structure
  and uses `hsame_trans` elementwise.
- Composite gap coverage because it constructs a new witness chain from two
  coverage layers.

Invalid examples:
- A generic preorder theorem that takes arbitrary `Le` plus `le_refl` and
  `le_trans`, then returns those assumptions.
- An interval certificate over arbitrary `lower` and `upper` predicates where all
  boundary transport is supplied as parameters and no concrete boundary theorem
  is proved.
- A rational carrier/spec file with no transport or certificate theorem.

If all selected targets reduce to these invalid shapes, abandon them and pick
new targets rather than committing shallow growth.

## Step 5.3: Paper-side discovery anti-parameter-echo (HARD GATE)

## Step 5.4: Paper-side label collision pre-check (HARD GATE)

When this round adds new paper-side blocks (from a `discovery` target — a `\\begin{{theorem}}…\\end{{theorem}}` block inserted into `papers/bedc/parts/`), the parallel paper pipeline may be inserting blocks at the same time. Before commit, run a fast collision scan:

```bash
git diff --cached -- papers/bedc/ \
  | grep -nE "^\+\\\\label\{" \
  | sed -E 's/.*label\{([^}]+)\}.*/\1/' \
  | while read L; do
      cnt=$(grep -rE "\\\\label\{${L}\}" papers/bedc/ | wc -l | tr -d ' ');
      if [ "$cnt" -gt 1 ]; then echo "DUPLICATE: $L"; fi;
    done
```

Any DUPLICATE line means this round is about to introduce a `\\label{}` that is already present in `papers/bedc/parts/` on the current worktree. Even if the rebased base branch does not yet have this label, a sibling paper round could land it before this round merges, and the combined audit would fail. Fix one of:

1. Drop the offending paper insertion entirely from this round and let the paper pipeline own that block; the round still keeps its Lean additions.
2. Rename the `\\label{}` to something that genuinely encodes new content (not a `_v2`, `_alt`, `_under`, `_again` suffix — those are version-narrative and disallowed). Update any internal `\\autoref{}` consumers in the same patch.

Do NOT commit until the duplicate scan returns no DUPLICATE lines.

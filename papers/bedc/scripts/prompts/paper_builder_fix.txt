## Common failure categories

### `! Undefined control sequence` — `\Foo` used but preamble missing
- `grep -rn '\\\\Foo' papers/bedc/parts/` to find the call site.
- Check `papers/bedc/preamble.tex` — if no `\\newcommand{{\\Foo}}` exists,
  add one. Use `\\ensuremath{{\\mathsf{{Foo}}}}` for math-only placeholders.
- This is the #1 cause: paper P-rounds invent new Lean-typeclass-like
  macros (`\\ChapterTasteGate`, `\\GroundCompiler`, etc.) but forget to
  declare them in preamble.

### `! LaTeX Error: \\mathsf allowed only in math mode`
- `\\mathsf{{X}}` used outside `$...$` — wrap the call site in math mode,
  OR redeclare the macro with `\\ensuremath` wrapping.

### `! Missing $ inserted`
- An identifier with subscripts/superscripts used in text mode.
  Wrap the offending phrase in `$...$`.

### `! Package XXX Error:` — package complaint
- Read the message. Often a misused `\\autoref` (label doesn't exist)
  or a `\\input` to a missing file. Fix the reference or the path.

### Other: read `papers/bedc/main.log` directly
- `grep -E '^! ' papers/bedc/main.log` lists every TeX-level error
  in order. Fix the first one — subsequent errors often cascade from
  it. Re-running `make check` does NOT re-trigger TeX errors (it
  skips pdflatex), so use the most recent `main.log` from the
  daemon's last build attempt.

\chapter{Project Changelog}

\section*{v1}
Created the master multi-file project. Frozen core: BEDC v10 finite kernel target. Included proof sprint v10.1 and implementation-facing contract.

\section*{v1.1}
Added the incremental hardening modules:
\begin{enumerate}
  \item Hardening Manifest v1.1;
  \item Package and Gap Policy Expansion;
  \item Typed Naming Certificate Instances;
  \item Machine-Checkable Milestones;
  \item Project Governance and Incremental Maintenance.
\end{enumerate}
The purpose of v1.1 is not to replace the frozen core. It clarifies the live edge of the project and prepares the next proof sprint.


\section*{v1.2}
Added project-navigation and proof-dependency hardening:
\begin{enumerate}
  \item Reader's Roadmap;
  \item Normative Status Table;
  \item Migration and Supersession Index;
  \item Finite-Kernel Theorem Dependency DAG;
  \item Build and Verification Log.
\end{enumerate}
The purpose of v1.2 is to make the master project easier to read, easier to extend, and safer to maintain without rewriting frozen material.


\section*{v1.3}
The v1.3 increment adds a proof-standing part. It does not expand the derived-object layer. It adds closed-generation and no-confusion proof blocks, internal sameness proof blocks, relational extension and continuation proof blocks, signature proof blocks, package/gap proof blocks, naming-certificate proof instances, and a proof status matrix. Machine verification remains pending.

\section*{v1.4}
The v1.4 increment adds concrete-instance modules: concrete signature packages, concrete signature gaps, a concrete finite-kernel Globalize instance, paper-level naming certificates for \(\NatUp\) and \(\AddUp\), and a refined proof status matrix. Machine verification remains pending.

\section*{v1.5}
The v1.5 increment is a corrective concrete-instance hardening release. It does not introduce new derived objects. It internalizes domain membership as \(\InDom(D,h)\), replaces function-like token notation by \(\TokIntro(\Pi,s,p)\), adds package reflection for the concrete signature package instance, internalizes the concrete signature gap policy, and strengthens the \(\AddUp\) certificate with a unary continuation commutativity obligation. Machine verification remains pending.


\section*{v1.5.1}
The v1.5.1 increment is a proof-obligation patch. It freezes the low-risk corrections from v1.5 and turns the remaining live risks into explicit obligations: \(\DomainPolicy(D)\), \(\PkgTokPol(\Pi)\), policy-supported \(\InGapSig\), unary right-shift, and \(\Eone\)-congruence. No new derived-object theory is added. Machine verification remains pending.


\section*{v1.5.2}
The v1.5.2 increment is a proof-readiness patch. It adds the first concrete unary-domain policy instance, an inductive signature-token package-sameness policy, and a narrowed proof plan for unary right-shift and unary continuation commutativity. It introduces no new derived objects and prepares the concrete-instance layer for local machine verification.

\section*{v1.5.3}
The v1.5.3 increment is a local proof patch. It introduces \(\TokUnique(\Pi)\) as the missing token-introduction uniqueness obligation, restates concrete Globalize exactness under explicit policies, decomposes unary right-shift into base and step lemmas, and records unary continuation commutativity as the condition licensing \(\AddUp\). No new derived-object theory is added. Machine verification remains pending.


\section*{v1.5.4}
The v1.5.4 increment is a minimal verification contract. It fixes \(\psame_{\Pi}^{\mathrm{base}}\) as the active one-constructor package-sameness relation, treats \(\psame_{\Pi}^{\mathrm{sig}}\) as its alias, separates \(\psame_{\Pi}^{\mathrm{eq}}\) as an optional future equivalence closure, and records \(\ClosureReflect(\Pi)\) as the missing theorem required before equivalence-closure exactness can be exported. No new derived-object theory is added. Machine verification remains pending.


\section*{v1.5.5}
The v1.5.5 increment is a base-reflection check-target patch. It introduces \(\GeneratedSameSig\) and \(\ExactGlobalizeBase\), isolates base package inversion, token replacement, and base package reflection as checker-facing theorem names, and records \(\ClosureReflect(\Pi)\) as an optional deferred theorem for equivalence-closure exactness. No new derived-object theory is added. Machine verification remains pending.

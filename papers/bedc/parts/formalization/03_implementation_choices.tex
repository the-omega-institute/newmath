\section{Implementation choices}

The first sprint should implement a finite evidence-erased signature layer.  That is, use
$$
\texttt{SigRel Ask bundle history answerHistory}
$$
without an explicit evidence ledger.  Once the finite kernel is checked, add evidence aggregation as a refinement.  This avoids blocking signature determinacy on unresolved ledger aggregation details.

Probe bundles should be internally generated by constructors, not exposed publicly as host lists.

Probe behavior should be represented as a relation:
$$
\texttt{Ask : ProbeName -> BHist -> BMark -> Evidence -> Prop}.
$$
A deterministic ask policy should be a structure containing the theorem that two ask events for the same probe and history emit internally same marks.

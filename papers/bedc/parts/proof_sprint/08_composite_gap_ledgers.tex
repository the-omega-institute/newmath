\section{Composite gap ledgers}

\begin{definition}[Composite gap]
Suppose a first visible interface compresses sources into intermediate tokens and a second visible interface compresses intermediate tokens into final tokens.  The composite gap of a final token contains a source exactly when there exists an intermediate token connecting the two gap memberships.  Symbolically:
\[
\mathsf{CompGap}_{D\circ C}(z,x)
\quad\text{iff}\quad
\exists y,\ \mathsf{Gap}_D(z,y)\ \text{and}\ \mathsf{Gap}_C(y,x).
\]
This is a meta-kernel ledger law, not an object-language primitive.
\end{definition}

\begin{theorem}[Composite gap coverage]
If the first and second gap interfaces satisfy coverage, then the composite gap satisfies coverage.
\end{theorem}
\begin{proof}
Given a source, use coverage of the first interface to obtain an intermediate token.  Then use coverage of the second interface to obtain a final token.  The intermediate token witnesses composite membership.
\end{proof}

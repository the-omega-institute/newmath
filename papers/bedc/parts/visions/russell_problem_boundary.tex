\chapter{Russell Problem Boundary}
\label{ch:visions-russell-problem-boundary}

BEDC inherits a problem from Russell, not Russell's logicist system. The inherited problem is the demand that mature mathematics be read from a small, explicit discipline rather than accepted as a stock of primitive objects. BEDC does not import the axioms, ramified type theory, reducibility principle, or class ontology of \emph{The Principles of Mathematics} or \emph{Principia Mathematica}. It imports a reconstruction pressure: every apparently familiar mathematical object must be unfolded into a licensed pattern of relations, descriptions, contexts, strata, and witnesses.

\section{The inherited problem}
\label{sec:russell-problem-inherited}

Russell's foundational project can be read as a discipline for preventing mature mathematical grammar from smuggling in mature mathematical ontology. A phrase that looks like it denotes an object must be analysed before it is admitted. A class phrase must not automatically become a class object. A relation must not be replaced by a unary property when the relation is doing the mathematical work. A term at one type level must not be treated as a term at another type level merely because the prose makes the passage sound harmless.

BEDC accepts this discipline in a kernel-facing form. The point is not to reproduce Russell's system, but to keep the following five restrictions visible at every upward interface:

\begin{enumerate}
  \item relation first: the behaviour that licenses a name is relational before it is object-like;
  \item description expansion: a definite description is not a primitive denotation, but an analysed certificate obligation;
  \item contextual class reading: a class-like expression is read through the context that classifies it, not as an automatic object;
  \item stratification: a passage between levels requires an explicit bridge and does not collapse the levels;
  \item witnessed descent: no downward use of an upward name is allowed without a certificate.
\end{enumerate}

\section{Kernel reading}
\label{sec:russell-problem-kernel-reading}

The finite kernel already supplies the place where this discipline lives. An asking event has the form
$$
\Ask(\pi,h,m,\delta),
$$
where $\pi$ is the probe name, $h$ is the history, $m$ is the emitted mark, and $\delta$ is the evidence token. Read structurally, this gives five roles:

\begin{center}
\begin{tabular}{lll}
\textbf{BEDC role} & \textbf{Name-certificate role} & \textbf{Russell discipline} \\
\hline
probe or question & source entry & description expansion \\
history & source material & contextual class reading \\
observation relation & pattern and classifier & relation first \\
result mark & stability-consumable result & stratification \\
evidence token & ledger policy & witnessed descent \\
\end{tabular}
\end{center}

Thus Russell's discipline is not an external philosophical decoration. It is a reading rule for the same fivefold structure that the BEDC kernel already uses to keep an observation from becoming an object without a certificate.

\section{The upward-name reading}
\label{sec:russell-problem-xup-reading}

For a mature mathematical expression $X$, the expression $X^{\uparrow}$ does not mean that BEDC has found the host object $X$ inside the kernel. It means that BEDC has licensed an interface whose source specification, pattern specification, classifier specification, stability certificate, and ledger policy allow the traditional grammar of $X$ to be used without making $X$ primitive.

For example, $\mathsf{Nat}^{\uparrow}$ is not the host natural-number type. It is a certificate-licensed reading of unary repetition, successor behaviour, induction, and recursion as stable public operations over histories. Likewise $\mathsf{ContinuousMap}^{\uparrow}$ is not a host-level function equipped with an imported modulus. It is a relation-plus-ledger interface whose continuity claims are consumed only through the certificate that licenses them.

This is the BEDC form of Russell's analysis of descriptions. The apparent object is replaced by the statement that a certificate exists with the right source, pattern, classifier, stability, and ledger obligations. The public name is usable only where those obligations are in scope.

\section{Boundary}
\label{sec:russell-problem-boundary}

The Russell problem boundary rejects five shortcuts.

\begin{enumerate}
  \item A class expression may not be promoted to an object merely because later prose wants to quantify over it.
  \item A description may not be used as a denoting term without existence, uniqueness or stability, and ledger obligations appropriate to the interface.
  \item A host quotient, host choice principle, proof-irrelevance principle, or proposition-extensionality principle may not enter the public surface as if it were a BEDC derivation.
  \item A level descent may not be justified by a reducibility-style slogan; it must be represented by a certificate that names the bridge.
  \item A bridge readback may not be treated as kernel identity; it is a consumed certificate, not a collapse of the source and target strata.
\end{enumerate}

These rejections are not additions to the kernel. They are consequences of taking $\NameCert$ as the universal admission rule for mature names.

\section{Analysis spine}
\label{sec:russell-problem-analysis-spine}

The reconstruction of analysis follows the same spine. First, unary repetition is licensed as $\mathsf{Nat}^{\uparrow}$. Relation calculus and order are then licensed as classifier-stable behaviours over the arithmetic layer. Series, segments, cuts, rationals, regular rational sequences, and real numbers are admitted only as upward names whose ledger records the hidden convergence or approximation work. Limits, continuous maps, derivatives, integrals, measure, and functional analysis are then read as later certificates over that real-analysis base.

In compressed form, the intended path is
$$
\mathsf{Nat}^{\uparrow}
\to
\mathsf{RelationCalculus}^{\uparrow}
\to
\mathsf{Order}^{\uparrow}
\to
\mathsf{Series}^{\uparrow}
\to
\mathsf{Segment}^{\uparrow},\mathsf{Cut}^{\uparrow}
\to
\mathsf{Rat}^{\uparrow}
\to
\mathsf{RegSeqRat}^{\uparrow}
\to
\mathsf{Real}^{\uparrow}
\to
\mathsf{Limit}^{\uparrow}
\to
\mathsf{ContinuousMap}^{\uparrow}
\to
\mathsf{Derivative}^{\uparrow}
\to
\mathsf{Integral}^{\uparrow}
\to
\mathsf{Measure}^{\uparrow}
\to
\mathsf{FunctionalAnalysis}^{\uparrow}.
$$

This path is the BEDC version of proving analysis from minimal disciplined beginnings. The phrase ``from minimal self-evident axioms'' is therefore not read as permission to import axiom lists. It is read as a demand that every later object be reconstructed through the smallest available kernel roles and then admitted only through a visible certificate.

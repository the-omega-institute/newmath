\chapter{v1.5.5 Manifest: Base Reflection Check Target}
\label{ch:v155-manifest}

This v1.5.5 increment is a checker-facing hardening layer. It does not add new derived objects and it does not broaden the finite kernel. Its purpose is to replace the remaining prose form of exact concrete Globalize by explicit theorem-shape objects that a proof assistant can target.

\section{Scope}
This increment adds the following active refinements:
\begin{enumerate}
  \item a checker-facing predicate \(\GeneratedSameSig(\Pi,h,k)\) for generated signature sameness of histories;
  \item a structure \(\ExactGlobalizeBase(\Pi,D)\) collecting coverage, soundness, completeness, and the classification equivalence for the base package relation;
  \item a named proof spine for \(\PBase\)-inversion, token replacement, and base package reflection;
  \item an explicit policy that \(\psame_{\Pi}^{\mathrm{eq}}\)-level export remains blocked until \(\ClosureReflect(\Pi)\) is checked;
  \item a Lean-style scaffold file whose role is theorem-shape documentation, not a verified proof result.
\end{enumerate}

\section{Non-goals}
This increment does not claim:
\begin{enumerate}
  \item a checked Lean, Coq, Agda, or other proof-assistant build;
  \item a proof of \(\ClosureReflect(\Pi)\) in a checker;
  \item a public activation of \(\AddUp\) beyond the conditional license already recorded in v1.5.4;
  \item a new construction of \(\RealUp\), \(\CircleUp\), \(\FoldUp\), or any completion-layer object;
  \item a rewrite of the frozen core or of the v1.5.4 base/closure boundary.
\end{enumerate}

\section{Development rule}
The rule of v1.5.5 is:
\[
  \text{turn exactness into a typed predicate before extending the theory.}
\]
Thus the current release hardens the proof target, not the object layer. A later release may be called a checked proof report only after it records the exact checker, command, theorem list, and list of admitted or postulated statements.

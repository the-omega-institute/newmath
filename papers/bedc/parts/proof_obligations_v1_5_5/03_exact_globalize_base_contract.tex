\chapter{Exact Globalize Base Contract}
\label{ch:v155-exact-globalize-base-contract}

This chapter states the v1.5.5 replacement for the prose theorem exported in v1.5.4. The content is intentionally narrow: it is the exactness of the concrete signature Globalize instance at the base package relation.

\begin{theorem}[v1.5.5 exact Globalize base target]
\label{thm:v155-exact-globalize-base-target}
Assume:
\begin{enumerate}
  \item \(\DomainPolicy(D)\);
  \item \(\AskPol(\Pi,D)\);
  \item signature generation and token introduction for generated signatures;
  \item \(\TokUnique(\Pi)\), or a checked canonical-constructor theorem implying it;
  \item the active base package relation \(\psame_{\Pi}^{\mathrm{base}}\);
  \item the policy-supported gap relation \(\InGapSig(\Pi,D,p,h)\);
  \item the base reflection theorem \texttt{PackageReflection.base}.
\end{enumerate}
Then
\[
  \ExactGlobalizeBase(\Pi,D).
\]
\end{theorem}

\begin{proof}
Coverage is the policy-supported gap coverage theorem from the v1.5.3/v1.5.4 spine, with internal domain transport supplied by \(\DomainPolicy(D)\). Base soundness follows from generated signature sameness plus the single constructor of \(\psame_{\Pi}^{\mathrm{base}}\). Base completeness follows by applying base package reflection to the visible packages and then unpacking \(\GeneratedSameSig\). The local classification equivalence is Proposition~\ref{prop:v155-iff-derived}.
\end{proof}

\begin{corollary}[No closure export]
\label{cor:v155-no-closure-export}
Theorem~\ref{thm:v155-exact-globalize-base-target} exports exactness only at \(\psame_{\Pi}^{\mathrm{base}}\). It does not export exactness at \(\psame_{\Pi}^{\mathrm{eq}}\).
\end{corollary}

\begin{proof}
The statement concludes \(\ExactGlobalizeBase(\Pi,D)\), whose relation field is \(\psame_{\Pi}^{\mathrm{base}}\) by Definition~\ref{def:v155-exact-globalize-base}. The optional equivalence closure remains governed by \(\ClosureReflect(\Pi)\), which is not a checked theorem in this release.
\end{proof}

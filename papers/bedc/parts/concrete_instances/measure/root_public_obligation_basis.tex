\begin{definition}[Measure root public obligation basis]
\label{def:measure-root-public-obligation-basis}
The $\MeasureUp$ root public obligation basis is the counted ten-field basis
that exposes the constructive root measure certificate before any analytic
completion or bridge layer is admitted:
\begin{enumerate}
  \item traditional target: a constructive measure root with visible
  measurable-event rows, countable-family rows, and $\RealUp$ endpoint
  values;
  \item BEDC source: $\mathsf{BHist}$, componentwise $\hsame$,
  $\NameCert_{\RealUp}$, $\StreamNameUp$, and the additive $\AbGroupUp$ real
  rows used for countable endpoint sums;
  \item carrier: the root predicate $\mathsf{MeasCode}$ on
  $\mathsf{BHist}$;
  \item classifier: the public measure classifier $\sim_M$, with endpoint
  motion read through $\NameCert_{\RealUp}$ and $\sim_R$;
  \item stability rows: carrier and classifier transport, visible event-row
  transport, endpoint-row transport, and countable-family transport along
  $\sim_M$ and componentwise $\hsame$;
  \item ledger: hidden measurable-event witnesses, endpoint-value witnesses,
  disjointness witnesses, relative-difference witnesses, and countable-sum
  witnesses attached to the displayed rows they justify;
  \item core theorem list:
  \autoref{thm:measure-root-public-obligation-basis-carrier},
  \autoref{thm:measure-root-public-obligation-basis-additivity},
  \autoref{thm:measure-root-public-obligation-basis-consumer}, and
  \autoref{thm:measure-root-public-obligation-basis-exactness};
  \item exactness: visible-row exhaustion for the carrier face, the additive
  face, and the consumer face through the rows of
  \autoref{def:measure-event-row-coverage};
  \item standard bridge: no $\StdBridge$ is asserted or imported by this
  basis;
  \item status: paper proof target under
  $\mathsf{BEDC.Derived.MeasureUp}$.
\end{enumerate}
This basis is read from
\autoref{def:measure-root-seed-obligation-front},
\autoref{def:measure-root-threshold-obligation-package}, and
\autoref{def:measure-root-countable-premeasure-front}. It is the public
obligation surface available to downstream analysis chapters.
\end{definition}

\begin{theorem}[Measure root public obligation basis carrier]
\label{thm:measure-root-public-obligation-basis-carrier}
The basis of \autoref{def:measure-root-public-obligation-basis} supplies the
carrier and classifier transport needed by the public $\MeasureUp$ root. If
$h,k:\mathsf{BHist}$ satisfy $\mathsf{MeasCode}(h)$ and $h\sim_M k$, then the
basis transports $\mathsf{MeasCode}(k)$, every visible event row, every
endpoint row, every countable-family row, and every ledger witness attached
to the carried measure code.
\end{theorem}
\begin{proof}
Start with the carrier and classifier fields of
\autoref{def:measure-root-public-obligation-basis}. They are the
$\mathsf{MeasCode}$ and $\sim_M$ rows first exposed in
\autoref{def:measure-root-seed-obligation-front}.

Apply \autoref{thm:measure-root-seed-carrier-classifier-obligation}. It
transports the carried measure code, visible event rows, ledger rows, and
$\RealUp$ endpoint comparisons along the supplied $\sim_M$ comparison.

Next apply \autoref{thm:measure-root-threshold-carrier-classifier-obligation}.
The threshold reading keeps the same transported rows and adds the public
countable-family and endpoint-readback boundary used by the root package.

The ledger field of the basis attaches the hidden witnesses only to displayed
$\mathsf{BHist}$ rows. Therefore the transported rows exhaust the public
carrier and classifier obligation without adding host set equality or
quotient event equality.
\end{proof}

\begin{theorem}[Measure root public obligation basis additivity]
\label{thm:measure-root-public-obligation-basis-additivity}
The basis of \autoref{def:measure-root-public-obligation-basis} supplies the
public additive face of the $\MeasureUp$ root: empty-zero, finite readback,
relative-difference additivity, countable disjoint-additivity, and
countable-sum transport through the additive $\AbGroupUp$ real row.
\end{theorem}
\begin{proof}
Project the additive fields of
\autoref{def:measure-root-public-obligation-basis}. They name the covered
empty event, finite disjoint unions, relative differences, countable
disjoint families, countable unions, and countable-sum endpoints in the same
visible $\mathsf{BHist}$ ledger.

Apply \autoref{thm:measure-root-seed-sigma-additive-obligation}. It supplies
the root comparison between the value of a covered countable disjoint union
and the corresponding countable endpoint sum.

Apply \autoref{thm:measure-root-threshold-sigma-additive-obligation}. It
keeps the empty-zero row, finite readback row, relative-difference row,
head-tail row, and countable-sum classifier transport visible at the public
threshold.

Finally apply
\autoref{thm:measure-root-countable-premeasure-additivity-obligation}. This
countable-premeasure reading identifies finite readback and
relative-difference additivity as projections of the same countable additive
surface, with endpoint movement through $\NameCert_{\RealUp}$ and the
additive $\AbGroupUp$ real row.
\end{proof}

\begin{theorem}[Measure root public obligation basis consumer]
\label{thm:measure-root-public-obligation-basis-consumer}
The basis of \autoref{def:measure-root-public-obligation-basis} exports
exactly the $\IntegralUp$, $\ProbSpaceUp$, and $\FunctionalAnalysisUp$ rows
available from the public $\MeasureUp$ root. No other analysis row is part of
this basis.
\end{theorem}
\begin{proof}
Begin with the consumer field of
\autoref{def:measure-root-public-obligation-basis}. It lists only the three
outgoing faces $\IntegralUp$, $\ProbSpaceUp$, and $\FunctionalAnalysisUp$.

Apply \autoref{thm:measure-root-threshold-analysis-consumer-obligation}. It
reads those three faces from the root threshold package and identifies the
covered event rows, endpoint classifier rows, countable-sum rows,
empty-zero, finite readback, relative-difference, monotone inclusion,
nonnegative endpoint, and total-event normalization rows available to them.

Next apply \autoref{thm:measure-root-countable-premeasure-consumer-threshold}.
It restricts the same three consumers to the countable-premeasure front and
keeps the outgoing rows attached to covered measurable-event and endpoint
rows.

The exactness clause of the basis has no fourth consumer slot and no
analytic completion slot. Hence the exported consumer boundary is exactly
the stated triad and no larger analysis interface is claimed.
\end{proof}

\begin{theorem}[Measure root public obligation basis exactness]
\label{thm:measure-root-public-obligation-basis-exactness}
The carrier face, additive face, and consumer face of
\autoref{def:measure-root-public-obligation-basis} form the deps-ready
$\MeasureUp$ root package. The package asserts no host set equality, no
quotient event equality, no analytic completion object, and no standard
bridge.
\end{theorem}
\begin{proof}
First read the carrier face through
\autoref{thm:measure-root-public-obligation-basis-carrier}. It supplies the
public $\mathsf{MeasCode}$ carrier, $\sim_M$ classifier transport, visible
event rows, endpoint rows, countable-family rows, and attached ledger
witnesses.

Next read the additive face through
\autoref{thm:measure-root-public-obligation-basis-additivity}. It supplies
the empty-zero, finite readback, relative-difference, countable
disjoint-additivity, and countable-sum transport rows over the same visible
ledger.

Finally read the consumer face through
\autoref{thm:measure-root-public-obligation-basis-consumer}. It exports only
the $\IntegralUp$, $\ProbSpaceUp$, and $\FunctionalAnalysisUp$ projections
from that public root package.

The exactness field of
\autoref{def:measure-root-public-obligation-basis} says that every consumed
hidden witness is attached to a displayed row covered by
\autoref{def:measure-event-row-coverage}. Since none of the three faces
contains a host-set equality field, quotient-event equality field, analytic
completion object, or $\StdBridge$ field, the resulting package is precisely
the deps-ready public obligation basis.
\end{proof}

\begin{definition}[Measure root NameCert front]
\label{def:measure-root-namecert-front}
The $\MeasureUp$ root $\NameCert$ front is the ten-field public package that
downstream consumers read before forming any analytic completion or
consumer-local quotient:
\begin{enumerate}
  \item event carrier field: visible $\mathsf{BHist}$ event rows and the
  $\mathsf{MeasCode}$ predicate certifying that each event row is carried by
  the root measure surface;
  \item event classifier field: componentwise $\hsame$ transport on
  $\mathsf{BHist}$ event presentations, read externally as the public
  $\sim_M$ classifier;
  \item NameCert carrier field: the root $\NameCert_{\MeasureUp}$ carrier row
  that packages the measurable-event witnesses without host set equality;
  \item NameCert classifier field: the root $\NameCert_{\MeasureUp}$
  classifier row identifying exactly the displayed event and endpoint
  classifier motion;
  \item StreamName family field: countable families named through
  $\StreamNameUp$, including displayed component rows, disjointness rows, and
  union rows;
  \item Real endpoint field: value rows, finite-sum rows, relative-difference
  rows, continuity-from-below rows, and countable-sum rows compared only by
  $\NameCert_{\RealUp}$ endpoint classifiers;
  \item stability field: finite additivity, relative-difference transport,
  continuity from below, and countable-additivity stability over the same
  visible rows;
  \item ledger field: the visible root ledger attaching every hidden witness
  to a displayed event row, endpoint row, countable-family row, or consumer
  row;
  \item downstream consumer field: the three outgoing faces named
  $\IntegralUp$, $\ProbSpaceUp$, and $\FunctionalAnalysisUp$;
  \item exclusion field: no host set equality, quotient event equality,
  analytic completion, or open-cover compactness row is part of the front.
\end{enumerate}
\end{definition}

\begin{theorem}[Measure root NameCert front carrier]
\label{thm:measure-root-namecert-front-carrier}
The carrier face of \autoref{def:measure-root-namecert-front} is exactly the
$\mathsf{BHist}$ event-row carrier, $\mathsf{MeasCode}$ predicate,
$\sim_M$ classifier, root $\NameCert_{\MeasureUp}$ carrier-classifier row,
and countable-classifier carrier scope supplied by
\autoref{def:measureup-carrier-obligation-inventory},
\autoref{def:measure-classifier-specification},
\autoref{thm:measure-root-namecert-carrier-classifier}, and
\autoref{thm:measure-root-countable-classifier-carrier-scope}.
\end{theorem}
\begin{proof}
Read the first four fields of
\autoref{def:measure-root-namecert-front}. They name the visible
$\mathsf{BHist}$ event rows, the $\mathsf{MeasCode}$ carrier predicate,
componentwise $\hsame$ transport, the public $\sim_M$ classifier, and the
root $\NameCert_{\MeasureUp}$ carrier-classifier row.

Apply \autoref{def:measureup-carrier-obligation-inventory}. Its inventory
lists the carried event rows, measurable witnesses, classifier transport,
endpoint rows, and ledger attachments that may appear on the root carrier
surface. Apply \autoref{def:measure-classifier-specification} to restrict
the classifier to event-witness motion and $\RealUp$ endpoint comparison.

Next apply \autoref{thm:measure-root-namecert-carrier-classifier}. It
identifies the root $\NameCert_{\MeasureUp}$ carrier and classifier with
exactly those displayed rows. Apply
\autoref{thm:measure-root-countable-classifier-carrier-scope} to include the
countable-family carrier side without adding a second carrier.

The exclusions field of the front is therefore inherited from the cited
carrier rows: none of them supplies host set equality, quotient event
equality, analytic completion, or open-cover compactness. Hence the front
carrier is precisely the carrier face stated above.
\end{proof}

\begin{theorem}[Measure root NameCert front classifier]
\label{thm:measure-root-namecert-front-classifier}
The classifier face of \autoref{def:measure-root-namecert-front} is stable
under the root $\NameCert_{\MeasureUp}$ exactness rows, the countable
classifier presentation, and the countable-additivity event-value closure.
Equivalently, all classifier motion in the front is obtained from
\autoref{thm:measure-root-namecert-stability-exactness},
\autoref{thm:measure-root-countable-classifier-stability}, and
\autoref{thm:measure-root-unblock-countable-additivity-event-value-closure}.
\end{theorem}
\begin{proof}
Project the event classifier, Real endpoint, stability, and ledger fields of
\autoref{def:measure-root-namecert-front}. They allow event motion only
through componentwise $\hsame$ and endpoint motion only through
$\NameCert_{\RealUp}$ classifier rows.

Apply \autoref{thm:measure-root-namecert-stability-exactness}. Its exactness
clauses attach every root $\NameCert_{\MeasureUp}$ stability witness to the
displayed event, endpoint, and ledger rows of the root surface.

For countable families, apply
\autoref{thm:measure-root-countable-classifier-stability}. It transports
finite additivity, relative differences, continuity from below, and
countable-sum comparison through the countable-classifier presentation.

Finally use
\autoref{thm:measure-root-unblock-countable-additivity-event-value-closure}.
That theorem identifies the countable-additivity event-value face with
displayed $\mathsf{BHist}$ families, $\StreamNameUp$ observation rows,
disjoint-union readback, and $\RealUp$ endpoint-sum comparison. Thus the
front classifier contains only stable visible rows and no host equality,
quotient event equality, analytic completion, or open-cover compactness.
\end{proof}

\begin{theorem}[Measure root NameCert front ledger]
\label{thm:measure-root-namecert-front-ledger}
The downstream ledger of \autoref{def:measure-root-namecert-front} exports to
$\IntegralUp$, $\ProbSpaceUp$, and $\FunctionalAnalysisUp$ exactly the
carrier rows, classifier rows, stability rows, event-value rows, and minimum
consumer obligations supplied by
\autoref{thm:measure-root-countable-classifier-downstream-interface},
\autoref{thm:measure-root-unblock-analysis-consumer-event-value-closure},
and \autoref{thm:measure-root-minimum-downstream-consumer-obligation}.
\end{theorem}
\begin{proof}
Read the ledger and downstream consumer fields of
\autoref{def:measure-root-namecert-front}. They name a visible root ledger
and exactly three consumers: $\IntegralUp$, $\ProbSpaceUp$, and
$\FunctionalAnalysisUp$.

Apply \autoref{thm:measure-root-countable-classifier-downstream-interface}.
It exports the carrier, classifier, and stability rows of the
countable-classifier presentation to the three named downstream chapters and
blocks any additional host equality, quotient equality, or completion row.

Next apply
\autoref{thm:measure-root-unblock-analysis-consumer-event-value-closure}.
It identifies the event-value rows consumed by the same three analysis
chapters: displayed event rows, endpoint rows, countable-sum rows,
finite-readback rows, normalization rows, monotone rows,
relative-difference rows, and visible ledger rows.

Finally apply
\autoref{thm:measure-root-minimum-downstream-consumer-obligation}. It
restricts the consumer package to carrier/classifier rows,
countable-additivity rows, nonnegative endpoint rows, and root ledger rows.
Intersecting these three projections gives the ledger exported by the front,
with no host set equality, quotient event equality, analytic completion, or
open-cover compactness.
\end{proof}

\begin{theorem}[Measure root NameCert front obligation status]
\label{thm:measure-root-namecert-front-obligation-status}
\autoref{def:measure-root-namecert-front}, together with
\autoref{thm:measure-root-namecert-front-carrier},
\autoref{thm:measure-root-namecert-front-classifier}, and
\autoref{thm:measure-root-namecert-front-ledger}, enumerates the
$\MeasureUp$ root obligation set: the carrier and classifier rows,
countable-family rows through $\StreamNameUp$, $\RealUp$ endpoint rows,
stability and exactness rows, ledger rows, and the
$\IntegralUp$/$\ProbSpaceUp$/$\FunctionalAnalysisUp$ consumer faces.  The
same front excludes host set equality, quotient event equality, analytic
completion, and standard bridges.
\end{theorem}
\begin{proof}
Project the ten fields of \autoref{def:measure-root-namecert-front}.  The
first four fields name the event carrier, event classifier, root
$\NameCert_{\MeasureUp}$ carrier, and root $\NameCert_{\MeasureUp}$
classifier.  By \autoref{thm:measure-root-namecert-front-carrier}, these are
exactly the carrier-classifier rows of the root surface.

The fifth through seventh fields name the $\StreamNameUp$ countable-family
rows, the $\RealUp$ endpoint rows, and the stability rows.  Apply
\autoref{thm:measure-root-namecert-front-classifier}; it identifies this
motion with the exactness rows, countable-classifier presentation, and
countable-additivity event-value closure.

The eighth and ninth fields name the visible ledger and the three outgoing
consumer faces.  By \autoref{thm:measure-root-namecert-front-ledger}, these
fields export exactly the ledger and consumer rows read by $\IntegralUp$,
$\ProbSpaceUp$, and $\FunctionalAnalysisUp$.

The tenth field is the exclusion field.  It rules out host set equality,
quotient event equality, analytic completion, and standard bridges, so the
front enumerates precisely the obligation set stated above.
\end{proof}

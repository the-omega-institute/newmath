\chapter{A Concrete Naming Certificate for $\BareObjectRefusalUp$}
\label{ch:concrete-instances-bareobjectrefusal-namecert}
\origin{ai}

The $\BareObjectRefusalUp$ packet realizes the bare-object refusal direction of \autoref{ch:visions-philosophy-object-knowledge-namecert} as finite NameCert data. It sits beside the licensed object-knowledge packet of \autoref{ch:concrete-instances-objectknowledgecertificate-namecert}, the description-expansion boundary of \autoref{ch:concrete-instances-definitedescriptionboundary-namecert}, the apophatic refusal boundary of \autoref{ch:concrete-instances-negativecertificateboundary-namecert}, and the classifier boundary of \autoref{ch:concrete-instances-classifierboundary-namecert}. Those siblings license object names, regulate descriptions, record apophatic refusal rows, and expose classifier-boundary rows; this packet records the failed five-field object license itself.

\begin{definition}[Bare-object refusal carrier]
\label{def:bare-object-refusal-carrier}
A $\BareObjectRefusalUp$ carrier is a finite $\BHist$ packet
$$
B=(O,M,R,W,L,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $O$ is the requested object-name row whose proposed use is tested against the object-knowledge certificate discipline.
\item $M$ is the missing-field row, recording which of source, pattern, classifier, stability, and ledger is absent or refused for the requested object license.
\item $R$ is the refusal row that prevents the requested name from being exported as an object-layer license while $M$ is nonempty.
\item $W$ is the witness-audit row recording that no auditable witness row has been supplied for the requested object read.
\item $L$ is the anti-primitive ledger row recording that the refusal cannot be bypassed by treating the name as a bare object primitive.
\item $H$ records componentwise $\hsame$ transport for the requested name, missing-field row, refusal row, witness audit, ledger row, continuation row, package row, and local naming row.
\item $C$ records the displayed $\Cont$ routes by which consumers read the missing fields, refusal, witness audit, and ledger nonescape rows.
\item $P$ is the $\Pkg$ provenance over $\ObjectKnowledgeCertificateUp$, $\DefiniteDescriptionBoundaryUp$, $\NegativeCertificateBoundaryUp$, $\ClassifierBoundaryUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.
\item $N$ is the local $\NameCert_{\BareObjectRefusalUp}$ row.
\end{enumerate}
The carrier has no coordinate for a licensed object name, witness-free knowledge export, primitive object token, host ontology, or object-layer license detached from the displayed five-field certificate check.
\end{definition}

\begin{theorem}[Bare-object refusal NameCert obligations]
\label{thm:bare-object-refusal-namecert-obligations}
For the carrier of \autoref{def:bare-object-refusal-carrier}, the local $\NameCert_{\BareObjectRefusalUp}$ surface consists of five obligation rows:
\begin{enumerate}
\item source refusal records the requested object-name row $O$ and the absence of an accepted source field in $M$;
\item pattern refusal records that no displayed pattern row licenses $O$ as an object read;
\item classifier refusal records that no accepted classifier row turns $O$ into an object-identity surface;
\item stability refusal records that no componentwise transport may convert the missing fields into a stable object license;
\item ledger refusal records the anti-primitive row $L$, witness-audit row $W$, refusal row $R$, routes $C$, provenance $P$, and local naming row $N$ as the only public export.
\end{enumerate}
Consequently a public consumer of an accepted $\BareObjectRefusalUp$ packet receives a refusal certificate for the failed object license, not a weak object license.
\end{theorem}
\begin{proof}
Carrier admission is projection from the displayed tuple $B=(O,M,R,W,L,H,C,P,N)$. The missing-field row $M$ is the only place where the five object-knowledge fields are named, and it records absence or refusal rather than a licensed object packet.

The first four obligation rows inspect the failed source, pattern, classifier, and stability fields. They show that the requested name has not met the object-knowledge discipline of \autoref{ch:concrete-instances-objectknowledgecertificate-namecert}; description expansion, negative-certificate refusal, and classifier-boundary rows remain sibling surfaces rather than substitutes for the missing object license.

The ledger obligation then closes the export. The witness-audit row $W$, refusal row $R$, anti-primitive ledger $L$, transports $H$, routes $C$, provenance $P$, and local naming row $N$ expose only the failed-license packet. Since no carrier coordinate contains a licensed object name or witness row, the consumer cannot repack the refusal as object knowledge.
\end{proof}

\begin{theorem}[Bare-object refusal object-license nonescape]
\label{thm:bare-object-refusal-object-license-nonescape}
Every accepted $\BareObjectRefusalUp$ packet exports only the requested-name, missing-field, refusal, witness-audit, anti-primitive ledger, transport, route, provenance, and local naming rows
$$
O,M,R,W,L,H,C,P,N.
$$
It cannot export an object-layer license unless the read factors through an existing $\ObjectKnowledgeCertificateUp$ packet with displayed source, pattern, classifier, stability, ledger, object-name, and witness rows.
\end{theorem}
\begin{proof}
Start with the NameCert obligations above. They make the missing source, pattern, classifier, stability, and ledger fields part of the refusal surface, so the local packet records a failed object license rather than a partial one.

Use the object-knowledge sibling. \autoref{ch:concrete-instances-objectknowledgecertificate-namecert} exposes licensed object knowledge only through its object-name, witness, five-field certificate, audit-route, transport, and local naming rows. None of those licensed rows is present in the $\BareObjectRefusalUp$ carrier except as refused or missing data.

Finally apply the anti-primitive ledger row. The ledger $L$ records that a consumer cannot treat $O$ as a primitive object token after the five-field certificate check has failed. Thus any accepted object-layer license must come from the sibling $\ObjectKnowledgeCertificateUp$ route, while this packet exports only the refusal rows listed above.
\end{proof}

\falsifiablePrediction{Any accepted bare-object-refusal packet with an empty missing-field row cannot export an object-layer license without factoring through an existing ObjectKnowledgeCertificateUp packet.}

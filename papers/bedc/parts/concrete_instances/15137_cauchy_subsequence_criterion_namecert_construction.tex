\chapter{A Concrete Naming Certificate for $\CauchySubsequenceCriterionUp$}
\label{ch:concrete-instances-cauchy-subsequence-criterion-namecert}
\origin{human}

Following Bishop and Bridges constructive analysis, the $\CauchySubsequenceCriterionUp$ packet records the finite route by which a displayed source sequence is accepted as having a Cauchy subsequence before any completion-facing consumer reads it. It sits between the finite stream windows of \autoref{ch:concrete-instances-streamname-namecert}, the regular rational readback of \autoref{ch:concrete-instances-regseqrat-namecert}, the inspected fast-subsequence route of \autoref{ch:concrete-instances-fastcauchysubsequence-namecert}, the monotone-cofinal regular-subsequence surface of \autoref{ch:concrete-instances-regular-cauchy-subsequence-namecert}, and the terminal real-name boundary of \autoref{ch:concrete-instances-real-namecert}. The packet is a finite criterion surface for downstream Bolzano--Weierstrass and completion consumers; it does not select an infinite subsequence by choice, identify quotient streams, compare host real equalities, or assert completeness of $\RealUp$.

\begin{definition}[CauchySubsequenceCriterion carrier]
\label{def:cauchy-subsequence-criterion-carrier}
A $\CauchySubsequenceCriterionUp$ carrier is a finite $\BHist$ packet
$$
K=(S,T,W,Q,D,F,R,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the source sequence row whose finite $\StreamNameUp$ windows are inspected before any subsequence is accepted;
\item $T$ is the monotone subsequence schedule, recorded as a finite unary reindexing ledger with visible order and later-index comparisons on inspected indices;
\item $W$ records the inspected $\StreamNameUp$ windows before and after applying the displayed schedule $T$;
\item $Q$ is the $\RegSeqRatUp$ readback row obtained from the selected windows and no independent rational stream;
\item $D$ is the dyadic tolerance ledger tying the selected windows to the requested Cauchy radii and pointwise comparison rows;
\item $F$ is the $\FastCauchySubsequenceUp$ witness that the selected windows satisfy the displayed fast-Cauchy bounds;
\item $R$ is the $\RegularCauchySubsequenceUp$ handoff row that repacks the same schedule, windows, dyadic tolerance ledger, and regular-rational readback for downstream consumers;
\item $E$ is the $\RealUp$ seal handoff reached only after $Q,F,R$ and the tolerance ledger have been displayed;
\item $H$ records componentwise $\hsame$ transport for the source row, schedule, window rows, regular-rational readback, dyadic ledger, fast witness, regular-subsequence handoff, real seal, and support rows;
\item $C$ records displayed $\Cont$ replay from source inspection through schedule selection, finite windows, tolerance ledger, fast-Cauchy witness, regular-rational readback, regular-subsequence handoff, and real-seal delivery;
\item $P$ is the $\Pkg$ provenance over $\CauchySubsequenceCriterionUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\FastCauchySubsequenceUp$, $\RegularCauchySubsequenceUp$, $\CauchySequenceBoundedUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$;
\item $N$ is the local $\NameCert_{\CauchySubsequenceCriterionUp}$ row.
\end{enumerate}
The carrier stores no selected infinite subsequence object, arbitrary choice functional, quotient stream, host real equality, independent real carrier, or completeness principle for $\RealUp$.
\end{definition}

\begin{theorem}[CauchySubsequenceCriterion NameCert obligations]
\label{thm:cauchy-subsequence-criterion-namecert-obligations}
For an accepted $\CauchySubsequenceCriterionUp$ carrier $K=(S,T,W,Q,D,F,R,E,H,C,P,N)$, the local $\NameCert_{\CauchySubsequenceCriterionUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the source row, monotone schedule, inspected window rows, regular-rational readback, dyadic tolerance ledger, fast-Cauchy witness, regular-subsequence handoff, real seal, transport, replay, provenance, and local naming row;
\item schedule monotonicity for $T$, requiring each displayed later index to be read from the finite unary order ledger rather than from a host-level selector;
\item window transport, requiring every selected $\StreamNameUp$ window in $W$ to be transported from an inspected source window along $T$;
\item fast-Cauchy handoff, requiring the witness $F$ to consume exactly the transported windows and dyadic tolerance rows displayed by $W$ and $D$;
\item regular-rational readback, requiring $Q$ to be obtained from the same selected windows and not from an independent rational sequence;
\item Real seal non-escape, requiring $E$ to be reached only through the displayed fast witness, regular-rational readback, and $\RegularCauchySubsequenceUp$ handoff;
\item ledger exactness for $D,H,C,P,N$, requiring dyadic tolerance, transport, replay, package provenance, and local naming to mention only the rows of the carrier.
\end{enumerate}
No obligation row exports a selected infinite subsequence, arbitrary choice functional, quotient stream, host real equality, independent real carrier, or completeness principle for $\RealUp$.
\end{theorem}
\begin{proof}
Project the carrier. The tuple displays exactly the source row, monotone schedule, inspected windows, regular-rational readback, dyadic tolerance ledger, fast witness, regular-subsequence handoff, real seal, componentwise transport, continuation replay, package provenance, and local naming row.

Read the schedule and window rows first. The row $T$ supplies only finite monotone later-index comparisons, and $W$ records only the source and selected windows transported along that ledger. Thus the criterion has no coordinate from which an arbitrary subsequence selector could be read.

Use the analytic handoff rows next. The row $D$ records the dyadic tolerances, $F$ verifies the fast-Cauchy subsequence route over those same rows, $Q$ gives the $\RegSeqRatUp$ readback, and $R$ repacks the displayed data as a $\RegularCauchySubsequenceUp$ consumer surface.

Finally the real seal and structural rows preserve the same dependency order. The row $E$ sits downstream of $Q,F,R$ and $D$, while $H,C,P,N$ transport, replay, source, and name only the displayed coordinates. Since the carrier is exhausted by $S,T,W,Q,D,F,R,E,H,C,P,N$, the excluded choice, quotient, host-equality, independent-real, and completeness data have no row.
\end{proof}

\begin{theorem}[CauchySubsequenceCriterion fast-subsequence handoff]
\label{thm:cauchy-subsequence-criterion-fast-handoff}
Every accepted $\CauchySubsequenceCriterionUp$ packet hands any downstream $\BolzanoWeierstrassUp$, $\RealUp$, or completion-facing consumer only the displayed subsequence schedule, finite windows, dyadic tolerance rows, fast-Cauchy witness, $\RegSeqRatUp$ handoff, $\RegularCauchySubsequenceUp$ route, and real seal. In the notation of \autoref{def:cauchy-subsequence-criterion-carrier}, every such read factors through
$$
S,T,W,D,F,Q,R,E,H,C,P,N.
$$
The consumer receives no selected infinite subsequence, arbitrary choice functional, quotient stream, host real equality, independent real carrier, or $\RealUp$ completeness principle.
\end{theorem}
\begin{proof}
Start with a downstream read of an accepted criterion packet. The only exposed source of subsequence data is the schedule row $T$ together with the selected window row $W$.

The dyadic tolerance ledger $D$ is read before the fast witness $F$, so the Cauchy assertion is witnessed by finite radius and comparison rows. The readback row $Q$ and the regular-subsequence row $R$ consume the same selected windows, preventing any independent rational stream or separate subsequence route from entering the handoff.

The real seal $E$ is reached only after those rows have been displayed. Transport through $H$, replay through $C$, provenance through $P$, and local naming through $N$ preserve the same finite route for equivalent packets.

No other coordinate exists in the carrier. Hence every Bolzano--Weierstrass, real-name, or completion-facing consumer receives exactly the displayed finite subsequence criterion package and cannot read a selected infinite subsequence, choice functional, quotient stream, host real equality, independent real carrier, or completeness field.
\end{proof}

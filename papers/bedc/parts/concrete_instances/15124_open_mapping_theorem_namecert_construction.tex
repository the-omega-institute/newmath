\chapter{A Concrete Naming Certificate for $\OpenMappingTheoremUp$}
\label{ch:concrete-instances-open-mapping-theorem-namecert}
\origin{human}

Following Banach and Rudin-style functional analysis, the $\OpenMappingTheoremUp$ packet records the finite BEDC certificate for reading a surjective bounded linear map between Banach rows as an open-ball image route with a bounded-inverse-facing handoff. It is downstream of the source and target Banach certificates of \autoref{ch:concrete-instances-banach-namecert}, the bounded linear map surface of \autoref{ch:concrete-instances-linearmap-namecert}, the normed real-line and real-seal surfaces of \autoref{ch:concrete-instances-normed-real-line-namecert} and \autoref{ch:concrete-instances-real-namecert}, and the closed-graph consumer boundary of \autoref{ch:concrete-instances-closed-graph-theorem-namecert}. The packet exposes only displayed finite rows: a source Banach row, a target Banach row, a bounded linear map row, a surjective image ledger, an open-ball image witness, an inverse-window ledger, transport and replay rows, package provenance, and a local NameCert row.

\begin{definition}[Open mapping theorem carrier]
\label{def:open-mapping-theorem-carrier}
An $\OpenMappingTheoremUp$ carrier is a finite $\BHist$ packet
$$
O=(X,Y,T,S,B,I,H,C,P,N)
$$
with the following visible rows:
\begin{enumerate}
\item $X$ is the source $\BanachUp$ row, including the displayed norm and metric observations through the $\NormedRealLineUp$ and $\RealUp$ surfaces used to read source balls.
\item $Y$ is the target $\BanachUp$ row, including the displayed norm and metric observations through the same real-facing surfaces used to read target balls.
\item $T$ is the bounded $\LinearMapUp$ row from $X$ to $Y$, carrying only the finite action, linearity, and boundedness data displayed by the map certificate.
\item $S$ is the surjective image ledger, recording that every displayed target window admitted by $Y$ has a source preimage window scheduled through $X$ and read through $T$.
\item $B$ is the open-ball image witness, recording finite radii, norm comparisons, image windows, and ball-containment rows that turn source ball data into a target-neighborhood read.
\item $I$ is the inverse-window ledger, recording the bounded-inverse-facing handoff: a target window may request a displayed source window only through $S$, $B$, and the same Banach and linear-map rows.
\item $H$ records componentwise $\hsame$ transport for the source, target, map, surjectivity, open-ball, inverse-window, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from Banach admission through bounded linear map inspection, surjective image lookup, open-ball witness readback, inverse-window handoff, and closed-graph consumer routing.
\item $P$ is the $\Pkg$ provenance over $\OpenMappingTheoremUp$, $\BanachUp$, $\LinearMapUp$, $\NormedRealLineUp$, $\RealUp$, $\ClosedGraphTheoremUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$ rows.
\item $N$ is the local $\NameCert_{\OpenMappingTheoremUp}$ row.
\end{enumerate}
The carrier has no coordinate for host quotient function spaces, choice-selected inverse maps, non-displayed inverse functions, or ambient completeness beyond the Banach rows $X$ and $Y$.
\end{definition}

\begin{theorem}[Open mapping theorem NameCert obligations]
\label{thm:open-mapping-theorem-namecert-obligations}
For an accepted $\OpenMappingTheoremUp$ carrier $O=(X,Y,T,S,B,I,H,C,P,N)$, the local $\NameCert_{\OpenMappingTheoremUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the source Banach row, target Banach row, bounded linear map row, surjective image ledger, open-ball image witness, inverse-window ledger, componentwise transport, continuation replay, provenance, and local naming rows;
\item classifier stability, requiring transported packets with the same displayed rows to be identified by the carried $\hsame$ data and replayed by the same $\Cont$ route;
\item image-open ledger, requiring every target-neighborhood read to pass through the source ball data in $X$, the target ball data in $Y$, the bounded map row $T$, the surjective ledger $S$, and the open-ball witness $B$;
\item bounded-inverse handoff, requiring every inverse-facing consumer to read a source window through $S$, $B$, and $I$ before any inverse boundary is exposed;
\item no-escape, stating that no host quotient function space, choice-selected inverse map, non-displayed inverse function, or ambient completeness principle is exported beyond the displayed Banach rows.
\end{enumerate}
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:open-mapping-theorem-carrier}. The tuple supplies exactly the source, target, map, surjectivity, open-ball, inverse-window, transport, replay, provenance, and local name rows, so a consumer has no other coordinate from which to read an open-mapping conclusion.

Classifier stability is componentwise. The transport row $H$ compares the same displayed coordinates, and the continuation row $C$ replays the same route after transport. Thus moving the packet cannot replace the Banach, LinearMap, norm, Real, or ClosedGraphTheorem dependencies by a host function-space principle.

For the image-open ledger, a target-neighborhood read begins with the source and target Banach rows $X$ and $Y$. The norm and real observations used by those rows pass through \autoref{ch:concrete-instances-normed-real-line-namecert} and \autoref{ch:concrete-instances-real-namecert}. The bounded map row $T$ is then read through \autoref{ch:concrete-instances-linearmap-namecert}, the surjective ledger $S$ supplies the displayed preimage window, and $B$ records the finite ball-containment witness.

For the bounded-inverse handoff, the inverse-facing consumer receives only the ledger row $I$. That row is downstream of $S$ and $B$, so it cannot expose a selected inverse map; it can only report the displayed source-window request already justified by the open-ball image witness.

Finally $P$ and $N$ package the same finite route. Since the carrier fields are exhausted by $X,Y,T,S,B,I,H,C,P,N$, the excluded quotient, selection, inverse-function, and ambient-completeness principles have no row in the packet.
\end{proof}

\begin{theorem}[Open mapping theorem closed-graph consumer route]
\label{thm:open-mapping-theorem-closed-graph-consumer-route}
For every accepted $\OpenMappingTheoremUp$ carrier $O=(X,Y,T,S,B,I,H,C,P,N)$, every bounded-inverse or closed-graph consumer reads through the displayed route
$$
X,Y,T,S,B,I,H,C,P,N .
$$
In particular, the consumer must pass through the Banach rows of \autoref{ch:concrete-instances-banach-namecert}, the bounded linear-map row of \autoref{ch:concrete-instances-linearmap-namecert}, the norm and Real rows of \autoref{ch:concrete-instances-normed-real-line-namecert} and \autoref{ch:concrete-instances-real-namecert}, and the closed-graph boundary of \autoref{ch:concrete-instances-closed-graph-theorem-namecert} before any inverse boundary is exposed.
\end{theorem}
\begin{proof}
Start with the Banach source and target rows. A consumer can inspect source and target balls only through $X$ and $Y$, and the finite norm data in those rows is read through the normed real-line and real-seal surfaces.

Next inspect the map route. The row $T$ supplies bounded linear-map data, while $S$ supplies the displayed surjective image ledger. No inverse-facing conclusion is available before these two rows have been read.

The open-ball and inverse-window rows then determine the handoff. The row $B$ records the finite image-open witness, and $I$ records the bounded-inverse-facing source-window request that depends on that witness.

Finally the route is transported, replayed, and packaged by $H,C,P,N$. A closed-graph consumer may compare this handoff with \autoref{ch:concrete-instances-closed-graph-theorem-namecert}, but it still reads only the displayed Banach, LinearMap, norm, Real, and closed-graph rows before any inverse boundary is exposed.
\end{proof}

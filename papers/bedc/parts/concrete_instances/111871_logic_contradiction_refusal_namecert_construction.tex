\label{thm:logic-contradiction-refusal-namecert-obligations}

\begin{proof}

The contradiction request and local refutation rows separate the proposed exporter from the finite context that licenses a local negative result. Thus $Q$ may record a request for global explosion while $L$ records only the local refutation boundary.

Finally $B,H,C,P,N$ preserve the sibling boundary, transport, replay, provenance, and naming rows. Since none of the excluded host-level coordinates occurs in the carrier, no local NameCert obligation can export them.

\label{thm:logic-contradiction-refusal-nonescape}

$$

$$

It cannot export a host explosion rule, total truth row, unrestricted choice step, quotient collapse, global proof-search result, halting closure, or closed meta-loop classifier.
\end{theorem}
\begin{proof}
Start with an accepted carrier $K=(Q,L,R,G,B,H,C,P,N)$. The only row that records the requested contradiction use is $Q$, and the only row that records the locally licensed negative content is $L$.

The refusal row $R$ is downstream of both. It rejects the global exporter while leaving the local refutation row visible, so the packet does not erase contradiction management; it blocks only the attempt to turn it into a host-level proof primitive.

The gate and sibling-boundary rows $G$ and $B$ keep the refusal aligned with the broader meta-loop discipline and with the adjacent refutation, host-leakage, and quotient-boundary packets. Transport, replay, provenance, and naming are then carried only by $H,C,P,N$.

\chapter{A Concrete Naming Certificate for $\DerivedFunctorUp$}
\label{ch:concrete-instances-derivedfunctor-namecert}

The derived-functor interface $\DerivedFunctorUp$ packages the right- or left-derived $R^iF$ / $L_iF$ of an additive functor between abelian categories. It depends on $\NameCert_{\AbelianCatUp}$ (\autoref{ch:concrete-instances-abeliancat-namecert}) and $\NameCert_{\HomologyUp}$ (\autoref{ch:concrete-instances-homology-namecert}).

\section{Carrier definition}
\label{sec:derivedfunctor-carrier}

\begin{definition}[Derived-functor history carrier]\label{def:derivedfunctor-carrier}
For $h:\Hist$, the predicate $\mathsf{DerivedFunctorCarrier}(h)$ holds exactly when there are histories
$$
F,R,H,i,c:\Hist
$$
read respectively as an additive-functor trace, a chosen resolution trace, a homology trace, a unary degree row, and the intermediate resolved trace, such that
$$
\mathsf{UnaryHistory}(i)\wedge \Cont(F,R,c)\wedge \Cont(c,H,h).
$$
The carrier is the concrete $\mathsf{BHist}$ predicate generated by this tuple. The degree row is not a host-level integer choice; it is the displayed unary history used by the homology trace, while the two $\Cont$ rows record first the functor-to-resolution passage and then the homology readback that produces $h$.
\end{definition}
\leandef{BEDC.Derived.DerivedFunctorUp.DerivedFunctorCarrier}

\begin{definition}[DerivedFunctorUp resolution-spine carrier]\label{def:derivedfunctor-resolution-spine-carrier}
A $\DerivedFunctorUp$ resolution-spine carrier is a $\mathsf{BHist}$ packet whose public rows consist of an $\AbelianCatUp$ source certificate, a $\HomologyUp$ source certificate, an object row $X:\mathsf{BHist}$, a finite resolution spine
$$
R_0\to R_1\to \cdots \to R_n\to X,
$$
cycle rows $Z_i:\mathsf{BHist}$, boundary rows $B_i:\mathsf{BHist}$, a displayed additive-functor row $F:\mathsf{BHist}$, and a $\Pkg$ provenance row for the resolution ledger. The arrows in the finite spine are admissible exactly when their source and target rows are classified by the $\NameCert_{\AbelianCatUp}$ source surface, and the rows $Z_i$ and $B_i$ are read through the $\NameCert_{\HomologyUp}$ cycle and boundary classifiers at the displayed degree.
\end{definition}

\begin{theorem}[Resolution-spine carrier obligation]\label{thm:derivedfunctor-resolution-carrier-obligation}
The resolution-spine carrier of \autoref{def:derivedfunctor-resolution-spine-carrier} is closed under $\hsame$ transport of the object row and under $\Cont$ extension of the finite resolution spine. Transporting $X$ to $X'$ along $X\hsame X'$ replaces the endpoint object row while preserving the admissible $\AbelianCatUp$ source and arrow rows; appending one admissible $\Cont$ step to the finite spine gives another carried resolution ledger with the same $\Pkg$ provenance extended by that visible step.
\end{theorem}
\begin{proof}
Unpack the carrier packet into its $\AbelianCatUp$ source rows, $\HomologyUp$ source rows, object row, finite spine, functor row, and $\Pkg$ provenance row.

For object transport, apply the $\NameCert_{\AbelianCatUp}$ classifier transport to the object row. The admissible arrow rows are then read with the transported endpoint, and the homology cycle and boundary rows remain attached to the same displayed degree because no cycle row is changed.

For spine extension, append the extra admissible arrow as one more $\Cont$ ledger row. The finite spine is still a $\mathsf{BHist}$ packet, the new endpoint is classified by the same $\AbelianCatUp$ source surface, and the provenance row is repacked as the old $\Pkg$ row followed by the displayed continuation step.
\end{proof}

\begin{theorem}[Cycle-boundary classifier obligation]\label{thm:derivedfunctor-cycle-boundary-classifier-obligation}
For two accepted $\DerivedFunctorUp$ resolution-spine packets, the public classifier compares derived-functor rows by the $\HomologyUp$ cycle classifier modulo the displayed boundary rows. The $\AbelianCatUp$ source certificate supplies the admissible object and arrow transport needed to compare the selected resolution spines, while the $\HomologyUp$ source certificate supplies the cycle and boundary comparison rows.
\end{theorem}
\begin{proof}
Start with two carried packets and project their object, arrow, cycle, boundary, functor, and provenance rows. The $\NameCert_{\AbelianCatUp}$ source compares the object row and every admissible arrow row in the selected finite spines, so both packets are read over classified resolution data.

Next project the $\NameCert_{\HomologyUp}$ source. Its cycle classifier compares the displayed cycle rows at the selected degree, and its boundary classifier records exactly which boundary rows may be used to identify two cycles.

Repack these two comparisons as the $\DerivedFunctorUp$ classifier. The abelian-category part accounts only for transport of the resolution spine, while the homology part decides the public endpoint class by cycle comparison modulo the listed boundaries.
\end{proof}

\begin{theorem}[Homology readback obligation]\label{thm:derivedfunctor-homology-readback-obligation}
Every accepted $\DerivedFunctorUp$ endpoint reads back to a $\HomologyUp$ cycle class attached to the selected resolution degree. If two accepted endpoints are boundary-equivalent through the displayed boundary rows, then they have the same public $\DerivedFunctorUp$ classifier.
\end{theorem}
\begin{proof}
Read an accepted endpoint by unpacking its resolution-spine carrier. The selected degree row chooses the cycle row $Z_i$ and boundary row $B_i$ already carried by the packet, and the $\NameCert_{\HomologyUp}$ source classifies $Z_i$ as the visible cycle class.

For two endpoints, use the displayed boundary-equivalence row as a $\HomologyUp$ boundary classifier witness. The homology certificate transports the cycle classifier across that boundary row, producing the same public homology class.

The $\DerivedFunctorUp$ classifier is defined by this homology readback after the $\AbelianCatUp$ resolution rows have been transported. Therefore boundary-equivalent endpoints have the same public classifier.
\end{proof}

\begin{theorem}[Functorial transport obligation]\label{thm:derivedfunctor-functorial-transport-obligation}
An $\AbelianCatUp$ morphism row transported through the displayed additive functor sends accepted resolution spines to accepted resolution spines and preserves the $\HomologyUp$ classifier on endpoints. The transported packet keeps the same $\Pkg$ provenance form, with each functorial arrow image recorded as a visible $\Cont$ row.
\end{theorem}
\begin{proof}
Project the source morphism row and the displayed additive-functor row from the carrier. The $\NameCert_{\AbelianCatUp}$ source transports the morphism through admissible object and arrow rows, so every arrow in the resolution spine has an admissible functorial image.

Assemble the image spine by the same finite $\Cont$ order as the original spine. Each image arrow contributes one continuation row, and the $\Pkg$ provenance row records these continuations as the functorial image of the original ledger.

Finally project the endpoint cycle and boundary rows through the $\NameCert_{\HomologyUp}$ classifier. Since the functorial image respects the displayed boundary rows, the endpoint cycle class is transported to the same public classifier value in the image packet.
\end{proof}

\begin{theorem}[DerivedFunctorUp ledger exactness obligation]\label{thm:derivedfunctor-ledger-exactness-obligation}
The public $\NameCert_{\DerivedFunctorUp}$ surface consists exactly of the $\AbelianCatUp$ source rows, $\HomologyUp$ cycle and boundary rows, the finite resolution spine, the functor row, the classifier transport rows, and $\Pkg$ provenance. It exports no host quotient of complexes and no external abelian-category equality beyond the displayed $\mathsf{BHist}$, $\hsame$, $\Cont$, $\Pkg$, $\NameCert$, $\AbelianCatUp$, and $\HomologyUp$ rows.
\end{theorem}
\begin{proof}
Soundness is obtained by unpacking a carried packet. The only public rows are the two source certificates, the finite spine, the functor row, the cycle and boundary rows, classifier transport, and the provenance row.

Completeness repacks exactly those rows. Given the $\AbelianCatUp$ object and arrow rows, the $\HomologyUp$ cycle and boundary rows, the finite $\Cont$ spine, the functor row, and the $\Pkg$ provenance row, the resolution-spine carrier of \autoref{def:derivedfunctor-resolution-spine-carrier} is reconstructed without adding any host quotient.

The exactness boundary follows because every classifier theorem above projects from this same packet and every accepted endpoint reads back through the $\HomologyUp$ cycle classifier. There is no exported row naming an equality of complexes or an external abelian-category equality, so such data cannot enter the public $\NameCert_{\DerivedFunctorUp}$ surface.
\end{proof}

\begin{theorem}[DerivedFunctorUp source-coupled homology readback]\label{thm:derivedfunctor-source-coupled-homology-readback}
Every packet satisfying the resolution-spine carrier of \autoref{def:derivedfunctor-resolution-spine-carrier} projects one finite $\mathsf{BHist}$ resolution spine whose object and arrow rows are consumed by $\NameCert_{\AbelianCatUp}$ and whose cycle, boundary, and unary-degree rows are consumed by $\NameCert_{\HomologyUp}$. The two readings have the same $\Pkg$ provenance row, so the public $\NameCert_{\DerivedFunctorUp}$ surface cannot split them into unrelated resolution and homology sources.
\end{theorem}
\begin{proof}
Unpack the carried packet into its $\AbelianCatUp$ source certificate, $\HomologyUp$ source certificate, object row, finite resolution spine, functor row, displayed unary degree, cycle rows, boundary rows, and $\Pkg$ provenance.

Project the object and arrow rows to the $\NameCert_{\AbelianCatUp}$ source surface. These are exactly the rows that classify the endpoint object and every admissible arrow in the finite spine.

Project the displayed unary degree, cycle rows, and boundary rows to the $\NameCert_{\HomologyUp}$ source surface. They are read at the same selected degree and over the spine already classified by the abelian-category source rows.

The shared $\Pkg$ provenance row is part of the same carrier packet, not a separate input. Therefore the abelian-category transport side and the homology readback side are two projections of one $\mathsf{BHist}$ ledger, and no unrelated second spine is available to the public certificate.
\end{proof}

\begin{theorem}[DerivedFunctorUp boundary over transported resolution]\label{thm:derivedfunctor-boundary-over-transported-resolution}
After $\hsame$ transport of the object row and $\Cont$ extension of the finite resolution spine, the selected boundary-equivalence row still compares the $\HomologyUp$ cycle rows over the transported $\AbelianCatUp$ resolution data. Thus boundary readback remains attached to the transported source resolution rather than to an independent homology packet.
\end{theorem}
\leanchecked{BEDC.Derived.DerivedFunctorUp.DerivedFunctorCarrier\_boundary\_over\_transported\_resolution}
\begin{proof}
Use \autoref{thm:derivedfunctor-resolution-carrier-obligation} to transport the endpoint object row and extend the finite spine by the displayed continuation step. The result is again a carried resolution ledger with the same visible provenance form.

Apply \autoref{thm:derivedfunctor-cycle-boundary-classifier-obligation} to the transported carried packets. The abelian-category rows compare the transported object and arrow data, while the homology rows compare the selected cycles modulo the displayed boundary rows.

Finally read the endpoint through \autoref{thm:derivedfunctor-homology-readback-obligation}. The boundary-equivalence witness is interpreted by the $\HomologyUp$ source over the transported resolution data, so the public derived-functor classifier is preserved by the combined transport and readback.
\end{proof}

\begin{theorem}[DerivedFunctorUp downstream consumer package]\label{thm:derivedfunctor-downstream-consumer-package}
A downstream $\DerivedCatUp$ or $\SpectralSeqUp$ consumer may receive precisely the shared source certificates, finite spine, functor row, unary degree, cycle-boundary rows, classifier transport, and $\Pkg$ provenance carried by the public $\NameCert_{\DerivedFunctorUp}$ surface. It receives no host quotient of complexes and no external abelian-category equality.
\end{theorem}
\begin{proof}
Start with the coupled source packet of \autoref{thm:derivedfunctor-source-coupled-homology-readback}. It supplies one finite $\mathsf{BHist}$ spine together with the $\AbelianCatUp$ source rows and the $\HomologyUp$ cycle, boundary, and degree rows that read that same spine.

Use \autoref{thm:derivedfunctor-boundary-over-transported-resolution} for the consumer-facing transport step. The classifier rows remain tied to the transported resolution data, and the selected boundary comparison continues to read the homology endpoint over those transported rows.

Combine these projections with \autoref{thm:derivedfunctor-ledger-exactness-obligation}. The exact public package is the source certificates, finite spine, functor row, unary degree, cycle-boundary rows, classifier transport, and $\Pkg$ provenance, with no exported host quotient of complexes and no external abelian-category equality.
\end{proof}

\begin{definition}[DerivedFunctorUp exact-triangle boundary carrier]\label{def:derivedfunctor-exact-triangle-boundary-carrier}
A $\DerivedFunctorUp$ exact-triangle boundary carrier is a concrete $\mathsf{BHist}$ packet whose public rows consist of one $\AbelianCatUp$ source row, the $\HomologyUp$ cycle and boundary rows at the displayed degree, three resolution-spine packets
$$
R_A\to A,\qquad R_B\to B,\qquad R_C\to C,
$$
the connecting-morphism row $\partial:C\to A[1]$, and a $\Pkg$ provenance row for the boundary ledger. Each resolution-spine packet is a row of the form named in \autoref{def:derivedfunctor-resolution-spine-carrier}; the $\AbelianCatUp$ source row classifies the admissible arrows in the exact-triangle source row, the $\HomologyUp$ rows classify cycles and displayed boundaries, and the $\Pkg$ row records precisely the three spines, the connecting row, and the visible continuation ledgers that relate them.
\end{definition}

\begin{theorem}[Exact-triangle boundary carrier scope]\label{thm:derivedfunctor-exact-triangle-boundary-carrier-scope}
The carrier of \autoref{def:derivedfunctor-exact-triangle-boundary-carrier} is closed under $\hsame$ transport of the $\AbelianCatUp$ source row and under $\Cont$ extension of each of the three displayed resolution spines. Transporting the source row preserves the admissible arrow rows for the exact triangle, and extending any one of $R_A\to A$, $R_B\to B$, or $R_C\to C$ by a displayed continuation step produces another exact-triangle boundary carrier with the same connecting-morphism row and with the $\Pkg$ provenance row extended by that visible step.
\end{theorem}
\begin{proof}
Unpack \autoref{def:derivedfunctor-exact-triangle-boundary-carrier}. Each of the three spine rows is an instance of the resolution-spine packet of \autoref{def:derivedfunctor-resolution-spine-carrier}, so the object rows, arrow rows, cycle rows, boundary rows, functor row, and provenance row are all visible $\mathsf{BHist}$ data.

For $\hsame$ transport of the source row, apply \autoref{thm:derivedfunctor-resolution-carrier-obligation} to read each spine over the transported $\AbelianCatUp$ classifier. The homology cycle and boundary rows remain attached to the same displayed degree, and the connecting-morphism row is transported as an admissible arrow row over the same source surface.

For $\Cont$ extension, apply \autoref{thm:derivedfunctor-resolution-carrier-obligation} separately to the affected spine. The other two spines and the connecting row are unchanged, and the boundary carrier is repacked by appending the displayed continuation row to the $\Pkg$ provenance ledger.
\end{proof}

\begin{theorem}[Connecting-morphism readback]\label{thm:derivedfunctor-connecting-morphism-readback}
For every accepted exact-triangle boundary carrier, the connecting morphism $\partial:C\to A[1]$ is read back from the $\HomologyUp$ boundary classifier together with the $\AbelianCatUp$ admissible-arrow transport. The boundary classifier supplies the row identifying the target cycle class modulo displayed boundaries, while the admissible-arrow transport supplies the source and target rows over the three resolution spines.
\end{theorem}
\begin{proof}
Begin with an accepted carrier and project the three resolution-spine packets supplied by \autoref{def:derivedfunctor-exact-triangle-boundary-carrier}. By \autoref{thm:derivedfunctor-cycle-boundary-classifier-obligation}, their endpoint comparisons are computed by the $\HomologyUp$ cycle classifier modulo the displayed boundary rows.

Next project the connecting-morphism row. The source and target rows are admissible because the same $\AbelianCatUp$ source surface classifies the exact-triangle arrows, and \autoref{thm:derivedfunctor-functorial-transport-obligation} transports those admissible rows through the displayed functorial image of the resolution data.

Finally read the target endpoint with \autoref{thm:derivedfunctor-homology-readback-obligation}. The $\HomologyUp$ boundary classifier is exactly the displayed witness that turns the target cycle into the connecting class, so the public readback of $\partial$ is obtained without adding a quotient of complexes or any external equality of abelian categories.
\end{proof}

\begin{theorem}[Long-exact ledger surface]\label{thm:derivedfunctor-long-exact-ledger-surface}
The long-exact ledger surface of an accepted exact-triangle boundary carrier consists exactly of the three resolution-spine rows, the displayed cycle and boundary rows, the connecting-morphism row, the $\Cont$ ledgers relating the three spines and their endpoint readbacks, and the $\Pkg$ provenance row. It exports no host quotient of complexes and no external exact-category equality beyond the displayed $\mathsf{BHist}$, $\hsame$, $\Cont$, $\Pkg$, $\NameCert$, $\AbelianCatUp$, and $\HomologyUp$ rows.
\end{theorem}
\begin{proof}
Soundness follows by unpacking the boundary carrier of \autoref{def:derivedfunctor-exact-triangle-boundary-carrier}. The only public rows named by the packet are the $\AbelianCatUp$ source row, the $\HomologyUp$ cycle and boundary rows, the three resolution spines, the connecting row, the $\Cont$ ledgers, and the $\Pkg$ provenance row.

Completeness repacks exactly those rows. Use \autoref{def:derivedfunctor-resolution-spine-carrier} to recognize each of the three spines, then use \autoref{thm:derivedfunctor-exact-triangle-boundary-carrier-scope} to close the packet under the displayed $\hsame$ and $\Cont$ transports.

The connecting part is supplied by \autoref{thm:derivedfunctor-connecting-morphism-readback}, and the endpoint classes are the homology readbacks of \autoref{thm:derivedfunctor-homology-readback-obligation}. Applying \autoref{thm:derivedfunctor-ledger-exactness-obligation} then shows that the public surface is exhausted by the listed rows, so no host quotient of complexes or external exact-category equality is exported.
\end{proof}

\begin{theorem}[Derived-functor resolution source]\label{thm:derivedfunctor-resolution-source}
If $\mathsf{DerivedFunctorCarrier}(h)$ holds, then there are histories $F,R,H,i,c:\Hist$ witnessing the additive-functor trace, resolution trace, homology trace, unary degree row, and intermediate resolved trace with
$$
\mathsf{UnaryHistory}(i)\wedge \Cont(F,R,c)\wedge \Cont(c,H,h).
$$
The functor and resolution rows are scoped by the cited $\NameCert_{\AbelianCatUp}$ source surface, and the homology and degree rows are scoped by the cited $\NameCert_{\HomologyUp}$ source surface.
\end{theorem}
\begin{proof}
Unpack \autoref{def:derivedfunctor-carrier}. The carrier witness is already the five-history tuple $(F,R,H,i,c)$ together with the displayed unary and continuation rows.

The additive-functor and resolution histories are interpreted in the abelian-category dependency because the resolution is a chain-object witness over the same abelian source. Thus their carrier and classifier obligations are read through $\NameCert_{\AbelianCatUp}$.

The homology trace and the degree row are interpreted in the homology dependency. The row $\mathsf{UnaryHistory}(i)$ supplies the degree index, and the second continuation row $\Cont(c,H,h)$ records the homology readback from the resolved trace to the carried derived-functor history.
\end{proof}

\begin{theorem}[Derived-functor classifier stability]\label{thm:derivedfunctor-classifier-stability}
Let $h,h':\Hist$ be carried histories with $\mathsf{DerivedFunctorCarrier}(h)$ and $\mathsf{DerivedFunctorCarrier}(h')$. If $h\hsame h'$, then the $\DerivedFunctorUp$ classifier transports from $h$ to $h'$: the additive-functor, resolution, homology, unary-degree, and intermediate rows of any carrier witness for $h$ may be replaced by $\hsame$-related rows in a carrier witness for $h'$, and the two continuation rows remain valid by $\Cont$ congruence.
\end{theorem}
\leanchecked{BEDC.Derived.DerivedFunctorUp.DerivedFunctorCarrier\_classifier\_stability}
\begin{proof}
Start with a carrier witness for $h$ and unpack it as $(F,R,H,i,c)$ with $\mathsf{UnaryHistory}(i)$, $\Cont(F,R,c)$, and $\Cont(c,H,h)$. Also unpack the carrier witness for $h'$ as $(F',R',H',i',c')$ with the corresponding unary and continuation rows.

The source certificate $\NameCert_{\AbelianCatUp}$ transports the additive-functor and resolution rows along their classifier comparisons. The source certificate $\NameCert_{\HomologyUp}$ transports the homology trace and the unary degree row, preserving the unary carrier of the degree row.

It remains to transport the two continuation rows. Apply the $\Cont$ congruence facts to the transported functor and resolution rows to obtain the resolved intermediate row, and then apply $\Cont$ congruence again to the transported intermediate and homology rows. The final endpoint is identified with $h'$ by the displayed hypothesis $h\hsame h'$, so the transported tuple is again a valid $\DerivedFunctorUp$ classifier witness.
\end{proof}

\section{The certificate}
\label{sec:derivedfunctor-certificate}

\begin{theorem}[Derived-functor NameCert obligation surface]\label{thm:derivedfunctor-namecert-obligation-surface}
The $\NameCert_{\DerivedFunctorUp}$ obligation surface consists of the carrier predicate of \autoref{def:derivedfunctor-carrier}, the classifier transported in \autoref{thm:derivedfunctor-classifier-stability}, the source-witness extraction of \autoref{thm:derivedfunctor-resolution-source}, the exactness clause that every accepted derived-functor row unpacks into exactly those $\mathsf{BHist}$ witnesses, and the ledger clause that records only the resolution, degree, homology, and boundary rows named by the carrier.
\end{theorem}
\begin{proof}
The carrier field is \autoref{def:derivedfunctor-carrier}: it is a predicate on $\Hist$ with concrete additive-functor, resolution, homology, degree, and intermediate witnesses. The source field is \autoref{thm:derivedfunctor-resolution-source}, which assigns those witnesses to $\NameCert_{\AbelianCatUp}$ and $\NameCert_{\HomologyUp}$.

The classifier and stability fields are exactly \autoref{thm:derivedfunctor-classifier-stability}. Transport is not a new relation; it is the restriction of $\hsame$ to the five carrier rows, with $\Cont$ congruence preserving the two displayed continuation ledgers.

Exactness is the bidirectional reading of the same data. Soundness reads each accepted row from the carrier tuple, and completeness repacks any tuple with $\mathsf{UnaryHistory}(i)$, $\Cont(F,R,c)$, and $\Cont(c,H,h)$ into the carrier. These rows reassemble the public $\NameCert_{\DerivedFunctorUp}$ obligation surface.
\end{proof}

\begin{theorem}[Derived-functor ledger policy scope]\label{thm:derivedfunctor-ledger-policy-scope}
The $\DerivedFunctorUp$ ledger records only the chosen resolution row $R$, the unary degree row $i$, the homology row $H$, the two continuation rows $\Cont(F,R,c)$ and $\Cont(c,H,h)$, and the exactness boundary that repacks these rows into $\mathsf{DerivedFunctorCarrier}(h)$. It records no quotient representatives, no choice data, and no additional host presentation of derived functors.
\end{theorem}
\begin{proof}
Read the ledger from \autoref{def:derivedfunctor-carrier}. The only hidden rows in the carrier are the resolution row, the unary degree row, the homology row, and the intermediate continuation boundary from the additive-functor trace through the resolved trace to the carried history.

Apply \autoref{thm:derivedfunctor-resolution-source} to identify which source certificate scopes each row. The abelian-category certificate scopes the functor and resolution source, while the homology certificate scopes the homology and unary degree source.

Finally apply \autoref{thm:derivedfunctor-namecert-obligation-surface}. Its exactness clause says that accepted rows are exhausted by this carrier tuple. Since neither the carrier nor the exactness repacking contains quotient representatives, choice data, or an ambient host presentation, the ledger cannot expose any such row.
\end{proof}

\begin{definition}[Derived-functor delta-splice surface]\label{def:derivedfunctor-delta-splice-surface}
A $\DerivedFunctorUp$ delta-splice surface is a displayed $\mathsf{BHist}$ packet over $\NameCert_{\AbelianCatUp}$ and $\NameCert_{\HomologyUp}$ whose public rows consist of short-exact source rows
$$
0\to A\to B\to C\to 0,
$$
selected finite resolution rows for $A$, $B$, and $C$, cycle rows $Z_i(A)$, $Z_i(B)$, $Z_i(C)$, boundary rows $B_i(A)$, $B_i(B)$, $B_i(C)$, and a connecting row
$$
\delta_i: Z_i(C)/B_i(C)\to Z_{i-1}(A)/B_{i-1}(A).
$$
The packet also carries finite $\Cont$ projection ledgers from the displayed boundary rows to $\delta_i$, $\hsame$ transport witnesses comparing the chosen resolution and homology rows across the short-exact source row, and a $\Pkg$ provenance row naming the AbelianCatUp and HomologyUp certificates used to read the packet.
\end{definition}

\begin{theorem}[Derived-functor short-exact source scope]\label{thm:derivedfunctor-short-exact-source-scope}
The packet scope of a $\DerivedFunctorUp$ delta-splice surface is exactly the $\AbelianCatUp$ short-exact source-row data together with the $\HomologyUp$ cycle and boundary data named in \autoref{def:derivedfunctor-delta-splice-surface}. It exports no additional complex equality, no ambient abelian-category object equality, and no host quotient representative for the displayed homology rows.
\end{theorem}
\begin{proof}
Project the delta-splice packet into its named rows. The short-exact source row, the three selected resolution rows, and the admissible arrows between them are read through $\NameCert_{\AbelianCatUp}$.

The cycle rows and boundary rows are read through $\NameCert_{\HomologyUp}$ at the displayed degrees. These rows supply the only homology data used by the packet; the notation $Z_i(-)/B_i(-)$ is the public readback of those rows, not an extra host quotient.

Conversely, given exactly these AbelianCatUp source rows and HomologyUp cycle and boundary rows, the packet is recovered by repacking the displayed $\Cont$ ledgers, $\hsame$ witnesses, and $\Pkg$ provenance. No row remains available for an ambient equality of complexes or for an external object equality, so the scope is precisely the listed source data.
\end{proof}

\begin{theorem}[Derived-functor connecting row ledger]\label{thm:derivedfunctor-connecting-row-ledger}
In every $\DerivedFunctorUp$ delta-splice surface, the connecting row $\delta_i$ is reached by finite $\Cont$ readback from the displayed HomologyUp boundary rows and transports along the listed $\hsame$ witnesses. Thus replacing the selected resolutions or cycle-boundary representatives by $\hsame$-related rows preserves the same public connecting row.
\end{theorem}
\begin{proof}
Unpack the packet and isolate the boundary rows for $C$ and $A$ in adjacent degrees. The finite projection ledger contains a chain of $\Cont$ rows from these displayed boundary rows to the row named $\delta_i$.

Use $\Cont$ associativity to compose the projection rows into a single readback ledger. This produces the connecting row from the same visible boundary data and does not require a chosen host representative of a quotient class.

Now apply the listed $\hsame$ transport witnesses to the selected resolutions and to the cycle-boundary rows. The HomologyUp classifier transports boundary readback along these witnesses, and the composed $\Cont$ ledger transports with it. Repacking the transported rows gives the same public $\delta_i$ row in the delta-splice surface.
\end{proof}

\begin{theorem}[Derived-functor long exact splice coverage]\label{thm:derivedfunctor-long-exact-splice-coverage}
Adjacent degree rows in a $\DerivedFunctorUp$ delta-splice surface splice into the visible long exact row by $\Cont$ associativity and HomologyUp boundary transport. Each neighboring pair uses only the carried short-exact source row, the selected resolution rows, the displayed cycle and boundary rows, the connecting row $\delta_i$, and their $\Pkg$ provenance.
\end{theorem}
\begin{proof}
Fix two adjacent displayed degree rows in the packet. Project their cycle rows, boundary rows, and connecting row from \autoref{def:derivedfunctor-delta-splice-surface}.

The HomologyUp boundary classifier supplies the transport from cycles modulo boundaries at degree $i$ to the boundary data read at degree $i-1$. The connecting row ledger of \autoref{thm:derivedfunctor-connecting-row-ledger} identifies this transport with the finite $\Cont$ readback named $\delta_i$.

Compose the degree-$i$ readback, the connecting-row readback, and the degree-$(i-1)$ readback by $\Cont$ associativity. The resulting finite ledger is exactly the adjacent segment of the long exact row, and repeating the same projection for each neighboring pair covers the displayed long exact splice without adding any row beyond the packet.
\end{proof}

\begin{theorem}[Derived-functor AbelianCat-Homology consumer surface]\label{thm:derivedfunctor-abeliancat-homology-consumer-surface}
Every downstream consumer of a $\DerivedFunctorUp$ delta-splice surface receives only the AbelianCatUp short-exact source rows, the HomologyUp cycle and boundary rows, the connecting row $\delta_i$, the finite $\Cont$ ledgers, the $\hsame$ witnesses, and the $\Pkg$ provenance. Any consumer row that is invariant under these projections is therefore invariant under the whole delta-splice packet.
\end{theorem}
\begin{proof}
Project the packet to the rows listed in \autoref{def:derivedfunctor-delta-splice-surface}. These are exactly the source rows identified by \autoref{thm:derivedfunctor-short-exact-source-scope} together with the connecting ledger identified by \autoref{thm:derivedfunctor-connecting-row-ledger}.

For a downstream consumer, supply those projected rows as its input packet. The long exact coverage theorem then provides every adjacent splice row as a finite $\Cont$ consequence of the same input, so no consumer has access to a hidden equality of complexes, an unlisted representative, or an external abelian-category presentation.

If the consumer predicate is invariant under the projected rows, transport the input packet by the displayed $\hsame$ witnesses and repack the same $\Pkg$ provenance. The consumer output is unchanged because every visible row it can read is either one of the projections or a finite $\Cont$ composite of them.
\end{proof}

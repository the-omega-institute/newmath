\chapter{A Concrete Naming Certificate for $\RecursionAuthorizationLedgerUp$}
\label{ch:concrete-instances-recursion-authorization-ledger-namecert}
\origin{human}

The $\RecursionAuthorizationLedgerUp$ packet turns an accepted recursor call into finite NameCert ledger data. It records the inductive signature accepted by the kernel, the emitted eliminator, the checked motive, the branch package, the structural descent row, the generated output row, transport and continuation routes, package provenance, and the local naming row that later audit and generator-closure consumers can inspect.

\begin{definition}[RecursionAuthorizationLedger carrier]
\label{def:recursion-authorization-ledger-carrier}
A $\RecursionAuthorizationLedgerUp$ carrier is a finite $\BHist$ packet
$$
L=(I,R,M,B,D,O,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $I$ is the accepted inductive signature row, including the constructor inventory and the recursive argument positions admitted by the kernel;
\item $R$ is the emitted eliminator row for $I$, read as the authorized recursor generated from that signature;
\item $M$ is the motive row checked for the eliminator call;
\item $B$ is the branch-package row, with one displayed branch for each constructor in $I$;
\item $D$ is the structural descent ledger recording that every recursive subcall is made only on constructor-subdata admitted by $I$;
\item $O$ is the generated output row whose public name is being certified by the accepted recursor call;
\item $H$ records componentwise $\hsame$ transports for the signature, eliminator, motive, branch package, descent ledger, output row, provenance row, and local naming row;
\item $C$ records the displayed $\Cont$ routes by which consumers read the signature, eliminator, motive, branch, descent, output, audit, and naming coordinates;
\item $P$ is the $\Pkg$ provenance over $\BHist$, $\hsame$, $\Cont$, the kernel-accepted signature, emitted eliminator, branch-completeness check, structural descent check, purity audit boundary, and neighboring generator certificates;
\item $N$ is the local $\NameCert_{\RecursionAuthorizationLedgerUp}$ row.
\end{enumerate}
The classifier compares accepted carriers only through the displayed rows $I,R,M,B,D,O,H,C,P,N$.
\end{definition}

\begin{theorem}[RecursionAuthorizationLedger NameCert obligations]
\label{thm:recursion-authorization-ledger-namecert-obligations}
For the carrier of \autoref{def:recursion-authorization-ledger-carrier}, the local $\NameCert_{\RecursionAuthorizationLedgerUp}$ surface consists of carrier admission for
$$
L=(I,R,M,B,D,O,H,C,P,N),
$$
kernel-acceptance of the constructor signature $I$, emitted-eliminator exactness for $R$, motive admission for $M$, branch completeness for $B$, structural descent exactness for $D$, output-row exposure for $O$, componentwise $\hsame$ transport, displayed $\Cont$ route replay, $\Pkg$ provenance, local naming, and purity-audit non-escape.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:recursion-authorization-ledger-carrier}. The signature, emitted eliminator, motive, branch package, descent ledger, output row, transport row, continuation row, provenance row, and local naming row are all coordinates of one finite packet.

The kernel-facing obligations are the accepted signature and emitted recursor rows. The recursor-as-authorized-generator vision supplies the boundary: a constructor signature is usable only after kernel acceptance, and the eliminator row is the kernel output for that signature rather than a separate recursion principle.

Branch completeness and structural descent are read from $B$ and $D$. The branch package is indexed by the displayed constructors of $I$, and every recursive call recorded by $D$ descends through constructor-subdata accepted by the same signature.

The output and audit obligations are the remaining rows. Consumers reach $O$ only along $C$, transport it only through $H$, and cite only the provenance in $P$ and $N$. The source gate, build gate, and purity-audit boundary named by $P$ exclude axiom channels and forbidden host principles from the ledger surface.
\end{proof}

\begin{theorem}[RecursionAuthorizationLedger branch completeness and descent exactness]
\label{thm:recursion-authorization-ledger-obligation-branch-completeness}
For an accepted $\RecursionAuthorizationLedgerUp$ packet
$$
L=(I,R,M,B,D,O,H,C,P,N),
$$
every constructor row displayed by $I$ has exactly one branch row in $B$, and every recursive call certified by the output row $O$ is justified by the structural descent ledger $D$. Thus the exported ledger contains no omitted branch and no non-structural recursive call.
\end{theorem}
\begin{proof}
Project the signature and branch coordinates $I$ and $B$ from the carrier. The inventory in \autoref{def:recursion-authorization-ledger-carrier} records $B$ as the branch-package row with one displayed branch for each constructor in $I$.

Project the descent and output coordinates $D$ and $O$. The same carrier records $D$ as the structural descent ledger and $O$ as the generated output row whose recursive calls are certified by that ledger.

Use the NameCert obligations of \autoref{thm:recursion-authorization-ledger-namecert-obligations}. Branch completeness and structural descent exactness are local obligations for the same finite packet, so no consumer can read $O$ without the constructor-indexed branch package and the descent rows that justify its recursive subcalls.

The remaining rows $H,C,P,N$ only transport, expose, package, and name those displayed coordinates. Since they add no branch coordinate and no recursion coordinate, the public ledger exports neither an omitted branch nor a non-structural recursive call.
\end{proof}

\begin{theorem}[RecursionAuthorizationLedger non-escape]
\label{thm:recursion-authorization-ledger-nonescape}
Every consumer read of an accepted $\RecursionAuthorizationLedgerUp$ packet factors through
$$
I,R,M,B,D,O,H,C,P,N.
$$
It exports no unlisted constructor, no omitted branch, no branch outside the accepted signature, no non-structural recursive call, no hidden eliminator, no unrestricted recursion principle, and no output whose dependency route escapes the build, source-audit, and purity-audit boundary recorded in the packet.
\end{theorem}
\begin{proof}
Start with the obligation theorem. It places the generated output downstream of the accepted signature, emitted eliminator, motive, branch package, structural descent ledger, transport rows, continuation routes, provenance, and local naming row.

For constructors and branches, inspect $I$ and $B$. The visible signature supplies the only constructor inventory, and the branch package supplies exactly the branch rows available to the eliminator call.

For recursion, inspect $D$. A recursive subcall can be consumed only when the descent ledger records it as constructor-subdata of the accepted signature. There is no coordinate for a non-structural recursive call or a second hidden eliminator.

Finally inspect $P$ and $N$. They record the build, source-audit, and purity-audit boundary for the named output. Since the carrier contains no axiom channel, unrestricted recursion principle, or forbidden host dependency row, the public ledger cannot export any of those objects.
\end{proof}

\chapter{A Concrete Naming Certificate for $\CompactMetricUp$}
\label{ch:concrete-instances-compactmetric-namecert}

The compact-metric interface $\CompactMetricUp$ packages a metric carrier that is both totally bounded and complete in the senses of $\NameCert_{\TotallyBoundedUp}$ (\autoref{ch:concrete-instances-totallybounded-namecert}) and $\NameCert_{\CompleteMetricUp}$ (\autoref{ch:concrete-instances-completemetric-namecert}). It is the metric form of compactness used by the Heine--Cantor capstone (\autoref{ch:capstones-compact-uniform-continuity}).

\section{Carrier definition}
\label{sec:compactmetric-carrier}

\begin{definition}[Compact metric certificate carrier]
\label{def:compactmetric-certificate-carrier}
Let $X$ be a certified metric carrier over $\mathsf{BHist}$. The compact
metric certificate carrier
$$
\mathsf{CompactMetricCertificate}(X)
$$
is the conjunction of the following BEDC certificate fields:
\begin{enumerate}
  \item a $\MetricSpaceUp$ carrier and distance package for $X$, with
  distance rows encoded by $\mathsf{MetricDistanceWitness}$ and classifier
  transport on point histories;
  \item for every positive $\RatUp$ tolerance history $\varepsilon$, a
  $\mathsf{TotallyBoundedProbeBundleNet}(X,\varepsilon,\mathcal{P}_{\varepsilon},net_{\varepsilon})$
  package of
  \autoref{thm:totallybounded-probebundle-net-namecert-target};
  \item for every regular Cauchy $\StreamNameUp$ stream name $s$ whose
  observations are carried by $X$, a complete metric limit witness
  $\mathsf{CompleteMetricLimitWitness}(X,s,\mu_s,\ell_s,L_s)$ as in
  \autoref{def:completemetric-limit-witness}.
\end{enumerate}
Thus compactness is represented by the metric carrier, the finite
ProbeBundle-net family at each visible positive tolerance, and the explicit
limit-witness family for all regular Cauchy streams. No external compact set
or host-level subsequence principle is part of the carrier.
\end{definition}

\section{The certificate}
\label{sec:compactmetric-certificate}

\begin{theorem}[Compact metric package from total boundedness and completeness]
\label{thm:compactmetric-certificate-package}
For a certified metric carrier $X$, the three fields of
\autoref{def:compactmetric-certificate-carrier} form the paper-level
$\NameCert$ target for $\CompactMetricUp$. The classifier is inherited from
the $\MetricSpaceUp$ point and distance classifier; the totally-bounded field
is stable by
\autoref{thm:totallybounded-probebundle-net-coverage-stability}; and the
complete-metric field is stable by
\autoref{thm:completemetric-limit-witness-stability}.
\end{theorem}
\leanchecked{BEDC.Derived.CompactMetricUp.CompactMetricCertificate\_hsame\_transport}
\leanchecked{BEDC.Derived.CompactMetricUp.CompactMetricSingleton\_certificate\_package}
\begin{proof}
Start with the metric carrier field. It supplies the source object, point
classifier, distance rows, and metric-distance transport used by both
subcertificates, so the public $\CompactMetricUp$ carrier can use the same
histories and the same $\mathsf{MetricDistanceWitness}$ ledger policy.

For total boundedness, fix a transported positive tolerance history. The
certificate carrier provides the corresponding closed $\mathsf{ProbeBundle}$
net, its bundled center histories, positive radius histories, membership
rows, distance witnesses, and rational comparison ledgers. Apply
\autoref{thm:totallybounded-probebundle-net-coverage-stability} to transport
this finite-net row along the metric and $\RatUp$ classifier data.

For completeness, fix a transported regular Cauchy stream name whose
observations are carried by $X$. The certificate carrier supplies the modulus
history, limit history, threshold ledgers, and late-observation distance
bounds. Apply
\autoref{thm:completemetric-limit-witness-stability} to transport these rows
pointwise and recover the displayed limit witness for the transported stream.

The ledger policy for $\CompactMetricUp$ is exactly the union of these three
visible packages: metric carrier data, ProbeBundle-net data at each positive
tolerance, and complete-limit data for each regular Cauchy stream. Since each
component has classifier transport and the shared source classifier is the
metric one, the assembled fields satisfy the paper-level $\NameCert$ target
for $\CompactMetricUp$.
\end{proof}

\begin{definition}[Public CompactMetricUp export source]
\label{def:compactmetric-public-export-source}
For a certified metric carrier $X$, the public $\CompactMetricUp$ export
source is the same shared carrier used in
\autoref{def:compactmetric-certificate-carrier}: the $\MetricSpaceUp$ point
carrier and distance package for $X$, together with the
$\mathsf{TotallyBoundedProbeBundleNet}(X,\varepsilon,\mathcal{P}_{\varepsilon},net_{\varepsilon})$
family for every positive $\RatUp$ tolerance history $\varepsilon$, and the
$\mathsf{CompleteMetricLimitWitness}(X,s,\mu_s,\ell_s,L_s)$ family for every
regular Cauchy $\StreamNameUp$ stream name $s$ carried by $X$.

The public surface exposes exactly these three component families as the
compact-metric constructor source. It does not add an open-cover selector,
subsequence extractor, or external compact set; downstream consumers receive
the finite-net and complete-limit rows already certified by the component
NameCerts.
\end{definition}

\begin{theorem}[Public compact-metric constructor package]
\label{thm:compactmetric-public-constructor-package}
The export source of \autoref{def:compactmetric-public-export-source}
produces the public compact-metric constructor package for $\CompactMetricUp$.
The package has the same metric carrier, the same positive-tolerance
$\mathsf{ProbeBundle}$ net family, and the same regular-Cauchy limit-witness
family as \autoref{thm:compactmetric-certificate-package}; classifier
transport is inherited componentwise from the metric, totally-bounded, and
complete-metric certificates. The natural formal target name for this
constructor package is
\texttt{BEDC.Derived.CompactMetricUp.CompactMetricPublicConstructor\_package}.
\end{theorem}

\begin{proof}
Project the metric field from
\autoref{thm:compactmetric-certificate-package}. This gives the public source
carrier, the point classifier, the distance rows, and the
$\mathsf{MetricDistanceWitness}$ ledgers used by the constructor package.

For each positive $\RatUp$ tolerance history $\varepsilon$, project the
totally-bounded field. The projected row is already the closed
$\mathsf{ProbeBundle}$ net
$\mathsf{TotallyBoundedProbeBundleNet}(X,\varepsilon,\mathcal{P}_{\varepsilon},net_{\varepsilon})$,
and its stability is the componentwise transport supplied by
\autoref{thm:totallybounded-probebundle-net-coverage-stability}.

For each regular Cauchy $\StreamNameUp$ stream name $s$, project the
complete-metric field. The projected row is the limit witness
$\mathsf{CompleteMetricLimitWitness}(X,s,\mu_s,\ell_s,L_s)$, transported by
\autoref{thm:completemetric-limit-witness-stability}.

Repack these three projected component families under the shared metric
classifier. Since the public export source contains no additional fields, the
componentwise transports assemble directly into the public
$\CompactMetricUp$ constructor package.
\end{proof}

\begin{theorem}[Heine--Cantor consumer bridge for CompactMetricUp]
\label{thm:compactmetric-heine-cantor-consumer-bridge}
Let
$$
\mathsf{ContinuousMapCarrier}(source,map,target,modulus,cert,distance)
$$
be a metric $\ContinuousMapUp$ consumer whose source metric carrier is the
same BEDC carrier $X$ as the export source of
\autoref{def:compactmetric-public-export-source}. Then the public
$\CompactMetricUp$ export supplies the compact finite-net input and
complete-limit input required by the Heine--Cantor capstone
\autoref{thm:compact-uniform-continuity}. The finite-net input is the
positive-tolerance
$\mathsf{TotallyBoundedProbeBundleNet}(X,\varepsilon,\mathcal{P}_{\varepsilon},net_{\varepsilon})$
family, and the complete-limit input is the regular-Cauchy
$\mathsf{CompleteMetricLimitWitness}(X,s,\mu_s,\ell_s,L_s)$ family; both are
consumed over the same metric distance rows as the continuous-map source
certificate. The natural formal target name for this consumer bridge is
\texttt{BEDC.Derived.CompactMetricUp.CompactMetricHeineCantor\_consumer\_bridge}.
\end{theorem}

\begin{proof}
Read the continuous-map source through
\autoref{def:continuousmapup-metric-source-specification}. The consumer
therefore uses a categorical graph row, a modulus witness, and a
$\mathsf{MetricDistanceWitness}$ row over the displayed source carrier.

Identify this source carrier with the metric carrier in the public
$\CompactMetricUp$ export. The metric classifier then transports the
continuous-map distance row and the compact-metric distance row to the same
BEDC history relation.

For a positive tolerance history $\varepsilon$, pass the exported
$\mathsf{ProbeBundle}$ net family to the finite-cover input of the
Heine--Cantor construction. This is exactly the located finite-net datum used
by \autoref{def:compact-uniform-finite-net} and
\autoref{lem:compact-uniform-subordinate-modulus-cover}.

For a regular Cauchy stream name $s$, pass the exported complete-limit witness
to the complete metric input. The capstone proof may then consume finite-net
coverage and complete-limit convergence over the same source metric, so the
uniform-modulus ledger is obtained from the public $\CompactMetricUp$ export
rather than from a private component package.
\end{proof}

\chapter{A Concrete Naming Certificate for $\FiniteVectorUp$}
\label{ch:concrete-instances-finitevector-namecert}

The $\FiniteVectorUp$ interface names fixed-length finite component data over the already displayed $\NatUp$, $\ListUp$, $\ProdUp$, and optional $\VecSpaceUp$ certificate surfaces. Pierce, Types and Programming Languages (2002), treats length-indexed finite sequences as typed finite data; the BEDC packet below keeps the same finite-index reading at the $\mathsf{BHist}$ level. It records the length row, component spine, index-component pairing rows, componentwise classifier ledger, read continuations, and package provenance, without forming an infinite sequence, quotient vector, basis choice, dimension theorem, or linear-independence principle.

\section{Finite-vector carrier}
\label{sec:finitevector-carrier}

\begin{definition}[FiniteVector carrier]
\label{def:finitevector-carrier}
A $\FiniteVectorUp$ carrier is a finite $\mathsf{BHist}$ packet
$$
V=(n,L,I,C,R,P,H)
$$
with the following visible rows:
\begin{enumerate}
\item $n$ is a $\NatUp$ length row;
\item $L$ is a $\ListUp$ component spine whose displayed length is $n$;
\item $I$ is a finite $\ProdUp$ spine of index-component pairing rows, one row for each visible index below $n$;
\item $C$ is the optional component provenance row, read through $\VecSpaceUp$ exactly when the entries are vector-space rows and omitted otherwise;
\item $R$ is the componentwise $\hsame$ ledger comparing displayed components at matching indices;
\item $P$ is the $\Pkg$ provenance tying the packet to the consumed $\NatUp$, $\ListUp$, $\ProdUp$, and optional $\VecSpaceUp$ certificate boundaries;
\item $H$ is the ledger that hides only unused positions outside the displayed length and records every public component read through the displayed $\Cont$ routes.
\end{enumerate}
The carrier predicate is $\mathsf{FiniteVectorPacket}(n,L,I,C,R,P,H)$. It contains no infinite stream, no quotient of component rows, no ambient basis selection, and no dimension or linear-independence datum beyond the displayed finite length.
\end{definition}

\begin{definition}[FiniteVector classifier]
\label{def:finitevector-classifier}
For accepted $\FiniteVectorUp$ carriers $V=(n,L,I,C,R,P,H)$ and $V'=(n',L',I',C',R',P',H')$, the classifier $\mathsf{FiniteVectorSame}(V,V')$ holds exactly when the following finite rows are displayed:
\begin{enumerate}
\item an $\hsame$ row comparing the $\NatUp$ length rows $n$ and $n'$;
\item a $\ListUp$ classifier row comparing the component spines $L$ and $L'$ after length transport;
\item for each displayed index below the transported length, a $\ProdUp$ classifier row comparing the corresponding index-component pairing rows in $I$ and $I'$;
\item when vector-space component provenance is present, a $\VecSpaceUp$ classifier row comparing the corresponding component provenance rows $C$ and $C'$;
\item the same $\Cont$ routes and $\Pkg$ provenance showing that every public read uses the displayed length, spine, pair rows, and componentwise $\hsame$ ledger.
\end{enumerate}
No other classifier is exported by the seed packet.
\end{definition}

\begin{theorem}[FiniteVector length-index stability]
\label{thm:finitevector-length-index-stability}
Let $V=(n,L,I,C,R,P,H)$ be an accepted $\FiniteVectorUp$ carrier. If the $\NatUp$ length row, $\ListUp$ component spine, $\ProdUp$ index-component pair rows, optional $\VecSpaceUp$ component provenance, componentwise $\hsame$ ledger, $\Cont$ routes, $\Pkg$ provenance, and outside-length ledger are transported by displayed $\hsame$ along the same public routes named by $P$, then the transported packet $V'=(n',L',I',C',R',P',H')$ is again an accepted $\FiniteVectorUp$ carrier.

Moreover, every public component read at a displayed index below $n$ transports to the matching read below $n'$, and the transported packet hides only positions outside the transported displayed length.
\end{theorem}
\begin{proof}
Project the seven coordinates of \autoref{def:finitevector-carrier}. The $\NatUp$ row transports first, so the displayed length of the target packet is fixed before any component read is considered.

Transport the $\ListUp$ spine along that length row and then transport each $\ProdUp$ index-component pair. The pair rows keep the index and the component in the same finite slot, so a read below the displayed length remains below the transported displayed length.

If the component provenance row is present, transport it through the stated $\VecSpaceUp$ classifier; otherwise the packet stays in the component-agnostic branch. The componentwise $\hsame$ ledger then repacks over the transported spine.

Finally transport the $\Cont$ routes, $\Pkg$ provenance, and outside-length ledger. Since $H$ hides only positions outside the displayed length, its transported copy hides only positions outside the transported length, and the result is an accepted $\FiniteVectorUp$ carrier.
\end{proof}

\begin{theorem}[FiniteVector componentwise ledger exactness]
\label{thm:finitevector-componentwise-ledger-exactness}
For accepted $\FiniteVectorUp$ carriers $V$ and $V'$, a proof of $\mathsf{FiniteVectorSame}(V,V')$ is equivalent, as public BEDC data, to the finite componentwise ledger consisting of the transported length row, the transported list spine, the transported index-component pairing rows, the optional transported $\VecSpaceUp$ component provenance, and the displayed $\hsame$ comparison at every visible index.

Consequently, the classifier does not compare hidden positions, does not form a quotient of component spines, and does not add a host vector equality outside the displayed ledger.
\end{theorem}
\begin{proof}
From a classifier proof, read off the five rows listed in \autoref{def:finitevector-classifier}. They are exactly the length comparison, spine comparison, pair-row comparisons, optional vector-space provenance comparison, and componentwise $\hsame$ rows.

Conversely, suppose the finite ledger rows are displayed. Repack them in the order required by \autoref{def:finitevector-classifier}: first the length row, then the list spine, then the pair rows, then the optional component provenance row, and finally the shared route and provenance rows.

The outside-length ledger is not part of the component comparison except for recording which positions are not publicly read. Therefore the equivalence exposes precisely the visible component rows and does not introduce a quotient, an infinite tail, or host vector equality.
\end{proof}

\begin{theorem}[FiniteVector NameCert obligation surface]
\label{thm:finitevector-namecert-obligation-surface}
The local $\NameCert_{\FiniteVectorUp}$ obligation surface consists of five finite rows:
\begin{enumerate}
\item carrier: the $\mathsf{BHist}$ packet $(n,L,I,C,R,P,H)$ of \autoref{def:finitevector-carrier};
\item classifier and stability: the classifier of \autoref{def:finitevector-classifier} together with length-index transport from \autoref{thm:finitevector-length-index-stability};
\item length-index exactness: the visible $\NatUp$ length row and the assertion that the $\ListUp$ component spine has exactly that displayed length;
\item componentwise ledger: the $\ProdUp$ index-component pair rows and the $\hsame$ comparisons of \autoref{thm:finitevector-componentwise-ledger-exactness};
\item downstream consumer boundary: the $\Pkg$ and $\Cont$ rows exposing finite component reads to later vector, matrix, and finite-dimensional certificate consumers while hiding only positions outside the displayed length.
\end{enumerate}
These rows are the full public seed surface for the finite-vector name.
\end{theorem}
\begin{proof}
The carrier row fixes the displayed length, component spine, pair rows, optional component provenance, componentwise ledger, provenance, and outside-length ledger. This gives the source domain of the finite-vector name.

The classifier and stability row is supplied by \autoref{def:finitevector-classifier} and \autoref{thm:finitevector-length-index-stability}. Together they say that every accepted replacement is componentwise $\hsame$ transport along the displayed $\Cont$ routes.

The length-index and componentwise rows are projections of the same carrier. The length row bounds the public read surface, and the pair rows identify which component belongs to each visible index.

Finally the downstream boundary is exactly the $\Pkg$, $\Cont$, and outside-length ledger. Since no coordinate contains an infinite sequence, quotient vector, basis choice, dimension theorem, or linear-independence witness, the five listed rows exhaust the $\NameCert_{\FiniteVectorUp}$ obligation surface.
\end{proof}

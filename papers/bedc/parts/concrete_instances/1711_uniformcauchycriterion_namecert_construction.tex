\begin{definition}[UniformCauchyCriterion carrier]
\label{def:uniform-cauchy-criterion-carrier}

$$

\begin{enumerate}

\item $W$ is the shared finite $\StreamNameUp$ window family, indexed by $I$, through which every source observation is read;
\item $M$ is one $\CauchyModulusUp$ schedule used uniformly for all indices in $I$;
\item $Q$ is the $\DyadicRatCoreUp$ tolerance row whose dyadic budget is used by the family test;
\item $T$ is the finite $\DyadicTailBoundUp$ tail-comparison ledger recording all accepted comparisons beyond the threshold selected by $M$ and inside the windows named by $W$;
\item $E$ is the $\RealUp$ seal handoff row reached after the finite family tail criterion has been accepted;
\item $H$ records the componentwise $\hsame$ rows for the family ledger, window family, modulus schedule, tolerance row, tail ledger, and seal handoff;

The classifier compares two accepted carriers only through these displayed coordinates, their componentwise transports, their continuation routes, and their package and local naming provenance.
\end{definition}

\begin{theorem}[UniformCauchyCriterion NameCert obligations]
\label{thm:uniform-cauchy-criterion-namecert-obligations}

\item classifier transport by componentwise $\hsame$ and displayed $\Cont$ routes over the finite family ledger, shared window family, modulus schedule, tolerance row, tail ledger, seal handoff, package provenance, and local naming row;
\item uniform window stability, requiring every transported precision request to keep the same $I$-indexed $\StreamNameUp$ window family and the same shared modulus row $M$ before tail comparisons are read;
\item $\RealUp$ seal handoff, requiring the family tail ledger to pass through $T$ and the real-facing row $E$ after the shared threshold and dyadic tolerance have been displayed;
\item non-escape from quotient streams, host real equality, selected limits, and ambient completeness.

\end{theorem}
\begin{proof}

\begin{theorem}[UniformCauchyCriterion window stability]
\label{thm:uniform-cauchy-criterion-window-stability}
Let $U=(I,W,M,Q,T,E,H,C,P,N)$ and $U'=(I',W',M',Q',T',E',H',C',P',N')$ be accepted $\UniformCauchyCriterionUp$ carriers related by the displayed classifier. Every transported family precision request preserves the same finite index ledger, the same $\StreamNameUp$ window family up to the componentwise $\hsame$ rows, and the same shared $\CauchyModulusUp$ schedule up to the transported row. Thus every tail comparison in the transported carrier is still read through one uniform threshold and one displayed dyadic tolerance row for all family members.
\end{theorem}
\begin{proof}
Project both carriers through \autoref{def:uniform-cauchy-criterion-carrier}. The classifier has access only to $I,W,M,Q,T,E,H,C,P,N$ and to the corresponding rows of $U'$.

Transport the family index and window rows first. The rows in $H$ compare the finite family ledger and the window family componentwise, and the routes in $C$ describe how the consumer reaches those rows before any tail ledger is inspected.

Now transport the shared modulus row. Since $M$ is a single coordinate of the packet rather than an index-dependent family of moduli, the transported request still reaches one threshold schedule before reading $Q$ and $T$.

The tail ledger can therefore record only comparisons inside the transported window family and beyond the transported threshold. Repacking these projections gives the stated window stability and leaves no coordinate for a host-level pointwise criterion.
\end{proof}

\label{thm:uniform-cauchy-criterion-real-seal-handoff}

\begin{proof}

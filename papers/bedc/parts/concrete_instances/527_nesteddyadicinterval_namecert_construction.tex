\chapter{A Concrete Naming Certificate for $\NestedDyadicIntervalUp$}
\label{ch:concrete-instances-nesteddyadicinterval-namecert}

The $\NestedDyadicIntervalUp$ interface names finite decreasing dyadic interval chains positioned between the public $\DyadicIntervalUp$ endpoint packet and the regular rational and real-name layers. Moore--Kearfott--Cloud interval analysis treats nested interval enclosures as standard constructive data; here the carrier is only a finite-window BEDC packet and carries no host completeness principle.

\section{Finite decreasing dyadic chains}
\label{sec:nesteddyadicinterval-finite-decreasing-dyadic-chains}

\begin{definition}[NestedDyadicInterval finite-chain carrier]
\label{def:nesteddyadicinterval-carrier}
A $\NestedDyadicIntervalUp$ carrier is a finite $\mathsf{BHist}$ packet
$$
K=(I_0,\ldots,I_n,S,R,P,L)
$$
with the following visible rows:
\begin{enumerate}
\item each $I_i$ is a $\DyadicIntervalUp$ endpoint packet with lower and upper $\DyadicRatCoreUp$ endpoint rows and its displayed interval-order ledger;
\item $S$ is the unary $\StreamNameUp$ schedule naming the finite prefix that selects the displayed interval chain;
\item $R$ is the nested-refinement ledger: for every displayed successor pair, $R$ stores the $\Cont$ route and $\hsame$ rows witnessing that $I_{i+1}$ refines inside $I_i$ endpointwise;
\item $P$ is the $\Pkg$ provenance naming the consumed $\DyadicRatCoreUp$, $\DyadicIntervalUp$, $\StreamNameUp$, $\RegSeqRatUp$, and $\RealUp$ certificate surfaces;
\item $L$ is the ledger that hides only the unobserved tail beyond the displayed finite chain and records that every public read factors through the shown intervals, $S$, $R$, and $P$.
\end{enumerate}
The carrier predicate is $\mathsf{NestedDyadicIntervalPacket}(K)$. Its dependency list is exactly $\DyadicRatCoreUp$, $\DyadicIntervalUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, $\mathsf{BHist}$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$.
\end{definition}

\begin{definition}[NestedDyadicInterval finite-window classifier]
\label{def:nesteddyadicinterval-classifier}
For accepted carriers $K=(I_0,\ldots,I_n,S,R,P,L)$ and $K'=(I'_0,\ldots,I'_m,S',R',P',L')$, the predicate $\mathsf{NestedDyadicIntervalClassifier}(K,K')$ says that every shared scheduled window transports endpointwise and that every common window has the same refined dyadic enclosure.

Concretely, for each unary index requested by both $S$ and $S'$, the classifier supplies displayed $\hsame$ rows between the lower endpoints, displayed $\hsame$ rows between the upper endpoints, the matching $\Cont$ routes for the interval provenance, and equality of the nested-refinement ledger read on that common prefix. It does not quotient infinite chains and does not identify any completed intersection point.
\end{definition}

\begin{theorem}[NestedDyadicInterval classifier laws]
\label{thm:nesteddyadicinterval-classifier-laws}
For accepted $\NestedDyadicIntervalUp$ finite-chain carriers, the componentwise classifier on interval rows, dyadic endpoint rows, containment ledgers, scheduled window rows, $\hsame$, $\Cont$, $\mathsf{Pkg}$ provenance, and $\NameCert$ provenance is reflexive, symmetric, and transitive.

Moreover, if a packet is related to an accepted carrier by this classifier and the displayed interval, endpoint, containment, window, route, package, and certificate rows transport through the classifier data, then the related packet is again an accepted $\NestedDyadicIntervalUp$ carrier.
\end{theorem}
\leanchecked{BEDC.Derived.NestedDyadicIntervalUp.NestedDyadicIntervalClassifier\_laws}
\begin{proof}
Project an accepted carrier $K=(I_0,\ldots,I_n,S,R,P,L)$ through \autoref{def:nesteddyadicinterval-carrier}. Reflexivity is obtained rowwise from the displayed $\hsame$ reflexivity on each dyadic endpoint row, the identity $\Cont$ route on each containment ledger entry, and the unchanged schedule, provenance, and tail ledger.

For symmetry, reverse each displayed endpoint $\hsame$ row and read the common scheduled window in the opposite direction. The containment rows are not replaced by a completed interval object; they are the same finite ledger entries read through the reversed classifier comparison on the shared window.

For transitivity, compose the endpoint $\hsame$ rows and compose the displayed $\Cont$ routes in each adjacent refinement slot. The scheduled-window rows align by the common finite prefix, so the containment ledger and provenance rows compose component by component.

If a classifier-related packet is transported from an accepted carrier, the preceding rowwise transports rebuild the finite list of interval packets, the nested-refinement ledger, the schedule, the $\mathsf{Pkg}$ row, the $\NameCert$ provenance, and the hidden-tail ledger. Repacking these rows gives an accepted carrier. No completed real point, quotient interval object, open-cover compactness witness, or choice of an infinite tail is introduced.
\end{proof}

\begin{theorem}[NestedDyadicInterval window transport]
\label{thm:nesteddyadicinterval-window-transport}
Let $K=(I_0,\ldots,I_n,S,R,P,L)$ be an accepted $\NestedDyadicIntervalUp$ carrier. If each displayed interval $I_i$ is transported endpointwise through $\hsame$ rows and through the same $\Cont$ routes named by $R$ and $P$, and if the transported schedule names the same unary finite prefix, then the transported packet is again an accepted $\NestedDyadicIntervalUp$ carrier.

The transported classifier reads the same finite decreasing dyadic chain on every shared scheduled window.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:nesteddyadicinterval-carrier}. Each visible interval row is already a $\DyadicIntervalUp$ packet, so endpointwise transport supplies transported lower endpoint rows, transported upper endpoint rows, and the interval-order ledger attached to the same displayed slot.

Now transport the refinement ledger. For each adjacent pair, the $\Cont$ route and $\hsame$ rows in $R$ say that the successor interval refines inside its predecessor. Applying the same routes after endpoint transport gives the corresponding adjacent refinement row for the transported chain.

Finally transport $S$, $P$, and $L$. The schedule still exposes the same unary finite prefix, the provenance still names the same source certificate boundaries, and the ledger still hides only the tail beyond the displayed chain. Repacking these finite rows yields the accepted transported carrier and the same classifier surface on common windows.
\end{proof}
\leanchecked{BEDC.Derived.NestedDyadicIntervalUp.NestedDyadicIntervalPacket\_window\_transport}

\begin{theorem}[NestedDyadicInterval RegSeqRat handoff]
\label{thm:nesteddyadicinterval-regseqrat-handoff}
An accepted $\NestedDyadicIntervalUp$ carrier hands a $\RegSeqRatUp$ consumer only finite-window data: the unary $\StreamNameUp$ schedule, the displayed $\DyadicIntervalUp$ endpoint packets, the nested-refinement ledger connecting adjacent visible intervals, and the $\Pkg$ provenance that ties these rows to $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$.

A regular rational consumer may read the midpoint, endpoint, or width information available in the displayed dyadic enclosures, but the handoff does not certify a completed limit, a Dedekind cut, a quotient interval, or a host real interval.
\end{theorem}
\begin{proof}
Read the packet through \autoref{def:nesteddyadicinterval-carrier}. The only $\RegSeqRatUp$-facing coordinates are the scheduled finite prefix $S$, the endpoint packets $I_i$, the refinement ledger $R$, and the source provenance $P$.

For each displayed index, the $\DyadicIntervalUp$ row exposes lower and upper $\DyadicRatCoreUp$ endpoints and its interval-order ledger. The adjacent rows of $R$ show that later visible intervals refine earlier visible intervals, so a regular rational consumer receives explicit finite enclosures and their displayed nesting evidence.

The tail ledger $L$ states that nothing beyond the finite chain is public. Therefore the handoff cannot supply regularity beyond the displayed window, cannot select an intersection point, and cannot seal a host real interval.
\end{proof}

\begin{theorem}[NestedDyadicInterval NameCert obligation surface]
\label{thm:nesteddyadicinterval-namecert-obligation-surface}
The local $\NameCert_{\NestedDyadicIntervalUp}$ obligation surface consists of five finite rows:
\begin{enumerate}
\item carrier: the finite $\mathsf{BHist}$ packet $K=(I_0,\ldots,I_n,S,R,P,L)$ of \autoref{def:nesteddyadicinterval-carrier};
\item classifier: the finite-window endpoint and refinement classifier of \autoref{def:nesteddyadicinterval-classifier};
\item window stability: the endpointwise transport statement of \autoref{thm:nesteddyadicinterval-window-transport};
\item handoff: the $\RegSeqRatUp$ and $\RealUp$ finite enclosure surface of \autoref{thm:nesteddyadicinterval-regseqrat-handoff};
\item ledger exactness: the assertion that the hidden part of the name is only the unobserved tail beyond the displayed chain, while all public interval, schedule, refinement, provenance, and certificate reads are listed in the carrier.
\end{enumerate}
These rows are the full seed certificate surface for finite decreasing dyadic interval-chain names.
\end{theorem}
\begin{proof}
The carrier row fixes the visible source of every read: each interval is a $\DyadicIntervalUp$ packet, the schedule is a $\StreamNameUp$ prefix, and the provenance names exactly the dyadic rational, interval, stream, regular-rational, and real-name certificate surfaces.

The classifier row is the common-window comparison from \autoref{def:nesteddyadicinterval-classifier}. It compares only displayed endpoints and displayed refinement ledgers, so it supplies the required stability relation without creating a quotient of chains.

Window stability is \autoref{thm:nesteddyadicinterval-window-transport}. The handoff row is \autoref{thm:nesteddyadicinterval-regseqrat-handoff}. Ledger exactness follows from the coordinate $L$: every public read factors through $I_0,\ldots,I_n$, $S$, $R$, and $P$, while only the tail outside the finite chain is hidden. Thus the five rows exhaust the local naming obligation.
\end{proof}

\begin{theorem}[NestedDyadicInterval singleton chain vacuity]
\label{thm:nesteddyadicinterval-singleton-chain-vacuity}
Let $K$ be an accepted $\NestedDyadicIntervalUp$ carrier as in \autoref{def:nesteddyadicinterval-carrier}. If the displayed interval chain of $K$ has exactly one row, so that
$$
K=(I_0,S,R,P,L)
$$
with no displayed interval $I_i$ for $i>0$, then the nested-refinement ledger $R$ has no successor-pair read. Consequently the public finite-window handoff of $K$ is exactly the single $\DyadicIntervalUp$ endpoint packet $I_0$, the unary $\StreamNameUp$ schedule $S$, the provenance row $P$, and the tail ledger $L$; there is no public adjacent-refinement datum to expose.
\end{theorem}
\leanchecked{BEDC.Derived.NestedDyadicIntervalUp.NestedDyadicIntervalPacket\_singleton\_chain\_vacuity}
\begin{proof}
By \autoref{def:nesteddyadicinterval-carrier}, the only entries recorded in the nested-refinement ledger $R$ are the rows attached to displayed successor pairs $I_{i+1}$ inside $I_i$. A carrier whose displayed chain consists only of $I_0$ has no displayed source index $i$ for which both $I_i$ and $I_{i+1}$ occur in the chain. Hence the indexing set of successor-pair reads is empty, and every requested read of $R$ over a displayed adjacent pair is vacuous.

The remaining public coordinates are read directly from the same carrier definition. The visible interval coordinate is the single endpoint packet $I_0$; the schedule coordinate is $S$; the provenance coordinate is $P$; and the ledger coordinate is $L$, which hides only the unobserved tail beyond the displayed finite chain and records that public reads factor through the shown data. Since $R$ contributes no displayed successor-pair row in the singleton case, the handoff surface described in \autoref{thm:nesteddyadicinterval-regseqrat-handoff} reduces exactly to $I_0$, $S$, $P$, and $L$ together with the assertion that the adjacent-refinement component is empty.
\end{proof}
\leanchecked{BEDC.Derived.NestedDyadicIntervalUp.NestedDyadicIntervalPacket\_empty\_refinement\_source}

\begin{theorem}[NestedDyadicInterval prefix truncation stability]
\label{thm:nesteddyadicinterval-prefix-truncation-stability}
Let $K=(I_0,\ldots,I_n,S,R,P,L)$ be an accepted $\NestedDyadicIntervalUp$ carrier as in \autoref{def:nesteddyadicinterval-carrier}. For any displayed prefix $I_0,\ldots,I_k$ with $k\leq n$, the restricted packet
$$
K_{\leq k}=(I_0,\ldots,I_k,S_{\leq k},R_{\leq k},P,L_{\leq k})
$$
is again an accepted $\NestedDyadicIntervalUp$ carrier, where $S_{\leq k}$ requests only the prefix rows, $R_{\leq k}$ contains exactly the successor-pair rows with source index below $k$, and $L_{\leq k}$ hides the unrequested suffix together with the original tail. The classifier on $K_{\leq k}$ is the restriction of $\mathsf{NestedDyadicIntervalClassifier}$ to the displayed prefix window, and its $\RegSeqRatUp$ handoff is the corresponding restriction of \autoref{thm:nesteddyadicinterval-regseqrat-handoff}.
\end{theorem}
\leanchecked{BEDC.Derived.NestedDyadicIntervalUp.NestedDyadicIntervalPacket\_prefix\_truncation\_stability}
\begin{proof}
Project the accepted packet through \autoref{def:nesteddyadicinterval-carrier}. The interval rows $I_0,\ldots,I_k$ are already displayed $\DyadicIntervalUp$ endpoint packets, so no endpoint object is reconstructed.

Restrict the schedule to the unary prefix positions at most $k$. Restrict the refinement ledger by deleting only those successor rows whose source interval is outside the requested prefix; every remaining row still carries the same $\Cont$, $\hsame$, $\mathsf{Pkg}$, and $\NameCert$ provenance as in $K$. The endpoint rows on the smaller window are the transported endpoint rows supplied by \autoref{thm:nesteddyadicinterval-window-transport}.

Finally repack the shared provenance row $P$ with the ledger $L_{\leq k}$ that records both the original hidden tail and the now-unrequested suffix $I_{k+1},\ldots,I_n$. Since every public read of the truncated packet is a public read already admitted by $K$, the carrier predicate holds. The classifier and the $\RegSeqRatUp$ handoff are exactly the old classifier and handoff rows read on the smaller scheduled window; no completed point, quotient interval, hidden tail, or host real interval is introduced.
\end{proof}

\begin{theorem}[NestedDyadicInterval adjacent refinement coverage]
\label{thm:nesteddyadicinterval-adjacent-refinement-coverage}
Let $K=(I_0,\ldots,I_n,S,R,P,L)$ be an accepted $\NestedDyadicIntervalUp$ carrier. Every public read of the refinement ledger $R$ is either empty when the displayed chain is the singleton chain, or is attached to a displayed adjacent pair $I_{i+1}$ inside $I_i$ for some $i<n$. In the nonempty case, the read exposes exactly the pair's $\Cont$ route, endpointwise $\hsame$ rows, $\mathsf{Pkg}$ source provenance, and $\NameCert_{\NestedDyadicIntervalUp}$ provenance recorded in the carrier.
\end{theorem}
\leanchecked{BEDC.Derived.NestedDyadicIntervalUp.NestedDyadicIntervalPacket\_adjacent\_refinement\_coverage}
\begin{proof}
If the displayed chain has one interval row, \autoref{thm:nesteddyadicinterval-singleton-chain-vacuity} says that there is no successor-pair index. Hence $R$ contributes no public adjacent-refinement datum in that case.

Otherwise read $R$ through \autoref{def:nesteddyadicinterval-carrier}. The carrier definition assigns refinement-ledger rows only to displayed successor pairs, and each such row is indexed by a visible pair $I_{i+1}$ inside $I_i$.

The same projection lists the row contents: the continuation route for the containment read, the endpointwise $\hsame$ witnesses, the package provenance naming the consumed dyadic-rational, interval, stream, regular-rational, and real-name certificate surfaces, and the local naming certificate provenance. There is no separate ledger coordinate from which a hidden refinement row could be read.
\end{proof}

\begin{theorem}[NestedDyadicInterval RegSeqRat window exhaustion]
\label{thm:nesteddyadicinterval-regseqrat-window-exhaustion}
A $\RegSeqRatUp$ consumer of an accepted $\NestedDyadicIntervalUp$ packet can read only prefix-truncated endpoint packets, adjacent-refinement rows, the finite $\StreamNameUp$ schedule, and source provenance from $\DyadicRatCoreUp$, $\DyadicIntervalUp$, $\RegSeqRatUp$, $\RealUp$, $\mathsf{BHist}$, $\hsame$, $\Cont$, $\mathsf{Pkg}$, and $\NameCert$. \begin{proof}

Apply \autoref{thm:nesteddyadicinterval-prefix-truncation-stability} to any scheduled finite window requested by the consumer. The requested endpoint data is therefore a prefix-truncated packet whose classifier is the restriction of the accepted packet classifier.

Apply \autoref{thm:nesteddyadicinterval-adjacent-refinement-coverage} to the ledger reads inside that prefix. Every refinement read is either empty in the singleton case or attached to a displayed adjacent pair with its recorded $\Cont$, $\hsame$, $\mathsf{Pkg}$, and $\NameCert$ provenance.

The remaining coordinates are the finite $\StreamNameUp$ schedule and the same source provenance rows. Since the carrier ledger hides only unrequested suffix and tail data, no completed intersection point, hidden tail, quotient interval, or host-completeness witness can be projected by the regular rational consumer.
\end{proof}

\begin{theorem}[NestedDyadicInterval shared-prefix handoff]
\label{thm:nesteddyadicinterval-shared-prefix-handoff}
Let $K=(I_0,\ldots,I_n,S,R,P,L)$ and
$K'=(I'_0,\ldots,I'_{n'},S',R',P',L')$ be accepted
$\NestedDyadicIntervalUp$ packets related by the finite-window classifier. If
their schedules share a displayed unary prefix, then any $\RegSeqRatUp$ or
$\RealUp$ consumer read on that prefix factors through the corresponding
prefix-truncated endpoint packets, adjacent-refinement rows, $\hsame$ rows,
$\Cont$ routes, $\mathsf{Pkg}$ provenance, and $\NameCert$ provenance on both
sides.

The handoff is determined by the displayed finite prefix alone. It does not
export a completed intersection point, hidden infinite tail, quotient interval,
host real interval, completeness witness, open-cover compactness datum, or
standard bridge row.
\end{theorem}
\begin{proof}
Start with the classifier on $K$ and $K'$. By
\autoref{thm:nesteddyadicinterval-window-transport}, a shared unary prefix
transports only the displayed endpoint packets, schedules, refinement rows, and
provenance coordinates.

Apply \autoref{thm:nesteddyadicinterval-prefix-truncation-stability} on both
sides. The requested consumer window is therefore represented by
prefix-truncated accepted packets, and the classifier restricts to the same
displayed prefix rather than to an unobserved tail.

For each adjacent-refinement read inside the prefix, use
\autoref{thm:nesteddyadicinterval-adjacent-refinement-coverage}. The read is
empty in the singleton case, or else attached to one displayed adjacent pair
with its recorded $\Cont$, $\hsame$, $\mathsf{Pkg}$, and $\NameCert$ rows.

Finally apply
\autoref{thm:nesteddyadicinterval-regseqrat-window-exhaustion}. It exhausts
the $\RegSeqRatUp$ and $\RealUp$ consumer boundary by prefix endpoint packets,
adjacent-refinement rows, finite schedules, and source provenance. Since the
carrier ledger hides only data outside the displayed prefix, the shared-prefix
read factors through the corresponding finite rows on both accepted packets.
\end{proof}

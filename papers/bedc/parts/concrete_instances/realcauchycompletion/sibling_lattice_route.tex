\begin{theorem}[RealCauchyCompletion sibling lattice route]
\label{thm:real-cauchy-completion-sibling-lattice-route}
Every accepted $\RealCauchyCompletionUp$ public seal route factors through the already displayed sibling surfaces \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, \autoref{ch:concrete-instances-dyadicratcore-namecert}, and \autoref{ch:concrete-instances-real-namecert}, with \autoref{ch:concrete-instances-regularcauchyname-namecert} supplying the regular-family input route.

Equivalently, a public $\RealCauchyCompletionUp$ read is a finite five-sibling NameCert factorization through the regular-family input, stream window, regular-rational readback, dyadic observation ledger, and exported $\RealUp$ seal. It exports no quotient stream, no selected host limit, no host real equality, no countable-choice row, and no ambient completeness coordinate.
\end{theorem}
\begin{proof}
Start with an accepted $\RealCauchyCompletionUp$ carrier. The diagonal handoff of \autoref{thm:real-cauchy-completion-diagonal-handoff} routes each precision request through the modulus row, the diagonal packet, one finite $\StreamNameUp$ window, one $\RegSeqRatUp$ readback row, and the displayed $\DyadicRatCoreUp$ observation ledger.

Next apply the stream-regseq-tail seal factorization, \autoref{thm:real-cauchy-completion-stream-regseq-tail-seal-factorization}. It keeps the same stream window, regular-rational readback, dyadic ledger, diagonal packet, and $\RealUp$ seal on one completion-facing route, so the sibling surfaces are not optional decorations around the carrier.

The regular-family side enters through \autoref{ch:concrete-instances-regularcauchyname-namecert}. The terminal handoff of \autoref{thm:real-cauchy-completion-regularcauchyname-terminal-handoff} identifies that input as the finite RegularCauchyName ledger feeding the same stream schedule, dyadic-radius row, regular readback, diagonal packet, and exported seal.

Finally use the real seal readback scope of \autoref{thm:real-cauchy-completion-real-seal-readback-scope} and the public real-name export of \autoref{thm:real-cauchy-completion-public-real-name-export}. These rows expose the consumer-facing endpoint only as the displayed $\RealUp$ seal over $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ data, and \autoref{thm:real-cauchy-completion-non-escape-boundary} excludes quotient streams, selected host limits, host real equality, countable choice, and ambient completeness from the public route.
\end{proof}

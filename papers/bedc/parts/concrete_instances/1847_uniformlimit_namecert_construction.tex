\chapter{A Concrete Naming Certificate for $\UniformLimitUp$}
\label{ch:concrete-instances-uniformlimit-namecert}
\origin{human}

The $\UniformLimitUp$ interface names the constructive-analysis packet for a uniformly convergent family of displayed real-valued windows. Following Bishop--Bridges constructive analysis, the limiting object is read through a uniform tail ledger, a regular rational handoff, and a visible $\CauchyLimitSealUp$ row. The packet does not select a host function limit, quotient a function space, or appeal to ambient completeness of $\RealUp$.

\begin{definition}[UniformLimit carrier]
\label{def:uniform-limit-carrier}
A $\UniformLimitUp$ carrier is a finite $\BHist$ packet
$$
U=(F,M,T,R,S,E,H,C,P,N)
$$
with the following displayed coordinates:
\begin{enumerate}
\item $F$ is the displayed function/window family, read only through finite $\StreamNameUp$ windows and $\RealUp$ endpoint rows;
\item $M$ is the shared modulus ledger assigning each requested tolerance to one family-wide window threshold;
\item $T$ is the $\UniformCauchyCriterionUp$ tail ledger that records the accepted comparisons beyond the threshold selected by $M$;
\item $R$ is the $\RegSeqRatUp$ handoff row obtained from the same windows and tail ledger;
\item $S$ is the $\CauchyLimitSealUp$ seal row attached to the regular handoff;
\item $E$ is the exposed $\RealUp$ endpoint row read after the seal row;
\item $H$ stores componentwise $\hsame$ transports for the function/window family, modulus ledger, uniform tail ledger, regular handoff, limit seal, endpoint row, package provenance, and local naming row;
\item $C$ stores the displayed $\Cont$ routes for reading the windows, applying the shared threshold, consuming the uniform tail ledger, reaching the regular handoff, and exposing the seal endpoint;
\item $P$ is the $\Pkg$ provenance over $\StreamNameUp$, $\UniformCauchyCriterionUp$, $\RegSeqRatUp$, $\CauchyLimitSealUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$;
\item $N$ is the local $\NameCert_{\UniformLimitUp}$ row.
\end{enumerate}
The classifier compares accepted carriers only through these rows, their displayed transports, their continuation routes, package provenance, and local naming provenance.
\end{definition}

\begin{theorem}[UniformLimit NameCert obligations]
\label{thm:uniform-limit-namecert-obligations}
For the carrier of \autoref{def:uniform-limit-carrier}, the local $\NameCert_{\UniformLimitUp}$ surface consists exactly of five finite obligations:
\begin{enumerate}
\item carrier admission for the packet $U=(F,M,T,R,S,E,H,C,P,N)$;
\item classifier transport by componentwise $\hsame$ over the function/window family, modulus ledger, tail ledger, regular handoff, seal row, and endpoint row;
\item uniform-tail exactness, requiring each endpoint-facing read to pass through the same threshold row $M$ and the $\UniformCauchyCriterionUp$ ledger $T$;
\item regular-seal routing, requiring every limit-facing read to pass from $F,M,T$ through $R$ and $S$ before reaching $E$;
\item non-escape from pointwise convergence without a uniform ledger, open-cover compactness, selected host limits, quotient function spaces, and $\RealUp$ completeness.
\end{enumerate}
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:uniform-limit-carrier}. The displayed coordinates name the family windows, shared modulus, uniform tail ledger, regular handoff, seal row, endpoint row, transports, continuations, package provenance, and local certificate.

Classifier transport is componentwise. The rows in $H$ move each named coordinate, and the routes in $C$ replay the same consumer order, so a transported packet remains inside the displayed carrier surface.

For exactness, inspect a requested tolerance. The consumer route first reads $M$, then reads the corresponding comparisons in $T$, and only then reaches the regular handoff and seal rows. The packet inventory contains no coordinate for pointwise-only convergence, open-cover compactness, selected host limits, quotient function spaces, or real completeness.
\end{proof}

\begin{theorem}[UniformLimit seal factorization]
\label{thm:uniform-limit-seal-factorization}
Every $\RealUp$ endpoint read exported by an accepted $\UniformLimitUp$ carrier factors through the finite route
$$
F,M,T \longmapsto R \longmapsto S \longmapsto E.
$$
Thus the endpoint is exposed only after the displayed function/window family has been read through the shared threshold ledger, the $\UniformCauchyCriterionUp$ tail row, the $\RegSeqRatUp$ handoff, and the $\CauchyLimitSealUp$ row, together with the visible $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ provenance.
\end{theorem}
\begin{proof}
Begin with an endpoint-facing consumer request. By the carrier definition, the only $\RealUp$ endpoint coordinate is $E$, and the continuation route to $E$ passes through the named rows of the same finite packet.

Read the family windows through $F$ and the shared threshold through $M$. The uniform comparisons are then supplied by $T$, so the request has not used an indexwise or pointwise-only convergence witness.

Next consume the regular rational handoff $R$ and the limit seal $S$. These rows mediate the passage from the uniform tail ledger to the endpoint row, while $H,C,P,N$ record the transport, route, package, and naming provenance. Hence every endpoint read has the stated factorization.
\end{proof}

\begin{theorem}[UniformLimit non-escape boundary]
\label{thm:uniform-limit-non-escape-boundary}
An accepted $\UniformLimitUp$ carrier exports only the displayed function/window family, shared modulus ledger, $\UniformCauchyCriterionUp$ tail ledger, $\RegSeqRatUp$ handoff, $\CauchyLimitSealUp$ seal row, $\RealUp$ endpoint, and the visible $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows of \autoref{def:uniform-limit-carrier}. It exports no pointwise convergence object lacking the uniform ledger, no open-cover compactness principle, no selected host limit, no quotient function space, and no completeness theorem for $\RealUp$.
\end{theorem}
\begin{proof}
Inspect the packet coordinates. Every listed row is a finite display, transport, continuation, provenance, or local naming row, and the only endpoint coordinate is the seal-facing row $E$.

Apply \autoref{thm:uniform-limit-namecert-obligations}. Uniform-tail exactness keeps each read inside $F,M,T$, and regular-seal routing forces the route through $R$ and $S$ before $E$ is exposed.

The excluded analytic objects would require additional carrier coordinates or a different consumer route. Since neither is present, the public surface is exhausted by the finite BEDC rows listed in the statement.
\end{proof}

\begin{theorem}[UniformLimit finite endpoint certificate]
\label{thm:uniform-limit-finite-endpoint-certificate}
The carrier, local $\NameCert$ obligations, seal factorization, and non-escape boundary assemble a finite endpoint certificate for $\UniformLimitUp$ over $\StreamNameUp$, $\UniformCauchyCriterionUp$, $\RegSeqRatUp$, $\CauchyLimitSealUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$. Every downstream completion or real-consumer read is confined to that packet and its factorization route.
\end{theorem}
\begin{proof}
Start with the carrier of \autoref{def:uniform-limit-carrier}. It supplies exactly the function/window family, shared modulus, uniform tail ledger, regular handoff, seal row, endpoint row, transports, continuations, package provenance, and local certificate.

Use \autoref{thm:uniform-limit-namecert-obligations} to obtain carrier admission, classifier transport, uniform-tail exactness, regular-seal routing, and the non-escape clauses. Then use \autoref{thm:uniform-limit-seal-factorization} to identify the endpoint route.

Finally apply \autoref{thm:uniform-limit-non-escape-boundary}. The route has no coordinate outside the listed BEDC rows, so the finite endpoint certificate is the whole downstream consumer surface.
\end{proof}

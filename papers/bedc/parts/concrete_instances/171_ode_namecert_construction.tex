\chapter{A Concrete Naming Certificate for $\OdeUp$}
\label{ch:concrete-instances-ode-namecert}

The ordinary-differential-equation interface $\OdeUp$ packages a system $\dot x = F(t, x)$ together with the local existence and uniqueness data on Lipschitz vector fields. It depends on $\NameCert_{\DerivativeUp}$ (\autoref{ch:concrete-instances-derivative-namecert}) and $\NameCert_{\BanachUp}$ (\autoref{ch:concrete-instances-banach-namecert}).

\section{Carrier definition}
\label{sec:ode-carrier}

\begin{definition}[OdeUp BHist local-flow source]
\label{def:ode-bhist-local-flow-source}
An $\OdeUp$ BHist local-flow source is a time-state carrier over the
$\BanachUp$ carrier supplied by $\NameCert_{\BanachUp}$. Its points are pairs
of histories $(t,x):\mathsf{BHist}\times\mathsf{BHist}$ whose time component
is carried by the scalar surface used by $\NameCert_{\DerivativeUp}$ and whose
state component is carried by the Banach carrier. The vector-field row is a
displayed operation
$$
F:\mathsf{BHist}\to\mathsf{BHist}\to\mathsf{BHist}
$$
whose value $F(t,x)$ is again a Banach-carried state history after transport
along the time and state classifiers.

A local-flow row consists of an initial time cell $t_0$, an initial state cell
$x_0$, an endpoint time cell $t_1$, an endpoint state cell $x_1$, a
vector-field evaluation ledger $v$, a Picard-step history $p$, and a limit
witness $\ell$ such that the step and limit are linked by displayed
continuations
$$
\Cont(t_0,x_0,p),\qquad
\Cont(p,v,\ell),\qquad
\Cont(t_1,\ell,x_1).
$$
The classifier is inherited from the Banach and derivative surfaces: two rows
are equivalent when their time cells are identified by the derivative scalar
classifier, their state cells by the Banach carrier classifier, and their
displayed ledgers by $\hsame$ transport through the same $\NameCert$ rows.
The ledger records the time cell, state cell, vector-field evaluation,
Picard-step continuation route, and limit witness. It does not assert equality
in any host function space.
\end{definition}

\section{The certificate}
\label{sec:ode-certificate}

\begin{theorem}[OdeUp root vector-field carrier stability]
\label{thm:ode-root-vector-field-carrier-stability}
For an $\OdeUp$ BHist local-flow source, if $t$ is carried by the derivative
scalar surface and $x$ is carried by the Banach state carrier, then the
displayed vector-field value $F(t,x)$ is carried by the same Banach state
carrier. This assignment is stable under classifier transport: replacing
$t$ and $x$ by $t'$ and $x'$ with the corresponding $\NameCert_{\DerivativeUp}$
and $\NameCert_{\BanachUp}$ classifier witnesses gives
$F(t,x)\hsame F(t',x')$ in the endpoint classifier recorded by $\NameCert$.
\end{theorem}
\begin{proof}
Unpack the local-flow source. The time cell is read through
$\NameCert_{\DerivativeUp}$, and the state cell is read through
$\NameCert_{\BanachUp}$. The vector-field row is part of the source data, so
its evaluation ledger records a Banach-carried output whenever both input
cells are carried.

For transport, move first along the derivative classifier on the time cell and
then along the Banach carrier classifier on the state cell. The displayed
evaluation ledger is functorial for these two classifier moves because it is
stored as a $\NameCert$ row rather than as host equality. Repacking the
transported output row gives the required Banach-carried value and its
$\hsame$ endpoint classifier.
\end{proof}

\begin{theorem}[OdeUp root Picard continuation scope]
\label{thm:ode-root-picard-continuation-scope}
In an $\OdeUp$ BHist local-flow source, Picard-step packages compose inside
the scope generated by $\NameCert_{\BanachUp}$, $\NameCert_{\DerivativeUp}$,
and $\hsame$ transport. If two adjacent Picard ledgers expose
$\Cont(t_0,x_0,p)$, $\Cont(p,v,\ell)$, and $\Cont(t_1,\ell,x_1)$ together with
a second compatible step from $x_1$ to $x_2$, then associativity of $\Cont$
builds the composite continuation route from $(t_0,x_0)$ to $(t_2,x_2)$
without adding any carrier outside those two certificate surfaces.
\end{theorem}
\begin{proof}
Read the first Picard package as a chain of displayed continuations from the
initial time-state cell to its limit witness and then to its endpoint. Read
the second package in the same way, with the endpoint state of the first row
as the carried input state of the second.

Use $\Cont$ associativity to reassociate the terminal continuation of the
first package with the initial continuation of the second package. Whenever a
shared endpoint is represented by a classifier-equal history rather than by
the same syntax, transport across $\hsame$ using the Banach and derivative
rows of $\NameCert_{\BanachUp}$ and $\NameCert_{\DerivativeUp}$.

The resulting chain is again a list of BHist continuations whose endpoints are
classified by the same $\NameCert$ surfaces. Thus the composite Picard route
remains scoped to Banach states, derivative time cells, and $\hsame$
transport; no ambient function-space carrier is introduced.
\end{proof}

\begin{theorem}[OdeUp root local-uniqueness classifier]
\label{thm:ode-root-local-uniqueness-classifier}
For an $\OdeUp$ BHist local-flow source, two local-flow witnesses with the same
initial time cell, the same initial state cell, and the same Lipschitz ledger
are classified by the same endpoint classifier. Concretely, if their
vector-field evaluation rows, Picard-step rows, and limit witnesses are
transported through the shared $\NameCert_{\DerivativeUp}$,
$\NameCert_{\BanachUp}$, $\Cont$, and $\hsame$ ledgers, then their endpoint
state histories are related by the Banach classifier recorded in $\NameCert$.
\end{theorem}
\leanchecked{BEDC.Derived.OdeUp.OdeLocalFlow\_endpoint\_classifier\_deterministic}
\begin{proof}
Unpack the two local-flow witnesses into their common initial time-state cell,
their vector-field evaluation ledgers, Picard-step histories, limit witnesses,
and endpoint histories. The shared Lipschitz ledger classifies the
vector-field rows through the derivative and Banach input classifiers, so the
two Picard steps have the same continuation data up to $\hsame$.

Transport the second Picard chain along these $\hsame$ witnesses and use
determinism of the displayed $\Cont$ result at each continuation stage. This
identifies the two limit witnesses in the classifier inherited from the
Banach carrier.

Finally transport through the endpoint continuation rows
$\Cont(t_1,\ell,x_1)$ and $\Cont(t_1,\ell',x_1')$. Since the time endpoint is
the same cell and the limit witnesses have just been classified together, the
Banach endpoint classifier relates $x_1$ and $x_1'$. This is precisely the
endpoint classifier exposed by the $\OdeUp$ source through $\NameCert$.
\end{proof}

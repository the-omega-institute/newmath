Following Tietze, Urysohn, Kelley, and Bourbaki, the $\TietzeExtensionUp$ packet records the finite BEDC route from displayed normal-space separation data to displayed continuous $\RealUp$-valued extension data. It sits above the normal-space carrier of \autoref{ch:concrete-instances-normalspace-namecert} and the separation-function carrier of \autoref{ch:concrete-instances-urysohn-lemma-namecert}, and it reads its function, codomain, and topological boundaries through \autoref{ch:concrete-instances-continuousmap-namecert}, \autoref{ch:concrete-instances-real-namecert}, \autoref{ch:concrete-instances-topology-namecert}, \autoref{ch:concrete-instances-zero-set-namecert}, and \autoref{ch:concrete-instances-cozero-set-namecert}. The packet names only a local finite extension certificate for a displayed closed-domain row and a bounded displayed function on that row; it does not select complements, invoke excluded middle, identify host functions extensionally, or appeal to ambient compactness.

\begin{definition}[Tietze extension carrier]
\label{def:tietze-extension-carrier}
A $\TietzeExtensionUp$ carrier is a finite $\BHist$ packet
$$
E=(N,D,B,U,M,G,R,H,K,P,L)
$$
with the following displayed rows:
\begin{enumerate}
\item $N$ is the $\NormalSpaceUp$ source row, exposing the ambient $\TopologyUp$ row and the finite closed-pair separation discipline used by the extension read.
\item $D$ is the closed-domain row, displayed over the topology source as a finite closed-set witness rather than as a selected complement of an open row.
\item $B$ is the bounded $\RealUp$ target-interval row for the input map on $D$, recording the two endpoint bounds and the displayed nonescape ledger for the interval.
\item $U$ is the $\UrysohnLemmaUp$ ladder row, recording the finite sequence of separation-function reads consumed to extend the bounded data stage by stage.
\item $M$ is the $\ContinuousMapUp$ row for the input map on the closed-domain surface, exposing only graph cells and continuity replay visible on $D$.
\item $G$ is the $\ContinuousMapUp$ extension-graph row, tying the ambient topology source to the extended $\RealUp$-valued graph and recording agreement with $M$ on the displayed closed-domain row.
\item $R$ is the $\RealUp$ seal boundary, certifying that every displayed extension value remains inside the bounded target interval named by $B$.
\item $H$ records componentwise $\hsame$ transport for the normal-space source, closed-domain row, bounded interval row, Urysohn ladder, input map, extension graph, Real seal, replay, provenance, and local naming rows.
\item $K$ records displayed $\Cont$ replay from a Tietze-extension request through $N,D,B,U,M,G,R,H,P,L$.
\item $P$ is the $\Pkg$ provenance over $\TietzeExtensionUp$, $\NormalSpaceUp$, $\UrysohnLemmaUp$, $\ContinuousMapUp$, $\RealUp$, $\TopologyUp$, $\ZeroSetUp$, $\CozeroSetUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $L$ is the local $\NameCert_{\TietzeExtensionUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $N,D,B,U,M,G,R,H,K,P,L$. It has no coordinate for selected complements, excluded middle, host function extensionality, arbitrary open-cover compactness, ambient Real completeness, Lean verification, or bridge checking.
\end{definition}

\begin{theorem}[Tietze extension NameCert obligations]
\label{thm:tietze-extension-namecert-obligations}
For every accepted $\TietzeExtensionUp$ carrier
$$
E=(N,D,B,U,M,G,R,H,K,P,L),
$$
the local $\NameCert_{\TietzeExtensionUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the normal-space source, closed-domain row, bounded interval row, Urysohn ladder, input map, extension graph, Real seal, transport, replay, provenance, and local naming;
\item closed-domain exposure, requiring the extension request to display $D$ over the topology source named by $N$ before any ambient extension graph can be consumed;
\item Urysohn-step route, requiring every stage of the bounded extension read to pass through the $\UrysohnLemmaUp$ ladder row $U$ rather than through selected complements or excluded middle;
\item continuous endpoint exactness, requiring $M$ and $G$ to agree on the displayed closed-domain row and requiring the agreement read to use the visible $\ContinuousMapUp$ graph cells;
\item Real boundary nonescape, requiring $R$ to preserve the bounded interval named by $B$ for all displayed extension values;
\item $\hsame$, $\Cont$, and $\Pkg$ transport, requiring componentwise transport, displayed replay, and provenance to preserve exactly the listed rows and no hidden host-function data.
\end{enumerate}
No local Tietze-extension obligation is stored outside these six rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:tietze-extension-carrier}. The accepted tuple displays exactly the normal-space source, closed-domain row, bounded interval, Urysohn ladder, input map, extension graph, Real seal, and structural transport, replay, provenance, and naming rows.

Closed-domain exposure follows from the order of the carrier. An extension request first reads $N$ and then $D$ over the same topological source; the extension graph row $G$ is unavailable until the closed-domain row has been displayed.

The Urysohn-step route is the projection to $U$. Each bounded correction step is named as a $\UrysohnLemmaUp$ ladder read, so the packet has no row from which a selected complement, excluded middle argument, or unrecorded separation theorem can be consumed.

Continuous endpoint exactness is read through $M$ and $G$. The row $M$ displays the original $\RealUp$-valued continuous graph on $D$, while $G$ displays the ambient graph and its agreement with $M$ along that closed-domain row.

Real boundary nonescape is the projection to $B$ and $R$. The interval row records the endpoint bounds, and the Real seal row records that the displayed extension graph stays inside those bounds.

Finally, transport and provenance are exhausted by $H,K,P,L$. These rows move and replay exactly the displayed carrier coordinates, so no local obligation exports host function extensionality, arbitrary open-cover compactness, ambient Real completeness, Lean verification, or bridge checking.
\end{proof}

\begin{theorem}[Tietze extension Urysohn route]
\label{thm:tietze-extension-urysohn-route}
Every extension read exported by an accepted $\TietzeExtensionUp$ packet factors through
$$
\NormalSpaceUp \longrightarrow \UrysohnLemmaUp \longrightarrow \ContinuousMapUp \longrightarrow \RealUp
$$
before the local $\NameCert_{\TietzeExtensionUp}$ row can be inspected. No extension read factors through a selected complement, excluded middle, host function extensionality, arbitrary open-cover compactness, or ambient Real completeness.
\end{theorem}
\begin{proof}
Begin with an accepted carrier $E=(N,D,B,U,M,G,R,H,K,P,L)$. Carrier admission exposes the $\NormalSpaceUp$ source $N$ and the closed-domain row $D$ before any ladder, graph, or Real-boundary row is available.

The next gate is $U$. This row is precisely the $\UrysohnLemmaUp$ ladder consumed by the extension read, so every separation-function stage is read from the displayed normal-space route and not from a selected complement or excluded-middle branch.

The function gate is $M$ and $G$. The input graph and the ambient extension graph are both $\ContinuousMapUp$ rows, and their agreement is visible only along the displayed closed-domain row.

The value gate is $B$ and $R$. These rows tie the extension graph to the bounded $\RealUp$ interval and record nonescape inside that interval.

The replay and local naming rows $H,K,P,L$ preserve this order. Since the classifier has no further coordinate, every exported extension read factors through $\NormalSpaceUp$, $\UrysohnLemmaUp$, $\ContinuousMapUp$, and $\RealUp$, with none of the excluded classical or host-level principles available.
\end{proof}

\begin{theorem}[$\BoolUp$ sibling dependency surface]
\label{thm:bool-sibling-dependency-surface}
The $\BoolUp$ finite-truth carrier is the L6 endpoint surface consumed by the adjacent finite-data routes for $\OptionUp$, $\SumUp$, $\ProdUp$, $\ListUp$, and $\CantorSpaceUp$. The $\OptionUp$ route of \autoref{ch:concrete-instances-option-namecert} reads the two endpoints as constructor discriminators, the $\SumUp$ and $\ProdUp$ routes read them as finite branch and component selectors, the $\ListUp$ route reads them as finite traversal endpoints, and the $\CantorSpaceUp$ route reads them as binary coordinate endpoints. Each consumer receives only the displayed Boolean endpoint rows, endpoint classifier, $\hsame$ transport, $\Cont$ replay, $\Pkg$ provenance, and local $\NameCert$ row.

No sibling consumer receives host booleans, external equality reflection, hidden enumeration, product-pair equality, option payload data, list-recursion data, or Cantor-space compactness from the $\BoolUp$ packet.
\end{theorem}
\begin{proof}
Start with the finite-truth route already exposed by the $\BoolUp$ carrier and its local sibling lattice. The readable rows are the two endpoint histories, the endpoint classifier, componentwise transport, displayed replay, provenance, and local naming.

The option route consumes the endpoints only as absent and present branch discriminators, as in \autoref{ch:concrete-instances-option-namecert}. The sum, product, and list routes consume the same finite distinction as branch, component, and traversal data, while their payload, pair, and recursion certificates remain owned by their own packets.

For the Cantor-space endpoint route, the Boolean row supplies only the binary coordinate endpoint. Compactness, product topology, and infinite-choice data are not coordinates of the Boolean carrier, so they cannot be exported through this dependency surface.
\end{proof}

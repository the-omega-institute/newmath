\begin{definition}[Uniform Cauchy window exhaustion carrier]
\label{def:uniform-cauchy-window-exhaustion-carrier}

$$

with the following displayed rows:
\begin{enumerate}
\item $S$ is the $\StreamNameUp$ row of finite stream windows inspected by the exhaustion request.
\item $T$ is the uniform threshold row, assigning each displayed tolerance request a single threshold that governs all later windows named by $S$.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger, recording the dyadic radii, dominance checks, and comparison budgets used by $T$.
\item $R$ is the $\RegSeqRatUp$ handoff row assembled from the window observations in $S$ after the threshold and dyadic rows have been displayed.
\item $E$ is the $\RealUp$ seal row reached only after $S,T,D,R$ have been replayed.
\item $H$ records componentwise $\hsame$ transport for the stream-window, threshold, dyadic-ledger, regular-readback, real-seal, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay of the finite route

\item $P$ is the $\Pkg$ provenance over $\UniformCauchyWindowExhaustionUp$, $\UniformCauchyCriterionUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.

The classifier compares accepted packets only through $S,T,D,R,E,H,C,P,N$. It has no coordinate for quotient stream equality, a selected limit, a countable-choice selector, host real equality, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Uniform Cauchy window exhaustion NameCert obligations]
\label{thm:uniform-cauchy-window-exhaustion}
For every accepted $\UniformCauchyWindowExhaustionUp$ carrier $U=(S,T,D,R,E,H,C,P,N)$, the local $\NameCert_{\UniformCauchyWindowExhaustionUp}$ surface consists exactly of the following obligation rows:
\begin{enumerate}
\item carrier admission for the displayed stream-window row, uniform threshold row, dyadic tolerance ledger, regular rational handoff, real seal, transport, replay, provenance, and local naming row;
\item finite-window exposure, requiring every exhaustion read to inspect the $\StreamNameUp$ row $S$ before it can consult $T,D,R,E$;
\item uniform-threshold exactness, requiring the threshold row $T$ to govern all later windows named by $S$ for the same displayed tolerance request;
\item dyadic-to-regular handoff, requiring the $\DyadicRatCoreUp$ ledger $D$ to feed the $\RegSeqRatUp$ row $R$ before the $\RealUp$ seal $E$ is visible;
\item ledger refusal, excluding quotient streams, selected limits, countable choice, host real equality, and ambient real completeness as certificate data.
\end{enumerate}
No NameCert obligation is stored outside those rows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:uniform-cauchy-window-exhaustion-carrier}. The accepted inventory is exactly $S,T,D,R,E,H,C,P,N$, so every local obligation must be read from those rows.

The finite-window and uniform-threshold rows are upstream. A consumer cannot read a regular rational handoff until the displayed stream windows $S$, the threshold row $T$, and the dyadic tolerance ledger $D$ have supplied the requested tail comparison.

The real seal is downstream of the regular handoff. The row $R$ records the $\RegSeqRatUp$ readback assembled from the same windows and budgets, and $E$ seals only that displayed readback.

Transport by $H$, replay by $C$, provenance by $P$, and local naming by $N$ preserve the same route. Since quotient stream equality, selected limits, countable choice, host real equality, and ambient real completeness have no carrier coordinate, the five listed rows exhaust the local certificate surface.
\end{proof}

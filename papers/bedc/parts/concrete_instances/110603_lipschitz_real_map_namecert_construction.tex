\begin{definition}[Lipschitz real-map carrier]\label{def:lipschitz-real-map-carrier}

$$

\begin{enumerate}
\item $S$ is the source $\RealUp$ seal row naming the accepted real input surface through displayed $\StreamNameUp$ and $\RegSeqRatUp$ data.
\item $T$ is the target $\RealUp$ seal row naming the accepted real output surface through the same kind of finite readback.
\item $K$ is the $\DyadicRatCoreUp$ Lipschitz constant row, including the positive dyadic bound and the displayed tolerance ledger used by every modulus request.
\item $W$ is the finite $\StreamNameUp$ window row for the source and target names inspected by the request.
\item $R$ is the $\RegSeqRatUp$ readback row translating the displayed source and target windows into rational comparison data.
\item $M$ is the modulus-inequality row, recording that every accepted finite pair of source reads with displayed apartness or distance bound is sent to a target read whose displayed difference is bounded by the dyadic constant in $K$ times the source bound, up to the requested displayed tolerance.
\item $H$ records componentwise $\hsame$ transport for source seal, target seal, dyadic constant, finite windows, regular-rational readback, modulus inequality, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a finite map request through $S,T,K,W,R,M,H,P,N$.

The classifier compares accepted carriers only through $S,T,K,W,R,M,H,C,P,N$. It has no coordinate for host function equality, quotient equality of real presentations, selected Cauchy representatives, global uniform continuity, or ambient completeness of $\RealUp$.
\end{definition}

\begin{theorem}[Lipschitz real-map NameCert obligations]
\label{thm:lipschitz-real-map-namecert-obligations}
For every accepted $\LipschitzRealMapUp$ carrier
$$
F=(S,T,K,W,R,M,H,C,P,N),
$$
the local $\NameCert_{\LipschitzRealMapUp}$ surface consists exactly of carrier admission for the source $\RealUp$ seal, target $\RealUp$ seal, dyadic Lipschitz constant, finite StreamName windows, RegSeqRat readback, modulus inequality, componentwise transport, replay, provenance, and local naming rows.

The obligation surface requires finite source and target visibility through $S$ and $T$, dyadic bound visibility through $K$, finite window visibility through $W$, rational readback through $R$, modulus-inequality exposure through $M$, transport and replay stability through $H$ and $C$, provenance through $P$, and local naming through $N$. It explicitly refuses host function extensionality, quotient equality of real presentations, selected representatives, global uniform-continuity principles, and ambient completeness of $\RealUp$.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:lipschitz-real-map-carrier}. The source and target real seals are the only real-name endpoints, and both are read through finite StreamName and RegSeqRat windows rather than through host real equality.

The Lipschitz datum is the dyadic row $K$. A map request can inspect only the finite windows $W$ and their rational readback $R$ before the modulus row $M$ exposes the displayed inequality for the requested tolerance.

The structural rows do not add analytic content. The transport row $H$ preserves the already displayed source, target, constant, window, readback, and modulus rows componentwise, while $C$ replays the same route and $P,N$ record provenance and local naming.

Since no carrier coordinate stores host function equality, quotient real equality, selected representatives, global uniform continuity, or ambient completeness, no local NameCert obligation can require those objects.
\end{proof}

\begin{theorem}[Lipschitz real-map modulus stability]

The stability claim is confined to the finite rows $S,T,K,W,R,M,H,C,P,N$. It does not identify host functions, quotient real presentations, selected Cauchy names, or ambient complete metric structures.
\end{theorem}
\begin{proof}
Start with a displayed modulus request for $F_0$. Its source and target real seals are read only through the finite StreamName windows and RegSeqRat readback rows named by $S,T,W,R$.

Use the componentwise $\hsame$ hypotheses. They transport the dyadic constant row $K$ and the finite window/readback rows into the corresponding rows of $F_1$, so the same displayed source tolerance and target tolerance are available after transport.

The modulus row $M$ is a row of the carrier rather than a host theorem about all representatives. Therefore the transported request reads the $F_1$ modulus row with the same displayed dyadic bound and the transported finite readback data.

Finally $C,P,N$ replay, source, and name the transported route. No quotient equality, host function equality, selected representative, or ambient completeness principle is introduced by this transport.

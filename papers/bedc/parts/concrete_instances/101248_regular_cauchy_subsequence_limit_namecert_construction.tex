\begin{theorem}[Regular-Cauchy subsequence limit NameCert obligations]
\label{thm:regular-cauchy-subsequence-limit-namecert-obligations}
For every accepted $\RegularCauchySubsequenceLimitUp$ carrier
$$
T=(S,L,W,R,D,E,H,C,P,N),
$$
the local $\NameCert_{\RegularCauchySubsequenceLimitUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the regular-Cauchy subsequence row, subsequence-limit consumer boundary, selected stream windows, regular rational readback, dyadic tolerance ledger, terminal real seal, componentwise transport, displayed replay, provenance, and local naming rows;
\item selected-window alignment, requiring every window read by $L$ and $R$ to be one of the finite windows named by $W$ and admitted by the regular-Cauchy subsequence row $S$;
\item tolerance preservation, requiring the dyadic ledger $D$ to be the same ledger consumed by $S$, $W$, $R$, and the limit-facing row $L$;
\item Real-seal handoff, requiring the seal row $E$ to be exposed only after the regular-Cauchy subsequence row, limit boundary, selected windows, regular rational readback, and tolerance ledger have been displayed;
\item non-escape, requiring $H,C,P,N$ to transport, replay, source, and name no arbitrary subsequence choice, quotient stream equality, host sequence equality, ambient Real-completeness datum, Lean credential, or bridge credential.
\end{enumerate}
No local obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:regular-cauchy-subsequence-limit-carrier}. The visible analytic rows are exactly the regular-Cauchy subsequence row $S$, the subsequence-limit boundary $L$, selected windows $W$, regular rational readback $R$, dyadic ledger $D$, and terminal seal $E$.

Selected-window alignment is read from the triple $(S,L,W)$. The row $S$ admits the finite subsequence windows, while $L$ can consume only the windows displayed by the same ledger $W$.

Tolerance preservation is the projection to $D$. The regular-Cauchy row, selected-window row, regular rational readback, and limit boundary all name the same dyadic tolerance ledger, so no second budget can alter the limit read.

The Real-seal handoff follows the row order $S,L,W,R,D,E$. A Real-facing consumer reaches $E$ only after the selected subsequence data, finite windows, regular rational readback, and dyadic tolerance ledger are already visible.

The remaining rows are structural. Componentwise transport, continuation replay, provenance, and local naming preserve only the displayed packet rows, so the excluded host choices, equality principles, completeness claim, Lean credential, and bridge credential cannot enter the local surface.
\end{proof}

\begin{theorem}[Regular-Cauchy subsequence limit Real seal boundary]

$$

$$
Consequently the terminal seal is reached only through the regular-Cauchy subsequence row, the existing subsequence-limit boundary, selected stream windows, regular rational readback, and the shared dyadic tolerance ledger.
\end{theorem}
\begin{proof}
Begin with a $\RealUp$-facing read of the accepted packet. The carrier places the terminal seal $E$ downstream of $S,L,W,R,D$, so the read must first inspect the regular-Cauchy subsequence row and the existing subsequence-limit consumer boundary.

The selected windows and readback rows are then forced by \autoref{thm:regular-cauchy-subsequence-limit-namecert-obligations}. Every window consumed by the limit boundary is recorded in $W$, and every regular rational observation in $R$ uses the same selected-window ledger.

The dyadic tolerance ledger is shared by all analytic rows. Since $D$ is the only displayed tolerance coordinate, the Real-facing read cannot switch to a detached modulus, quotient representative, or host sequence equality before reaching the seal.

Finally $H,C,P,N$ transport, replay, source, and name the same finite route. Thus every Real-facing read factors through the listed rows and exports no arbitrary subsequence choice, quotient stream equality, host sequence equality, ambient completeness theorem, Lean credential, or bridge credential.
\end{proof}

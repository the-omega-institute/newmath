\item $A$ is the located evidence row. It records the displayed positive apartness witness when the request separates the endpoints, or the displayed equality-facing refusal row when the request reaches only an overlap tolerance.

\begin{theorem}[Located real comparison modulus obligations]
\label{thm:located-real-comparison-modulus-obligations}
For every accepted $\LocatedRealComparisonModulusUp$ carrier
$$
M=(Q,A,D,B,R,E,H,C,P,N),
$$
the local $\NameCert_{\LocatedRealComparisonModulusUp}$ surface consists exactly of comparison-request admission, located apartness-or-equality evidence, dyadic tolerance scheduling, $\CauchyModulusBridgeUp$ routing, $\RegSeqRatUp$ readback, $\RealUp$ sealing, componentwise $\hsame$ transport, displayed $\Cont$ replay, $\Pkg$ provenance, and local naming. Any consumer of the comparison schedule must read those rows in the displayed order and receives no trichotomy split, quotient real equality, hidden selector, countable-choice schedule, host comparison oracle, or ambient $\RealUp$ completeness theorem.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:located-real-comparison-modulus-carrier}. The comparison request is $Q$, and the only evidence row attached to it is $A$.

The schedule is then forced through $D$ and $B$. The dyadic row $D$ records the finite tolerances to be inspected, while the $\CauchyModulusBridgeUp$ row $B$ turns those tolerance requests into explicit tail thresholds before any rational readback is exposed.

The terminal read is downstream. The regular rational row $R$ reads only the displayed tails opened by $B$, and the $\RealUp$ seal $E$ is available only after that readback.

Finally $H,C,P,N$ transport, replay, source, and name the same rows. Since the carrier has no coordinate for the excluded host-level or choice-level objects, the listed rows exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[Located real comparison modulus nontrichotomy boundary]
\label{thm:located-real-comparison-modulus-nontrichotomy-boundary}
Every accepted $\LocatedRealComparisonModulusUp$ packet exports a positive schedule for comparison reads without exporting trichotomy. If the evidence row $A$ records positive apartness, a downstream consumer may read that apartness only through $Q,D,B,R,E$; if $A$ records only an equality-facing overlap tolerance, the consumer receives the displayed tolerance route and no forced left-or-right order verdict.
\end{theorem}
\begin{proof}
Read the evidence row $A$ together with the comparison request $Q$. The carrier does not contain a three-way order coordinate, so the two possible reads are positive displayed apartness or finite overlap tolerance.

For the positive branch, the dyadic tolerance row $D$ and bridge row $B$ determine which tails are inspected before $R$ and $E$ can be consumed. The apartness read is therefore a finite schedule read, not a global order split.

For the overlap branch, the same schedule exposes only the inspected tolerance data. Since no trichotomy coordinate or hidden selector appears in $M=(Q,A,D,B,R,E,H,C,P,N)$, the packet refuses any forced left-or-right verdict not displayed by the rows.
\end{proof}

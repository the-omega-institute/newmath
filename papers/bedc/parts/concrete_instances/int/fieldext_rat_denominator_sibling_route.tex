\begin{theorem}[IntUp FieldExt and Rat denominator sibling route]
\label{thm:int-fieldext-rat-denominator-sibling-route}
Every $\FieldExtUp$ base-endpoint read that consumes $\RatUp$ denominator data ultimately depends on the $\IntUp$ signed-pair denominator route. The consumer in \autoref{ch:concrete-instances-fieldext-namecert} first reads the rational denominator surface of \autoref{ch:concrete-instances-rat-namecert}; that surface has already consumed the displayed $\IntUp$ sign row, unary magnitude row, balancing route, and positive denominator witness before any field-extension endpoint can be named.
\end{theorem}
\begin{proof}
Start with a $\FieldExtUp$ base-endpoint read whose rational endpoint data includes a denominator row. The field-extension carrier can read that denominator only through the displayed rational surface, so the first dependency is the $\RatUp$ NameCert chapter.

The rational surface is not independent of $\IntUp$. Its numerator and denominator normalization rows use the signed-pair route of the integer packet before the positive denominator witness is paired with rational data.

Thus the field-extension endpoint consumes the same finite dependency route: $\IntUp$ signed-pair data, $\RatUp$ denominator normalization, and then the $\FieldExtUp$ endpoint row. No detached host integer, quotient rational, or ambient field equality is introduced between these sibling chapters.
\end{proof}

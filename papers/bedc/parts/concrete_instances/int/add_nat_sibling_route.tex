\begin{theorem}[Int Add Nat sibling route]
\label{thm:int-add-nat-sibling-route}
Every signed-pair $\IntUp$ packet consumes the $\NatUp$ unary rows of \autoref{ch:concrete-instances-nat-namecert} and the $\AddUp$ balance rows of \autoref{ch:concrete-instances-add-namecert} before the $\RatUp$ denominator routes of \autoref{ch:concrete-instances-rat-namecert} can read it. Thus a rational denominator consumer receives an already displayed signed-pair row with Nat magnitude data and Add balance data, not a host integer quotient, primitive subtraction row, or field-level rational equality at the integer boundary.
\end{theorem}
\begin{proof}
Start with the signed-pair surfaces of the Int chapter. The signed-pair interface and the signed-pair additive lattice dependency expose the sign row, unary magnitude row, and balance context before any Rat-facing denominator read is available.

Apply \autoref{thm:int-signed-pair-additive-lattice-dependency} and \autoref{thm:int-nat-add-rat-sibling-lattice-route}. These rows cite the Nat and Add sibling chapters and place the $\IntUp$ carrier between their unary and balance data and the later Rat denominator consumer.

Finally use \autoref{thm:int-rat-additive-denominator-route} and \autoref{thm:int-rat-denominator-dependency-surface}. The Rat route reads the balanced signed row beside a positive denominator only after the Nat and Add rows have already been consumed, so the downstream read remains a sibling route through the displayed Int packet.
\end{proof}

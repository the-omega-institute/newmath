\begin{lemma}[Int signed-pair Rat dependency lattice]
\label{lem:int-rat-signed-pair-dependency-lattice}
The $\IntUp$ signed-pair rows consumed by $\RatUp$ are already arithmetic rows over $\NatUp$ and $\AddUp$. The sign coordinate is paired with a unary magnitude row from \autoref{ch:concrete-instances-nat-namecert}, the zero-balancing and signed continuation discipline are routed through \autoref{ch:concrete-instances-add-namecert}, and the resulting signed numerator surface is the row later read by the denominator and numerator packets of \autoref{ch:concrete-instances-rat-namecert}.
\end{lemma}

Thus an accepted rational numerator read does not rebuild an integer quotient. It consumes the displayed $\IntUp$ signed pair, whose $\BHist$ magnitude, additive balancing, $\hsame$ transport, $\Cont$ replay, $\Pkg$ provenance, and local $\NameCert$ row have already been exposed. The sibling edge is therefore Nat source row plus Add balancing row into Int signed pair, then Int signed pair into Rat numerator and denominator readback.

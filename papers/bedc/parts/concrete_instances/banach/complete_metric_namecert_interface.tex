\begin{definition}[Banach complete-metric NameCert interface]
\label{def:banach-complete-metric-namecert-interface}
For a concrete Banach carrier $X$ of
\autoref{def:banach-concrete-carrier}, a stream name $s$, a modulus history
$M$, and a displayed limit history $\ell$, the Banach complete-metric
$\NameCert$ interface is the predicate
$$
\mathsf{CompleteMetricLimitWitness}(X,s,M,\ell)
$$
read over the norm-induced Banach metric rows. Its visible ledger consists of
the carrier row $X$, the carried observations of $s$, the modulus row $M$, the
carried limit row $\ell$, and the late-observation distance ledgers named as
$\mathsf{BanachMetricWitness}_X(s(n),\ell,d_{n,\varepsilon})$ together with the
corresponding $\RatUp$ comparison rows.

Equivalently, the displayed row
$$
\mathsf{CompleteMetricLimitWitness}(X,s,M,\ell,L)
$$
is accepted at the Banach surface exactly when every metric-distance component
of $L$ is presented by the norm-induced Banach ledger
$\mathsf{BanachMetricWitness}_X$ rather than by an anonymous metric row. Thus
the public interface exposes the same carrier $X$, stream $s$, modulus $M$,
limit $\ell$, and convergence ledgers consumed by $\NameCert_{\CompleteMetricUp}$.
\end{definition}

\begin{theorem}[Banach complete-metric NameCert coupling]
\label{thm:banach-complete-metric-namecert-coupling}
Let $X$ be a concrete Banach carrier, let $s$ be a regular Cauchy
$\StreamNameUp$ stream over the norm-induced metric
$\mathsf{BanachMetricWitness}_X$, let $M$ be its displayed modulus, and let
$\ell$ be a displayed limit. Carrier transport along $\hsame$, together with a
displayed Banach-surface row
$$
\mathsf{CompleteMetricLimitWitness}(X,s,M,\ell,L),
$$
yields the $\mathsf{SemanticNameCert}$ package for the predicate
$$
\mathsf{CompleteMetricLimitWitness}(X,s,M)
$$
on limit histories. The coupling depends on
\autoref{def:banach-concrete-carrier}, \autoref{thm:banach-hsame-transport},
\autoref{thm:banach-complete-metric-witness-exactness}, and the complete-metric
certificate theorem
$\mathsf{BEDC.Derived.CompleteMetricUp.CompleteMetricLimitWitness\_name\_certificate}$.
\end{theorem}
\leanchecked{BEDC.Derived.CompleteMetricUp.CompleteMetricLimitWitness\_empty\_modulus\_observation\_metric\_row}
\leanchecked{BEDC.Derived.CompleteMetricUp.CompleteMetricLimitWitness\_observation\_distance\_deterministic}
\begin{proof}
Unpack the concrete Banach carrier. Its carrier component supplies the
$\hsame$ transport operation on carried vector histories, and its norm component
identifies each induced distance row with a displayed
$\mathsf{BanachMetricWitness}_X$ row.

Next transport the complete-metric data componentwise. Use
\autoref{thm:banach-hsame-transport} for the stream observations, modulus
history, limit history, threshold rows, metric-distance rows, and $\RatUp$
comparison rows. The result is still a Banach carrier package because the
transported metric rows are repacked through the norm-induced ledger.

Now invoke the complete-metric NameCert constructor represented on the Lean
side by
$\mathsf{BEDC.Derived.CompleteMetricUp.CompleteMetricLimitWitness\_name\_certificate}$.
Its carrier-transport input is the Banach carrier transport just extracted, and
its witness input is the displayed row of
\autoref{def:banach-complete-metric-namecert-interface}.

Finally apply \autoref{thm:banach-complete-metric-witness-exactness}. It
ensures that no metric row has left the Banach surface: every distance ledger
used by the complete-metric package is the same norm-distance row displayed as
$\mathsf{BanachMetricWitness}_X$. Repacking these rows gives the required
$\mathsf{SemanticNameCert}$ for
$\mathsf{CompleteMetricLimitWitness}(X,s,M)$.
\end{proof}

\begin{theorem}[Banach singleton limit NameCert uniqueness]
\label{thm:banach-singleton-limit-namecert-uniqueness}
In the singleton empty-history Banach carrier of
\autoref{def:banach-singleton-empty-history-instance}, any two Banach-surface
complete-metric rows for the same stream,
$$
\mathsf{CompleteMetricLimitWitness}(C_{\emp},s,M_0,\ell_0,L_0)
\quad\text{and}\quad
\mathsf{CompleteMetricLimitWitness}(C_{\emp},s,M_1,\ell_1,L_1),
$$
have $\hsame(\ell_0,\ell_1)$ limits. Consequently the two displayed vector
limits are classified by the singleton vector classifier
$\rho_{\emp}(\ell_0,\ell_1)$.
\end{theorem}
\leanchecked{BEDC.Derived.CompleteMetricUp.CompleteMetricLimitWitness\_singleton\_classifier\_distance}
\begin{proof}
First read both displayed rows through the interface of
\autoref{def:banach-complete-metric-namecert-interface}. The carrier predicate
is $C_{\emp}$, so each limit row is a singleton empty-history endpoint and each
metric-distance row is the constant zero Banach metric row.

Apply the complete-metric singleton uniqueness theorem
$\mathsf{BEDC.Derived.CompleteMetricUp.CompleteMetricLimitWitness\_singleton\_uniqueness}$
to the two rows for the same stream. It returns
$\hsame(\ell_0,\ell_1)$ because the complete-metric carrier is the singleton
predicate $h\mapsto\hsame(h,\emp)$.

It remains to read that relation at the vector endpoint surface. The singleton
Banach carrier is the singleton norm carrier, so
\autoref{thm:singleton-empty-history-vecspace-endpoint-collapse} converts the
two carried limit endpoints and their $\hsame$ relation into the classifier row
$\rho_{\emp}(\ell_0,\ell_1)$.
\end{proof}

\begin{corollary}[Banach complete-metric interface determinacy]
\label{cor:banach-complete-metric-interface-determinacy}
In the singleton empty-history Banach carrier $C_{\emp}$ of
\autoref{def:banach-singleton-empty-history-instance}, fix a stream name $s$ and
fix a displayed Banach complete-metric interface row
$$
\mathsf{CompleteMetricLimitWitness}(C_{\emp},s,M,\ell,L).
$$
Every accepted consumer-facing use of this row factors through the same visible
interface data: the carrier row $C_{\emp}$, the stream observations of $s$, the
modulus row $M$, the carried limit row $\ell$, the late-distance rows in $L$
displayed as
$$
\mathsf{BanachMetricWitness}_{C_{\emp}}(s(n),\ell,d_{n,\varepsilon}),
$$
and the corresponding $\RatUp$ comparison rows. If a second accepted use for
the same stream displays
$$
\mathsf{CompleteMetricLimitWitness}(C_{\emp},s,M',\ell',L'),
$$
then its limit normal form is the same singleton endpoint, classified by
$\rho_{\emp}(\ell,\ell')$. Thus the interface admits no alternate hidden metric
coordinate, no anonymous distance classifier, and no duplicate endpoint
classifier source beyond the rows named in
\autoref{def:banach-complete-metric-namecert-interface} and
\autoref{thm:banach-singleton-limit-namecert-uniqueness}.
\end{corollary}
\begin{proof}
Unfold the interface of
\autoref{def:banach-complete-metric-namecert-interface}. An accepted
Banach-surface row is not merely a complete-metric row with an unspecified
metric component. Its visible ledger consists of the carrier $C_{\emp}$, the
stream observations of $s$, the modulus history, the carried limit history,
the late-observation distance rows, and the $\RatUp$ comparison rows. The same
definition requires every metric-distance component of the late ledger to be
presented as a norm-induced Banach row
$\mathsf{BanachMetricWitness}_{C_{\emp}}$ rather than as an anonymous metric
row. Hence any consumer use of the fixed row must read exactly these displayed
coordinates.

For the singleton carrier, \autoref{def:banach-singleton-empty-history-instance}
identifies the vector carrier with the empty-history singleton, the endpoint
classifier with $\rho_{\emp}$, and the norm-induced metric distance with the
constant RealUp zero row. Therefore each accepted distance entry in $L$ has the
fixed Banach source
$\mathsf{BanachMetricWitness}_{C_{\emp}}(s(n),\ell,d_{n,\varepsilon})$, and the
only exactness decision available at the complete-metric slot is the one that
keeps this norm-induced row at the Banach surface.

Now compare any second accepted use for the same stream. Both displayed rows
are Banach-surface complete-metric rows for $C_{\emp}$, so
\autoref{thm:banach-singleton-limit-namecert-uniqueness} applies to them. It
first gives $\hsame(\ell,\ell')$ and then reads that comparison as the
singleton vector classifier $\rho_{\emp}(\ell,\ell')$. Thus an alleged alternate
limit normal form is already classified by the same singleton endpoint row.
Since the interface ledger has no additional carrier coordinate, no extra
metric slot, and no classifier other than the displayed singleton classifier,
every accepted consumer-facing use factors through the listed finite Banach
route.
\end{proof}

$$

\begin{enumerate}
\item $L_X$ and $L_Y$ are the two $\SorgenfreyLineUp$ source rows, each exposing lower-limit interval windows only through the finite carrier of \autoref{def:sorgenfrey-line-carrier}.
\item $Q$ is the $\ProductTopologyUp$ product row, reading paired histories through component topology rows and rectangular subbasis data.
\item $O_X$ and $O_Y$ are the two component $\TopologyUp$ indexed-open rows exported by the lower-limit line carriers.
\item $B$ is the lower-limit rectangle ledger, recording basic rows of the form $[a,b)\times[c,d)$ as paired component windows rather than as a host powerset of $\RealUp\times\RealUp$.
\item $S$ is the $\SierpinskiSpaceUp$ observable-open row used when a consumer asks whether a displayed rectangle or generated product open has been observed.

The classifier compares accepted packets only through these rows. It stores no host plane, no metric witness, no second-countable base, no powerset topology, no quotient real-pair equality, no selected open family, and no hidden product carrier outside the displayed $\ProductTopologyUp$ route.
\end{definition}

\begin{theorem}[Sorgenfrey plane NameCert obligations]
\label{thm:sorgenfrey-plane-namecert-obligations}
For every accepted $\SorgenfreyPlaneUp$ carrier
$$
P=(L_X,L_Y,Q,O_X,O_Y,B,S,H,C,R,N),
$$
the local $\NameCert_{\SorgenfreyPlaneUp}$ obligations are exactly the displayed finite product rows:
\begin{enumerate}
\item component lower-limit admission through $L_X$ and $L_Y$;
\item product-topology admission through $Q$, with the two topology rows $O_X,O_Y$ exposed before any product read is consumed;
\item rectangle-ledger exactness through $B$, requiring every basic lower-limit product neighbourhood to be a paired half-open window row;
\item observable-open readback through $S$, so generated product opens are inspected through finite truth rows rather than host propositions;
\item transport, replay, provenance, and local naming through $H,C,R,N$.
\end{enumerate}
These obligations do not assert metrizability, normality, second countability, host separation theorems, powerset topology on pairs, quotient real-pair equality, selected bases, Lean verification, or bridge checking.
\end{theorem}
\begin{proof}
Project the accepted carrier. The only component sources are the lower-limit line rows $L_X,L_Y$, and each component topology read must pass through $O_X,O_Y$ before the product row is consumed.

The product route is then fixed by $Q$. Its rectangular subbasis access is refined by $B$, whose entries are paired half-open windows and not arbitrary subsets of a host plane.

Observable-open requests pass through $S$, so membership reads return displayed finite truth data. The structural rows $H,C,R,N$ only transport, replay, source, and name the same route. Since no row stores a metric, a selected basis, a host powerset, quotient real-pair equality, or an ambient plane, the listed rows exhaust the local NameCert obligations.
\end{proof}

\begin{theorem}[Sorgenfrey plane product-separation route]
\label{thm:sorgenfrey-plane-product-separation-route}
Every separation-facing read exported by an accepted $\SorgenfreyPlaneUp$ carrier factors through the finite route
$$
L_X,L_Y,O_X,O_Y,Q,B,S,H,C,R,N.
$$
The route may distinguish product observations by component lower-limit windows and observable-open truth rows, but it cannot claim a host metric, a normality theorem, a powerset topology, or a selected countable base for the classical Sorgenfrey plane.
\end{theorem}
\begin{proof}
Start with a separation-facing request. The carrier has no direct separation coordinate, so the request must first read one or both component lower-limit line rows and their topology exports.

The product row $Q$ then pairs those component reads. Any product-open evidence must pass through the rectangle ledger $B$, whose entries are generated from the two lower-limit component windows.

The observable-open row $S$ records the finite truth readback for the generated product observation. Transport and replay through $H,C$, together with provenance and local naming through $R,N$, preserve that same componentwise route. Because the carrier lacks coordinates for host metrics, normality proofs, powerset opens, selected countable bases, and quotient real-pair equality, no such claim can be exported by the product-separation route.
\end{proof}

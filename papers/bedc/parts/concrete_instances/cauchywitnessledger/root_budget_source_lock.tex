\begin{theorem}[Cauchy witness ledger root source lock]
\label{thm:cauchy-witness-ledger-root-source-lock}
Let $L=(Q,B,S,K,H,C,P,N)$ be an accepted $\CauchyWitnessLedgerUp$ packet. The diagonal-budget row $Q$, observation-budget row $B$, synchronizer row $S$, and completion-seal classifier row $K$ use one shared finite $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ source surface. Thus the public root read of $L$ cannot attach $K$ to a stream, rational, dyadic, or Real source separate from the source already read by $Q$, $B$, and $S$.
\end{theorem}
\leanchecked{BEDC.Derived.CauchyWitnessLedgerUp.CauchyWitnessLedgerCarrier\_root\_source\_lock}
\begin{proof}
Begin with \autoref{def:cauchy-witness-ledger-carrier}. The accepted packet exposes only $Q,B,S,K,H,C,P,N$, so every source visible at the root must be read through one of those rows.

Apply \autoref{thm:cauchy-witness-ledger-budget-factorization} and \autoref{thm:cauchy-witness-ledger-diagonal-observation-route}. The budget factorization orders the analytic route as $Q \longmapsto B \longmapsto S \longmapsto K$, and the diagonal-observation route identifies the $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ surface seen by $B$ with the one already fixed by $Q$.

Finally use \autoref{thm:cauchy-witness-ledger-synchronizer-classifier-lock}. The synchronizer carries the same $Q \longmapsto B$ source into $K$, so the classifier has no separate source row from which an alternate completion-facing read could be exposed.
\end{proof}

\begin{theorem}[Cauchy witness ledger root source-window route lock]
\label{thm:cauchy-witness-ledger-root-source-window-route-lock}
Every accepted $\CauchyWitnessLedgerUp$ completion-facing root read uses one shared finite $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ source-window route through
$$
Q \longmapsto B \longmapsto S \longmapsto K.
$$
The read cannot attach a second stream window, rational readback, dyadic tolerance, real seal, hidden tail, quotient stream, host real equality, or ambient completeness row to the displayed carrier rows $Q,B,S,K,H,C,P,N$.
\end{theorem}
\leanchecked{BEDC.Derived.CauchyWitnessLedgerUp.CauchyWitnessLedgerCarrier\_root\_source\_window\_route\_lock}
\begin{proof}
First apply the root source lock of \autoref{thm:cauchy-witness-ledger-root-source-lock}. It identifies the stream, regular-rational, dyadic, and real source surface of the completion-seal classifier with the source surface already read by $Q,B,S$.

Next use \autoref{thm:cauchy-witness-ledger-streamname-window-source-lock}. The stream window visible to a completion consumer is the window supplied by the same finite budget route, so the consumer cannot attach a separate window source before reading $K$.

Apply the refusal boundary of \autoref{thm:cauchy-witness-ledger-root-budget-refusal-boundary}. Any proposed rational readback, dyadic tolerance, real seal, hidden tail, quotient stream, host real equality, or ambient completeness coordinate outside the route $Q \longmapsto B \longmapsto S \longmapsto K$ is refused at the root boundary.

Finally, \autoref{thm:cauchy-witness-ledger-root-consumer-exhaustion} says every downstream completion-facing consumer factors through the displayed ledger rows. The common source-window route is therefore the whole consumer-facing route, not one branch among several.
\end{proof}

\begin{theorem}[Cauchy witness ledger root budget coherence]
\label{thm:cauchy-witness-ledger-root-budget-coherence}
Let $L=(Q,B,S,K,H,C,P,N)$ and $L'=(Q',B',S',K',H',C',P',N')$ be accepted $\CauchyWitnessLedgerUp$ packets. If the two public reads agree on $Q,B,S,H,C,P,N$, then they determine the same public completion-seal classifier read $K$. In particular, no accepted root read can split the classifier from the displayed budget route $Q \longmapsto B \longmapsto S \longmapsto K$.
\end{theorem}
\begin{proof}
Start with \autoref{thm:cauchy-witness-ledger-route-determinacy}. Agreement on the displayed route and structural rows fixes the completion-facing route, so the public read has no second branch on which $K$ could vary.

Use \autoref{thm:cauchy-witness-ledger-budget-source-coherence}. The rows $Q$, $B$, and $S$ share one finite $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ source before $K$ is exposed.

Now apply \autoref{thm:cauchy-witness-ledger-completion-classifier-readback}. The classifier readback is exactly the displayed route through $Q \longmapsto B \longmapsto S \longmapsto K$ together with $H,C,P,N$. Since those route and structural rows already agree, the public classifier read agrees as well.
\end{proof}
\leanchecked{BEDC.Derived.CauchyWitnessLedgerUp.CauchyWitnessLedgerCarrier\_root\_budget\_coherence}

\begin{theorem}[Cauchy witness ledger root budget refusal boundary]
\label{thm:cauchy-witness-ledger-root-budget-refusal-boundary}
Let $L=(Q,B,S,K,H,C,P,N)$ be an accepted $\CauchyWitnessLedgerUp$ packet. Any attempted public root read that gives the completion-seal classifier $K$ a stream, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, tail, limit, or seal source not already exposed through
$$
Q \longmapsto B \longmapsto S \longmapsto K
$$
is routed to the displayed refusal boundary. Such a read cannot export a quotient stream, host $\RealUp$ equality, selected external limit, countable-choice row, hidden tail, alternate seal, or ambient completeness coordinate.
\end{theorem}
\leanchecked{BEDC.Derived.CauchyWitnessLedgerUp.CauchyWitnessLedgerCarrier\_root\_budget\_refusal\_boundary}
\begin{proof}
Read the accepted carrier through \autoref{def:cauchy-witness-ledger-carrier}. Its public root surface is exactly $Q,B,S,K,H,C,P,N$, and the completion-facing rows are displayed through the finite route $Q \longmapsto B \longmapsto S \longmapsto K$.

Apply \autoref{thm:cauchy-witness-ledger-root-source-lock}. The stream, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ sources visible to $K$ are the same sources already read by $Q$, $B$, and $S$, so a proposed second source for the classifier is not a carrier row.

Next apply \autoref{thm:cauchy-witness-ledger-root-budget-coherence}. Once $Q,B,S,H,C,P,N$ are fixed, the public classifier read $K$ is determined by the displayed budget route, so a different tail, limit, or seal source cannot produce a second accepted classifier read.

Finally use \autoref{thm:cauchy-witness-ledger-root-consumer-exhaustion}. Downstream completion consumers have no escape coordinate outside the displayed ledger, and the requested quotient stream, host $\RealUp$ equality, selected external limit, countable-choice row, hidden tail, alternate seal, or ambient completeness coordinate is therefore refused rather than exported.
\end{proof}

\begin{theorem}[Cauchy witness ledger root budget classifier dichotomy]
\label{thm:cauchy-witness-ledger-root-budget-classifier-dichotomy}
Let $L=(Q,B,S,K,H,C,P,N)$ be an accepted $\CauchyWitnessLedgerUp$ packet. Every public completion-seal classifier read of $L$ is exactly one of the following:
\begin{enumerate}
\item it follows the displayed route
$$
Q \longmapsto B \longmapsto S \longmapsto K
$$
over the shared $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ source surface exposed by the root ledger;
\item it is routed to the refusal boundary and therefore exports no quotient stream, host $\RealUp$ equality, selected external limit, countable-choice row, hidden tail, alternate seal, or ambient completeness coordinate.
\end{enumerate}
\end{theorem}
\begin{proof}
First fix the source surface. By \autoref{thm:cauchy-witness-ledger-root-source-lock}, the rows $Q,B,S,K$ read one shared $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ source, so an accepted classifier read cannot begin from a second stream, rational, dyadic, or Real source.

Next bind the accepted branch. \autoref{thm:cauchy-witness-ledger-root-budget-coherence} says that once the displayed budget route and structural rows are fixed, the completion-seal classifier read is determined by $Q \longmapsto B \longmapsto S \longmapsto K$. This gives the first side of the dichotomy.

It remains to account for every read not carried by that route. \autoref{thm:cauchy-witness-ledger-root-budget-refusal-boundary} sends any classifier read with a detached source, tail, limit, or seal coordinate to the refusal boundary.

Finally apply \autoref{thm:cauchy-witness-ledger-root-consumer-exhaustion}. Downstream $\RealUp$-completion consumers have no coordinate outside the displayed ledger, so a classifier read is either the accepted route above or a refused request, with no third exported surface.
\end{proof}

\begin{theorem}[Cauchy witness ledger root diagnostic refusal dichotomy]
\label{thm:cauchy-witness-ledger-root-diagnostic-refusal-dichotomy}
Let $L=(Q,B,S,K,H,C,P,N)$ be an accepted $\CauchyWitnessLedgerUp$ packet. Every root diagnostic read or completion-facing consumer read of $L$ is exactly one of the following:
\begin{enumerate}
\item it follows the displayed route
$$
Q \longmapsto B \longmapsto S \longmapsto K
$$
over the shared $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$ faces exposed by the ledger;
\item it is refused at the displayed root boundary and exports no quotient stream, selected limit, host equality, countable-choice row, ambient completeness coordinate, hidden tail, alternate seal, or standard bridge row.
\end{enumerate}
\end{theorem}
\begin{proof}
First fix the source boundary. The root source lock of \autoref{thm:cauchy-witness-ledger-root-source-lock} and the source-window route lock of \autoref{thm:cauchy-witness-ledger-root-source-window-route-lock} place every accepted root read on the same $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$ source route through $Q \longmapsto B \longmapsto S \longmapsto K$.

Next classify diagnostic reads. Apply \autoref{thm:cauchy-witness-ledger-root-budget-classifier-dichotomy}; the classifier side has only the accepted displayed route or the refusal branch.

Finally attach the consumer boundary. The refusal boundary of \autoref{thm:cauchy-witness-ledger-root-budget-refusal-boundary} blocks the excluded quotient, limit, host-equality, countable-choice, ambient-completeness, tail, seal, and bridge rows, while \autoref{thm:cauchy-witness-ledger-root-consumer-exhaustion} says completion-facing consumers have no coordinate outside the displayed ledger. Thus the root diagnostic surface has no third export case.
\end{proof}

\begin{theorem}[Cauchy witness ledger root consumer exhaustion]
\label{thm:cauchy-witness-ledger-root-consumer-exhaustion}
Every downstream $\RealUp$-completion consumer of an accepted $\CauchyWitnessLedgerUp$ packet $L=(Q,B,S,K,H,C,P,N)$ factors through
$$
Q \longmapsto B \longmapsto S \longmapsto K
$$
and the structural rows $H,C,P,N$. Such a consumer has no quotient stream, host real equality, selected external limit, countable-choice row, hidden tail, alternate seal, or ambient completeness coordinate outside the displayed ledger.
\end{theorem}
\leanchecked{BEDC.Derived.CauchyWitnessLedgerUp.CauchyWitnessLedgerCarrier\_root\_consumer\_exhaustion}
\begin{proof}
First apply \autoref{thm:cauchy-witness-ledger-ledger-exhaustion} and \autoref{thm:cauchy-witness-ledger-real-completion-nonescape}. Ledger exhaustion confines every public read to $Q,B,S,K,H,C,P,N$, and Real-completion nonescape places every admitted completion read on the route $Q \longmapsto B \longmapsto S \longmapsto K$.

Next use \autoref{thm:cauchy-witness-ledger-streamname-window-source-lock} and \autoref{thm:cauchy-witness-ledger-regseqrat-readback-totality}. The StreamName window source and RegSeqRat readback are both supplied by the same finite budget source, so a downstream consumer cannot detach a stream window, rational readback, dyadic tolerance row, or hidden tail from the ledger.

Finally apply \autoref{thm:cauchy-witness-ledger-real-seal-refusal-boundary}. Any request outside the displayed route lands at the refusal boundary rather than at a quotient stream, host Real equality, selected external limit, countable-choice row, alternate seal, or ambient completeness coordinate.
\end{proof}

\begin{corollary}[Cauchy witness ledger root source-lock strict obstruction]
\label{cor:cauchy-witness-ledger-root-source-lock-strict-obstruction}
Let $L=(Q,B,S,K,H,C,P,N)$ be an accepted $\CauchyWitnessLedgerUp$ packet. The refusal boundary of \autoref{thm:cauchy-witness-ledger-root-budget-refusal-boundary}, read under the source lock of \autoref{thm:cauchy-witness-ledger-root-source-lock}, is a strict local obstruction: it blocks exactly an attempted completion-seal classifier read that detaches $K$ from the displayed route
$$
Q \longmapsto B \longmapsto S \longmapsto K
$$
or that tries to reuse the classifier through an unlisted quotient stream, host $\RealUp$ equality, selected external limit, countable-choice row, hidden tail, alternate seal, or ambient completeness coordinate. It does not assert a global failure of Cauchy completion, does not supply a host theorem, and does not validate any hidden external completion route.
\end{corollary}
\begin{proof}
The root source lock fixes the source side first. By \autoref{thm:cauchy-witness-ledger-root-source-lock}, the stream, regular-rational, dyadic, and Real source surface visible to $K$ is the same finite source already read by $Q$, $B$, and $S$. Hence a request that reaches $K$ by a separate stream, rational, dyadic, or Real source is not a second accepted root read of $L$; it is precisely a detached classifier-source request.

Now apply the refusal boundary of \autoref{thm:cauchy-witness-ledger-root-budget-refusal-boundary}. That theorem lists the refused coordinates: quotient stream, host $\RealUp$ equality, selected external limit, countable-choice row, hidden tail, alternate seal, and ambient completeness coordinate, together with any tail, limit, or seal source not exposed through $Q \longmapsto B \longmapsto S \longmapsto K$. These are the only coordinates named by the displayed obstruction, so the obstruction is confined to the named converse or reuse route for the root classifier.

Finally use the consumer exhaustion statement of \autoref{thm:cauchy-witness-ledger-root-consumer-exhaustion}. Every downstream completion-facing consumer factors through the displayed ledger rows and has no coordinate outside $Q,B,S,K,H,C,P,N$. Therefore the refusal cannot be reinterpreted as a statement about all Cauchy completions, as a theorem of an ambient host theory, or as evidence for a hidden validation route. It only rejects off-route uses of the local source-locked witness ledger.
\end{proof}

\chapter{A Concrete Naming Certificate for \(\OptionUp\)}
\label{ch:concrete-instances-option-namecert}

This chapter separates two certificate scopes. The checked BEDC object is the nullable history envelope $\mathsf{OptionHistoryCarrier}_S(h)\equiv \hsame(h,\emp)\lor S(h)$ together with the classifier endpoint relation carried by $\hsame$. The constructor-tagged optional value interface for $\OptionUp$ is a stronger interface: it requires the typed tag classifier, none/some separation, present inversion, the tag-plus-source ledger, and the compatibility hypothesis $\mathsf{TaggedOptionSourceHsameCompatible}$ used below.

\section{Roadmap template status}

\begin{enumerate}
  \item \textbf{Traditional target}: optional data with one absent value and present values over a source object.
  \item \textbf{BEDC source}: any licensed source carrier $A$ equipped with its existing name certificate.
  \item \textbf{Carrier choice}: the typed $\mathsf{OptionCarrier}$ has $\mathsf{none}$ and $\mathsf{some}$ constructors, while the checked history carrier is the nullable envelope $\mathsf{OptionHistoryCarrier}_S(h)\equiv \hsame(h,\emp)\lor S(h)$.
  \item \textbf{Classifier choice}: the typed $\mathsf{OptionClassifierSpec}$ relates only matching option tags through the source classifier; $\mathsf{OptionHistoryClassifier}$ is the nullable-history classifier consisting of endpoint envelope-carrier witnesses and endpoint $\hsame$.
  \item \textbf{Stability obligations}: typed optional carrier transport, typed absent/present separation, lifted typed classifier symmetry and transitivity, nullable-history endpoint transport, nullable-history classifier symmetry and transitivity, and for the constructor-tagged optional-history certificate a payload coherence condition converting source-carrier $\hsame$ between middle payloads into the source classifier.
  \item \textbf{Ledger}: the typed option interface records the absent-or-present tag and present payload evidence, while the checked nullable-history envelope ledger records only the raw envelope-carrier witness and the raw-to-visible $\hsame$ link.
  \item \textbf{Core theorem list}: typed carrier cases, typed absent/present no-confusion, present inversion, typed classifier equivalence, tagged same-tag exactness, tagged mixed-tag separation, source-compatibility for the tagged semantic target, nullable-history carrier transport, nullable-history classifier transport, and the nullable-history semantic certificate.
  \item \textbf{Exactness predicate}: matching-tag exactness belongs to $\mathsf{OptionClassifierSpec}$; the tagged history classifier has same-tag exactness through visible absent and present endpoints plus mixed-tag separation, while the checked nullable-history classifier exactness is endpoint envelope membership plus endpoint $\hsame$ and carries no present constructor tag.
  \item \textbf{Standard bridge}: the checked nullable envelope has only empty-or-source endpoint readback. Ordinary $\mathsf{none}$ and $\mathsf{some}(a)$ readback belongs to the typed and tagged option interface under $\mathsf{TaggedOptionHistoryCarrier}$, $\mathsf{TaggedOptionHistoryClassifier}$, same-tag exactness, mixed-tag separation, and the tag-plus-source ledger.
  \item \textbf{Status}: the nullable history-envelope $\mathsf{SemanticNameCert}$ is $\mathsf{CheckedCert}$; the constructor-tagged optional-history certificate is $\mathsf{ProofTarget}$.
\end{enumerate}

\begin{definition}[Tagged option history carrier]
\label{def:tagged-option-history-carrier}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, define $\mathsf{TaggedOptionHistoryCarrier}_S(h)$ by the two constructor-shaped cases:
\begin{enumerate}
  \item absent: $\hsame(h,\emp)$;
  \item present: there is a payload history $a$ such that $S(a)$ and $\hsame(h,\Eone(a))$.
\end{enumerate}
\end{definition}

\begin{theorem}[Tagged option history present inversion]
\label{thm:tagged-option-history-present-inversion}
For any BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$ and
history $h$, a present endpoint in the tagged carrier exposes a source payload:
$$
\mathsf{TaggedOptionHistoryCarrier}_S(\Eone(h))
\quad\Longrightarrow\quad
\exists a:\mathsf{BHist}.\; S(a)\land \hsame(h,a).
$$
\end{theorem}
\begin{proof}
Case-analyze the tagged carrier witness for $\Eone(h)$. The absent branch gives
$\hsame(\Eone(h),\emp)$ and is eliminated by generated-history no-confusion
between a present endpoint and the empty endpoint. The present branch gives a
payload history $a$, a source witness $S(a)$, and
$\hsame(\Eone(h),\Eone(a))$; applying the one-constructor inversion theorem for
$\hsame$ strips the visible present tags and yields $\hsame(h,a)$, so $a$ is the
required payload.
\end{proof}

For the finite-data certificate template, the traditional target is
absent-vs-present option tags; the BEDC source is $\mathsf{BHist}$ with
$\hsame$; the carrier is $\mathsf{TaggedOptionHistoryCarrier}$; no classifier
participates in this obligation. The stability obligation is constructor
no-confusion, the ledger exposes the visible tag, the core theorem is this
disjointness lemma, exactness is absent/present separation, the standard bridge
is $\mathsf{none}$/$\mathsf{some}$, and the site status is
$\mathsf{ProofTarget}$.

\begin{lemma}[Tagged option history carrier constructor separation]
\label{lem:tagged-option-history-carrier-constructor-separation}
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, history $h$, and payload history $a$, an absent endpoint and a present endpoint cannot be witnessed by the same carrier endpoint:
$$
\hsame(h,\emp)\land S(a)\land \hsame(h,\Eone(a))
\quad\Longrightarrow\quad
\bot.
$$
\end{lemma}
\begin{proof}
Compose symmetry of $\hsame(h,\emp)$ with $\hsame(h,\Eone(a))$ to obtain $\hsame(\emp,\Eone(a))$. BEDC generated-history no-confusion between the empty constructor and the one-step constructor eliminates this endpoint equality. The source witness $S(a)$ records that the present payload is a valid source history but is not needed for the constructor contradiction.
\end{proof}

\begin{definition}[Tagged option history classifier]
\label{def:tagged-option-history-classifier}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$ and source classifier $\sim_S$, define $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,k)$ by the two constructor-matching cases:
\begin{enumerate}
  \item absent-absent: $\hsame(h,\emp)$ and $\hsame(k,\emp)$;
  \item present-present: there are payload histories $a,b$ such that $S(a)$, $S(b)$, $a\sim_S b$, $\hsame(h,\Eone(a))$, and $\hsame(k,\Eone(b))$.
\end{enumerate}
\end{definition}

\begin{definition}[Tagged option source hsame compatibility]
\label{def:tagged-option-source-hsame-compatibility}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$ and source classifier $\sim_S$, define $\mathsf{TaggedOptionSourceHsameCompatible}(S,\sim_S)$ to be the property that for all payload histories $a,b$, if $S(a)$, $S(b)$, and $\hsame(a,b)$, then $a\sim_S b$.
\end{definition}

\begin{theorem}[Tagged option history classifier endpoint carriers]
\label{thm:tagged-option-history-classifier-endpoint-carriers}
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$,
source classifier $\sim_S$, and histories $h,k$, if
$\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,k)$ holds, then both
$\mathsf{TaggedOptionHistoryCarrier}_S(h)$ and
$\mathsf{TaggedOptionHistoryCarrier}_S(k)$ hold.
\end{theorem}
\begin{proof}
Case-analyze the tagged classifier witness. The absent-absent branch stores
$\hsame(h,\emp)$ and $\hsame(k,\emp)$, so the absent carrier clause rebuilds
both endpoint carriers. The present-present branch stores payload histories
$a,b$, source witnesses $S(a)$ and $S(b)$, the payload classifier witness
$a\sim_S b$, and visible endpoint links $\hsame(h,\Eone(a))$ and
$\hsame(k,\Eone(b))$. The two present carrier clauses reuse the corresponding
payload histories, source witnesses, and endpoint links, so both classified
endpoints are tagged option carrier endpoints.
\end{proof}

\begin{theorem}[Tagged option history present-present exactness]
\label{thm:tagged-option-history-present-present-exactness}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, source classifier $\sim_S$, and histories $h,k$, the tagged option history classifier is exact on two visible present endpoints:
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\Eone(h),\Eone(k))
\quad\Longleftrightarrow\quad
\exists a,b:\mathsf{BHist}.\; S(a)\land S(b)\land \hsame(h,a)\land \hsame(k,b)\land a\sim_S b.
$$
\end{theorem}
\begin{proof}
For the forward direction, case-analyze the tagged classifier witness. The absent-absent branch would contain $\hsame(\Eone(h),\emp)$ and is eliminated by one-versus-empty no-confusion. The present-present branch gives payload histories $a,b$, source witnesses $S(a)$ and $S(b)$, a source classifier witness $a\sim_S b$, and endpoint witnesses $\hsame(\Eone(h),\Eone(a))$ and $\hsame(\Eone(k),\Eone(b))$; one-constructor inversion strips the visible $e1$ tags and yields $\hsame(h,a)$ and $\hsame(k,b)$. For the reverse direction, the displayed payload data rebuilds the present-present classifier branch after applying one-constructor congruence for $\hsame$ to the two payload endpoint witnesses.
\end{proof}

\begin{theorem}[Tagged option history visible present exactness]
\label{thm:tagged-option-history-visible-present-exactness}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, source classifier $\sim_S$, source certificate $\mathsf{cert}:\NameCert(S,\sim_S)$, and $\mathsf{TaggedOptionSourceHsameCompatible}(S,\sim_S)$, if $S(h)$ and $S(k)$, then the tagged option history classifier is exact on the two source-visible present endpoints:
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\Eone(h),\Eone(k))
\quad\Longleftrightarrow\quad
h\sim_S k.
$$
\end{theorem}
\begin{proof}
For the forward direction, apply Theorem~\ref{thm:tagged-option-history-present-present-exactness} to the tagged classifier witness. It provides payload histories $a,b$, source witnesses $S(a)$ and $S(b)$, endpoint witnesses $\hsame(h,a)$ and $\hsame(k,b)$, and a payload classifier witness $a\sim_S b$. The compatibility hypothesis gives $h\sim_S a$ from $S(h)$, $S(a)$, and $\hsame(h,a)$; symmetry of $\hsame$ turns $\hsame(k,b)$ into $\hsame(b,k)$, and compatibility gives $b\sim_S k$ from $S(b)$, $S(k)$, and that symmetric endpoint witness. Two uses of $\mathsf{cert.equiv\_trans}$ compose $h\sim_S a$, $a\sim_S b$, and $b\sim_S k$ into $h\sim_S k$.

For the reverse direction, use Theorem~\ref{thm:tagged-option-history-present-present-exactness} with payloads $h$ and $k$. The source witnesses are the assumptions $S(h)$ and $S(k)$, the endpoint witnesses are reflexivity of $\hsame$ at $h$ and $k$, and the payload classifier witness is the assumed $h\sim_S k$.
\end{proof}

\begin{theorem}[Tagged option hsame present exactness]
\label{thm:tagged-option-history-hsame-present-exactness}
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$ and histories $h,k$,
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\hsame}(\Eone(h),\Eone(k))
\quad\Longleftrightarrow\quad
S(h)\land S(k)\land \hsame(h,k).
$$
\end{theorem}
\begin{proof}
Apply Theorem~\ref{thm:tagged-option-history-present-present-exactness} with source classifier $\hsame$. In the forward direction, the exactness witness gives payload histories $a,b$, source witnesses $S(a)$ and $S(b)$, endpoint links $\hsame(h,a)$ and $\hsame(k,b)$, and a payload link $\hsame(a,b)$. Eliminate the endpoint witnesses $\hsame(h,a)$ and $\hsame(k,b)$; after substitution, the displayed source witnesses are $S(h)$ and $S(k)$, and the remaining payload link is an $\hsame(h,k)$ witness. In the reverse direction, choose payloads $h$ and $k$ in Theorem~\ref{thm:tagged-option-history-present-present-exactness}; the two source witnesses are displayed, the endpoint links are reflexivity of $\hsame$, and the payload classifier witness is the displayed $\hsame(h,k)$.
\end{proof}

\begin{theorem}[Tagged option history carrier, pattern, and ledger equivalence]
\label{thm:tagged-option-history-carrier-pattern-ledger-equivalence}
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$ and history $h$, the tagged option history carrier is equivalent to both visible certificate presentations:
$$
\mathsf{TaggedOptionHistoryCarrier}_S(h)
\quad\Longleftrightarrow\quad
\mathsf{TaggedOptionHistoryPattern}_S(h)
$$
and
$$
\mathsf{TaggedOptionHistoryCarrier}_S(h)
\quad\Longleftrightarrow\quad
\mathsf{TaggedOptionHistoryLedger}_S(h).
$$
\end{theorem}
\begin{proof}
Unfold the three tagged option predicates. In the absent case, each presentation stores exactly the witness $\hsame(h,\emp)$, so the witness is reused unchanged in both directions. In the present case, each presentation stores the same payload history $a$ together with $S(a)$ and $\hsame(h,\Eone(a))$, so the same triple rebuilds the carrier, pattern, and ledger witnesses. The two cases exhaust the tagged option history carrier and the two visible certificate presentations.
\end{proof}

\begin{theorem}[Tagged option history carrier transport]
\label{thm:tagged-option-history-carrier-hsame-transport}
For any BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, if $\mathsf{TaggedOptionHistoryCarrier}_S(h)$ and $\hsame(h,k)$, then $\mathsf{TaggedOptionHistoryCarrier}_S(k)$.
\end{theorem}
\begin{proof}
Case-analyze the tagged carrier witness for $h$. In the absent case, compose the symmetry of $\hsame(h,k)$ with the stored endpoint witness $\hsame(h,\emp)$ to obtain $\hsame(k,\emp)$, then rebuild the absent clause. In the present case, keep the same payload $a$ and source witness $S(a)$. Compose the symmetry of $\hsame(h,k)$ with the stored endpoint witness $\hsame(h,\Eone(a))$ to obtain $\hsame(k,\Eone(a))$, then rebuild the present clause.
\end{proof}

\begin{theorem}[Tagged option history classifier endpoint transport]
\label{thm:tagged-option-history-classifier-hsame-transport}
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$,
source classifier $\sim_S$, and histories $h,k,h',k'$, if
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,k)\land

$$
holds, then
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h',k')
$$
holds.
\end{theorem}
\begin{proof}
Case-analyze the tagged classifier witness for $h$ and $k$. In the absent-absent

$\hsame(h,\emp)$ to obtain $\hsame(h',\emp)$, and compose the symmetry of
$\hsame(k,k')$ with the stored endpoint witness $\hsame(k,\emp)$ to obtain
$\hsame(k',\emp)$; these two witnesses rebuild the absent-absent classifier
branch. In the present-present branch, keep the same payload histories $a,b$,
source witnesses $S(a)$ and $S(b)$, and payload classifier witness
$a\sim_S b$. Compose the symmetry of $\hsame(h,h')$ with the stored endpoint
witness $\hsame(h,\Eone(a))$ to obtain $\hsame(h',\Eone(a))$, and compose the
symmetry of $\hsame(k,k')$ with the stored endpoint witness
$\hsame(k,\Eone(b))$ to obtain $\hsame(k',\Eone(b))$; these data rebuild the
present-present classifier branch.
\end{proof}

\begin{theorem}[Tagged option history classifier stability fields]
\label{thm:tagged-option-history-stability-fields}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, source classifier $\sim_S$, source certificate $\mathsf{cert}:\NameCert(S,\sim_S)$, and $\mathsf{TaggedOptionSourceHsameCompatible}(S,\sim_S)$, the following four clauses are exactly the core stability fields for the tagged option history carrier and classifier:
\begin{enumerate}
  \item carrier-relative reflexivity: if $\mathsf{TaggedOptionHistoryCarrier}_S(h)$ holds, then $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,h)$ holds;
  \item classifier symmetry: if $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,k)$ holds, then $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(k,h)$ holds;
  \item classifier transitivity: if $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,k)$ and $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(k,u)$ hold, then $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,u)$ holds;
  \item classifier-induced carrier transport: if $\mathsf{TaggedOptionHistoryCarrier}_S(h)$ and $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,k)$ hold, then $\mathsf{TaggedOptionHistoryCarrier}_S(k)$ holds.
\end{enumerate}
\end{theorem}
\begin{proof}
For reflexivity, case-analyze the carrier witness. The absent case rebuilds the absent classifier branch from the stored empty endpoint witness and reflexivity, symmetry, and transitivity of $\hsame$; the present case keeps the same payload and applies the reflexivity field of $\mathsf{cert}$ to the source carrier witness. For symmetry, case-analyze the tagged classifier witness. The absent branch reverses the stored endpoint links by symmetry of $\hsame$, and the present branch reverses the payload classifier with $\mathsf{cert.equiv\_symm}$ while swapping the two endpoint witnesses.

For transitivity, split the two classifier witnesses by their visible branches. In the absent/absent case, both witnesses expose the shared middle history as an empty endpoint, so transitivity of $\hsame$ composes the empty endpoint presentations and rebuilds the absent classifier branch from $h$ to $u$. In the absent/present case, the first witness presents the middle history as $\emp$, while the second presents the same middle history as $\Eone(b)$ for some source payload $b$; symmetry and transitivity of $\hsame$ give $\hsame(\emp,\Eone(b))$, contradicting empty-versus-one no-confusion. In the present/absent case, the first witness presents the middle history as $\Eone(b)$ and the second presents it as $\emp$, so the same no-confusion argument applies with the endpoints exchanged. In the present/present case, invert the two classifier witnesses around the shared middle history to obtain endpoint witnesses for $\Eone(b)$ and $\Eone(b')$. Symmetry and transitivity of $\hsame$ yield $\hsame(\Eone(b),\Eone(b'))$, $e1$-constructor inversion yields $\hsame(b,b')$, and source hsame compatibility gives $b\sim_S b'$. The field $\mathsf{cert.equiv\_trans}$ then composes $a\sim_S b$, $b\sim_S b'$, and $b'\sim_S c$ into $a\sim_S c$.

For carrier transport, case-analyze the classifier witness; the absent branch gives the empty endpoint carrier, and the present branch stores the target payload carrier directly. Same-tag exactness, right-absent exactness, and mixed constructor separation are isolated in Theorems~\ref{thm:tagged-option-history-same-tag-exactness}, \ref{thm:tagged-option-history-right-absent-exactness}, and \ref{thm:tagged-option-history-mixed-tag-separation}.
\end{proof}

\begin{theorem}[Tagged option history same-tag exactness]
\label{thm:tagged-option-history-same-tag-exactness}
For a BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$ and source classifier $\sim_S$, the classifier $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}$ from Definition~\ref{def:tagged-option-history-classifier} is exact on equal visible tags:
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\emp,k)
\quad\Longleftrightarrow\quad
\hsame(k,\emp)
$$
and
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\Eone(h),\Eone(k))
\quad\Longleftrightarrow\quad
\exists a,b.\; S(a)\land S(b)\land \hsame(h,a)\land \hsame(k,b)\land a\sim_S b.
$$
\end{theorem}
\begin{proof}
For the absent endpoint, case-analyze the tagged classifier. The absent-absent branch stores $\hsame(k,\emp)$, while the present-present branch would contain $\hsame(\emp,\Eone(a))$ and is eliminated by empty-versus-one no-confusion. Conversely, $\hsame(k,\emp)$ combines with reflexivity of $\hsame(\emp,\emp)$ to build the absent-absent classifier branch.

For the present endpoint, the absent-absent branch would contain $\hsame(\Eone(h),\emp)$ and is eliminated by one-versus-empty no-confusion. The present-present branch gives payload histories $a,b$, source witnesses $S(a)$ and $S(b)$, a source classifier witness $a\sim_S b$, and endpoint witnesses $\hsame(\Eone(h),\Eone(a))$ and $\hsame(\Eone(k),\Eone(b))$; one-constructor inversion turns these endpoint witnesses into $\hsame(h,a)$ and $\hsame(k,b)$. Conversely, such payload data rebuilds the present-present classifier branch after applying the one-constructor congruence for $\hsame$ to the two payload endpoint witnesses.
\end{proof}

\begin{theorem}[Tagged option history right absent exactness]
\label{thm:tagged-option-history-right-absent-exactness}
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$, source classifier $\sim_S$, and history $h$, the tagged option history classifier has the exact right absent endpoint:
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(h,\emp)
\quad\Longleftrightarrow\quad
\hsame(h,\emp).
$$
\end{theorem}
\begin{proof}
For the forward direction, case-analyze the tagged classifier witness. The absent-absent branch directly provides the endpoint witness $\hsame(h,\emp)$. The present-present branch would force $\hsame(\emp,\Eone(b))$ for some source payload $b$, contradicting BEDC history no-confusion between the empty endpoint and a one-extension endpoint. For the reverse direction, combine the assumed $\hsame(h,\emp)$ with reflexivity of $\hsame$ at $\emp$ and rebuild the absent-absent classifier branch.
\end{proof}

\begin{theorem}[Tagged option history mixed-tag separation]

are impossible:
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\emp,\Eone(k))\to\bot
$$
and
$$
\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\Eone(h),\emp)\to\bot.

$\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\emp,\Eone(k))$, the

contradict BEDC history no-confusion between empty and one-extension

$\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}(\Eone(h),\emp)$ is

$\hsame(\Eone(h),\emp)$, and the present-present branch forces an endpoint

\begin{theorem}[Tagged option history carrier-level mixed-tag separation]
\label{thm:tagged-option-history-carrier-mixed-tag-separation}
This theorem fits the per-object template as follows: the traditional target is
absent/present option tag separation; the BEDC source is $\mathsf{BHist}$ with
$\hsame$; the carrier is $\mathsf{TaggedOptionHistoryCarrier}$; the classifier is
$\mathsf{TaggedOptionHistoryClassifier}$; the stability obligation is mixed-tag
no-confusion under carrier presentations; the ledger is the visible tag; the
core theorem is this separation; exactness is absent-vs-present impossibility;
the standard bridge is none/some; and the status is $\mathsf{ProofTarget}$.
For every BEDC history source carrier $S:\mathsf{BHist}\to\mathsf{Prop}$,
source classifier $\mathsf{sameS}:\mathsf{BHist}\to\mathsf{BHist}\to\mathsf{Prop}$,
histories $h,k$, and payload history $a$, the classifier cannot relate endpoints
that are merely presented as opposite tags:
$$
\hsame(h,\emp)\to S(a)\to \hsame(k,\Eone(a))\to
\mathsf{TaggedOptionHistoryClassifier}_{S,\mathsf{sameS}}(h,k)\to\bot
$$
and
$$
S(a)\to \hsame(h,\Eone(a))\to \hsame(k,\emp)\to
\mathsf{TaggedOptionHistoryClassifier}_{S,\mathsf{sameS}}(h,k)\to\bot.
$$
\end{theorem}
\begin{proof}
Case-analyze the tagged classifier witness. In the first direction, the
absent-absent branch stores an endpoint witness from $k$ to $\emp$; composing it
with the displayed witness from $k$ to $\Eone(a)$ gives empty-versus-one
no-confusion. The present-present branch stores an endpoint witness from $h$ to
some $\Eone(b)$; composing it with the displayed witness from $h$ to $\emp$ again
gives empty-versus-one no-confusion. The second direction is symmetric: the
absent-absent branch contradicts the displayed present presentation of $h$, and
the present-present branch contradicts the displayed absent presentation of $k$.
\end{proof}

\begin{theorem}[Tagged option history semantic certificate target]
\label{thm:tagged-option-history-semantic-certificate}
Let $S:\mathsf{BHist}\to\mathsf{Prop}$ be a source carrier, let $\sim_S$ be a source classifier on source histories, and assume a source $\NameCert(S,\sim_S)$ together with $\mathsf{TaggedOptionSourceHsameCompatible}(S,\sim_S)$. Then $\mathsf{SemanticNameCert}(\mathsf{TaggedOptionHistoryCarrier}_S,\mathsf{TaggedOptionHistoryPattern}_S,\mathsf{TaggedOptionHistoryLedger}_S,\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S})$ is the tagged option history semantic certificate.
\end{theorem}
\begin{proof}
Inhabitation is supplied by the absent carrier witness at $\emp$. Reflexivity is built by case analysis on $\mathsf{TaggedOptionHistoryCarrier}_S(h)$: the absent endpoint gives the absent-absent classifier at $\emp$, while the present endpoint uses the source certificate field $\mathsf{equiv\_refl}$ on the displayed payload. Symmetry is obtained by inverting the classifier witness; absent-absent symmetry uses symmetry of $\hsame$ at $\emp$, and present-present symmetry applies the source certificate field $\mathsf{equiv\_symm}$ to the payload classifier. For present-present transitivity, unpack the two classifier witnesses into payload data $(a,b)$ and $(b',c)$. The shared middle history has endpoint witnesses $\hsame(m,\Eone(b))$ and $\hsame(m,\Eone(b'))$, so symmetry and transitivity of $\hsame$ give $\hsame(\Eone(b),\Eone(b'))$. Applying $e1$-constructor inversion gives $\hsame(b,b')$; source hsame compatibility classifies the two middle payload presentations as $b\sim_S b'$. The source certificate transitivity field first composes $a\sim_S b$ with $b\sim_S b'$, then composes the result with $b'\sim_S c$. Theorems~\ref{thm:tagged-option-history-mixed-tag-separation} and \ref{thm:tagged-option-history-same-tag-exactness} eliminate mixed absent-present classifications and invert present-present classifications to payload histories. Carrier transport follows by cases on $\mathsf{TaggedOptionHistoryClassifier}_{S,\sim_S}$: absent transport uses the empty endpoint case, and present transport uses the $S(b)$ field already stored in the present-present classifier case. The visible tag pattern and visible tag ledger fields are supplied by Theorem~\ref{thm:tagged-option-history-carrier-pattern-ledger-equivalence}, which identifies both certificate presentations with the tagged option history carrier.
\end{proof}

\begin{corollary}[Tagged option history certificate over hsame]
\label{cor:tagged-option-history-semantic-certificate-hsame}
Let $S:\mathsf{BHist}\to\mathsf{Prop}$ be a source carrier. If a source $\NameCert(S,\hsame)$ is supplied, then $\mathsf{SemanticNameCert}(\mathsf{TaggedOptionHistoryCarrier}_S,\mathsf{TaggedOptionHistoryPattern}_S,\mathsf{TaggedOptionHistoryLedger}_S,\mathsf{TaggedOptionHistoryClassifier}_{S,\hsame})$ is the tagged option history semantic certificate with no separate compatibility hypothesis.
\end{corollary}
\begin{proof}
Apply Theorem~\ref{thm:tagged-option-history-semantic-certificate} with source classifier $\hsame$. The required compatibility input is exactly Theorem~\ref{thm:tagged-option-source-hsame-compatibility-hsame}.
\end{proof}

\begin{theorem}[Option history classifier endpoint transport]
\label{thm:option-history-classifier-hsame-transport}
If the optional history classifier relates \(h\) to \(k\), and
\(\hsame(h,h')\) and \(\hsame(k,k')\), then it relates \(h'\) to \(k'\).
\end{theorem}
\begin{proof}
Unpack the classifier into its endpoint carrier witnesses and the stored
\(\hsame(h,k)\) link. Transport both carriers along the endpoint sameness
links, then compose symmetry of \(\hsame(h,h')\), the stored link, and
\(\hsame(k,k')\).
\end{proof}

\begin{theorem}[Option history classifier empty left iff]
\label{thm:option-history-classifier-empty-left-iff}
For any source predicate on BEDC histories, the optional history classifier
relates the empty history on the left to \(k\) exactly when \(k\) is
history-same as the empty history.
\end{theorem}
\begin{proof}
In the forward direction, unpack the classifier and reverse its stored
\(\hsame(\emp,k)\) link. In the reverse direction, both endpoint carrier
witnesses are supplied by the absent-history branch, and the classifier link is
the symmetric form of the assumed \(\hsame(k,\emp)\).
\end{proof}
\leanchecked{BEDC.Derived.OptionUp.OptionHistoryClassifier\_empty\_left\_iff}

\begin{theorem}[Option history classifier empty right iff]
\label{thm:option-history-classifier-empty-right-iff}
For any source predicate $S:\mathsf{BHist}\to\mathsf{Prop}$ and any history $h$,
$$
\mathsf{OptionHistoryClassifier}_S(h,\emp)\quad\Longleftrightarrow\quad\hsame(h,\emp).
$$
\end{theorem}
\begin{proof}
In the forward direction, apply classifier symmetry to reduce to
$\mathsf{OptionHistoryClassifier}_S(\emp,h)$, then use
Theorem~\ref{thm:option-history-classifier-empty-left-iff}. In the reverse
direction, use the reverse implication of
Theorem~\ref{thm:option-history-classifier-empty-left-iff} to obtain
$\mathsf{OptionHistoryClassifier}_S(\emp,h)$, then apply classifier symmetry.
\end{proof}

\begin{theorem}[Option classifier symmetry from a source certificate]

\begin{definition}[Option stability certificate]
\label{def:option-stability-certificate}
The stability certificate separates the typed optional classifier fields from
the nullable history-envelope fields.

For the typed $\mathsf{OptionClassifierSpec}$ classifier, the fields are:
\begin{enumerate}
  \item reflexivity for $\mathsf{none}$;
  \item reflexivity for $\mathsf{some}(a)$ from source reflexivity;
  \item transitivity from a source-transitivity hypothesis;
  \item no-confusion between $\mathsf{none}$ and $\mathsf{some}(a)$ in either order;
  \item present inversion, exposing the source classifier witness from a $\mathsf{some}$--$\mathsf{some}$ classification.
\end{enumerate}
Symmetry for this typed classifier is supplied only from a source-symmetry
hypothesis; the checked optional-history instance specializes that principle to
heads classified by $\hsame$.

For the nullable $\mathsf{OptionHistoryCarrier}$ envelope and its
$\mathsf{OptionHistoryClassifier}$, the certificate fields are:
\begin{enumerate}
  \item carrier transport across $\hsame$;
  \item endpoint transport for the classifier across $\hsame$ on both endpoints;
  \item classifier symmetry;
  \item classifier transitivity;
  \item ledger transport across raw and visible endpoint $\hsame$ links.
\end{enumerate}
\end{definition}
\leanchecked{BEDC.Derived.OptionUp.option\_stability\_certificate\_fields}

\begin{corollary}[OptionUp certificate scope]
\label{cor:option-certificate-scope}
The checked envelope semantic certificate licenses the empty-or-source history
envelope only. It is not a checked constructor-tagged $\OptionUp$ certificate:
citations to Theorem~\ref{thm:concrete-option-namecert} assert the nullable
history-envelope certificate, while the explicit optional value interface is
licensed only when the typed tag classifier $\mathsf{OptionClassifierSpec}$,
none/some no-confusion, some-inversion, the tag-plus-source ledger, and either
a $\mathsf{TaggedOptionSourceHsameCompatible}(S,\sim_S)$ witness or the
specialized $\hsame$ classifier certificate are supplied together with that
envelope.
\end{corollary}

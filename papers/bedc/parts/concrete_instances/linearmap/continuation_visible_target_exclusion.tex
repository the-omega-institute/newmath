\begin{theorem}[Singleton evaluation continuation visible target exclusion]
\label{thm:linearmap-singleton-eval-continuation-visible-target-exclusion}
For histories $f,x,y,r,p:\mathsf{BHist}$, if
$$
\ContR(\mathsf{LinearMapSingletonEval}(f,x),y,r),
$$
then both visible target classifier claims are impossible:
$$
\mathsf{LinearMapSingletonClassifier}(r,\Ezero(p))\Longrightarrow\bot,
\qquad
\mathsf{LinearMapSingletonClassifier}(r,\Eone(p))\Longrightarrow\bot.
$$
The ten-field fit is: traditional target terminal linear-map continuation
result exclusion; BEDC source $\mathsf{BHist}$, $\ContR$, and the singleton
$\LinearMapUp$ evaluator; carrier $\mathsf{LinearMapSingletonCarrier}$;
classifier $\mathsf{LinearMapSingletonClassifier}$; stability from
continuation classifier readback and constructor no-confusion; ledger
continuation result endpoint; core theorem this block; exactness visible-target
impossibility; standard bridge terminal linear hom; status
$\mathsf{ProofTarget}$.
\end{theorem}
\begin{proof}
Apply Theorem~\ref{thm:linearmap-singleton-eval-continuation-classifier} to
the displayed continuation with target $\Ezero(p)$. A classifier witness for
$\Ezero(p)$ then forces
$\mathsf{LinearMapSingletonCarrier}(\Ezero(p))$.

Apply the same continuation classifier readback with target $\Eone(p)$. A
classifier witness for $\Eone(p)$ forces
$\mathsf{LinearMapSingletonCarrier}(\Eone(p))$.

Unfolding the singleton carrier makes each witness a history-sameness proof
from the visible endpoint to $\emp$. Constructor no-confusion for $\Ezero$ and
$\Eone$ contradicts those histories being empty, so both displayed classifier
claims imply $\bot$.
\end{proof}

\chapter{A Concrete Naming Certificate for $\SequentialCauchyRegularizationUp$}
\label{ch:concrete-instances-sequential-cauchy-regularization-namecert}
\origin{human}

Following Bishop and Bridges, the $\SequentialCauchyRegularizationUp$ packet records the finite route by which ordinary Cauchy stream data is regularized into a displayed regular Cauchy name before the terminal Real seal is read. It sits over the displayed threshold discipline of \autoref{ch:concrete-instances-cauchyregularization-namecert}, the finite window service of \autoref{ch:concrete-instances-streamname-namecert}, the rational readback of \autoref{ch:concrete-instances-regseqrat-namecert}, the tolerance ledger of \autoref{ch:concrete-instances-dyadicratcore-namecert}, and the Real seal of \autoref{ch:concrete-instances-real-namecert}. The packet names a sequential regularization certificate only; it does not identify quotient streams, choose a limit, use countable choice, invoke trichotomy, or assume ambient $\RealUp$ completeness.

\begin{definition}[Sequential Cauchy regularization carrier]
\label{def:sequential-cauchy-regularization-carrier}
A $\SequentialCauchyRegularizationUp$ carrier is a finite $\BHist$ packet
$$
S=(X,T,W,R,D,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $X$ is the source Cauchy stream row. It records the ordinary sequential Cauchy data through finite requests, not through a quotient stream or selected limit.
\item $T$ is the displayed regularization threshold row imported from the $\CauchyRegularizationUp$ route. It records, for each requested precision, the finite stage after which the source windows are regular enough to serve the requested name.
\item $W$ is the $\StreamNameUp$ window row naming the finite source windows and the regularized output windows read at the threshold supplied by $T$.
\item $R$ is the $\RegSeqRatUp$ readback row for the regularized rational name exposed by $T$ and $W$.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger for the source-window comparisons, threshold requests, regularized readbacks, and Real-facing output tolerance.
\item $E$ is the terminal $\RealUp$ seal reached only after the source row, regularization threshold, stream windows, readback, and dyadic tolerance ledger have been displayed.
\item $H$ records componentwise $\hsame$ transport for the source stream, threshold row, window row, readback row, tolerance ledger, Real seal, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a sequential regularization request through $X,T,W,R,D,E,H,P,N$.
\item $P$ is the $\Pkg$ provenance over $\SequentialCauchyRegularizationUp$, $\CauchyRegularizationUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\SequentialCauchyRegularizationUp}$ row.
\end{enumerate}
The carrier predicate asserts exactly that these rows form one finite sequential regularization history. It has no coordinate for quotient stream equality, selected limits, countable choice, trichotomy, host sequence equality, or ambient Real completeness.
\end{definition}

\begin{theorem}[Sequential Cauchy regularization NameCert obligations]
\label{thm:sequential-cauchy-regularization-namecert-obligations}
For every accepted $\SequentialCauchyRegularizationUp$ carrier $S=(X,T,W,R,D,E,H,C,P,N)$, the local $\NameCert_{\SequentialCauchyRegularizationUp}$ surface consists exactly of the following obligation rows:
\begin{enumerate}
\item carrier admission for the source Cauchy stream, displayed regularization threshold, StreamName windows, RegSeqRat readback, DyadicRatCore tolerance ledger, Real seal, transport, replay, provenance, and local naming;
\item threshold stability under componentwise $\hsame$ transport of $X,T,W,D$;
\item regularized readback compatibility between the threshold row $T$, the window row $W$, the rational readback row $R$, and the tolerance ledger $D$;
\item $\RealUp$ seal nonescape through $E$, excluding any read that bypasses $X,T,W,R,D$;
\item displayed $\Cont$ replay and $\Pkg$ provenance for the whole route;
\item refusal of quotient streams, selected limits, countable choice, trichotomy, host sequence equality, and ambient Real completeness as certificate data.
\end{enumerate}
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:sequential-cauchy-regularization-carrier}. The analytic rows are the source stream $X$, the regularization threshold $T$, the window row $W$, the readback row $R$, the tolerance ledger $D$, and the terminal seal $E$.

The threshold obligation is read from $T$ together with $W$ and $D$. The row $T$ names the finite stage supplied by the $\CauchyRegularizationUp$ route, while $W$ and $D$ record the finite windows and tolerance comparisons checked at that stage.

The readback and seal obligations are read from $R$ and $E$. The regularized rational name is exposed only through $R$, and the Real-facing seal is downstream of the same threshold, window, and tolerance data.

Finally $H,C,P,N$ transport, replay, record provenance for, and locally name the same finite route. Since the carrier has no quotient-stream, selected-limit, choice, trichotomy, host-equality, or completeness coordinate, the listed rows exhaust the local certificate surface.
\end{proof}

\begin{theorem}[Sequential Cauchy regularization Real-seal route]
\label{thm:sequential-cauchy-regularization-real-seal-route}
Every $\RealUp$-facing consumer of an accepted $\SequentialCauchyRegularizationUp$ packet factors through the finite route
$$
X \longmapsto T \longmapsto W,D \longmapsto R \longmapsto E,H,C,P,N .
$$
Thus the consumer first reads the source Cauchy stream, then the displayed regularization threshold from $\CauchyRegularizationUp$, then the $\StreamNameUp$ windows and $\DyadicRatCoreUp$ tolerance ledger, then the $\RegSeqRatUp$ readback, and only then the terminal $\RealUp$ seal. The route exports no quotient stream, selected limit, countable-choice selector, trichotomy row, host sequence equality, or ambient completeness theorem.
\end{theorem}
\begin{proof}
Start with an accepted packet $S=(X,T,W,R,D,E,H,C,P,N)$. The only source coordinate is $X$, so any sequential Cauchy read begins with the displayed source stream row.

The regularization stage is $T,W,D$. The threshold row $T$ supplies the finite stage used by the regularization route, and the window and tolerance rows record the exact finite observations made at that stage.

The readback stage is $R$. It packages the regularized windows as a regular rational name before any Real-facing seal is available.

The terminal stage is $E,H,C,P,N$. These rows seal, transport, replay, record provenance for, and locally name the same route, and none of them adds quotient, selection, choice, trichotomy, host-equality, or completeness data.
\end{proof}

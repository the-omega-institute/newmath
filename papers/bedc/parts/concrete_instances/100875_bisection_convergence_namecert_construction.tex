\begin{definition}[Bisection convergence carrier]
\label{def:bisection-convergence-carrier}
A $\BisectionConvergenceUp$ carrier is a finite $\BHist$ packet
$$
B=(I,D,W,T,S,Q,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $I$ is the located interval source row. It records the initial lower and upper endpoint windows, locatedness reads, endpoint order data, and the interval identity consumed by the finite bisection request.
\item $D$ is the $\DyadicBisectionUp$ branch ledger. It stores the finite left-or-right branch prefix, midpoint cells, child-window records, and branch depth used by the requested tolerance.
\item $W$ is the retained $\NestedIntervalUp$ window row. It records the lower and upper retained windows after each branch listed in $D$.
\item $T$ is the tail-diameter schedule. It assigns to each requested tolerance a displayed branch depth and the retained-window diameter bound available at that depth.
\item $S$ is the $\StreamNameUp$ window row opened from $W$ at the depth named by $T$.
\item $Q$ is the $\RegSeqRatUp$ readback row obtained from the finite StreamName windows and the displayed diameter budget.
\item $E$ is the terminal $\RealUp$ seal row exposed only after the retained windows, tail-diameter schedule, StreamName windows, and RegSeqRat readback have been displayed.
\item $H$ records componentwise $\hsame$ transport for the located interval source, branch ledger, retained windows, tail schedule, stream windows, regular readback, Real seal, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a convergence request through interval inspection, finite branch retention, tail-diameter scheduling, finite window opening, regular readback, and Real-seal consumption.
\item $P$ is the $\Pkg$ provenance over $\BisectionConvergenceUp$, $\DyadicBisectionUp$, $\LocatedIntervalBisectionUp$, $\NestedIntervalUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\BisectionConvergenceUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $I,D,W,T,S,Q,E,H,C,P,N$. It has no coordinate for selected limits, quotient stream equality, real trichotomy, countable choice, open-cover compactness, ambient $\RealUp$ completeness, host interval equality, Lean verification, or bridge checking.
\end{definition}

\begin{theorem}[Bisection convergence NameCert obligations]
\label{thm:bisection-convergence-namecert-obligations}
For every accepted $\BisectionConvergenceUp$ carrier
$$
B=(I,D,W,T,S,Q,E,H,C,P,N),
$$
the local $\NameCert_{\BisectionConvergenceUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the located interval source, dyadic branch ledger, retained nested windows, tail-diameter schedule, StreamName windows, RegSeqRat readback, Real seal, transports, replay, provenance, and local naming;
\item source discipline, requiring every endpoint, locatedness, and order read to factor through $I$ before any branch or tail row is inspected;
\item branch-retention discipline, requiring $D$ to display the finite branch prefix and requiring $W$ to store only the retained windows produced by that prefix;
\item tail-diameter exactness, requiring $T$ to attach each requested tolerance to a displayed branch depth and a displayed retained-window diameter bound;
\item finite readback discipline, requiring $S$ and $Q$ to consume the retained window and tail schedule before any regular rational observation is visible;
\item Real-seal non-escape, requiring $E,H,C,P,N$ to export only the displayed finite route and no selected limit, quotient stream equality, real trichotomy, countable choice, open-cover compactness, ambient $\RealUp$ completeness, host interval equality, Lean verification, or bridge data.
\end{enumerate}
No bisection convergence obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:bisection-convergence-carrier}. The visible packet consists exactly of the source interval row $I$, branch ledger $D$, retained-window row $W$, tail-diameter schedule $T$, StreamName row $S$, RegSeqRat readback $Q$, Real seal $E$, and structural rows $H,C,P,N$.

Source discipline is the first stage of the route. The endpoint windows, locatedness reads, and order data are supplied by $I$, so a convergence request cannot replace the source by a host interval equality theorem or by a completed limit object.

Branch-retention discipline is the middle stage. The ledger $D$ displays the finite branch prefix and midpoint cells, while $W$ records only the retained windows produced by that ledger.

Tail-diameter exactness is local to $T$. For each requested tolerance, the schedule stores the branch depth and the corresponding retained-window diameter bound; no unrecorded convergence principle or countable choice operator chooses the depth.

Finally, $S,Q,E,H,C,P,N$ expose finite windows, regular readback, terminal seal, transport, replay, provenance, and naming along the same displayed route. Since the excluded objects have no carrier coordinate, no local NameCert obligation can export them.
\end{proof}

\begin{theorem}[Bisection convergence tail-diameter route]
\label{thm:bisection-convergence-tail-diameter}
Let
$$
B=(I,D,W,T,S,Q,E,H,C,P,N)
$$
be an accepted $\BisectionConvergenceUp$ carrier. Every accepted convergence request factors through the finite route
$$
I \longmapsto D \longmapsto W \longmapsto T
$$
before any $\StreamNameUp$ or $\RegSeqRatUp$ readback is exposed. Consequently each requested tolerance is answered by a displayed branch depth and retained-window diameter bound rather than by a selected limit or ambient completeness principle.
\end{theorem}
\begin{proof}
Project the accepted carrier. The only source interval coordinate is $I$, and the only branch coordinate is $D$, so the request first reads the located interval data and then the finite branch prefix.

The retained-window row $W$ is downstream of that branch prefix. It stores the lower and upper windows that remain after the displayed bisection choices, and it provides the visible window data consumed by the tail schedule.

The tail-diameter row $T$ then answers the tolerance request. It names the branch depth and diameter bound available for the retained window at that depth.

Since $S$ and $Q$ occur only after $T$ in the carrier route, no StreamName or RegSeqRat readback is visible before the finite diameter schedule has been displayed. The route therefore supplies finite convergence data without selected limits or ambient completeness.
\end{proof}

\label{thm:bisection-convergence-real-seal-handoff}

I \longmapsto D \longmapsto W \longmapsto T \longmapsto S \longmapsto Q \longmapsto E

\begin{proof}

The finite observation stage is $S$ followed by $Q$. The StreamName row opens the retained windows at the scheduled depth, and the RegSeqRat row reads back only those displayed finite observations under the recorded diameter budget.

The Real seal $E$ is terminal with respect to that route. Transport, replay, provenance, and local naming through $H,C,P,N$ preserve the same displayed data and introduce no analytic coordinate beyond it.

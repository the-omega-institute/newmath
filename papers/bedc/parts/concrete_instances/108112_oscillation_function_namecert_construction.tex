\begin{theorem}[OscillationFunction NameCert obligations]
\label{thm:oscillation-function-namecert-obligations}
For every accepted $\OscillationFunctionUp$ carrier $O=(P,B,D,S,R,V,H,C,K,N)$, the local $\NameCert_{\OscillationFunctionUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the pointwise-limit source, Baire-one consumer, Dini-continuity consumer, stream schedule, regular rational readback, RealUp oscillation seal, transport, replay, provenance, and local naming row;
\item pointwise-window exposure, requiring every oscillation request to read the $\PointwiseLimitUp$ route of \autoref{ch:concrete-instances-pointwise-limit-namecert} before any value-difference comparison is inspected;
\item consumer compatibility, requiring the $\BaireOneFunctionUp$ and $\DiniContinuityUp$ rows to consume the same displayed windows and not a second host function-space object;
\item Real-seal exactness, requiring the oscillation value row $V$ to be downstream of $P,B,D,S,R$ and transported only by $H$ and $C$;
\item nonescape from host oscillation functions, quotient real equality, selected suprema, ambient Baire-one representation theorems, unrestricted Dini compactness, and completeness of $\RealUp$.
\end{enumerate}
No NameCert obligation is stored outside $P,B,D,S,R,V,H,C,K,N$.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:oscillation-function-carrier}. The accepted packet exposes exactly the pointwise-limit source row, the Baire-one and Dini consumer rows, the finite stream schedule, the regular rational readback, the RealUp value seal, transport, replay, provenance, and local naming.

The pointwise-window exposure follows from the source row $P$. The route cited in \autoref{ch:concrete-instances-pointwise-limit-namecert} already displays the point-index, approximant windows, regular rational readback, and RealUp value seal before a pointwise value can be consumed.

Consumer compatibility is then read through $B$ and $D$. Both rows are coordinates of the same packet, so the Baire-one and Dini-facing consumers receive the finite schedule $S$ and readback $R$ already attached to $P$ rather than an external function-space witness.

Finally the seal row $V$ is downstream of this route. Componentwise transport by $H$ and replay by $C$ preserve only the displayed rows, while $K$ and $N$ carry provenance and local naming. Since the excluded objects are not coordinates of the carrier, the listed rows exhaust the local NameCert surface.
\end{proof}

$$

\item $B$ is the target $\BorelSetUp$ row whose generated-set boundary supplies the target event being pulled back.
\item $R$ is the pullback ledger assigning the displayed target Borel row in $B$ to a displayed source Borel row over $X$.
\item $Q$ is the $\RealUp$ readback row used when the target Borel query is real-facing; it is consumed only through the displayed Borel row and the pullback ledger.

\begin{theorem}[BorelFunction NameCert obligations]
\label{thm:borel-function-namecert-obligations}

\end{theorem}
\begin{proof}

Read a real-facing target query through $B$ and $Q$. The readback row $Q$ is downstream of the displayed Borel row, while the source-side result is named only by the pullback ledger $R$ over $X$.

\label{thm:borel-function-pullback-stability}

\chapter{A Concrete Naming Certificate for $\AuthorizedGeneratorRecursorUp$}
\label{ch:concrete-instances-authorizedgeneratorrecursor-namecert}
\origin{ai}

The $\AuthorizedGeneratorRecursorUp$ surface turns the recursor authorization principle of \autoref{ch:recursor-as-authorized-generator} into a finite NameCert packet. The packet records an accepted constructor signature, a legal eliminator name, complete branch data, structural descent evidence, output naming, audit provenance, and the visible boundary at which BEDC relies on the host kernel's acceptance of the recursor call. It does not add a new eliminator, a new induction principle, or a hidden source of proof terms outside the accepted signature.

The audit-provenance row $A$ is the local consumer of the $\OperationalMembraneUp$ audit carrier in \autoref{ch:concrete-instances-operationalmembrane-namecert}. It reads source-axiom refusal, axiom-purity ancestry, and import-separation rows before the recursor output row $O$ becomes public.

\begin{definition}[AuthorizedGeneratorRecursor carrier]
\label{def:authorized-generator-recursor-carrier}
An $\AuthorizedGeneratorRecursorUp$ carrier is a finite $\BHist$ packet
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $I$ is the accepted constructor-signature row, naming the finite constructors of the inductive carrier and the strict-positivity gate that admits them;
\item $E$ is the emitted eliminator row, naming the corresponding recursor $I.\mathrm{rec}$ as the only generator admitted by this packet;
\item $M$ is the checked motive row, recording the output family or target judgment selected for the recursor call;
\item $B$ is the branch-package row, containing exactly one accepted branch for each constructor listed by $I$;
\item $D$ is the structural-descent row, recording the recursive arguments and the kernel-visible decrease or constructor subterm relation used by the call;
\item $O$ is the output-name row, naming the term, proof object, or definition produced by the recursor call;
\item $A$ is the audit-provenance row, recording the build check, source-axiom refusal, and transitive dependency refusal relevant to the generated output;
\item $H$ records componentwise $\hsame$ transport for $I,E,M,B,D,O,A$, package provenance, boundary data, and the local naming row;
\item $C$ records the displayed $\Cont$ routes by which downstream consumers read the signature, eliminator, motive, branch package, descent row, output name, and audit provenance;
\item $P$ is the $\Pkg$ provenance over $\BHist$, constructor signatures, recursor names, motive rows, branch packages, structural-descent rows, output names, audit reports, $\hsame$, $\Cont$, $\mathsf{GAP}$, and $\NameCert$;
\item $G$ is the $\mathsf{GAP}$ boundary row recording that the packet relies on host-kernel acceptance of the displayed signature and recursor call, while refusing any unlisted branch, non-structural recursion channel, external axiom, or hidden proof-producing generator;
\item $N$ is the local $\NameCert_{\AuthorizedGeneratorRecursorUp}$ row.
\end{enumerate}
The carrier contains no coordinate for an extra constructor, a missing branch, an unchecked motive, a non-decreasing recursive call, an output without a name, an unaudited dependency channel, or a proof term generated outside $I.\mathrm{rec}$ and the accepted constructor signature.
\end{definition}

\begin{theorem}[AuthorizedGeneratorRecursor branch exhaustion]
\label{thm:authorized-generator-recursor-branch-exhaustion}
For an accepted $\AuthorizedGeneratorRecursorUp$ packet, the branch-package row $B$ exhausts the generator surface named by $E$. Every public use of the packet reads one branch for each constructor displayed in $I$, transports those branches only through the componentwise row $H$, and cannot recover any additional branch or constructor case from the packet.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:authorized-generator-recursor-carrier}. The constructor signature $I$ and emitted eliminator $E$ are displayed together, and $B$ is declared to contain exactly one accepted branch for each constructor listed by $I$.

Read downstream access through $C$. A consumer reaches the branch package only by the displayed continuation route, so the public surface is the finite list already paired with $I$.

Transport through $H$ preserves the same component rows. Since the tuple has no coordinate for an extra constructor or unlisted branch, the branch surface is exhausted by the entries already recorded in $B$.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor NameCert surface]
\label{thm:authorized-generator-recursor-namecert-surface}
The local $\NameCert_{\AuthorizedGeneratorRecursorUp}$ surface for the carrier of \autoref{def:authorized-generator-recursor-carrier} consists exactly of signature admission, recursor naming, motive checking, branch exhaustion, structural-descent visibility, output naming, audit provenance, componentwise transport, continuation locality, package provenance, and the displayed $\mathsf{GAP}$ boundary. A downstream consumer may treat $O$ as a recursor-generated named output only through the packet
$$
(I,E,M,B,D,O,A,H,C,P,G,N).
$$
\end{theorem}
\begin{proof}
Carrier exposure supplies the finite tuple. The rows $I,E,M,B,D,O,A,H,C,P,G,N$ are the only rows admitted to the local NameCert surface.

Signature admission and recursor naming are read from $I$ and $E$. Motive checking, branch exhaustion, and structural-descent visibility are read from $M,B,D$, with branch exhaustion supplied by \autoref{thm:authorized-generator-recursor-branch-exhaustion}.

The output becomes public through $O$ only after the audit row $A$, transport row $H$, continuation row $C$, package row $P$, boundary row $G$, and local certificate row $N$ are retained. No coordinate in the packet can replace these rows with an unnamed proof term or an external generator.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor audit boundary]
\label{thm:authorized-generator-recursor-audit-boundary}
An accepted $\AuthorizedGeneratorRecursorUp$ packet binds the audit boundary for recursor-authorized proof generation. The packet certifies that the displayed output is public only as the named result of the accepted signature, emitted eliminator, checked motive, complete branch package, structural-descent row, and audit-provenance row. It does not certify kernel completeness, global termination theory, semantic conservativity for all inductive declarations, or any output whose dependency chain escapes the displayed audit provenance.
\end{theorem}
\begin{proof}
Start with the NameCert surface of \autoref{thm:authorized-generator-recursor-namecert-surface}. It forces the output row $O$ to be read together with the constructor signature, eliminator, motive, branch package, descent row, and audit row.

The boundary row $G$ records the host-kernel handoff. It names the reliance on kernel acceptance of the signature and recursor call while refusing unlisted branches, non-structural recursion, external axioms, and hidden proof generators.

The excluded claims have no coordinate in the carrier. Therefore the packet gives a seed certificate for the displayed recursor-generated output and does not export global kernel completeness, unrestricted termination, general semantic conservativity, or an unaudited dependency surface.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor structural descent locality]
\label{thm:authorized-generator-recursor-structural-descent-locality}
Every public recursive read of an accepted $\AuthorizedGeneratorRecursorUp$ packet factors through the displayed constructor signature $I$, emitted eliminator $E$, checked motive $M$, branch package $B$, and structural-descent row $D$. The packet admits no public recursive call whose descent is not recorded in $D$, and it admits no branch read outside the constructor list displayed by $I$ and exhausted by $B$.
\end{theorem}
\begin{proof}
Project the carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N).
$$
The only recursive generator named by the packet is the emitted eliminator row $E$, and it is paired with the constructor signature $I$, motive row $M$, branch package $B$, and descent row $D$.

Read access through $C$ exposes those rows as the local public interface. By \autoref{thm:authorized-generator-recursor-branch-exhaustion}, the branch package already contains exactly the public branches available for the constructors displayed in $I$.

The boundary row $G$ refuses non-structural recursion and unlisted branches. Since the tuple has no coordinate for a second recursion channel, a non-decreasing recursive call, or an extra branch source, every public recursive read is local to $I,E,M,B,D$.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor audit provenance factorization]
\label{thm:authorized-generator-recursor-audit-provenance-factorization}
For an accepted $\AuthorizedGeneratorRecursorUp$ packet, the output row $O$ becomes public only after the audit-provenance row $A$, componentwise transports $H$, continuation routes $C$, package row $P$, boundary row $G$, and local name row $N$ are retained. Thus public output exposure factors through the audit boundary of \autoref{thm:authorized-generator-recursor-audit-boundary} and cannot be read as an unaudited recursor result.
\end{theorem}
\begin{proof}
Project the carrier and retain the rows $O,A,H,C,P,G,N$. The NameCert surface of \autoref{thm:authorized-generator-recursor-namecert-surface} states that $O$ is part of the local certificate only inside the displayed packet.

Apply \autoref{thm:authorized-generator-recursor-audit-boundary}. It binds the public output to the accepted signature, emitted eliminator, checked motive, complete branch package, structural-descent row, and audit-provenance row.

The rows $H,C,P,G,N$ keep the transports, public routes, package provenance, boundary refusal, and local naming certificate attached to that exposure. No coordinate remains from which a consumer can read $O$ apart from $A,H,C,P,G,N$, so the output factors through the displayed audit provenance.
\end{proof}

\chapter{A Concrete Naming Certificate for \(\VecSpaceUp\)}
\label{ch:concrete-instances-vecspace-namecert}

The vector space interface \(\VecSpaceUp\) is the specialization of \autoref{ch:concrete-instances-module-namecert} to the case where the scalar ring satisfies \autoref{ch:concrete-instances-field-namecert}. No new fields are required beyond the module ones; the change is the strengthened scalar source. The \(n\)-fold rational tuple space (an iterated product over \(\RatUp\)) is the canonical concrete instance.

\section{Carrier definition}

\begin{definition}[VecSpace carrier]
\label{def:vecspace-carrier}
Use a module setup whose scalar source is a certified field.
\end{definition}

\section{Concrete singleton history vector space instance}
\label{sec:vecspace-concrete-singleton-history-instance}

\begin{definition}[Concrete singleton empty-history vector space instance]
\label{def:singleton-empty-history-vecspace-instance}
Let the scalar carrier and vector carrier both be the singleton history source
$A_{\emp}$ of Definition~\ref{def:singleton-empty-history-module-instance}.
Define the shared classifier
$$
\rho_{\emp}(h,k):=
A_{\emp}(h)\land A_{\emp}(k)\land\hsame(h,k).
$$
Use the scalar addition, scalar negation, scalar multiplication, scalar zero,
scalar one, nonzero predicate, and proof-indexed scalar inverse from
Definition~\ref{def:singleton-empty-history-field-schema-instance}. Use the
vector addition, vector negation, vector zero, and scalar action from
Definition~\ref{def:singleton-empty-history-module-instance}, with
$$
\mathsf{smul}(r,m):=\emp.
$$
The certificate fit has ten fields:
\begin{enumerate}
  \item Traditional target: the one-element vector space over the one-element
  proof-indexed field.
  \item BEDC source: $\mathsf{BHist}$.
  \item Carrier choice: $A_{\emp}$ for both scalars and vectors.
  \item Classifier choice: $\rho_{\emp}$.
  \item Stability obligations: the singleton field laws together with the
  singleton module laws and the vacuous nonzero-indexed scalar inverse
  reasoning.
  \item Ledger: singleton scalar, vector, and action provenance.
  \item Core theorem list: Theorem~\ref{thm:singleton-empty-history-vecspace-laws}.
  \item Exactness predicate: singleton endpoint classification.
  \item Standard bridge: the terminal one-object vector space.
  \item Status: $\mathsf{ProofTarget}$.
\end{enumerate}
\end{definition}

\begin{theorem}[Singleton empty-history vector space laws]
\label{thm:singleton-empty-history-vecspace-laws}
The tuple
$$
(A_{\emp},\rho_{\emp},\mathsf{add},0,\mathsf{neg},
\mathsf{mul},1,\mathsf{smul},\mathsf{NonZero}_{\bot},\mathsf{inv})
$$
satisfies the $\VecSpaceUp$ schema over the concrete $\mathsf{BHist}$
singleton carrier:
scalar $\FieldUp$ laws, additive module laws, scalar-action closure,
scalar-action congruence, scalar associativity, distributivity over vector
addition, distributivity over scalar addition, scalar unit law, and
inverse-enabled scalar reasoning for explicitly nonzero scalars.
\end{theorem}
\begin{proof}
The module part is exactly the singleton empty-history module package of
Theorem~\ref{thm:module-singleton-scalar-action-laws}. It supplies closure,
the additive abelian-group laws, scalar-action congruence, scalar
associativity, both distributivity laws, and the scalar unit law for the same
$A_{\emp}$ carrier and $\rho_{\emp}$ classifier.

The scalar field part is exactly the singleton empty-history field schema of
Theorem~\ref{thm:singleton-empty-history-field-schema-laws}. It supplies the
scalar commutative-ring laws together with the proof-indexed nonzero transport,
inverse closure, inverse congruence, and inverse laws. The certificate
packaging is the one recorded in
Theorem~\ref{thm:singleton-empty-history-field-schema-certificate}.

It remains only to check that the two packages share the displayed scalar
source. Both use the constant empty-history scalar operations on
$A_{\emp}$, the same classifier $\rho_{\emp}$, and the same scalar unit. Hence
their scalar ring fragments coincide as the scalar side of the vector-space
schema.

For scalar congruence and each scalar-action law, every endpoint reduces to
$\emp$. The required classifier witness is assembled from
$A_{\emp}(\emp)$ and $\hsame(\emp,\emp)$.

For inverse-enabled scalar reasoning, every such clause begins with a witness
$p:\mathsf{NonZero}_{\bot}(h)$. Theorem~\ref{thm:field-singleton-nonzero-absurd}
identifies the singleton nonzero predicate as contradictory at the only
possible endpoint. Since
$\mathsf{NonZero}_{\bot}(h)$ is $\bot$, eliminating $p$ closes the clause over
the empty witness domain. No classical nonzero test or further endpoint datum
is needed.
\end{proof}

\begin{theorem}[Singleton empty-history vector space endpoint collapse]
\label{thm:singleton-empty-history-vecspace-endpoint-collapse}
With the notation of
Definition~\ref{def:singleton-empty-history-vecspace-instance}, let
$r,s:\mathsf{BHist}$ be scalar endpoints and $m,n:\mathsf{BHist}$ be vector
endpoints carried by $A_{\emp}$. Then the concrete singleton tuple has the
following endpoint classifications:
\begin{enumerate}
  \item For any $x,y:\mathsf{BHist}$, if $A_{\emp}(x)$ and $A_{\emp}(y)$,
  then $\rho_{\emp}(x,y)$.
  \item $A_{\emp}(\mathsf{smul}(r,m))$ and
  $\rho_{\emp}(\mathsf{smul}(r,m),\emp)$.
  \item The vector endpoints $\mathsf{add}(m,n)$, $\mathsf{neg}(m)$, and
  the vector zero are all carried by $A_{\emp}$ and classified with $\emp$.
  \item The scalar endpoints $0$, $1$, and $\mathsf{mul}(r,s)$ are all
  carried by $A_{\emp}$ and classified with $\emp$.
  \item For any witness $p:\mathsf{NonZero}_{\bot}(r)$, the inverse endpoint
  $\mathsf{inv}(r,p)$ is carried by $A_{\emp}$ and classified with $\emp$.
\end{enumerate}
\end{theorem}
\begin{proof}
Apply Theorem~\ref{thm:module-singleton-scalar-action-laws} to the carried
scalar and vector inputs. Its closure and congruence clauses cover the
additive vector endpoints and the scalar-action endpoint for the same
$A_{\emp}$ carrier and $\rho_{\emp}$ classifier.

Apply Theorem~\ref{thm:singleton-empty-history-field-schema-laws} to the
scalar side. It supplies closure for scalar zero, scalar one, scalar
multiplication, and the proof-indexed inverse clauses over the same singleton
source.

Unfold the concrete operations in
Definition~\ref{def:singleton-empty-history-vecspace-instance}. The displayed
scalar action is definitionally $\emp$, and the imported singleton vector and
scalar operations reduce to the singleton endpoint as recorded by the module
and field law packages.

For any two carried endpoints $x$ and $y$, unfold
$\rho_{\emp}(x,y)$. The carrier components are the two assumptions
$A_{\emp}(x)$ and $A_{\emp}(y)$, and the singleton endpoint witnesses for
$x$ and $y$ compose to give the required $\hsame(x,y)$ component.

Each operation endpoint already reduces to $\emp$ or is closed by the cited
law package at the singleton endpoint. Pair the relevant
$A_{\emp}(\emp)$ witness with reflexivity $\hsame(\emp,\emp)$ to obtain the
displayed classifier witness against $\emp$.

For the inverse-indexed clause, assume
$p:\mathsf{NonZero}_{\bot}(r)$. Since $A_{\emp}(r)$ holds,
Theorem~\ref{thm:field-singleton-nonzero-absurd} eliminates the nonzero
witness at the only possible endpoint. From that contradiction, the carried
and classified inverse conclusion follows over the empty witness domain.
\end{proof}

$$
\mathsf{VecSpaceSingletonClassifier}(a,b)
\quad\text{and}\quad
\mathsf{VecSpaceSingletonClassifier}(b,c),

then
$$
a\hsame\emp\wedge c\hsame\emp\wedge
\mathsf{VecSpaceSingletonClassifier}(a,c).

Unfold the concrete singleton classifier. The first classifier supplies the
empty endpoint witness for $a$ and the comparison $a\hsame b$. The second
classifier supplies the empty endpoint witness for $c$ and the comparison
$b\hsame c$. Compose the two comparisons by transitivity to obtain
$a\hsame c$, and package it with the carried endpoint witnesses for $a$ and
$c$.
\end{proof}

\section{The certificate}

\begin{definition}[VecSpace source specification]
\label{def:vecspace-source-specification}
The source is \(\FieldUp(F)\) together with a certified \(F\)-module carrier.
\end{definition}

\begin{definition}[VecSpace pattern specification]
\label{def:vecspace-pattern-specification}
The pattern is exactly module expression formation over field scalars.
\end{definition}

\begin{definition}[VecSpace classifier specification]
\label{def:vecspace-classifier-specification}
The classifier is the module carrier classifier, with scalar congruence inherited from the field.
\end{definition}

\begin{definition}[VecSpace stability certificate]
\label{def:vecspace-stability-certificate}
The stability certificate consists of:
\begin{enumerate}
  \item all module stability fields
  \item all scalar field stability fields
  \item inverse-enabled scalar reasoning for apart-from-zero scalars
\end{enumerate}
\end{definition}

\begin{definition}[VecSpace ledger policy]
\label{def:vecspace-ledger-policy}
Record module ledgers plus the field ledger for scalar operations.
\end{definition}

\begin{definition}[\(\VecSpaceUp\) certificate obligations]
\label{def:vecspace-certificate-obligations}
The carrier, source, pattern, classifier, stability, and ledger data listed above are the obligations required for the interface name \(\VecSpaceUp\).
\end{definition}

\paragraph{Primitive boundary.}
\(\VecSpaceUp\) is not a BEDC primitive. It is admitted only when those obligations are supplied.

\begin{remark}[Zero-axiom encoding note]
\label{rem:vecspace-zero-axiom-note}
Inverses use field apartness witnesses rather than LEM-based nonzero tests.
\end{remark}

\begin{proposition}[VecSpace forgets to Module certificate]
\label{prop:vecspace-forgets-module-certificate}
Let $F$ carry a $\FieldUp(F)$ certificate, and let $R$ be the $\CommRingUp$ base coordinate contained in that field certificate. Let $V$ carry a $\VecSpaceUp(F,V)$ certificate with scalar action $\mathsf{smul}:F\times V\to V$. If the constructive apartness predicate, the proof-indexed scalar inverse operation, and every inverse-indexed stability or ledger row are omitted, then the retained carrier, source, pattern, classifier, stability, and ledger fields form a $\ModuleUp(R,V)$ certificate, with $R$ read through its ring reduct. The vector carrier, scalar action, additive vector operations, scalar ring operations, classifier, exactness predicate, and all non-inverse ledger provenance are unchanged.
\end{proposition}
\begin{proof}
By \autoref{def:vecspace-certificate-obligations}, the given vector-space certificate supplies carrier, source, pattern, classifier, stability, and ledger data for $\VecSpaceUp$. By \autoref{def:vecspace-carrier}, its carrier is already a module setup whose scalar source is strengthened from a scalar ring to a certified field. Therefore the vector carrier, additive vector operations, scalar action, and classifier are the module-shaped carrier data listed in \autoref{def:module-carrier}.

The scalar source is handled by projection of the field side. By \autoref{def:field-source-specification}, the $\FieldUp(F)$ source contains a certified commutative-ring carrier together with constructive nonzero evidence. Omitting the nonzero and inverse-specific coordinates leaves the $\CommRingUp$ certificate $R$. By \autoref{prop:commring-forgets-ring-certificate}, this same scalar coordinate supplies the $\RingUp$ source required by \autoref{def:module-source-specification}. Together with the additive vector carrier already present in the module setup, this gives the source field for $\ModuleUp(R,V)$.

For patterns, \autoref{def:vecspace-pattern-specification} says that the vector-space pattern is module expression formation over field scalars. Removing scalar inverse formation leaves expressions built from vector addition, vector zero, vector negation, scalar multiplication, and the scalar ring operations. This is exactly the module expression formation of \autoref{def:module-pattern-specification}. For classifiers, \autoref{def:vecspace-classifier-specification} uses the module carrier classifier, with scalar congruence inherited from the field. After the scalar source is read through the commutative-ring and ring reducts, the same classifier is the one required by \autoref{def:module-classifier-specification}; no endpoint relation is replaced.

For stability, \autoref{def:vecspace-stability-certificate} contains all module stability fields, all scalar field stability fields, and inverse-enabled scalar reasoning. The first component is precisely the scalar-action and additive-vector stability package of \autoref{def:module-stability-certificate}. The scalar ring laws needed there are present because \autoref{def:field-stability-certificate} contains all commutative-ring stability fields, and \autoref{prop:commring-forgets-ring-certificate} retains the ring part. The inverse closure, nonzero transport, and inverse laws are not obligations for $\ModuleUp$.

For the ledger, \autoref{def:vecspace-ledger-policy} records the module ledgers together with the field ledger for scalar operations. By \autoref{def:field-ledger-policy}, the field ledger consists of the ring ledger plus the nonzero witness used for each inverse. Omitting those inverse-witness entries leaves the scalar and module expression provenance described in \autoref{def:module-ledger-policy}.

Thus the retained carrier, source, pattern, classifier, stability, and ledger fields are exactly the obligations named in \autoref{def:module-certificate-obligations}. Since every retained operation and classifier is copied from the original $\VecSpaceUp(F,V)$ certificate, the resulting $\ModuleUp(R,V)$ certificate has the same carrier, action, classifier, stability rows, and ledger outside the omitted inverse-specific data.
\end{proof}
\begin{proposition}[VecSpace module-fragment projection]
\label{prop:vecspace-module-fragment-projection}
Let $F$ carry a $\FieldUp(F)$ certificate. Let $F_{\mathsf{Ring}}$ be the scalar certificate obtained from it by first taking the commutative-ring reduct and then the ring reduct. Let $V$ carry a $\VecSpaceUp(F,V)$ certificate with vector carrier $C_V$, vector classifier $\sim_V$, additive operations $+_V$, $0_V$, $-_V$, and scalar action $\odot:F\times V\to V$. If the apartness predicate, proof-indexed scalar inverse operation, inverse-indexed stability rows, and inverse-witness ledger rows are omitted, the retained fields form a $\ModuleUp(F_{\mathsf{Ring}},V)$ certificate. The vector carrier $C_V$, vector classifier $\sim_V$, vector addition, vector zero, vector negation, scalar action $\odot$, scalar addition, scalar zero, scalar negation, scalar multiplication, scalar one, exactness predicate, and non-inverse ledger rows are unchanged.
\end{proposition}
\begin{proof}
By \autoref{def:vecspace-certificate-obligations}, the vector-space certificate supplies carrier, source, pattern, classifier, stability, and ledger data for $\VecSpaceUp$. By \autoref{def:vecspace-carrier}, its carrier is already a module setup whose scalar source has been strengthened from a certified ring source to a certified field source. Thus the vector carrier, additive vector operations, scalar action, and vector classifier are exactly the module-shaped carrier data required by \autoref{def:module-carrier}.

The scalar source is obtained by successive reducts. Applying \autoref{thm:field-certificate-forgets-to-commring-certificate} to the $\FieldUp(F)$ scalar certificate keeps the same scalar carrier, classifier, addition, zero, negation, multiplication, and one while dropping the apartness and proof-indexed inverse rows. Applying \autoref{prop:commring-forgets-ring-certificate} to that commutative-ring coordinate keeps the same scalar operations and classifier and supplies the $\RingUp$ source required by \autoref{def:module-source-specification}. This scalar source is $F_{\mathsf{Ring}}$.

For patterns, \autoref{def:vecspace-pattern-specification} identifies vector-space expression formation with module expression formation over field scalars. After inverse formation on the scalar side is omitted, the remaining vector expressions are generated by vector addition, vector zero, vector negation, scalar multiplication, and scalar ring expressions. This is exactly the pattern listed in \autoref{def:module-pattern-specification}.

For classifiers, \autoref{def:vecspace-classifier-specification} uses the module carrier classifier, with scalar congruence inherited from the field. The scalar reducts above preserve the scalar classifier, and the vector classifier is not replaced. Hence the retained classifier is the module carrier classifier required by \autoref{def:module-classifier-specification}.

For stability, \autoref{def:vecspace-stability-certificate} contains all module stability fields, all scalar field stability fields, and inverse-enabled scalar reasoning. The module rows are precisely the scalar-action and additive-vector rows listed in \autoref{def:module-stability-certificate}. The scalar ring rows needed there are retained because \autoref{def:field-stability-certificate} contains the commutative-ring stability fields and \autoref{prop:commring-forgets-ring-certificate} keeps the ring part. The inverse closure, nonzero transport, and inverse laws are not obligations for $\ModuleUp$.

For the ledger, \autoref{def:vecspace-ledger-policy} records module ledgers together with the field ledger for scalar operations. By \autoref{def:field-ledger-policy}, the field ledger consists of the ring ledger plus inverse-witness entries. Omitting those witness entries leaves the scalar and module expression provenance described in \autoref{def:module-ledger-policy}.

Thus the retained carrier, source, pattern, classifier, stability, and ledger fields satisfy \autoref{def:module-certificate-obligations}. Every retained operation, carrier, classifier, exactness predicate, and non-inverse ledger row is copied from the original $\VecSpaceUp(F,V)$ certificate, so the projected $\ModuleUp(F_{\mathsf{Ring}},V)$ certificate has the same module fragment.
\end{proof}
\leanchecked{BEDC.Derived.VecSpaceUp.VecSpaceSingleton\_module\_fragment\_projection}

\begin{corollary}[VecSpace additive carrier projection]
\label{cor:vecspace-additive-carrier-projection}
Let $F$ carry a $\FieldUp(F)$ certificate. Let $F_{\mathsf{Ring}}$ be the scalar certificate obtained from it by first taking the commutative-ring reduct and then the ring reduct. Let $V$ carry a $\VecSpaceUp(F,V)$ certificate with vector carrier $C_V$, vector classifier $\sim_V$, vector addition $+_V$, vector zero $0_V$, vector negation $(-)_V$, and scalar action $\odot:F\times V\to V$. If the vector-space certificate is first read through the module-fragment projection of \autoref{prop:vecspace-module-fragment-projection}, and the resulting module certificate is then read through the additive reduct of \autoref{prop:module-forgets-abgroup-certificate}, the retained vector-side data form an $\AbGroupUp(V)$ certificate. The carrier $C_V$, classifier $\sim_V$, vector addition $+_V$, vector zero $0_V$, and vector negation $(-)_V$ are unchanged from the original $\VecSpaceUp(F,V)$ certificate.
\end{corollary}
\leanchecked{BEDC.Derived.VecSpaceUp.VecSpaceSingleton\_additive\_carrier\_projection}
\begin{proof}
Apply \autoref{prop:vecspace-module-fragment-projection} to the given vector-space certificate. It supplies a $\ModuleUp(F_{\mathsf{Ring}},V)$ certificate after deleting only the apartness predicate, proof-indexed scalar inverse operation, inverse-indexed stability rows, and inverse-witness ledger rows. The proposition states that this module certificate keeps the original vector carrier $C_V$, vector classifier $\sim_V$, vector addition, vector zero, vector negation, and scalar action.

Now apply \autoref{prop:module-forgets-abgroup-certificate} to that $\ModuleUp(F_{\mathsf{Ring}},V)$ certificate. Its scalar input is available because $F_{\mathsf{Ring}}$ is a certified $\RingUp$ source. This second reduct removes the scalar-ring package, the scalar action, and the scalar-action stability rows, and retains exactly the additive carrier, additive source, additive pattern, carrier classifier, additive stability fields, and additive ledger.

By \autoref{def:abgroup-certificate-obligations}, those retained fields are the obligations for an $\AbGroupUp(V)$ certificate. The first projection did not alter the vector-side additive data, and the second projection explicitly keeps the additive carrier data of the module certificate. Therefore the final $\AbGroupUp(V)$ certificate is over the same carrier $C_V$ with the same classifier $\sim_V$, addition $+_V$, zero $0_V$, and additive inverse $(-)_V$ carried by the original vector-space certificate.
\end{proof}

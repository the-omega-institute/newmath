\begin{definition}[InscriptionNoveltySocket carrier]
\label{def:inscription-novelty-socket-carrier}
An $\InscriptionNoveltySocketUp$ carrier is a finite $\BHist$ packet
$$
S=(B,I,R,G,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $B$ is the prior-boundary row, listing the records visible before the inscription event is read;
\item $I$ is the inscription-event row, consumed through \autoref{ch:concrete-instances-inscriptionevent-namecert};
\item $R$ is the non-reduction and aspect-separation row, recording that consumers may compare $I$ with $B$ without extracting a closed records-side reducer from $B$ to $I$;
\item $G$ is the Type VI audit-gate row, read from \autoref{sec:transcendental-supply-type-VI} and preserving the distinction between proof replay and inscription-event actuality;
\item $H$ records componentwise $\hsame$ transport for $B,I,R,G$, package provenance, and local naming;
\item $C$ records displayed $\Cont$ replay routes through the prior boundary, event row, aspect-separation row, and audit-gate row;
\item $P$ is the $\Pkg$ provenance over $\InscriptionEventUp$, $\TranscendentalSupplyInscriptionUp$, $\FreeWillInscriptionCommitmentUp$, $\SupplySocketLedgerUp$, $\SocketKindClassifierUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows;
\item $N$ is the local $\NameCert_{\InscriptionNoveltySocketUp}$ row.
\end{enumerate}
The carrier has no coordinate for a hidden novelty source, deterministic reducer, agency oracle, host psychology predicate, unrecorded proof rule, or second socket-kind classifier.
\end{definition}

\begin{theorem}[InscriptionNoveltySocket NameCert obligations]
\label{thm:inscription-novelty-socket-namecert-obligations}
For the carrier of \autoref{def:inscription-novelty-socket-carrier}, the local $\NameCert_{\InscriptionNoveltySocketUp}$ surface has five finite obligations:
\begin{enumerate}
\item carrier exposure for the tuple $S=(B,I,R,G,H,C,P,N)$;
\item prior-boundary accountability for the records row $B$;
\item inscription-event accountability for the event row $I$ and its Type VI site;
\item aspect-separation stability for the non-reduction row $R$, audit-gate row $G$, componentwise transport row $H$, and replay row $C$;
\item package exactness and non-escape for $P,N$, excluding hidden novelty sources, deterministic reducers, agency oracles, host psychology predicates, unrecorded proof rules, and unlisted socket-kind classifiers.
\end{enumerate}
\end{theorem}
\begin{proof}
Carrier exposure is projection from the finite packet $S=(B,I,R,G,H,C,P,N)$. Every consumer-facing coordinate is one of these displayed rows.

Prior-boundary accountability reads $B$, while inscription-event accountability reads $I$ through the inscription-event sibling. These rows make the event public as a finite record site before any Type VI audit gate is consulted.

Aspect-separation stability is carried by $R,G,H,C$. The row $R$ prevents a consumer from turning comparison between $B$ and $I$ into a closed reducer, while $G$ records the Type VI gate that separates proof replay from inscription-event actuality. The rows $H$ and $C$ transport and replay only that displayed data.

Finally read $P$ and $N$. The package and local naming rows list the concrete sibling surfaces and do not add a hidden source, reducer, agency oracle, host psychology predicate, proof rule, or socket-kind classifier. Thus the five obligations exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[InscriptionNoveltySocket inscription-event route]
\label{thm:inscription-novelty-socket-inscription-event-route}
Every accepted $\InscriptionNoveltySocketUp$ packet reads its inscription-event row $I$ through the finite route
$$
B \to I \to R \to G \to H \to C \to P \to N.
$$
The route preserves the prior-boundary comparison and the Type VI audit gate, and it exports no hidden novelty source, deterministic reducer from $B$ to $I$, agency oracle, host psychology predicate, or unrecorded proof rule.
\end{theorem}
\begin{proof}
Project the accepted carrier as $S=(B,I,R,G,H,C,P,N)$. The only inscription-event coordinate is $I$, and it is displayed beside the prior-boundary row $B$ before the aspect-separation and audit-gate rows are read.

Read $I$ through the concrete inscription-event sibling named in \autoref{ch:concrete-instances-inscriptionevent-namecert}. The row $R$ then records that the event may be compared with $B$ without yielding a closed records-side reducer, and $G$ records the Type VI audit gate from \autoref{sec:transcendental-supply-type-VI}.

The support rows $H,C,P,N$ transport, replay, package, and name the same visible route. Since the carrier contains no coordinate for a hidden novelty source, deterministic reducer, agency oracle, host psychology predicate, or unrecorded proof rule, the inscription-event route cannot export any of those objects.
\end{proof}

\begin{theorem}[InscriptionNoveltySocket sibling independence]
\label{thm:inscription-novelty-socket-sibling-independence}
The $\InscriptionNoveltySocketUp$ carrier is not bijective, under NameCert-row preservation, to $\InscriptionEventUp$, $\TranscendentalSupplyInscriptionUp$, $\FreeWillInscriptionCommitmentUp$, $\SupplySocketLedgerUp$, or $\SocketKindClassifierUp$.
\end{theorem}
\begin{proof}
Compare the displayed rows. $\InscriptionEventUp$ supplies the event record, $\TranscendentalSupplyInscriptionUp$ supplies a supply-boundary socket, $\FreeWillInscriptionCommitmentUp$ supplies a commitment ledger, $\SupplySocketLedgerUp$ accounts for requested supply, and $\SocketKindClassifierUp$ classifies the socket kind.

The $\InscriptionNoveltySocketUp$ packet binds $B,I,R,G,H,C,P,N$ together, with $R$ carrying non-reduction and aspect separation and $G$ carrying the Type VI audit gate for the same event. None of the siblings carries that paired prior-boundary, inscription-event, aspect-separation, and Type VI gate surface as one local NameCert packet. A row-preserving bijection would have to preserve those paired rows, so no such bijection exists.
\end{proof}

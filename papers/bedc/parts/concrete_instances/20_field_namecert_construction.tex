\begin{theorem}[Concrete $\FieldUp(\RatUp)$ semantic naming certificate]
\label{thm:ratup-fieldup-semantic-namecert}
The concrete source tuple $\mathsf{RatUpFieldSource}_F$ of
\autoref{def:ratup-fieldup-concrete-source-tuple}, with carrier
$\mathsf{RatHistoryCarrier}$, classifier $\mathsf{RatHistoryClassifier}$,
denominator tuple
$(\mathsf{RatDenomUnitCarrier},\mathsf{RatDenomUnitClassifier},
\mathsf{append},\emp,\mathsf{RatHistoryCarrier})$, contextual action
$\Gamma$, context-pair monoid, and affine source $\mathsf{AffAut}_F$,
supplies the semantic naming certificate for $\FieldUp(\RatUp)$. Consequently
$\FieldUp(\RatUp)$ is the open phase exit object carried by the assembled
BHist-level field source.
\end{theorem}
\begin{proof}
The base name is fixed by \autoref{thm:concrete-rat-namecert}: the rational
history carrier and classifier are the named $\RatUp$ object on $\mathsf{BHist}$.
This supplies the semantic target over which the field source tuple is read.

Next apply \autoref{thm:ratup-fieldup-assembled-certificate-boundary}. It
identifies $\mathsf{RatUpFieldSource}_F$ as satisfying
\autoref{def:field-certificate-obligations} with the same carrier,
classifier, denominator tuple, contextual source, context-pair source, and
affine source displayed in the statement.

The denominator-action coordinate is the component isolated by
\autoref{thm:ratup-fieldup-denominator-action-envelope-certificate-component}.
It records the envelope closure, normalized row coverage, and context-pair
action certificate for the same concrete denominator tuple and the same action
$\Gamma$.

Finally use \autoref{thm:field-affine-automorphism-group-certificate} for the
affine source $\mathsf{AffAut}_F$. Combining the RatUp name certificate, the
assembled boundary, the denominator-action component, and the affine group row
fills exactly the carrier, source, classifier, stability, ledger, and exactness
slots specified by \autoref{def:field-certificate-obligations}. This repacks the
assembled source as the semantic naming certificate for $\FieldUp(\RatUp)$.
\end{proof}

Per-object certificate: traditional target constructive field over rational
histories; BEDC source $\mathsf{BHist}$, $\RatUp$, $\CommRingUp$, denominator
envelope, contextual action, and affine rows; carrier
$\mathsf{RatHistoryCarrier}$; classifier $\mathsf{RatHistoryClassifier}$;
stability obligations commutative-ring rows, nonzero and inverse rows,
denominator rows, contextual action rows, context-pair rows, and affine closure;
ledger ring expressions, nonzero witnesses, denominator histories, contextual
pairs, and affine coefficient witnesses; core theorem list headed by
\autoref{thm:ratup-fieldup-semantic-namecert} and
\autoref{thm:ratup-fieldup-assembled-certificate-boundary}; exactness predicate
the conjunction of inverse, product, denominator, contextual-action,
context-pair, and affine exactness rows; standard bridge deferred; status
$\mathsf{ProofTarget}$.

\begin{theorem}[RatUp FieldUp semantic certificate assembly]
\label{thm:ratup-fieldup-semantic-certificate-assembly}
For the concrete $\mathsf{BHist}$ data
$\mathsf{RatHistoryCarrier}$, $\mathsf{RatHistoryClassifier}$,
$\mathsf{RatDenomUnitCarrier}$, $\mathsf{RatDenomUnitClassifier}$,
$\mathsf{append}$, $\Cont$, $\emp$, and strict support
$\mathsf{RatHistoryCarrier}$, assume the $\RatUp$ source already carries the
$\CommRingUp$ base rows certified by
\autoref{thm:singleton-empty-history-commring-laws} and
\autoref{thm:singleton-empty-history-commring-schema-certificate}. With those
base rows in place, the FieldUp-specific inverse and apart-zero rows, the
two-sided zero-product and nonzero-product rows, the denominator multiplicative
and contextual rows, the denominator context-pair action rows, and the packaged
affine $\GroupUp$ consequence supply the carrier, source, pattern, classifier,
stability, ledger, and exactness rows of
\autoref{def:field-certificate-obligations} for the concrete target
$\FieldUp(\mathsf{RatUp})$.
\end{theorem}
\begin{proof}
Bind the rational carrier and classifier first. The object carrier is
$\mathsf{RatHistoryCarrier}$, the object classifier is
$\mathsf{RatHistoryClassifier}$, and the source pattern is the BEDC
continuation pattern $\Cont$ over $\mathsf{BHist}$. The denominator tuple is
$(\mathsf{RatDenomUnitCarrier},\mathsf{RatDenomUnitClassifier},
\mathsf{append},\emp,\mathsf{RatHistoryCarrier})$.

The $\CommRingUp$ base rows are the first dependency. Theorem~\ref{thm:singleton-empty-history-commring-laws}
supplies the additive and multiplicative ring laws, distributivity, operation
congruence, multiplicative commutativity, and commutativity compatibility needed
by the field record.

Theorem~\ref{thm:singleton-empty-history-commring-schema-certificate} packages
those laws with the carrier, source, classifier, pattern, and ring ledger fields
over the concrete singleton source, so the base part of
\autoref{def:field-certificate-obligations} is already licensed before any
inverse or denominator row is attached.

The inverse rows are the apart-zero rows. Theorem~\ref{thm:field-apart-zero-inverse-classifier-exact-from-apartness}
supplies classifier exactness for proof-indexed inverses over the concrete
apart-zero predicate, so the inverse classifier, inverse stability, and
nonzero transport entries are read from the displayed apartness witnesses.

The zero-product rows come from
\autoref{thm:field-two-sided-zero-product-cancel-from-apartness}. Its
two-sided multiplication equivalence gives exact zero classification after
certified nonzero left and right factors, and the same witness data supply the
nonzero-product contradiction row in the per-object certificate record.

For the denominator part, use
\autoref{thm:field-rat-denominator-fieldup-multiplicative-obligation-coverage}
for the multiplicative denominator rows and
\autoref{thm:ratup-fieldup-denominator-package-closure} for the closed
denominator-side package. The exact carrier, classifier, multiplication, unit,
and strict-support entries are the concrete tuple fixed above.

The contextual denominator action is supplied by
\autoref{thm:ratup-fieldup-denominator-context-pair-action-certificate}. It
packages the pair classifier, pair product, unit-pair action, product-pair
composition law, and strict-support readback over the same BHist denominator
tuple, so the contextual exactness rows align with the denominator ledger
entries.

Finally, \autoref{thm:field-affine-automorphism-group-certificate} supplies
the packaged affine $\GroupUp$ consequence consumed by the field certificate.

Together the $\CommRingUp$ base rows, the inverse rows, the zero-product rows,
the denominator multiplicative rows, the contextual denominator rows, the
context-pair action rows, and the affine $\GroupUp$ row exhaust the ten-entry
per-object template listed after \autoref{def:field-certificate-obligations}:
target, source, carrier, classifier, stability obligations, ledger, core
theorem list, exactness predicate, standard bridge, and status.
\end{proof}

Per-object template: traditional target constructive field on $\RatUp$; BEDC
source $\RatUp$ plus $\CommRingUp$ and concrete denominator $\mathsf{BHist}$
rows; carrier $\mathsf{RatHistoryCarrier}$; classifier
$\mathsf{RatHistoryClassifier}$ together with
$\mathsf{RatDenomUnitClassifier}$ on denominator rows; stability obligations
inverse apartness, zero-product exactness, denominator package closure,
context-pair action, and affine automorphism packaging; ledger rational
history rows, denominator unit histories, continuation witnesses, and
apart-zero inverse witnesses; core theorem
\autoref{thm:ratup-fieldup-semantic-certificate-assembly}; exactness predicate
row coverage for $\FieldUp(\mathsf{RatUp})$; standard bridge deferred; status
$\mathsf{ProofTarget}$.

\begin{definition}[Concrete RatUp FieldUp certificate instance]
\label{def:concrete-ratup-fieldup-certificate-instance}
The concrete certificate instance for $\FieldUp(\mathsf{RatUp})$ is the BEDC
tuple with field carrier $\mathsf{RatHistoryCarrier}$, field classifier
$\mathsf{RatHistoryClassifier}$, denominator carrier
$\mathsf{RatDenomUnitCarrier}$, denominator classifier
$\mathsf{RatDenomUnitClassifier}$, denominator multiplication
$\mathsf{append}$, continuation predicate $\Cont$, empty denominator endpoint
$\emp$, apart-zero predicate $\mathsf{FieldApartZero}$, the contextual
denominator action $\Gamma$ of
\autoref{def:ratup-fieldup-denominator-contextual-action-instance}, and the
affine automorphism package $\mathsf{AffAut}_F$ of
\autoref{def:field-affine-automorphism-package}.

Its per-object fields are:
\begin{enumerate}
  \item traditional target: the constructive field target
  $\FieldUp(\mathsf{RatUp})$;
  \item BEDC source: $\RatUp$ together with $\CommRingUp$ and the concrete
  denominator BHist rows;
  \item carrier: $\mathsf{RatHistoryCarrier}$;
  \item classifier: $\mathsf{RatHistoryClassifier}$ together with
  $\mathsf{RatDenomUnitClassifier}$ for denominator rows;
  \item stability obligations: inverse apartness, two-sided zero-product
  exactness, denominator package closure, denominator contextual action,
  context-pair action, context-pair strict-support commutation, and affine
  automorphism closure;
  \item ledger: rational history rows, denominator unit histories,
  continuation witnesses, apart-zero inverse witnesses, context-pair action
  witnesses, and affine coefficient witnesses;
  \item core theorem list:
  \autoref{thm:ratup-fieldup-denominator-action-envelope-certificate-component},
  \autoref{thm:ratup-fieldup-denominator-context-pair-action-certificate},
  \autoref{thm:field-rat-denominator-context-pair-product-strict-support-commutes},
  \autoref{thm:field-affine-automorphism-group-certificate}, and
  \autoref{thm:ratup-fieldup-assembled-certificate-boundary};
  \item exactness predicate: row coverage for $\FieldUp(\mathsf{RatUp})$
  including inverse, zero-product, denominator multiplication,
  denominator-action envelope, contextual action, context-pair action,
  context-pair strict-support commutation, and affine automorphism rows;
  \item standard bridge: deferred conservative bridge to the ordinary
  constructive rational field presentation;
  \item status: $\mathsf{ProofTarget}$.
\end{enumerate}
\end{definition}

\begin{theorem}[Concrete $\FieldUp(\RatUp)$ semantic naming certificate]
\label{thm:ratup-fieldup-semantic-namecert}
The concrete source tuple $\mathsf{RatUpFieldSource}_F$ of
\autoref{def:ratup-fieldup-concrete-source-tuple}, with carrier
$\mathsf{RatHistoryCarrier}$, classifier $\mathsf{RatHistoryClassifier}$,
denominator tuple
$(\mathsf{RatDenomUnitCarrier},\mathsf{RatDenomUnitClassifier},
\mathsf{append},\emp,\mathsf{RatHistoryCarrier})$, contextual action
$\Gamma$, context-pair monoid, and affine source $\mathsf{AffAut}_F$,
supplies the semantic naming certificate for $\FieldUp(\RatUp)$. Consequently
$\FieldUp(\RatUp)$ is the open phase exit object carried by the assembled
BHist-level field source.
\end{theorem}
\begin{proof}
The base name is fixed by \autoref{thm:concrete-rat-namecert}: the rational
history carrier and classifier are the named $\RatUp$ object on $\mathsf{BHist}$.
This supplies the semantic target over which the field source tuple is read.

Next apply \autoref{thm:ratup-fieldup-assembled-certificate-boundary}. It
identifies $\mathsf{RatUpFieldSource}_F$ as satisfying
\autoref{def:field-certificate-obligations} with the same carrier,
classifier, denominator tuple, contextual source, context-pair source, and
affine source displayed in the statement.

The denominator-action coordinate is the component isolated by
\autoref{thm:ratup-fieldup-denominator-action-envelope-certificate-component}.
It records the envelope closure, normalized row coverage, and context-pair
action certificate for the same concrete denominator tuple and the same action
$\Gamma$.

The context-pair product support row is
\autoref{thm:field-rat-denominator-context-pair-product-strict-support-commutes}.
It is registered in
Definition~\ref{def:concrete-ratup-fieldup-certificate-instance} as the
strict-support commutation entry for the same denominator tuple.

Finally use \autoref{thm:field-affine-automorphism-group-certificate} for the
affine source $\mathsf{AffAut}_F$. Combining the RatUp name certificate, the
assembled boundary, the denominator-action component, the strict-support
commutation row, and the affine group row fills exactly the carrier, source,
classifier, stability, ledger, and exactness slots specified by
\autoref{def:field-certificate-obligations}. This repacks the assembled source
as the semantic naming certificate for $\FieldUp(\RatUp)$.
\end{proof}

Per-object certificate: traditional target constructive field over rational
histories; BEDC source $\mathsf{BHist}$, $\RatUp$, $\CommRingUp$, denominator
envelope, contextual action, and affine rows; carrier
$\mathsf{RatHistoryCarrier}$; classifier $\mathsf{RatHistoryClassifier}$;
stability obligations commutative-ring rows, nonzero and inverse rows,
denominator rows, contextual action rows, context-pair rows,
context-pair strict-support commutation, and affine closure; ledger ring
expressions, nonzero witnesses, denominator histories, contextual pairs, and
affine coefficient witnesses; core theorem list headed by
\autoref{thm:ratup-fieldup-semantic-namecert},
\autoref{thm:ratup-fieldup-assembled-certificate-boundary},
\autoref{thm:ratup-fieldup-denominator-action-envelope-certificate-component},
and
\autoref{thm:field-rat-denominator-context-pair-product-strict-support-commutes};
exactness predicate the conjunction of inverse, product, denominator,
denominator-action envelope, contextual-action, context-pair,
strict-support commutation, and affine exactness rows; standard bridge
deferred; status $\mathsf{ProofTarget}$.

\begin{theorem}[RatUp FieldUp contextual action support transports across context-pair classifiers]
\label{thm:field-rat-denominator-contextual-action-pair-transport-support}
For the concrete denominator tuple
$(C_D^u,R_D^u,\mu_D^u,e_D^u,N_D^u)$ and the contextual action $\Gamma$ of
\autoref{def:ratup-fieldup-denominator-contextual-action-instance}, suppose
$C_D^u(h)$ and suppose that $(l,r)$, $(s,t)$, $(l',r')$, and $(s',t')$ are
carried denominator context pairs. If
$\mathsf{PairClass}_D^u((l,r),(s,t))$ and
$\mathsf{PairClass}_D^u((l',r'),(s',t'))$, and if
$p\hsame\emp$, $q\hsame\emp$, $p'\hsame\emp$, and $q'\hsame\emp$, then strict
support is invariant under the transported two-layer contextual action:
$$
N_D^u(\Gamma_{p',q',l',r'}(\Gamma_{p,q,l,r}(h)))
\Longleftrightarrow
N_D^u(\Gamma_{p',q',s',t'}(\Gamma_{p,q,s,t}(h))).
$$
\end{theorem}
\begin{proof}
Apply
\autoref{thm:ratup-fieldup-denominator-contextual-action-composition-closure}
to the left pair of carried contexts and the empty endpoint hypotheses. Its
support row reads the left nested action as the disjunction of the common core
$N_D^u(h)$ with the four strict-support terms from $(l,r)$ and $(l',r')$.

Apply the same composition-closure theorem to $(s,t)$ and $(s',t')$. This reads
the right nested action as the same core term $N_D^u(h)$ together with the four
strict-support terms from the transported context pairs.

Next use
\autoref{thm:field-rat-denominator-context-pair-product-strict-support} on both
context-pair products. The four support terms on the left are exactly strict
support of $\mu_D^u(\mu_D^u(l',l),\mu_D^u(r,r'))$, while the four support terms
on the right are strict support of
$\mu_D^u(\mu_D^u(s',s),\mu_D^u(t,t'))$.

The componentwise pair classifiers are the hypotheses required by
\autoref{thm:field-rat-denominator-context-pair-product-strict-support-transport}.
They transport the product strict-support term between the two context-pair
products.

For the forward direction, use the left composition readback. A core-support
witness $N_D^u(h)$ is inserted directly into the right composition readback. A
context-product support witness is transported across the product pair and then
inserted into the corresponding four context disjuncts on the right.

The reverse direction is symmetric. Use the right composition readback, transport
the product-pair support term back across the same classifier equivalence, keep
the common core support fixed, and close with the left composition readback.
\end{proof}

\begin{theorem}[RatUp FieldUp semantic certificate assembly]
\label{thm:ratup-fieldup-semantic-certificate-assembly}
For the concrete $\mathsf{BHist}$ data
$\mathsf{RatHistoryCarrier}$, $\mathsf{RatHistoryClassifier}$,
$\mathsf{RatDenomUnitCarrier}$, $\mathsf{RatDenomUnitClassifier}$,
$\mathsf{append}$, $\Cont$, $\emp$, and strict support
$\mathsf{RatHistoryCarrier}$, assume the $\RatUp$ source already carries the
$\CommRingUp$ base rows certified by
\autoref{thm:singleton-empty-history-commring-laws} and
\autoref{thm:singleton-empty-history-commring-schema-certificate}. With those
base rows in place, the FieldUp-specific inverse and apart-zero rows, the
two-sided zero-product and nonzero-product rows, the denominator
multiplicative rows, the denominator-action envelope rows, contextual
denominator rows, the denominator context-pair action rows, the context-pair
strict-support commutation row, and the packaged affine $\GroupUp$ consequence
supply the carrier, source, pattern, classifier, stability, ledger, and
exactness rows of
\autoref{def:field-certificate-obligations} for the concrete target
$\FieldUp(\mathsf{RatUp})$. More explicitly,
Definition~\ref{def:concrete-ratup-fieldup-certificate-instance} supplies the
concrete $\FieldUp(\mathsf{RatUp})$ certificate rows.
\end{theorem}
\begin{proof}
Bind the rational carrier and classifier first. The object carrier is
$\mathsf{RatHistoryCarrier}$, the object classifier is
$\mathsf{RatHistoryClassifier}$, and the source pattern is the BEDC
continuation pattern $\Cont$ over $\mathsf{BHist}$. The denominator tuple is
$(\mathsf{RatDenomUnitCarrier},\mathsf{RatDenomUnitClassifier},
\mathsf{append},\emp,\mathsf{RatHistoryCarrier})$, with contextual action
$\Gamma$ and affine coordinate $\mathsf{AffAut}_F$ as fixed in
Definition~\ref{def:concrete-ratup-fieldup-certificate-instance}.

The $\CommRingUp$ base rows are the first dependency. Theorem~\ref{thm:singleton-empty-history-commring-laws}
supplies the additive and multiplicative ring laws, distributivity, operation
congruence, multiplicative commutativity, and commutativity compatibility needed
by the field record.

Theorem~\ref{thm:singleton-empty-history-commring-schema-certificate} packages
those laws with the carrier, source, classifier, pattern, and ring ledger fields
over the concrete singleton source, so the base part of
\autoref{def:field-certificate-obligations} is already licensed before any
inverse or denominator row is attached.

The inverse rows are the apart-zero rows. Theorem~\ref{thm:field-apart-zero-inverse-classifier-exact-from-apartness}
supplies classifier exactness for proof-indexed inverses over the concrete
apart-zero predicate, so the inverse classifier, inverse stability, and
nonzero transport entries are read from the displayed apartness witnesses.

The zero-product rows come from
\autoref{thm:field-two-sided-zero-product-cancel-from-apartness}. Its
two-sided multiplication equivalence gives exact zero classification after
certified nonzero left and right factors, and the same witness data supply the
nonzero-product contradiction row in the per-object certificate record.

For the denominator part, use
\autoref{thm:field-rat-denominator-fieldup-multiplicative-obligation-coverage}
for the multiplicative denominator rows and
\autoref{thm:ratup-fieldup-denominator-package-closure} for the closed
denominator-side package.

The denominator-action envelope component is supplied by
\autoref{thm:ratup-fieldup-denominator-action-envelope-certificate-component}.
It records the multiplicative coverage, contextual boundary, normalized row
coverage, and context-pair product and action rows over the concrete tuple
fixed above.

The contextual denominator action is supplied by
\autoref{thm:ratup-fieldup-denominator-context-pair-action-certificate}. It
packages the pair classifier, pair product, unit-pair action, product-pair
composition law, and strict-support readback over the same BHist denominator
tuple, so the contextual exactness rows align with the denominator ledger
entries.

Separately,
\autoref{thm:field-rat-denominator-context-pair-product-strict-support-commutes}
supplies the strict-support commutation row for the context-pair product. This
is the support-transport row consumed by
Definition~\ref{def:concrete-ratup-fieldup-certificate-instance} between the
context-pair action certificate and the final FieldUp record.

Finally, \autoref{thm:field-affine-automorphism-group-certificate} supplies
the packaged affine $\GroupUp$ consequence consumed by the field certificate.

Together the $\CommRingUp$ base rows, the inverse rows, the zero-product rows,
the denominator multiplicative rows, the denominator-action envelope component,
the contextual denominator rows, the context-pair action rows, the
strict-support commutation row, and the affine $\GroupUp$ row exhaust the
ten-entry per-object template of
Definition~\ref{def:concrete-ratup-fieldup-certificate-instance}: target,
source, carrier, classifier, stability obligations, ledger, core theorem list,
exactness predicate, standard bridge, and status.
\end{proof}

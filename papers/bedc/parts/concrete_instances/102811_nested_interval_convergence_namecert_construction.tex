\chapter{A Concrete Naming Certificate for $\NestedIntervalConvergenceUp$}
\label{ch:concrete-instances-nested-interval-convergence-namecert}
\origin{human}

Following Bishop and Bridges constructive analysis, the $\NestedIntervalConvergenceUp$ packet records the handoff by which a displayed shrinking closed interval route is consumed as a regular rational completion name before a terminal real seal is read. It sits between the finite nested-packet surface of \autoref{ch:concrete-instances-nestedinterval-namecert}, the completion-facing consumer of \autoref{ch:concrete-instances-cantor-intersection-namecert}, the endpoint-cell discipline of \autoref{ch:concrete-instances-locatedinterval-namecert}, the closed-interval Cauchy route of \autoref{ch:concrete-instances-closed-interval-cauchy-completion-namecert}, and the $\RegSeqRatUp$ to $\RealUp$ seal route. The packet separates convergence handoff data from raw nested-interval rows and from the Cantor-intersection consumer: it names the finite diagonal enclosure, the regular-rational readback, and the real-seal nonescape surface without adding point choice, open-cover compactness, host real equality, or ambient $\RealUp$ completeness.

\begin{definition}[Nested interval convergence carrier]
\label{def:nested-interval-convergence-carrier}
A $\NestedIntervalConvergenceUp$ carrier is a finite $\BHist$ packet
$$
G=(N,L,W,D,E,R,Q,H,C,P,M)
$$
with the following displayed rows:
\begin{enumerate}
\item $N$ is an accepted $\NestedIntervalUp$ packet, carrying the finite endpoint, width, inclusion, schedule, regular-rational, and seal-facing rows already displayed by the nested-interval certificate.
\item $L$ is a $\LocatedIntervalUp$ row binding the visible lower and upper endpoint reads to rational cells and optional dyadic refinements.
\item $W$ is the shrinking-window ledger. For each displayed precision request it records a visible nested interval row, its endpoint enclosure, and a dyadic width bound small enough for the requested regular-rational tolerance.
\item $D$ is the diagonal selection row. It chooses from $N$ and $W$ a finite window for each displayed request; it does not select a completed intersection point.
\item $E$ records the endpoint enclosure rows selected by $D$ and the dependency path back to $N,L,W$.
\item $R$ is the $\RegSeqRatUp$ handoff row obtained from the selected endpoint enclosures and their displayed width bounds.
\item $Q$ is the closed-interval completion handoff row, reading the same selected enclosures through the $\ClosedIntervalCauchyCompletionUp$ route before any real-facing seal is exposed.
\item $H$ records the terminal $\RealUp$ seal row reached only after $N,L,W,D,E,R,Q$ have been displayed.
\item $C$ records componentwise $\hsame$ transport and displayed $\Cont$ replay for nested packets, located endpoints, shrinking ledgers, diagonal selections, enclosures, regular-rational handoffs, completion handoffs, and seal rows.
\item $P$ is the $\Pkg$ provenance over $\NestedIntervalConvergenceUp$, $\NestedIntervalUp$, $\LocatedIntervalUp$, $\ClosedIntervalCauchyCompletionUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $M$ is the local $\NameCert_{\NestedIntervalConvergenceUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $N,L,W,D,E,R,Q,H,C,P,M$. It has no coordinate for a raw infinite chain, a chosen intersection point, open-cover compactness, host real equality, quotient equality of real names, countable choice, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Nested interval convergence NameCert obligations]
\label{thm:nested-interval-convergence-namecert-obligations}
For every accepted $\NestedIntervalConvergenceUp$ carrier
$$
G=(N,L,W,D,E,R,Q,H,C,P,M),
$$
the local $\NameCert_{\NestedIntervalConvergenceUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the nested-interval packet, located-interval endpoint row, shrinking-window ledger, diagonal selection row, selected endpoint enclosures, regular-rational handoff, closed-interval completion handoff, real seal, transport, replay, provenance, and local naming rows;
\item nested-source exactness, requiring every interval read to open $N$ and every endpoint-cell read to open $L$ before the shrinking ledger $W$ is consumed;
\item diagonal-window exactness, requiring $D$ to select only a displayed finite window already recorded in $N,W$ for the current precision request;
\item enclosure-width exactness, requiring every row of $E$ to carry the selected endpoint enclosure together with the displayed dyadic width bound that justifies the request;
\item $\RegSeqRatUp$ handoff exactness, requiring $R$ to be read only from $E$ and its width ledger;
\item closed-interval completion ordering, requiring $Q$ to consume $N,L,W,D,E,R$ before the terminal $\RealUp$ seal $H$ is exposed;
\item provenance exhaustion, requiring $C,P,M$ to transport, replay, source, and name only the displayed route.
\end{enumerate}
No convergence obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:nested-interval-convergence-carrier}. The accepted packet displays the nested source $N$, located endpoint row $L$, shrinking ledger $W$, diagonal selector $D$, selected enclosures $E$, regular-rational handoff $R$, completion handoff $Q$, terminal seal $H$, and structural rows $C,P,M$.

Nested-source exactness and diagonal-window exactness are the first two route constraints. A convergence request must read the finite nested packet and the located endpoint cells before it can inspect $W$, and the selector $D$ can point only to a window already present in the displayed ledger.

Enclosure-width exactness then identifies the data used by the regular-rational handoff. Each row of $E$ carries the selected endpoint enclosure together with the dyadic width bound that answers the displayed precision request, so $R$ has no hidden source outside those rows.

The completion ordering is downstream of the handoff. The row $Q$ consumes the nested source, located endpoints, shrinking ledger, selector, enclosures, and regular-rational readback before the seal row $H$ becomes visible.

Finally, the transport, provenance, and local naming rows preserve the same finite inventory. Since the carrier contains no other coordinate, the listed obligations exhaust the local NameCert surface and cannot export an infinite intersection point, compactness theorem, host real equality, quotient equality, choice principle, or ambient completeness theorem.
\end{proof}

\begin{theorem}[Nested interval convergence carrier admission]
\label{thm:nested-interval-convergence-carrier-admission}
For every accepted $\NestedIntervalConvergenceUp$ carrier
$$
G=(N,L,W,D,E,R,Q,H,C,P,M),

carrier admission projects exactly the nested source $N$, the located endpoint-cell row $L$, the shrinking ledger $W$, the diagonal window selections $D$, the endpoint enclosures $E$, the $\RegSeqRatUp$ handoff $R$, the closed-interval completion handoff $Q$, the terminal $\RealUp$ seal $H$, componentwise $\hsame$ transport and displayed $\Cont$ replay in $C$, $\Pkg$ provenance $P$, and local $\NameCert_{\NestedIntervalConvergenceUp}$ naming row $M$.

No admitted coordinate is available outside these rows.
\end{theorem}
\begin{proof}
Project the accepted carrier through \autoref{def:nested-interval-convergence-carrier}. The tuple displays exactly the coordinates $N,L,W,D,E,R,Q,H,C,P,M$, and the classifier compares accepted packets only through those coordinates.

The carrier-admission row of \autoref{thm:nested-interval-convergence-namecert-obligations} records the same inventory as the local NameCert surface. Hence admission opens the nested source, located endpoint-cell row, shrinking ledger, diagonal selector, selected enclosures, handoff rows, seal row, transport and replay row, provenance row, and local naming row, with no extra projection.

Any selected point, raw infinite chain, host interval-intersection theorem, quotient equality, choice schedule, or ambient $\RealUp$ completeness datum would require a further carrier coordinate. The definition supplies none, so the admitted packet is exhausted by the displayed rows.
\end{proof}

\begin{theorem}[Nested interval convergence diagonal-window exactness]
\label{thm:nested-interval-convergence-diagonal-window-exactness}
For every accepted $\NestedIntervalConvergenceUp$ carrier
$$
G=(N,L,W,D,E,R,Q,H,C,P,M),
$$
each diagonal finite-window selection used by a $\RegSeqRatUp$ consumer is justified by the displayed shrinking ledger $W$ and by a dyadic width bound recorded in the selected endpoint enclosure row $E$ before the readback row $R$ can be consumed.

Thus $R$ cannot read a diagonal window not already displayed by $N,W,D,E$ for the current precision request.
\end{theorem}
\begin{proof}
Begin with the diagonal selector $D$. By the diagonal-window row of \autoref{thm:nested-interval-convergence-namecert-obligations}, $D$ may select only a finite window already recorded in the nested packet $N$ and the shrinking ledger $W$.

The selected enclosure row $E$ supplies the quantitative witness for that choice. Its enclosure-width row carries the endpoint enclosure together with the displayed dyadic width bound, so the precision request is justified before the regular-rational row is visible.

The handoff row $R$ is downstream from $E$. Combining the diagonal-window, enclosure-width, and $\RegSeqRatUp$ handoff rows gives the required route
$$
N,W,D,E \longmapsto R .
$$

No alternative readback remains, because a hidden diagonal choice or an unrecorded width certificate would lie outside the carrier coordinates admitted by \autoref{thm:nested-interval-convergence-carrier-admission}.
\end{proof}

\begin{proof}

The remaining structural data are $C,P,M$. The provenance-exhaustion row of \autoref{thm:nested-interval-convergence-namecert-obligations} requires these rows to transport, replay, source, and name only the displayed route, so the consumer cannot bypass the local NameCert surface.

The exclusions follow from carrier admission. A selected point, host interval-intersection theorem, quotient equality, choice schedule, or ambient completeness theorem would have to be stored as a further provenance branch or naming coordinate, but the accepted carrier contains only $N,L,W,D,E,R,Q,H,C,P,M$.

\begin{theorem}[Nested interval convergence RegSeqRat handoff]
\label{thm:nested-interval-convergence-regseqrat-handoff}
Every $\RegSeqRatUp$ consumer of an accepted $\NestedIntervalConvergenceUp$ carrier factors its read through the finite route
$$
N,L \longmapsto W \longmapsto D \longmapsto E \longmapsto R .
$$
Thus the consumer receives only a displayed nested source packet, located endpoint-cell data, a shrinking-window ledger, a diagonal finite-window selection, selected endpoint enclosures, and the dyadic width bounds used to build the regular-rational readback. It receives no chosen intersection point, no invisible tail of the interval chain, no quotient equality of real names, and no ambient $\RealUp$ completeness datum.
\end{theorem}
\begin{proof}
Start with the regular-rational row $R$ in \autoref{def:nested-interval-convergence-carrier}. The carrier makes $R$ downstream from the selected enclosures $E$, not from a separate real name or an unrecorded interval tail.

The selected enclosures are themselves downstream from $D$. By the diagonal-window obligation in \autoref{thm:nested-interval-convergence-namecert-obligations}, the selector can choose only a visible window from $N,W$ for the displayed precision request, with located endpoint-cell data supplied through $L$.

The dyadic width row in $E$ is the quantitative part of the handoff. It is the only tolerance witness available to $R$, so the regular-rational consumer receives exactly the finite enclosure and width data recorded in the packet.

No excluded source remains. A chosen intersection point, invisible tail, quotient real equality row, or ambient completeness theorem would require a coordinate outside $N,L,W,D,E,R$, and the carrier has none.
\end{proof}

\begin{theorem}[Nested interval convergence real-seal nonescape]
\label{thm:nested-interval-convergence-real-seal-nonescape}
For every accepted $\NestedIntervalConvergenceUp$ carrier
$$

The seal route does not claim open-cover compactness, a selected intersection point, uniqueness of the point, host real equality, quotient equality of real names, countable choice, or ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
Project the accepted carrier through the seal-facing coordinate $H$. The only route to $H$ named by the carrier passes through the regular-rational handoff $R$ and the closed-interval completion handoff $Q$.

Use \autoref{thm:nested-interval-convergence-regseqrat-handoff} to identify the source of $R$. The handoff reads only the finite nested source, located endpoints, shrinking ledger, diagonal selector, selected enclosures, and displayed width bounds.

Next use the completion-ordering obligation of \autoref{thm:nested-interval-convergence-namecert-obligations}. The row $Q$ consumes the same selected enclosure route before $H$ is exposed, so a real-facing read cannot bypass the finite completion surface.

The remaining rows $C,P,M$ transport, replay, source, and name this route. Because no carrier coordinate stores compactness, point selection, uniqueness, host equality, quotient equality, countable choice, or ambient completeness, every real-seal read remains inside the displayed finite packet.
\end{proof}

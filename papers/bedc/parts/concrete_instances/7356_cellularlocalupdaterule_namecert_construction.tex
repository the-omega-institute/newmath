\begin{theorem}[CellularLocalUpdateRule NameCert obligations]
\label{thm:cellular-local-update-rule-namecert-obligations}
For the carrier of \autoref{def:cellular-local-update-rule-carrier}, the local $\NameCert_{\CellularLocalUpdateRuleUp}$ surface consists exactly of five finite obligation rows:
\begin{enumerate}
\item carrier admission for the packet $U=(S,W,U,S',H,C,P,N)$;
\item neighbourhood visibility for the finite window row $W$;
\item rule-row accountability for the displayed table row $U$ and its visible output cell $S'$;
\item componentwise stability and replay through $H,C,P,N$;
\item non-escape from global orbit reads, hidden boundary conditions, host evaluators, and rule entries absent from $U$.
\end{enumerate}
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cellular-local-update-rule-carrier}. The packet displays the current state row, finite neighbourhood window, selected rule-table row, visible next-row cell, transport row, continuation route, provenance row, and local naming row before a consumer can read the local update.

Neighbourhood visibility and rule-row accountability are the two local coordinates of the carrier. The window row $W$ supplies the only input pattern, while $U$ supplies the only rule-table row and output cell available to the classifier.

The stability and replay rows are $H,C,P,N$. The $\hsame$ row transports corresponding displayed rows, the $\Cont$ row records the read route, and the package and local-name rows expose the certificate as a public NameCert object.

No remaining coordinate is present. A global orbit, hidden boundary condition, host evaluator, or absent rule entry therefore cannot be exported by the local $\NameCert_{\CellularLocalUpdateRuleUp}$ surface.
\end{proof}

\begin{theorem}[CellularLocalUpdateRule falsifiable boundary]
\label{thm:cellular-local-update-rule-falsifiable-boundary}
Every accepted $\CellularLocalUpdateRuleUp$ packet determines its visible next-row cell by the finite route
$$
W,U \longrightarrow S' \longrightarrow H,C,P,N.
$$
If a consumer obtains the next-row cell from a global orbit ledger, a hidden boundary condition, a host cellular evaluator, or a rule-table entry not displayed in $U$, then the consumer has left the accepted $\CellularLocalUpdateRuleUp$ packet.
\end{theorem}
\begin{proof}
Start with the obligation surface of \autoref{thm:cellular-local-update-rule-namecert-obligations}. The only neighbourhood input is $W$, and the only rule information is the displayed rule-table row $U$.

The output row $S'$ is then read along $H,C,P,N$. These rows transport, replay, package, and name the displayed local rule surface; they do not add an orbit ledger or evaluator.

Thus a read that depends on a global orbit ledger, hidden boundary condition, host evaluator, or absent table row is not a read through $W,U,S',H,C,P,N$. It is outside the accepted packet.
\end{proof}

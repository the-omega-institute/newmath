\chapter{A Concrete Naming Certificate for $\ContinuousIntervalImageUp$}
\label{ch:concrete-instances-continuous-interval-image-namecert}
\origin{human}

Following Bishop and Bridges constructive analysis, the $\ContinuousIntervalImageUp$ packet records the finite route by which the image of a connected closed bounded real interval under a displayed continuous Real-facing map remains available as a connected Real-facing image surface. It sits over the closed bounded interval boundary of \autoref{ch:concrete-instances-closedboundedinterval-namecert}, the connected interval route of \autoref{ch:concrete-instances-connectedinterval-namecert}, the intermediate-value bisection discipline of \autoref{ch:concrete-instances-intermediatevalue-namecert}, the graph and modulus rows of \autoref{ch:concrete-instances-continuousmap-namecert}, and the terminal real-name seal of \autoref{ch:concrete-instances-real-namecert}. The packet is a finite image certificate for Real and Completion consumers; it is not a host image set, an endpoint-selection principle, quotient equality of real names, trichotomy, countable choice, or ambient $\RealUp$ completeness.

\begin{definition}[Continuous interval image carrier]
\label{def:continuous-interval-image-carrier}
A $\ContinuousIntervalImageUp$ carrier is a finite $\BHist$ packet
$$
I=(B,J,F,V,W,R,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $B$ is the $\ClosedBoundedIntervalUp$ source row, exposing the displayed lower and upper $\RealUp$ endpoint names, rational bounds, endpoint-window requests, and interval membership ledger.
\item $J$ is the $\ConnectedIntervalUp$ row for the same source interval, recording the finite connectedness handoff consumed by image-connectedness reads.
\item $F$ is the $\ContinuousMapUp$ graph and modulus row, carrying the displayed Real-facing map, finite graph observations, modulus budget, and compatibility with the source interval windows.
\item $V$ is the $\IntermediateValueUp$ sign-route row, naming the bisection and sign-change surface used whenever an image request must pass between two displayed values.
\item $W$ is the $\StreamNameUp$ finite-window row opened for source endpoint names, graph observations, bisection probes, and image output reads.
\item $R$ is the $\RegSeqRatUp$ readback row attached to those windows, recording the rational approximants used by the map, intermediate-value route, and output seal.
\item $E$ is the output $\RealUp$ seal row; it is reached only after the source interval, connectedness row, continuous-map row, intermediate-value route, finite windows, and regular readbacks have been replayed.
\item $H$ records componentwise $\hsame$ transport for the closed bounded interval, connected interval row, continuous map, intermediate-value route, window row, readback row, output seal, replay, provenance, and local naming.
\item $C$ records displayed $\Cont$ replay from an image request through $B,J,F,V,W,R,E,H,P,N$.
\item $P$ is the $\Pkg$ provenance over $\ContinuousIntervalImageUp$, $\ClosedBoundedIntervalUp$, $\ConnectedIntervalUp$, $\ContinuousMapUp$, $\IntermediateValueUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\ContinuousIntervalImageUp}$ row.
\end{enumerate}
The classifier compares accepted image packets only through the displayed rows $B,J,F,V,W,R,E,H,C,P,N$. It has no coordinate for a host image subset, selected extrema, endpoint-choice functionals, quotient equality of real names, trichotomy, countable choice, ambient Real completeness, Lean verification, or bridge checking.
\end{definition}

\begin{theorem}[Continuous interval image NameCert obligations]
\label{thm:continuous-interval-image-namecert-obligations}
For every accepted $\ContinuousIntervalImageUp$ carrier
$$
I=(B,J,F,V,W,R,E,H,C,P,N),
$$
the local $\NameCert_{\ContinuousIntervalImageUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the closed bounded interval row, connected interval row, continuous-map graph and modulus row, intermediate-value sign route, finite window row, regular readback row, output Real seal, transport, replay, provenance, and local naming;
\item source-interval exactness, requiring every endpoint, rational bound, membership, and interval-window request to pass through the displayed $\ClosedBoundedIntervalUp$ row $B$;
\item connected-source exactness, requiring connectedness reads to pass through the displayed $\ConnectedIntervalUp$ row $J$ and the same source interval boundary $B$;
\item map-modulus exactness, requiring graph observations and continuity budgets to pass through the displayed $\ContinuousMapUp$ row $F$ and its finite window requests;
\item intermediate-value routing, requiring every image handoff between displayed source values to pass through the $\IntermediateValueUp$ sign-route row $V$ rather than through host order completeness;
\item output-seal ordering, requiring $W$ and $R$ to expose StreamName windows and RegSeqRat readbacks before the terminal $\RealUp$ seal $E$ is available;
\item nonescape, excluding host image subsets, selected endpoints or extrema, quotient real equality, trichotomy, countable choice, ambient $\RealUp$ completeness, Lean verification, and bridge checking.
\end{enumerate}
No continuous interval image obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:continuous-interval-image-carrier}. The accepted packet displays exactly one closed bounded interval source, one connected interval row, one continuous-map graph and modulus row, one intermediate-value sign route, one finite window row, one regular readback row, one output real seal, and the structural rows.

Source-interval exactness follows from $B$. Endpoint, rational-bound, membership, and source-window reads have no second interval coordinate, so each such read enters the image packet through the displayed $\ClosedBoundedIntervalUp$ carrier.

Connected-source exactness is supplied by $J$ over the same source row $B$. Since the packet contains no independent connectedness object, connected image reads must consume the connected interval route already attached to the closed bounded interval boundary.

Map-modulus exactness is supplied by $F$. Graph observations, continuity budgets, and compatibility checks all pass through the displayed $\ContinuousMapUp$ row before any image-output seal can be inspected.

Intermediate-value routing is supplied by $V$. When a consumer asks for a value between two displayed image values, the sign-route row provides the bisection surface and prevents the request from being replaced by host order completeness or a selected preimage.

The remaining rows $W,R,E,H,C,P,N$ impose readback order and local naming. StreamName windows and RegSeqRat readbacks precede the output Real seal, while transport, replay, provenance, and naming contain none of the excluded host image, choice, quotient, trichotomy, completeness, verification, or bridge data.
\end{proof}

\begin{theorem}[Continuous interval image Real seal route]
\label{thm:continuous-interval-image-real-seal-route}
Every Real-facing output read of an accepted $\ContinuousIntervalImageUp$ carrier factors through the finite route
$$
B,J,F,V \longmapsto W,R \longmapsto E,H,C,P,N .
$$
Thus the consumer first receives the closed bounded source interval, its connected interval handoff, the continuous-map graph and modulus data, and the intermediate-value sign route; only then may it read finite windows, regular rational approximants, the output $\RealUp$ seal, transport, replay, provenance, and local naming. The route exports no host image set, selected endpoint or extremum, quotient real equality, trichotomy, countable choice, or ambient $\RealUp$ completeness theorem.
\end{theorem}
\begin{proof}
Start with an accepted packet $I=(B,J,F,V,W,R,E,H,C,P,N)$. The source side is forced through $B$ and $J$: the closed bounded interval row supplies the endpoint and membership boundary, and the connected interval row supplies the connectedness handoff over that same boundary.

The image computation is then forced through $F$ and $V$. The continuous-map row provides graph observations and modulus budgets, while the intermediate-value row supplies the sign-route surface needed to pass between displayed values.

The Real-facing output is downstream of finite observation. The window row $W$ and readback row $R$ expose the finite stream and regular rational data before $E$ seals the output as a $\RealUp$ name.

Finally $H,C,P,N$ transport, replay, source, and name the same route. Since no row contains a host image subset, endpoint selector, extremum selector, quotient-real equality, trichotomy, countable choice, or ambient completeness witness, every output read remains inside the displayed factorization.
\end{proof}

\begin{theorem}[Continuous interval image connected handoff]
\label{thm:continuous-interval-image-connected-handoff}
Every connectedness read exported by an accepted $\ContinuousIntervalImageUp$ carrier factors through
$$
J \longmapsto F,V \longmapsto W,R,E,H,C,P,N .
$$
The read uses the source $\ConnectedIntervalUp$ row, the continuous-map graph and modulus row, the intermediate-value sign route, finite windows, regular readbacks, the output $\RealUp$ seal, and the structural NameCert rows. It does not introduce a separate host connected image object or a choice-selected preimage.
\end{theorem}
\begin{proof}
The source connectedness coordinate is $J$, and it is tied to the closed bounded interval row $B$ in the carrier. A connectedness read therefore begins with the displayed $\ConnectedIntervalUp$ handoff rather than a host connected-subset assertion.

Continuity transport is then read through $F$. The graph and modulus rows determine how source-window data are carried to image-window data, while $V$ supplies the intermediate-value sign route needed to keep the image between displayed values connected at the finite certificate level.

The output side is read only after $W,R,E$ have been replayed. These rows expose finite StreamName windows, RegSeqRat approximants, and the output Real seal before the structural rows $H,C,P,N$ transport, replay, source, and name the packet.

Because the packet has no coordinate for a host image set, selected preimage, endpoint selector, quotient real equality, trichotomy, countable choice, or ambient completeness, the displayed handoff is the whole connectedness surface exported by the certificate.
\end{proof}

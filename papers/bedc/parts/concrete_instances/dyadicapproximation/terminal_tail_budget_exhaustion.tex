\begin{theorem}[DyadicApproximation terminal tail-budget exhaustion]
\label{thm:dyadicapproximation-terminal-tail-budget-exhaustion}
Let $T$ be an accepted terminal $\DyadicApproximationUp$ finite packet, let $D$ be an accepted $\RealDiagonalCompletionUp$ packet matched to $T$ by the terminal diagonal handoff, and let $B$ be a $\FiniteErrorBudgetUp$ precision request consumed by the same terminal packet. Suppose the displayed transports and continuation routes identify the terminal precision row used by $T$, the diagonal-completion readback of $D$, and the finite budget request of $B$.

Then the consumed budget factors through exactly the terminal precision row, the transported dyadic endpoint, the $\RationalIntervalUp$ cell, the $\DyadicMeshUp$ selector, the $\ValidatedNumericsUp$ enclosure, the $\StreamNameUp$ window, the $\RegSeqRatUp$ readback, the $\DyadicTailBoundUp$ ledger, the $\RealUp$ seal, visible $\hsame$ rows, visible $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ rows already exposed by the terminal packet and its matched diagonal-completion surface.

The terminal packet exports no completed real equality, quotient stream, selected host limit, solver correctness, metric completeness, or host real equality to the finite budget consumer.
\end{theorem}
\begin{proof}
First normalize the terminal finite-packet real readback. Apply \autoref{thm:dyadicapproximation-terminal-finite-packet-real-readback}. The terminal packet is read through the displayed terminal precision, endpoint, rational interval cell, validated enclosure, stream window, regular-rational readback, real seal, transports, continuation routes, package provenance, and local certificate.

Next attach the terminal dyadic-net route. Apply \autoref{thm:dyadicapproximation-terminal-dyadic-net-consumption}. This identifies the dyadic endpoint with the $\RationalIntervalUp$ cell and $\DyadicMeshUp$ selector used by the terminal consumer, keeping the validated enclosure and real seal on the same finite surface.

Now consume the precision request. Apply \autoref{thm:finite-error-budget-tail-ledger-consumption-exactness} to force the $\FiniteErrorBudgetUp$ request through the displayed terminal tail ledger. Apply \autoref{thm:dyadic-tail-bound-budget-composition} to compose that ledger with the $\DyadicTailBoundUp$ row recorded by the dyadic packet.

Finally use the diagonal side. The terminal diagonal readback composition of \autoref{thm:dyadicapproximation-terminal-diagonal-readback-composition} keeps the matched $\RealDiagonalCompletionUp$ packet on the same terminal precision, endpoint, stream window, readback, real seal, transports, continuations, package provenance, and local certificate rows.

The cited surfaces contain the same terminal rows and no additional consumer channel. Repacking them gives exactly the factorization stated above, and the non-escape clauses from the terminal readback and diagonal-completion surfaces exclude completed real equality, quotient streams, selected host limits, solver correctness, metric completeness, and host real equality.
\end{proof}

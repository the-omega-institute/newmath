\begin{theorem}[Real convergent sequence NameCert obligations]\label{thm:real-convergent-sequence-namecert-obligations}

$$

\begin{enumerate}
\item carrier admission for the source sequence row, candidate $\RealUp$ limit, dyadic tolerance ledger, regular-Cauchy modulus, finite windows, regular rational readback, transport, replay, provenance, and local naming rows;
\item source-limit exposure, requiring $S$ and $L$ to be the only source of the displayed sequence and the displayed limit used by the convergence classifier;
\item tolerance and modulus exactness, requiring every convergence request to read its dyadic tolerance from $D$ and its tail threshold from $M$ before any source observation is compared with the limit;
\item window and readback alignment, requiring $W$ and $R$ to expose only the source windows and limit windows selected by the same tolerance request;
\item limit-law compatibility, requiring uses of real sequence limit laws to consume only the displayed rows $S,L,D,M,W,R$ and never a second limit surface;

\end{theorem}
\begin{proof}

The convergence request first reads $D$ and $M$. The dyadic row fixes the requested tolerance and comparison budget, while the modulus row fixes the tail threshold for source observations.

The selected observations are then opened through $W$ and $R$. The same tolerance request chooses both the source windows and the candidate-limit windows, and the regular rational row reads back only those scheduled observations.

Limit-law compatibility follows because every law-facing read has access only to $S,L,D,M,W,R$ before the structural rows. A second limit surface or host equality proof is not a carrier coordinate.

Finally, $H,K,P,N$ transport, replay, record, and name the same finite route. Since the excluded principles do not occur as carrier coordinates, the listed rows exhaust the local NameCert surface.

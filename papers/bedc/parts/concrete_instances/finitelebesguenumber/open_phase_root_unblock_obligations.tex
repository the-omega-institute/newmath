\begin{theorem}[Finite Lebesgue-number open Phase root-unblock radius coverage]
\label{thm:finite-lebesgue-number-open-phase-root-unblock-radius-coverage}
Every accepted $\FiniteLebesgueNumberUp$ root radius read exported to the open Phase Real route factors through the finite radius-unblock packet
$$
D,B,M,E,S,Q,R,H,C,P,N
$$
where $D$ is the displayed $\DyadicCoverUp$ cover row, $B$ is the $\RationalBallUp$ positive-radius row, $M$ is the $\DyadicMeshUp$ mesh row, $E$ is the $\DyadicRatCoreUp$ source face, $S,Q,R$ are the $\StreamNameUp$, $\RegSeqRatUp$, and $\RealUp$ faces, and the surrounding $\BHist$ records $H,C,P,N$ as the $\hsame$ transport, $\Cont$ replay, $\Pkg$ provenance, and compatible local $\NameCert$ rows. Thus the root-unblock obligation exposed to compact-uniform consumers is a finite BEDC packet over these rows, not a host-side radius source.
\end{theorem}
\begin{proof}
Start with the root radius-window surface. \autoref{thm:finite-lebesgue-number-root-radius-window-obligation-surface} identifies the positive radius and finite window through the dyadic cover, rational-ball radius, dyadic mesh, stream-name, regular-rational, Real, history, transport, replay, provenance, and naming rows.

Next attach the Phase Real source route. \autoref{thm:finite-lebesgue-number-phase-real-source-lock} forces each public radius read through the same dyadic, stream, regular-rational, and Real rows before a consumer can inspect it, while \autoref{thm:finite-lebesgue-number-real-phase-coverage-export} adds the $\DyadicRatCoreUp$ face to the compact, compact-net, continuous-map, and uniform-modulus export.

Finally read the exit and consumer ledgers. \autoref{thm:finite-lebesgue-number-root-unblock-exit-triad} gives the open Phase root-unblock exit surface, and \autoref{thm:finite-lebesgue-number-phase-real-radius-consumer-exhaustion} exhausts the consumer-visible radius source by the same finite rows. The displayed packet therefore covers every accepted root radius read exported to the open Phase Real route.
\end{proof}

\begin{theorem}[Finite Lebesgue-number open Phase root-unblock choice refusal]
\label{thm:finite-lebesgue-number-open-phase-root-unblock-choice-refusal}
The open Phase root-unblock obligation packet for $\FiniteLebesgueNumberUp$ exports no open-cover compactness principle, selected subcover, host minimum, quotient stream equality, countable choice coordinate, detached $\RealUp$ completeness row, or external modulus row. Any admitted compact, continuous, totally bounded, or uniform radius use must be read from the finite dyadic, four-face Real, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ packet of \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-radius-coverage}.
\end{theorem}
\begin{proof}
Read the positive side from the coverage theorem. \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-radius-coverage} leaves only the dyadic cover, rational-ball radius, dyadic mesh, $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, and BEDC support rows available to the open Phase root-unblock route.

The source and exit theorems supply the negative side. \autoref{thm:finite-lebesgue-number-phase-real-source-lock} prevents the source read from being replaced by an outside compactness or completeness primitive, and \autoref{thm:finite-lebesgue-number-root-unblock-exit-triad} states that the exit surface supplies no host Real equality, open-cover compactness, selected subcover, selected limit, detached completeness primitive, or external modulus data.

The consumer ledger closes the refusal at the endpoints. \autoref{thm:finite-lebesgue-number-phase-real-radius-consumer-exhaustion} removes open-cover compactness, host minima, selected subcovers, quotient stream equality, selected limits, countable choice, metric-completeness data, and detached $\RealUp$ completeness from compact, continuous, and uniform reads. The same finite packet is the only admitted source, so none of the listed outside rows can be exported.
\end{proof}

\begin{theorem}[Finite Lebesgue-number open Phase root-unblock consumer readback]
\label{thm:finite-lebesgue-number-open-phase-root-unblock-consumer-readback}
The $\CompactMetricUp$, $\ContinuousMapUp$, $\TotallyBoundedUp$, and $\UniformModulusUp$ consumers downstream of an accepted $\FiniteLebesgueNumberUp$ open Phase root-unblock route read the same finite packet. \end{theorem}
\begin{proof}
First identify the shared source. \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-radius-coverage} factors every accepted root radius read through the finite dyadic and four-face Real packet, and \autoref{thm:finite-lebesgue-number-root-unblock-exit-triad} presents that packet as the root-unblock exit surface for compact-uniform consumers.

Finally compare the endpoint consumers. \end{proof}

\begin{theorem}[Finite Lebesgue-number root-unblock endpoint uniformity]
\label{thm:finite-lebesgue-number-root-unblock-endpoint-uniformity}
If two downstream endpoint reads for $\CompactMetricUp$, $\ContinuousMapUp$, $\TotallyBoundedUp$, or $\UniformModulusUp$ are accepted from the same $\FiniteLebesgueNumberUp$ open Phase root-unblock route, then their visible positive-radius source is one finite packet over $\DyadicCoverUp$, $\RationalBallUp$, $\DyadicMeshUp$, $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$. Conversely, any endpoint read displaying exactly that packet is an admissible consumer-visible radius read for the same route.
\end{theorem}
\begin{proof}
Start from the shared source assertion. \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-radius-coverage} says that every accepted root radius read exported to the open Phase Real route factors through the displayed dyadic, rational-ball, mesh, dyadic-rational, stream, regular-rational, Real, history, transport, replay, provenance, and naming rows.

The refusal theorem removes every possible endpoint-specific replacement. By \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-choice-refusal}, the route exports no consumer-specific radius source, open-cover compactness, selected subcover, host minimum, quotient equality, choice coordinate, external modulus, selected limit, or detached $\RealUp$-completeness row. Thus two accepted endpoint reads cannot differ by importing any outside radius witness.

Now compare the endpoint reads. \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-consumer-readback} states that the $\CompactMetricUp$, $\ContinuousMapUp$, $\TotallyBoundedUp$, and $\UniformModulusUp$ consumers read exactly the same finite packet and add no endpoint-specific source. Hence any two accepted reads from the same root-unblock route have the same visible positive-radius source.

Conversely, an endpoint read that displays exactly the finite packet named above already presents the consumer-visible route required by \autoref{thm:finite-lebesgue-number-open-phase-root-unblock-consumer-readback}. Since the refusal theorem excludes only outside rows and the displayed packet is precisely the covered source, such a read is admissible for the same open Phase root-unblock route.
\end{proof}

\begin{definition}[Cauchy continuity principle carrier]
\label{def:cauchy-continuity-principle-carrier}
A $\CauchyContinuityPrincipleUp$ carrier is a finite $\BHist$ packet
$$
Q=(S,W,D,M,U,T,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the source $\RegSeqRatUp$ row naming the regular rational Cauchy source data.
\item $W$ is the finite $\StreamNameUp$ window family through which source and target observations are inspected.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger recording the requested target tolerances, source-window budgets, and comparison rows.
\item $M$ is the $\CauchyModulusOfContinuityUp$ data that converts each target tolerance into the source-window read used by the map.
\item $U$ is the $\UniformContinuityUp$ compatibility row, requiring the pointwise Cauchy-modulus route to agree with the available uniform read whenever a uniform packet is supplied.
\item $T$ is the $\RegularCauchyTailEstimateUp$ target handoff, recording the target tail estimate obtained after $S,W,D,M,U$ have been replayed.
\item $E$ is the $\RealUp$ seal row reached only after the target $\RegSeqRatUp$ handoff carried by $T$ is displayed.
\item $H$ records componentwise $\hsame$ transport for $S,W,D,M,U,T,E$ and for the replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a target tolerance request through $D$, then $M$, then the source row $S$ and finite windows $W$, then $U$, then the target tail estimate $T$, and finally the seal row $E$.
\item $P$ is the $\Pkg$ provenance over $\CauchyContinuityPrincipleUp$, $\RegSeqRatUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, $\CauchyModulusOfContinuityUp$, $\UniformContinuityUp$, $\RegularCauchyTailEstimateUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CauchyContinuityPrincipleUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $S,W,D,M,U,T,E,H,C,P,N$. It has no coordinate for host function equality, quotient completion, countable choice, selected limits, ambient $\RealUp$ completeness, or equality of completed real values.
\end{definition}

\begin{theorem}[Cauchy continuity principle NameCert obligations]
\label{thm:cauchy-continuity-principle-namecert-obligations}
For every accepted $\CauchyContinuityPrincipleUp$ carrier $Q=(S,W,D,M,U,T,E,H,C,P,N)$, the local $\NameCert_{\CauchyContinuityPrincipleUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the source regular Cauchy row, finite windows, dyadic tolerance ledger, Cauchy-continuity modulus, uniform compatibility row, target tail estimate, Real seal, transport, replay, provenance, and local naming row;
\item classifier transport, requiring $H$ and $C$ to preserve the ordered route $D,M,S,W,U,T,E$ componentwise;
\item modulus exactness, requiring every target tolerance to reach its source-window budget only through the displayed row $M$ and the dyadic ledger $D$;
\item regular-target handoff, requiring the target Cauchy read to pass through $T$ before the Real seal $E$ is exposed;
\item ledger non-escape, requiring $P$ and $N$ to export no host function equality, quotient completion, countable choice, selected limit, ambient Real completeness, or equality of completed real values.
\end{enumerate}
No local NameCert obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:cauchy-continuity-principle-carrier}. The projection exposes exactly the regular source row $S$, finite windows $W$, dyadic ledger $D$, modulus data $M$, uniform compatibility row $U$, target tail estimate $T$, Real seal $E$, and the structural rows $H,C,P,N$.

Classifier transport is componentwise. The transport row $H$ moves the displayed coordinates to a common finite surface, and the replay row $C$ repeats the same dependency order $D,M,S,W,U,T,E$ for every accepted tolerance request.

Modulus exactness is exhausted by $M$ and $D$. A target tolerance first reads the dyadic ledger, then the Cauchy-continuity modulus row, and only then obtains the source-window budget used to inspect $S$ through $W$.

The regular-target handoff is the row $T$. It records the finite target tail estimate produced after the source row, windows, ledger, modulus, and compatibility row have been replayed, so the Real seal $E$ is downstream of the regular target data.

Finally inspect $P$ and $N$. Since the carrier contains no host function equality, quotient completion, countable-choice source, selected limit, ambient completeness witness, or equality of completed real values, the listed rows exhaust the local NameCert obligations.
\end{proof}

Use the assumed handoff identification. The row $S_2$ is not an ambient equality of completed values; it is the target $\RegSeqRatUp$ handoff displayed by $T_1$.

\begin{theorem}[Reality-constrained methodology-ledger public export readiness]
\label{thm:reality-constrained-methodology-ledger-public-export-readiness}
For every accepted $\RealityConstrainedMethodologyLedgerUp$ packet
$L=(X,A,O,T,I,S,D,U,F,H,C,Q,N)$, the public export surface is the finite
methodology-ledger interface carried by the explanation row $X$, audit row
$A$, objectivity row $O$, tower-compression row $T$, open-fit row $I$, scope
row $S$, refusal row $D$, upgrade row $U$, failure row $F$, transport row $H$,
consumer row $C$, provenance row $Q$, and local $\NameCert$ row $N$. Any public
export of methodology authority must pass through those rows, the existing
failure-consumer coverage, the formal-target route totality surface, and the
TasteGate readback surface before it can be consumed.
\end{theorem}
\begin{proof}
Project the carrier using
\autoref{def:reality-constrained-methodology-ledger-carrier}. The displayed
coordinates are exactly the five sibling rows $X,A,O,T,I$ together with the
methodology scope, refusal, upgrade, failure, transport, consumer, provenance,
and local naming rows.

Apply the NameCert obligations,
\autoref{thm:reality-constrained-methodology-ledger-namecert-obligations}.
They bind carrier admission, scope exactness, refusal, upgrade, failure
exposure, sibling accountability, and ledger non-escape to the same local
certificate surface.

Next use failure-consumer coverage and formal-target route totality,
\autoref{thm:reality-constrained-methodology-ledger-failure-consumer-coverage}
and
\autoref{thm:reality-constrained-methodology-ledger-formal-target-route-totality}.
These results place the failure row and consumer route before public
methodology consumption and keep TruthCert-facing access inside the same
$\BHist/\hsame/\Cont/\Pkg/\NameCert$ support.

Finally apply the TasteGate readback surface,
\autoref{thm:reality-constrained-methodology-ledger-tastegate-readback-surface}.
It identifies the event-flow decoding with the displayed carrier, so the
public export is a readback of $L$ rather than an off-ledger methodology object.
\end{proof}

\begin{theorem}[Reality-constrained methodology-ledger TruthCert consumer handoff]
\label{thm:reality-constrained-methodology-ledger-truthcert-consumer-handoff}
For every accepted $\RealityConstrainedMethodologyLedgerUp$ packet
$L=(X,A,O,T,I,S,D,U,F,H,C,Q,N)$, any TruthCert-facing consumer handoff receives
the packet only after the carrier rows, local $\NameCert$ obligations,
failure-consumer coverage, formal-target route totality, and TasteGate
readback have displayed the same finite methodology ledger. The handoff cannot
replace the explanation, audit, objectivity, tower-compression, open-fit,
scope, refusal, upgrade, failure, transport, consumer, provenance, or local
NameCert rows by a domain-truth oracle or a physical-model adequacy theorem.
\end{theorem}
\begin{proof}
Begin with the root TruthCert handoff theorem,
\autoref{thm:reality-constrained-methodology-ledger-root-truthcert-handoff}.
It already forces a TruthCert-facing read through the local methodology rows
instead of through an external truth coordinate.

Use public export readiness,
\autoref{thm:reality-constrained-methodology-ledger-public-export-readiness},
to identify the available consumer surface. The handoff receives the same
finite packet rows and no additional methodology primitive.

Now apply the formal-target route totality theorem. It puts refusal-upgrade,
failure-consumer, and prediction-facing reads before TruthCert consumption and
keeps all of them in the displayed $\BHist/\hsame/\Cont/\Pkg/\NameCert$
support.

Finally use the TasteGate readback surface. Since the decoded event-flow
carrier is the same $L$, a TruthCert-facing consumer handoff reads the public
methodology ledger without replacing it by a separate domain-truth or adequacy
claim.
\end{proof}

\begin{theorem}[Reality-constrained methodology-ledger root dependency totality]
\label{thm:reality-constrained-methodology-ledger-root-dependency-totality}
For every accepted root methodology consumer of a
$\RealityConstrainedMethodologyLedgerUp$ packet
$L=(X,A,O,T,I,S,D,U,F,H,C,Q,N)$, the consumer reads the explanation, audit,
objectivity, tower-compression, open-fit, scope, refusal, upgrade, failure,
transport, consumer, provenance, and local NameCert rows before any TruthCert
consumer handoff. In particular, the five sibling dependencies
$\RealityConstrainedExplanationUp$, $\RealityConstrainedModelAuditUp$,
$\RealityConstrainedObjectivityUp$, $\RealityConstrainedTowerCompressionUp$,
and $\OpenFitPacketUp$ remain displayed as rows of the same finite ledger.
\end{theorem}
\begin{proof}
Start from the methodology root triad,
\autoref{thm:reality-constrained-methodology-ledger-methodology-root-triad}.
It exposes the five sibling rows, the scope-refusal-upgrade rows, and the
failure-consumer-provenance-name rows inside one carrier.

Apply root sibling accountability and the public obligation surface,
\autoref{thm:reality-constrained-methodology-ledger-root-sibling-accountability}
and
\autoref{thm:reality-constrained-methodology-ledger-public-obligation-surface}.
These keep the explanation, audit, objectivity, tower-compression, and
open-fit dependencies visible as rows of $L$ rather than as hidden sources.

Next use failure-consumer public scope,
\autoref{thm:reality-constrained-methodology-ledger-failure-consumer-public-scope}.
It requires the failure row and displayed sibling, scope, refusal, and upgrade
rows before the consumer route $C$ may be read.

Finally apply the TruthCert consumer handoff theorem,
\autoref{thm:reality-constrained-methodology-ledger-truthcert-consumer-handoff}.
The handoff therefore occurs only after the root consumer has read every row
listed in the carrier and the five sibling dependencies remain part of the
same public methodology ledger.
\end{proof}

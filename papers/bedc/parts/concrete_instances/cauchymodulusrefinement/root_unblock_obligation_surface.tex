\begin{theorem}[Cauchy modulus refinement root source binding]
\label{thm:cauchy-modulus-refinement-root-source-binding}
Every root consumer of an accepted $\CauchyModulusRefinementUp$ packet first reads the source rows $M_0,M_1$, the shared meet row $U$, and the extraction row $V$ through the carrier route of \autoref{def:cauchy-modulus-refinement-carrier}. The root consumer has no admissible access to a downstream selector, seal, or structural row before that shared meet and extraction route is displayed.
\end{theorem}
\leanchecked{BEDC.Derived.CauchyModulusRefinementUp.CauchyModulusRefinementCarrier\_root\_source\_binding}
\begin{proof}
Read the carrier definition. Its packet orders the two source schedules, the shared meet, and the extraction witness as the visible front of the root route.

Apply \autoref{thm:cauchy-modulus-refinement-shared-meet-dependency}. The theorem binds any extraction read of $V$ to the single displayed meet $U$ after $M_0,M_1$; hence a root consumer cannot substitute a second meet or a hidden source.

Finally use \autoref{thm:cauchy-modulus-refinement-real-consumer-coverage}. Since every Real or Completion consumer is covered by the displayed packet route, every such consumer inherits this first read of $M_0,M_1,U,V$ before any later row is available.
\end{proof}

\begin{theorem}[Cauchy modulus refinement root selector readback]
\label{thm:cauchy-modulus-refinement-root-selector-readback}
For every accepted $\CauchyModulusRefinementUp$ packet, selected-window and tolerance readback consume $T,W,Q$ only after the source-meet-extraction row $M_0,M_1,U,V$ has been displayed. Thus any root consumer that asks for the selected-window ledger or the tolerance-readback ledger reaches those rows through the same selector-budget route $T$.
\end{theorem}
\leanchecked{BEDC.Derived.CauchyModulusRefinementUp.CauchyModulusRefinementCarrier\_root\_selector\_readback}
\begin{proof}
Start from \autoref{def:cauchy-modulus-refinement-carrier}. The selected-window ledger $W$ and tolerance-readback ledger $Q$ are carrier rows following $M_0,M_1,U,V,T$.

Apply \autoref{thm:cauchy-modulus-refinement-shared-meet-dependency} to fix the source side. The extraction row $V$ is already bound to the shared meet route before any selector-budget read is meaningful.

Now apply \autoref{thm:cauchy-modulus-refinement-selector-budget-dependency}. It identifies $T$ as the selector-budget row consumed by the selected-window and tolerance-readback rows, so $W,Q$ are read only after the extraction row and through $T$.
\end{proof}

\end{theorem}
\begin{proof}
Apply \autoref{thm:cauchy-modulus-refinement-root-source-binding}. The shared source rows force the same $M_0,M_1,U,V$ spine for both reads, so there is no separate source-meet branch on which a second selector route could start.

Next apply \autoref{thm:cauchy-modulus-refinement-root-selector-readback}. The selector-budget, selected-window, and tolerance-readback rows are read back through the same displayed selector surface $T,W,Q$, and the carrier exposes no hidden selector coordinate beyond that surface.

The Real seal row and structural rows remain inside the same $E,H,C,P,N$ packet under componentwise $\hsame$ transport and displayed $\Cont$ replay, so quotient streams, selected limits, host equalities, second meets, and ambient completeness rows are not root-facing witnesses.

\end{proof}

\begin{theorem}[Cauchy modulus refinement selector-meet noninterference]
\label{thm:cauchy-modulus-refinement-selector-meet-noninterference}
For an accepted $\CauchyModulusRefinementUp$ packet whose root route displays
$$
M_0,M_1,U,V,T,W,Q,E,H,C,P,N,
$$
any consumer request for a hidden selector, second meet, quotient stream,
selected limit, host equality, or ambient completeness row cannot alter the
accepted selector route. Every accepted read remains determined by the shared
meet $U$, extraction row $V$, selector-budget row $T$, selected-window row
$W$, tolerance-readback row $Q$, Real-seal row $E$, and structural rows
$H,C,P,N$.
\end{theorem}
\begin{proof}
First apply \autoref{thm:cauchy-modulus-refinement-root-source-binding}. The
accepted root route must begin with the source rows $M_0,M_1$, the shared meet
$U$, and the extraction row $V$, so a hidden second meet has no source position
from which to change the selector route.

Next use \autoref{thm:cauchy-modulus-refinement-root-selector-readback}. The
selector-facing rows $T,W,Q$ are read only after the source-meet-extraction
route has been displayed, and their readback is tied to the same
selector-budget row $T$.

Apply \autoref{thm:cauchy-modulus-refinement-root-real-seal-nonescape} and
\autoref{thm:cauchy-modulus-refinement-root-route-completeness}. They keep the
Real seal and structural rows on the displayed route while excluding quotient
streams, selected limits, host equalities, hidden selectors, second meets, and
ambient completeness rows from the exported packet.

Finally invoke \autoref{thm:cauchy-modulus-refinement-shared-selector-determinacy}.
Once the displayed rows agree up to componentwise $\hsame$ transport and
displayed $\Cont$ replay, no remaining root-facing coordinate can distinguish
another accepted selector route. The requested hidden data are therefore
noninterfering with the accepted route.
\end{proof}

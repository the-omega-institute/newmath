\chapter{A Concrete Naming Certificate for $\SetUp$}

The set interface $\SetUp$ packages a carrier together with a $\psame$-classifier interpretable as set-membership equality. It is the type-layer counterpart of $\NameCert_{\PreorderUp}$ (\autoref{ch:concrete-instances-preorder-namecert}) at the discrete level, and the substrate for $\NameCert_{\FinSetUp}$ (\autoref{ch:concrete-instances-finset-namecert}) and $\NameCert_{\SubtypeUp}$ (\autoref{ch:concrete-instances-subtype-namecert}).

\begin{theorem}[Set carrier obligation]
\label{thm:set-carrier-obligation}
The carrier row of a $\NameCert_{\SetUp}$ is a predicate $A:\mathsf{BHist}\to\mathsf{Prop}$ over BEDC histories. It is generated by membership-visible histories: a history $h$ is carried exactly when the $\Hist$ presentation exposes a membership row for $h$, and the visible row is read through $\mathsf{BHist}$ rather than through a host-side set object.
\end{theorem}
\begin{proof}
Unpack the source package as a $\Pkg$ row over $\Hist$ and read its public histories through the $\mathsf{BHist}$ interface. The carrier predicate records precisely those histories whose membership row is visible to the package boundary.

The construction is constructive because no host enumeration or host subset is selected. A carried witness is the displayed membership row together with its $\Pkg$ provenance, and forgetting the witness returns the original $\mathsf{BHist}$ history. This is the carrier surface consumed later by $\FinSetUp$ and $\SubtypeUp$.
\end{proof}

\begin{theorem}[Set classifier obligation]
\label{thm:set-classifier-obligation}
For a fixed $\NameCert_{\SetUp}$ carrier $A$, the classifier row is a $\psame$-style membership classifier on carried histories. Two carried histories are classified together exactly when their membership-visible rows are identified by the BEDC presentation classifier; the row is not host equality of set elements and is not a quotient by an external relation.
\end{theorem}
\begin{proof}
The classifier is read from the $\NameCert$ row attached to the package, so its inputs are carried histories with membership witnesses in $A$. Applying the $\psame$ classifier compares the visible membership rows and returns the certificate-level sameness relation.

Because the comparison is made inside $\psame$, no appeal is made to host equality. Because the witnesses remain in the $\Pkg$ ledger, no quotient carrier is formed. The result is the classifier boundary required before finite enumeration in $\FinSetUp$ or predicate restriction in $\SubtypeUp$ may consume the set source.
\end{proof}
\leanchecked{BEDC.Derived.SetUp.SetPkgMembership\_semantic\_name\_certificate}

\begin{theorem}[Set membership transport obligation]
\label{thm:set-membership-transport-obligation}
The $\SetUp$ membership row is stable under $\hsame$ transport and under the $\psame$ classifier. If $h$ is carried by $A$, $h \hsame h'$, and the visible membership rows for $h$ and $h'$ are related by the classifier, then $h'$ is carried by $A$ with the transported membership witness and the same package provenance boundary.
\end{theorem}
\begin{proof}
First transport the displayed $\Hist$ membership row along $\hsame$. The transport changes only the presentation of the history, while the $\Pkg$ provenance records that the read remains inside the same certificate boundary.

Next apply the $\psame$ classifier row to the transported membership presentation. This supplies the certificate-level identification between the original membership read and the transported read.

Combining the transported witness with the classifier evidence yields a carried witness for $h'$. The dependency remains constructive and visible: the proof uses only $\NameCert$, $\psame$, $\hsame$, and $\Pkg$, so downstream $\FinSetUp$ and $\SubtypeUp$ consumers receive a stable membership row rather than an external equality principle.
\end{proof}
\leanchecked{BEDC.Derived.SetUp.SetMembershipVisibleTransportChain\_hsame}

\begin{theorem}[Set extensionality ledger obligation]
\label{thm:set-extensionality-ledger-obligation}
The $\SetUp$ extensionality ledger records exactly the visible membership rows and their exact package boundary. Finite consumers such as $\FinSetUp$ and restricted consumers such as $\SubtypeUp$ may consume the carrier $A$, the $\psame$ classifier, and the $\hsame$-stable membership transport row, but they may not inspect hidden package data outside the ledger boundary.
\end{theorem}
\leanchecked{BEDC.Derived.SetUp.SetExtensionalityLedger\_semantic\_name\_certificate}
\begin{proof}
The extensionality row compares two $\SetUp$ presentations by their visible membership classifications. If the same $\mathsf{BHist}$ histories are carried and their visible membership rows agree by $\psame$, the public set presentation agrees at the certificate boundary.

The exact ledger boundary is the accompanying $\Pkg$ provenance for those membership reads. It records which rows are exposed and which witnesses remain hidden, so finite enumeration in $\FinSetUp$ can use only occurrence of visible members and $\SubtypeUp$ can use only visible restriction of the carrier.

Thus the ledger gives an extensionality obligation without exporting a host set, a host equality, or any hidden construction witness. The only downstream dependency made available is the membership-visible carrier, classifier, transport, and exact boundary described above.
\end{proof}

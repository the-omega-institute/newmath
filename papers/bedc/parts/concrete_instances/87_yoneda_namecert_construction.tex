The Yoneda interface $\YonedaUp$ packages, for each object of a category, the representable functor and the natural-isomorphism witness of the Yoneda lemma. It depends on $\NameCert_{\FunctorUp}$ (\autoref{ch:concrete-instances-functor-namecert}) and $\NameCert_{\NatTransUp}$ (\autoref{ch:concrete-instances-nattrans-namecert}), supplying the embedding into presheaf categories.

\begin{theorem}[Yoneda representable empty component family characterization]
\label{thm:yoneda-representable-empty-component-family-iff}
For histories $p$ and $q$, an empty natural-transformation component exists
over every unary object exactly when $p$ and $q$ are unary prefixes with the
same history:
$$
\Bigl(\forall a,\ \mathsf{UnaryHistory}(a)\Rightarrow
\mathsf{NatTransPrefixComponentCarrier}(p,q,a,\emp)\Bigr)
\quad\Longleftrightarrow\quad
\mathsf{UnaryHistory}(p)\wedge\mathsf{UnaryHistory}(q)\wedge\HSame(p,q).
$$
\end{theorem}
\begin{proof}
For the forward direction, evaluate the component family at the empty unary
history and apply empty-component identity exactness. For the reverse direction,
the same exactness theorem rebuilds each object component from the common prefix
equality and the supplied unary object carrier.
\end{proof}

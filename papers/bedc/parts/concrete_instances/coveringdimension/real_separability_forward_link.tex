\begin{theorem}[Covering dimension real-separability forward link]
\label{thm:coveringdimension-real-separability-forward-link}
Every finite refinement row used by the $\CoveringDimensionUp$ packet may be read as a consumer of the dense $\RealSeparabilityUp$ route of \autoref{ch:concrete-instances-real-separability-namecert} when the covering-dimension argument is applied to $\RealUp$-derived metric carriers. The dependency is the finite dense-window refinement surface: a displayed dense real window supplies sample points, a finite cover row supplies the cover cells inspected at those samples, and the refinement row records only the finite multiplicity bound visible at that window.

The link does not assert ambient separability, countable choice of dense points, a host open-cover compactness principle, selected real representatives, or a detached dimension oracle.
\end{theorem}
\begin{proof}
Project the $\CoveringDimensionUp$ carrier through \autoref{def:covering-dimension-carrier}. The refinement row is visible only together with the compact metric source, finite epsilon-net row, cover row, order-bound row, Lebesgue ledger, transport, replay, provenance, and local naming rows.

Use the RealSeparability route of \autoref{ch:concrete-instances-real-separability-namecert}. For a $\RealUp$-derived metric carrier, the dense-window row supplies the finite real samples that the covering-dimension refinement consumes; the refinement is therefore read from displayed dense windows rather than from an ambient separability theorem.

Finally apply the existing separability cover-refinement route, \autoref{lem:covering-dimension-separability-cover-refinement-route}. It identifies the finite compact-metric, epsilon-net, cover, refinement, order-bound, Lebesgue-ledger, and structural rows through which the consumer reads the dependency, and it excludes detached host covers, choice-selected refinements, and external multiplicity oracles.
\end{proof}

\begin{proposition}[CoveringDimension metric separability dependency route]
\label{prop:coveringdimension-metric-real-separability-route}
Every $\CoveringDimensionUp$ consumer that uses $\RealSeparabilityUp$ data factors through the metric-ball surface of \autoref{ch:concrete-instances-metric-namecert}. The exported route is only the finite chain
$$
\MetricUp \longrightarrow \RealSeparabilityUp \longrightarrow \CoveringDimensionUp
$$
read through the carrier of \autoref{def:covering-dimension-carrier}; it does not select dense points, arbitrary open covers, or external refinements.
\end{proposition}
\begin{proof}
Project the carrier of \autoref{def:covering-dimension-carrier}. The finite cover row and refinement row can be inspected only through displayed metric balls and the compact metric source already named by the packet.

Read the $\RealSeparabilityUp$ sibling through rational dense windows. Those windows contribute finite sample rows for the displayed metric balls, not selected points outside the certificate surface.

The order-bound ledger is then read from the same finite refinement packet. Since the route has no coordinate for arbitrary open covers or external refinements, the dependency remains the displayed metric-to-separability route.
\end{proof}

\begin{theorem}[Covering dimension Real and Metric forward route]
\label{thm:covering-dimension-real-metric-forward-route}
Every $\CoveringDimensionUp$ consumer that enters the Real and Metric dependency lattice reads $\RealSeparabilityUp$ and $\MetricUp$ only through the finite cover, finite-net, dyadic-density, compact-metric, and Lebesgue-ledger rows already displayed by the accepted carrier. The route exposes compact metric source rows, metric ball inspection, finite epsilon-net centers, dyadic density witnesses, cover-refinement rows, and the order-bound ledger before any dimension bound is consumed.

It exports no ambient metric completion, selected dense sequence, host open-cover compactness theorem, quotient real equality, or external multiplicity oracle.
\end{theorem}
\begin{proof}
Project the accepted carrier $D=(K,E,C,R,O,L,H,T,P,N)$ from \autoref{def:covering-dimension-carrier}. The compact metric source $K$, finite epsilon-net row $E$, cover row $C$, refinement row $R$, order-bound row $O$, and Lebesgue ledger $L$ are the only analytic rows visible to the packet.

The Real-facing information is read through the existing $\RealSeparabilityUp$ routes in this chapter. Those routes supply rational and dyadic density witnesses as finite rows before the cover and refinement rows are inspected; they do not select an ambient dense sequence.

The Metric-facing information is read through the compact metric and metric ball rows used by $K$ and $E$. Thus pointwise cover inspection is finite and row-local, and the order-bound ledger cannot be replaced by a host compactness or multiplicity theorem.

Finally $H,T,P,N$ transport, replay, package, and name the same finite route. Since no extra completion, selected dense sequence, host open-cover theorem, quotient real equality, or external oracle occurs in the carrier, the Real and Metric forward route is exactly the displayed finite dependency surface.
\end{proof}

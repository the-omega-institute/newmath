\begin{theorem}[CoveringDimension RealSeparability MetricCompletion cover route]
\label{thm:coveringdimension-real-separability-metric-completion-cover-route}
Every finite-cover dimension read over Real-facing metric data must expose $\RealSeparabilityUp$ finite dense windows and $\MetricCompletionUp$ completion boundary rows before a $\CoveringDimensionUp$ consumer may inspect cover order. The route presents compact metric source rows, finite cover and refinement rows, separability witnesses, completion-stable metric rows, the order-bound ledger, transport, replay, provenance, and local naming, but it exports no arbitrary open-cover compactness, selected dense points, choice-selected refinements, quotient point equality, or ambient completion theorem.
\end{theorem}
\begin{proof}
Begin with the carrier of \autoref{def:covering-dimension-carrier}. Its cover-order read is available only through the compact metric source, finite epsilon-net schedule, finite cover row, refinement row, order-bound witness, compact-cover Lebesgue ledger, transport, replay, provenance, and local naming.

Apply the RealSeparability routes \autoref{lem:covering-dimension-separability-cover-refinement-route} and \autoref{thm:coveringdimension-real-separability-forward-bound}. They place finite rational and dyadic dense windows before any cover-refinement or order-bound row is consumed.

Use the metric-completion route \autoref{thm:coveringdimension-cauchy-filter-completion-route}. It sends completion-facing cover reads through the displayed completion boundary rather than through an ambient completion theorem.

Together the cited routes force finite dense windows and completion boundary rows to appear before cover order is inspected. Since none of the rows contains arbitrary open covers, selected dense points, choice-selected refinements, quotient equality, or ambient completion data, the composed route exports only the finite cover dependency.
\end{proof}

\begin{theorem}[Prod hub Bool sibling audit link]
\label{thm:prod-hub-bool-sibling-audit-link}
The public $\ProdUp$ ordered-pair read consumes $\BoolUp$ from \autoref{ch:concrete-instances-bool-namecert} only as the finite endpoint discriminator before any component row is exposed. The Bool row distinguishes the left and right endpoint positions of the displayed-pair ledger; payload carriers, component classifiers, source $\NameCert$ rows, $\hsame$ transports, $\Cont$ replays, and $\Pkg$ provenance remain the product rows named by the $\ProdUp$ packet.
\end{theorem}
\begin{proof}
Read the product hub through its child rows. The componentwise interface and displayed-pair ledger expose endpoint positions before they expose payload components.

Apply the Bool endpoint-discriminator route. It identifies the $\BoolUp$ dependency as the finite left-versus-right tag discipline for those endpoint positions.

The public finite-data exit package then supplies the rest of the product route. Since payload, classifier, transport, replay, and provenance rows are still owned by the product and source certificates, the Bool sibling is a genuine horizontal dependency but not a replacement for the $\ProdUp$ carrier.
\end{proof}

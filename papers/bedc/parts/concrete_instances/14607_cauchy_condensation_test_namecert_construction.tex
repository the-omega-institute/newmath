\begin{theorem}[Cauchy condensation test NameCert obligations]
\label{thm:cauchy-condensation-test-namecert-obligations}
For every accepted $\CauchyCondensationTestUp$ carrier $K=(A,B,I,D,M,R,E,H,C,P,N)$, the local $\NameCert_{\CauchyCondensationTestUp}$ surface consists exactly of the following obligations:
\begin{enumerate}

\item nonnegative monotone source exposure for $A$ through finite $\SeriesUp$ summand and partial-sum rows;
\item dyadic block index readback for $B$ through the $\DyadicRatCoreUp$ scale ledger;
\item condensed block term formation for $I$ on the same displayed dyadic indices;
\item block domination compatibility for $D$, stating that monotonicity turns each inspected original tail block into the corresponding finite comparison against $I$;
\item comparison-budget routing for $M$ into the $\RegularCauchyComparisonTestUp$ and $\FastConvergentSeriesUp$ surfaces;
\item $\RegSeqRatUp$ handoff and $\RealUp$ seal exposure for $R$ and $E$;
\item transport stability, continuation replay, provenance, and local naming for exactly the route recorded by $H,C,P,N$.
\end{enumerate}
These obligations are finite-row obligations over the sibling surfaces named in the chapter introduction.
\end{theorem}
\begin{proof}
Project the carrier $K=(A,B,I,D,M,R,E,H,C,P,N)$. For a displayed tail request, read the dyadic index ledger $B$. The ledger selects finitely many dyadic blocks, and the condensed row $I$ records the corresponding block terms on those same indices.

Monotonicity is used only inside the displayed domination ledger $D$. Each inspected original tail block is compared with the condensed block row by finite endpoint reads, so the comparison budget $M$ receives a block comparison rather than a hidden infinite tail argument.

Finally apply the structural rows. The $\hsame$ rows in $H$ preserve the source, block, domination, budget, handoff, and seal data componentwise, while $C,P,N$ only replay, source, and name the displayed route. No omitted coordinate remains from which an additional NameCert obligation could be read.

\begin{theorem}[Cauchy condensation test dyadic-block handoff]
\label{thm:cauchy-condensation-test-dyadic-block-handoff}
Every accepted $\CauchyCondensationTestUp$ carrier sends a monotone nonnegative tail to the comparison-test route only through the finite dyadic-block handoff
$$
A,B,I\longmapsto D,M\longmapsto R,E,H,C,P,N.
$$
Thus a downstream consumer can read the condensed comparison only after it has read the displayed dyadic block indices, the condensed block term rows, the finite block domination ledger, and the comparison-test budget.
\end{theorem}
\begin{proof}
Begin with an accepted carrier and fix a displayed precision or tail request. The dyadic ledger $B$ supplies the finite block endpoints, and the condensed row $I$ supplies the block term associated with each endpoint.

Read the monotone source row $A$ only on those displayed endpoints. The domination ledger $D$ binds the finite collection of original tail terms to the condensed block row, using monotonicity as a finite block comparison.

The comparison budget $M$ then packages the resulting block estimates for the sibling comparison-test surface. Since $M$ receives only the displayed rows $A,B,I,D$, the handoff cannot bypass the dyadic block ledger.

Transport and replay preserve the same route. The $\hsame$ rows in $H$ carry the dyadic indices, condensed block terms, domination rows, and comparison budget together, while $C,P,N$ attach the displayed route to the regular handoff and local certificate. This gives the announced finite dyadic-block handoff.
\end{proof}

\label{thm:cauchy-condensation-test-real-seal-boundary}

\end{theorem}

Apply the dyadic-block handoff theorem. It shows that any comparison route reaching $R$ has already passed through the finite rows $A,B,I,D,M$ and through the displayed transport and replay rows attached to the same carrier.

\end{proof}

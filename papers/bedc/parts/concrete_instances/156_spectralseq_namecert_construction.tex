\chapter{A Concrete Naming Certificate for $\SpectralSeqUp$}
\label{ch:concrete-instances-spectralseq-namecert}

The spectral-sequence interface $\SpectralSeqUp$ packages a sequence of bigraded objects with differentials whose successive cohomologies converge to a target. It depends on $\NameCert_{\HomologyUp}$ (\autoref{ch:concrete-instances-homology-namecert}) and $\NameCert_{\AbelianCatUp}$ (\autoref{ch:concrete-instances-abeliancat-namecert}).

\section{Carrier definition}
\label{sec:spectralseq-carrier}

\begin{definition}[SpectralSeqUp BHist page carrier]
\label{def:spectralseq-bhist-page-carrier}
A $\SpectralSeqUp$ $\mathsf{BHist}$ page carrier is a finite page packet
$$
\mathcal S=(A,E,d,h,c,\lambda,\rho)
$$
over the source rows $\NameCert_{\AbelianCatUp}$ and
$\NameCert_{\HomologyUp}$. The row $A$ is the displayed abelian-category
surface in which the page objects and differential arrows live. The finite
family $E=(E_r^{p,q})$ is the page-object ledger; each $E_r^{p,q}$ is an
$A$-classified object row. The finite family $d=(d_r^{p,q})$ is the
differential ledger, with
$$
d_r^{p,q}:E_r^{p,q}\to E_r^{p+r,q-r+1}
$$
recorded as an $A$-classified morphism row. The row $h$ records the homology
readback identifying each successor page with the $\HomologyUp$ cycle row of
$d_r$ modulo its displayed boundary row. The row $c$ records the finite
convergence and filtration rows selected for the target object. The row
$\lambda$ is the $\Cont$ ledger ordering the finite page-to-page transitions,
and $\rho$ is the $\Pkg$ provenance row for the displayed source certificates,
page rows, differential rows, homology readbacks, convergence rows, and
continuation entries.
\end{definition}

\begin{definition}[SpectralSeqUp page classifier]
\label{def:spectralseq-page-classifier}
For two $\SpectralSeqUp$ page carriers
$\mathcal S=(A,E,d,h,c,\lambda,\rho)$ and
$\mathcal S'=(A',E',d',h',c',\lambda',\rho')$, the page classifier holds when
the following componentwise transport rows are displayed.
\begin{enumerate}
  \item $A\hsame A'$ as $\NameCert_{\AbelianCatUp}$ source surfaces.
  \item $E_r^{p,q}\hsame E_r'^{p,q}$ for every carried page object.
  \item $d_r^{p,q}\hsame d_r'^{p,q}$ for every differential row, with source
  and target read through the transported page rows.
  \item $h_r^{p,q}\hsame h_r'^{p,q}$ for every $\HomologyUp$ cycle-boundary
  readback row relating a differential to the successor page.
  \item $c\hsame c'$ for the displayed filtration and convergence rows.
  \item $\Cont(\lambda,\lambda')$ for the finite page-transition ledger.
  \item $\rho\hsame\rho'$ as $\Pkg$ provenance rows for exactly the listed
  page, differential, homology, convergence, and continuation entries.
\end{enumerate}
Thus the public classifier compares the finite spectral-sequence packet by
transporting the displayed rows themselves, not by importing a host exact
couple or an ambient equality of abelian categories.
\end{definition}

\section{The certificate}
\label{sec:spectralseq-certificate}

\begin{theorem}[SpectralSeqUp obligation surface]
\label{thm:spectralseq-obligation-surface}
Every $\SpectralSeqUp$ page carrier exposes exactly the source abelian-category
surface, the source homology surface, the finite page-object ledger, the page
classifier, the differential-square rows, the homology-page readback rows, the
finite convergence rows, the $\Cont$ transition ledger, and the $\Pkg$
provenance row. These rows form the public $\NameCert_{\SpectralSeqUp}$
obligation surface available to downstream $\HomologyUp$,
$\AbelianCatUp$, and derived-category consumers.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:spectralseq-bhist-page-carrier}. The
projections give the abelian-category source row, the homology source row, the
finite page ledger, the differential ledger, the homology readback row, the
convergence row, the continuation ledger, and the provenance row.

The classifier of \autoref{def:spectralseq-page-classifier} is then read
component by component. It transports page objects, differential rows,
homology readbacks, convergence rows, continuation entries, and provenance
without adding any unlisted carrier.

Repack these projections as the named obligation surface. Since every exposed
entry is a projection of the displayed packet or the classifier attached to
that packet, the public certificate is exhausted by the listed rows.
\end{proof}

\begin{theorem}[SpectralSeqUp differential ledger stability]
\label{thm:spectralseq-differential-ledger}
If two $\SpectralSeqUp$ page carriers are related by the page classifier, then
each transported differential row $d_r^{p,q}$ remains an admissible
$\AbelianCatUp$ morphism row with the transported source and target pages, and
the corresponding successor-page homology readback remains classified by the
$\HomologyUp$ source surface. Finite $\Cont$ extension of the page-transition
ledger preserves the same property for the extended carried ledger.
\end{theorem}
\begin{proof}
Start with one differential row
$d_r^{p,q}:E_r^{p,q}\to E_r^{p+r,q-r+1}$ from the carrier. The page
classifier supplies $\hsame$ transport for the source page, target page, and
the differential row itself, so the transported row is again typed by the
transported $\AbelianCatUp$ source surface.

Project the homology readback row attached to this differential. The
classifier also transports the corresponding $\HomologyUp$ cycle-boundary
entry, so the successor page continues to be read as the displayed homology
page of the transported differential.

For a finite $\Cont$ extension, append the displayed transition row to
$\lambda$ and extend the provenance row by the same entry. The page packet is
then repacked with all previous differential and homology rows unchanged and
with the new transition recorded in the finite ledger.
\end{proof}

\begin{theorem}[SpectralSeqUp convergence boundary]
\label{thm:spectralseq-convergence-boundary}
The $\SpectralSeqUp$ certificate exports only the displayed page, differential,
homology-readback, filtration, convergence, $\Cont$, and $\Pkg$ rows of
\autoref{def:spectralseq-bhist-page-carrier}. It does not export a host exact
couple, a quotient object outside the displayed $\HomologyUp$ boundary rows,
or an ambient equality of abelian categories beyond the transported
$\NameCert_{\AbelianCatUp}$ source surface.
\end{theorem}
\begin{proof}
Unpack the carrier and inspect the convergence component $c$. Its public
content is the finite filtration and convergence ledger selected in the
$\mathsf{BHist}$ packet, together with the page and homology rows needed to
name its source.

The quotient information used by a successor page appears only through the
$\HomologyUp$ cycle-boundary readback row $h$. The theorem therefore reads no
separate quotient object: it uses the displayed boundary classifier already
carried by the homology source surface.

Finally, the abelian-category comparison is exactly the source-surface
transport in the page classifier. Since the carrier has no row naming a host
exact couple or an ambient category equality, those data cannot be exported by
the public $\NameCert_{\SpectralSeqUp}$ boundary.
\end{proof}

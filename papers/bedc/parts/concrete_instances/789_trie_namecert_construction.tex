\chapter{A Concrete Naming Certificate for $\TrieUp$}
\label{ch:concrete-instances-trie-namecert}

The $\TrieUp$ interface names finite prefix-indexed data without importing host
strings, host maps, or a host trie primitive. The reference point is Okasaki,
\emph{Purely Functional Data Structures} (1998), where tries are finite
prefix trees for keyed lookup; the BEDC surface below reads that structure as
finite $\BoolUp$, $\ListUp$, $\OptionUp$, $\NatUp$, and $\BitVectorUp$ rows.

\begin{definition}[Trie BHist source]\label{def:trie-bhist-source}
A $\TrieUp$ source packet is a finite $\mathsf{BHist}$ record
$$
T=(k,v,l,b,p)
$$
with the following visible coordinates:
\begin{enumerate}
\item $k$ is a finite key-path row, read either as a $\BitVectorUp$ path or as
a $\ListUp$ spine of branch histories;
\item $v$ is an $\OptionUp$ terminal payload row, recording whether the
current prefix carries a value and, when present, exposing only the displayed
value row;
\item $l$ is a $\NatUp$ length and depth ledger recording the finite prefix
length, the depth of each inspected node, and the fact that every public read
is below that depth;
\item $b$ is the branch ledger whose branch tags are $\BoolUp$ rows and whose
left and right subprefix reads are routed only through displayed finite list
or bit-vector positions;
\item $p$ is the $\Pkg$ provenance naming the consumed $\BoolUp$, $\ListUp$,
$\OptionUp$, $\NatUp$, $\BitVectorUp$, $\hsame$, and $\Cont$ certificate rows.
\end{enumerate}
The carrier predicate is $\mathsf{TrieSource}(k,v,l,b,p)$. Its classifier
compares accepted packets by componentwise key-path comparison, optional
payload comparison, branch-tag comparison, length-ledger comparison, and the
same displayed $\hsame$, $\Cont$, and $\Pkg$ routes. The packet contains no
host string equality, host finite map, function extensionality, hidden
ordering on keys, or unbounded tree object.
\end{definition}

\begin{theorem}[Trie carrier stability]\label{thm:trie-carrier-stability}
Let $T=(k,v,l,b,p)$ be an accepted $\TrieUp$ source packet. If the key path,
optional payload, depth ledger, branch ledger, and provenance row are
transported by displayed $\hsame$ along the $\Cont$ routes named by $p$, then
the transported packet $T'=(k',v',l',b',p')$ is again an accepted $\TrieUp$
source packet.
\end{theorem}
\begin{proof}
Project the five coordinates of \autoref{def:trie-bhist-source}. The
$\NatUp$ ledger transports first, so every later prefix and branch read is
bounded by the transported finite depth row.

Transport the key row through its displayed $\BitVectorUp$ or $\ListUp$
presentation. Each branch tag is a $\BoolUp$ row, so the branch ledger
transports componentwise along the same key-path positions.

Finally transport the optional payload and provenance rows. The $\OptionUp$
row preserves the absent case and transports the present payload only when the
present tag is displayed, while the $\Pkg$ row keeps the same finite
certificate dependencies. Thus the transported record is another accepted
trie source packet.
\end{proof}

\begin{theorem}[Trie classifier laws]\label{thm:trie-classifier-laws}
The $\TrieUp$ classifier is reflexive, symmetric, and transitive on accepted
packets by componentwise use of the $\BitVectorUp$ or $\ListUp$ key-path
classifier, the $\OptionUp$ payload classifier, the $\BoolUp$ branch
classifier, and the $\NatUp$ ledger classifier.
\end{theorem}
\begin{proof}
For reflexivity, take the reflexive classifier row for the key path, the
optional payload, each boolean branch tag, and the finite depth ledger. The
provenance and continuation rows are read unchanged.

For symmetry, reverse each displayed component comparison. The absent and
present cases of the optional payload reverse by the $\OptionUp$ classifier,
and every branch tag reverses by the $\BoolUp$ classifier.

For transitivity, compose the two key-path comparisons, the two optional
payload comparisons, the two branch-ledger comparisons, and the two depth
ledger comparisons. The depth ledger fixes the finite set of visible prefix
positions, so no hidden branch can enter the composite classifier.
\end{proof}

\begin{theorem}[Trie ledger coverage]\label{thm:trie-ledger-coverage}
Every public read of an accepted $\TrieUp$ packet factors through the finite
ledger $(k,v,l,b,p)$. Consumers see exactly finite prefix rows, boolean branch
rows, optional terminal payload rows, depth and length rows, $\hsame$
transports, $\Cont$ routes, and $\Pkg$ provenance.

Consequently the packet exports no host string type, host map lookup, ambient
function space, infinite tree, or implicit key ordering.
\end{theorem}
\begin{proof}
Start from the carrier definition. Its tuple lists exactly the key, terminal
payload, depth ledger, branch ledger, and provenance coordinates.

A prefix-facing consumer reads $k$ through the displayed $\BitVectorUp$ or
$\ListUp$ path. A branch-facing consumer reads $b$ only through the boolean
tags admitted by the finite depth ledger $l$. A value-facing consumer reads
$v$ only through the $\OptionUp$ absent-or-present row.

The remaining public reads are the displayed $\hsame$ comparisons, $\Cont$
routes, and $\Pkg$ provenance named in $p$. Since no further coordinate is
present, every consumer projection is one of these finite rows.
\end{proof}

\begin{theorem}[Trie key-path exhaustion]\label{thm:trie-key-path-exhaustion}
Every public key-path read of an accepted $\TrieUp$ packet is exhausted by
the displayed $\BitVectorUp$ or $\ListUp$ key row, the $\NatUp$ depth ledger,
the $\hsame$ transports and $\Cont$ routes named by the packet, and the
$\Pkg$ provenance row. No host string, host ordering, ambient finite map, or
hidden key predicate contributes an additional public key-path coordinate.
\end{theorem}
\begin{proof}
Project the key coordinate $k$ and depth ledger $l$ from
\autoref{def:trie-bhist-source}. The definition gives $k$ only as a displayed
$\BitVectorUp$ path or as a $\ListUp$ spine of branch histories, and it gives
$l$ as the bound for every public prefix read.

Use \autoref{thm:trie-carrier-stability} to see that transport preserves this
same finite key presentation along the displayed $\hsame$ and $\Cont$ routes.
Then apply \autoref{thm:trie-classifier-laws} componentwise to the accepted
key rows.

Finally apply \autoref{thm:trie-ledger-coverage}. Since every public read is
a projection from $(k,v,l,b,p)$, the only remaining key-path data is the
provenance named in $p$, and no host key structure is exported.
\end{proof}

\begin{theorem}[Trie prefix-branch determinacy]\label{thm:trie-prefix-branch-determinacy}
At each displayed finite prefix below the $\NatUp$ depth row of an accepted
$\TrieUp$ packet, the $\BoolUp$ branch tag selects exactly the recorded left
or right subprefix. No unrecorded branch at that prefix is public.
\end{theorem}
\begin{proof}
Project the branch ledger $b$ and the depth ledger $l$ from
\autoref{def:trie-bhist-source}. The ledger $l$ bounds the finite prefix
positions that a consumer may inspect, and $b$ records branch tags only at
those displayed positions.

For any such prefix, read the branch tag through its $\BoolUp$ row. The two
displayed boolean cases select the recorded left or right subprefix row, and
\autoref{thm:trie-classifier-laws} transports equality of branch reads
componentwise through the same finite coordinates.

The stability theorem \autoref{thm:trie-carrier-stability} preserves the
branch ledger under $\hsame$ and $\Cont$. Applying
\autoref{thm:trie-ledger-coverage} then excludes any branch coordinate not
recorded in $(k,v,l,b,p)$, so no extra branch is public.
\end{proof}

\begin{theorem}[Trie lookup payload projection]\label{thm:trie-lookup-payload-projection}
Every lookup payload read of an accepted $\TrieUp$ packet factors through the
$\OptionUp$ absent-or-present terminal row at the displayed key path. The
read never factors through a host finite map, host lookup function, or ambient
function space.
\end{theorem}
\begin{proof}
Project the terminal payload coordinate $v$ from
\autoref{def:trie-bhist-source}. The coordinate is an $\OptionUp$ row at the
displayed key path, so a consumer can observe only the absent tag or the
present tag together with the displayed payload row.

The key path used by the lookup is fixed by
\autoref{thm:trie-key-path-exhaustion}. The payload row transports along the
same displayed $\hsame$ and $\Cont$ routes by
\autoref{thm:trie-carrier-stability}, and the accepted-row comparison is
componentwise by \autoref{thm:trie-classifier-laws}.

Apply \autoref{thm:trie-ledger-coverage} to close the projection. Since the
finite tuple exports no host map or function coordinate, lookup payload reads
can only pass through the displayed $\OptionUp$ terminal row.
\end{proof}

\begin{theorem}[Trie NameCert obligation surface]\label{thm:trie-namecert-obligation-surface}
The local $\NameCert_{\TrieUp}$ obligation surface consists of five finite
rows:
\begin{enumerate}
\item carrier habitation: the empty trie is carried by the empty key path, an
absent $\OptionUp$ payload, the zero $\NatUp$ depth ledger, the empty branch
ledger, and the corresponding $\Pkg$ provenance;
\item classifier laws: reflexivity, symmetry, and transitivity are inherited
componentwise from \autoref{thm:trie-classifier-laws};
\item prefix-branch determinacy: at each displayed finite prefix, a $\BoolUp$
branch tag selects exactly the recorded left or right subprefix row, and no
unrecorded branch is public, as witnessed by
\autoref{thm:trie-key-path-exhaustion} and
\autoref{thm:trie-prefix-branch-determinacy};
\item transport under $\hsame$ and $\Cont$: the stability theorem
\autoref{thm:trie-carrier-stability} repacks transported key, payload, branch,
ledger, and provenance rows as another accepted packet, with lookup payload
projection fixed by \autoref{thm:trie-lookup-payload-projection};
\item ledger exactness: downstream consumers may read only finite prefix
rows, branch rows, optional terminal payload rows, length and depth rows, and
the displayed provenance rows of \autoref{thm:trie-ledger-coverage}, with the
key-path, branch, and payload projections separated by
\autoref{thm:trie-key-path-exhaustion},
\autoref{thm:trie-prefix-branch-determinacy}, and
\autoref{thm:trie-lookup-payload-projection}.
\end{enumerate}
These rows are the complete seed obligation surface for prefix-tree
NameCert data over finite BEDC rows.
\end{theorem}
\begin{proof}
For carrier habitation, choose the empty $\BitVectorUp$ or empty $\ListUp$
path, the absent $\OptionUp$ payload, the zero $\NatUp$ depth row, and the
empty branch ledger. The provenance row records only the finite data
certificates consumed by the packet.

For the classifier laws, apply \autoref{thm:trie-classifier-laws}. Because the
classifier is componentwise, the three equivalence laws are inherited from the
key, option, boolean branch, and natural ledger classifiers.

For prefix-branch determinacy, apply
\autoref{thm:trie-key-path-exhaustion} and
\autoref{thm:trie-prefix-branch-determinacy}. The key-path theorem fixes the
finite prefix positions, and the branch theorem fixes the recorded left or
right subprefix at each displayed position.

For transport, apply \autoref{thm:trie-carrier-stability}. The same $\hsame$
and $\Cont$ routes move the visible rows without adding host string equality,
host maps, or unbounded trees. Lookup payload reads are then projected through
\autoref{thm:trie-lookup-payload-projection}.

For ledger exactness, apply \autoref{thm:trie-ledger-coverage}. Every
downstream read is a projection from the finite tuple $(k,v,l,b,p)$, so the
NameCert obligation surface is exhausted by the listed rows.
\end{proof}

\begin{definition}[Displayed Trie prefix subledger]\label{def:trie-displayed-prefix-subledger}
Let $T=(k,v,l,b,p)$ be an accepted $\TrieUp$ source packet. A displayed prefix subledger of $T$ is a finite packet
$$
U=(k_U,v_U,l_U,b_U,p_U;\iota)
$$
with a displayed inclusion row $\iota$ into the visible prefix positions of $T$ such that:
\begin{enumerate}
\item $k_U$ is a finite $\BitVectorUp$ path or $\ListUp$ spine whose entries are exactly the key-path entries of $k$ at the positions selected by $\iota$;
\item the selected positions are prefix-closed: whenever a displayed position is selected, every displayed predecessor prefix of that position in the key-path ledger is also selected;
\item $v_U$ is the $\OptionUp$ payload row of $T$ restricted to selected terminal positions, so the absent tag stays absent and a present tag exposes only the same displayed payload row already exposed by $v$;
\item $l_U$ is the $\NatUp$ depth ledger obtained by restricting $l$ to the selected positions, with each selected read still below the corresponding depth bound;
\item $b_U$ is the branch ledger obtained by retaining exactly the $\BoolUp$ branch tags and left or right subprefix reads of $b$ whose source and target prefixes are selected by $\iota$;
\item $p_U$ is the projected $\Pkg$ provenance row naming only the consumed $\BoolUp$, $\ListUp$, $\OptionUp$, $\NatUp$, $\BitVectorUp$, $\hsame$, and $\Cont$ rows that occur in the retained coordinates.
\end{enumerate}
The associated restricted packet is $T|_U=(k_U,v_U,l_U,b_U,p_U)$.
\end{definition}

\begin{theorem}[Trie prefix subledger restriction carrier]\label{thm:trie-prefix-subledger-restriction-carrier}
Let $T=(k,v,l,b,p)$ be an accepted $\TrieUp$ source packet, and let $U$ be a displayed prefix subledger of $T$ in the sense of \autoref{def:trie-displayed-prefix-subledger}. Then the restricted packet
$$
T|_U=(k_U,v_U,l_U,b_U,p_U)
$$
is again an accepted $\TrieUp$ source packet under the same $\BoolUp$, $\ListUp$, $\OptionUp$, $\NatUp$, and $\BitVectorUp$ dependency certificates.
\end{theorem}
\begin{proof}
Unpack the accepted source packet by \autoref{def:trie-bhist-source}. The original carrier consists only of the key-path row, optional terminal payload row, finite depth ledger, boolean branch ledger, and package provenance row. The displayed subledger definition selects rows from exactly those five coordinates and records the inclusion row $\iota$.

First consider the key coordinate. Since $k_U$ is obtained by selecting displayed positions of the finite $\BitVectorUp$ path or $\ListUp$ branch-history spine of $k$, it is again a finite key-path row of the same kind. Prefix-closedness of the selected positions ensures that every selected non-root prefix has its predecessor prefix in the same restricted packet, so no branch read in $k_U$ points outside the restricted carrier.

Next consider the $\NatUp$ ledger. The row $l_U$ is the restriction of $l$ to selected positions. Every public read admitted by $l_U$ was already a public read admitted by $l$, and the depth inequality for that read is the same displayed $\NatUp$ row. Thus the restricted depth ledger is finite and still bounds all selected key and branch reads.

For the branch coordinate, each retained entry of $b_U$ is a $\BoolUp$ tag already present in $b$. The definition of displayed prefix subledger keeps a branch entry only when its source prefix and the selected left or right subprefix row are both selected by $\iota$. Therefore the branch ledger of the restricted packet has no public unrecorded branch, and every visible branch selection remains routed through displayed finite list or bit-vector positions.

For the optional payload coordinate, the absent case is inherited unchanged from $v$. In the present case, $v_U$ exposes only a payload row already exposed by the original $\OptionUp$ row $v$ at a selected terminal prefix. Thus no payload comparison, hidden value equality, or terminal read is introduced by the restriction.

Finally, $p_U$ is the projection of the original provenance row $p$ to the dependency rows actually used by $k_U$, $v_U$, $l_U$, and $b_U$. It names the same kinds of certificates as $p$: $\BoolUp$, $\ListUp$, $\OptionUp$, $\NatUp$, $\BitVectorUp$, $\hsame$, and $\Cont$. Since \autoref{thm:trie-ledger-coverage} says every public read of an accepted trie packet factors through the finite ledger coordinates, deleting unselected rows cannot create a host string equality, a host finite map lookup, an ambient function space, an infinite tree, or an implicit ordering on keys.

The five projected coordinates therefore satisfy the same carrier clauses as \autoref{def:trie-bhist-source}. Hence $T|_U=(k_U,v_U,l_U,b_U,p_U)$ is an accepted $\TrieUp$ source packet under the same finite dependency certificates.
\end{proof}

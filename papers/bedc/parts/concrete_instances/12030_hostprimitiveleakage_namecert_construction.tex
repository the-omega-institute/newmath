\chapter{A Concrete Naming Certificate for $\HostPrimitiveLeakageUp$}
\label{ch:concrete-instances-hostprimitiveleakage-namecert}
\origin{ai}

The $\HostPrimitiveLeakageUp$ packet realizes the host-smuggling branch of \autoref{ch:visions-philosophy-auditable-generativity} as a finite diagnostic refusal carrier. It sits beside \autoref{ch:concrete-instances-auditgatefailurewitness-namecert}, \autoref{ch:concrete-instances-quotientsoundnessboundary-namecert}, \autoref{ch:concrete-instances-negativecertificateboundary-namecert}, \autoref{ch:concrete-instances-auditgateboundary-namecert}, and \autoref{ch:concrete-instances-kernelauditwitness-namecert}. Those siblings expose failed audit rows, quotient-boundary refusals, negative certificates, audit-gate verdicts, and kernel acceptance witnesses; this packet isolates the public-surface import of host primitives as the refused item.

\begin{definition}[HostPrimitiveLeakage carrier]
\label{def:host-primitive-leakage-carrier}
A $\HostPrimitiveLeakageUp$ carrier is a finite $\BHist$ packet
$$
L=(S,Q,E,D,F,G,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the public-surface site where a consumer requests host equality, host numerals, host lists, host function spaces, quotient soundness, choice, or propositional extensionality;
\item $Q$ is the requested primitive row, naming the host feature that would otherwise enter as substrate content;
\item $E$ is the exposed BEDC replacement row, containing the finite $\BHist$, $\msame$, $\hsame$, $\psame$, $\Cont$, $\Pkg$, or local $\NameCert$ route offered instead;
\item $D$ is the diagnostic row explaining which public route attempted the import;
\item $F$ is the failed-gate row consumed from $\AuditGateFailureWitnessUp$;
\item $G$ is the audit-boundary row consumed from $\AuditGateBoundaryUp$;
\item $H$ records componentwise $\hsame$ transport for the site, request, replacement, diagnostic, failed-gate, audit-boundary, replay, provenance, and local naming rows;
\item $C$ records displayed $\Cont$ replay for the refusal route and for the replacement route when one is available;
\item $P$ is the $\Pkg$ provenance over $\HostPrimitiveLeakageUp$, $\AuditGateFailureWitnessUp$, $\QuotientSoundnessBoundaryUp$, $\NegativeCertificateBoundaryUp$, $\AuditGateBoundaryUp$, $\KernelAuditWitnessUp$, $\BHist$, $\msame$, $\hsame$, $\psame$, $\Cont$, $\Pkg$, and $\NameCert$ rows;
\item $N$ is the local $\NameCert_{\HostPrimitiveLeakageUp}$ row.
\end{enumerate}
The carrier has no coordinate for a host equality token, host-level numeric object, host list eliminator, host function-space extensionality principle, quotient soundness dependency, choice principle, or propositional-extensionality transport.
\end{definition}

\begin{theorem}[HostPrimitiveLeakage NameCert obligations]
\label{thm:host-primitive-leakage-namecert-obligations}
For the carrier of \autoref{def:host-primitive-leakage-carrier}, the local $\NameCert_{\HostPrimitiveLeakageUp}$ surface consists exactly of carrier admission for
$$
L=(S,Q,E,D,F,G,H,C,P,N),
$$
public-site exposure through $S$, requested-host-primitive exposure through $Q$, BEDC replacement exposure through $E$, diagnostic readback through $D$, failed-gate witness exposure through $F$, audit-boundary verdict exposure through $G$, componentwise $\hsame$ transport through $H$, displayed $\Cont$ replay through $C$, package provenance through $P$, and local naming through $N$.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:host-primitive-leakage-carrier}. The rows $S,Q,E,D,F,G,H,C,P,N$ account for the public site, requested host primitive, BEDC replacement, diagnostic readback, failed-gate witness, audit boundary, transport, replay, provenance, and local naming surface.

The failed-gate and audit rows are consumed from the sibling certificates \autoref{ch:concrete-instances-auditgatefailurewitness-namecert} and \autoref{ch:concrete-instances-auditgateboundary-namecert}. The quotient-specific branch is read through \autoref{ch:concrete-instances-quotientsoundnessboundary-namecert}, and the named refusal branch is read through \autoref{ch:concrete-instances-negativecertificateboundary-namecert}.

The kernel witness sibling \autoref{ch:concrete-instances-kernelauditwitness-namecert} supplies the accepted-declaration readback surface, not a permission to import host primitives. Repacking these rows gives the local naming certificate, and every public consumer route is forced to display the requested host feature before the route can be refused or replaced.
\end{proof}

\begin{theorem}[HostPrimitiveLeakage non-escape]
\label{thm:host-primitive-leakage-nonescape}
Every accepted $\HostPrimitiveLeakageUp$ packet $L=(S,Q,E,D,F,G,H,C,P,N)$ admits public reads only through the displayed diagnostic route $S,Q,D,F,G,H,C,P,N$ and, when present, the replacement route $E,H,C,P,N$. A consumer that reads host equality, host numerals, host lists, host function spaces, quotient soundness, choice, or propositional extensionality as BEDC-internal content has left the accepted packet.
\end{theorem}
\begin{proof}
Inspect the carrier. The requested host feature is present only as the request row $Q$, and the diagnostic row $D$ records the public route that tried to expose it.

Apply \autoref{thm:host-primitive-leakage-namecert-obligations}. The failed-gate row $F$ and audit-boundary row $G$ keep the request on the refusal side of the packet, while $H,C,P,N$ preserve only the displayed diagnostic and replacement routes.

Finally use the excluded coordinates in \autoref{def:host-primitive-leakage-carrier}. Since no host primitive is a packet coordinate, every accepted consumer route either reads it as a refused request through $Q,D,F,G$ or uses the visible BEDC replacement row $E$.
\end{proof}

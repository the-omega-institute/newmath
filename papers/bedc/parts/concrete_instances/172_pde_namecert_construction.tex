\chapter{A Concrete Naming Certificate for $\PdeUp$}
\label{ch:concrete-instances-pde-namecert}
\origin{human}

The partial-differential-equation interface $\PdeUp$ packages a relation among partial derivatives of a function on a manifold carrier together with classification by order, type, and boundary data. It depends on $\NameCert_{\ManifoldUp}$ (\autoref{ch:concrete-instances-manifold-namecert}) and $\NameCert_{\DerivativeUp}$ (\autoref{ch:concrete-instances-derivative-namecert}).

\section{Carrier definition}
\label{sec:pde-carrier}

\begin{definition}[PdeUp carried source row]\label{def:pde-carried-source-row}
A carried source row for $\PdeUp$ consists of the following BEDC-facing data: a $\ManifoldUp$ carrier row naming the underlying charted carrier; the $\DerivativeUp$ partial-derivative observation rows whose endpoints are the named derivative observations used by the equation; a finite PDE relation row whose entries refer only to those derivative endpoints together with their order and type tags; a boundary-data classifier row over the same carrier and endpoint names; and a ledger recording hidden chart choices, coordinate choices, representative functions, and solver-side presentation data.

The carried source row does not identify host functions, host real values, analytic solution spaces, or coordinate representatives. Those objects remain ledger entries unless they are re-expressed through the displayed $\ManifoldUp$, $\DerivativeUp$, relation, and boundary rows.
\end{definition}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_source\_scope}

\begin{theorem}[PdeUp carrier obligation]
\label{thm:pde-carrier-obligation}
A carried $\PdeUp$ endpoint is a $\mathsf{BHist}$ row whose visible source
data are exactly the $\NameCert_{\ManifoldUp}$ source, the
$\NameCert_{\DerivativeUp}$ source, the PDE relation row, the boundary-data
classifier row, $\Pkg$ provenance, and the probe bundles used to read the
relation. Chart coordinates, derivative witnesses, and boundary presentations
are ledger data for the certificate rather than additional carrier-equality
coordinates.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_carrier\_obligation\_surface}
\begin{proof}
Read the endpoint through the ten-field certificate template. The
traditional target is a partial-differential-equation endpoint on a certified
manifold, while the BEDC source is the displayed $\mathsf{BHist}$ package
assembled from the manifold certificate, derivative certificate, relation
row, boundary classifier row, provenance, and probe bundles.

The carrier field therefore projects only those visible histories. The
coordinate charts and derivative witnesses enter through the stability and
ledger fields: they justify that the displayed relation and boundary rows can
be transported across the selected presentations, but they are not compared
as carrier coordinates.

Exactness is readback from the same package. Any accepted carrier row must be
one of the displayed $\mathsf{BHist}$ rows, and any hidden chart, derivative
order, or boundary presentation is recoverable only as ledger evidence.
\end{proof}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_carrier\_obligation}

\begin{theorem}[PdeUp classifier obligation]
\label{thm:pde-classifier-obligation}
The $\PdeUp$ classifier compares two carried endpoints exactly when the
manifold source row, derivative-tuple row, PDE relation row, and
boundary-data classifier row agree under the same $\hsame$ discipline and the
same $\Pkg$ provenance. It does not compare endpoints by host function
equality.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeRelationClassifier\_visible\_append\_surface}
\begin{proof}
Project the classifier slot from the ten-field template. Its source rows are
the same source, carrier, classifier, stability, ledger, theorem-list,
exactness, bridge, and boundary fields used by the carrier obligation.

The source field fixes the manifold row and the derivative tuple row. The
carrier and classifier fields then compare the PDE relation and boundary
classifier rows by $\hsame$ and by the provenance equality recorded in
$\Pkg$. Since each comparison is a comparison of displayed $\mathsf{BHist}$
rows, no host-level function equality is available as an extra rule.

Conversely, if these four rows and the provenance agree under the displayed
$\hsame$ witnesses, the classifier field has exactly the data it requires.
Repacking those witnesses gives the carried-endpoint comparison.
\end{proof}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_classifier\_obligation}

\begin{theorem}[PdeUp stability ledger exactness obligation]
\label{thm:pde-stability-ledger-exactness-obligation}
The $\PdeUp$ certificate is admitted only with transport stability for the
derivative relation and boundary data, ledger entries for hidden
chart/order/boundary choices, exactness rows for coverage, soundness, and
completeness, and dependency rows citing the $\ManifoldUp$ and
$\DerivativeUp$ sources.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_stability\_ledger\_exactness\_obligation\_surface}
\leanchecked{BEDC.Derived.PdeUp.PdeStabilityLedger\_relation\_boundary\_append\_surface}
\begin{proof}
Separate the remaining certificate fields after the carrier and classifier
rows have been fixed. Stability requires transport of the derivative-relation
row and the boundary-data row along the displayed $\hsame$ comparisons of the
manifold and derivative sources.

The ledger field records every chart, derivative-order, and boundary
presentation choice used to witness that transport. These entries are
evidence rows over $\mathsf{BHist}$; they do not enlarge the carrier and do
not create a second classifier.

The exactness field splits into coverage, soundness, and completeness rows:
every admitted endpoint has displayed source and probe data, every accepted
relation or boundary row is supported by the ledger, and every required
source dependency cites $\NameCert_{\ManifoldUp}$ and
$\NameCert_{\DerivativeUp}$. These rows are precisely the non-bridge
obligation surface claimed here.
\end{proof}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_stability\_ledger\_exactness\_obligation}

\section{The certificate}
\label{sec:pde-certificate}

\begin{theorem}[PdeUp carrier source obligation]\label{thm:pde-carrier-source-obligation}
For any carried source row of $\PdeUp$, the carrier obligation is discharged exactly by the named $\ManifoldUp$ row, the named $\DerivativeUp$ partial-derivative observation rows, the finite PDE relation row on those derivative endpoints, and the boundary-data classifier row. No host function carrier, host coordinate equality, analytic existence predicate, or solver object is part of the carrier source.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_visible\_source\_readback}
\begin{proof}
Unpack \autoref{def:pde-carried-source-row}. Its source entries are the $\ManifoldUp$ carrier row, the $\DerivativeUp$ observation rows, the relation row, and the boundary row.

The remaining listed host objects occur only in the ledger component of the same definition. Hence they can be recorded for provenance, but they cannot act as carrier-source dependencies for the $\PdeUp$ certificate.
\end{proof}

\begin{theorem}[PdeUp relation classifier obligation]\label{thm:pde-relation-classifier-obligation}
Let two $\PdeUp$ carried source rows have $\ManifoldUp$ carrier rows related by the supplied $\ManifoldUp$ classifier, and let each partial-derivative endpoint appearing in their PDE relation rows be related by the supplied $\DerivativeUp$ classifier. Then the PDE relation rows are transported by replacing each relation entry with its transported derivative endpoint, order tag, and type tag; no further host equation equality is required.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeRelationClassifier\_endpoint\_transport}
\begin{proof}
The relation row in \autoref{def:pde-carried-source-row} is finite and refers only to displayed derivative endpoints with their order and type tags. Transport the carrier name by the $\ManifoldUp$ classifier and transport every endpoint name by the corresponding $\DerivativeUp$ classifier.

After these transports, each relation entry has the same BEDC-facing shape as before: a finite relation entry over named derivative endpoints. Since host equations and coordinate representatives are ledger entries rather than relation entries, no host equation equality is introduced.
\end{proof}

\begin{theorem}[PdeUp boundary-data obligation]\label{thm:pde-boundary-data-obligation}
For any $\PdeUp$ carried source row, boundary-data rows are transported by the same $\ManifoldUp$ carrier classifier and $\DerivativeUp$ endpoint classifiers used for the PDE relation row. The boundary-data obligation adds no host boundary equality and no host trace-space equality beyond those carried classifier rows.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeBoundaryData\_relation\_transport\_surface}
\leanchecked{BEDC.Derived.PdeUp.PdeRelationClassifier\_boundary\_transport\_surface}
\begin{proof}
Unpack the boundary-data classifier row from \autoref{def:pde-carried-source-row}. Its carrier reference is the same $\ManifoldUp$ carrier row, and its derivative references are among the displayed $\DerivativeUp$ endpoint rows.

Therefore the boundary row uses the same two classifier families as the relation row: carrier transport from $\ManifoldUp$ and endpoint transport from $\DerivativeUp$. Host boundary representatives and trace-space presentations remain ledger data, so they do not become additional equality obligations.
\end{proof}

\begin{theorem}[PdeUp ledger exactness obligation]\label{thm:pde-ledger-exactness-obligation}
The exactness data of a $\PdeUp$ carried source row is precisely the pair consisting of the carried PDE relation classifier and the carried boundary-data classifier. Host functions, host real equality, analytic existence of solutions, regularity theorems, numerical solvers, and solver data are ledger-external to this exactness claim.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdeCarriedSourceRow\_ledger\_exactness\_surface}
\begin{proof}
The only classifier rows displayed in \autoref{def:pde-carried-source-row} are the finite PDE relation classifier and the boundary-data classifier. Exactness is therefore the assertion that these two rows account for all BEDC-facing relation and boundary comparisons in the certificate.

Every analytic or solver-side object named in the statement is listed in the ledger component of the carried source row. Because ledger entries record presentation choices without adding classifier rows, they remain outside the exactness pair.
\end{proof}

\begin{theorem}[PdeUp public interface export]\label{thm:pde-public-interface-export}
The public $\NameCert_{\PdeUp}$ surface is exactly the $\mathsf{BHist}$ carried source row of \autoref{def:pde-carried-source-row} together with the $\ManifoldUp$ carrier row, the $\DerivativeUp$ partial-derivative observation rows, the $\Pkg$ provenance row, the $\mathsf{ProbeBundle}$ rows used to read the finite PDE relation, and the following obligation surface:
\begin{enumerate}
  \item the carrier obligation of \autoref{thm:pde-carrier-obligation};
  \item the classifier obligation of \autoref{thm:pde-classifier-obligation};
  \item the stability-ledger-exactness obligation of \autoref{thm:pde-stability-ledger-exactness-obligation};
  \item the carrier-source obligation of \autoref{thm:pde-carrier-source-obligation};
  \item the relation classifier obligation of \autoref{thm:pde-relation-classifier-obligation};
  \item the boundary-data obligation of \autoref{thm:pde-boundary-data-obligation};
  \item the ledger exactness obligation of \autoref{thm:pde-ledger-exactness-obligation}.
\end{enumerate}
No host function equality, host coordinate equality, analytic existence theorem, solution space, solver object, host real equality, regularity theorem, or numerical solver datum is part of this public interface.
\end{theorem}
\leanchecked{BEDC.Derived.PdeUp.PdePublicInterface\_export\_semantic\_name\_certificate}
\begin{proof}
Start with the carried source row of \autoref{def:pde-carried-source-row}. Its visible $\mathsf{BHist}$ entries bind the $\ManifoldUp$ carrier row, the $\DerivativeUp$ derivative-observation rows, the finite PDE relation row, the boundary-data classifier row, $\Pkg$ provenance, and the $\mathsf{ProbeBundle}$ rows used to read the relation.

Repack those entries through the ten-field certificate template. The carrier, classifier, stability, ledger, theorem-list, exactness, bridge, and boundary slots are supplied by the seven cited obligation theorems, with the standard bridge slot absent and the dependency rows pointing back to the $\ManifoldUp$ and $\DerivativeUp$ source certificates.

Conversely, any public $\NameCert_{\PdeUp}$ endpoint must project through the same ten fields. The carrier and classifier projections recover the named source rows, the stability and ledger projections recover only transport and presentation evidence, and the exactness projection recovers the carried relation and boundary classifiers. Therefore the displayed surface is exact.

The excluded host objects occur only as ledger-side presentation data or analytic objects outside the BEDC-facing relation and boundary rows. Since no cited obligation introduces a classifier for them, they cannot enter the public $\NameCert_{\PdeUp}$ interface.
\end{proof}

\begin{theorem}[PdeUp conservative standard bridge]
\label{thm:pde-conservative-standard-bridge}
The public $\NameCert_{\PdeUp}$ surface of \autoref{thm:pde-public-interface-export} has a conservative standard interpretation whose source is exactly the displayed $\mathsf{BHist}$ carried source row, the $\NameCert_{\ManifoldUp}$ carrier row, the $\NameCert_{\DerivativeUp}$ partial-derivative observation rows, the finite PDE relation row, the boundary-data classifier row, the $\Pkg$ provenance row, and the $\mathsf{ProbeBundle}$ read rows. Conversely, any standard PDE presentation admitted through precisely those displayed manifold, derivative, relation, boundary, package, and probe rows repacks to the same public BEDC surface.

This bridge adds no analytic existence theorem, solution space, solver semantics, host function equality, host coordinate equality, host real equality, or regularity theory.
\end{theorem}
\begin{proof}
Apply \autoref{thm:pde-public-interface-export}. It exposes the carried source row and the seven obligation rows: carrier, classifier, stability-ledger-exactness, carrier-source, relation-classifier, boundary-data, and ledger exactness. The standard reading is allowed to inspect only the displayed $\ManifoldUp$ carrier row, $\DerivativeUp$ partial-derivative rows, finite PDE relation row, boundary-data classifier row, $\Pkg$ provenance, and $\mathsf{ProbeBundle}$ read rows.

The ledger exactness theorem \autoref{thm:pde-ledger-exactness-obligation} keeps chart selections, derivative-order witnesses, boundary presentations, representative functions, and solver-side presentation choices as provenance. Thus the standard interpretation is conservative: it reads the ordinary PDE presentation through the visible rows and does not turn hidden ledger choices into BEDC carrier coordinates or classifier clauses.

For the converse direction, start with a standard PDE presentation admitted through exactly those displayed rows. Repack its manifold carrier, derivative observations, relation entries, boundary classifier, package provenance, and probe reads into the carried source row of \autoref{def:pde-carried-source-row}. The seven obligations cited by \autoref{thm:pde-public-interface-export} then reconstruct the same public $\NameCert_{\PdeUp}$ surface.

The excluded analytic and host-level objects remain outside the bridge because no row in the exported surface supplies classifiers for them. Hence the bridge certifies only conservative readback and repacking for the displayed PDE certificate surface.
\end{proof}
\leanchecked{BEDC.Derived.PdeUp.PdeUp\_StdBridge}

\begin{theorem}[PdeUp finite relation subrow projection]
\label{thm:pde-finite-relation-subrow-projection}
Let $S$ be a carried source row for $\PdeUp$. Write its displayed source data as a $\ManifoldUp$ carrier row $M$, a finite row $D$ of $\DerivativeUp$ partial-derivative endpoints, a finite PDE relation row $R$, a boundary-data classifier row $B$, a $\Pkg$ provenance row $P$, and $\ProbeBundle$ rows $Q$ reading the entries of $R$. Let $R'$ be a displayed finite subrow of $R$, obtained by selecting relation entries from $R$, and suppose that each selected entry names only derivative endpoints occurring in $D$ and carries only the order tags and type tags already displayed in $R$. Let $Q'$ be the displayed restriction of $Q$ to the reads of the selected entries. Then $R'$ is again admitted as the relation coordinate of a carried $\PdeUp$ source row over the same $M$, $D$, $B$, $P$, and inherited probe-source rows $Q'$. This projection adds no host function equality, host coordinate equality, analytic solution predicate, trace-space equality, or solver object.
\end{theorem}
\begin{proof}
Unpack the carried source row in \autoref{def:pde-carried-source-row}. The relation coordinate is finite, and every one of its entries is required to refer only to displayed $\DerivativeUp$ endpoints together with displayed order and type tags. Since $R'$ is obtained by selecting entries of $R$, no selected relation entry is created outside the original finite row. The endpoint, order, and type conditions for $R'$ are therefore inherited entry by entry from the corresponding entries of $R$.

The source coordinates $M$, $D$, $B$, and $P$ are not altered by the selection. The carrier-source obligation in \autoref{thm:pde-carrier-source-obligation} says that these displayed manifold, derivative, relation, and boundary rows are exactly the carrier-source data for the certificate. Replacing the relation coordinate by the selected finite subrow keeps the same manifold carrier, the same derivative endpoint list, the same boundary-data classifier row, and the same package provenance.

The relation classifier obligation in \autoref{thm:pde-relation-classifier-obligation} permits relation rows only through their displayed derivative endpoints, order tags, and type tags. Here the transport is the identity on the retained entries: each selected entry keeps its original endpoint names and tags. Thus the selected relation row has the same BEDC-facing relation shape as the original row, with no appeal to host equation equality.

For the boundary coordinate, \autoref{thm:pde-boundary-data-obligation} uses the same $\ManifoldUp$ carrier classifier and $\DerivativeUp$ endpoint classifiers as the relation row. Because the selected relation entries use only endpoints already present in $D$, the boundary-data classifier row remains compatible with the selected relation subrow. Restricting $Q$ to $Q'$ similarly changes only which displayed relation reads are consumed; it introduces no new probe source.

Finally apply the public export theorem \autoref{thm:pde-public-interface-export}. The exported surface consists only of the carried source row, the manifold row, the derivative rows, the finite relation row, the boundary row, $\Pkg$ provenance, and the probe reads. The projected packet has exactly those same kinds of rows, with $R'$ and $Q'$ in place of the original relation reads. Since none of the cited obligations exports host functions, host coordinates, analytic solution data, trace-space equality, or solver objects, the projected packet exports none of them either.
\end{proof}

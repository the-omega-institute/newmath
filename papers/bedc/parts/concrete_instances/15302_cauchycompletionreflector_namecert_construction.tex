\chapter{A Concrete Naming Certificate for $\CauchyCompletionReflectorUp$}
\label{ch:concrete-instances-cauchycompletionreflector-namecert}
\origin{human}

Following Bishop and Bridges constructive completion, the $\CauchyCompletionReflectorUp$ packet records the public finite reflector surface for the L10 completion route. It gathers the unit surface of \autoref{ch:concrete-instances-cauchycompletionunit-namecert}, the counit surface of \autoref{ch:concrete-instances-cauchy-completion-counit-namecert}, the idempotent surface of \autoref{ch:concrete-instances-cauchy-completion-idempotent-namecert}, the extension route of \autoref{ch:concrete-instances-completionuniversalproperty-namecert}, and the reflection boundary of \autoref{ch:concrete-instances-completionreflection-namecert}. Its local real-facing rows are read through \autoref{ch:concrete-instances-real-cauchy-completion-namecert}, \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, \autoref{ch:concrete-instances-dyadicratcore-namecert}, and \autoref{ch:concrete-instances-real-namecert}. The packet is a finite reflector certificate; it is not a quotient completion, not a selected limit, not host equality of completed objects, not countable choice, and not ambient completeness.

\begin{definition}[CauchyCompletionReflector carrier]
\label{def:cauchy-completion-reflector-carrier}
A $\CauchyCompletionReflectorUp$ carrier is a finite $\BHist$ packet
$$
K=(S,M,U,Q,I,E,R,T,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the source row for the displayed metric, regular-Cauchy, or real-name data being sent toward completion.
\item $M$ is the completion-object row, read through the $\RealCauchyCompletionUp$ and $\MetricCompletionUp$ surfaces rather than through a host quotient carrier.
\item $U$ is the $\CauchyCompletionUnitUp$ row admitting $S$ into the completion-facing surface.
\item $Q$ is the $\CauchyCompletionCounitUp$ row returning already completed or real-sealed reads through the displayed regular route.
\item $I$ is the $\CauchyCompletionIdempotentUp$ row recording that a second completion pass reuses the same completion-object surface.
\item $E$ is the $\CompletionUniversalPropertyUp$ extension row for uniformly or continuously routed consumers attached to the same unit route.
\item $R$ is the $\CompletionReflectionUp$ row binding the construction, extension, separated boundary, and real-seal rows into one reflection surface.
\item $T$ is the local transport row through $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ windows used by $U,Q,I,E,R$.
\item $H$ records componentwise $\hsame$ transport for source, completion object, unit, counit, idempotence, extension, reflection, local transport, continuation, provenance, and naming rows.
\item $C$ records displayed $\Cont$ replay from a source read through unit admission, completion-object access, counit return, idempotent replay, extension use, reflection, and the local real-name route.
\item $P$ is the $\Pkg$ provenance over $\CauchyCompletionReflectorUp$, $\CauchyCompletionUnitUp$, $\CauchyCompletionCounitUp$, $\CauchyCompletionIdempotentUp$, $\CompletionUniversalPropertyUp$, $\CompletionReflectionUp$, $\RealCauchyCompletionUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CauchyCompletionReflectorUp}$ row.
\end{enumerate}
The classifier on accepted packets compares only these twelve rows under the displayed transport and replay data. It has no coordinate for quotient completions, selected limits, host equality, countable choice, or ambient completeness.
\end{definition}

\begin{theorem}[CauchyCompletionReflector NameCert obligations]
\label{thm:cauchy-completion-reflector-namecert-obligations}
For every accepted $\CauchyCompletionReflectorUp$ carrier $K=(S,M,U,Q,I,E,R,T,H,C,P,N)$, the local $\NameCert_{\CauchyCompletionReflectorUp}$ surface consists exactly of the following obligation rows:
\begin{enumerate}
\item carrier admission for the displayed source, completion object, unit, counit, idempotence, universal-property extension, reflection, local transport, continuation, provenance, and naming rows;
\item unit admission, requiring every completion-facing source read to pass through $U$ before $M$ can be consumed;
\item counit return, requiring every completed read that is already real-sealed to return through $Q$ before the terminal $\RealUp$ route is exposed;
\item idempotent replay, requiring a repeated completion request to reuse $I$ and the same completion-object row $M$;
\item extension and reflection compatibility, requiring every universal-property consumer to read $E$ and every reflection consumer to read $R$ over the same $S,M,U,Q,I,T$ route;
\item reflector non-escape, excluding quotient completions, selected limits, host equality, countable choice, ambient completeness, and detached $\RealUp$ completeness.
\end{enumerate}
No reflector obligation is stored outside those rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cauchy-completion-reflector-carrier}. The packet exposes the source, completion object, unit, counit, idempotence, extension, reflection, local transport, continuation, provenance, and naming rows, and no other rows.

The unit and counit obligations are then read in order. A completion-facing source read must first pass through the unit row $U$ before $M$ is available, while an already completed real-facing read returns through $Q$ before the $\RealUp$ route carried by $T$ can be consumed.

Idempotent replay uses the idempotent row $I$. Since $I$ is stored beside the same $M,U,Q,T$ rows, a second completion pass has no independent completion-object coordinate to inspect.

Finally, the extension row $E$ and reflection row $R$ are downstream of the same displayed route. Componentwise transport by $H$, replay by $C$, provenance by $P$, and naming by $N$ preserve exactly that finite route, so quotient completions, selected limits, host equality, countable choice, ambient completeness, and detached real completeness cannot be exported.
\end{proof}

\begin{theorem}[CauchyCompletionReflector unit-counit route]
\label{thm:cauchy-completion-reflector-unit-counit-route}
Let $K=(S,M,U,Q,I,E,R,T,H,C,P,N)$ be an accepted $\CauchyCompletionReflectorUp$ carrier. Every public reflector consumer factors through the finite route
$$
S\longmapsto U\longmapsto M\longmapsto Q\longmapsto I\longmapsto E,R\longmapsto T\longmapsto H,C,P,N .
$$
The route exposes the unit and counit faces of completion, replays idempotence, and then serves universal-property and reflection consumers through the same real-name transport. It does not supply quotient completion equality, a selected limit, host equality of completed objects, countable choice, or ambient completeness.
\end{theorem}
\begin{proof}
Begin with a public reflector consumer. The source row $S$ is visible only as the source of the displayed unit route $U$, so the consumer first reads unit admission before it can inspect the completion-object row $M$.

The counit row $Q$ supplies the return route for reads that are already on the real-seal side. This return is tied to the same local transport row $T$, whose $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, and $\RealUp$ windows are the only real-facing rows available to the packet.

Next read the idempotent row $I$. It states that repeating the completion pass reuses the same completion-object row and the same local transport, so the consumer cannot branch to a second completed carrier.

The universal-property row $E$ and reflection row $R$ are finally consumed over that fixed unit-counit-idempotent route. The structural rows $H,C,P,N$ transport, replay, source, and name only the displayed packet, which proves the factorization and the stated non-escape boundary.
\end{proof}

\chapter{A Concrete Naming Certificate for \(\ComplexUp\)}
\label{ch:concrete-instances-complex-namecert}

The complex number interface \(\ComplexUp\) is licensed by ordered pairs of Bishop reals from \autoref{ch:concrete-instances-real-namecert}, read as Cartesian real and imaginary components. The classifier is componentwise real sameness. It is not a primitive number system; it is licensed only via the real source and the pair constructor of \autoref{ch:concrete-instances-prod-namecert}.

\begin{remark}[Roadmap chapter---formalization pending]
\label{rem:concrete-instances-complex-namecert-roadmap}
This chapter sketches the licensing target. Lean implementations of the
listed specifications and certificate are not yet present; the chapter
is a spec roadmap, not a formalized license.
\end{remark}

\section{Carrier definition}

\begin{definition}[Complex carrier]
\label{def:complex-carrier}
Use ordered pairs of Bishop reals \((x,y)\) from \(\RealUp\).
\end{definition}

\section{The certificate}

\begin{definition}[Complex source specification]
\label{def:complex-source-specification}
The source is \(\ProdUp(\RealUp,\RealUp)\).
\end{definition}

\begin{definition}[Complex pattern specification]
\label{def:complex-pattern-specification}
The pattern is Cartesian pairing of real and imaginary components.
\end{definition}

\begin{definition}[Complex classifier specification]
\label{def:complex-classifier-specification}
The classifier is componentwise \(\RealUp\) sameness.
\end{definition}

\begin{definition}[Complex stability certificate]
\label{def:complex-stability-certificate}
The stability certificate consists of:
\begin{enumerate}
  \item componentwise reflexivity
  \item componentwise transitivity
  \item closure under pair construction
  \item closure of complex arithmetic through real arithmetic
\end{enumerate}
\end{definition}

\begin{definition}[Complex ledger policy]
\label{def:complex-ledger-policy}
Record both real component ledgers.
\end{definition}

\begin{theorem}[Concrete \(\ComplexUp\) naming certificate]
\label{thm:concrete-complex-namecert}
The real-pair tuple forms a valid NameCert for \(\ComplexUp\).
\end{theorem}

\begin{corollary}[\(\ComplexUp\) is licensed, not primitive]
\label{cor:complex-licensed-not-primitive}
\(\ComplexUp\) is licensed as a derived pair interface over \(\RealUp\).
\end{corollary}

\begin{theorem}[Dyadic interval hull NameCert obligations]
\label{thm:dyadic-interval-hull-namecert-obligations}
For every accepted $\DyadicIntervalHullUp$ carrier $H=(L,U,O,A,B,S,R,E,T,C,P,N)$, the local $\NameCert_{\DyadicIntervalHullUp}$ surface consists exactly of carrier admission, endpoint ordering, hull containment, located-real route, non-escape, transport, replay, provenance, and local naming obligations. No obligation is stored outside those displayed rows.
\end{theorem}
\begin{proof}
Project the accepted carrier. The lower and upper endpoint rows $L,U$ are both dyadic rational core rows, and $O$ is the only order witness available to the packet.

The enclosure ledger $A$ is then read before any interval consumer sees the hull. It records finite containment of the displayed source endpoints or cells in the same lower-upper dyadic boundary.

The located-real route is the ordered handoff through $B,S,R,E$. The dyadic interval row packages the endpoint ledger, the stream row exposes finite endpoint windows, the regular rational row records readback, and the real row seals the terminal boundary.

Finally $T,C,P,N$ preserve and name the route. Since the carrier has no coordinate for a host interval, selected endpoint pair, quotient stream, trichotomy, choice principle, or ambient completeness theorem, the listed obligations exhaust the local certificate.
\end{proof}

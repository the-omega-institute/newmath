Following Bishop--Bridges constructive analysis, unique relation readback is admitted only through displayed existence and uniqueness rows. The $\UniqueChoicePrincipleUp$ packet records that finite surface as a NameCert carrier: a source row and relation row are not turned into a selected representative or host function, but into a deterministic readback row whose use is justified by visible existence evidence and a uniqueness ledger. It sits beside the extension consumers of \autoref{ch:concrete-instances-uniform-continuous-extension-namecert} and \autoref{ch:concrete-instances-cauchy-continuous-extension-namecert}, the completion interface of \autoref{ch:concrete-instances-completionuniversalproperty-namecert}, and the separated comparison boundary of \autoref{ch:concrete-instances-separatedmetric-namecert}.

The packet is a finite relation-to-readback certificate. It does not assert countable choice, selected representatives, host function equality, quotient completion, real trichotomy, or ambient completeness; it records only the displayed route by which an accepted source and relation row expose one deterministic readback.

\begin{definition}[UniqueChoicePrinciple carrier]
\label{def:unique-choice-principle-carrier}
A $\UniqueChoicePrincipleUp$ carrier is a finite $\mathsf{BHist}$ packet
$$
Q=(S,R,E,U,D,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the source carrier row whose accepted elements may be queried.
\item $R$ is the relation row over $S$ and the readback surface, stored as finite classifier data rather than as a host function.
\item $E$ is the existence witness row. For each accepted source read, it displays a relation output row before deterministic readback is visible.
\item $U$ is the uniqueness ledger. It compares any two displayed relation outputs for the same accepted source by componentwise $\hsame$ transport and classifier agreement.
\item $D$ is the deterministic readback row obtained only after $E$ and $U$ have been displayed.
\item $H$ records componentwise $\hsame$ transport for source rows, relation rows, existence rows, uniqueness-ledger rows, deterministic readback rows, continuation routes, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a source query through $S,R,E,U,D,H,P,N$.
\item $P$ is the $\Pkg$ provenance over $\UniqueChoicePrincipleUp$, $\mathsf{BHist}$, $\hsame$, classifier data, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\UniqueChoicePrincipleUp}$ row.
\end{enumerate}
The carrier predicate asserts exactly that these rows form one finite unique-relation readback packet. It has no coordinate for countable choice, selected representatives, host function equality, quotient completion, real trichotomy, ambient completeness, Lean verification, or bridge checking.
\end{definition}

\begin{theorem}[UniqueChoicePrinciple NameCert obligations]
\label{thm:unique-choice-principle-namecert-obligations}
For every accepted $\UniqueChoicePrincipleUp$ carrier $Q=(S,R,E,U,D,H,C,P,N)$, the local $\NameCert_{\UniqueChoicePrincipleUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the source row $S$, relation row $R$, existence witness row $E$, uniqueness ledger $U$, deterministic readback row $D$, transport, continuation, provenance, and local naming rows;
\item existence visibility, requiring each accepted source query to display an $E$ row before any deterministic readback row is consumed;
\item uniqueness-ledger determinacy, requiring two relation outputs over the same accepted source to be compared only through $U$ by $\hsame$ transport and classifier agreement;
\item readback stability under $\hsame$ transport, requiring the deterministic row $D$ to transport componentwise with $S,R,E,U,H,C,P,N$;
\item ledger non-escape, requiring every deterministic readback to factor through the displayed existence and uniqueness rows and forbidding countable choice, selected representatives, host function equality, quotient completion, real trichotomy, and ambient completeness from the local surface.
\end{enumerate}
No unique-choice obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:unique-choice-principle-carrier}. The packet displays the source, relation, existence witness, uniqueness ledger, deterministic readback, transport, continuation, provenance, and local naming rows before any readback can be consumed.

Existence visibility follows from the position of $E$ in the carrier route. A source query can reach $D$ only through the replay row $C$, and that replay passes through the relation row and the existence witness row first.

Uniqueness-ledger determinacy is the role of $U$. If two relation outputs are displayed for the same accepted source row, the packet compares them by the $\hsame$ and classifier data recorded in $U$, not by host equality or a hidden selector.

Readback stability is componentwise. The transport ledger $H$ moves the source, relation, existence, uniqueness, deterministic readback, replay, provenance, and naming rows together, so the deterministic row has no independent equality principle.

Finally inspect the carrier inventory. Since no coordinate of $Q$ is a countable-choice schedule, selected representative, host function equality proof, quotient-completion object, trichotomy decision, or ambient-completeness theorem, none can occur in the local NameCert surface.
\end{proof}

\begin{theorem}[UniqueChoicePrinciple deterministic readback]
\label{thm:unique-choice-principle-deterministic-readback}
Let $Q=(S,R,E,U,D,H,C,P,N)$ be an accepted $\UniqueChoicePrincipleUp$ carrier. Any two deterministic readbacks over the same accepted source row and relation row are compared only through the displayed uniqueness ledger $U$ and the componentwise $\hsame$ transport recorded by $H$.

Thus deterministic readback is a finite relation-ledger boundary. It is not countable choice, a selected representative, host function equality, quotient completion, real trichotomy, or ambient completeness.
\end{theorem}
\begin{proof}
Start with two accepted deterministic readbacks over the same source row $S$ and relation row $R$. By \autoref{def:unique-choice-principle-carrier}, each readback must be the row $D$ reached through displayed existence evidence $E$ and replay $C$ inside the same packet.

The NameCert obligations of \autoref{thm:unique-choice-principle-namecert-obligations} force comparison of the corresponding relation outputs through the uniqueness ledger $U$. That ledger exposes only $\hsame$ transport and classifier agreement for the displayed rows.

The transport ledger $H$ then carries this comparison componentwise across $S,R,E,U,D,C,P,N$. There is no separate host function equality or representative selector available to the readback.

The excluded principles have no coordinate in $Q$ and no replay edge in $C$. Therefore any accepted comparison of deterministic readbacks is exactly the displayed uniqueness-ledger comparison transported by $H$.
\end{proof}

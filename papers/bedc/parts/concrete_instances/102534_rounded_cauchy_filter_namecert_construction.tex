$$

\item $F$ is the $\CauchyFilterUp$ row carried by the same base tokens and by the displayed Cauchy comparison requests.
\item $R$ is the rounded-base refinement row: every accepted base token exports a strictly smaller displayed token below it, together with the finite inclusion route witnessing the refinement.
\item $U$ is the $\CauchyFilterCompletionUp$ handoff row that may be read only after the base, Cauchy, and rounded-refinement rows have been displayed.

\item $Q$ is the $\RegSeqRatUp$ readback row attached to the same windows and to the rational observations exported by the rounded filter route.

The classifier compares accepted packets only through $B,F,R,U,W,Q,D,E,H,C,P,N$. It has no coordinate for quotient equality of filters, an ultrafilter extension, selected cluster points, trichotomy, host real completeness, Lean verification, or bridge data.

\begin{theorem}[Rounded Cauchy filter NameCert obligations]\label{thm:rounded-cauchy-filter-namecert-obligations}
For every accepted $\RoundedCauchyFilterUp$ carrier $K=(B,F,R,U,W,Q,D,E,H,C,P,N)$, the local $\NameCert_{\RoundedCauchyFilterUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the displayed filter base, Cauchy filter, rounded refinement, completion handoff, stream-window, regular-readback, dyadic-ledger, real-seal, transport, replay, provenance, and local naming rows;
\item base alignment, requiring the $\FilterBaseUp$ row $B$ and the $\CauchyFilterUp$ row $F$ to read the same displayed base tokens before any completion-facing observation is accepted;
\item roundedness, requiring each accepted base token to pass through the row $R$ and receive a displayed smaller token with its finite inclusion route;
\item Cauchy comparison stability, requiring every tolerance request to use the same rounded base tokens and dyadic ledger $D$ before the completion handoff $U$ is read;
\item regular readback and real-seal boundary, requiring $W,Q,D$ to mediate the route from rounded filter data to the terminal $\RealUp$ seal $E$;
\item componentwise transport, displayed replay, provenance, and local naming through $H,C,P,N$ for exactly the rows displayed above.
\end{enumerate}
No rounded Cauchy filter obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Project the accepted packet $K=(B,F,R,U,W,Q,D,E,H,C,P,N)$. Carrier admission exposes exactly the filter-base row, Cauchy-filter row, rounded refinement row, completion handoff, stream-window row, regular readback, dyadic ledger, real seal, and structural rows.

Base alignment is finite. The same displayed base tokens feed $B$ and $F$, so a consumer cannot read a Cauchy comparison apart from the filter-base surface that names its neighborhoods.

Roundedness is then read through $R$. For each accepted base token, $R$ supplies the smaller displayed token and the inclusion route that places it below the original token. The Cauchy comparison rows use those rounded tokens and the dyadic ledger $D$ before the handoff $U$ can be consumed.

Finally $W,Q,D,E,H,C,P,N$ provide the observation, readback, tolerance, endpoint, transport, replay, source, and local name of the same route. Since the carrier has no slot for quotient equality, ultrafilter extension, selected cluster points, trichotomy, host completeness, Lean verification, or bridge data, the listed rows exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[Rounded Cauchy filter regular readback route]\label{thm:rounded-cauchy-filter-regular-readback-route}

$$

$$

\end{theorem}
\begin{proof}

The request then opens finite windows through $W$. Those windows inspect the base token, its rounded refinement, and the Cauchy comparison rows attached to that refinement.

Regular readback is forced through $Q$, and the tolerance accounting is forced through $D$. Only after this finite observation route has been displayed may the completion handoff $U$ and the real seal $E$ be consumed.

The structural rows $H,C,P,N$ transport, replay, source, and name the same route. No excluded quotient, representative sequence, ultrafilter, or completeness coordinate appears in the carrier, so the displayed factorization is the only completion-facing read.
\end{proof}

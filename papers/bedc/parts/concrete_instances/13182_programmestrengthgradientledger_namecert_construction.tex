\begin{theorem}[Programme strength gradient no over-reporting]
\label{thm:programme-strength-gradient-no-overreporting}
An accepted $\ProgrammeStrengthGradientLedgerUp$ packet cannot export a reported strength above the minimum grade licensed by its displayed theory-closure row $T$, formal-status row $F$, bridge row $B$, machine-interface row $M$, verified-output harness row $V$, downgrade row $D$, upgrade-registry row $U$, empirical-boundary row $E$, and refusal row $R$. A consumer that asks for a stronger grade receives the downgrade row rather than a promoted report.
\end{theorem}
\begin{proof}
Read $T,F,$ and $B$ first. These rows record the independent paper, verification, and bridge axes, so no one axis can raise the others.

Read $M$ and $V$ next. A machine-facing report is available only through the displayed interface and checker-facing harness; unchecked machine output is therefore not a higher grade inside the packet.

Finally read $D,U,E,$ and $R$. The downgrade row supplies the lower report, the upgrade registry names the next obligation, the empirical-boundary row blocks value substitution, and the refusal row names over-strong reports. The carrier has no remaining coordinate that can convert these blockers into discharged obligations.
\end{proof}

Thus the over-strong report cannot be a consumer of the accepted carrier. It is a falsifying observation for the purported $\ProgrammeStrengthGradientLedgerUp$ packet.

\begin{definition}[Adaptive Cauchy modulus carrier]
\label{def:adaptive-cauchy-modulus-carrier}
An $\AdaptiveCauchyModulusUp$ carrier is a finite $\BHist$ packet
$$
A=(S,D,T,W,R,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the source $\StreamNameUp$ row, exposing the finite name windows available to a regular Cauchy request.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger, recording the requested positive dyadic thresholds and the refinements used by the threshold search.
\item $T$ is the adaptive threshold schedule, assigning to each displayed tolerance request the finite stage bound selected from the inspected evidence.
\item $W$ is the inspected-window row, listing exactly the stream windows opened while validating the schedule $T$.
\item $R$ is the $\RegSeqRatUp$ readback row obtained from the source stream, tolerance ledger, threshold schedule, and inspected windows.
\item $E$ is the terminal $\RealUp$ seal row available only after the regular-rational readback has been displayed.
\item $H$ records componentwise $\hsame$ transport for $S,D,T,W,R,E,C,P,N$.
\item $C$ records displayed $\Cont$ replay from an adaptive-modulus request through the same source, tolerance, schedule, window, readback, seal, transport, provenance, and naming rows.
\item $P$ is the $\Pkg$ provenance over $\AdaptiveCauchyModulusUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, $\CauchyModulusUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\AdaptiveCauchyModulusUp}$ row.
\end{enumerate}
The classifier accepts only packets whose threshold read is mediated by $S,D,T,W,R,E,H,C,P,N$. There is no coordinate for a selected limit, quotient stream equality, host real equality, ambient completeness, countable choice, or an uninspected global modulus.
\end{definition}

\begin{theorem}[Adaptive Cauchy modulus NameCert obligations]
\label{thm:adaptive-cauchy-modulus-namecert-obligations}
For every accepted $\AdaptiveCauchyModulusUp$ carrier
$$
A=(S,D,T,W,R,E,H,C,P,N),
$$
the local $\NameCert_{\AdaptiveCauchyModulusUp}$ surface consists exactly of source-stream admission, dyadic tolerance-ledger exposure, adaptive threshold-schedule admission, inspected-window replay, regular-rational readback, terminal Real seal readback, componentwise $\hsame$ transport, displayed $\Cont$ replay, $\Pkg$ provenance, and local naming through the rows of $A$.

No obligation in the packet is discharged by a selected Cauchy limit, quotient equality of streams, host real equality, ambient completeness, countable choice, or an oracle for a modulus outside the displayed finite windows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:adaptive-cauchy-modulus-carrier}. The analytic rows are precisely the source stream $S$, tolerance ledger $D$, adaptive threshold schedule $T$, inspected windows $W$, regular readback $R$, and terminal Real seal $E$.

The access order is forced by the rows. A consumer first reads a dyadic request from $D$, then uses $T$ to select a finite stage bound, and then opens only the windows recorded by $W$ in the source stream $S$.

The readback rows add no external completion principle. The row $R$ repacks the inspected stream and tolerance data as a regular rational sequence, and $E$ seals that readback for Real-facing consumers only after $R$ is present.

Finally $H,C,P,N$ transport, replay, source, and name the same packet. Since no coordinate of $A$ stores a selected limit, quotient equality, host equality, ambient completeness, countable choice, or an uninspected global modulus, the displayed rows exhaust the local NameCert obligations.
\end{proof}

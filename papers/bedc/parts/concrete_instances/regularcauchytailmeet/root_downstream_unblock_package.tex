\begin{theorem}[RegularCauchyTailMeet root downstream window exhaustion]
\label{thm:regular-cauchy-tail-meet-root-downstream-window-exhaustion}

$\RegularCauchyTailMeetUp$ packet. Every downstream $\RealUp$ or
$\CompletionUp$ root consumer that reads the TailMeet threshold must first
exhaust the paired window-source route through the displayed $\BHist$ rows
$R_0,R_1,W_0,W_1$, the componentwise $\hsame$ row $H$, the displayed $\Cont$
route $C$, the finite ledger $L$, the package row $\Pkg$, and the local
$\NameCert$ row $N$. No downstream read can reach $\tau$, $Q$, or a Real seal
handoff before those four source-window rows have been exposed.
\end{theorem}
\begin{proof}
Start with TailMeet window-source coverage. It forces any threshold-facing read
to expose the two regular-Cauchy source rows $R_0,R_1$ and the two stream-window
rows $W_0,W_1$ before the shared dyadic threshold is available.

Next attach the structural rows. Shared-threshold transport carries the exposed
window-source route through the componentwise $\hsame$ row $H$ and the displayed
$\Cont$ route $C$, while consumer exactness records the same route in $L$,
$\Pkg$, and $N$.

Finally read the Real seal handoff. The handoff can consume only the already
displayed finite TailMeet route, so a root downstream consumer cannot bypass the
window-source rows to reach $\tau$, $Q$, or any completion-facing seal row.
\end{proof}

\chapter{A Concrete Naming Certificate for $\FastCauchyUp$}
\label{ch:concrete-instances-fastcauchy-namecert}

The $\FastCauchyUp$ interface records a finite, modulus-normalized Cauchy-name surface feeding the regular-sequence and real-name layers. In Bishop-style constructive analysis, a Cauchy name is useful only together with visible rate data; the BEDC packet keeps that rate as finite certificate data over $\StreamNameUp$, $\CauchyModulusUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$.

\section{Finite modulus-normalized Cauchy names}
\label{sec:fastcauchy-finite-modulus}

\begin{definition}[FastCauchy carrier]
\label{def:fastcauchy-carrier}
A $\FastCauchyUp$ carrier is a finite $\mathsf{BHist}$ packet
$$
C=(s,\mu,E,R,L,T,P,N)
$$
with the following visible rows:
\begin{enumerate}
\item $s$ is a scheduled $\StreamNameUp$ observation table whose displayed entries are dyadic-rational observations;
\item $\mu$ is a $\CauchyModulusUp$ threshold table normalized so that each requested precision row names the finite index after which all scheduled comparisons are read;
\item $E$ is the dyadic endpoint ledger read through $\DyadicRatCoreUp$ for every scheduled observation and precision row used by $\mu$;
\item $R$ is the late-pair radius ledger, recording for each precision row that any two scheduled observations beyond the threshold named by $\mu$ fit inside the displayed dyadic radius;
\item $L$ records the componentwise $\hsame$ transport rows for the stream, threshold, dyadic endpoint, and late-pair ledgers;
\item $T$ records the $\Cont$ windows through which those finite reads are consumed by downstream certificates;
\item $P$ is the $\mathsf{Pkg}$ provenance for the named source chapters $\DyadicRatCoreUp$, $\CauchyModulusUp$, $\StreamNameUp$, $\RegSeqRatUp$, and $\RealUp$;
\item $N$ is the local $\NameCert_{\FastCauchyUp}$ row for the displayed packet.
\end{enumerate}
The carrier is a finite explicit-rate Cauchy name. It contains no quotient stream, no host real equality, no selected limit, and no standard bridge datum.
\end{definition}

\begin{definition}[FastCauchy classifier]
\label{def:fastcauchy-classifier}
For accepted $\FastCauchyUp$ packets $C$ and $C'$, the classifier $\mathsf{FastCauchyClassifier}(C,C')$ holds when the scheduled observation windows, modulus threshold rows, dyadic endpoint ledgers, late-pair radius ledgers, $\hsame$ transports, $\Cont$ windows, $\mathsf{Pkg}$ provenance, and local $\NameCert$ rows compare componentwise through their displayed source classifiers.

The classifier is exact only for the finite modulus-normalized ledger. It does not assert equality of completed real numbers, equality of quotient Cauchy streams, or existence of a chosen limit.
\end{definition}

\begin{theorem}[FastCauchy modulus transport]
\label{thm:fastcauchy-modulus-transport}
Let $C$ be an accepted $\FastCauchyUp$ packet. If the scheduled observation rows of $C$ transport along the displayed $\hsame$ rows and the threshold table $\mu$ transports along the displayed $\Cont$ windows, then the transported packet preserves the same modulus-normalized late-pair radius obligations.

Concretely, every precision row in $\mu$ keeps its finite threshold, every scheduled pair beyond that threshold keeps its dyadic endpoint rows from $\DyadicRatCoreUp$, and the late-pair radius ledger remains readable by the same $\CauchyModulusUp$ classifier.
\end{theorem}
\leanchecked{BEDC.Derived.FastCauchyUp.FastCauchyFiniteCarrier\_modulus\_transport}
\begin{proof}
Project the carrier tuple of \autoref{def:fastcauchy-carrier}. The scheduled stream, threshold table, dyadic endpoint ledger, and late-pair ledger are separate finite rows, so each row can be transported through its displayed $\hsame$ coordinate.

The $\Cont$ row supplies the finite window in which downstream consumers may read the transported data. Because the threshold row is preserved, every pair of scheduled observations whose indices are beyond that threshold remains in the same late-pair comparison window.

Repack the transported stream, threshold, endpoint, radius, transport, window, provenance, and local certificate rows. The result has the same carrier shape and satisfies the classifier of \autoref{def:fastcauchy-classifier}, so the modulus-normalized obligations are preserved.
\end{proof}

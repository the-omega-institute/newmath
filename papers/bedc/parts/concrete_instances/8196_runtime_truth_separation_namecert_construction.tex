\begin{definition}[Runtime/truth separation carrier]
\label{def:runtime-truth-separation-carrier}
A $\RuntimeTruthSeparationUp$ carrier is a finite $\BHist$ packet
$$
R=(S,E,T,L,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the structural description row for the finite $\mathrm{CIC}^{*}$ self-description accepted by the reflection-limit packet;
\item $E$ is the runtime-request row, recording a consumer request to execute, step, normalize, or validate the structural description by an external host runtime;
\item $T$ is the truth-validation request row, recording a consumer request for a truth predicate, host validation oracle, global truth classifier, or semantic correctness judgment over the same structural description;
\item $L$ is the refusal ledger, routing every request in $E$ or $T$ back to the finite packet unless the consumer reads only the structural row $S$ and its displayed bookkeeping;
\item $H$ records componentwise $\hsame$ transport for the structural description, runtime-request, truth-validation request, refusal ledger, continuation, provenance, and local naming rows;
\item $C$ records displayed $\Cont$ routes for reading $S,E,T,L,H,P,N$ without adding an evaluator, truth predicate, or validation oracle;
\item $P$ is the $\Pkg$ provenance over $\ReflectionLimitBoundaryUp$, $\RuntimeCandidateBoundaryUp$, $\FiniteTraceEvaluatorUp$, $\GroundLoopBoundaryUp$, $\QuotientSoundnessBoundaryUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows;
\item $N$ is the local $\NameCert_{\RuntimeTruthSeparationUp}$ row.
\end{enumerate}
The carrier exports structural self-description and its refusal ledger only. It contains no runtime executor, no truth predicate, no host validation oracle, and no off-packet evaluator.
\end{definition}

\begin{theorem}[Runtime/truth separation non-escape]
\label{thm:runtime-truth-separation-nonescape}
Every accepted $\RuntimeTruthSeparationUp$ packet $R=(S,E,T,L,H,C,P,N)$ exposes structural $\mathrm{CIC}^{*}$ data only through $S,H,C,P,N$ and routes runtime or truth-validation requests only through $E,T,L$. It exports no runtime executor, truth predicate, host validation oracle, or off-packet evaluator.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:runtime-truth-separation-carrier}. The row $S$ is the only structural self-description row, while $E$ and $T$ are request rows rather than execution or truth witnesses.

The refusal ledger $L$ is the only row that consumes those requests. A runtime read must pass through $E$ and then $L$; a truth-validation read must pass through $T$ and then $L$.

Finally inspect the bookkeeping rows. The transport row $H$, continuation row $C$, provenance row $P$, and local name $N$ preserve and route the displayed finite packet. Since none of these coordinates contains an executor, truth predicate, host validation oracle, or evaluator, every accepted consumer read remains inside the structural row or the refusal ledger.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor ground compiler authorization]
\label{thm:authorized-generator-recursor-ground-compiler-authorization}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet, any ground-compiler consumer of the recursor output must read the packet through the displayed rows
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
The consumer may reach $O$ only after the accepted signature row $I$, emitted eliminator row $E$, motive row $M$, branch row $B$, structural descent row $D$, audit row $A$, $\BHist$ carrier row, displayed $\Cont$ row $C$, package row $\Pkg$ at $P$, boundary row $G$, and local $\NameCert$ row $N$ have all been retained in the same route.
\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursor\_ground\_compiler\_authorization}
\begin{proof}
Start with the carrier of \autoref{def:authorized-generator-recursor-carrier}. It presents the accepted recursor output as one row of the finite $\BHist$ packet, so the ground compiler has no initial read outside $I,E,M,B,D,O,A,H,C,P,G,N$.

The computational rows are fixed by the existing branch and descent obligations. Apply \autoref{thm:authorized-generator-recursor-branch-exhaustion} and \autoref{thm:authorized-generator-recursor-structural-descent-locality}; the branch row is exhausted by the displayed constructors, and the recursive use is local to the motive and descent rows.

The audit and naming side is fixed by \autoref{thm:authorized-generator-recursor-audit-provenance-factorization}, \autoref{thm:authorized-generator-recursor-root-host-boundary-namecert-exhaustion}, and \autoref{thm:authorized-generator-recursor-ground-compiler-handoff}. These rows retain audit provenance, carrier admission, boundary exhaustion, $\Pkg$ provenance, and the local $\NameCert$ certificate before any ground-compiler consumer may read $O$.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor finite route exposure]
\label{thm:authorized-generator-recursor-finite-route-exposure}
Every public route exposed by an accepted $\AuthorizedGeneratorRecursorUp$ root packet is exhausted by the finite packet rows
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
Consequently the public surface cannot introduce an unregistered branch, a hidden induction channel, or an off-packet generator path beyond the accepted signature, eliminator, motive, branch, descent, output, audit, transport, continuation, package, boundary, and local certificate rows.
\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorFiniteRouteExposure}
\begin{proof}
Read the route from \autoref{thm:authorized-generator-recursor-root-carrier-route}. It already factors every closed-substitution-facing public route through the displayed packet and leaves no carrier coordinate outside the finite row list.

Branch exposure is exhausted by \autoref{thm:authorized-generator-recursor-branch-exhaustion}. A public route can name only the branch package associated with the accepted constructors and the emitted eliminator, so an unregistered branch has no row from which to enter.

The remaining escape routes are closed by descent locality and ledger descent. Use \autoref{thm:authorized-generator-recursor-structural-descent-locality}, \autoref{thm:authorized-generator-recursor-audit-ledger-nonescape}, and \autoref{thm:authorized-generator-recursor-ground-compiler-handoff}; together they keep recursive use, audit provenance, and ground-compiler consumption inside the same finite packet.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor output audit nonescape]
\label{thm:authorized-generator-recursor-output-audit-nonescape}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet, the output row $O$ cannot be consumed apart from the audit provenance row $A$, the ground-compiler boundary row $G$, and the local certificate row $N$. Any consumer of $O$ must preserve the same displayed route through audit provenance, boundary exposure, and $\NameCert_{\AuthorizedGeneratorRecursorUp}$ certification.
\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorOutputAuditNonescape}
\begin{proof}
Begin with \autoref{thm:authorized-generator-recursor-audit-provenance-factorization}. It attaches public exposure of $O$ to the audit and provenance rows, so the output cannot be read as an isolated generator result.

The boundary side is supplied by \autoref{thm:authorized-generator-recursor-root-host-boundary-namecert-exhaustion}. The row $G$ is exhausted by the displayed packet and cannot export a hidden proof generator, external axiom, or detached induction channel.

Finally apply \autoref{thm:authorized-generator-recursor-ground-compiler-authorization}. Since every ground-compiler consumer reads $O$ only after the carrier, audit, boundary, package, and local $\NameCert$ rows are retained, the output row has no nonescaping consumer route apart from $A,G,N$ and the finite packet that carries them.
\end{proof}

\begin{corollary}[AuthorizedGeneratorRecursor ground-compiler authorization strict obstruction]
\label{cor:authorized-generator-recursor-ground-compiler-authorization-strict-obstruction}
Let an accepted $\AuthorizedGeneratorRecursorUp$ root packet be read by a ground-compiler consumer. Suppose a displayed failure, refusal, separation, or countermodel row is cited against a named converse or reuse route for the output row $O$. Then the row is a strict local obstruction: it blocks only the named route whose consumer-facing use tries to read, omit, detach, replace, or bypass one of the displayed packet rows
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
In particular every permitted consumer-facing use of the cited row factors through the audit row $A$, the componentwise transport row $H$, the displayed $\Cont$ row $C$, the package row $P$, the ground-compiler boundary row $G$, and the local $\NameCert_{\AuthorizedGeneratorRecursorUp}$ row $N$ before it can speak about $O$. The same row cannot be reinterpreted as a global impossibility theorem, a host theorem, hidden external validation, or a certificate for any coordinate not displayed by the accepted packet.
\end{corollary}
\begin{proof}
Start with finite route exposure. By \autoref{thm:authorized-generator-recursor-finite-route-exposure}, every public route exposed by the accepted packet is exhausted by
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
The theorem also excludes an unregistered branch, a hidden induction channel, and an off-packet generator path beyond the accepted signature, eliminator, motive, branch, descent, output, audit, transport, continuation, package, boundary, and local certificate rows. Hence any consumer-facing use of a displayed failure, refusal, separation, or countermodel row has no public coordinate outside that finite list.

Now apply output-audit nonescape, \autoref{thm:authorized-generator-recursor-output-audit-nonescape}. A consumer of $O$ cannot detach the output row from audit provenance $A$, boundary exposure $G$, and local certification $N$; it must preserve the same displayed route through those rows. Together with the ground-compiler authorization route already displayed in the chapter, this forces the consumer-facing readback through $A,H,C,P,G,N$, where $H$ carries componentwise transport, $C$ carries the displayed $\Cont$ replay, $P$ carries package provenance, $G$ carries the ground-compiler boundary, and $N$ carries the local name certificate.

Consider the cited displayed row. If the named converse or reuse route asks to read $O$ while omitting, replacing, detaching, or bypassing one of the rows above, the two cited theorems place the obstruction exactly at that missing packet coordinate. Thus the row blocks that named route and no other route that remains inside the displayed carrier.

It remains to rule out stronger readings. A global impossibility theorem would require a public conclusion not confined to the accepted packet and not indexed by the named failed route. A host theorem would require a host-facing theorem coordinate not among $I,E,M,B,D,O,A,H,C,P,G,N$. Hidden external validation would require an extra validation or certificate row outside $A,H,C,P,G,N$. Finite route exposure gives no such coordinate, and output-audit nonescape prevents $O$ from being read apart from the displayed audit, boundary, transport, continuation, package, and local naming route. Therefore the obstruction remains strict and local to the named ground-compiler converse or reuse route.
\end{proof}

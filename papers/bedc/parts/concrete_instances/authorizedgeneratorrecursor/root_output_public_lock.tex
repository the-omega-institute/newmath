\begin{theorem}[AuthorizedGeneratorRecursor accepted signature route]
\label{thm:authorized-generator-recursor-accepted-signature-route}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet with displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N),
$$
any public read that consumes the output row $O$ must first expose the accepted signature row $I$, emitted recursor row $E$, checked motive row $M$, authorized branch package $B$, and structural-descent row $D$. The route may then read $O$ only as the output row of the same carrier; it has no detached branch table, motive-free recursive channel, hidden generator source, or off-packet signature route.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:authorized-generator-recursor-carrier}. It places $I,E,M,B,D$ before $O$ in one finite $\BHist$ packet, so the public consumer begins with the displayed signature, recursor, motive, branch, and descent rows rather than with a separate generator source.

Use \autoref{thm:authorized-generator-recursor-carrier-admission} and \autoref{thm:authorized-generator-recursor-route-authorization}. These results identify accepted route consumption with the same finite carrier and require the route to retain the source signature, emitted recursor, motive, branch package, descent row, output row, audit row, and local certificate together.

Finally apply branch exhaustion and descent locality. Theorems~\ref{thm:authorized-generator-recursor-branch-exhaustion} and~\ref{thm:authorized-generator-recursor-structural-descent-locality} keep the branch package on the accepted constructor list and keep recursive use local to the motive and descent rows. Therefore $O$ is reached only after $I,E,M,B,D$ have become visible on the accepted signature route.
\end{proof}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorAcceptedSignatureRoute}

\begin{theorem}[AuthorizedGeneratorRecursor output boundary totality]
\label{thm:authorized-generator-recursor-output-boundary-totality}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet with displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N),
$$
the output row $O$ becomes public only while the audit row $A$, componentwise $\hsame$ transport $H$, displayed $\Cont$ route $C$, $\Pkg$ provenance $P$, boundary row $G$, and local $\NameCert$ row $N$ are retained. The public output boundary exports no hidden generator, free induction principle, external axiom channel, or unaudited dependency route outside $R$.
\end{theorem}
\begin{proof}
Start with the output side of the displayed packet. The rows $O,A,H,C,P,G,N$ are not separate public surfaces; they are the output, audit, transport, continuation, provenance, boundary, and local naming coordinates of the same carrier.

Apply \autoref{thm:authorized-generator-recursor-root-boundary-ledger-totality}. It exhausts the output boundary ledger by $I,E,M,B,D,O,A,H,C,P,G,N$ and rules out a hidden generator, unaudited dependency route, external axiom channel, host-kernel completeness claim, or naming certificate outside the displayed packet.

Use \autoref{thm:authorized-generator-recursor-terminal-authorized-output-cover} for terminal consumption. That theorem keeps terminal readers on the same authorized output rows, so public exposure of $O$ cannot discard $A,H,C,P,G,N$ while retaining an output name.
\end{proof}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursor\_output\_boundary\_totality}

\begin{lemma}[AuthorizedGeneratorRecursor accepted signature downstream coverage]
\label{lem:authorized-generator-recursor-accepted-signature-downstream-coverage}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet with displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N),
$$
every accepted local consumer of the accepted signature output is exhausted by the finite rows displayed in $R$. More precisely, the consumer can read only an admission alternative among $I,E,M,B,D$, an output-boundary ledger row among $O,A,H,C,P,G,N$, or the same-packet coverage obtained by keeping those two surfaces coupled inside the carrier. A request for a detached branch table, motive-free recursive channel, hidden generator source, off-packet signature route, free induction principle, external axiom channel, unaudited dependency route, or naming certificate outside $R$ remains outside the accepted $\BEDC$ packet.
\end{lemma}
\begin{proof}
Fix an accepted root packet and project the displayed carrier $R$. By \autoref{thm:authorized-generator-recursor-accepted-signature-route}, any public read that consumes $O$ must first expose the accepted signature row $I$, emitted recursor row $E$, checked motive row $M$, authorized branch package $B$, and structural-descent row $D$. The same theorem also says that the subsequent read of $O$ is a read of the output row of this carrier, with no detached branch table, motive-free recursive channel, hidden generator source, or off-packet signature route. Thus the admission side of any accepted local consumer is already exhausted by the finite alternatives $I,E,M,B,D$.

Now apply \autoref{thm:authorized-generator-recursor-output-boundary-totality} to the output side. Once $O$ is public, the audit row $A$, componentwise $\hsame$ transport $H$, displayed $\Cont$ route $C$, $\Pkg$ provenance $P$, boundary row $G$, and local $\NameCert$ row $N$ remain attached to the same carrier. Hence a downstream consumer that passes the admission side can only proceed through the displayed output-boundary ledger $O,A,H,C,P,G,N$.

Combining the two cited theorems leaves no third source of consumer-facing data. A local consumer either stops on the admission alternatives $I,E,M,B,D$, reads the output-boundary ledger $O,A,H,C,P,G,N$, or uses their same-packet coupling as the coverage row for $R$. The alternatives excluded by the accepted signature route are absent before $O$, and the alternatives excluded by output boundary totality are absent after $O$. Therefore every unlisted coordinate remains outside the accepted $\BEDC$ packet.
\end{proof}

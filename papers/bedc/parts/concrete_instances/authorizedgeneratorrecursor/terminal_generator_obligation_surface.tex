\begin{theorem}[AuthorizedGeneratorRecursor terminal generator admission]
\label{thm:authorized-generator-recursor-terminal-generator-admission}
For every accepted $\AuthorizedGeneratorRecursorUp$ packet, every terminal generator-facing consumer route is admitted only through the displayed packet rows
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
In particular, a terminal normalization, closed-substitution, or generator-audit consumer may read the output row $O$ only as a row of this finite carrier and never as a detached generator authority.
\end{theorem}
\begin{proof}
Start with the carrier admission and total route surface. By \autoref{thm:authorized-generator-recursor-carrier-admission}, \autoref{thm:authorized-generator-recursor-acceptance-totality}, and \autoref{thm:authorized-generator-recursor-finite-route-exposure}, the accepted packet displays exactly the signature, eliminator, motive, branch, descent, output, audit, transport, continuation, provenance, boundary, and local naming rows.

Next use terminal consumer readiness. \autoref{thm:authorized-generator-recursor-terminal-consumer-readiness} keeps terminal normalization and closed-substitution consumers behind the same displayed carrier, while \autoref{thm:authorized-generator-recursor-ground-compiler-authorization} gives the ground compiler no read outside the same packet rows.

Finally apply output-audit nonescape. By \autoref{thm:authorized-generator-recursor-output-audit-nonescape}, the terminal consumer cannot detach $O$ from the audit, boundary, and local $\NameCert$ rows, so admission is exhausted by $I,E,M,B,D,O,A,H,C,P,G,N$.
\end{proof}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorTerminalGeneratorAdmission}

\label{thm:authorized-generator-recursor-terminal-branch-descent-exactness}
For every accepted $\AuthorizedGeneratorRecursorUp$ packet, any terminal branch consumption that reaches the output row $O$ must first pass through the branch row $B$ and the structural-descent row $D$. No terminal route may consume a branch, recursive call, or output-facing generator result unless the branch is carried by $B$ and the descent evidence is carried by $D$ inside the same displayed packet.
\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorTerminalBranchDescentExactness}
\begin{proof}
Read the branch surface first. \autoref{thm:authorized-generator-recursor-branch-exhaustion} and \autoref{thm:authorized-generator-recursor-root-branch-package-coverage} force every public branch read to stay on the accepted constructor list and the displayed branch package.

Now attach descent locality. By \autoref{thm:authorized-generator-recursor-structural-descent-locality}, \autoref{thm:authorized-generator-recursor-branch-descent-totality}, and \autoref{thm:authorized-generator-recursor-root-descent-audit-coupling}, a recursive read is public only when its descent evidence is recorded in $D$ together with the accepted signature, emitted eliminator, motive, and branch row.

\end{proof}

\begin{corollary}[AuthorizedGeneratorRecursor terminal generator strict obstruction]
\label{cor:authorized-generator-recursor-terminal-generator-strict-obstruction}
Let
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N)
$$
be an accepted $\AuthorizedGeneratorRecursorUp$ packet. Suppose a displayed failure, refusal, separation, or countermodel row is cited against a named terminal-generator converse or reuse route in the terminal generator obligation surface of $R$. Then the row is a strict local obstruction: it blocks only the named route whose consumer-facing use tries to read, omit, detach, replace, or bypass one of the displayed packet coordinates
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
Every permitted terminal-generator use of the row factors through the same route block $I,E,M,B,D,O,A$, the componentwise transport row $H$, the displayed continuation readback row $C$, the provenance row $P$, the ground-boundary row $G$, and the local $\NameCert_{\AuthorizedGeneratorRecursorUp}$ row $N$. The same displayed row cannot be reinterpreted as a global impossibility theorem, a host theorem about generators, hidden external validation, or an obstruction to any terminal consumer route that still remains inside those displayed rows.
\end{corollary}
\begin{proof}
Start with terminal generator admission, \autoref{thm:authorized-generator-recursor-terminal-generator-admission}. It admits every terminal generator-facing consumer route only through the finite packet rows
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
It also states that terminal normalization, closed-substitution, and generator-audit consumers may read $O$ only as a row of this finite carrier and never as detached generator authority. Hence a consumer-facing use of the cited failure, refusal, separation, or countermodel row has no initial terminal-generator coordinate outside the displayed packet.

Now apply terminal audit $\NameCert$ nonescape, \autoref{thm:authorized-generator-recursor-terminal-audit-namecert-nonescape}. Terminal output cannot be consumed apart from the audit row $A$, the ground-boundary row $G$, and the local $\NameCert$ row $N$ carried by the same packet. Thus any terminal use that speaks about $O$ must retain the route rows that carry the output and audit, the transport and continuation rows $H,C$, the provenance row $P$, and the boundary-certificate rows $G,N$.

Consider the named converse or reuse route against which the displayed row is cited. If that route tries to read $O$ before the route block $I,E,M,B,D,O,A$ is present, or tries to omit the transport row $H$, the continuation readback row $C$, the provenance row $P$, the ground-boundary row $G$, or the local certificate row $N$, the two cited theorems locate the obstruction at that missing displayed coordinate. Therefore the row blocks precisely that named terminal-generator route, namely the route whose consumer-facing use fails to remain inside the admitted finite carrier and the audit-certificate nonescape route.

It remains to exclude stronger readings. A global impossibility theorem would require a conclusion not indexed by the accepted packet $R$ and not confined to the named failed route. A host theorem about generators would require a host-facing generator coordinate outside $I,E,M,B,D,O,A,H,C,P,G,N$. Hidden external validation would require an additional validation or certificate row beyond $A,G,N$ and the same displayed packet rows that carry them. Terminal generator admission exposes no such coordinate, and terminal audit $\NameCert$ nonescape prevents $O$ from being read apart from the displayed audit, boundary, provenance, transport, continuation, and local naming data. Hence the obstruction is strict and local to the named terminal-generator converse or reuse route.
\end{proof}

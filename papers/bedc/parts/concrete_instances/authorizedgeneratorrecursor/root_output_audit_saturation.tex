\begin{theorem}[AuthorizedGeneratorRecursor root branch audit coupling]
\label{thm:authorized-generator-recursor-root-branch-audit-coupling}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet with displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N),
$$
every public read of the output row $O$ couples the branch package $B$ to the audit row $A$ through the same signature, eliminator, motive, descent, transport, continuation, package, boundary, and local-name rows. No downstream recursor or generator-audit consumer can read $O$ from a branch package that is detached from the audit ledger.
\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorRootBranchAuditCoupling}
\begin{proof}
Project the displayed carrier $R$. The branch row $B$ and audit row $A$ occur as coordinates of the same finite packet, with $I,E,M,D,O,H,C,P,G,N$ recording the surrounding admission, output, transport, route, provenance, boundary, and local naming rows.

Apply \autoref{thm:authorized-generator-recursor-root-branch-package-coverage}. It forces every branch exposed to a consumer to pass through the accepted signature, emitted eliminator, checked motive, branch package, and structural-descent rows $I,E,M,B,D$ before $O$ is public.

Now apply \autoref{thm:authorized-generator-recursor-root-audit-ledger-exactness}. That theorem attaches public output reads to $O,A,H,C,P,G,N$ after branch and descent admission. The TasteGate marker \texttt{BEDC.Derived.AuthorizedGeneratorRecursorUp.taste\_gate} is used only as the carrier alignment cited by the chapter, not as a replacement for this branch-audit obligation. Thus every public branch read remains coupled to the audit ledger of $R$.
\end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor root descent output factorization]
\label{thm:authorized-generator-recursor-root-descent-output-factorization}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet with displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N),
$$
any output-facing recursive read factors through the checked motive row $M$ and structural-descent row $D$ before it reaches $O$, and then through $A,H,C,P,G,N$ before any downstream closed-normal or generator-audit consumer can cite the output. The factorization has no hidden proof generator, non-structural recursion channel, external axiom row, or off-packet naming certificate.
\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorRootDescentOutputFactorization}
\begin{proof}
Read $M,D,O$ as coordinates of the displayed packet $R$. The packet contains no second motive row, no detached descent row, and no output coordinate outside the same finite carrier.

Apply \autoref{thm:authorized-generator-recursor-root-motive-descent-stability}. It keeps structural descent through $D$ tied to the checked motive row $M$ before output exposure, and it blocks a motive-free descent surface.

Then apply \autoref{thm:authorized-generator-recursor-root-audit-ledger-exactness}. The output read continues through $A,H,C,P,G,N$ after branch and descent admission, so the public consumer receives an audited output route rather than a hidden generator route. The TasteGate marker \texttt{BEDC.Derived.AuthorizedGeneratorRecursorUp.taste\_gate} remains only the aligned-carrier witness; the paper-side descent and output factorization is carried by the cited root theorems.
\end{proof}

Together they force branch coverage and descent stability to remain inside $I,E,M,B,D$ before public output is read.

Use \autoref{thm:authorized-generator-recursor-root-audit-ledger-exactness} for the output side. It exhausts public output reads by $O,A,H,C,P,G,N$ after branch and descent admission. The TasteGate marker \texttt{BEDC.Derived.AuthorizedGeneratorRecursorUp.taste\_gate} only aligns this finite carrier with the chapter handoff, so it does not supply an additional boundary or ledger row. \end{proof}

\begin{theorem}[AuthorizedGeneratorRecursor root branch-audit downstream coverage]
\label{thm:authorized-generator-recursor-root-branch-audit-downstream-coverage}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet with displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N),
$$
every accepted local downstream consumer of the root branch-audit output is exhausted by the finite rows displayed in this carrier. More precisely, the consumer can read only a branch-admission alternative in $I,E,M,B,D$, an output-audit ledger row in $O,A,H,C,P,G,N$, or the boundary coverage row obtained by keeping these two surfaces coupled inside the same packet. Any candidate consumer that asks for a detached branch package, unaudited output route, hidden generator, external axiom channel, host-kernel completeness claim, or off-packet naming certificate remains outside the accepted $\BEDC$ packet.
\end{theorem}
\begin{proof}
Fix an accepted root packet and project the displayed carrier
$$
R=(I,E,M,B,D,O,A,H,C,P,G,N).
$$
The branch side of any local consumer is finite because the exposed branch package is not a free generator surface. By \autoref{thm:authorized-generator-recursor-root-branch-audit-coupling}, every public read of $O$ couples the branch package $B$ to the audit row $A$ through the same signature, eliminator, motive, descent, transport, continuation, package, boundary, and local-name rows. Hence a local consumer cannot first read a branch alternative and then detach that alternative from the audit ledger of $R$.

The downstream side is total because the boundary ledger has no further public coordinates. By \autoref{thm:authorized-generator-recursor-root-boundary-ledger-totality}, the output boundary visible to recursor, closed-normal, generator-audit, and TasteGate consumers is exhausted by
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
Thus any accepted consumer that reaches the output must land either on the admission rows $I,E,M,B,D$ before $O$, or on the audit, transport, continuation, package, boundary, and local-name rows $O,A,H,C,P,G,N$ after $O$.

It remains only to exclude apparent alternatives. A detached branch package is rejected by the coupling theorem, since no downstream recursor or generator-audit consumer can read $O$ from a branch package separated from $A$. An unaudited output route, hidden generator, external axiom channel, host-kernel completeness claim, or off-packet naming certificate is rejected by the boundary-ledger totality theorem, since none of those objects is a coordinate of the displayed carrier. Therefore the accepted local downstream coverage is exactly the finite branch alternatives, ledgers, and coverage rows already displayed in $R$, and every excluded alternative remains outside the accepted $\BEDC$ packet.
\end{proof}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorRootBranchAuditDownstreamCoverage}

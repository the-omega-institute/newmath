\begin{theorem}[Authorized generator recursor root-output totality lock]
\label{thm:authorized-generator-recursor-root-output-totality-lock}
Let $R=(I,E,M,B,D,O,A,H,C,P,G,N)$ be an accepted $\AuthorizedGeneratorRecursorUp$ carrier. Every public exposure of the root output row $O$ factors through the same displayed packet: branch exhaustion in $B$, structural descent in $D$, audit provenance in $A$, componentwise transport in $H$, displayed $\Cont$ routes in $C$, $\Pkg$ provenance in $P$, the $\mathsf{GAP}$ boundary in $G$, and the local $\NameCert$ row $N$.

The checked TasteGate marker
$\mathsf{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorTasteGate\_single\_carrier\_alignment}$
is used only as the single-carrier alignment witness for this packet. It does not expose host induction, an external axiom route, an unaudited dependency route, off-packet substitution, or a second output certificate.
\end{theorem}
\begin{proof}
Start from the carrier definition, \autoref{def:authorized-generator-recursor-carrier}. It displays the root packet $R$ and leaves no output coordinate outside $I,E,M,B,D,O,A,H,C,P,G,N$.

Apply branch exhaustion and structural descent locality, \autoref{thm:authorized-generator-recursor-branch-exhaustion} and \autoref{thm:authorized-generator-recursor-structural-descent-locality}. These force every recursive output read to retain the constructor signature, branch package, and descent row before $O$ can be public.

Next attach audit and route provenance. Theorems~\ref{thm:authorized-generator-recursor-audit-boundary} and~\ref{thm:authorized-generator-recursor-audit-provenance-factorization} keep $A,H,C,P,G,N$ attached to the output exposure.

Finally use the ground-compiler handoff, \autoref{thm:authorized-generator-recursor-ground-compiler-handoff}. The TasteGate marker is read only as the aligned-carrier witness, so no hidden generator, host induction principle, external axiom, unaudited route, or off-packet substitution can replace the displayed root-output packet.
\end{proof}

Use the root-output audit totality theorem, \autoref{thm:authorized-generator-recursor-root-output-audit-totality}. It places every public output read behind the branch, descent, audit, route, package, boundary, and local naming rows.

The audit provenance factorization, \autoref{thm:authorized-generator-recursor-audit-provenance-factorization}, then records that the output row becomes public only after the audit provenance and route rows remain attached.

Finally apply the root boundary nonescape and ground-compiler handoff, \autoref{thm:authorized-generator-recursor-root-boundary-nonescape} and \autoref{thm:authorized-generator-recursor-ground-compiler-handoff}. \begin{lemma}[Authorized generator recursor root-output totality downstream coverage]
\label{lem:authorized-generator-recursor-root-output-totality-downstream-coverage}
Let $R=(I,E,M,B,D,O,A,H,C,P,G,N)$ be an accepted $\AuthorizedGeneratorRecursorUp$ carrier. Every accepted local downstream consumer that reads the root output row $O$ is exhausted by the finite surface displayed in the root-output packet: constructor and branch alternatives are read through $I,E,M,B,D$, audit and output ledgers are read through $O,A,H,C,P$, and boundary, coverage, and local naming rows are read through $G,N$ together with the displayed $\Cont$ route $C$ and componentwise transport row $H$.

Consequently an unlisted constructor alternative, detached branch table, unchecked motive, non-structural recursion channel, hidden generator, host induction principle, external axiom route, unaudited dependency route, off-packet substitution authority, second output certificate, or naming certificate outside $R$ remains outside the BEDC packet exposed to that consumer.
\end{lemma}
\begin{proof}
Start with the root-output totality lock, \autoref{thm:authorized-generator-recursor-root-output-totality-lock}. It says that every public exposure of $O$ factors through the displayed packet: branch exhaustion in $B$, structural descent in $D$, audit provenance in $A$, componentwise transport in $H$, displayed $\Cont$ routes in $C$, $\Pkg$ provenance in $P$, the $\mathsf{GAP}$ boundary in $G$, and the local $\NameCert$ row $N$. Since the carrier itself is the finite tuple $R=(I,E,M,B,D,O,A,H,C,P,G,N)$ of \autoref{def:authorized-generator-recursor-carrier}, this first step leaves no output coordinate from which a downstream consumer could obtain a second route to $O$.

Now apply root audit route exactness, \autoref{thm:authorized-generator-recursor-root-audit-route-exactness}. Any audit-facing consumer that reads $O$ must also read the branch-exhaustion row $B$, structural-descent row $D$, audit-provenance row $A$, displayed $\Cont$ route $C$, $\Pkg$ row $P$, $\mathsf{GAP}$ boundary $G$, and local $\NameCert$ row $N$ from the same packet. Thus the consumer-facing use of $O$ is not merely named by the output row; it is forced to carry the route, transport, package, boundary, and local naming rows that the totality lock lists.

These two constraints exhaust the downstream cases. A constructor or recursive-call demand is covered by the admitted signature, emitted eliminator, motive, branch package, and descent row $I,E,M,B,D$. An output-ledger or audit demand is covered by $O,A,H,C,P$. A boundary, refusal, or local-readback demand is covered by $G,N$ together with the same displayed transport and continuation rows. Any proposed consumer fact outside those alternatives would need either a public exposure of $O$ not factored by the totality lock or an audit-facing route not exact in the sense of \autoref{thm:authorized-generator-recursor-root-audit-route-exactness}. Both possibilities are excluded, so all unlisted alternatives remain outside the BEDC packet.
\end{proof}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorRootOutputTotalityDownstreamCoverage}

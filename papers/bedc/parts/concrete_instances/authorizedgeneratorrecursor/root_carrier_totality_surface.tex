\begin{corollary}[AuthorizedGeneratorRecursor root-totality strict obstruction]
\label{cor:authorized-generator-recursor-root-totality-strict-obstruction}
Let
$$
Q=(U,J,K,O,L,H,C,P,N)
$$
be an accepted $\AuthorizedGeneratorRecursorUp$ readiness route, read from the root carrier rows
$$
I,E,M,B,D,O,A,H,C,P,G,N.
$$
Suppose a displayed failure, refusal, separation, or countermodel row is cited against a named root-totality converse or reuse route for $Q$. Then that row is a strict local obstruction: it blocks only the named route whose consumer-facing read omits, replaces, detaches, or bypasses one of the displayed output, source, transport, replay, provenance, refusal, boundary, or local naming coordinates
$$
I,E,M,B,D,O,A,H,C,P,G,N
$$
or their readiness packaging
$$
U,J,K,O,L,H,C,P,N.
$$
In particular, the obstruction has no force as a global impossibility theorem, no force as a host theorem about recursors or totality, and no force as hidden external validation for any coordinate not displayed by the accepted packet.
\end{corollary}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorReadinessRoute\_root\_totality\_strict\_obstruction}
\begin{proof}
Start with terminal output non-escape, \autoref{thm:authorized-generator-recursor-terminal-output-nonescape}. It lists the only rows through which terminal normalization, closed-substitution, and ground-compiler consumers may read the public output $O$: the authorization row $U$, refusal ledger $L$, ground-compiler recognition row $K$, componentwise $\hsame$ transport $H$, displayed $\Cont$ replay $C$, $\Pkg$ provenance $P$, and local $\NameCert$ row $N$. The same theorem excludes hidden generator state, host evaluation, unchecked recursion, external axioms, detached proof production, and off-packet substitution as output coordinates.

Now attach closed-substitution source exposure, \autoref{thm:authorized-generator-recursor-closed-substitution-source-exposure}. A closed-substitution consumer must preserve the source rows that authorize the output: $I,E,M,B,D,A,G,H,C,P,L,N$. Thus any consumer-facing use of the displayed obstruction factors through the source rows $I,E,M,B,D,A,G$, the refusal ledger $L$, the transport and readback rows $H,C,P$, and the local naming row $N$ before it can speak about the output row $O$.

Consider the cited failure, refusal, separation, or countermodel row. If the named converse or reuse route asks to read $O$ while omitting the authorization row $U$, the refusal ledger $L$, the ground-recognition row $K$, the source rows $I,E,M,B,D,A,G$, or the transport and readback rows $H,C,P,N$, the two cited theorems place the failure exactly at that missing coordinate. The row therefore blocks the named route at the point where the route leaves the displayed packet. If a consumer instead keeps all listed rows, this corollary supplies no rejection of that consumer; the obstruction is tied to the named omitted, replaced, detached, or bypassed coordinate.

It remains to rule out stronger readings. A global impossibility theorem would require a conclusion not indexed by the named failed route and not confined to $Q$. A host theorem about recursors or totality would require a host evaluator, host induction rule, unchecked recursive call, or external axiom, each of which is excluded by terminal output non-escape. Hidden external validation would require a certificate or substitution channel outside $H,C,P,L,N$, which closed-substitution source exposure does not export. Since no such coordinate is present in the root carrier or in the readiness packaging, the displayed row remains a strict local obstruction to the named root-totality converse or reuse route only.
\end{proof}

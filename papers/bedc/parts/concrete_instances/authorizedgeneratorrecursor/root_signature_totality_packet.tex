\label{thm:authorized-generator-recursor-root-signature-totality}

$$

$$

the accepted signature is total over the displayed constructor row $I$, emitted recursor row $E$, motive row $M$, branch package row $B$, structural descent row $D$, output row $O$, audit row $A$, componentwise $\hsame$ row $H$, displayed $\Cont$ row $C$, $\Pkg$ row $P$, boundary row $G$, and local $\NameCert$ row $N$. \end{theorem}
\begin{proof}
Start with carrier admission and acceptance totality. They expose the finite packet $R=(I,E,M,B,D,O,A,H,C,P,G,N)$ and require every accepted signature read to pass through the same displayed constructor, recursor, motive, branch, descent, output, audit, transport, continuation, package, boundary, and local naming rows.

Next use the finite route exposure and root authorization obligations. The generator route is not a hidden host eliminator; it is the accepted signature carried by $I$, checked by the local NameCert surface, and replayed through the displayed $\Cont$ and $\Pkg$ rows.

Finally attach the ground compiler authorization. Since the compiler-facing route reads only the displayed packet, no unchecked generator row or unregistered recursion principle can supply a second signature source. The accepted signature is therefore total over the displayed root rows.
\end{proof}

Next use root host boundary NameCert exhaustion together with terminal audit NameCert non-escape. The output-facing route remains visible only through $A,H,C,P,G,N$ and the displayed rows that carry them.

\end{proof}

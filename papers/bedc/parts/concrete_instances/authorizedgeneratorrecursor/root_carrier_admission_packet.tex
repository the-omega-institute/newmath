\label{thm:authorized-generator-recursor-root-carrier-admission}

$$

\end{theorem}
\leanchecked{BEDC.Derived.AuthorizedGeneratorRecursorUp.AuthorizedGeneratorRecursorRootCarrierAdmission}
\begin{proof}

Theorems~\ref{thm:authorized-generator-recursor-branch-exhaustion} and~\ref{thm:authorized-generator-recursor-structural-descent-locality} keep every public recursive read on the signature, emitted eliminator, motive, branch package, and structural descent rows.

Finally attach audit and handoff data. The audit boundary of \autoref{thm:authorized-generator-recursor-audit-boundary} keeps output exposure behind audit provenance, and \autoref{thm:authorized-generator-recursor-ground-compiler-handoff} reads the checked carrier alignment only as the same finite packet. Hence the accepted root carrier exposes no rows beyond the displayed list.
\end{proof}

\begin{lemma}[AuthorizedGeneratorRecursor root admission downstream coverage]
\label{lem:authorized-generator-recursor-root-admission-downstream-coverage}
For every accepted $\AuthorizedGeneratorRecursorUp$ root packet, every local downstream consumer admitted by the root carrier is covered by the finite consumer interface of \autoref{thm:authorized-generator-recursor-ground-compiler-route-totality}. More explicitly, a consumer-facing use is exhausted by one of the displayed roles: the accepted signature row $I$, emitted eliminator row $E$, motive row $M$, branch package row $B$, structural descent row $D$, output row $O$, audit row $A$, componentwise transport row $H$, displayed continuation row $C$, package provenance row $P$, boundary row $G$, or local $\NameCert$ row $N$. An unlisted constructor alternative, detached branch source, non-structural recursion channel, unaudited ledger, detached output name, or external compiler certificate is therefore not an accepted coordinate of the $\AuthorizedGeneratorRecursorUp$ BEDC packet.
\end{lemma}
\begin{proof}
Start with root carrier admission, \autoref{thm:authorized-generator-recursor-root-carrier-admission}. It presents the accepted root carrier as the displayed packet of signature, eliminator, motive, branch, descent, output, audit, transport, continuation, provenance, boundary, and local certificate rows, and it gives no additional coordinate for an off-packet generator, hidden branch source, or detached output name.

Now apply ground-compiler route totality, \autoref{thm:authorized-generator-recursor-ground-compiler-route-totality}. Any ground-compiler or $\TasteGate$-facing route out of the accepted packet factors through exactly those named roles. The computational alternatives are therefore exhausted by the accepted signature row $I$, emitted eliminator row $E$, motive row $M$, branch package row $B$, and structural descent row $D$; a purported constructor case or recursive branch not admitted by those rows has no consumer-facing route.

It remains to account for ledger, readback, and output use. The same totality theorem keeps public output and readback on $O,A,H,C,P,G,N$: the output row $O$ is read only together with audit row $A$, componentwise transport row $H$, displayed continuation row $C$, package provenance row $P$, boundary row $G$, and local $\NameCert$ row $N$. Thus every accepted local consumer factors through the finite alternatives, ledger rows, and coverage rows already displayed by the root carrier admission theorem. Since the admitted carrier has no coordinate beyond those roles, every excluded alternative remains outside the BEDC packet rather than becoming a hidden consumer route.
\end{proof}

\chapter{A Concrete Naming Certificate for $\FiniteKernelCategoryUp$}
\label{ch:concrete-instances-finitekernelcategory-namecert}

The $\FiniteKernelCategoryUp$ packet records the category-shaped reading of the finite BEDC kernel as a concrete naming certificate. Its objects are finite object histories, its arrows are displayed continuation histories, its identity and composition rows are finite $\Cont$ ledgers, and its equality rows are $\hsame$ comparisons on the same displayed histories. The packet is native to $\BHist$ and does not import an external category library, quotient morphism type, or host equality of arrows.

It is the finite, displayed-history specialization of the broader kernel-category surface in \autoref{ch:concrete-instances-kernelcategory-namecert}, and it also consumes the displayed continuation-composition route of \autoref{ch:concrete-instances-unary-continuation-category-namecert}. Its object and hom readings come from the kernel-category packet, while the identity, composition, associativity, and unit ledgers repackage those rows through finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and local $\NameCert$ evidence rather than importing an external category library.

\begin{definition}[Finite-kernel category carrier]
\label{def:finite-kernel-category-carrier}
A $\FiniteKernelCategoryUp$ carrier is a finite packet
$$
K=(O,M,I,Q,A,U,H,C,P,N)
$$
with the following visible rows:
\begin{enumerate}
\item $O$ is the finite object-history row: every object is a displayed $\BHist$ object history accepted by the kernel classifier.
\item $M$ is the displayed hom-history row: a morphism from object history $a$ to object history $b$ is a finite history $f$ equipped with the displayed continuation ledger relating the endpoint histories.
\item $I$ records identity rows, one for each accepted object history, using the empty continuation ledger from the object to itself.
\item $Q$ records composition rows: for displayed hom histories $f:a\to b$ and $g:b\to c$, the composite history is accepted only through the finite $\Cont$ ledger joining the two displayed continuations.
\item $A$ records associativity ledgers comparing the two displayed threefold composites by $\hsame$ after their $\Cont$ rows are expanded.
\item $U$ records left and right unit ledgers comparing empty-continuation composites with the original displayed hom history.
\item $H$ stores the componentwise $\hsame$ transports for object histories, hom histories, identity rows, composition rows, associativity rows, and unit rows.
\item $C$ stores the $\Cont$ routes by which a consumer reads identities, composites, associativity witnesses, and unit witnesses.
\item $P$ is the $\Pkg$ provenance row naming the consumed $\BHist$, $\hsame$, $\Cont$, identity, composition, associativity, and unit ledgers.
\item $N$ is the local $\NameCert_{\FiniteKernelCategoryUp}$ registration row.
\end{enumerate}
The local classifier compares carriers only through these displayed rows. It has no coordinate for a global category universe, quotient morphisms, external functor categories, or host equality of arrows.
\end{definition}

\begin{theorem}[Finite-kernel category NameCert obligations]
\label{thm:finite-kernel-category-namecert-obligations}
The local $\NameCert_{\FiniteKernelCategoryUp}$ obligation surface consists of five finite rows:
\begin{enumerate}
\item carrier habitation by the empty object history and by unary object histories with their empty-continuation identity rows;
\item classifier stability for object histories, displayed hom histories, identity ledgers, composition ledgers, associativity ledgers, and unit ledgers by componentwise $\hsame$;
\item identity exactness, requiring each identity row to be the empty $\Cont$ continuation attached to the displayed object history;
\item composition exactness, requiring every public composite to factor through the displayed $\Cont$ row named in $Q$;
\item ledger non-escape, requiring all public category reads to pass through $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$, excluding external category libraries and quotient morphisms.
\end{enumerate}
\end{theorem}
\begin{proof}
Carrier habitation uses the empty object history and the unary object histories already accepted by the kernel. The empty continuation gives the identity row, so the object and identity coordinates are inhabited inside the finite carrier.

Classifier stability is componentwise. Transport object histories and hom histories by the displayed $\hsame$ rows, then transport the identity, composition, associativity, and unit ledgers along the same endpoint comparisons.

Identity and composition exactness are projections of the carrier. The identity coordinate is $I$, while each composite is read through the $\Cont$ row stored in $Q$ before it can be consumed as a displayed hom history.

The non-escape row follows by inspecting $K$. Since every public route is stored in $H$, $C$, $P$, and $N$, a consumer cannot read an external category library, quotient morphism, or host arrow equality from this certificate.
\end{proof}

\begin{theorem}[Finite-kernel category object and hom boundary]
\label{thm:finite-kernel-category-object-hom-boundary}
Let
$$
K=(O,M,I,Q,A,U,H,C,P,N)
$$
be an accepted $\FiniteKernelCategoryUp$ carrier. Every accepted object read is exposed only through the object-history row $O$, and every accepted hom read is exposed only through the hom-history row $M$. Endpoint compatibility for those hom reads is guarded by the displayed $\BHist$ histories, the componentwise $\hsame$ rows in $H$, and the $\Cont$ rows in $C$.
\end{theorem}
\begin{proof}
Project the carrier $K=(O,M,I,Q,A,U,H,C,P,N)$. The carrier definition makes $O$ the only object-history coordinate and $M$ the only hom-history coordinate.

Apply \autoref{thm:finite-kernel-category-namecert-obligations}. Carrier exposure and classifier stability transport object histories and hom histories by the displayed $\hsame$ rows, while composition exactness and ledger non-escape keep every endpoint read inside the visible $\BHist$ and $\Cont$ ledgers.

Finally repack the result as a boundary statement for consumers. Since no quotient morphism type or host arrow equality occurs in $K$, object and hom reads cannot escape the displayed $O,M,H,C,P,N$ surface.
\end{proof}

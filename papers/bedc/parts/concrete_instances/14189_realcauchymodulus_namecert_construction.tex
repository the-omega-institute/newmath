\begin{theorem}[Real Cauchy modulus NameCert obligations]
\label{thm:real-cauchy-modulus-namecert-obligations}

\begin{proof}

Read the analytic rows in order. The precision request first consumes the threshold row $M$, then the finite windows $S$, then the dyadic ledger $D$, and only then the regular readback $Q$ and terminal seal $E$.

The structural rows do not add an analytic source. They transport, replay, source, and name the same route, while the excluded objects have no coordinate in the carrier. Hence the listed rows exhaust the seed obligation surface.

\label{thm:real-cauchy-modulus-window-modulus-route}

$$

$$
The consumer receives no Real completeness theorem, quotient stream equality, host real equality, countable-choice row, trichotomy row, selected-limit row, open-cover compactness row, hidden selector, or detached Real seal.

\begin{proof}

The regular readback is then forced through the same finite window. The row $Q$ is read only after $M,S,D$ are displayed, so no quotient stream or host equality row can substitute for the scheduled window comparison.

Finally read the Real seal coordinate. The row $E$ is reached after $Q$, and the rows $H,C,P,N$ only preserve and name that route. This exhausts the advertised finite-window path for the precision request.

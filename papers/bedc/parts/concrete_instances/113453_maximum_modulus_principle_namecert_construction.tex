Following Ahlfors and Conway on the maximum-modulus principle for holomorphic functions, the $\MaximumModulusPrincipleUp$ packet records the boundary-maximum route as finite NameCert data. It consumes the complex carrier of \autoref{ch:concrete-instances-complex-namecert}, the holomorphic source rows of \autoref{ch:concrete-instances-holomorphic-namecert}, the circle-boundary surface of \autoref{ch:concrete-instances-s1-namecert}, the terminal real-name seal of \autoref{ch:concrete-instances-real-namecert}, and the three-circle annulus ledger of \autoref{ch:concrete-instances-hadamard-three-circle-namecert}. The packet names a displayed boundary upper-bound certificate; it is not an ambient supremum theorem, a selected maximizer, analytic continuation beyond the named domain, quotient function equality, or a bridge certificate.

\begin{definition}[Maximum-modulus principle carrier]
\label{def:maximum-modulus-principle-carrier}
A $\MaximumModulusPrincipleUp$ carrier is a finite $\BHist$ packet
$$
M=(D,F,B,R,U,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $D$ is the named complex domain or annulus row, read through $\ComplexUp$ and the relevant boundary carrier.
\item $F$ is the $\HolomorphicUp$ map row over $D$, with values in the same $\ComplexUp$ carrier.
\item $B$ is the finite boundary-read ledger, using $\SOneUp$ or an equivalent displayed boundary source to expose the inspected boundary probes.
\item $R$ is the $\RealUp$ modulus-read row for the boundary and interior comparison values.
\item $U$ is the boundary upper-bound row, recording the displayed maximum-modulus inequality available to downstream consumers.
\item $H$ stores componentwise $\hsame$ transport for the domain, holomorphic map, boundary reads, modulus reads, upper-bound row, replay, provenance, and local naming rows.
\item $C$ stores displayed $\Cont$ replay from a maximum-modulus request through $D,F,B,R,U,H$.
\item $P$ is the $\Pkg$ provenance over $\MaximumModulusPrincipleUp$, $\ComplexUp$, $\HolomorphicUp$, $\SOneUp$, $\RealUp$, $\HadamardThreeCircleUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$ dependency rows.
\item $N$ is the local $\NameCert_{\MaximumModulusPrincipleUp}$ row.
\end{enumerate}
The classifier compares accepted carriers only through these rows. No coordinate stores a host compactness theorem, selected boundary point, ambient supremum, quotient function equality, analytic continuation outside $D$, or countable-choice schedule.
\end{definition}

\begin{theorem}[Maximum-modulus principle NameCert obligations]
\label{thm:maximum-modulus-principle-namecert-obligations}
For every accepted $\MaximumModulusPrincipleUp$ carrier $M=(D,F,B,R,U,H,C,P,N)$, the local $\NameCert_{\MaximumModulusPrincipleUp}$ surface consists exactly of domain admission, holomorphic-map exposure, boundary-read visibility, real modulus readback, boundary upper-bound compatibility, componentwise transport, displayed replay, package provenance, and local naming. A consumer may not read a maximum-modulus bound until these rows are displayed, and the certificate does not export an ambient supremum theorem, selected maximizer, quotient function equality, analytic continuation, or countable choice.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:maximum-modulus-principle-carrier}. The domain row $D$ and holomorphic row $F$ fix the complex source before any boundary comparison is available.

The boundary and modulus data are then read in order. The row $B$ exposes only the finite boundary probes, while $R$ turns the inspected values into $\RealUp$ modulus rows. The upper-bound row $U$ is therefore attached to the displayed boundary ledger rather than to an external supremum.

Finally $H,C,P,N$ transport, replay, source, and name the same finite route. Since none of these rows adds a selected boundary point, host compactness proof, analytic continuation coordinate, quotient equality, or countable-choice schedule, the listed obligations exhaust the NameCert surface.
\end{proof}

\begin{theorem}[Maximum-modulus principle Hadamard boundary handoff]
\label{thm:maximum-modulus-principle-hadamard-boundary-handoff}
Every $\HadamardThreeCircleUp$ or holomorphic-boundary consumer that reads an accepted $\MaximumModulusPrincipleUp$ packet receives the displayed route
$$
\ComplexUp \longmapsto \HolomorphicUp \longmapsto \SOneUp \longmapsto \RealUp \longmapsto \MaximumModulusPrincipleUp
$$
together with the rows $D,F,B,R,U,H,C,P,N$. When the consumer is an annulus packet, the boundary read may be handed to \autoref{thm:hadamard-three-circle-annulus-maximum-handoff}; the handoff preserves the same finite boundary ledger and does not add selected maximizers, host analytic continuation, quotient equality, or an ambient maximum theorem.
\end{theorem}
\begin{proof}
Start with the holomorphic source rows $D$ and $F$. They determine the complex carrier and displayed map row that any boundary consumer is allowed to inspect.

Next read the boundary ledger $B$ and the modulus row $R$. These rows expose the finite boundary probes and real modulus comparisons before $U$ can be consumed as an upper-bound row.

For an annulus consumer, use \autoref{thm:hadamard-three-circle-annulus-maximum-handoff}. The Hadamard packet receives exactly the displayed boundary maximum row already attached to $B,R,U$, so the three-circle ledger is a downstream read of the same finite data.

The structural rows $H,C,P,N$ keep the handoff inside the NameCert route. No row supplies a selected extremal point, an ambient supremum proof, analytic continuation beyond the named source, quotient equality, or countable choice.
\end{proof}

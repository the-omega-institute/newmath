The $\FiniteTraceEvaluatorUp$ packet realizes the runtime-candidate surface of \autoref{def:reflection-runtime-candidate} inside the reflection-limit boundary of \autoref{ch:visions-reflection-and-limits}. It sits beside the runtime-candidate boundary packet of \autoref{ch:concrete-instances-runtimecandidateboundary-namecert}, the diagonal finite-trace packet of \autoref{ch:concrete-instances-haltingdistinctiontrace-namecert}, and the finite-trace GAP socket of \autoref{ch:concrete-instances-finite-trace-gap-socket-namecert}. Its bounded evaluator trace is consumed by the runtime checkpoint packet of \autoref{ch:concrete-instances-reflection-runtime-checkpoint-namecert}, where checkpoint-state and validation-boundary rows read the finite trace evidence without importing host validation.

\begin{definition}[Finite trace evaluator carrier]
\label{def:finite-trace-evaluator-carrier}
A $\FiniteTraceEvaluatorUp$ carrier is a finite $\BHist$ packet
$$
E=(I,T,A,V,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $I$ is the input row of coded $\mathrm{CIC}^{*}$ expressions, proof objects, and states admitted to the bounded evaluator;
\item $T$ is the finite trace-step row, listing the displayed evaluator steps from the input row $I$ without adding an unbounded runtime;
\item $A$ is the accepted-result classifier row, grouping exactly the results reached by the displayed trace-step row $T$;
\item $V$ is the validation-boundary row, recording that host validation of the evaluator is not a packet coordinate;
\item $H$ records componentwise $\hsame$ transports for the input, trace-step, accepted-result, validation-boundary, provenance, and local naming rows;
\item $C$ records the displayed $\Cont$ routes by which a consumer reads $I$, replays $T$, reaches $A$, checks the boundary row $V$, and returns through provenance and local naming;
\item $P$ is the $\Pkg$ provenance over $\BHist$, $\hsame$, $\Cont$, $\Pkg$, $\NameCert$, the structural $\mathrm{CIC}^{*}$ coding interface, and \autoref{ch:concrete-instances-runtimecandidateboundary-namecert};
\item $N$ is the local $\NameCert_{\FiniteTraceEvaluatorUp}$ row.
\end{enumerate}
The classifier accepts exactly packets whose visible coordinates are $I,T,A,V,H,C,P,N$. The carrier has no coordinate for a host evaluator, a total truth predicate, an unbounded execution stream, or the judgment that the bounded evaluator validates itself.
\end{definition}

\label{thm:finite-trace-evaluator-namecert-obligations}

Finally, $P$ and $N$ bind provenance and local naming without adding new evaluator content. Ledger exhaustion follows because every available consumer route has already passed through $I,T,A,V,H,C,P,N$, so an unlisted evaluator, truth predicate, total halting predicate, or infinite trace would require an extra coordinate absent from the carrier.

\begin{theorem}[FiniteTraceEvaluator nontrivial carrier pair]
\label{thm:finite-trace-evaluator-nontrivial-carrier-pair}
The $\FiniteTraceEvaluatorUp$ carrier has at least two distinguishable packet
shapes. One admitted shape exposes an input-only finite trace packet with rows
$I,T,H,C,P,N$ and no accepted-result row. A second admitted shape exposes the
same finite input and trace surface together with a displayed accepted-result
row $A$ and validation-boundary row $V$. The local classifier can therefore
distinguish an input-only trace packet from a packet whose finite trace carries
an accepted-result row.
\end{theorem}
\begin{proof}
Start from the carrier row of \autoref{def:finite-trace-evaluator-carrier}.
The row $I$ supplies the coded input, $T$ supplies the finite trace-window
coordinate, and $H,C,P,N$ record transport, continuation, provenance, and the
local name.

For the input-only shape, read only $I,T,H,C,P,N$. This is still a finite trace
packet because \autoref{thm:finite-trace-evaluator-bounded-trace-coverage}
places trace evidence inside $T$ and does not require an accepted result to be
displayed by every finite trace read.

For the accepted-result shape, add the rows $A,V$. By
\autoref{thm:finite-trace-evaluator-result-classifier-exactness}, the row $A$
is an accepted-result classifier only after the finite trace has been read from
$I,T$, and by \autoref{thm:finite-trace-evaluator-validation-refusal-package}
the row $V$ records the validation boundary rather than a host validator.

The two shapes are not interchangeable as public rows. Removing $A,V$ loses
the accepted-result and validation-boundary route, while adding them creates a
displayed result row that the input-only packet does not expose. Hence the
carrier is not a one-shape packet.
\end{proof}

\begin{theorem}[FiniteTraceEvaluator field-faithful rows]

\begin{proof}

public coordinate beyond the coded input, finite trace, accepted-result

local name.

Next identify the four route-sensitive fields. Changing $I$ changes the coded
input route before any finite trace can be read. Changing $T$ changes the
finite trace-window route controlled by
\autoref{thm:finite-trace-evaluator-bounded-trace-coverage}.

Changing $A$ changes the accepted-result classifier route of
\autoref{thm:finite-trace-evaluator-result-classifier-exactness}. Changing
$V$ changes the validation-boundary route of
\autoref{thm:finite-trace-evaluator-validation-row-nonescape}.

The remaining rows $H,C,P,N$ transport and name those displayed routes. Thus
row equality fixes the public evaluator read, and a change in any of
$I,T,A,V$ is visible as a change in the corresponding route.
\end{proof}

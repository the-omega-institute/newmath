\label{thm:finite-trace-evaluator-ground-checkpoint-readback-totality}

$\ReflectionRuntimeCheckpointUp$ consumer is total along the displayed finite

$$

$$

$\hsame$ row $H$, replayed by the displayed $\Cont$ route $C$, packaged by

runtime acceptance predicate, total truth classifier, unbounded runtime, or
infinite trace.
\end{theorem}
\begin{proof}

the coded input $I$, finite trace $T$, accepted-result row $A$, validation
boundary $V$, componentwise transport $H$, continuation route $C$, provenance

Apply checkpoint-budget exhaustion and the reflection-runtime route. These
results force every checkpoint readback through the same finite
$\BHist/\hsame/\Cont/\Pkg/\NameCert$ packet and prevent the runtime consumer
from asking for any row outside the displayed route.

Finally use checkpoint-source locking. Since the source is already the finite
packet $I,T,A,V,H,C,P,N$, the readback is total exactly on that packet and has
no second runtime source from which to import host validation, total truth,
unbounded evaluation, or an infinite trace.
\end{proof}

$$

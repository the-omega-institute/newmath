\begin{theorem}[FiniteTraceEvaluator terminal runtime consumer factorization]
\label{thm:finite-trace-evaluator-terminal-runtime-consumer-factorization}
Every terminal runtime consumer of an accepted $\FiniteTraceEvaluatorUp$
route carrier factors through the finite packet
$$
E=(T,Q,R,X,H,C,P,N)
$$
and through the sibling $\RuntimeCandidateBoundaryUp$ and
$\ReflectionRuntimeCheckpointUp$ surfaces before reading any evaluator result.
The rows $T,Q,R,X,H,C,P,N$ are respectively the finite $\BHist$ trace row,
checkpoint row, runtime row, image and readback row, componentwise $\hsame$
transport row, displayed $\Cont$ replay row, $\Pkg$ provenance row, and local
$\NameCert_{\FiniteTraceEvaluatorUp}$ naming row. The carrier gate is the
$\FiniteTraceEvaluatorUp$ TasteGate readiness surface of
\autoref{thm:finite-trace-evaluator-tastegate-readiness}.
\end{theorem}
\begin{proof}
Start from the terminal route carrier and terminal route exposure of
\autoref{def:finite-trace-evaluator-terminal-route-carrier} and
\autoref{thm:finite-trace-evaluator-terminal-route-exposure}. They display the
whole terminal packet as $T,Q,R,X,H,C,P,N$ and leave no terminal coordinate
outside those rows.

Next apply the terminal checkpoint accountability surface,
\autoref{thm:finite-trace-evaluator-terminal-checkpoint-accountability}. It
forces checkpoint-facing reads to remain accountable to a finite $\BHist$
route before runtime access is consumed.

Now use checkpoint-runtime acceptance lock and runtime route coverage,
\autoref{thm:finite-trace-evaluator-checkpoint-runtime-acceptance-lock} and
\autoref{thm:finite-trace-evaluator-runtime-route-coverage}. These route the
runtime consumer through $\ReflectionRuntimeCheckpointUp$ and
$\RuntimeCandidateBoundaryUp$ before the evaluator result is read.

Finally attach the terminal runtime-candidate bridge and the TasteGate
readiness surface. The bridge keeps the consumer on the displayed terminal
rows, while \autoref{thm:finite-trace-evaluator-tastegate-readiness} supplies
the carrier gate that makes the public read field-faithful over the same
packet.
\end{proof}

\begin{theorem}[FiniteTraceEvaluator terminal NameCert exhaustion]
\label{thm:finite-trace-evaluator-terminal-namecert-exhaustion}
For an accepted $\FiniteTraceEvaluatorUp$ terminal route carrier
$E=(T,Q,R,X,H,C,P,N)$, the local
$\NameCert_{\FiniteTraceEvaluatorUp}$ terminal obligations are exhausted by
the following finite rows: trace finiteness for $T$, checkpoint admission for
$Q$, runtime totality for $R$, readback exactness for $X$, componentwise
$\hsame$ transport for $H$, displayed $\Cont$ replay for $C$, $\Pkg$
provenance for $P$, and local naming for $N$. No further
$\NameCert_{\FiniteTraceEvaluatorUp}$ clause is available outside these
rows.
\end{theorem}
\begin{proof}
Read the local terminal packet from
\autoref{thm:finite-trace-evaluator-namecert-terminal-obligation-surface}. It
already identifies the terminal NameCert surface with trace, checkpoint,
runtime, readback, transport, replay, provenance, and naming obligations over
$T,Q,R,X,H,C,P,N$.

The checkpoint part is closed by terminal checkpoint accountability,
\autoref{thm:finite-trace-evaluator-terminal-checkpoint-accountability}. It
binds checkpoint admission to the finite $\BHist$ route and prevents the
checkpoint row from becoming a hidden source.

The runtime and readback part is closed by
\autoref{thm:finite-trace-evaluator-runtime-route-totality} and
\autoref{thm:finitetrace-evaluator-checkpoint-replay-totality}. These
theorems make the runtime route total on the displayed packet and force the
readback replay through the same finite rows.

It remains to account for transport, replay, provenance, and naming. Terminal
route exposure supplies $H,C,P,N$ as the displayed $\hsame$, $\Cont$, $\Pkg$,
and local $\NameCert$ rows, and the TasteGate readiness theorem keeps those
rows as the public carrier surface. Thus the listed obligations exhaust the
terminal local NameCert package.
\end{proof}

\begin{theorem}[FiniteTraceEvaluator terminal source lock]
\label{thm:finite-trace-evaluator-terminal-source-lock}
No terminal runtime source of an accepted $\FiniteTraceEvaluatorUp$ route
carrier exposes a host validator, hidden runtime state, unbounded evaluator,
infinite trace, total truth predicate, total halting classifier, or unchecked
compiler theorem. Every source available to a terminal consumer is locked to
the displayed finite rows $T,Q,R,X,H,C,P,N$, to the sibling
$\RuntimeCandidateBoundaryUp$ and $\ReflectionRuntimeCheckpointUp$ surfaces
named by provenance, and to the carrier gate of
\autoref{thm:finite-trace-evaluator-tastegate-readiness}.
\end{theorem}
\begin{proof}
First apply terminal route exposure. The carrier
$E=(T,Q,R,X,H,C,P,N)$ has no coordinate for host validation, hidden runtime
state, unbounded evaluation, infinite traces, or unchecked compiler theorems.

Next use checkpoint route nonescape,
\autoref{thm:finite-trace-evaluator-checkpoint-route-nonescape}. It prevents a
checkpoint-facing consumer from replacing the displayed finite route by an
alternate evaluator output or a detached runtime source.

Then apply runtime acceptance nonescape and validation refusal closure,
\autoref{thm:finitetrace-evaluator-runtime-acceptance-nonescape} and
\autoref{thm:finite-trace-evaluator-validation-refusal-closure}. These exclude
total truth predicates, total halting classifiers, host-valid runtime
judgments, hidden runtime state, unbounded evaluator semantics, infinite
traces, and unchecked compiler theorems from the runtime acceptance row.

Finally combine runtime route coverage with terminal runtime-candidate bridge.
Every runtime-facing source passes through $\ReflectionRuntimeCheckpointUp$
and $\RuntimeCandidateBoundaryUp$ before it reaches the evaluator result, and
then returns to $T,Q,R,X,H,C,P,N$. The TasteGate readiness theorem supplies the
carrier gate over that same finite surface, so no other terminal runtime
source is exposed.
\end{proof}

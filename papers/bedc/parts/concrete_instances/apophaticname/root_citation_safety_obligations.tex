\begin{theorem}[ApophaticName root citation safety]
\label{thm:apophatic-name-root-citation-safety}
For every accepted $\ApophaticNameUp$ packet
$$
A=(S,R,G,L,H,C,P,N),
$$
every root consumer citation of the named externality may cite only the visible socket, request, refusal gate, refusal ledger, transport, continuation, provenance, and local naming surface $S,R,G,L,H,C,P,N$. Such a citation cannot cite a generated witness, host object, quotient representative, hidden theorem, supplied dependency, assumed proposition, or closure principle outside the visible refusal packet.
\end{theorem}
\begin{proof}
Start with the public boundary package of \autoref{thm:apophatic-name-public-boundary-package}. It identifies the root-facing object with exactly the finite rows $S,R,G,L,H,C,P,N$.

Next apply socket citation scope and consumer-boundary exhaustion, \autoref{thm:apophatic-name-socket-citation-scope} and \autoref{thm:apophatic-name-consumer-boundary-exhaustion}. These keep every downstream citation on the socket, request, refusal, transport, continuation, provenance, and naming rows.

Finally use the non-escape certificate, \autoref{thm:apophatic-name-non-escape-certificate}. Since the accepted carrier has no coordinate for generated witnesses, host objects, quotient representatives, hidden theorems, supplied dependencies, assumed propositions, or closure principles, the root citation is safe exactly on the visible refusal surface.
\end{proof}

\begin{theorem}[ApophaticName boundary request totality]
\label{thm:apophatic-name-boundary-request-totality}
For every accepted $\ApophaticNameUp$ packet
$$
A=(S,R,G,L,H,C,P,N),
$$
every boundary request for the named externality must expose the request row $R$ before it exposes the refusal gate row $G$. The accepted route then continues through the refusal ledger $L$, transport row $H$, continuation row $C$, provenance row $P$, and local naming row $N$; it cannot bypass $R$ by reading a generated witness, host object, quotient representative, hidden theorem, supplied dependency, assumed proposition, or closure principle.
\end{theorem}
\begin{proof}
Begin with visible request factorization, \autoref{thm:apophatic-name-visible-request-factorization}. It sends every visible socket request through the displayed request row $R$ as part of the packet $S,R,G,L,H,C,P,N$.

Use socket request gate exactness, \autoref{thm:apophatic-name-socket-request-gate-exactness}. The gate row $G$ is read as the refusal response to that displayed request, so a boundary request cannot reach the gate before it has exposed $R$.

Finally combine source-refusal totality and ledger nonescape, \autoref{thm:apophatic-name-source-refusal-totality} and \autoref{thm:apophatic-name-ledger-nonescape}. These route the accepted request through $G,L,H,C,P,N$ and exclude every bypass through generated or host-side content.
\end{proof}

\begin{theorem}[ApophaticName refusal route exhaustion]
\label{thm:apophatic-name-refusal-route-exhaustion}
For every accepted $\ApophaticNameUp$ packet
$$
A=(S,R,G,L,H,C,P,N),
$$
every refusal route for the named externality is exhausted by the visible rows $G,L,H,C,P,N$ after the socket and request rows $S,R$ have been exposed. No refusal route generates a witness, supplies a host object, selects a quotient representative, imports a hidden theorem, or invokes a closure principle outside the displayed packet.
\end{theorem}
\begin{proof}
First apply root-unblock refusal boundary and refusal ledger exactness, \autoref{thm:apophatic-name-root-unblock-refusal-boundary} and \autoref{thm:apophatic-name-refusal-ledger-exactness}. They identify root use of the named externality with the displayed refusal gate and ledger rather than with supplied content.

Next use socket-gate route exhaustion, \autoref{thm:apophatic-name-socket-gate-route-exhaustion}. Any socket read that reaches the refusal side must pass through $S,R,G,L,H,C,P,N$, so the refusal route is already a finite route through the displayed rows.

Finally apply boundary citation exhaustion and downstream named-externality refusal, \autoref{thm:apophatic-name-boundary-citation-exhaustion} and \autoref{thm:apophatic-name-downstream-named-externality-refusal}. They exhaust downstream citation by the same refusal packet and leave no route to generated witnesses, host objects, quotient representatives, hidden theorems, or closure principles.
\end{proof}

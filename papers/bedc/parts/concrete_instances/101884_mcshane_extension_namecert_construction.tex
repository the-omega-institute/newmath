\chapter{A Concrete Naming Certificate for $\McShaneExtensionUp$}
\label{ch:concrete-instances-mcshane-extension-namecert}
\origin{human}

Following McShane and Whitney Lipschitz extension and Bishop--Bridges constructive analysis, the $\McShaneExtensionUp$ packet records the finite certificate by which a displayed real-valued Lipschitz map on a displayed subcarrier is read as an extension candidate on the ambient metric carrier. It consumes the map-level bound surface of \autoref{ch:concrete-instances-lipschitzmap-namecert}, the dyadic tolerance route of \autoref{ch:concrete-instances-dyadic-lipschitz-modulus-namecert}, the uniformly continuous extension handoff of \autoref{ch:concrete-instances-uniform-continuous-extension-namecert}, and the separated completion uniqueness boundary of \autoref{ch:concrete-instances-completion-extension-uniqueness-namecert}. The packet names a finite Lipschitz-extension certificate; it is not a host formula for pointwise infima or suprema, not a quotient function-space construction, and not an appeal to selected bounds, function extensionality, countable choice, or ambient $\RealUp$ completeness.

\begin{definition}[McShaneExtension carrier]
\label{def:mcshane-extension-carrier}
A $\McShaneExtensionUp$ carrier is a finite $\BHist$ packet
$$
M=(X,A,L,D,E,S,R,K,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $X$ is the ambient source $\MetricUp$ row, exposing only the metric observations used by the extension request.
\item $A$ is the displayed subcarrier or domain row, recording the admitted source points, the inclusion route into $X$, and the finite membership evidence consumed by later rows.
\item $L$ is the source $\LipschitzMapUp$ row for the real-valued map on $A$, including the displayed Lipschitz constant and the finite graph windows read on the subcarrier.
\item $D$ is the $\DyadicLipschitzModulusUp$ row that turns the displayed Lipschitz bound into dyadic tolerance data for source and ambient reads.
\item $E$ is the candidate extension row, recording the finite McShane--Whitney envelope comparisons available to an ambient input without selecting a global infimum or supremum.
\item $S$ is the source-agreement row, requiring the candidate extension read to coincide with the original source graph when the ambient input is admitted by $A$.
\item $R$ records the $\RealUp$ seal rows for source values, envelope comparisons, and ambient extension reads.
\item $K$ records the $\MetricUp$ seal rows for the source metric, subcarrier inclusion, and ambient distance comparisons.
\item $H$ records componentwise $\hsame$ transport for the ambient metric, subcarrier, source Lipschitz row, dyadic modulus, candidate extension, source agreement, Real and Metric seals, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from an ambient precision request through $X,A,L,D,E,S,R,K,H,P,N$.
\item $P$ is the $\Pkg$ provenance over $\McShaneExtensionUp$, $\MetricUp$, $\LipschitzMapUp$, $\DyadicLipschitzModulusUp$, $\UniformContinuousExtensionUp$, $\CompletionExtensionUniquenessUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\McShaneExtensionUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through these twelve displayed rows. It has no coordinate for host function equality, quotient function spaces, selected infima or suprema, countable-choice selectors, trichotomy, ambient $\RealUp$ completeness, or a public bridge credential.
\end{definition}

\begin{theorem}[McShaneExtension NameCert obligations]
\label{thm:mcshane-extension-namecert-obligations}
For every accepted $\McShaneExtensionUp$ carrier
$$
M=(X,A,L,D,E,S,R,K,H,C,P,N),
$$
the local $\NameCert_{\McShaneExtensionUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the displayed ambient metric, subcarrier, source Lipschitz, dyadic modulus, candidate extension, source agreement, Real and Metric seal, componentwise transport, replay, provenance, and local naming rows;
\item domain compatibility, requiring every source-value read to pass through $A$ before it is compared with an ambient extension read;
\item Lipschitz-source admission, requiring $L$ to be the only row that exposes the source graph and displayed Lipschitz constant;
\item dyadic modulus transport, requiring $D$ to mediate every tolerance transfer from source distances to extension-value estimates;
\item candidate-extension exactness, requiring $E$ to store only finite envelope comparison data and no selected infimum, selected supremum, quotient function, or host function object;
\item source-agreement exactness, requiring $S$ to identify candidate and source reads only through the same displayed $A,L,D,R,K$ route;
\item ledger non-escape, requiring $H,C,P,N$ to preserve and replay the listed rows without adding host function equality, quotient function spaces, selected bounds, choice, or ambient $\RealUp$ completeness.
\end{enumerate}
No local obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:mcshane-extension-carrier}. The accepted packet displays exactly the ambient metric, subcarrier, source Lipschitz row, dyadic modulus, candidate extension row, source-agreement row, Real and Metric seals, and the structural rows.

The domain and source clauses are read next. The row $A$ is the only subcarrier admission row, and $L$ is the only row exposing the real-valued Lipschitz graph on that subcarrier. Thus a source-value comparison cannot be formed before the packet has read both rows.

The tolerance route is then forced by $D$. By \autoref{ch:concrete-instances-lipschitzmap-namecert} and \autoref{ch:concrete-instances-dyadic-lipschitz-modulus-namecert}, the displayed Lipschitz constant reaches ambient precision requests only through the finite dyadic modulus route, not through a host estimate.

The candidate and agreement clauses project $E$ and $S$. The candidate row stores finite McShane--Whitney envelope comparisons, while the source-agreement row checks the same data on points admitted by $A$. These rows do not select a global infimum or supremum and do not turn the candidate into a quotient function-space element.

Finally inspect $H,C,P,N$. Transport is componentwise, replay follows the displayed route, and provenance plus local naming mention only the packet and its sibling dependencies, including \autoref{ch:concrete-instances-uniform-continuous-extension-namecert} and \autoref{ch:concrete-instances-completion-extension-uniqueness-namecert}. Hence the local NameCert obligations are exactly the listed finite rows.
\end{proof}

\begin{theorem}[McShaneExtension Lipschitz bound]
\label{thm:mcshane-extension-lipschitz-bound}
Let $M=(X,A,L,D,E,S,R,K,H,C,P,N)$ be an accepted $\McShaneExtensionUp$ carrier. Every ambient extension read exported by $M$ factors through the finite route
$$
X,A\longmapsto L\longmapsto D\longmapsto E\longmapsto R,K\longmapsto H,C,P,N .
$$
Along that route the displayed source Lipschitz constant remains the only bound used for the ambient candidate. The packet therefore exports a Lipschitz-bound certificate for the candidate extension without using host function equality, quotient function spaces, selected infima or suprema, choice, or ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
Start with an ambient extension read. The ambient metric observations come from $X$, while subcarrier information and source comparison data come from $A$. No extension-value estimate is available before these two rows have been displayed.

Read the source Lipschitz and modulus rows. The row $L$ supplies the displayed source graph and Lipschitz constant, and $D$ converts that bound into dyadic tolerance data. This is the same finite tolerance discipline isolated by \autoref{ch:concrete-instances-dyadic-lipschitz-modulus-namecert}.

The candidate extension row $E$ then records only finite envelope comparisons. Its reads are sealed through $R$ and $K$, so Real-facing and Metric-facing consumers inspect the same finite route before reaching structural transport.

Transport, replay, provenance, and local naming preserve the route through $H,C,P,N$. The completion-facing siblings \autoref{ch:concrete-instances-uniform-continuous-extension-namecert} and \autoref{ch:concrete-instances-completion-extension-uniqueness-namecert} may consume this bound certificate downstream, but the carrier itself exports no host function equality, no quotient function space, no selected infimum or supremum, no choice principle, and no ambient $\RealUp$ completeness.
\end{proof}

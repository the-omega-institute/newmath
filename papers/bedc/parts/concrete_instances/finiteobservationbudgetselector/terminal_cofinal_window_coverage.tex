\begin{theorem}[Finite observation-budget selector Real-seal window saturation]
\label{thm:finite-observation-budget-selector-real-seal-window-saturation}
Let
$$
F=(B,S,W,D,R,E,H,C,P,N)
$$
be an accepted $\FiniteObservationBudgetSelectorUp$ packet. Any Real-completion or diagonal consumer that reaches the same terminal row $E$ through the same budget, schedule, dyadic, regular, transport, continuation, provenance, and local naming rows $B,S,D,R,H,C,P,N$ must use the displayed finite $\StreamNameUp$ window family $W$. Adding or replacing a window without changing the budget or schedule is not a refinement of the same selector packet; it is a different accepted packet with a different window row.
\end{theorem}
\begin{proof}
Start with the route into the terminal Real seal. By \autoref{thm:finite-observation-budget-selector-real-seal-handoff}, every Real-facing read of $E$ is reached only after the finite budget, schedule, selected window family, dyadic ledger, and regular handoff have been exposed.

Use window exhaustion for the lower observation segment. By \autoref{thm:finite-observation-budget-selector-dyadic-window-exhaustion}, the dyadic observations in $D$ are already covered by the displayed finite windows $W$, so a diagonal or completion consumer has no independent dyadic source outside that family.

Now apply route determinacy. The theorem \autoref{thm:finite-observation-budget-selector-budget-route-determinacy} fixes the displayed route from $B$ through $S,W,D,R$ to $E$ whenever the structural rows $H,C,P,N$ are preserved.

Finally use the terminal theorems to exclude a hidden selector. By \autoref{thm:finite-observation-budget-selector-terminal-route-no-branching} and \autoref{thm:finite-observation-budget-selector-terminal-route-endpoint-exactness}, a consumer reaching the same endpoint through the same displayed rows cannot pass through a second window source. A changed window family therefore changes the accepted selector packet rather than refining the original one.
\end{proof}

\begin{theorem}[Finite observation-budget selector terminal window minimality]
\label{thm:finite-observation-budget-selector-terminal-window-minimality}
Let
$$
F=(B,S,W,D,R,E,H,C,P,N)
$$
be an accepted $\FiniteObservationBudgetSelectorUp$ packet. $$
B \to S \to W \to D \to R \to E.
$$
If any one of the displayed budget, schedule, window, dyadic ledger, regular handoff, seal, transport, continuation, provenance, or local naming rows is dropped, the remaining rows no longer determine the terminal endpoint exactness of \autoref{thm:finite-observation-budget-selector-terminal-route-endpoint-exactness}.
\end{theorem}
\begin{proof}
First identify the only route available to a terminal consumer. The budget-route determinacy theorem \autoref{thm:finite-observation-budget-selector-budget-route-determinacy} fixes the path from $B$ through $S,W,D,R$ to $E$, and terminal selector synchronization \autoref{thm:finite-observation-budget-selector-terminal-selector-synchronization} keeps the selected schedule and window rows attached to that path.

Next remove terminal branching. By \autoref{thm:finite-observation-budget-selector-terminal-route-no-branching}, two consumers exposing the same displayed rows determine the same terminal route, so there is no alternate endpoint route through quotient streams, host real equality, countable choice, hidden selectors, side-channel completion, or ambient completeness.

It remains to record sufficiency and minimality. Sufficiency is exactly endpoint exactness from \autoref{thm:finite-observation-budget-selector-terminal-route-endpoint-exactness}: the displayed rows identify the terminal endpoint with $E$ by componentwise $\hsame$ transport. For minimality, each listed row is used by one segment of the displayed route or by its transport, continuation, provenance, and naming certificate. Dropping such a row removes the corresponding segment or certificate witness, so the endpoint can no longer be recovered by the cited exactness theorem.
\end{proof}

\begin{theorem}[RatUp FieldUp final certificate noncollapse action package]
\label{thm:ratup-fieldup-final-certificate-noncollapse-action-package}
For Definition~\ref{def:ratup-fieldup-concrete-exit-object} and
$\mathsf{RatUpFieldSource}_F$, the concrete $\FieldUp(\RatUp)$ exit certificate
contains one package whose rows are all anchored at the same concrete
$\mathsf{BHist}$ source object. The package consists of:
\begin{enumerate}
  \item the source anchor, carrier $\mathsf{RatHistoryCarrier}$, classifier
  $\mathsf{RatHistoryClassifier}$, denominator tuple, and contextual action from
  Theorem~\ref{thm:ratup-fieldup-canonical-machine-checkable-exit};
  \item the algebra-reduct row, inverse-product classifier row, and affine-action
  rows from
  Theorem~\ref{thm:ratup-fieldup-canonical-exit-action-inverse-package};
  \item the transported contextual-pair noncollapse clauses from
  Theorem~\ref{thm:ratup-fieldup-contextual-pair-transport-full-noncollapse-invariance};
  \item the affine inverse uniqueness row from
  Theorem~\ref{thm:ratup-fieldup-affine-inverse-uniqueness-exit}.
\end{enumerate}
Thus the Phase Algebra Hierarchy exit criterion may consume this object as one
concrete certificate boundary, not as a family indexed by schematic operation
parameters.
\end{theorem}
\begin{proof}
Apply
Theorem~\ref{thm:ratup-fieldup-canonical-machine-checkable-exit}. This fixes the
concrete $\mathsf{RatUpFieldSource}_F$ source, the $\mathsf{BHist}$ carrier
object, the history classifier, the denominator tuple, and the contextual
action in the concrete exit certificate.

Apply
Theorem~\ref{thm:ratup-fieldup-canonical-exit-action-inverse-package} to the
same exit certificate. It attaches the algebra-reduct, inverse-product
classifier, and affine-action rows without changing the source object selected
by the canonical certificate.

Apply
Theorem~\ref{thm:ratup-fieldup-contextual-pair-transport-full-noncollapse-invariance}.
The transported contextual-pair clauses use the same $\mathsf{BHist}$ endpoint
source and its denominator contexts, so the noncollapse rows are certificate
rows over the concrete object rather than fresh parameters.

Finally apply
Theorem~\ref{thm:ratup-fieldup-affine-inverse-uniqueness-exit}. The affine
inverse uniqueness row is read in the affine action carrier supplied above, so
it closes the same concrete certificate package and supplies the displayed
single boundary consumed by the hierarchy exit criterion.
\end{proof}

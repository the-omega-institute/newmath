\begin{theorem}[RatUp FieldUp nondegenerate semantic exit source]
\label{thm:ratup-fieldup-nondegenerate-semantic-exit}
Let $d_1:=\Eone(\emp)$. For the concrete source tuple
$\mathsf{RatUpFieldSource}_F$ of
\autoref{def:ratup-fieldup-concrete-source-tuple}, the same $\mathsf{BHist}$
data simultaneously give the following four pieces of the
$\FieldUp(\mathsf{RatUp})$ exit source:
\begin{enumerate}
  \item the assembled semantic naming certificate rows for
  $\FieldUp(\mathsf{RatUp})$;
  \item source anchoring of all certificate rows at
  $\mathsf{RatHistoryCarrier}$ and $\mathsf{RatHistoryClassifier}$;
  \item the assembled support-neutrality row for the denominator contextual
  action $\Gamma$;
  \item the endpoint separation
  $$
  \mathsf{RatHistoryCarrier}(d_1)
  $$
  together with
  $$
  \mathsf{fieldSingletonEmptyCarrier}(d_1)\to\bot.
  $$
\end{enumerate}
Consequently the semantic exit source for $\FieldUp(\mathsf{RatUp})$ over
$\mathsf{RatUpFieldSource}_F$ is not collapsible to the singleton empty-history
$\FieldUp$ source.
\end{theorem}
\begin{proof}
The semantic certificate rows are those assembled in
\autoref{thm:ratup-fieldup-semantic-certificate-assembly}. They include the
carrier, source, classifier, stability, ledger, exactness, and affine rows of
the concrete $\FieldUp(\mathsf{RatUp})$ certificate.

The common source location is fixed by
\autoref{thm:ratup-fieldup-certificate-rows-source-anchored}. It identifies the
field carrier as $\mathsf{RatHistoryCarrier}$, the field classifier as
$\mathsf{RatHistoryClassifier}$, and the denominator, contextual-action,
context-pair, and affine rows as entries of the same
$\mathsf{RatUpFieldSource}_F$ tuple.

The denominator contextual action contributes the support-neutrality component
through \autoref{thm:ratup-fieldup-assembled-support-neutrality-certificate}.
Thus the assembled certificate contains the carried nested action, the
collected-action classifier comparison, strict-support neutrality, and the
second-endpoint classifier exactness row over the denominator tuple.

Finally,
\autoref{thm:ratup-fieldup-source-non-singleton-witness} supplies the displayed
endpoint $d_1=\Eone(\emp)$, its
$\mathsf{RatHistoryCarrier}(d_1)$ witness, and the rejection map
$\mathsf{fieldSingletonEmptyCarrier}(d_1)\to\bot$. Any collapse to the singleton
empty-history $\FieldUp$ source would have to send this carried endpoint to the
singleton carrier, contradicting that rejection map.
\end{proof}

\begin{theorem}[RatUp FieldUp concrete exit row closure]
\label{thm:ratup-fieldup-concrete-exit-row-closure}
Let $d_1:=\Eone(\emp)$. For the concrete source tuple
$\mathsf{RatUpFieldSource}_F$ of
\autoref{def:ratup-fieldup-concrete-source-tuple}, the
$\FieldUp(\mathsf{RatUp})$ exit row package is the concrete
$\mathsf{BHist}$-anchored closure jointly supplied by the following data:
\begin{enumerate}
  \item the semantic certificate rows of
  \autoref{thm:ratup-fieldup-semantic-certificate-assembly};
  \item the source anchoring rows of
  \autoref{thm:ratup-fieldup-certificate-rows-source-anchored};
  \item the support-sensitive row coverage of
  \autoref{thm:ratup-fieldup-support-neutrality-row-completeness};
  \item the endpoint facts $\mathsf{RatHistoryCarrier}(d_1)$ and
  $\mathsf{fieldSingletonEmptyCarrier}(d_1)\to\bot$;
  \item the shared concrete row language over
  $\mathsf{RatHistoryCarrier}$, $\mathsf{RatHistoryClassifier}$,
  $\mathsf{RatDenomUnitCarrier}$, $\mathsf{RatDenomUnitClassifier}$,
  $\mathsf{append}$, $\Cont$, $\emp$, $\Gamma$, and $\mathsf{AffAut}_F$.
\end{enumerate}
Thus the displayed exit package is a concrete closure over
$\mathsf{RatUpFieldSource}_F$ and not an instance of an abstract operator schema.
\end{theorem}
\begin{proof}
Apply \autoref{thm:ratup-fieldup-semantic-certificate-assembly} first. It gives
the assembled carrier, source, pattern, classifier, stability, ledger, and
exactness rows for the concrete target $\FieldUp(\mathsf{RatUp})$.

Apply \autoref{thm:ratup-fieldup-certificate-rows-source-anchored} next. It pins
those rows to the single source tuple $\mathsf{RatUpFieldSource}_F$, with field
carrier $\mathsf{RatHistoryCarrier}$, field classifier
$\mathsf{RatHistoryClassifier}$, denominator tuple
$(\mathsf{RatDenomUnitCarrier},\mathsf{RatDenomUnitClassifier},
\mathsf{append},\emp,\mathsf{RatHistoryCarrier})$, contextual action $\Gamma$,
and affine coordinate $\mathsf{AffAut}_F$.

The support-sensitive part is closed by
\autoref{thm:ratup-fieldup-support-neutrality-row-completeness}. It records that
the support rows consumed by the assembled semantic certificate are precisely
the concrete $\mathsf{BHist}$ rows already present in the named certificate
instance.

Finally, \autoref{thm:ratup-fieldup-nondegenerate-semantic-exit} supplies the
endpoint $d_1=\Eone(\emp)$, the witness
$\mathsf{RatHistoryCarrier}(d_1)$, and the rejection map
$\mathsf{fieldSingletonEmptyCarrier}(d_1)\to\bot$. Combining these four cited
sites repacks the existing rows as one concrete closure theorem for the exit
package. The result is therefore a composite closure statement, not a variant of
any single checked theorem.
\end{proof}

\begin{theorem}[RatUp FieldUp empty-context nondegenerate witness]
\label{thm:ratup-fieldup-empty-context-nondegenerate-witness}
Let $d_1:=\Eone(\emp)$. For histories $L,R:\mathsf{BHist}$, assume
$\mathsf{fieldSingletonEmptyCarrier}(L)$ and
$\mathsf{fieldSingletonEmptyCarrier}(R)$. Put
$w:=\mathsf{append}(L,\mathsf{append}(d_1,R))$. Then the wrapped endpoint
satisfies
$$
\mathsf{RatHistoryCarrier}(w),
$$
the rejection map
$$
\mathsf{fieldSingletonEmptyCarrier}(w)\to\bot,
$$
and the nonzero-complement clause
$$
\mathsf{fieldSingletonEmptyNonZero}(w).
$$
Thus the non-singleton rational witness remains stable under empty singleton
context wrapping on both sides.
\end{theorem}
\begin{proof}
The hypotheses place both contextual wrappers in the singleton empty carrier.
By the singleton-empty carrier inversion for $\FieldUp$, each wrapper compares
by $\mathsf{hsame}$ with $\emp$.

Use the left and right append-unit laws, together with the continuation
compatibility of $\mathsf{append}$ and transitivity of $\mathsf{hsame}$, to
compare
$$
w=\mathsf{append}(L,\mathsf{append}(d_1,R))
$$
with the endpoint $d_1$. This comparison is the transport path for all carrier
and classifier predicates used below.

Apply \autoref{thm:ratup-fieldup-nondegenerate-semantic-exit}. It supplies
$\mathsf{RatHistoryCarrier}(d_1)$ and the rejection map
$\mathsf{fieldSingletonEmptyCarrier}(d_1)\to\bot$. Transport the rational
carrier witness from $d_1$ to $w$ along the displayed $\mathsf{hsame}$
comparison.

For rejection at $w$, take any hypothetical
$\mathsf{fieldSingletonEmptyCarrier}(w)$ witness. Transport it back along the
same comparison to obtain a singleton-empty carrier witness for $d_1$, and
contradict the rejection map from
\autoref{thm:ratup-fieldup-nondegenerate-semantic-exit}.

It remains to obtain the nonzero-complement clause. A failure of
$\mathsf{fieldSingletonEmptyNonZero}(w)$ would build the forbidden singleton
empty classifier from the carried singleton-empty witness, so the established
rejection map for $w$ closes the complement.
\end{proof}

Per-object template: traditional target context-stable nondegenerate rational
field witness; BEDC source $\mathsf{BHist}$, $\RatUp$, $\Cont$, and singleton
$\FieldUp$ carrier; carrier choice $\mathsf{RatHistoryCarrier}$; classifier
choice singleton empty classifier for the obstruction; stability obligations
append-unit transport and carrier transport; ledger empty context wrappers; core
theorem \autoref{thm:ratup-fieldup-empty-context-nondegenerate-witness} together
with \autoref{thm:ratup-fieldup-nondegenerate-semantic-exit}; exactness
predicate preservation of the non-singleton witness under empty wrappers;
standard bridge deferred; status $\mathsf{ProofTarget}$.

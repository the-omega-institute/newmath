\begin{theorem}[RatUp FieldUp assembled certificate operation-table coverage]
\label{thm:ratup-fieldup-assembled-certificate-operation-table-coverage}
For the concrete source tuple $\mathsf{RatUpFieldSource}_F$ of
\autoref{def:ratup-fieldup-concrete-source-tuple} and the concrete operation
table of Definition~\ref{def:ratup-fieldup-concrete-operation-table}, one and
the same $\mathsf{BHist}$ row package satisfies both
\autoref{def:field-certificate-obligations} and the concrete
$\FieldUp(\RatUp)$ operation-table laws. The shared carrier and classifier are
$\mathsf{RatHistoryCarrier}$ and $\mathsf{RatHistoryClassifier}$. The shared
denominator coordinate is
$$
(\mathsf{RatDenomUnitCarrier},\mathsf{RatDenomUnitClassifier},
\mathsf{append},\emp,\mathsf{RatHistoryCarrier}).
$$
The shared contextual coordinate is the action $\Gamma$, the shared
context-pair coordinate is the monoid of
\autoref{def:ratup-fieldup-denominator-context-pair-monoid}, and the shared
affine coordinate is $\mathsf{AffAut}_F$. The inverse and nonzero rows, together
with the exit-noncollapse rows, are the rows consumed by both cited interfaces.
\end{theorem}
\begin{proof}
Unpack \autoref{def:ratup-fieldup-concrete-source-tuple}. It names
$\mathsf{RatHistoryCarrier}$, $\mathsf{RatHistoryClassifier}$, the displayed
denominator tuple, $\Gamma$, the context-pair monoid, and
$\mathsf{AffAut}_F$ as coordinates of the single source
$\mathsf{RatUpFieldSource}_F$.

Apply \autoref{thm:ratup-fieldup-assembled-certificate-boundary}. This gives
\autoref{def:field-certificate-obligations} for exactly those coordinates.
Apply \autoref{thm:ratup-fieldup-concrete-operation-table-laws}. This gives the
carrier, classifier, denominator, contextual-action, context-pair, affine,
inverse, nonzero, and exit-noncollapse laws for the concrete operation table of
Definition~\ref{def:ratup-fieldup-concrete-operation-table}.

Both cited results name the same concrete carrier, classifier, denominator
tuple, contextual action, context-pair monoid, affine row source, inverse rows,
nonzero rows, and exit rows. Therefore the certificate-obligation interface and
the operation-table interface are satisfied by the same concrete
$\mathsf{BHist}$ rows.
\end{proof}

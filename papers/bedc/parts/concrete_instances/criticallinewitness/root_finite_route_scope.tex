\begin{theorem}[Critical-line witness root finite strip route]
\label{thm:critical-line-witness-root-finite-strip-route}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ packet. Every root zero read and every root critical-strip read of $W$ exposes only the finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows carried by $Z,S,H,C,P,N$ before any downstream $\mathsf{RH}$-facing consumer reads the packet. The later modulus, real-part, and rational-comparison rows may be consulted only through the same displayed carrier route $Z,S,M,R,Q,H,C,P,N$.
\end{theorem}
\begin{proof}
Project the accepted packet through \autoref{def:critical-line-witness-carrier}. The carrier supplies exactly the displayed rows $Z,S,M,R,Q,H,C,P,N$, with $H,C,P,N$ recording the finite transport, continuation, provenance, and local naming surface.

Apply the root source exposure theorem \autoref{thm:critical-line-witness-root-source-exposure}. It forces a root-facing consumer to see the supplied zero and strip rows as displayed source data before it can use the packet downstream.

Finally use the zero-strip source totality theorem \autoref{thm:critical-line-witness-zero-strip-source-totality}. The total route remains inside finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows, so no downstream $\mathsf{RH}$-facing consumer receives an unexposed source coordinate.
\end{proof}

\begin{theorem}[Critical-line witness root modulus read totality]
\label{thm:critical-line-witness-root-modulus-read-totality}
For every accepted $\CriticalLineWitnessUp$ packet $W=(Z,S,M,R,Q,H,C,P,N)$, every modulus, depth, real-part, or rational-comparison read factors through the displayed rows $M,R,Q,H,C,P,N$ and the existing $\RatUp/\RealUp$ comparison surface. No hidden numeric oracle, host real equality, quotient representative, analytic-continuation row, or ambient completeness input can answer the read.
\end{theorem}
\begin{proof}
Start with the root modulus-window totality theorem \autoref{thm:critical-line-witness-root-modulus-window-totality}. It identifies $M,R,Q,H,C,P,N$ as the whole finite route for modulus-window reads.

Now apply the root comparison ledger \autoref{thm:critical-line-witness-root-comparison-ledger}. The rational and real comparison data are read through the displayed $\RatUp$ and $\RealUp$ rows, transported by $H$, replayed by $C$, recorded by $P$, and named by $N$.

Since the displayed carrier has no coordinate for a hidden numeric oracle, host equality, quotient representative, analytic continuation, or ambient completeness, the modulus read is total only through the stated route.
\end{proof}

\begin{theorem}[Critical-line witness root refusal ledger scope]
\label{thm:critical-line-witness-root-refusal-ledger-scope}
For an accepted $\CriticalLineWitnessUp$ packet $W=(Z,S,M,R,Q,H,C,P,N)$, every request for $\mathsf{RH}$ proof data, global zero classifiers, analytic-continuation certificates, quotient representatives, host equality, or ambient completeness is refused inside the local ledger over $Z,S,M,R,Q,H,C,P,N$. The refusal ledger is not exported as a proof surface or as any of the refused objects.
\end{theorem}
\begin{proof}
Apply the root refusal route \autoref{thm:critical-line-witness-root-refusal-route}. It sends root-facing off-route demands to the displayed refusal boundary rather than to analytic or host-level data.

Then use RH-refusal ledger completeness \autoref{thm:critical-line-witness-rh-refusal-ledger-completeness}. That theorem accounts for $\mathsf{RH}$ proof requests, zero-search and global-classifier requests, quotient representatives, host equality, analytic continuation, and ambient completeness inside the same finite ledger.

The ledger rows remain $Z,S,M,R,Q,H,C,P,N$. They can be read as local refusal data, but no theorem in the route repacks them as an exported $\mathsf{RH}$ proof, global zero classifier, analytic-continuation certificate, quotient representative, host equality witness, or completeness principle.
\end{proof}

\begin{theorem}[Critical-line witness root budget-selector totality]
\label{thm:critical-line-witness-root-budget-selector-totality}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ root packet, and let $B$ be an accepted finite observation-budget selector row for the same root readback surface. Every root consumer that reads $W$ through $B$ is exhausted by the displayed $\BHist$ rows $Z,S,M,R,Q$, the componentwise $\hsame$ support $H$, the displayed $\Cont$ route $C$, the $\Pkg$ provenance $P$, and the local $\NameCert$ row $N$. The selector may bound finite requests, but it cannot add host $\mathsf{RH}$ truth, analytic continuation, ambient completeness, or a hidden zero-location selector.
\end{theorem}
\leanchecked{BEDC.Derived.CriticalLineWitnessUp.CriticalLineWitnessCarrier\_root\_budget\_selector\_totality}
\begin{proof}
Begin with root finite readback totality, \autoref{thm:critical-line-witness-root-finite-readback-totality}. It identifies the positive root readback surface as the finite packet $Z,S,M,R,Q,H,C,P,N$.

Attach the selector side through \autoref{thm:finite-observation-budget-selector-budget-route-determinacy} and \autoref{thm:finite-observation-budget-selector-public-export}. Those rows bound the observation request and keep selector-facing reads inside finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support.

Finally apply the root refusal boundary, \autoref{thm:critical-line-witness-root-refusal-boundary-totality}. Every request for host $\mathsf{RH}$ truth, analytic continuation, ambient completeness, or hidden zero-location data is refused outside the displayed packet.
\end{proof}

\begin{theorem}[Critical-line witness root strip-window exhaustion]
\label{thm:critical-line-witness-root-strip-window-exhaustion}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ root packet whose finite selector route displays the same fixed-strip window rows as the root source surface. Then every fixed-strip or zero-strip readback over $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ factors through $Z,S,M,R,Q,H,C,P,N$ before any refusal row is exported. No consumer may replace the fixed-strip window by analytic continuation, ambient completeness, host $\mathsf{RH}$ truth, or an unrecorded zero-location selector.
\end{theorem}
\begin{proof}
Use the fixed-strip source rows first. The fixed-strip source totality and source readback theorems, \autoref{thm:critical-line-witness-fixed-strip-source-totality} and \autoref{thm:critical-line-witness-fixed-strip-source-readback-totality}, place strip-facing reads on the displayed root packet.

Now route the window through the finite selector. By \autoref{thm:critical-line-witness-root-budget-selector-totality}, any selector-bound root consumer keeps the same $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows.

The exhaustion claim follows from the fixed-strip row exhaustion theorem \autoref{thm:critical-line-witness-fixed-strip-row-exhaustion}. Its refusal side excludes off-packet strip data, and the selector route adds no analytic-continuation, completeness, host-truth, or hidden-zero coordinate.
\end{proof}
\leanchecked{BEDC.Derived.CriticalLineWitnessUp.CriticalLineWitnessCarrier\_root\_strip\_window\_exhaustion}

\begin{theorem}[Critical-line witness root refusal-ledger pullback]
\label{thm:critical-line-witness-root-refusal-ledger-pullback}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ root packet, and suppose a finite observation-budget selector and an RH-boundary sibling read the same displayed root rows over $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$. Then their refusal ledgers pull back to the same finite packet $Z,S,M,R,Q,H,C,P,N$: every refused request for host $\mathsf{RH}$ truth, analytic continuation, ambient completeness, or hidden zero-location selection is recorded on that packet-local ledger and nowhere else.
\end{theorem}
\begin{proof}
Start with the root refusal consumer exactness theorem \autoref{thm:critical-line-witness-root-refusal-consumer-exactness}. It makes refusal-facing consumers read the displayed finite root packet rather than a host analytic object.

Attach the RH boundary sibling through \autoref{thm:critical-line-witness-rh-boundary-downstream-lock} and \autoref{thm:critical-line-witness-root-rh-consumer-refusal}. These rows keep every $\mathsf{RH}$-facing request on the same root refusal surface.

Finally apply \autoref{thm:critical-line-witness-root-budget-selector-totality}. The finite observation-budget selector bounds the read but does not change the refusal ledger. Hence the selector-facing and RH-boundary-facing refusal rows pull back to the same packet-local $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ surface.
\end{proof}

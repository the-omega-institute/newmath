\begin{theorem}[Critical-line witness root modulus ledger scope]
\label{thm:critical-line-witness-root-modulus-ledger-scope}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ packet. The rows $M,R,Q,H,C,P,N$ are the complete root-facing modulus-comparison ledger: every modulus or rational-comparison read factors through the displayed modulus row, real-part row, rational comparison ledger, componentwise transport, displayed continuation, package provenance, and local naming row.

No root-facing comparison route may replace this ledger by a global zero classifier, host equality, quotient representative, analytic-continuation certificate, or ambient completeness row.
\end{theorem}
\begin{proof}
Begin with the root source triad of \autoref{thm:critical-line-witness-root-source-triad}. It exposes the supplied zero row, critical-strip row, and rational comparison ledger as finite $\BHist$ data before downstream access.

Apply the root comparison ledger of \autoref{thm:critical-line-witness-root-comparison-ledger}. It sends every comparison request through $M,R,Q,H,C,P,N$ and rules out global zero-set, quotient, analytic, and ambient-completeness replacements.

Finally use \autoref{thm:critical-line-witness-modulus-comparison-route-totality}. The modulus row and rational comparison row are read under the same transport, continuation, package, and local naming rows, so $M,R,Q,H,C,P,N$ exhaust the root-facing ledger.
\end{proof}

\begin{theorem}[Critical-line witness root zero-strip exclusion]
\label{thm:critical-line-witness-root-zero-strip-exclusion}
For an accepted $\CriticalLineWitnessUp$ packet $W=(Z,S,M,R,Q,H,C,P,N)$, the supplied zero row $Z$ and critical-strip row $S$ cannot be replaced by host zero search, quotient representatives, analytic continuation, ambient completeness, or any detached zero-strip oracle. Downstream root-facing consumers must read $Z$ and $S$ only through the finite source triad and the displayed transport, continuation, package, and naming rows.
\end{theorem}
\begin{proof}
Read the root source triad. The zero and strip rows are supplied finite source rows, and the rational comparison ledger is exposed with them before any downstream boundary can inspect the packet.

Use the supplied zero-strip transport lock of \autoref{thm:critical-line-witness-supplied-zero-strip-transport-lock}. It keeps $Z$ and $S$ attached to the displayed $\BHist$ route through $H,C,P,N$ and excludes host equality, zero search, global zero classification, quotient representatives, and analytic continuation.

Now apply the root refusal route of \autoref{thm:critical-line-witness-root-refusal-route}. The refusal route also removes ambient completeness and RH-facing proof data, so no detached zero-strip oracle remains available to root-facing consumers.
\end{proof}

\begin{theorem}[Critical-line witness root RH-boundary refusal]
\label{thm:critical-line-witness-root-rh-boundary-refusal}
Every RH-facing downstream consumer of an accepted $\CriticalLineWitnessUp$ packet receives only the finite refusal package over
$$
Z,S,M,R,Q,H,C,P,N.
$$
It receives no proof of $\mathsf{RH}$, no global zero classifier, no host zero search, no quotient representative, no host equality, no analytic-continuation certificate, and no ambient completeness witness.
\end{theorem}
\begin{proof}
First apply the root modulus ledger scope. The modulus-comparison side is exhausted by $M,R,Q,H,C,P,N$ and cannot read analytic or quotient replacements.

Next apply the root zero-strip exclusion. It fixes the supplied zero and strip rows as finite source data and prevents replacement by host zero search, analytic continuation, quotient representatives, or ambient completeness.

Finally use the RH-refusal ledger completeness theorem \autoref{thm:critical-line-witness-rh-refusal-ledger-completeness}. It identifies the downstream export as a refusal ledger rather than an RH proof surface, so the downstream consumer receives exactly the displayed finite package and none of the excluded proof data.
\end{proof}

\begin{theorem}[Critical-line witness candidate closed route]
\label{thm:critical-line-witness-candidate-closed-route}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ packet. \end{theorem}
\begin{proof}
Project the accepted packet through the carrier definition and the root source triad of \autoref{thm:critical-line-witness-root-source-triad}. These rows expose $Z$ and $S$ as the source side of the same finite $\BHist$ packet.

Next use the root unblock source triad and fixed-strip source boundary, \autoref{thm:critical-line-witness-root-unblock-source-triad} and \autoref{def:critical-line-witness-fixed-strip-source}. They keep the candidate route on $Z,S,H,C,P,N$ before the modulus coordinate $M$ is read.

The NameCert obligations then compose the displayed $\hsame$, $\Cont$, and $\Pkg$ rows with the local name $N$. Since the candidate has already been exhausted by $Z,S,H,C,P,N$, it exports no host zero source and no analytic-continuation route.
\end{proof}

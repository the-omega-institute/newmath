\begin{theorem}[Critical-line witness downstream zero-source readback]
\label{thm:critical-line-witness-downstream-zero-source-readback}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ packet. Every downstream zero-source readback of $W$ must first read the supplied zero row $Z$, the critical-strip row $S$, and the rational comparison ledger $Q$ as the root source triad of \autoref{thm:critical-line-witness-root-source-triad}, and it may then read only the displayed rows $M,R,H,C,P,N$ needed by the same accepted packet.

The readback is therefore a finite BEDC row export. It is not an $\mathsf{RH}$ proof, a global zero classifier, analytic continuation, a quotient representative, host equality, or ambient completeness.
\end{theorem}
\begin{proof}
Start with the root source triad. By \autoref{thm:critical-line-witness-root-source-triad}, the downstream source side exposes $Z,S,Q$ as finite $\BHist$ rows under the displayed transport, continuation, package, and local naming surface.

Next apply \autoref{thm:critical-line-witness-root-carrier-projection}. The accepted packet projects to exactly $Z,S,M,R,Q,H,C,P,N$, so a downstream readback has no additional carrier coordinate from which to extract analytic or host-level data.

Finally use \autoref{thm:critical-line-witness-rh-refusal-ledger-completeness}. The refusal ledger removes $\mathsf{RH}$ proof data, global zero classification, analytic continuation, quotient representatives, host equality, and ambient completeness from every downstream read, leaving only the finite BEDC rows.
\end{proof}

\begin{theorem}[Critical-line witness downstream modulus comparison totality]
\label{thm:critical-line-witness-downstream-modulus-comparison-totality}
For an accepted $\CriticalLineWitnessUp$ packet $W=(Z,S,M,R,Q,H,C,P,N)$, every downstream modulus or rational-comparison request is total through the displayed route
$$
Z,S \longmapsto M,R,Q \longmapsto H,C,P,N.
$$
The source side is the root source triad, the comparison side is the root comparison ledger of \autoref{thm:critical-line-witness-root-comparison-ledger}, and the downstream consumer receives no $\mathsf{RH}$ proof, global zero classifier, analytic-continuation certificate, quotient representative, host equality, or ambient completeness row.
\end{theorem}
\begin{proof}
Read the supplied source first. The root source triad theorem identifies $Z,S,Q$ as the finite source rows available before any downstream boundary inspects the packet.

Now use \autoref{thm:critical-line-witness-root-comparison-ledger}. It forces each downstream rational comparison through the ledger $Q$ together with $M,R,H,C,P,N$ and rules out global zero-set, quotient, analytic, and ambient-completeness replacements.

Finally compose with \autoref{thm:critical-line-witness-modulus-comparison-route-totality}. The modulus route and the rational comparison route share the displayed transport, continuation, provenance, and local naming rows, so the route is total through the finite packet and cannot acquire host equality or RH-facing proof data.
\end{proof}

\begin{theorem}[Critical-line witness downstream refusal export]
\label{thm:critical-line-witness-downstream-refusal-export}
Let $W=(Z,S,M,R,Q,H,C,P,N)$ be an accepted $\CriticalLineWitnessUp$ packet. The downstream export of $W$ is exactly the finite refusal package consisting of the root source triad $Z,S,Q$, the modulus and real-part rows $M,R$, the displayed $\hsame$ and $\Cont$ rows $H,C$, the $\Pkg$ provenance row $P$, and the local $\NameCert$ row $N$.

Any downstream RH-boundary chapter may cite that package only as a finite BEDC refusal export. It may not cite it as an $\mathsf{RH}$ proof, a global zero classifier, analytic continuation, a quotient representative, host equality, or ambient completeness.
\end{theorem}
\begin{proof}
First combine \autoref{thm:critical-line-witness-downstream-zero-source-readback} with \autoref{thm:critical-line-witness-downstream-modulus-comparison-totality}. The first theorem fixes the zero-source readback through the root source triad, and the second fixes the modulus and rational-comparison route through the displayed ledger.

Then apply the RH-refusal ledger completeness theorem, \autoref{thm:critical-line-witness-rh-refusal-ledger-completeness}. Its ledger clause says that downstream consumers can cite only $Z,S,M,R,Q,H,C,P,N$ and the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.

The export is therefore a refusal surface rather than an analytic proof surface. Since the accepted packet contains no coordinate for $\mathsf{RH}$ proof data, global zero classification, analytic continuation, quotient representatives, host equality, or ambient completeness, no downstream consumer can read such data from it.
\end{proof}

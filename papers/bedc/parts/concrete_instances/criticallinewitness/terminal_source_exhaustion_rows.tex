\begin{theorem}[Critical-line witness terminal source exhaustion]
\label{thm:critical-line-witness-terminal-source-exhaustion}
For an accepted $\CriticalLineWitnessUp$ packet $W=(Z,S,M,R,Q,H,C,P,N)$, every terminal read first exhausts the displayed finite source rows $Z,S,H,C,P,N$ before any modulus or comparison row is read. The read therefore begins inside the supplied $\ZetaZerosUp$ zero row, the supplied critical-strip row, and the local $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ route; it cannot begin from host equality, a quotient representative, a hidden numeric oracle, analytic continuation, ambient completeness, or a detached zero classifier.
\end{theorem}
\begin{proof}
Start with terminal zero-source totality, \autoref{thm:critical-line-witness-terminal-zero-source-totality}. It makes $Z,S,H,C,P,N$ the first exposed source surface of a terminal read.

Next project that source through the accepted packet carrier. The rows $H,C,P,N$ keep the read inside the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ route, so no host-level equality, quotient representative, hidden numeric oracle, analytic-continuation theorem, or ambient completeness row is available before the source surface is exhausted.

Finally use the terminal modulus-comparison scope of \autoref{thm:critical-line-witness-terminal-modulus-comparison-scope}. It allows modulus and comparison reads only after the finite source route has already been exposed, giving the stated source exhaustion order.
\end{proof}

\begin{theorem}[Critical-line witness terminal comparison ledger lock]
\label{thm:critical-line-witness-terminal-comparison-ledger-lock}
For an accepted $\CriticalLineWitnessUp$ packet $W=(Z,S,M,R,Q,H,C,P,N)$, the terminal comparison ledger is locked to the rows $M,R,Q,H,C,P,N$ together with the displayed $\RatUp$, $\RealUp$, and $\ZetaZerosUp$ source interfaces. The ledger exports only the local modulus, real-part, rational-comparison, transport, continuation, provenance, and naming rows; it excludes host equality, quotient representatives, hidden numeric oracle data, analytic continuation, and ambient completeness.
\end{theorem}
\begin{proof}
First apply terminal source exhaustion. The comparison ledger is reached only after the terminal read has exposed the supplied zero and local route rows.

Then apply terminal modulus-comparison scope, \autoref{thm:critical-line-witness-terminal-modulus-comparison-scope}. It binds every terminal comparison to $M,R,Q,H,C,P,N$ over the displayed $\RatUp$, $\RealUp$, $\ZetaZerosUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.

The exclusions follow from the same scope statement together with the terminal zero-source totality theorem. Since neither source theorem contains host equality, quotient representatives, hidden numeric oracles, analytic continuation, or ambient completeness, the terminal comparison ledger has no route to those objects.
\end{proof}

\begin{theorem}[Critical-line witness terminal refusal package]
\label{thm:critical-line-witness-terminal-refusal-package}
Every terminal RH-boundary consumer of an accepted $\CriticalLineWitnessUp$ packet receives the obligation package consisting of the source exhaustion of \autoref{thm:critical-line-witness-terminal-source-exhaustion}, the comparison ledger lock of \autoref{thm:critical-line-witness-terminal-comparison-ledger-lock}, and the RH-refusal exactness of \autoref{thm:critical-line-witness-terminal-rh-refusal-exactness}. \end{theorem}
\begin{proof}
Begin with source exhaustion. It supplies the terminal consumer with the finite source route $Z,S,H,C,P,N$ and blocks every external source row.

Next use the comparison ledger lock. It adds only $M,R,Q$ through the displayed $\RatUp$ and $\RealUp$ comparison surface while preserving the same $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ route.

Finally apply terminal RH-refusal exactness, \autoref{thm:critical-line-witness-terminal-rh-refusal-exactness}. This identifies the exported terminal boundary with $Z,S,M,R,Q,H,C,P,N$ and removes $\mathsf{RH}$ proof content, global zero classification, host equality, quotient representatives, hidden numeric oracles, analytic continuation, and ambient completeness. The three rows together form the root-unblock obligation package used by downstream RH-boundary chapters.
\end{proof}

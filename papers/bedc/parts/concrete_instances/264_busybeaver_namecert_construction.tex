\chapter{A Concrete Naming Certificate for $\BusyBeaverUp$}
\label{ch:concrete-instances-busybeaver-namecert}

The busy-beaver interface $\BusyBeaverUp$ packages a finite-state-bounded halting maximum predicate parametrised by a displayed state count and a candidate output bound. It depends on $\NameCert_{\NatUp}$ (\autoref{ch:concrete-instances-nat-namecert}), $\NameCert_{\TuringMachineUp}$ (\autoref{ch:concrete-instances-turingmachine-namecert}), and $\NameCert_{\ComputableUp}$ (\autoref{ch:concrete-instances-computable-namecert}).

\section{Carrier definition}
\label{sec:busybeaver-carrier}

A $\BusyBeaverUp$ token carries a unary state-count $n$, a finite enumeration of $n$-state two-symbol $\TuringMachineUp$ codes, a unary candidate output bound $B$, and a halted-step ledger. The carrier accepts a witness when, for every machine code in the enumeration that halts on the empty tape, its halted output and step count are bounded by $B$ in the sense of $\NameCert_{\ComputableUp}$. The classifier is $\hsame$ on output bounds at fixed unary state count; the ledger keeps the enumeration, the halted bound $B$, the per-machine readback, and the non-halting witness rows visible.

\begin{theorem}[BusyBeaver halted-bound admissibility]
\label{thm:busybeaver-halted-bound-admissibility}
For a $\BusyBeaverUp$ token with unary state-count $n$, displayed finite
enumeration $E$ of $n$-state two-symbol $\TuringMachineUp$ codes, unary bound
$B$, and halted-step ledger $L$, every code $M$ in $E$ whose empty-input row is
recorded in $L$ as halted has a $\ComputableUp$ bounded-trace row in $L$. That
row starts from the displayed empty tape, follows the finite
$\TuringMachineUp$ transition trace for $M$, reads back the halted output row,
records the halted step-count row, and compares both rows with $B$ through the
$\NatUp$ order surface.
\end{theorem}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverHaltedBound\_admissibility}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverHaltedBoundAdmissibility\_halted\_trace\_surface}
\begin{proof}
Project the accepted $\BusyBeaverUp$ packet to its finite enumeration and
halted-step ledger. Membership of $M$ in $E$ is a displayed finite row, and the
halted case is exactly the ledger branch that carries a finite
$\TuringMachineUp$ configuration trace from the empty input to a halted row.

Read that halted branch through the $\TuringMachineUp$ finite trace and tape
readback rows. They supply the endpoint configuration, output readback, and
unary step count without appealing to a host run or to a universal-machine
coding theorem.

The carrier clause for $\BusyBeaverUp$ requires this halted branch to be
repacked as a $\ComputableUp$ bounded trace. The same ledger row supplies the
bound comparisons against $B$ in the $\NatUp$ order, so the accepted witness is
admissible exactly for the halted machines already displayed in $E$.
\end{proof}

\begin{theorem}[BusyBeaver enumeration completeness]
\label{thm:busybeaver-enumeration-completeness}
The finite enumeration surface of a $\BusyBeaverUp$ token exhausts the displayed
$n$-state two-symbol $\TuringMachineUp$ code rows for its fixed unary state
count $n$. Every admitted machine row is either one of the listed enumeration
entries with its corresponding ledger branch, or it is outside the token's
carrier surface; no host universal-machine theorem, ambient coding theorem, or
external halting search is imported into the enumeration claim.
\end{theorem}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverEnumerationCompleteness\_finite\_ledger\_surface}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverEnumerationCompleteness\_append\_coverage}
\begin{proof}
Unpack the source coordinate of the token. The enumeration is a finite BEDC row
whose entries are already typed as $n$-state two-symbol $\TuringMachineUp$
codes, and the carrier consumes only entries addressed by that row.

For a displayed code row at the same state count and alphabet, enumeration
completeness is the finite coverage statement internal to the token: the row is
found at one listed position, with the halted or non-halting ledger branch
attached there. If no listed position is supplied, the row has no carrier
membership witness for this $\BusyBeaverUp$ token.

Thus the proof is finite projection from the enumeration surface, not an
argument that a universal simulator can decide which machines halt. The only
readback used downstream is the per-entry $\TuringMachineUp$ or
$\ComputableUp$ ledger row already attached to the finite list.
\end{proof}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverEnumerationCompleteness\_finite\_coverage}

\begin{theorem}[BusyBeaver bound monotonicity]
\label{thm:busybeaver-bound-monotonicity}
Let a $\BusyBeaverUp$ token with enumeration $E$, unary state-count $n$, bound
$B$, and halted-step ledger $L$ be accepted. If $B'$ is another unary
$\NatUp$ bound with $B \leq B'$ in the displayed order surface, then replacing
$B$ by $B'$ preserves the accepted witness with the same enumeration and the
same halted-step ledger. The transport changes only the bound rows; it does not
add a halting-decision row and does not assert computability of the
busy-beaver function.
\end{theorem}
\begin{proof}
Start with an accepted witness at bound $B$. Each halted entry in the finite
enumeration has, by
\autoref{thm:busybeaver-halted-bound-admissibility}, a $\ComputableUp$
bounded trace whose output and step-count rows are bounded by $B$.

Transport those two comparisons along the supplied $\NatUp$ order proof
$B \leq B'$. The enumeration row, halted trace, output readback, and step-count
ledger are unchanged; only the terminal bound comparison is replaced by the
larger unary bound.

Non-halting ledger branches are also unchanged, since they carry no halted
output comparison. Repacking the same finite ledger with the transported
comparisons yields an accepted $\BusyBeaverUp$ witness at $B'$, with no new
halting oracle and no computability claim about the global busy-beaver
function.
\end{proof}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverBoundMonotonicity\_bound\_transport}

\begin{theorem}[BusyBeaver non-halting exclusion obligation]
\label{thm:busybeaver-nonhalting-exclusion-obligation}
For a $\BusyBeaverUp$ token with unary state-count $n$, finite enumeration
$E$, unary bound $B$, and ledger $L$, a non-halting branch for an entry
$M \in E$ is only a visible finite ledger row attached to that enumeration
position. It records that the accepted packet has no halted output row for
$M$ inside $L$; it does not supply a halting-decision oracle, a complement
enumerator for halting rows, or a method for deciding whether arbitrary
$n$-state $\TuringMachineUp$ codes halt.
\end{theorem}
\begin{proof}
Project the token to the finite enumeration surface of
\autoref{thm:busybeaver-enumeration-completeness}. Each enumeration position
has exactly the ledger branch displayed at that position, so a non-halting row
is read as part of the finite packet rather than as the result of a search over
machine runs.

The halted branch has a different shape: by
\autoref{thm:busybeaver-halted-bound-admissibility}, it carries a finite
$\TuringMachineUp$ trace, a halted output row, a step-count row, and
$\ComputableUp$ bound comparisons. The non-halting branch lacks precisely
those halted endpoint fields.

Therefore the row excludes only a halted readback inside this accepted ledger.
It gives downstream consumers a visible finite boundary for this token and no
procedure for classifying machines outside the displayed branch.
\end{proof}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverNonhaltingExclusion\_obligation\_surface}

\begin{theorem}[BusyBeaver per-machine readback exactness]
\label{thm:busybeaver-per-machine-readback-exactness}
For every halted enumeration entry of an accepted $\BusyBeaverUp$ token, the
per-machine readback is exactly the bounded $\TuringMachineUp$ and
$\ComputableUp$ endpoint already attached to that entry. The readback does not
introduce a second simulator, a host computation trace, or a detached output
bound; it is the finite trace endpoint, halted output row, step-count row, and
$B$-comparison row carried by the same enumeration position.
\end{theorem}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverPerMachineReadback\_membership\_exactness}
\begin{proof}
Start from a halted entry $M$ of the finite enumeration. By
\autoref{thm:busybeaver-halted-bound-admissibility}, the ledger branch attached
to $M$ already contains the $\TuringMachineUp$ transition trace from the empty
tape to the halted configuration and the corresponding $\ComputableUp$ bounded
trace.

The finite projection from the enumeration position supplies the readback key.
It points to the same branch of $L$, so the output, step count, and bound
comparison are the endpoint rows already present there.

If the bound is transported as in
\autoref{thm:busybeaver-bound-monotonicity}, the machine trace and halted
readback remain unchanged. Only the terminal $\NatUp$ comparison is replaced,
which preserves exactness of the per-machine endpoint.
\end{proof}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverPerMachineReadback\_endpoint\_exactness}

\begin{theorem}[BusyBeaver scoped name-certificate surface]
\label{thm:busybeaver-scoped-namecert-surface}
The scoped $\NameCert_{\BusyBeaverUp}$ surface over a displayed unary state
count $n$, finite enumeration $E$, unary bound $B$, and ledger $L$ is exactly
the package consisting of halted-bound admissibility, enumeration completeness,
bound monotonicity, non-halting exclusion, and per-machine readback exactness.
Its public read exposes the enumeration, the halted bound, the halted
per-entry $\TuringMachineUp$ and $\ComputableUp$ rows, and the non-halting
ledger rows, but it does not expose busy-beaver computability or halting
decidability.
\end{theorem}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverHaltingLedger\_scoped\_namecert\_surface}
\begin{proof}
First assemble the halted side. Apply
\autoref{thm:busybeaver-halted-bound-admissibility} to obtain the bounded
$\ComputableUp$ row for each halted entry, and apply
\autoref{thm:busybeaver-per-machine-readback-exactness} to identify each
consumer read with the endpoint attached to its enumeration position.

Next assemble the enumeration and transport side. Apply
\autoref{thm:busybeaver-enumeration-completeness} to keep every admitted
machine row inside the displayed finite list, and apply
\autoref{thm:busybeaver-bound-monotonicity} to show that increasing the unary
bound changes only the terminal comparison rows.

Finally handle the negative branch. Apply
\autoref{thm:busybeaver-nonhalting-exclusion-obligation}: the non-halting rows
are finite ledger projections, not a decision procedure. Repacking these five
rows gives the scoped $\NameCert_{\BusyBeaverUp}$ surface and preserves the
stated non-claims.
\end{proof}

\begin{theorem}[BusyBeaver per-machine halted-readback determinacy]
\label{thm:busybeaver-per-machine-halted-readback-determinacy}
Let $W$ and $W'$ be accepted $\BusyBeaverUp$ witnesses with the same unary
state-count $n$ and the same displayed finite enumeration $E$ of $n$-state
two-symbol $\TuringMachineUp$ codes. Let $M$ be one entry of $E$. Suppose the
ledger of $W$ records for $M$ a halted branch from the displayed empty tape
with finite transition trace $T$, halted output row $o$, and halted step-count
row $s$, and the ledger of $W'$ records for the same entry $M$ a halted branch
from the displayed empty tape with finite transition trace $T'$, halted output
row $o'$, and halted step-count row $s'$. Then the carrier ledger classifies
the two readbacks componentwise:
$$
  T\hsame T',\qquad o\hsame o',\qquad s\hsame s'.
$$
Thus, at fixed enumeration entry, the halted readback of an accepted
$\BusyBeaverUp$ witness is determined by the deterministic machine code and the
empty-tape initial row, not by the choice of accepted witness packet.
\end{theorem}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverPerMachineReadback\_halted\_determinacy}
\begin{proof}
Project both accepted witnesses to their common enumeration row. Since the
entry is the same displayed $M\in E$, both halted branches use the same
finite-state code, the same deterministic transition table, the same blank-tape
initial configuration, and the same halting predicate. By
\autoref{thm:busybeaver-per-machine-readback-exactness}, each public readback
is the endpoint already attached to the ledger branch at that enumeration
position; no detached simulator or host computation trace is introduced.

Compare the two finite transition traces from the empty-tape initial row. Their
zero configurations are classified together because they are the displayed
initial configuration of the same machine on the same input. If two compared
current configurations are classified by $\hsame$, then
\autoref{thm:turingmachine-deterministic-step-stability} transports the common
transition-table row and classifies the next configurations by $\hsame$ again.
Induction over the finite trace rows therefore identifies the two deterministic
runs up to the first halted row exposed by the halted-step ledger.

It remains to rule out a mismatch of the displayed halted index. Once either
trace reaches a halted configuration, the halted-repeat clause of
\autoref{thm:turingmachine-halted-repeat-obligation} says that later accepted
trace rows only repeat that halted configuration. The ledger exactness clause
of \autoref{thm:turingmachine-ledger-exactness-obligation} gives no second
branch choice or fresh transition row after that point. Hence the two halted
branches for the same $M$ expose the same halted-step row in the $\NatUp$
classifier, so $s\hsame s'$.

At that common halted row the configuration traces have already been compared,
so the visible tape components and head-position readbacks are compared by the
finite readback surface of
\autoref{thm:turingmachine-bounded-readback-soundness}. The halted output rows
read from those compared endpoints are therefore classified together,
$o\hsame o'$. Combining the trace comparison, the halted-step comparison, and
the output readback comparison gives $T\hsame T'$, $s\hsame s'$, and
$o\hsame o'$ as claimed.
\end{proof}
\leanchecked{BEDC.Derived.BusyBeaverUp.BusyBeaverPerMachineReadback\_halted\_readback\_determinacy}

\begin{theorem}[BusyBeaver public enumeration-bound export]
\label{thm:busybeaver-public-enumeration-bound-export}
For a fixed unary state count and displayed enumeration, the finite
enumeration rows, bounded halted-readback rows, non-halting ledger rows,
monotone bound transport, and per-machine determinacy assemble a public
$\NameCert_{\BusyBeaverUp}$ export. The public rows are exactly the unary state
count, the finite $\TuringMachineUp$ enumeration, the unary candidate bound,
the halted trace and readback rows, the non-halting branch rows, the $\NatUp$
order comparisons, and the visible $\hsame$, $\Cont$, $\mathsf{Pkg}$, and
$\NameCert$ provenance.

The export asserts no computability of the global busy-beaver function, no
halting decidability, no asymptotic growth theorem, and no standard
recursion-theory bridge.
\end{theorem}
\begin{proof}
Start from \autoref{thm:busybeaver-scoped-namecert-surface}. It packages the
fixed unary state count, displayed finite enumeration, candidate unary bound,
halted-step ledger, non-halting witness rows, $\mathsf{BHist}$, $\hsame$,
$\Cont$, $\mathsf{Pkg}$, and local $\NameCert$ data as the scoped certificate
surface.

For each displayed enumeration entry, use
\autoref{thm:busybeaver-per-machine-halted-readback-determinacy}. The halted
trace, halted output row, and halted step-count row are determined by the
displayed machine code and empty-tape initial row, so the public export does
not depend on an additional readback choice.

Use \autoref{thm:busybeaver-bound-monotonicity} to transport the unary
candidate bound along the displayed $\NatUp$ order comparisons. Use
\autoref{thm:busybeaver-nonhalting-exclusion-obligation} to keep off-table
branches recorded as non-halting ledger rows rather than as missing public
data.

Combining these rows gives the public
$\NameCert_{\BusyBeaverUp}$ export for the fixed enumeration and bound. Since
the construction only repacks finite enumeration, bounded readback,
non-halting, order-comparison, and provenance rows, it does not export global
busy-beaver computability, halting decidability, asymptotic growth, or a
standard recursion-theory bridge.
\end{proof}

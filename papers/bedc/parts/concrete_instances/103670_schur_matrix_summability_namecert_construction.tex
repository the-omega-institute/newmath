\begin{theorem}[Schur matrix summability NameCert obligations]\label{thm:schur-matrix-summability-namecert-obligations}
For every accepted $\SchurMatrixSummabilityUp$ carrier $K=(M,B,C,W,T,R,E,H,Q,P,N)$, the local $\NameCert_{\SchurMatrixSummabilityUp}$ surface consists exactly of the following seven finite obligation rows:
\begin{enumerate}
\item carrier admission for the displayed matrix coefficient windows, row-bound ledger, column-tail ledger, source windows, transformed finite rows, regular readback, real seal, transport, continuation, provenance, and naming rows;
\item row-bound exactness, requiring every bounded-kernel claim used by the transform to be read from $B$ after the corresponding coefficient windows in $M$ have been displayed;
\item column-tail exactness, requiring every tail-smallness claim to pass through the displayed column-tail ledger $C$ and the same finite matrix-window row $M$;
\item source-window transform construction, requiring every transformed row in $T$ to be computed from $M,B,C,W$ before any transformed read is exposed;
\item regular readback, requiring source-window and transformed-row reads to pass through the $\RegSeqRatUp$ row $R$ with dyadic budgets supplied by the same finite ledger data;
\item Real-seal nonescape, requiring the endpoint row $E$ to remain downstream of $M,B,C,W,T,R$ and to export no host summability or ambient completeness field;
\item ledger preservation, requiring $H,Q,P,N$ to preserve the same finite route under componentwise transport, continuation replay, package provenance, and local naming.
\end{enumerate}
No obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:schur-matrix-summability-carrier}. Carrier admission is exactly the displayed inventory $M,B,C,W,T,R,E,H,Q,P,N$, so the local certificate has no detached coordinate for an infinite matrix, a host Schur theorem, or an external completion principle.

The bounded-kernel rows are read before the transform. The inspected coefficient windows in $M$ expose the finite row and column data, the row-bound ledger $B$ controls the displayed row kernels, and the column-tail ledger $C$ records the tail-smallness requests that remain after the finite windows have been chosen.

The source and transform rows are then forced. The only source observations are the $\StreamNameUp$ windows $W$, and the transformed finite-row surface $T$ is constructed from $M,B,C,W$. This is the Schur-bounded counterpart of the Toeplitz and Silverman--Toeplitz matrix routes referenced by $P$.

Finally the readback and seal rows close the route. The $\RegSeqRatUp$ row $R$ reads both source and transformed windows, and the $\RealUp$ seal $E$ is downstream of that readback. The structural rows $H,Q,P,N$ transport, replay, source, and name the same packet, so the seven listed obligations exhaust the local NameCert surface.
\end{proof}

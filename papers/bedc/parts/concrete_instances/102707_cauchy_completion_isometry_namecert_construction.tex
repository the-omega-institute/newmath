\chapter{A Concrete Naming Certificate for $\CauchyCompletionIsometryUp$}
\label{ch:concrete-instances-cauchy-completion-isometry-namecert}
\origin{human}

Bishop--Bridges and Kelley-style completion language treats the unit map from a metric source into its Cauchy completion as distance preserving before density or extension consumers use the completed object. The $\CauchyCompletionIsometryUp$ packet records that boundary as finite BEDC data: it reads the source metric route of \autoref{ch:concrete-instances-real-metric-namecert}, the unit route of \autoref{ch:concrete-instances-cauchycompletionunit-namecert}, the distance-preserving interface of \autoref{ch:concrete-instances-isometric-embedding-namecert}, the dense-image handoff of \autoref{ch:concrete-instances-completion-density-namecert}, and the terminal real-name seal of \autoref{ch:concrete-instances-real-namecert}. It names the unit distance-preservation surface for completion consumers without identifying quotient completion classes, choosing representatives, asserting host function equality, invoking countable choice, using open-cover compactness, assuming ambient completeness, or proving completeness of $\RealUp$.

\begin{definition}[CauchyCompletionIsometry carrier]
\label{def:cauchy-completion-isometry-carrier}
A $\CauchyCompletionIsometryUp$ carrier is a finite $\BHist$ packet
$$
J=(X,U,D_X,D_{\widehat X},R,Q,I,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $X$ is the source metric row whose displayed endpoints and finite distance requests are being admitted into the completion route.
\item $U$ is the accepted $\CauchyCompletionUnitUp$ row that sends the displayed source endpoints into the completion-facing surface.
\item $D_X$ is the source-distance witness row for the displayed source endpoints, read through the source metric and real-metric comparison data.
\item $D_{\widehat X}$ is the completion-distance witness row for the corresponding unit images.
\item $R$ is the $\RealMetricUp$ comparison row witnessing that the displayed source-distance read and completion-distance read name the same $\RealUp$ distance observation.
\item $Q$ is the accepted $\CompletionDensityUp$ row available after the unit image has been admitted, so later density consumers can cite the same source route.
\item $I$ is the accepted $\IsometricEmbeddingUp$ row that packages the source metric row, completion-facing target row, unit graph, the two distance witnesses, and the real-distance comparison.
\item $H$ records componentwise $\hsame$ transport for the source metric row, unit row, both distance witnesses, real-metric comparison, density row, isometric-embedding row, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a source distance request through the unit images, source and completion distance witnesses, the real-metric comparison, the isometric-embedding row, and the density handoff.
\item $P$ is the $\Pkg$ provenance over $\CauchyCompletionIsometryUp$, $\CauchyCompletionUnitUp$, $\RealMetricUp$, $\CompletionDensityUp$, $\IsometricEmbeddingUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CauchyCompletionIsometryUp}$ row.
\end{enumerate}
The carrier has no coordinate for quotient completion equality, selected representatives, host function equality, countable choice, open-cover compactness, ambient completeness, or completeness of $\RealUp$.
\end{definition}

\begin{theorem}[CauchyCompletionIsometry NameCert obligations]
\label{thm:cauchy-completion-isometry-namecert-obligations}
For an accepted $\CauchyCompletionIsometryUp$ carrier $J=(X,U,D_X,D_{\widehat X},R,Q,I,H,C,P,N)$, the local $\NameCert_{\CauchyCompletionIsometryUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the displayed source metric row, completion unit row, source-distance witness, completion-distance witness, real-metric comparison, completion-density row, isometric-embedding row, transports, replay, provenance, and local naming row;
\item unit endpoint routing, requiring every source endpoint used by $D_X$ to enter the completion-facing target only through the row $U$;
\item distance-pair exposure, requiring the source distance in $D_X$ and the completion distance in $D_{\widehat X}$ to be displayed before the comparison row $R$ is read;
\item isometry packaging, requiring the accepted row $I$ to bind the same $X,U,D_X,D_{\widehat X},R$ route as the local distance-preservation certificate;
\item density handoff, requiring the row $Q$ to be downstream of the same unit images and not an independent dense subset;
\item provenance exhaustion, requiring $H,C,P,N$ to transport, replay, source, and name only the listed route.
\end{enumerate}
No NameCert obligation is stored outside those rows, and none exports quotient completion equality, selected representatives, host function equality, countable choice, open-cover compactness, ambient completeness, or completeness of $\RealUp$.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cauchy-completion-isometry-carrier}. The tuple displays the source metric row, completion unit, two distance witnesses, real-metric comparison, density handoff, isometric-embedding row, transport, replay, provenance, and local name.

Unit endpoint routing follows from the placement of $U$ before either completion-facing distance witness is consumed. A source endpoint can reach the target side of the packet only as the unit image recorded by the accepted $\CauchyCompletionUnitUp$ row.

Distance-pair exposure and isometry packaging are then finite row checks. The comparison row $R$ reads only the displayed source and completion distances, while $I$ packages that same comparison as an $\IsometricEmbeddingUp$ route.

The density handoff is downstream of the same unit images. Since $Q$ is a row of the carrier and is transported only componentwise by $H$ and replayed only by $C$, it cannot supply a detached dense subset or a second completion object.

Finally $P$ and $N$ source and name exactly the displayed packet. The excluded quotient, representative, host-equality, choice, compactness, ambient-completeness, and real-completeness principles have no carrier coordinate from which a consumer could read them.
\end{proof}

\begin{theorem}[CauchyCompletionIsometry unit distance preservation]
\label{thm:cauchy-completion-isometry-unit-distance-preservation}
Let $J=(X,U,D_X,D_{\widehat X},R,Q,I,H,C,P,N)$ be an accepted $\CauchyCompletionIsometryUp$ packet. Every finite source-distance read in $D_X$ is routed through the unit row $U$ to the completion-distance read in $D_{\widehat X}$, and the equality of the two displayed distance observations is read only through the $\RealMetricUp$ comparison row $R$ and the accepted $\IsometricEmbeddingUp$ row $I$.

Thus the unit map is distance preserving at the finite NameCert surface:
$$
X\longmapsto U\longmapsto (D_X,D_{\widehat X})\longmapsto R\longmapsto I .
$$
The route is not quotient completion equality, not selected representative equality, not host function equality, and not an appeal to countable choice, open-cover compactness, ambient completeness, or completeness of $\RealUp$.
\end{theorem}
\begin{proof}
Start with a source-distance request. The source metric row $X$ and source-distance witness $D_X$ display the two source endpoints and their finite distance observation.

Apply the completion-unit row $U$. It supplies the only completion-facing images of those endpoints, so the target distance witness $D_{\widehat X}$ is attached to the unit images rather than to a chosen quotient representative or host function.

The row $R$ compares the two displayed distance observations inside the real-metric surface. It reads the same source and completion distance witnesses and therefore gives the finite equality of distance observations needed by an isometry consumer.

Finally the accepted row $I$ packages the metric rows, unit graph, distance witnesses, and real-distance comparison as the $\IsometricEmbeddingUp$ route. The structural rows $H,C,P,N$ keep that route componentwise, replayable, sourced, and locally named, with no coordinate for any excluded completion principle.
\end{proof}

\begin{theorem}[CauchyCompletionIsometry density handoff]
\label{thm:cauchy-completion-isometry-density-handoff}
Every completion-density consumer of an accepted $\CauchyCompletionIsometryUp$ packet receives the dense source image only after the distance-preserving unit route
$$
X,U,D_X,D_{\widehat X},R,I
$$
has been displayed. The $\CompletionDensityUp$ row $Q$ is therefore a handoff from the same unit images and real-distance comparison, not a separate dense subset, not a quotient-completion selector, and not an ambient completeness theorem.
\end{theorem}
\begin{proof}
Begin with a density consumer. The carrier exposes the density row $Q$ only as a row of the same packet that already contains $X,U,D_X,D_{\widehat X},R$ and $I$.

Use the unit distance-preservation theorem. Before $Q$ can be read as a completion-density handoff, the source endpoints have passed through $U$, their source and completion distances have been displayed, and the real-distance comparison has been packaged by $I$.

Transport and replay preserve this dependency. The row $H$ moves the source, unit, distance, comparison, isometric, and density rows componentwise, while $C$ replays the same order for downstream consumers.

Thus $Q$ hands off density for the displayed unit image that has already carried its distance-preservation certificate. Since the packet contains no separate dense subset, quotient selector, representative choice, host equality, compactness, ambient completeness, or $\RealUp$ completeness coordinate, the density handoff exports only that finite route.
\end{proof}

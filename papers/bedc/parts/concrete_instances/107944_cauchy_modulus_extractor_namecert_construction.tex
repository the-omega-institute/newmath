\begin{theorem}[Cauchy modulus extractor NameCert obligations]\label{thm:cauchy-modulus-extractor-namecert-obligations}

\item source exposure, requiring the $\EffectiveCauchySequenceUp$ row $E$ to be read before $M$ is exported;
\item modulus extraction, requiring $M$ to be the explicit $\CauchyModulusUp$ row exposed from $E$ for the same precision query $Q$;
\item threshold compatibility, requiring $F$ to refine only the same selected threshold row, either directly or through the doubled-window route;
\item finite-window and dyadic accounting, requiring $S$ and $D$ to be downstream of $M$ and $F$ before any $\RegSeqRatUp$ or $\RealUp$ read is consumed;

\begin{theorem}[Cauchy modulus extractor stage determinacy]\label{thm:cauchy-modulus-extractor-stage-determinacy}

Thus two accepted reads with the same transported $E,F,M,Q,H,C,P,N$ rows cannot diverge on the stage used by $S$ or on the dyadic tolerance ledger $D$.
\end{theorem}
\begin{proof}
Project the accepted carrier. The only coordinates that mention stage selection are $E,F,M,Q$ together with the downstream window and ledger rows $S,D$.

The source theorem \autoref{thm:effective-cauchy-sequence-regseqrat-handoff} places every regular rational read behind the explicit modulus row carried by $E$. The extractor reads that row as $M$ for the same query $Q$.

The fast-modulus theorems \autoref{thm:fast-cauchy-modulus-threshold-dichotomy} and \autoref{thm:fast-cauchy-modulus-window-exactness} then determine whether the direct threshold or doubled threshold is used, and identify the corresponding $S$ and $D$ rows.

Since the carrier has no additional selector coordinate, transported agreement of $E,F,M,Q,H,C,P,N$ forces the same selected stage, the same finite window, and the same dyadic ledger.
\end{proof}

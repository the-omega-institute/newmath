\begin{theorem}[CertificateCompiler root landing obligation]
\label{thm:certificate-compiler-root-landing-obligation}
Every root consumer of an accepted $\CertificateCompilerUp$ packet
$$
K=(S,T,G,E,C,H,P,N)
$$
must first read the source $\NameCert$ endpoint row $S$, the target
$\NameCert$ endpoint row $T$, the displayed compiler graph $G$, and the
edge-landing row $E$. The consumer may treat a displayed graph edge as
landed only after $S$ supplies the source carrier and classifier row, $T$
supplies the target carrier and classifier row, $G$ supplies the finite edge,
and $E$ records that the edge starts in the carrier named by $S$ and lands in
the carrier named by $T$.

No root consumer can obtain a landing witness from host compiler equality,
ambient program equivalence, optimizer correctness, decompiler behavior, or an
unrecorded target behavior outside the rows $S,T,G,E$.
\end{theorem}
\begin{proof}
Unfold the carrier in \autoref{def:certificate-compiler-carrier}. Its first
four rows are exactly the source endpoint certificate $S$, the target endpoint
certificate $T$, the displayed compiler graph $G$, and the edge-landing row
$E$.

Use the classifier definition in
\autoref{def:certificate-compiler-classifier}. Even a paired-edge comparison
begins by reading the source classifier row in $S$ and the target landing row
in $T$, so classification cannot precede endpoint availability.

Apply \autoref{thm:certificate-compiler-namecert-obligations}. Its carrier
landing obligation is the finite row assertion that every edge of $G$ reads a
source row admitted by $S$ and lands in a target row admitted by $T$ through
$E$.

The carrier has no coordinate for host compiler equality or ambient program
equivalence. Therefore a root consumer that consumes landing data is forced
through the displayed $S,T,G,E$ rows and cannot source a landing witness
elsewhere.
\end{proof}
\leanchecked{BEDC.Derived.CertificateCompilerUp.CertificateCompilerCarrier\_root\_landing\_obligation}

\begin{theorem}[CertificateCompiler root classifier transport obligation]
\label{thm:certificate-compiler-root-classifier-transport-obligation}
For every accepted $\CertificateCompilerUp$ packet
$$
K=(S,T,G,E,C,H,P,N),
$$
paired-edge classifier transport is exhausted by the source and target
endpoint rows $S,T$, the componentwise $\hsame$ transport row $H$, and the
continuation row $C$. A root consumer may compare two displayed graph edges
only by sharing the source classifier row in $S$, landing through $T$, moving
the target classifier row through $H$, and following the paired $\Cont$ routes
recorded in $C$.

The transport obligation does not admit host compiler equality, source program
equivalence, target program equivalence, or an external semantic relation as a
classifier witness.
\end{theorem}
\leanchecked{BEDC.Derived.CertificateCompilerUp.CertificateCompilerCarrier\_root\_classifier\_transport\_obligation}
\begin{proof}
Unfold \autoref{def:certificate-compiler-carrier}. The transport-relevant rows
available to a root consumer are the endpoint certificates $S,T$, the
componentwise transport row $H$, and the continuation compatibility row $C$,
with $G$ and $E$ only displaying the edges whose classifiers are compared.

Now unfold \autoref{def:certificate-compiler-classifier}. It defines
compatibility for paired graph edges by a common source classifier row, a
target landing, componentwise $\hsame$ transport, and paired continuation
routes. These are precisely the rows $S,T,H,C$ named in the statement.

Apply the classifier-transport clause of
\autoref{thm:certificate-compiler-namecert-obligations}. That clause permits
transport from the source classifier row to the target classifier row only by
the displayed finite-row data already carried by the packet.

Since neither the carrier nor the classifier contains host compiler equality
or a program-equivalence coordinate, a root consumer cannot certify paired
edges by such an external equality. The finite row surface $S,T,H,C$ exhausts
classifier transport.
\end{proof}

\begin{theorem}[CertificateCompiler root continuation route obligation]
\label{thm:certificate-compiler-root-continuation-route-obligation}
Every downstream root consumer of an accepted $\CertificateCompilerUp$ packet,
including $\CertificatePipelineStageBoundaryUp$, $\KernelMorphismUp$, and
$\DecompilerRightInverseUp$, reads continuation compatibility only through the
displayed $\Cont$ row $C$, the package provenance row $P$, and the local
$\NameCert_{\CertificateCompilerUp}$ row $N$ together with the endpoint
$\NameCert$ rows $S,T$. The consumer route factors through the recorded source
and target continuation pair, the named package surface, and the local
certificate row; it does not factor through an unstated compiler semantics or
decompiler correctness theorem.
\end{theorem}
\leanchecked{BEDC.Derived.CertificateCompilerUp.CertificateCompilerCarrier\_root\_continuation\_route\_obligation}
\begin{proof}
Unfold the carrier in \autoref{def:certificate-compiler-carrier}. The only
continuation coordinate is $C$, and the rows that publish it to downstream
consumers are the provenance row $P$ and the local naming row $N$ over the
endpoint certificates $S,T$.

Use \autoref{def:certificate-compiler-classifier}. Its paired-edge comparison
does not stop at landing; it also reads the source and target $\Cont$ routes
recorded in $C$, so any classifier-compatible downstream route has already
passed through the displayed continuation row.

Apply \autoref{thm:certificate-compiler-namecert-obligations}. The
continuation-compatibility clause states that each source route used by a
graph edge is paired in $C$ with the target route followed after compiling,
while the package and local naming rows can only repack these displayed
obligations.

The three named root consumers each read a different downstream surface:
pipeline-stage boundaries read adjacent compiler routes, kernel morphisms read
source-to-target preservation, and decompiler right-inverse packets read the
compiler/decompiler interface. In all three cases the available route data are
the finite rows $C,P,N,S,T$ and the local $\NameCert$ obligations, so no
unstated compiler semantics or decompiler correctness theorem can enter the
continuation route.
\end{proof}

\begin{theorem}[CertificateCompiler root landing bridge determinacy]
\label{thm:certificate-compiler-root-landing-bridge-determinacy}
Let $K=(S,T,G,E,C,H,P,N)$ be an accepted $\CertificateCompilerUp$ packet. Every consumer-facing bridge use of root landing data for $K$, including a displayed edge route, classifier transport, endpoint readback, continuation readback, package handoff, or local naming handoff, factors through the local row surface
$$
S,T,G,E,C,H,P,N.
$$
More precisely, landing is determined by $S,T,G,E$, paired-edge classifier transport is determined by $S,T,H,C$, continuation route readback is determined by $C$ over the endpoint rows $S,T$, and the handoff rows $P,N$ can only repack those displayed obligations. No host compiler equality, ambient program equivalence, optimizer correctness, decompiler behavior, external semantic relation, or unrecorded target behavior supplies an additional bridge coordinate.
\end{theorem}
\leanchecked{BEDC.Derived.CertificateCompilerUp.CertificateCompilerCarrier\_root\_landing\_bridge\_determinacy}
\begin{proof}
Start with root landing. By \autoref{thm:certificate-compiler-root-landing-obligation}, a root consumer may treat a displayed graph edge as landed only after reading the endpoint rows $S,T$, the displayed graph row $G$, and the edge-landing row $E$. Thus every bridge use whose first demand is endpoint readback, graph readback, or target landing is already forced through $S,T,G,E$.

Next read classifier transport. By \autoref{thm:certificate-compiler-root-classifier-transport-obligation}, paired-edge comparison is exhausted by the source and target endpoint rows $S,T$, the componentwise $\hsame$ row $H$, and the continuation row $C$. Hence a consumer cannot turn a landing bridge into an independent classifier witness: the only accepted transport reads the same endpoints, moves the target classifier through $H$, and follows the paired $\Cont$ routes in $C$.

Now apply \autoref{thm:certificate-compiler-root-continuation-route-obligation}. A downstream route consumer reads continuation compatibility through $C$, package provenance through $P$, and local naming through $N$ together with the endpoint certificates $S,T$. Therefore route readback, continuation readback, package handoff, and local naming handoff do not create a second bridge surface; they publish the same finite landing and transport rows to downstream consumers.

Combining the three obligations gives determinacy. The landing rows $S,T,G,E$ decide which displayed edge has landed, the transport rows $S,T,H,C$ decide which paired-edge classifier comparison is available, and the rows $C,P,N$ decide how that route is handed to later consumers. Each listed row has a fixed role in the packet, while the carrier and classifier supply no coordinate for host compiler equality, ambient program equivalence, optimizer correctness, decompiler behavior, an external semantic relation, or unrecorded target behavior. Any attempted bridge coordinate outside $S,T,G,E,C,H,P,N$ therefore has no row to read, and every permitted consumer-facing use factors through the cited local rows.
\end{proof}

\begin{corollary}[CertificateCompiler root landing consumer determinacy]
\label{cor:certificate-compiler-root-landing-consumer-determinacy}
Let $K=(S,T,G,E,C,H,P,N)$ be an accepted $\CertificateCompilerUp$ packet, and fix the consumer-visible rows cited by \autoref{thm:certificate-compiler-root-landing-bridge-determinacy}: the source endpoint row $S$, the target endpoint row $T$, the displayed compiler graph $G$, the edge-landing row $E$, the continuation row $C$, the componentwise $\hsame$ transport row $H$, the package provenance row $P$, and the local $\NameCert_{\CertificateCompilerUp}$ row $N$. Then any accepted consumer-facing use of root landing data for $K$ must pick the same finite displayed edge route, the same continuation readback route, the same package or local naming handoff, and the same classifier-exactness decision determined by those rows. It cannot pick an alternate hidden coordinate, a duplicate source classifier row, a second endpoint readback source, host compiler equality, ambient program equivalence, optimizer correctness, decompiler behavior, an external semantic relation, or unrecorded target behavior.
\end{corollary}
\begin{proof}
By \autoref{thm:certificate-compiler-root-landing-obligation}, root landing can be consumed only after the consumer has read $S,T,G,E$. Thus the landing part of any accepted use is already pinned to the source endpoint certificate, target endpoint certificate, displayed graph edge, and edge-landing row named in the packet. There is no carrier coordinate from which the consumer could obtain another landing witness, endpoint readback, or source classifier row.

Apply \autoref{thm:certificate-compiler-root-landing-bridge-determinacy}. It lists the consumer-facing bridge uses of the same root landing data: displayed edge route, classifier transport, endpoint readback, continuation readback, package handoff, and local naming handoff. The theorem states that each such use factors through the finite row surface
$$
S,T,G,E,C,H,P,N.
$$
It also separates the roles of the rows: $S,T,G,E$ determine landing, $S,T,H,C$ determine paired-edge classifier transport, $C$ over $S,T$ determines continuation readback, and $P,N$ only repack those obligations for package and local naming handoff.

The remaining alternatives are therefore ruled out by absence of a row to read. A claimed alternate finite route would have to alter the displayed graph, edge landing, continuation row, or handoff row, but those are fixed as $G,E,C,P,N$. A claimed different normal form would have to use an identity, composite, or readback surface outside the displayed packet, but \autoref{thm:certificate-compiler-root-landing-bridge-determinacy} permits only the listed bridge surface for consumer-facing use. A claimed different classifier-exactness decision would have to replace the source classifier row in $S$, the target classifier row in $T$, the componentwise transport row $H$, or the paired $\Cont$ row $C$, but the classifier transport part of the bridge determinacy theorem exhausts exactly $S,T,H,C$.

Consequently, once the cited local rows are fixed, every accepted consumer-facing use lands on the same displayed route, readback route, handoff, or exactness decision named by the $\CertificateCompilerUp$ chapter. A hidden coordinate, duplicate classifier source, or external semantic witness is not an accepted consumer source for root landing data.
\end{proof}
\leanchecked{BEDC.Derived.CertificateCompilerUp.CertificateCompilerCarrier\_root\_landing\_consumer\_determinacy}

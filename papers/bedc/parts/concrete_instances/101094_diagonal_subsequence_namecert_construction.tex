\begin{theorem}[Diagonal subsequence NameCert obligations]
\label{thm:diagonal-subsequence-namecert-obligations}
For every accepted $\DiagonalSubsequenceUp$ carrier
$$
D=(S,W,J,R,Q,E,H,C,P,N),
$$
the local $\NameCert_{\DiagonalSubsequenceUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the source-sequence windows, finite diagonal window ledger, monotone unary selector row, $\RegSeqRatUp$ readback, $\DyadicRatCoreUp$ tolerance ledger, $\RealUp$ seal, componentwise transport, displayed replay, provenance, and local naming rows;
\item finite-window diagonal compatibility, requiring each diagonal stage in $W$ to expose only the source and target windows selected from $S$ for the same displayed tolerance request;
\item selector monotonicity, requiring $J$ to witness that later diagonal stages read strictly later source positions by a finite unary trace;
\item readback exactness, requiring $R$ to be assembled only from the selected windows and tolerance entries recorded in $S,W,J,Q$;
\item $\RealUp$ seal non-escape, requiring $E$ to be downstream of the finite source-window, diagonal-ledger, selector, readback, and tolerance route;
\item componentwise $\hsame$ transport through $H$ for all displayed analytic and structural rows;
\item displayed $\Cont$ replay through $C$ for every finite diagonal request;
\item $\Pkg$ provenance through $P$ for the same dependency surface;
\item local naming through $N$ and refusal of quotient streams, countable-choice selectors, host limits, trichotomy, ambient $\RealUp$ completeness, Lean verification, and bridge checking as certificate data.
\end{enumerate}
No diagonal subsequence obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:diagonal-subsequence-carrier}. The visible analytic rows are exactly the source windows $S$, finite diagonal ledger $W$, monotone selector row $J$, regular rational readback $R$, dyadic tolerance ledger $Q$, and terminal real seal $E$.

Finite-window diagonal compatibility is read from $S$ and $W$. A diagonal request chooses one finite stage, and the ledger exposes only the source and target windows assigned to that same displayed tolerance request.

Selector monotonicity is read from $J$. Since $J$ is a finite unary trace inside the packet, the local certificate can check that each later diagonal stage uses a later source position without importing a countable-choice schedule.

Readback exactness is the projection to $R$ and $Q$. The regular rational row is assembled from the selected windows and dyadic tolerance entries already displayed by $S,W,J,Q$, so no quotient stream or detached representative is available.

The seal, transport, replay, provenance, and naming rows preserve the same route. The terminal seal $E$ is downstream of $S,W,J,R,Q$, while $H,C,P,N$ transport, replay, record, and name only the listed packet rows.
\end{proof}

\begin{theorem}[Group center abelian normal subgroup package]
\label{thm:group-center-abelian-normal-subgroup-package}
For a BEDC history-level group operation $\mathsf{mul}$ over
$\mathsf{BHist}$ with associativity, left and right identity laws for $e$,
multiplication congruence, and two-sided inverse laws, the center carrier
$\mathsf{Z}(G)$ of \autoref{def:group-center-carrier} has all
normal-subgroup rows of \autoref{thm:group-center-normal-subgroup} and also
the abelian row
$$
\forall x,y:\mathsf{BHist},\quad
\mathsf{Z}(G)(x)\Rightarrow
\mathsf{Z}(G)(y)\Rightarrow
\hsame(\mathsf{mul}(x,y),\mathsf{mul}(y,x)).
$$
The ten-field fit is: traditional target center as an abelian normal subgroup;
BEDC source $\GroupUp$ over $\mathsf{BHist}$; carrier $\mathsf{Z}(G)$;
classifier $\hsame$; stability from the checked normal-subgroup theorem plus
center commutativity at each test history; ledger central endpoints $x,y$ and
the two product orders $\mathsf{mul}(x,y)$ and $\mathsf{mul}(y,x)$; core
theorem list \autoref{thm:group-center-normal-subgroup} and
\autoref{def:group-center-carrier}; exactness the displayed commutativity row
together with the normal-subgroup rows; standard bridge ordinary group center;
status $\mathsf{ProofTarget}$.
\end{theorem}
\begin{proof}
The normal-subgroup part is exactly
\autoref{thm:group-center-normal-subgroup}. It supplies the unit, product,
inverse, classifier transport, conjugation, and centralizer-normalizer
inclusion rows for the carrier $\mathsf{Z}(G)$.

For the abelian row, fix $x,y:\mathsf{BHist}$ and assume
$\mathsf{Z}(G)(x)$ and $\mathsf{Z}(G)(y)$. Unfold
\autoref{def:group-center-carrier}. The witness $\mathsf{Z}(G)(x)$ is a
universal commutativity certificate against every test history, so applying it
to the test history $y$ gives
$$
\hsame(\mathsf{mul}(x,y),\mathsf{mul}(y,x)).
$$

The carrier witness for $y$ is part of the abelian normal-subgroup package
interface, and it is used by the product and inverse rows inherited from
\autoref{thm:group-center-normal-subgroup}. The displayed pointwise
commutativity itself only needs the center witness for $x$ at the chosen test
history $y$.
\end{proof}
\begin{theorem}[Group singleton public operation surface]
\label{thm:group-singleton-public-operation-surface}
For the concrete singleton history group instance of
\autoref{def:concrete-singleton-history-group-instance}, the public carrier is
$A_{\emp}$, multiplication is $\mathsf{append}$, unit is $\emp$, inverse is the
constant operation $h\mapsto\emp$, and the public classifier is $R_{\emp}$.
This surface is packaged by the following four clauses.
\begin{enumerate}
  \item Multiplication, unit, and inverse endpoints remain in the carrier
  $A_{\emp}$ and are classified by $R_{\emp}$ whenever their inputs are carried.

  \item Every public expression generated from carried endpoints by
  multiplication, unit, and inverse has readback classified with $\emp$.

  \item Left and right singleton append contexts preserve and reflect the
  public classifier by the existing append-context cancellation rows.

  \item The source, pattern, and ledger predicates expose only singleton
  carrier witnesses and append or inverse expression provenance. Hence the
  public surface admits no endpoint outside $A_{\emp}$.
\end{enumerate}
\end{theorem}
\begin{proof}
The carrier, classifier, multiplication, unit, and inverse are exactly the
objects named in \autoref{def:concrete-singleton-history-group-instance}.
Theorem~\ref{thm:group-singleton-history-semantic-namecert} packages them as
the singleton-history $\mathsf{SemanticNameCert}$ for $\GroupUp$, so the source
and classifier fields of the public surface are not additional data.

For the endpoint clauses, use
Theorem~\ref{thm:concrete-singleton-history-group-laws} together with the
singleton endpoint readback row recorded by
$\mathsf{BEDC.Derived.GroupUp.GroupSingletonHistory\_laws}$. Multiplication
closure carries $\mathsf{append}(h,k)$ from carried inputs; the unit row carries
$\emp$; inverse closure carries the constant inverse endpoint $\emp$; and the
readback row classifies each generated endpoint against $\emp$ by $R_{\emp}$.

Public expressions are built by induction on the expression tree. The base
case is a carried endpoint, whose singleton witness reads back to $\emp$. The
unit and inverse cases use the constant endpoint $\emp$. The multiplication
case applies multiplication closure to the two inducted carrier witnesses and
then composes their readback classifiers through the append law supplied by the
same checked singleton group rows.

For contextual visibility, apply
Theorem~\ref{thm:group-singleton-classifier-append-context-cancel} to the left
append contexts. The right-context row in the same singleton block gives the
corresponding suffix form, and the two-sided row combines the two orientations.
Thus wrapping a public endpoint by singleton append contexts neither creates
nor hides a classifier witness.

Finally, the no-hidden-leakage clause is just the exactness content of the
singleton certificate. Source predicates expose carrier witnesses
$A_{\emp}(h)$, pattern predicates expose only the empty endpoint and append
shape used by the generated expression, and ledger rows record the concrete
append or constant-inverse provenance. Any admitted public endpoint therefore
carries a witness to $A_{\emp}$, so no endpoint outside the singleton carrier
is admitted by this operation surface.
\end{proof}
\begin{definition}[Group singleton terminal standard reference]
\label{def:group-terminal-standard-reference}
Let $T_G^\ast$ denote the host-facing terminal group reference with one element
$\star_G$. Its equality relation is terminal equality, so every pair of
host-facing elements is identified. Its multiplication row is
$\star_G\cdot\star_G=\star_G$, its unit row names $\star_G$ as the unit, and its
inverse row sends $\star_G$ to $\star_G$.

The finite theorem-preservation list for $T_G^\ast$ contains exactly the
terminal rows matched to the public singleton surface: multiplication closure,
left and right unit, associativity, two-sided inverse, and append-context
cancellation. The BEDC counterparts are the clauses of
\autoref{thm:group-singleton-public-operation-surface} backed by
$\mathsf{BEDC.Derived.GroupUp.GroupSingletonHistory\_laws}$.
\end{definition}

\begin{theorem}[Group singleton terminal standard bridge]
\label{thm:group-singleton-terminal-standard-bridge}
The public singleton $\GroupUp$ surface of
\autoref{thm:group-singleton-public-operation-surface} supplies a paper-level
$\StdBridge(\GroupUp,T_G^\ast)$. The encode field sends the unique point
$\star_G$ of $T_G^\ast$ to the BEDC endpoint $\emp$. The decode field sends
every endpoint carried by $A_{\emp}$ to $\star_G$. The decode-encode round trip
is terminal equality in $T_G^\ast$, and the encode-decode round trip is the
classifier witness $R_{\emp}(h,\emp)$ for each carried endpoint $h:A_{\emp}$.
The operation translation is the same carrier $A_{\emp}$, classifier
$R_{\emp}$, multiplication $\mathsf{append}$, unit $\emp$, and constant inverse
from the public surface. The theorem-preservation and no-host-leak fields are
the finite list and public-object restriction of
\autoref{def:group-terminal-standard-reference}.
\end{theorem}
\leanchecked{BEDC.Derived.GroupUp.GroupSingleton\_terminal\_standard\_bridge}
\begin{proof}
We check the six fields of \autoref{sec:acceptance-bridge-six-fields}. The
encode field is the constant map from $T_G^\ast$ to the empty BEDC endpoint
$\emp$. The endpoint $\emp$ is public by
\autoref{thm:group-singleton-public-operation-surface}, so the map lands in the
carrier $A_{\emp}$ and respects the classifier $R_{\emp}$.

The decode field sends every carried BEDC endpoint to the unique element of
$T_G^\ast$. Since the host carrier has one element, classifier-respect is
terminal equality: any two decoded carried endpoints are the same host point
$\star_G$.

For the decode-encode round trip, start with the unique host element, encode it
as $\emp$, and decode back to $\star_G$. This is exactly terminal equality in
$T_G^\ast$.

For the encode-decode round trip, start with a carried endpoint $h:A_{\emp}$.
Decoding gives $\star_G$, and encoding returns $\emp$. The singleton carrier
readback clause of \autoref{thm:group-singleton-public-operation-surface}
supplies $R_{\emp}(h,\emp)$, which is exactly the BEDC classifier witness
required for this round trip.

The operation translation keeps the BEDC public data fixed: carrier
$A_{\emp}$, classifier $R_{\emp}$, multiplication $\mathsf{append}$, unit
$\emp$, and constant inverse. The finite theorem-preservation field is the list
in \autoref{def:group-terminal-standard-reference}. Each row is read through
\autoref{thm:group-singleton-public-operation-surface} and the checked law
package $\mathsf{BEDC.Derived.GroupUp.GroupSingletonHistory\_laws}$. No other
host group theorem is claimed by this list.

Finally, the no-host-leak field follows from the public-object inventory in
\autoref{thm:group-singleton-public-operation-surface}. The bridge uses the
host terminal reference only to name the unique decode value and terminal
equality. The BEDC-facing surface remains $\mathsf{BHist}$, $A_{\emp}$,
$R_{\emp}$, $\mathsf{append}$, $\emp$, and the constant inverse, so no host
primitive enters the exported $\GroupUp$ object.
\end{proof}

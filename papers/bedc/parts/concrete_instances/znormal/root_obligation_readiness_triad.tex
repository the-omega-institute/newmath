\begin{theorem}[$\ZNormalUp$ componentwise NameCert transport]
\label{thm:znormal-componentwise-namecert-transport}
Let
$$
Z=(T,F,V,K,R,H,C,P,N)
$$
and
$$
Z'=(T',F',V',K',R',H',C',P',N')
$$
be accepted $\ZNormalUp$ root packets over displayed $\BHist$ rows. If the componentwise $\hsame$ transport row $H$ relates $T,F,V,K,R,C,P,N$ to $T',F',V',K',R',C',P',N'$, then the local $\NameCert_{\ZNormalUp}$ surface transports from $Z$ to $Z'$ without adding a host normalization row, quotient conversion row, or divergence oracle. The transported surface remains the same displayed terminal typed endpoint, bounded fuel, terminal-normality, total-host refusal, continuation readback, $\Cont$ replay, $\Pkg$ provenance, and local naming packet.
\end{theorem}
\begin{proof}
Project both packets through \autoref{def:znormal-root-carrier-boundary}. The only readable coordinates are the displayed $\BHist$ rows $T,F,V,K,R,H,C,P,N$ and their primed counterparts.

Apply \autoref{thm:znormal-namecert-root-obligations}. The certificate surface is carrier admission, classifier exposure, terminal-normality evidence, bounded-fuel replay, non-escape, $\Pkg$ provenance, and local naming over exactly those rows.

The sibling chapters provide the component witnesses rather than host normalization. The typed endpoint row is read through \autoref{ch:concrete-instances-typedreductionnormalform-namecert}, the normal-word row through \autoref{ch:concrete-instances-zeckendorf-normal-namecert}, and the terminal continuation row through \autoref{ch:concrete-instances-continuationtracenormalform-namecert}.

Transport along $H$ therefore moves each visible component and then rereads the same route through $C$. Since no host normalization, quotient conversion, or divergence-oracle coordinate occurs in the carrier, the transported NameCert remains kernel-scoped.
\end{proof}

\begin{theorem}[$\ZNormalUp$ total-host refusal boundary]
\label{thm:znormal-total-host-refusal-boundary}
For every accepted $\ZNormalUp$ root packet
$$
Z=(T,F,V,K,R,H,C,P,N),
$$
the total-host refusal row $K$ is part of the kernel-scoped $\NameCert_{\ZNormalUp}$ boundary. Any consumer that cites terminal normality from $V$ must also display $K$ together with the bounded fuel row $F$, continuation readback row $R$, $\hsame$ transport row $H$, $\Cont$ replay row $C$, $\Pkg$ provenance row $P$, and local $\NameCert$ row $N$. A consumer that omits $K$ has not crossed the total-host boundary and cannot export host conversion, quotient normalization, divergence detection, unrestricted evaluation, or global normalization as a $\ZNormalUp$ consequence.
\end{theorem}
\begin{proof}
Start with the root carrier boundary. The row $K$ is a coordinate of the displayed $\BHist$ packet, not a host-side comment about normalization.

Use \autoref{thm:znormal-namecert-root-obligations}. Terminal-normality evidence is accepted only after the bounded fuel row $F$ and the total-host refusal row $K$ have been displayed, and replay is restricted to the visible $\Cont$ route $C$.

Now read the sibling inputs. The typed reduction normal-form packet supplies a typed endpoint, the Zeckendorf normal packet supplies finite normal-word evidence, and the continuation trace normal-form packet supplies terminal continuation readback. None of these siblings supplies a host conversion, quotient normalization, divergence oracle, unrestricted evaluator, or global normalizer.

Therefore any consumer omitting $K$ lacks one of the boundary rows required by $\NameCert_{\ZNormalUp}$. The terminal-normality citation remains inside the finite kernel packet and cannot be promoted to a total-host export.
\end{proof}

\begin{theorem}[$\ZNormalUp$ terminal route exhaustion]
\label{thm:znormal-terminal-route-exhaustion}
Every terminal route exported by an accepted $\ZNormalUp$ root packet factors through
$$
T,F,V,K,R,H,C,P,N.
$$
After the terminal typed endpoint row $T$, bounded fuel row $F$, terminal-normality row $V$, total-host refusal row $K$, continuation readback row $R$, componentwise $\hsame$ transport row $H$, displayed $\Cont$ replay row $C$, $\Pkg$ provenance row $P$, and local $\NameCert_{\ZNormalUp}$ row $N$ have been read, no additional kernel row remains for host normalization, quotient conversion, divergence detection, unrestricted evaluation, global normalization, or a second terminal route.
\end{theorem}
\begin{proof}
Apply \autoref{thm:znormal-source-route-totality}. A root consumer that cites terminal normality must expose the complete source route $T,F,V,K,R,H,C,P,N$.

Apply \autoref{thm:znormal-namecert-ledger-exhaustion}. The local ledger is exhausted by the displayed $\Pkg$ and $\NameCert$ rows over that same packet, so there is no second provenance or naming coordinate for a hidden terminal route.

Finally use \autoref{thm:znormal-terminal-boundary-nonescape}. It rules out host conversion, quotient normalization, divergence oracles, unrestricted evaluation, and global normalization from the terminal boundary. Thus the displayed route is exhausted once the nine kernel rows have been read.
\end{proof}

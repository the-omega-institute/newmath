\chapter{A Concrete Naming Certificate for $\ZeckendorfCarryNormalizationUp$}
\label{ch:concrete-instances-zeckendorf-carry-normalization-namecert}
\origin{human}

The $\ZeckendorfCarryNormalizationUp$ interface packages the displayed
Zeckendorf carry witness as a finite naming certificate. It records the
source non-normal row, the target normal row, the Fibonacci-value ledger, the
non-$\hsame$ boundary, displayed transports, continuation routes, package
provenance, and a local naming row. The packet therefore separates
normalization-by-carry from history sameness while keeping value preservation
as an explicit ledger row.

The source non-normal row and target normal row are read through
\autoref{ch:concrete-instances-zeckendorf-normal-namecert}, while the carry
witness row is read through \autoref{ch:concrete-instances-zcarry-namecert}.
The value ledger and non-$\hsame$ boundary are therefore sibling-carried rows,
not standalone local data.

\begin{definition}[Zeckendorf carry-normalization carrier]
\label{def:zeckendorf-carry-normalization-carrier}
A $\ZeckendorfCarryNormalizationUp$ carrier is a finite $\BHist$ packet
$$
Z=(h,k,W,V,H,C,P,N)
$$
with the following visible coordinates:
\begin{enumerate}
\item $h$ is the displayed source history row with public carry window
$\mathtt{word\_011}$;
\item $k$ is the displayed target history row with public carry window
$\mathtt{word\_100}$;
\item $W$ is the Zeckendorf carry witness row, reading
$\mathsf{ZCarry}(h,k)$ through the local generator;
\item $V$ is the Fibonacci-value ledger row, recording that the value read
from $h$ and the value read from $k$ are the same displayed Fibonacci sum;
\item $H$ records the non-$\hsame$ boundary between $h$ and $k$ together with
the componentwise $\hsame$ transports that preserve the source row, target
row, carry witness, value ledger, package row, and local naming row;
\item $C$ stores the finite $\Cont$ routes by which consumers read the source
non-normal row, target normal row, carry row, value ledger, boundary row,
package provenance, and local naming row;
\item $P$ is the $\Pkg$ provenance over $\BHist$, $\ZeckendorfNormalUp$,
$\ZCarryUp$, Fibonacci-value ledgers, $\hsame$, $\Cont$, and local
$\NameCert$ rows;
\item $N$ is the local
$\NameCert_{\ZeckendorfCarryNormalizationUp}$ row.
\end{enumerate}
The carrier exports only the displayed carry packet and its finite value
ledger. It contains no source-target $\hsame$ proof, no global normalization
algorithm, no arithmetic model outside the Fibonacci rows, and no kernel
rewrite rule.
\end{definition}

\begin{theorem}[Zeckendorf carry-normalization NameCert obligations]
\label{thm:zeckendorf-carry-normalization-namecert-obligations}
The local $\NameCert_{\ZeckendorfCarryNormalizationUp}$ surface for
\autoref{def:zeckendorf-carry-normalization-carrier} consists of five
finite obligations:
\begin{enumerate}
\item carrier admission for the finite packet $Z=(h,k,W,V,H,C,P,N)$;
\item source and target normality, stating that $h$ is rejected by the
$\ZeckendorfNormalUp$ normal-form classifier and $k$ is accepted by it;
\item value-ledger exactness, stating that $V$ is the unique public
Fibonacci-value row exported by the packet;
\item carry-is-not-$\hsame$ preservation, stating that the carry witness $W$
does not supply $\hsame(h,k)$ and that all displayed transports preserve that
boundary;
\item local naming, stating that $P$ and $N$ name exactly the displayed
carrier rows and no additional normalization object.
\end{enumerate}
\end{theorem}
\leanchecked{BEDC.Derived.ZeckendorfCarryNormalizationUp.ZeckendorfCarryNormalizationCarrier\_namecert\_obligations}
\begin{proof}
Carrier admission is projection of the tuple in
\autoref{def:zeckendorf-carry-normalization-carrier}. The source row, target
row, carry witness, value ledger, boundary-and-transport row, continuation
routes, package provenance, and local naming row are all displayed
coordinates of one finite $\BHist$ packet.

The normality clause reads the source and target sides through $C$ and the
normal-form rows named by $P$. The source window is the non-normal
$\mathtt{word\_011}$ row, while the target window is the normal
$\mathtt{word\_100}$ row.

The value and boundary clauses are separate projections. The row $V$ records
the Fibonacci-value equality of the two displayed sums, while $H$ records
that the carry witness is not an $\hsame(h,k)$ proof and transports only the
named components.

Finally, the package and local naming rows repack the same finite surface.
Since no coordinate names a global normalizer, source-target $\hsame$ proof,
or kernel rewrite rule, the five obligations exhaust the local NameCert
surface.
\end{proof}

\begin{theorem}[Zeckendorf carry-normalization value ledger]
\label{thm:zeckendorf-carry-normalization-value-ledger}
For an accepted $\ZeckendorfCarryNormalizationUp$ carrier
$$
Z=(h,k,W,V,H,C,P,N),
$$
every public claim that the source and target carry rows have the same
Fibonacci value factors through the displayed ledger $V$. The ledger boundary
is exact: $V$ may be used to read equality of the two Fibonacci sums, but it
cannot be repacked as $\hsame(h,k)$, as a kernel rewrite, or as a global
normalization procedure.
\end{theorem}
\leanchecked{BEDC.Derived.ZeckendorfCarryNormalizationUp.ZeckendorfCarryNormalizationCarrier\_value\_ledger}
\begin{proof}
Read the accepted packet through the carrier definition. The only coordinate
that mentions Fibonacci value is $V$, and the continuation routes in $C$
expose that coordinate as a finite ledger row.

The carry witness $W$ supplies the local $\mathsf{ZCarry}(h,k)$ edge, while
the row $H$ keeps the non-$\hsame$ boundary visible under componentwise
transport. Thus a consumer can combine $W$ and $V$ to read value-preserving
carry normalization, but the consumer still lacks a source-target
$\hsame(h,k)$ witness.

The package provenance $P$ and local naming row $N$ name exactly this packet.
They do not add a global normalizer or a kernel rewrite coordinate, so every
public value-preservation read remains inside the displayed ledger boundary.
\end{proof}

\begin{theorem}[Zeckendorf carry-normalization carry and Nat dependency]
\label{thm:zeckendorf-carry-normalization-axis-nat-dependency}
For an accepted $\ZeckendorfCarryNormalizationUp$ packet
$$
Z=(h,k,W,V,H,C,P,N),
$$
the value ledger $V$ is readable only after the local carry row certified by \autoref{ch:concrete-instances-zeckendorfcarry-namecert} and the unary Fibonacci-index discipline of \autoref{ch:concrete-instances-nat-namecert} have been consumed. Thus $V$ is a sibling-dependent ledger row, not an isolated kernel-only row.
\end{theorem}
\leanchecked{BEDC.Derived.ZeckendorfCarryNormalizationUp.ZeckendorfCarryNormalizationCarrier\_axis\_nat\_dependency}
\begin{proof}
Project $Z=(h,k,W,V,H,C,P,N)$ from \autoref{def:zeckendorf-carry-normalization-carrier}. The row $W$ is the local carry witness, while $V$ is the row that records equality of the displayed Fibonacci sums.

Read $W$ through the Zeckendorf carry sibling of \autoref{ch:concrete-instances-zeckendorfcarry-namecert}. The source and target windows $h,k$ are indexed as unary Nat positions, so the Fibonacci indexing discipline is read through \autoref{ch:concrete-instances-nat-namecert} before the sums named by $V$ are meaningful.

Finally apply \autoref{thm:zeckendorf-carry-normalization-value-ledger}. Since every public value-preservation read factors through $V$ after the carry edge and Nat-index rows have been fixed, $V$ records a sibling-dependent ledger rather than a detached kernel-only certificate.
\end{proof}

\begin{theorem}[Zeckendorf carry-normalization source-target totality]
\label{thm:zeckendorf-carry-normalization-source-target-totality}
For an accepted $\ZeckendorfCarryNormalizationUp$ packet
$$
Z=(h,k,W,V,H,C,P,N),
$$
the visible source and target normality surface is total for the displayed
carry move: the source row $h$ is exposed as the public non-normal
$\mathtt{word\_011}$ row, the target row $k$ is exposed as the public normal
$\mathtt{word\_100}$ row, and every consumer route that reads the local carry
edge $W$ also reads those two source-target normality rows through the same
finite packet.
\end{theorem}
\leanchecked{BEDC.Derived.ZeckendorfCarryNormalizationUp.ZeckendorfCarryNormalizationCarrier\_source\_target\_window\_totality}
\begin{proof}
Begin with the carrier definition
\autoref{def:zeckendorf-carry-normalization-carrier}. The rows $h$, $k$, and
$W$ are coordinates of the same finite packet $Z$, with $h$ displaying the
non-normal carry window and $k$ displaying the normal target window.

Apply the NameCert obligation theorem
\autoref{thm:zeckendorf-carry-normalization-namecert-obligations}. Its source
and target normality clause makes the two classifier reads part of the admitted
certificate surface, while its carrier-admission clause keeps the carry edge
$W$ tied to those same rows.

Finally use the value ledger theorem
\autoref{thm:zeckendorf-carry-normalization-value-ledger}. Since public
value-preservation reads remain inside $V$ and the displayed packet, no
consumer can read the carry edge as a detached operation without also seeing
the source and target normality rows.
\end{proof}

\begin{theorem}[Zeckendorf carry-normalization boundary transport]
\label{thm:zeckendorf-carry-normalization-boundary-transport}
For an accepted $\ZeckendorfCarryNormalizationUp$ packet
$$
Z=(h,k,W,V,H,C,P,N),
$$
the carry-is-not-$\hsame$ boundary is preserved by the displayed transport rows
$H$ and continuation routes $C$. Transporting the source row, target row, carry
witness, Fibonacci-value ledger, provenance row, or local name along those
routes never produces an $\hsame(h,k)$ proof and never turns the carry witness
into a kernel rewrite rule.
\end{theorem}
\leanchecked{BEDC.Derived.ZeckendorfCarryNormalizationUp.ZeckendorfCarryNormalizationCarrier\_boundary\_transport}
\begin{proof}
Project the packet through
\autoref{def:zeckendorf-carry-normalization-carrier}. The row $H$ contains the
non-$\hsame$ boundary together with the componentwise transports, while $C$
records only the finite continuation routes by which consumers read the
displayed rows.

The boundary clause of
\autoref{thm:zeckendorf-carry-normalization-namecert-obligations} states that
$W$ does not supply $\hsame(h,k)$ and that all displayed transports preserve
that boundary. The value ledger theorem
\autoref{thm:zeckendorf-carry-normalization-value-ledger} keeps equality of
Fibonacci values confined to $V$.

Therefore transport through $H$ and $C$ may preserve source, target, carry,
value, package, and naming rows, but it cannot create source-target history
sameness or a kernel rewrite coordinate.
\end{proof}

\autoref{thm:zeckendorf-carry-normalization-namecert-obligations} says that

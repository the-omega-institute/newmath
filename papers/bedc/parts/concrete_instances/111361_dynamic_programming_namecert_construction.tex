Following Bellman's finite dynamic programming principle, the $\DynamicProgrammingUp$ packet records finite backward recursion data as BEDC NameCert rows. It sits after the finite path and edge ledger of \autoref{ch:concrete-instances-shortestpath-namecert}, the transition-row discipline of \autoref{ch:concrete-instances-markovchain-namecert}, the probability-row carrier of \autoref{ch:concrete-instances-probspace-namecert}, the terminal real-name seal of \autoref{ch:concrete-instances-real-namecert}, and the finite horizon counter of \autoref{ch:concrete-instances-nat-namecert}. The chapter names only finite states, finite actions, displayed transition and cost rows, horizon-indexed Bellman tables, policy readback, transport, replay, provenance, and local naming; it does not assert an infinite-horizon fixed point, host minimizer, choice-selected policy, quotient value equality, or ambient completeness theorem.

\begin{definition}[Dynamic programming carrier]
\label{def:dynamic-programming-carrier}
A $\DynamicProgrammingUp$ carrier is a finite $\BHist$ packet
$$
D=(S,A,T,C,H,V,P,R,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the finite displayed state row, sharing its path endpoints with \autoref{ch:concrete-instances-shortestpath-namecert}.
\item $A$ is the finite action row, listing exactly the actions admitted at the displayed states.
\item $T$ is the transition row, read through the finite Markov and probability surfaces of \autoref{ch:concrete-instances-markovchain-namecert} and \autoref{ch:concrete-instances-probspace-namecert}.
\item $C$ is the cost row, exposing the finite real-valued immediate costs and terminal costs through the $\RealUp$ seal.
\item $H$ is the horizon row, a $\NatUp$-indexed finite counter for the backward recursion stages.
\item $V$ is the Bellman value table, giving one displayed value row for each state and horizon stage.
\item $P$ is the policy readback row, recording only actions whose value entry is witnessed by the same finite transition and cost rows.
\item $R$ records componentwise $\hsame$ transport, displayed $\Cont$ replay, and $\Pkg$ provenance over $S,A,T,C,H,V,P$ and the cited sibling packets.
\item $N$ is the local $\NameCert_{\DynamicProgrammingUp}$ row.
\end{enumerate}
The classifier reads accepted packets only through $S,A,T,C,H,V,P,R,N$. It has no coordinate for a host minimizer, selected infinite policy, analytic contraction theorem, quotient value function, countable choice, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Dynamic programming Bellman obligations]
\label{thm:dynamic-programming-bellman-obligations}
For every accepted $\DynamicProgrammingUp$ carrier $D=(S,A,T,C,H,V,P,R,N)$, the local $\NameCert_{\DynamicProgrammingUp}$ surface consists exactly of carrier admission for finite states, finite actions, transition rows, cost rows, horizon rows, Bellman value-table rows, policy readback rows, componentwise $\hsame$ transport, displayed $\Cont$ replay, $\Pkg$ provenance, local naming, and ledger refusal for host minimizers, selected infinite policies, quotient value equality, countable choice, and ambient completeness.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:dynamic-programming-carrier}. The finite state and action rows are displayed before transition and cost rows can be read, and the horizon row fixes which backward stages may appear.

The Bellman table row $V$ is then read stage by stage over the same displayed horizon. A policy readback row $P$ is exposed only after the corresponding state, action, transition, cost, and value rows have all been displayed.

Transport, replay, provenance, and local naming are exhausted by $R$ and $N$. Since the carrier has no further coordinate beyond $S,A,T,C,H,V,P,R,N$, no accepted read can import a host minimizer, selected infinite policy, quotient value equality, countable choice, or ambient completeness.
\end{proof}

\begin{theorem}[Dynamic programming finite-horizon induction]
\label{thm:dynamic-programming-finite-horizon-induction}
Every accepted $\DynamicProgrammingUp$ Bellman table is generated by finite backward induction over the displayed $\NatUp$ horizon row. The terminal stage reads the terminal cost row from $C$ and the $\RealUp$ seal; each predecessor stage reads only the previously displayed successor-stage value row, the transition row $T$, the action row $A$, the immediate cost row $C$, and the same local NameCert route.
\end{theorem}
\begin{proof}
Project the horizon row $H$ and read it through the unary source discipline of \autoref{ch:concrete-instances-nat-namecert}. The empty endpoint is the terminal stage, so the value entry is the displayed terminal cost row sealed through $\RealUp$.

For a one-step predecessor, use the closed generation of the horizon row. The successor-stage table has already been displayed, and the new value row is computed from the finite transition row, finite action row, immediate cost row, and successor value row.

Repeating the one-step construction along the finite horizon supplies every value-table row. No step asks for an infinite horizon, a global fixed point, or an external minimizer; the induction is only the BEDC finite-history recursion named by the carrier.
\end{proof}

\begin{theorem}[Dynamic programming policy readback]
\label{thm:dynamic-programming-policy-readback}
Every policy row exported by an accepted $\DynamicProgrammingUp$ packet is a readback from the displayed Bellman table. For each displayed state and horizon stage, the row $P$ may expose an action only after the state row $S$, action row $A$, transition row $T$, cost row $C$, horizon row $H$, value row $V$, componentwise transport, replay, provenance, and local naming have all been read.
\end{theorem}
\begin{proof}
Start with a policy-facing read. The carrier forces the read through the finite state and action rows, then through the transition and cost rows that define the local Bellman entry.

Next read the horizon row and the value-table row. The policy row is not an independent selector: it is attached to the displayed value entry at the same state and horizon stage.

Finally apply the structural row $R$ and local naming row $N$. These rows preserve the finite route under transport and replay, so any action exposed by $P$ remains tied to the same Bellman table entry and cannot become a host choice function or infinite policy.
\end{proof}

\label{thm:dynamic-programming-namecert-package}

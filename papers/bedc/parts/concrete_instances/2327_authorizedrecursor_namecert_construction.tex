\chapter{A Branch-Coverage Certificate for $\AuthorizedRecursorUp$}
\label{ch:concrete-instances-authorizedrecursor-certificate-namecert}
\origin{human}

The $\AuthorizedRecursorUp$ packet records the finite certificate surface by which a kernel-emitted eliminator becomes an authorized BEDC generator. It sits between the accepted recursor authorization row and downstream recursor-generator consumers: the constructor signature, emitted eliminator, branch package, structural descent ledger, audit gate, generated output, and local \NameCert{} row are all visible packet coordinates.

The $\AuthorizedRecursorUp$ surface uses the coarser legal-eliminator packet as its authorization boundary. The sibling audit surface in \autoref{ch:concrete-instances-recursor-branch-audit-namecert} exposes the motive boundary, constructor-indexed branch matrix, and structural descent ledger as the finer branch-level read of the same recursor authorization route.

The finite authorized-recursor certificate is the upstream recursor ledger read by \autoref{ch:concrete-instances-authorizedgeneratorrecursor-namecert}. That downstream packet consumes the displayed branch exhaustion, structural descent, output naming, and audit provenance rows as a consumer relation; it does not add a second generator, a stronger eliminator principle, or an independent proof route.

\begin{definition}[AuthorizedRecursor certificate carrier]
\label{def:recursor-generator-certificate-carrier}
An $\AuthorizedRecursorUp$ certificate carrier is a finite $\BHist$ packet
$$
Q=(I,\Sigma,R,M,B,D,A,O,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $I$ is the accepted inductive-name row and $\Sigma$ is its closed constructor-signature row;
\item $R$ is the kernel-emitted recursor row for $\Sigma$;
\item $M$ is the motive row accepted for the recursor call;
\item $B$ is the finite branch package indexed by the constructors of $\Sigma$;
\item $D$ is the structural descent ledger for every recursive occurrence exposed to $B$;
\item $A$ is the audit-gate row recording kernel acceptance and source-dependency refusals for unlisted proof principles;
\item $O$ is the generated output row handed to the downstream certificate surface;
\item $H$ stores componentwise $\hsame$ transports for $\Sigma,R,M,B,D,A,O$;
\item $C$ stores displayed $\Cont$ routes for reading the signature, recursor, motive, branch, descent, audit, and output rows;
\item $P$ is the $\Pkg$ provenance over $\BHist$, constructor signatures, emitted eliminators, branch packages, structural descent, audit gates, $\hsame$, $\Cont$, and local naming;
\item $N$ is the local $\NameCert_{\AuthorizedRecursorUp}$ row.
\end{enumerate}
The carrier has no coordinate for an unlisted constructor, an extra branch, a non-structural recursive call, a hidden generator, or an external proof-trust channel.
\end{definition}

\begin{theorem}[AuthorizedRecursor branch coverage]
\label{thm:recursor-generator-branch-coverage}
For every accepted $\AuthorizedRecursorUp$ carrier $Q=(I,\Sigma,R,M,B,D,A,O,H,C,P,N)$, the branch package $B$ contains exactly the branch rows indexed by the constructor signature $\Sigma$ read by the emitted recursor $R$. A downstream consumer of $O$ has no route to an omitted constructor case and no route to an additional branch outside $\Sigma$.
\end{theorem}
\begin{proof}
Project the constructor signature $\Sigma$, emitted recursor $R$, and branch package $B$ from the carrier. The recursor row is emitted for that signature, so its branch inventory is read against the constructors named by $\Sigma$.

The branch package $B$ is a finite coordinate of the same packet. Its rows are therefore exactly the rows exposed for the constructor cases of $\Sigma$.

The transport and continuation rows $H$ and $C$ preserve the same displayed indexing for downstream reads of $O$. Since no additional branch coordinate is present in $Q$, branch coverage is exhausted by the displayed constructor signature.
\end{proof}

\begin{theorem}[AuthorizedRecursor non-escape ledger]
\label{thm:recursor-generator-non-escape-ledger}
Every generated output row $O$ of an accepted $\AuthorizedRecursorUp$ carrier factors through the emitted recursor $R$, the accepted motive $M$, the constructor-indexed branch package $B$, the structural descent ledger $D$, and the audit-gate row $A$. The packet exports no host generator, non-structural recursive call, unlisted constructor, unchecked proof principle, or channel around the local $\NameCert_{\AuthorizedRecursorUp}$ row.
\end{theorem}
\begin{proof}
Start with the carrier projection. The only generator row available to a consumer is the emitted recursor $R$, and the only accepted call data are $M$, $B$, and $D$.

Apply \autoref{thm:recursor-generator-branch-coverage}. Every branch read used to form $O$ is one of the constructor-indexed rows in $B$, and every recursive occurrence is recorded in the structural descent ledger $D$.

The audit row $A$ is part of the same finite certificate surface. It records the source and dependency boundary before the generated row is exported.

Finally, $H,C,P,N$ transport, route, package, and name the displayed rows. They add no independent generator coordinate, so every public read of $O$ remains inside the authorized recursor ledger.
\end{proof}

\begin{theorem}[AuthorizedRecursor certificate surface]
\label{thm:authorized-recursor-certificate-surface}
The carrier of \autoref{def:recursor-generator-certificate-carrier}, the branch coverage theorem of \autoref{thm:recursor-generator-branch-coverage}, and the non-escape ledger of \autoref{thm:recursor-generator-non-escape-ledger} assemble the finite $\AuthorizedRecursorUp$ NameCert surface for kernel-emitted eliminators. The surface is available to recursor-generator, TasteGate, and self-compilation consumers without treating host recursion as an untracked primitive.
\end{theorem}
\begin{proof}
The carrier names every row that may be inspected by a downstream consumer: signature, emitted recursor, motive, branch package, descent ledger, audit gate, output row, transports, routes, provenance, and local naming.

Branch coverage binds the branch package to the accepted constructor signature. The non-escape ledger binds generated outputs to the recursor, motive, branch, descent, and audit rows.

Repacking these rows gives the advertised finite \NameCert{} surface. Since every consumer read factors through the displayed packet, host recursion never appears as an untracked generator.
\end{proof}

\begin{theorem}[AuthorizedRecursor carrier exposure]
\label{thm:authorized-recursor-carrier-exposure}
Every public read of an accepted $\AuthorizedRecursorUp$ carrier projects only the displayed rows
$$
(I,\Sigma,R,M,B,D,A,O,H,C,P,N).
$$
The read may inspect the accepted inductive name, constructor signature, emitted recursor, motive, branch package, structural descent ledger, audit gate, output row, $\hsame$ transports, $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ row, and it has no coordinate for a host recursion primitive or an unlisted eliminator.
\end{theorem}
\begin{proof}
Apply \autoref{thm:recursor-generator-branch-coverage}. It binds all branch reads to the constructor signature $\Sigma$ and prevents an omitted or additional constructor case from entering through the public output row.

Use \autoref{thm:recursor-generator-non-escape-ledger}. It confines output reads to $R,M,B,D,A$ and records that no host generator, non-structural recursive call, unchecked proof principle, or naming bypass is exported.

Finally apply \autoref{thm:authorized-recursor-certificate-surface}. Repacking its finite \NameCert{} surface leaves exactly the carrier coordinates $I,\Sigma,R,M,B,D,A,O,H,C,P,N$ as the public exposure boundary.
\end{proof}

\begin{theorem}[AuthorizedRecursor structural descent locality]
\label{thm:authorized-recursor-structural-descent-locality}
Every recursive occurrence consumed by the output row $O$ of an accepted $\AuthorizedRecursorUp$ carrier is witnessed inside the displayed structural descent ledger $D$ and is transported only through $H,C,P,N$. No recursive occurrence may be supplied by a host-side recursion principle, an extra branch coordinate, or a route outside the local certificate.
\end{theorem}
\begin{proof}
Start with \autoref{thm:recursor-generator-branch-coverage}. It identifies the branch package $B$ as the constructor-indexed source for all recursive occurrences used by the emitted recursor row.

Apply \autoref{thm:recursor-generator-non-escape-ledger}. The output row $O$ factors through the structural descent ledger $D$, so every recursive occurrence consumed by $O$ must already be recorded there.

The certificate surface theorem \autoref{thm:authorized-recursor-certificate-surface} then transports the recorded occurrences only through the displayed $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows $H,C,P,N$. These rows preserve the finite ledger; they do not add a separate recursion source.
\end{proof}

\begin{theorem}[AuthorizedRecursor output consumer handoff]
\label{thm:authorized-recursor-output-consumer-handoff}
Downstream recursor-generator and self-compilation consumers may consume the output row $O$ of an accepted $\AuthorizedRecursorUp$ carrier only after the branch coverage row, non-escape ledger, and certificate-surface row are retained with the packet. The handoff exports the finite authorized recursor surface, not unrestricted host recursion or an unchecked generated eliminator.
\end{theorem}
\begin{proof}
First apply \autoref{thm:recursor-generator-branch-coverage}. It supplies the branch inventory required before any consumer can treat $O$ as an output of the accepted constructor signature.

Next apply \autoref{thm:recursor-generator-non-escape-ledger}. It attaches the motive, branch, descent, audit, and local naming restrictions that prevent $O$ from being read as a host-generated recursion principle.

Finally use \autoref{thm:authorized-recursor-certificate-surface}. With branch coverage and non-escape retained, the downstream consumer receives $O$ only as part of the finite $\AuthorizedRecursorUp$ \NameCert{} surface available to recursor-generator and self-compilation interfaces.
\end{proof}

\begin{corollary}[AuthorizedRecursor certificate strict obstruction]
\label{cor:authorized-recursor-certificate-strict-obstruction}
For an accepted $\AuthorizedRecursorUp$ carrier $Q=(I,\Sigma,R,M,B,D,A,O,H,C,P,N)$, any displayed failure, refusal, separation, or countermodel row attached to a consumer of the output row $O$ is a strict local obstruction to the attempted converse or reuse route that asks for one of the following undisplayed coordinates: a host generator, an unlisted constructor case, an extra branch, a non-structural recursive call, an unchecked proof principle, a bypass around the local $\NameCert_{\AuthorizedRecursorUp}$ row, or an external proof-trust channel. Such a row blocks only that named converse or reuse route. It cannot be reinterpreted as a global impossibility theorem, a host recursion theorem, an independent eliminator theorem, or hidden external validation for the packet.
\end{corollary}
\begin{proof}
By \autoref{def:recursor-generator-certificate-carrier}, the carrier exposes exactly the finite rows
$$
(I,\Sigma,R,M,B,D,A,O,H,C,P,N).
$$
The same definition states that the packet has no coordinate for an unlisted constructor, an extra branch, a non-structural recursive call, a hidden generator, or an external proof-trust channel. Thus any consumer-facing obstruction must be read against the displayed packet coordinates, not against an ambient host object.

Apply \autoref{thm:recursor-generator-non-escape-ledger}. Every generated output row $O$ factors through the emitted recursor $R$, the accepted motive $M$, the constructor-indexed branch package $B$, the structural descent ledger $D$, and the audit-gate row $A$. The theorem also excludes a host generator, non-structural recursive call, unlisted constructor, unchecked proof principle, and route around the local $\NameCert_{\AuthorizedRecursorUp}$ row. Hence a failure or refusal row can only reject a request that tries to add one of those missing coordinates to the displayed factorization.

The branch part is local as well. By \autoref{thm:recursor-generator-branch-coverage}, every branch read is indexed by the constructor signature $\Sigma$, and a downstream consumer has no route to an omitted constructor case or to an additional branch outside $\Sigma$. Therefore a separation or countermodel row at the branch face blocks only the attempted branch reuse that leaves the displayed signature, not the existence of recursors or eliminators in general.

Finally, \autoref{thm:authorized-recursor-carrier-exposure}, \autoref{thm:authorized-recursor-structural-descent-locality}, and \autoref{thm:authorized-recursor-output-consumer-handoff} keep all public reads of $O$ inside the same rows and their $H,C,P,N$ transport, continuation, provenance, and naming data. Since those rows add no independent generator, no external validation channel, and no host theorem coordinate, the obstruction remains exactly the local refusal of the named converse or reuse route.
\end{proof}

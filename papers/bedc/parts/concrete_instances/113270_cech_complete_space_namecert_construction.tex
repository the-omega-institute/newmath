\begin{definition}[Cech complete space carrier]\label{def:cech-complete-space-carrier}

$$

$$
with the following displayed rows:
\begin{enumerate}
\item $T$ is the $\TopologyUp$ source row, carrying the open-set classifier and the completely regular observation surface used by the completion request.
\item $U$ is the $\UniformCompleteSpaceUp$ envelope row, recording the uniform cover and Cauchy-filter completion data available from the same topological source.
\item $M$ is the $\CompleteMetricUp$ comparison row, used only when a metrizable sibling read is explicitly present.
\item $F$ is the finite Cech-filter row. It stores the nested cover, centered finite-intersection checks, and refusal boundary for non-convergent filter branches.
\item $W$ is the finite $\StreamNameUp$ window row through which cover refinements, filter stages, and completion-facing observations are inspected.
\item $S$ is the $\PolishSpaceUp$ sibling handoff row, present only as a consumer-facing comparison surface and not as the definition of the packet.

The classifier compares accepted carriers only through these eleven rows. It has no coordinate for ambient choice, quotient completion equality, selected ultrafilters, open-cover compactness, host equality, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Cech complete space NameCert obligations]\label{thm:cech-complete-space-namecert-obligations}

\end{theorem}
\begin{proof}
Project the accepted carrier in \autoref{def:cech-complete-space-carrier}. The projection exposes the topology source $T$, uniform-complete envelope $U$, complete-metric comparison row $M$, finite Cech-filter row $F$, StreamName windows $W$, Polish comparison row $S$, regularization ledger $R$, and structural rows $H,C,P,N$.

The completion-facing route starts from topology and uniform data. A request must read $T$ and $U$ before it can inspect the finite filter row $F$, and the stages of $F$ are visible only through the finite windows $W$.

The sibling comparison rows do not define the packet. Row $M$ supplies a complete-metric comparison only when such a comparison is displayed, and row $S$ supplies a Polish-space consumer surface only after the Cech-filter route has already been read.

The structural rows preserve that same route. Since none of $T,U,M,F,W,S,R,H,C,P,N$ stores ambient choice, quotient completion equality, selected ultrafilters, open-cover compactness, host equality, or ambient $\RealUp$ completeness, the listed obligations exhaust the local NameCert surface.
\end{proof}

$$

\begin{proof}

The Cech-filter evidence is then read through $F$ and $W$. The finite filter row records the cover-refinement and centered finite-intersection checks, while the StreamName row exposes only finite windows of that route.

The regularization ledger $R$ is downstream of those windows. Only after $R$ has been read may the optional complete-metric comparison $M$ or Polish comparison $S$ be consumed.

Finally $H,C,P,N$ transport, replay, source, and name the same displayed route. The excluded choice, quotient, ultrafilter, open-cover compactness, host-equality, and ambient-completeness data have no coordinate in the carrier.
\end{proof}

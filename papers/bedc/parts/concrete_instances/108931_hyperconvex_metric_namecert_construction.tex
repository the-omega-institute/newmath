\begin{theorem}[Hyperconvex metric NameCert obligations]\label{thm:hyperconvex-metric-namecert-obligations}

\item carrier admission for the displayed metric source, complete-metric seal, closed-ball family, formal-ball refinement, real-radius seal, pairwise-intersection ledger, handoff row, transport, replay, provenance, and local naming rows;
\item common-source discipline, requiring $K$, $B$, $F$, $R$, and $I$ to be read over the same $\MetricSpaceUp$ row $M$;
\item closed-ball visibility, requiring every radius and membership request to pass through $\MetricClosedBallUp$ and $\RealUp$ before the intersection handoff is exposed;
\item basis refinement, requiring the $\FormalBallBasisUp$ row to display the centre-radius refinement route used by the closed-ball family;

\end{enumerate}
No hyperconvex metric obligation is stored outside these rows.
\end{theorem}
\begin{proof}

The intersection obligation is finite. The ledger $I$ lists the pairwise compatibility inequalities for the displayed balls in $B$, and the handoff row $E$ can be consumed only after that ledger has been read.

The nonescape clauses follow from the carrier shape. None of the rows stores a selected injective hull, arbitrary ball-family intersection, quotient completion equality, countable-choice selector, or ambient $\RealUp$ completeness witness, so the listed obligations exhaust the local NameCert surface.

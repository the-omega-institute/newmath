\begin{definition}[Bishop completion carrier]
\label{def:bishopcompletion-carrier}
A $\BishopCompletionUp$ carrier is a finite $\BHist$ packet
$$

with the following displayed rows:
\begin{enumerate}
\item $R$ is the $\RegSeqRatUp$ regular rational name row, carrying the rational readback and regularity budget visible to completion-facing consumers.
\item $S$ is the $\StreamNameUp$ finite-window row from which the displayed rational observations in $R$ are read.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger attached to the same windows and precision requests.
\item $E$ is the $\RealUp$ seal row reached only after $R,S,D$ have been displayed.
\item $F$ is the $\CauchyFilterCompletionUp$ handoff row generated from the same finite tail windows and dyadic budgets.
\item $U$ is the $\CompletionUniversalPropertyUp$ extension row available only after the regular-name route and filter-completion handoff have both been exposed.
\item $H$ records componentwise $\hsame$ transport for the regular name, stream window, dyadic ledger, Real seal, filter-completion handoff, universal-property row, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a completion request through $S,R,D,E,F,U$.
\item $P$ is the $\Pkg$ provenance over $\BishopCompletionUp$, $\RegSeqRatUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, $\RealUp$, $\CauchyFilterCompletionUp$, $\CompletionUniversalPropertyUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\BishopCompletionUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through these ten rows and their displayed replay paths. It has no coordinate for quotient stream equality, a selected representative, countable choice, host real equality, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Bishop completion NameCert obligations]
\label{thm:bishopcompletion-namecert-obligations}
For every accepted $\BishopCompletionUp$ carrier $B=(R,S,D,E,F,U,H,C,P,N)$, the local $\NameCert_{\BishopCompletionUp}$ surface consists exactly of the following obligation rows:
\begin{enumerate}
\item carrier admission for the displayed regular rational name, stream window, dyadic tolerance ledger, Real seal, filter-completion handoff, universal-property row, transport, replay, provenance, and local naming row;
\item finite-window exposure, requiring every completion-facing observation to pass through $S$ before $R$ or $D$ can be read;
\item regular rational boundary exposure, requiring $R$ and $D$ to be displayed before the terminal $\RealUp$ seal $E$ becomes available;
\item filter-compatibility exposure, requiring the $\CauchyFilterCompletionUp$ row $F$ to be generated from the same $S,R,D$ route;
\item universal-property exposure, requiring $U$ to read only the already displayed regular-name, Real-seal, and filter-completion rows;
\item ledger non-escape, requiring $H,C,P,N$ to export no quotient stream equality, selected representative, countable choice, host real equality, or ambient Real completeness coordinate.
\end{enumerate}
No local NameCert obligation is stored outside those rows.
\end{theorem}
\begin{proof}
Project the accepted carrier through \autoref{def:bishopcompletion-carrier}. The visible packet consists exactly of $R,S,D,E,F,U,H,C,P,N$.

The finite-window and regular-name obligations are ordered by the carrier. A completion-facing observation first exposes the window row $S$, then reads the regular rational row $R$ and the dyadic ledger $D$ before reaching the terminal Real seal $E$.

The filter and universal-property rows are downstream of the same route. The row $F$ uses the displayed windows and dyadic budgets as its filter-completion handoff, while $U$ is read only after the regular-name, Real-seal, and filter-completion rows have been exposed.

Finally $H,C,P,N$ transport, replay, source, and name the same finite packet. Since quotient stream equality, selected representatives, countable choice, host real equality, and ambient completeness have no carrier row, the listed obligations exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[Bishop completion regular real boundary]
\label{thm:bishopcompletion-regular-real-boundary}

$$

The route exports no quotient stream, selected host limit, countable-choice schedule, host real equality, or ambient Real completeness datum.
\end{theorem}
\begin{proof}

The window row $S$ supplies the finite observations available to the read. The row $R$ records the regular rational name extracted from those observations, and $D$ records the dyadic tolerance ledger for the same precision requests.

The structural rows preserve this order. The $\hsame$ row $H$ transports the displayed coordinates componentwise, the $\Cont$ row $C$ replays the same route, and $P,N$ record provenance and local naming without adding analytic data. Thus every Real-facing read remains inside the listed finite route.

\begin{theorem}[Bishop completion filter compatibility]
\label{thm:bishopcompletion-filter-compatibility}

$$

\begin{proof}

\end{proof}

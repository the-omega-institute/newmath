$$

\begin{proof}

$$
T,V,d,C_{\mathsf{src}},C_{\mathsf{val}},Q,R,L,A,H,C,P,N.
$$

Now apply downstream non-escape, \autoref{thm:closed-term-substitution-boundary-downstream-nonescape}. After the generator, binder-budget, and compiler dependencies are combined, the downstream consumer still receives only the same displayed $\ClosedTermSubstitutionBoundaryUp$ route. The carried data are exactly the accepted closedness, operation, ledger, and audit rows transported by $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$, while the downstream surface exports no unregistered free variable, hidden binder convention, host substitution semantics, host syntax quotient, unchecked compiler correctness theorem, or ambient subject reduction.

Therefore any consumer-facing use of $O$ must factor through the source-total route and the downstream non-escape route. If the named converse or reuse route asks to omit a displayed coordinate, replace it by a host coordinate, detach it from the closed source surface, or bypass the listed structural rows, then $O$ blocks that named local attempt. Every stronger interpretation asks for a coordinate that neither source totality nor downstream non-escape exports. Hence $O$ has force only as the strict local obstruction to the named converse or reuse route, and it cannot supply a global impossibility statement, host theorem, hidden external validation, or any external validation route outside the displayed closed-term substitution boundary.
\end{proof}

\begin{theorem}[Closed-term substitution boundary primitive ledger scope]
\label{thm:closed-term-substitution-boundary-primitive-ledger-scope}
Every accepted $\ClosedTermSubstitutionBoundaryUp$ read that exposes closed shift or closed substitution binds its primitive ledger scope to the displayed $\mathsf{Term}$ rows, $\mathsf{ClosedAt}$ rows, $\mathsf{shift}$ row, $\mathsf{substitute}$ row, $\BHist$ packet, componentwise $\hsame$ transports, displayed $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ row. No accepted read obtains a closed substitution boundary from host syntax, host equality, unchecked compiler state, subject reduction, quotient representation, hidden evaluator state, or an operation row outside that primitive ledger.
\end{theorem}
\begin{proof}
Start with the carrier and classifier. They display the source term, closed value, inspected depth, closedness rows, closed shift, closed substitution, ledger, audit row, transport rows, continuation routes, package provenance, and local naming row as one finite packet.

Apply the NameCert obligations and ledger non-escape theorem. The closedness rows must be read before the operation rows, and the ledger has no projection for host substitution semantics, hidden equality, unchecked compiler state, or subject reduction.

Finally use the root route exhaustion package. Every root read is already routed through the same $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ surface, so the primitive ledger scope is exactly the listed boundary tuple.
\end{proof}

\label{thm:closed-term-substitution-boundary-downstream-factorization-triad}

\begin{proof}

Finally use the RecursorGenerator handoff theorem. The generated branch output may consume closed substitution only through the displayed continuation route, so the three downstream consumers share the same finite boundary tuple before any substitution read is visible.
\end{proof}

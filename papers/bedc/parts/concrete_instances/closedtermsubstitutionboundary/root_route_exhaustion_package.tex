\begin{theorem}[Closed-term substitution boundary root route exhaustion]
\label{thm:closed-term-substitution-boundary-root-route-exhaustion}
Every root route read of an accepted $\ClosedTermSubstitutionBoundaryUp$ packet is exhausted by the finite carrier rows
$$
T,V,d,C_{\mathsf{src}},C_{\mathsf{val}},Q,R,L,A,H,C,P,N.
$$
The route may read closedness through $C_{\mathsf{src}}$ and $C_{\mathsf{val}}$, shift through $Q$, substitution through $R$, ledger accounting through $L$, audit non-escape through $A$, componentwise $\hsame$ transport through $H$, displayed $\Cont$ replay through $C$, $\Pkg$ provenance through $P$, and local naming through $N$. It has no root coordinate for host substitution semantics, subject reduction, host syntax quotienting, hidden evaluator state, hidden compiler state, or an unlisted operation row.
\end{theorem}
\begin{proof}
Start with the carrier and the NameCert obligations. \autoref{def:closed-term-substitution-boundary-carrier} displays exactly the rows $T,V,d,C_{\mathsf{src}},C_{\mathsf{val}},Q,R,L,A,H,C,P,N$, and \autoref{thm:closed-term-substitution-boundary-namecert-obligations} makes the closedness, operation, ledger, audit, transport, continuation, package, and naming rows the local certificate surface.

Next apply the ledger and root route rows. \autoref{thm:closed-term-substitution-boundary-ledger-non-escape} blocks free-variable, host-equality, compiler-state, and subject-reduction escape, while \autoref{thm:closed-term-substitution-boundary-root-namecert-handoff} packages the root handoff as the same finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ route.

Finally use the root route determinacy package. Since any downstream route is already locked to the displayed operation and handoff rows, the root read is exhausted by the stated carrier coordinates and cannot be replaced by an unlisted host or compiler surface.
\end{proof}

\begin{theorem}[Closed-term substitution boundary root non-escape]
\label{thm:closed-term-substitution-boundary-root-nonescape}
No root consumer of an accepted $\ClosedTermSubstitutionBoundaryUp$ packet can escape from the displayed route
$$
T,V,d,C_{\mathsf{src}},C_{\mathsf{val}},Q,R,L,A,H,C,P,N
$$
to obtain a free-variable coordinate, host substitution semantics, subject reduction, host syntax quotienting, hidden evaluator state, hidden compiler state, or a compiler-correctness theorem. The closed shift and substitution reads remain guarded by the displayed closedness, ledger, audit, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.
\end{theorem}
\begin{proof}
First read the root consumer through \autoref{thm:closed-term-substitution-boundary-root-route-exhaustion}. That theorem fixes every accepted root route to the displayed carrier rows before any operation boundary is consumed.

Apply \autoref{thm:closed-term-substitution-boundary-ledger-non-escape}. The ledger and audit row have no projection for free variables, host equality, unchecked compiler state, or ambient subject reduction, so those objects cannot be exported by a root read.

Now use \autoref{thm:closed-term-substitution-boundary-root-namecert-handoff}. The handoff gives downstream consumers one finite packet over $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$, not a second host substitution or compiler-correctness surface.
\end{proof}

\begin{theorem}[Closed-term substitution boundary root downstream readiness]
\label{thm:closed-term-substitution-boundary-root-downstream-readiness}
The root route exhaustion and root non-escape rows make $\ClosedTermSubstitutionBoundaryUp$ ready for downstream self-compilation consumers exactly as a finite closed-boundary packet. A downstream chapter may consume the displayed $T,V,d,C_{\mathsf{src}},C_{\mathsf{val}},Q,R,L,A,H,C,P,N$ route, together with the NameCert obligation surface, ledger non-escape row, and root NameCert handoff, but it may not treat the packet as a host compiler theorem, a subject-reduction package, or an untracked substitution semantics.
\end{theorem}
\begin{proof}
Begin with \autoref{thm:closed-term-substitution-boundary-root-route-exhaustion}. It supplies the finite operation route that downstream consumers may read.

Next apply \autoref{thm:closed-term-substitution-boundary-root-nonescape}. The same route blocks host substitution semantics, subject reduction, host syntax quotienting, hidden evaluator state, hidden compiler state, and compiler-correctness import.

Finally compose \autoref{thm:closed-term-substitution-boundary-namecert-obligations}, \autoref{thm:closed-term-substitution-boundary-ledger-non-escape}, and \autoref{thm:closed-term-substitution-boundary-root-namecert-handoff}. These rows repack the closedness, shift, substitution, ledger, audit, transport, continuation, provenance, and naming data as the precise downstream readiness surface.
\end{proof}

\begin{corollary}[Closed-term substitution boundary root route strict local obstruction]
\label{cor:closed-term-substitution-boundary-root-route-strict-local-obstruction}
Let $O$ be a displayed failure, refusal, separation, or countermodel row cited against a named consumer-facing converse or reuse route in the root route exhaustion package for an accepted $\ClosedTermSubstitutionBoundaryUp$ packet. Then $O$ is a strict local obstruction. It blocks only the named route whose consumer-facing read factors through the finite packet rows
$$
T,V,d,C_{\mathsf{src}},C_{\mathsf{val}},Q,R,L,A,H,C,P,N
$$
and then tries to omit, replace, detach, or bypass one of those displayed coordinates. The same row cannot be reinterpreted as a global impossibility statement, a host theorem, hidden external validation, host substitution semantics, subject reduction, host syntax quotienting, hidden evaluator state, hidden compiler state, compiler-correctness evidence, or an obstruction to any consumer route that still remains inside the listed rows.
\end{corollary}
\begin{proof}
Start with \autoref{thm:closed-term-substitution-boundary-root-route-exhaustion}. Every root route read of the accepted packet is already exhausted by the listed rows. The theorem also identifies the only permitted readings of those rows: closedness is read through $C_{\mathsf{src}}$ and $C_{\mathsf{val}}$, shift through $Q$, substitution through $R$, ledger accounting through $L$, audit non-escape through $A$, componentwise $\hsame$ transport through $H$, displayed $\Cont$ replay through $C$, $\Pkg$ provenance through $P$, and local naming through $N$. Thus a consumer-facing converse or reuse route has no root entry point except the displayed finite route.

Apply \autoref{thm:closed-term-substitution-boundary-root-nonescape}. It excludes escape from the same route to a free-variable coordinate, host substitution semantics, subject reduction, host syntax quotienting, hidden evaluator state, hidden compiler state, or a compiler-correctness theorem. The closed shift and substitution reads therefore remain guarded by the displayed closedness, ledger, audit, transport, continuation, provenance, and local naming rows.

Now consider the cited row $O$. If the named converse or reuse route asks for one of the displayed coordinates and $O$ records the failure, refusal, separation, or countermodel at that coordinate, then $O$ blocks precisely that named local route. If the attempted reading asks instead for a global impossibility statement, a host theorem, hidden external validation, host substitution semantics, subject reduction, a host syntax quotient, hidden evaluator state, hidden compiler state, compiler-correctness evidence, or any unlisted operation row, the route exhaustion theorem supplies no coordinate for the request and the non-escape theorem explicitly excludes it. Hence $O$ has force only as the strict local obstruction to the named converse or reuse route, while every consumer read that continues to factor through the displayed packet rows remains a read of the accepted $\ClosedTermSubstitutionBoundaryUp$ boundary.
\end{proof}

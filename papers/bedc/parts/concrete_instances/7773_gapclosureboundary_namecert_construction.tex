\begin{definition}[GapClosureBoundary carrier]
\label{def:gap-closure-boundary-carrier}
A $\GapClosureBoundaryUp$ carrier is a finite $\BHist$ packet
$$
B=(G,S,R,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $G$ is the $\mathsf{GAP}$ request row, naming the forward-binding or socket-facing demand whose source is not generated by the closed substrate;
\item $S$ is the source observation or socket row, displaying the finite record, locality cell, aspect side, or inscription route that the request may lawfully read;
\item $R$ is the refusal and non-escape ledger, recording that an off-source closure request is not internalised;
\item $H$ records componentwise $\hsame$ transports for $G,S,R$ together with continuation, provenance, and naming rows;
\item $C$ records the displayed $\Cont$ routes from the request row to the source row, and from every unsupported closure read to the refusal row;
\item $P$ records $\Pkg$ provenance over $\BHist$, $\hsame$, $\Cont$, $\Pkg$, $\NameCert$, and the hard-problem sibling packets that consume this boundary;
\item $N$ is the local $\NameCert_{\GapClosureBoundaryUp}$ row.
\end{enumerate}
The classifier accepts exactly finite packets with these seven rows. It has no coordinate for internal universal closure, private other-minds access, host causal necessity, direct aspect reduction, or a $\mathsf{T}$ supply that bypasses the displayed source route.
\end{definition}

\label{thm:gap-closure-boundary-namecert-obligations}

\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:gap-closure-boundary-carrier}. The rows $G,S,R,H,C,P,N$ are the full inventory visible to a boundary reader, so carrier exposure contributes no hidden coordinate.

Read the source route next. Any accepted use of the $\mathsf{GAP}$ request must show the source observation or socket row $S$ before it can be consumed by a causal, refutation, other-minds, aspect-chain, or solipsism-refusal sibling.

Finally inspect a closure request with no displayed source route. The continuation row sends such a request to $R$, while $H,P,N$ merely transport, package, and name the same finite packet. Since the carrier has no internal-closure coordinate, the four obligations exhaust the local certificate surface.
\end{proof}

When $S$ is absent, the only available continuation is the refusal route into $R$. The refusal row is a ledger entry rather than a supply witness, so the excluded closure, access, necessity, reduction, hidden-source, and off-packet continuation coordinates cannot be exported.

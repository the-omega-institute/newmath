\chapter{A Concrete Naming Certificate for $\DenseExtensionMetricUp$}
\label{ch:concrete-instances-dense-extension-metric-namecert}
\origin{human}

Following Bishop and Bridges and Kelley, the $\DenseExtensionMetricUp$ packet records the finite BEDC surface for extending a uniformly continuous map from a dense metric source into a displayed complete metric target. It consumes the metric-completion route of \autoref{ch:concrete-instances-metriccompletion-namecert}, the unit admission route of \autoref{ch:concrete-instances-cauchycompletionunit-namecert}, the complete target boundary of \autoref{ch:concrete-instances-completemetric-namecert}, and the naming spine of \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, \autoref{ch:concrete-instances-dyadicratcore-namecert}, and \autoref{ch:concrete-instances-real-namecert}. The packet is a constructive dense-extension certificate over displayed rows; it is not quotient completion equality, not a selected host limit, not countable choice, and not a completeness theorem for $\RealUp$.

\begin{definition}[Dense extension metric carrier]
\label{def:dense-extension-metric-carrier}
A $\DenseExtensionMetricUp$ carrier is a finite $\BHist$ packet
$$
E=(X,J,\omega,M,Y,S,R,D,Q,T,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $X$ is the source metric row, including the source point histories and distance-read interface inspected before completion data are consumed.
\item $J$ is the dense embedding row, recording the displayed source-to-completion admission through the $\CauchyCompletionUnitUp$ and the dense image boundary used by completion consumers.
\item $\omega$ is the uniform-continuity modulus row for the source map on dense points, carrying the finite tolerance conversion from source metric reads to target metric reads.
\item $M$ is the $\MetricCompletionUp$ handoff row through which completion-facing source names are read.
\item $Y$ is the $\CompleteMetricUp$ target row, carrying the displayed complete-metric limit interface for the transported image stream.
\item $S$ is the $\StreamNameUp$ finite-window route for the dense source names and transported image names.
\item $R$ is the $\RegSeqRatUp$ readback route for rational regularity checks attached to $S$.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger, storing the dyadic budgets consumed by $X,J,\omega,M,Y,S,R$.
\item $Q$ is the $\RealUp$ readback seal reached only after the stream, regular-rational, and dyadic routes have been displayed.
\item $T$ is the extension transport row, assigning each admitted completion-side source name a target read through $Y$ and recording agreement with the dense source map on the image of $J$.
\item $H$ is the componentwise $\hsame$ transport ledger for the source metric, dense embedding, modulus, completion handoff, complete target, stream, rational, dyadic, real seal, and extension transport rows.
\item $C$ is the displayed $\Cont$ replay route forcing every extension read to pass through
$$
X,J,\omega \longmapsto M,S,R,D \longmapsto Y,Q,T .
$$
\item $P$ is the $\Pkg$ provenance over $\DenseExtensionMetricUp$, $\MetricCompletionUp$, $\CauchyCompletionUnitUp$, $\CompleteMetricUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\DenseExtensionMetricUp}$ row.
\end{enumerate}
No coordinate of the carrier stores quotient completion equality, selected representatives, countable-choice schedules, host equality of completed objects, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Dense extension metric NameCert obligations]
\label{thm:dense-extension-metric-namecert-obligations}
For an accepted $\DenseExtensionMetricUp$ carrier $E=(X,J,\omega,M,Y,S,R,D,Q,T,H,C,P,N)$, the local $\NameCert_{\DenseExtensionMetricUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the source metric, dense embedding, modulus, metric-completion handoff, complete target, stream windows, regular readbacks, dyadic ledger, real seal, extension transport, componentwise transport, replay, provenance, and local naming rows;
\item dense-entry exposure, requiring every completion-side source read to enter through $J$ before $M$ is consumed;
\item modulus transport, requiring every target Cauchy request to use $\omega$ after the source distance reads in $X$ and before the complete-target row $Y$ is queried;
\item completion handoff, requiring source completion names to pass through $M,S,R,D,Q$ before an extension value is offered;
\item target completeness handoff, requiring the transported image stream to reach a target read only through $Y$ and $T$;
\item dense agreement, requiring $T$ to restrict to the original dense-source map on the image carried by $J$;
\item ledger non-escape, stating that $H,C,P,N$ export no quotient completion, selected representative, countable-choice schedule, host equality, or ambient $\RealUp$ completeness theorem.
\end{enumerate}
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:dense-extension-metric-carrier}. The tuple lists the source metric, dense embedding, modulus, completion handoff, complete target, stream, rational, dyadic, real, transport, replay, provenance, and local naming rows.

Dense-entry exposure uses $J$ as the only carrier row connecting source points to completion-facing names. Thus a completion-side source read must pass through the displayed dense embedding before $M$ can be consumed.

Modulus transport uses $\omega$ between the source metric row $X$ and the target complete-metric row $Y$. The modulus row is the only tolerance-conversion coordinate in the packet, so every transported image request is routed through it.

Completion handoff is the ordered read of $M,S,R,D,Q$. The metric-completion row supplies the completion-facing source name, while the stream, regular-rational, dyadic, and real-seal rows provide the finite readback surface required before the target extension row is reached.

The complete-target and dense-agreement clauses are then projections to $Y$ and $T$. The target read is accepted only through the displayed complete metric row, and $T$ records agreement on the dense image carried by $J$.

Finally, $H,C,P,N$ transport, replay, source, and name the same finite route. Since no excluded host-side object appears as a coordinate, the listed obligations exhaust the local certificate surface.
\end{proof}

\begin{theorem}[Dense extension metric completion route]
\label{thm:dense-extension-metric-completion-route}
Let $E=(X,J,\omega,M,Y,S,R,D,Q,T,H,C,P,N)$ be an accepted $\DenseExtensionMetricUp$ carrier. Every completion-side extension read exported by $E$ factors through
$$
X,J,\omega \longmapsto M,S,R,D,Q \longmapsto Y,T \longmapsto H,C,P,N .
$$
Consequently the packet exports a displayed dense-extension handoff from source metric rows to complete target rows, not quotient completion equality, not a selected limit, not countable choice, and not ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
Start with a completion-side source read. The carrier first exposes the source metric row $X$ and the dense embedding row $J$, so the request enters the completion interface through displayed dense-source data.

The modulus row $\omega$ is then consumed before target convergence can be queried. This fixes the finite tolerance conversion used to transport source Cauchy evidence into the target metric row.

The completion handoff is read through $M,S,R,D,Q$. These rows provide the metric-completion surface, finite windows, regular rational readbacks, dyadic budgets, and terminal real seal for the source-side request.

Only then can the complete target row $Y$ and extension transport row $T$ be consumed. The structural rows $H,C,P,N$ preserve that route componentwise, so no quotient completion, selected limit, choice schedule, host equality, or $\RealUp$ completeness theorem is introduced.
\end{proof}

\begin{theorem}[Dense extension metric downstream non-escape]
\label{thm:dense-extension-metric-downstream-non-escape}
Every $\MetricCompletionUp$, $\CompleteMetricUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, or $\RealUp$ consumer of an accepted $\DenseExtensionMetricUp$ carrier receives only the rows named in \autoref{def:dense-extension-metric-carrier}. In particular, downstream consumers receive no quotient completion equality, selected representative, countable-choice schedule, host equality of completed objects, or completeness theorem for $\RealUp$.
\end{theorem}
\begin{proof}
A $\MetricCompletionUp$ consumer receives $M$ only after the source metric row $X$, the dense embedding row $J$, and the modulus row $\omega$ have been displayed. The stream, regular-rational, dyadic, and real-seal consumers are confined to $S,R,D,Q$ along the same route.

A $\CompleteMetricUp$ consumer receives $Y$ only after the source completion handoff and finite readback rows have been fixed. The extension transport row $T$ records the target read and dense agreement, but it has no coordinate for a host completion object or a chosen representative.

The structural rows $H,C,P,N$ preserve and name this route. Since every downstream read is a projection from the same finite packet, the excluded quotient, choice, host-equality, and ambient-completeness data cannot escape to any listed consumer.
\end{proof}

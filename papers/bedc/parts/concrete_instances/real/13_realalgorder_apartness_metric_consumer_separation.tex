\begin{theorem}[RealAlgOrder apartness metric consumer separation]
\label{thm:realalgorder-apartness-metric-consumer-separation}
For constant rational real names, any $\MetricSpaceUp$ or $\SOneUp$ consumer of
the public apartness and metric export of
\autoref{thm:realalgorder-public-apartness-metric-export} can read only the
rationally transported carrier, classifier, algebra, order, apartness,
positive-gap handoff, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows displayed
by the $\RealAlgOrderUp$ packet.

The same consumer surface does not supply a trichotomy row, completed-real
equality, quotient representative, or completeness datum. Apartness and the
positive-gap metric boundary are therefore consumer-visible separation rows,
not an ambient completed-real order or quotient-equality principle.
\end{theorem}
\begin{proof}
Start with
\autoref{thm:realalgorder-constant-rational-scoped-certificate}. The scoped
certificate fixes the constant-rational carrier by a $\RatUp$ witness embedded
into $\RealUp$, and it lists the classifier, algebra, order, apartness,
transport, continuation, provenance, and local name rows as the available
coordinates.

Apply \autoref{thm:realalgorder-public-apartness-metric-export}. Its public
projection is exactly the row set available to $\MetricSpaceUp$ and $\SOneUp$
consumers, so a consumer can only project the displayed rows already present in
the packet.

Next use
\autoref{thm:realalgorder-positive-apartness-nonzero-metric-boundary}. The
apartness-to-positive-gap handoff is a boundary row over the same transported
rational witnesses. It adds a metric-facing separation read but no independent
completed-real equality, quotient representative, or completeness coordinate.

Finally compare this projected row set with the constructive real algebra
boundary recorded by the roadmap constraints for
\autoref{sec:roadmap-l10-construction-order}. Since no displayed ledger
coordinate has the type of a trichotomy row, completed-real equality,
quotient representative, or completeness datum, those data cannot be obtained
by reading the apartness or metric rows.
\end{proof}

\begin{theorem}[RealAlgOrder public positive-distance handoff]
\label{thm:realalgorder-metricspace-public-positive-distance-handoff}
For constant rational real names, the positive-gap or apartness row exported by
\autoref{thm:realalgorder-public-apartness-metric-export} repacks as the
positive component of the $\MetricSpaceUp$ real-distance public surface of
\autoref{def:metricspace-real-distance-carrier}. The repacking reads only the
$\RatUp$ endpoint witnesses, their $\RealUp$ transported names, the displayed
$\hsame$ rows, $\Cont$ routes, $\Pkg$ provenance, local $\NameCert$ row, and the
$\RealAlgOrderUp$ budget packet. It does not add a completed-real equality row,
a trichotomy row, or a host metric structure.
\end{theorem}
\begin{proof}
Start from the exported apartness and positive-gap row of
\autoref{thm:realalgorder-public-apartness-metric-export}. The row already
records the two rational endpoints, their real-name transport, the visible
$\hsame$ coordinates, the continuation routes, provenance, and local
certificate.

Use \autoref{thm:realalgorder-positive-apartness-nonzero-metric-boundary} to
read this row as a metric-facing positive boundary. The theorem supplies the
positive component over the same transported endpoints and the same budget
packet, so no new equality or trichotomy coordinate is introduced.

Finally compare the resulting row with
\autoref{def:metricspace-real-distance-carrier} and
\autoref{thm:metricspace-public-law-package}. The $\MetricSpaceUp$ public
surface consumes the positive-distance component as a displayed ledger row, and
\autoref{thm:realalgorder-apartness-metric-consumer-separation} keeps the
consumer from reading any stronger completed-real or host-metric structure.
\end{proof}

\begin{theorem}[RealAlgOrder apartness triangle consumer]
\label{thm:realalgorder-metricspace-apartness-triangle-consumer}
Suppose two adjacent $\MetricSpaceUp$ positive-distance reads are both obtained
from $\RealAlgOrderUp$ apartness packets of
\autoref{thm:realalgorder-public-apartness-metric-export}. Then the outer
triangle-boundary consumer factors through the same visible endpoint rows,
apartness rows, $\hsame$ transports, $\Cont$ routes, $\Pkg$ provenance, local
$\NameCert$ rows, and metric triangle ledger rows. The factorization supplies
only the public triangle consumer boundary of $\MetricSpaceUp$ and does not
upgrade RealAlgOrder apartness into completed-real equality, trichotomy, or an
ambient host metric.
\end{theorem}
\begin{proof}
Apply the public export theorem to each adjacent apartness packet. Each packet
exposes a positive-distance read over its displayed rational endpoints, real
transport, apartness row, $\hsame$ coordinate, continuation route, provenance,
and local certificate.

Next apply
\autoref{thm:realalgorder-positive-apartness-nonzero-metric-boundary} to both
reads. The shared endpoint remains visible because both positive components are
repacked from the same RealAlgOrder endpoint ledger rather than from a hidden
host metric.

Use \autoref{def:metricspace-real-distance-carrier} and
\autoref{thm:metricspace-public-law-package} to feed the two displayed positive
components into the metric triangle ledger. The triangle consumer reads the
outer boundary through the same public metric rows.

Finally invoke
\autoref{thm:realalgorder-apartness-metric-consumer-separation}. The consumer
surface is limited to visible endpoints, apartness, transport, continuation,
provenance, local naming, and triangle-ledger rows, so the outer boundary does
not create completed-real equality, trichotomy, or host metric structure.
\end{proof}

\begin{theorem}[RealAlgOrder apartness metric consumer separation]
\label{thm:realalgorder-apartness-metric-consumer-separation}
For constant rational real names, any $\MetricSpaceUp$ or $\SOneUp$ consumer of
the public apartness and metric export of
\autoref{thm:realalgorder-public-apartness-metric-export} can read only the
rationally transported carrier, classifier, algebra, order, apartness,
positive-gap handoff, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows displayed
by the $\RealAlgOrderUp$ packet.

The same consumer surface does not supply a trichotomy row, completed-real
equality, quotient representative, or completeness datum. Apartness and the
positive-gap metric boundary are therefore consumer-visible separation rows,
not an ambient completed-real order or quotient-equality principle.
\end{theorem}
\begin{proof}
Start with
\autoref{thm:realalgorder-constant-rational-scoped-certificate}. The scoped
certificate fixes the constant-rational carrier by a $\RatUp$ witness embedded
into $\RealUp$, and it lists the classifier, algebra, order, apartness,
transport, continuation, provenance, and local name rows as the available
coordinates.

Apply \autoref{thm:realalgorder-public-apartness-metric-export}. Its public
projection is exactly the row set available to $\MetricSpaceUp$ and $\SOneUp$
consumers, so a consumer can only project the displayed rows already present in
the packet.

Next use
\autoref{thm:realalgorder-positive-apartness-nonzero-metric-boundary}. The
apartness-to-positive-gap handoff is a boundary row over the same transported
rational witnesses. It adds a metric-facing separation read but no independent
completed-real equality, quotient representative, or completeness coordinate.

Finally compare this projected row set with the constructive real algebra
boundary recorded by the roadmap constraints for
\autoref{sec:roadmap-l10-construction-order}. Since no displayed ledger
coordinate has the type of a trichotomy row, completed-real equality,
quotient representative, or completeness datum, those data cannot be obtained
by reading the apartness or metric rows.
\end{proof}

\begin{theorem}[RealUp finite-window exhaustion obligation]
\label{thm:realup-finite-window-exhaustion-obligation}
At the RealUp obligation boundary, the finite-window observation field is
supplied by the conjunction of
$\mathsf{RealStreamClassifier\_finite\_prefix\_endpoint\_denominator\_package}$,
$\mathsf{RealStreamPrefixClassifier\_truncated\_endpoint\_denominator\_package}$,
$\mathsf{RealStreamPrefixClassifier\_truncated\_endpoint\_common\_e1\_tail\_readback}$,
and the $\mathsf{RegSeqRat}$ finite-window dyadic source exhaustion package.
Consequently every selected finite $\RealUp$ observation window exposes
endpoint $\RatUp$ classifiers, positive unary denominators, nonempty unary
endpoint rows, common e-one tail readback, and exactly the
$\mathsf{DyadicRatCore}$ source rows consumed by the $\RealUp$ seal, with no
data outside the carriers, classifiers, stability rows, and ledger rows listed
in \autoref{thm:realup-scoped-certificate-boundary}.
\end{theorem}
\begin{proof}
First collect the endpoint denominator face. Apply
\autoref{thm:real-stream-classifier-finite-prefix-endpoint-denominator-package}
to the selected finite prefix. It supplies the endpoint $\RatUp$ classifier,
the positive unary denominator witnesses, the unary endpoint rows, and the
nonempty endpoint exclusions needed at the selected observation.

Next pass to truncated prefixes. Apply the truncated endpoint denominator
package recorded by
\autoref{thm:real-stream-prefix-finite-tail-transported-endpoint-denominator-package}.
This transports the same endpoint denominator data through the finite tail
presentation used by the stream-prefix carrier and classifier rows.

Now read the selected one-headed tails. Apply
\autoref{thm:real-stream-prefix-truncated-endpoint-common-e1-tail-readback};
the truncated endpoint package gives the hypotheses needed to expose common
e-one tails at the two selected endpoint histories.

Finally identify the consumed source rows. Apply
\autoref{thm:regseqrat-finite-window-dyadic-source-exhaustion}. It states that
the selected bounded RegSeqRat observation reads exactly the two
$\mathsf{DyadicRatCoreObservationRow}$ endpoint sources and the scoped
carrier, classifier, stability, and finite ledger faces named by the dyadic
source package. These are precisely the rows allowed by
\autoref{thm:realup-scoped-certificate-boundary}; hence the finite-window observation
field consumes no data outside that boundary.
\end{proof}

\begin{theorem}[RealUp StreamName window determinacy boundary]
\label{thm:realup-streamname-window-determinacy-boundary}
Let two finite-window observations be admitted by the $\RealUp$ seal through
the same selected unary observation and the same
$\mathsf{RatStreamNameFiniteWindowClassifier}$ endpoint package. Then the two
observations expose the same endpoint $\RatUp$ classifier rows, the same
positive denominator witnesses, the same common e-one tail readbacks, and the
same $\mathsf{DyadicRatCore}$ source rows, up to the displayed $\hsame$ ledger
rows permitted by \autoref{thm:realup-scoped-certificate-boundary}. No host
$\mathsf{Nat}$ index, host function, quotient, or unrecorded stream object is
available to separate the two exposed windows.
\end{theorem}
\begin{proof}
Begin with the two accepted finite-window observations. Apply
\autoref{thm:realup-streamname-finite-window-ledger-boundary} to each one. The
only exposed rows are the selected unary observation, the pointwise
$\mathsf{RatStreamNameFiniteWindowClassifier}$ row, the endpoint denominator
rows, and the dyadic source rows named by the finite-window obligation.

Now use the common StreamName package. The structural laws of
\autoref{thm:rat-stream-name-finite-window-structural-laws} and the selected
full-endpoint package of
\autoref{thm:rat-stream-name-finite-window-selected-full-endpoint-package}
identify the two selected observations and their endpoint classifier rows by
the displayed $\hsame$ ledger rows. Thus the endpoint $\RatUp$ classifiers and
positive denominator witnesses are the same readable rows of the package.

Finally repack through
\autoref{thm:realup-finite-window-exhaustion-obligation}. The common endpoint
package supplies the hypotheses for the common e-one tail readbacks, and the
dyadic source exhaustion supplies the same $\mathsf{DyadicRatCore}$ source
rows. Since \autoref{thm:realup-scoped-certificate-boundary} permits only these
carrier, classifier, stability, and finite ledger rows, the two windows have
the same exposed RealUp seal ledger.
\end{proof}

\begin{theorem}[RealUp StreamName finite-window ledger boundary]
\label{thm:realup-streamname-finite-window-ledger-boundary}
Every finite-window observation admitted by the $\RealUp$ seal factors through
the $\StreamNameUp$ finite-window classifier, the $\RatUp$ endpoint denominator
package, and the $\RegSeqRat$ dyadic source exhaustion rows. Its ledger is
exactly the selected unary observation, the pointwise
$\mathsf{RatStreamNameFiniteWindowClassifier}$ row, the endpoint denominator
rows, and the dyadic source rows named by
\autoref{thm:realup-finite-window-exhaustion-obligation}. No host
$\mathsf{Nat}$ index, host function, quotient, or unrecorded infinite stream
object is exposed at this boundary.
\end{theorem}
\begin{proof}
Start with an observation accepted by the $\RealUp$ seal. Apply
\autoref{thm:realup-finite-window-exhaustion-obligation}; it reduces the
accepted finite window to the denominator package, truncated endpoint
readback, and $\RegSeqRat$ dyadic source rows allowed by the scoped seal.

The finite-window stream-name component is supplied by
\autoref{thm:rat-stream-name-finite-window-from-classifier},
\autoref{thm:rat-stream-name-finite-window-structural-laws}, and
\autoref{thm:rat-stream-name-finite-window-selected-full-endpoint-package}.
Together these rows identify the selected unary observation, the bundled
pointwise classifier, and the endpoint denominator package read by the window.

Finally use \autoref{thm:realup-scoped-certificate-boundary}. That boundary
permits only the listed carrier, classifier, stability, and finite ledger rows.
The StreamName rows and the dyadic source rows therefore repack into the exact
ledger stated above, while host $\mathsf{Nat}$ data, host functions, quotients,
and unrecorded infinite objects have no exported row to inspect.
\end{proof}

\begin{theorem}[RealUp finite-window exhaustion subwindow closure]
\label{thm:realup-finite-window-exhaustion-subwindow-closure}
Whenever a $\RealUp$ finite-window exhaustion package is exposed for a
$\StreamNameUp$ finite window, every certified subwindow obtained by the
$\StreamNameUp$ finite-window subwindow restriction exposes the same $\RatUp$
endpoint classifier, the same ledger boundary, and the same $\RealUp$ read
surface restricted to that subwindow. The restricted package reads only the
subwindow observation rows, the inherited
$\mathsf{RatStreamNameFiniteWindowClassifier}$ rows, the endpoint denominator
rows, and the $\mathsf{DyadicRatCore}$ source rows already named by the ambient
window.
\end{theorem}
\begin{proof}
Start with the ambient finite-window exhaustion package. Apply
\autoref{thm:realup-finite-window-exhaustion-obligation} to identify its
$\RatUp$ endpoint classifiers, positive denominator witnesses, common e-one
tail readbacks, and $\mathsf{DyadicRatCore}$ source rows.

Restrict the StreamName component by
\autoref{thm:rat-stream-name-finite-window-subwindow-restriction}. The certified
subwindow keeps the same pointwise
$\mathsf{RatStreamNameFiniteWindowClassifier}$ rows and only changes the
displayed finite observation domain.

Next use \autoref{thm:realup-streamname-window-determinacy-boundary}. It rules
out a second endpoint reading for the restricted window once the selected
StreamName package and endpoint denominator rows are fixed.

Finally apply
\autoref{thm:realup-streamname-finite-window-ledger-boundary}. The ledger
boundary remains exactly the selected unary observation, the restricted
StreamName classifier row, the endpoint denominator rows, and the dyadic source
rows. Repacking these rows gives the same $\RealUp$ read surface restricted to
the certified subwindow.
\end{proof}

\begin{theorem}[RealUp finite-window exhaustion overlap closure]
\label{thm:realup-finite-window-exhaustion-overlap-closure}
Let two $\RealUp$ finite-window exhaustion packages expose certified
$\StreamNameUp$ subwindows. If the two restricted windows have a certified
common $\StreamNameUp$ subwindow overlap, then the overlap exposes the same
inherited $\mathsf{RatStreamNameFiniteWindowClassifier}$ rows, the same
endpoint denominator rows, the same common e-one tail readbacks, and the same
$\mathsf{DyadicRatCore}$ source rows, up to the $\hsame$ ledger permitted by
\autoref{thm:realup-scoped-certificate-boundary}. The overlap certificate
does not expose host $\mathsf{Nat}$ indices, host functions, quotients, or
unrecorded infinite stream data.
\end{theorem}
\begin{proof}
Apply
\autoref{thm:realup-finite-window-exhaustion-subwindow-closure} to each
ambient finite-window exhaustion package. Each restricted window therefore
reads only its subwindow observation rows, inherited
$\mathsf{RatStreamNameFiniteWindowClassifier}$ rows, endpoint denominator
rows, common e-one tail readbacks, and $\mathsf{DyadicRatCore}$ source rows.

Now restrict both sides to the certified common $\StreamNameUp$ overlap. The
overlap has the same selected StreamName window on both readings, so
\autoref{thm:realup-streamname-window-determinacy-boundary} identifies the
endpoint $\RatUp$ classifier rows, positive denominator witnesses, common
e-one tail readbacks, and dyadic source rows through the displayed $\hsame$
ledger.

Finally apply
\autoref{thm:realup-streamname-finite-window-ledger-boundary} on the overlap
itself. The only exported ledger rows are the selected unary observation, the
overlap $\mathsf{RatStreamNameFiniteWindowClassifier}$ rows, the endpoint
denominator rows, and the dyadic source rows named by the finite-window
exhaustion boundary.

Since \autoref{thm:realup-scoped-certificate-boundary} admits no exported row
for host $\mathsf{Nat}$ indices, host functions, quotient witnesses, or
unrecorded infinite stream data, no further datum can distinguish the two
restricted readings on their common overlap.
\end{proof}

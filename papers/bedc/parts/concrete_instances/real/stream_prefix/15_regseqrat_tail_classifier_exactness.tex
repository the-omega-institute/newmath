\begin{theorem}[RegSeqRat tail-classifier exactness]
\label{thm:regseqrat-tail-classifier-exactness}
Let $W_0$ and $W_1$ be selected finite $\RealUp$ observation-window packages.
Assume that they have matching $\StreamNameUp$ finite-window classifier rows,
matching $\RegSeqRat$ spine rows, matching DyadicRatCore observation rows, and
the same finite $\Cont/\hsame$ ledger over the visible endpoint and transport
entries. Then the selected $\RegSeqRat$ tail classifier extracted from $W_0$
is exactly the same selected $\RegSeqRat$ tail classifier extracted from
$W_1$ as a finite-window readback row, after the displayed $\hsame$ transports
in the common ledger are applied.

The exact classifier surface is only the finite $\StreamNameUp$ row, the
$\RegSeqRat$ spine, the DyadicRatCore observation rows, the endpoint transport
ledger, and the shared $\mathsf{RealLedgerPolicy}$ provenance. It exposes no
quotient row, completion row, Classical representative, metric-completeness
certificate, arithmetic or order operation, standard-bridge row, or external
regular-sequence selector.
\end{theorem}
\begin{proof}
Project both selected windows through the finite-window readback package of
\autoref{thm:realup-finite-window-streamname-readback-package}. This places
the $\StreamNameUp$ classifier row, the $\RegSeqRat$ spine rows, the
DyadicRatCore observation rows, the endpoint classifiers, the finite
$\Cont/\hsame$ ledger, and the $\mathsf{RealLedgerPolicy}$ row in one visible
finite package.

Apply
\autoref{thm:realup-streamname-regseqrat-tail-selector-determinacy}. The
matching stream-name, regular-sequence, dyadic, endpoint, and ledger rows force
the two selected tail readbacks to agree up to the displayed $\hsame$
transports, and the same argument identifies the exposed
$\mathsf{RealLedgerPolicy}$ row.

It remains to record that this agreement is exact on the selected
$\RegSeqRat$ classifier surface. Use
\autoref{thm:realup-scoped-finite-window-read-surface} together with the
finite-window readback package to list every row visible to a consumer. The
list is precisely the finite stream-name row, the regular-sequence spine, the
dyadic observation rows, the endpoint transport ledger, and the shared
provenance; the same scoped surface excludes quotient, completion, Classical,
metric-completeness, arithmetic, order, standard-bridge, and external selector
data. Hence the selected tail classifier is determined exactly by the common
finite rows.
\end{proof}

\begin{theorem}[RegSeqRat tail-classifier reindexing normal form]
\label{thm:regseqrat-tail-classifier-reindexing-normal-form}
Let $W$ be a finite $\RealUp$ observation-window package, and let
$\rho_1,\ldots,\rho_n$ be a finite certified unary reindexing schedule whose
adjacent stages preserve the displayed $\StreamNameUp$, $\RegSeqRat$,
DyadicRatCore, $\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$,
$\mathsf{InBundle}$, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RealLedgerPolicy}$ rows. At every stage, read the selected
$\RegSeqRat$ tail classifier from the displayed finite-window surface.

Then the selected tail row read after any adjacent certified stage, the row
read after the composite schedule $\rho_n\circ\cdots\circ\rho_1$, and the row
read directly from the normal-form finite-window surface of $W$ are the same
finite-window $\RegSeqRat$ tail-classifier row. This normal-form comparison
uses only the displayed finite readback rows and introduces no quotient row,
completion row, Classical representative, metric-completeness certificate,
arithmetic or order row, standard-bridge row, or hidden regular-sequence
selector.
\end{theorem}
\begin{proof}
Induct on the finite certified schedule. In the empty schedule case, the
normal-form read is the direct finite-window read of $W$; apply
\autoref{thm:regseqrat-tail-classifier-exactness} to obtain the selected
$\RegSeqRat$ tail row from the displayed StreamName, RegSeqRat, DyadicRatCore,
endpoint-transport, ledger, package, and provenance rows.

For the cons step, assume the statement for the initial schedule and append one
certified unary reindexer. The induction hypothesis identifies the selected
tail row after the initial schedule with the direct normal-form read. Apply
\autoref{thm:regseqrat-tail-classifier-reindexing-stability} to the appended
stage, using the preserved $\StreamNameUp$, $\RegSeqRat$, DyadicRatCore,
$\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$, $\mathsf{InBundle}$,
$\hsame$, $\Cont$, $\Pkg$, and $\mathsf{RealLedgerPolicy}$ rows.

The composite schedule has the same public readback as the stagewise schedule
by \autoref{thm:regseqrat-tail-classifier-reindexing-composition}. Therefore
the row read after the composite schedule agrees with each stagewise selected
tail row and with the normal-form finite-window row.

Finally apply
\autoref{thm:regseqrat-tail-classifier-pullback-uniqueness}. Any candidate
tail row compatible with all stagewise pullbacks factors through the same
displayed finite-window surface, so there is only one selected tail classifier.
\autoref{thm:realup-scoped-finite-window-read-surface} supplies the exclusion
boundary: quotient, completion, Classical, metric-completeness, arithmetic,
order, standard-bridge, and hidden selector rows are outside the scoped finite
surface.
\end{proof}

\begin{theorem}[RegSeqRat tail-classifier normal-form tail coverage]
\label{thm:regseqrat-tail-classifier-normal-form-tail-coverage}
Let $W$ be a finite $\RealUp$ observation-window package, and let
$\rho_1,\ldots,\rho_n$ be a finite certified unary reindexing schedule whose
adjacent stages preserve the displayed $\StreamNameUp$, $\RegSeqRat$,
DyadicRatCore, $\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$,
$\mathsf{InBundle}$, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RealLedgerPolicy}$ rows. For each stagewise read, for the read after
the composite schedule $\rho_n\circ\cdots\circ\rho_1$, and for the direct
normal-form finite-window read of $W$, the selected tail row is covered by one
and the same selected $\RegSeqRat$ classifier row.

The covering row is the finite-window classifier row exposed by the displayed
$\StreamNameUp$, $\RegSeqRat$, DyadicRatCore, endpoint-transport,
continuation-ledger, package, and $\mathsf{RealLedgerPolicy}$ data. The
coverage statement has no quotient row, completion row, Classical
representative, metric-completeness certificate, arithmetic or order row,
standard-bridge row, or hidden selector row.
\end{theorem}
\begin{proof}
Apply
\autoref{thm:regseqrat-tail-classifier-reindexing-normal-form}. It identifies
the selected tail row read at each certified stage, the selected row read after
the composite schedule, and the selected row read directly from the normal-form
finite-window surface.

Now use \autoref{thm:regseqrat-tail-classifier-exactness} on that common
finite-window surface. The exactness theorem says that the selected tail is not
a separate row but the displayed $\RegSeqRat$ classifier row read from the
shared StreamName, RegSeqRat, DyadicRatCore, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RealLedgerPolicy}$ data.

It remains to rule out a second mediator for one of the reads. Apply
\autoref{thm:regseqrat-tail-classifier-pullback-uniqueness}. Any candidate row
compatible with the stagewise, composite, and direct pullbacks factors through
the same selected finite-window classifier row, so all selected tails are
covered by that single row.

The exclusion boundary is inherited from the exact finite read surface: quotient
data, completion data, Classical representatives, metric-completeness
certificates, arithmetic or order structure, standard-bridge data, and hidden
selector rows are not part of the scoped package.
\end{proof}

\begin{theorem}[RegSeqRat tail-classifier pullback uniqueness]
\label{thm:regseqrat-tail-classifier-pullback-uniqueness}
Let $W_0$ and $W_1$ be finite-window $\RealUp$ packages whose displayed
$\StreamNameUp$ classifier rows, $\RegSeqRat$ spine rows, DyadicRatCore
observation rows, endpoint $\hsame$ transports, and finite $\Cont$ ledger
entries agree as in
\autoref{thm:regseqrat-tail-classifier-exactness}. Suppose a candidate
tail-classifier row $T$ pulls back to the selected tail of $W_0$ and also
pulls back to the selected tail of $W_1$ through exactly those displayed
rows. Then $T$ is identified with the unique selected $\RegSeqRat$ tail
classifier determined by the common finite-window readback data. No second
tail-classifier surface can mediate the finite-window $\RealUp$ seal while
remaining invisible to the displayed $\StreamNameUp$, $\RegSeqRat$,
DyadicRatCore, $\hsame$, and $\Cont$ rows.
\end{theorem}
\begin{proof}
By \autoref{thm:regseqrat-tail-classifier-exactness}, the two selected tails
of $W_0$ and $W_1$ are the same $\RegSeqRat$ tail classifier after the
displayed endpoint transports in the shared ledger are applied.

It remains to exclude a hidden mediator. The scoped read surface of
\autoref{thm:realup-scoped-finite-window-read-surface} lists every row that a
finite-window $\RealUp$ consumer may read: the stream-name classifier row, the
regular-sequence spine, the dyadic observation rows, the endpoint transport
ledger, and the finite continuation ledger.

Finally, \autoref{thm:realup-finite-window-streamname-readback-package}
packages exactly that surface as the readback boundary. Since $T$ pulls back
through no rows other than those displayed rows, it has no independent slot in
the finite package. Therefore the two pullbacks of $T$ factor through the
same selected tail, and $T$ is the unique selected $\RegSeqRat$ tail
classifier available at the finite-window seal.
\end{proof}

\begin{theorem}[RegSeqRat tail-classifier union exhaustion]
\label{thm:regseqrat-tail-classifier-union-exhaustion}
Let $\{W_i\}_{i\in I}$ be a finite family of selected finite $\RealUp$
observation-window packages. Assume that every certified common subwindow used
by the finite cover displays the same $\StreamNameUp$ row, the same
$\RegSeqRat$ spine row, the same DyadicRatCore observation row, matching
endpoint $\hsame$ transports, and the same finite $\Cont$ ledger entries. Also
assume that these certified common subwindows cover the union of the visible
windows. Then the union window exposes one selected $\RegSeqRat$
tail-classifier row.

Moreover, any candidate $\RegSeqRat$ tail row whose pullback agrees with the
selected tail row on every member window factors uniquely through the selected
tail-classifier row on the union window. The union read surface adds no
quotient row, completion row, Classical representative, metric-completeness
certificate, arithmetic or order row, standard-bridge row, or hidden
regular-sequence selector.
\end{theorem}
\begin{proof}
First restrict the family to a certified common subwindow. By
\autoref{thm:regseqrat-tail-classifier-exactness}, the shared
$\StreamNameUp$, $\RegSeqRat$, DyadicRatCore, endpoint-transport, and finite
ledger rows determine the same selected tail classifier on that subwindow.

Next compare two overlaps in the finite cover. Apply
\autoref{thm:regseqrat-tail-classifier-pullback-uniqueness} to the two
restricted readback surfaces. Any mediator that pulls back to both selected
tails has no independent row outside the displayed finite readback inventory,
so the overlap contributes the same mediating tail row.

Now exhaust the finite union. Use
\autoref{thm:realup-finite-window-exhaustion-overlap-closure} to repack the
certified common subwindows into one finite $\RealUp$ union surface, and use
\autoref{thm:streamname-regseqrat-finite-window-cover-determinacy} to keep the
StreamName and RegSeqRat handoff rows fixed across the cover. The repacked
surface therefore carries a single selected tail classifier.

Finally let a candidate tail row pull back to every member window. The previous
overlap comparison identifies all of its member-window pullbacks with the same
selected row, and the union repacking has no further tail-classifier slot. Thus
the candidate factors uniquely through the selected $\RegSeqRat$ tail row on
the finite union window.
\end{proof}

overlaps, then they glue to a unique selected global tail row on the finite
union window. Each local selected row is the pullback of this global row.

The glued surface contains only the displayed finite readback rows. It imports
no quotient, completion, Classical representative, metric-completeness, order,
arithmetic, standard bridge, or hidden selector row.
\end{theorem}
\begin{proof}
First apply \autoref{thm:regseqrat-tail-classifier-exactness} on each member

$\StreamNameUp$, $\RegSeqRat$, DyadicRatCore, $\mathsf{UnaryHistory}$,
$\mathsf{ProbeBundle}$, $\mathsf{InBundle}$, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RealLedgerPolicy}$ rows.

Next use \autoref{thm:regseqrat-tail-classifier-union-exhaustion}. The finite

is read through that surface after restriction to its member window.

Compatibility on overlaps is the partition condition of
\autoref{thm:regseqrat-tail-classifier-partition-coherence}. It identifies the
member-window restrictions with the direct selected read of the finite union, so
the union surface carries one candidate global tail row.

Finally apply \autoref{thm:regseqrat-tail-classifier-pullback-uniqueness}. Any
other global row with the same member-window pullbacks has no additional

global tail row. The exclusion sentence follows from the exactness and union
exhaustion boundaries: no quotient, completion, Classical representative,
metric-completeness, order, arithmetic, standard bridge, or hidden selector row

\end{proof}

\begin{theorem}[RegSeqRat tail-classifier reindexing composition]
\label{thm:regseqrat-tail-classifier-reindexing-composition}

$\rho$ be a certified unary reindexer from the displayed window of $W_0$ to an
intermediate displayed $\StreamNameUp$ window, and let $\sigma$ be a certified
unary reindexer from that intermediate window to the displayed window of
$W_1$. Suppose the two packages have matching $\StreamNameUp$ finite-window
classifier rows after first reindexing by $\rho$ and then by $\sigma$, matching
$\RegSeqRat$ spine rows, matching DyadicRatCore observation rows, matching
$\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$, and $\mathsf{InBundle}$
membership rows, and matching finite $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RealLedgerPolicy}$ rows at both displayed reindexing stages.

Then the $\RegSeqRat$ tail classifier selected after the composite
reindexing schedule $\sigma \circ \rho$ is the same finite-window readback row
as the classifier obtained by applying the two certified reindexing steps
successively. The composition surface is only the displayed finite readback
surface; it introduces no quotient row, completion row, Classical
representative, metric-completeness certificate, arithmetic or order row,
standard-bridge row, or hidden regular-sequence selector.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:streamname-regseqrat-reindexed-window-exhaustion} to the
$\rho$-reindexed handoff surface. The displayed $\StreamNameUp$ classifier,
$\RegSeqRat$ spine, $\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$, and
$\mathsf{InBundle}$ membership rows form the intermediate finite readback
boundary.

Apply the same exhaustion theorem to the $\sigma$-reindexed handoff surface.
The intermediate boundary is thereby transported to the final displayed
finite-window boundary with the same RegSeqRat spine, DyadicRatCore
observation rows, endpoint $\hsame$ transports, finite $\Cont$ ledger, $\Pkg$
row, and $\mathsf{RealLedgerPolicy}$ provenance.

Next use
\autoref{thm:streamname-regseqrat-reindexed-window-consumer-determinacy}. A
consumer reading after $\sigma \circ \rho$ sees exactly the same displayed
StreamName and RegSeqRat source rows as a consumer that reads after $\rho$ and
then after $\sigma$. Thus both schedules feed the tail selector with the same
finite-window row inventory.

Now invoke
\autoref{thm:regseqrat-tail-classifier-reindexing-stability} for the
$\rho$-stage and again for the $\sigma$-stage. Each stage preserves the
selected $\RegSeqRat$ tail classifier on the displayed finite rows, so the
successive readback row agrees with the row exposed by the composite
reindexing schedule.

It remains to show that this agreement is the exact public surface. Apply
\autoref{thm:regseqrat-tail-classifier-exactness} to the common StreamName,
RegSeqRat, DyadicRatCore, endpoint-transport, ledger, package, and provenance
rows. Then
\autoref{thm:regseqrat-tail-classifier-pullback-uniqueness} excludes any
second tail-classifier row pulling back through the two-stage schedule.

Finally use \autoref{thm:realup-scoped-finite-window-read-surface}. The scoped
row inventory contains only the finite-window readback rows displayed above,
so quotient, completion, Classical, metric-completeness, arithmetic, order,
standard-bridge, and hidden selector data have no slot in the composite
surface.
\end{proof}

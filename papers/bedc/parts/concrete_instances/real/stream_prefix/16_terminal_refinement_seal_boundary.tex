\begin{theorem}[RealUp terminal refinement seal boundary]
\label{thm:realup-terminal-refinement-seal-boundary}
Let $E$ be a $\RealUp$ finite-window refinement envelope over a fixed
$\StreamNameUp$ source. Suppose two refined finite-window presentations
$W$ and $W'$ are carried by $E$, are connected by the directed-refinement seal
of \autoref{thm:realup-finite-window-directed-refinement-seal}, and are then
read by a terminal finite-window $\RealUp$ seal consumer. Then every terminal
consumer of either presentation reads exactly the same finite source
boundary:
\begin{enumerate}
  \item the $\mathsf{RatStreamNameFiniteWindowClassifier}$ rows;
  \item the bundled $\mathsf{UnaryHistory}$ and $\mathsf{InBundle}$ rows;
  \item the pointwise $\mathsf{RatHistoryClassifier}$ rows;
  \item the accepted $\RegSeqRat$ finite-window spine;
  \item the $\mathsf{DyadicRatCore}$ source rows;
  \item the visible $\Cont$, $\hsame$, $\Pkg$, and
  $\mathsf{RealLedgerPolicy}$ ledger rows.
\end{enumerate}
No terminal read exposes a quotient row, completion object, host
$\mathsf{Nat}$ index, host function, $\mathsf{Classical}$ datum,
metric-completeness certificate, arithmetic row, order row, or standard bridge
row.
\end{theorem}
\begin{proof}
First identify both refined presentations with the same envelope boundary.
Apply
\autoref{thm:realup-finite-window-refinement-envelope-coherence} to the two
windows carried by $E$. The result gives one public finite $\mathsf{BHist}$
surface containing exactly the envelope classifier, endpoint, dyadic-source,
$\hsame$, and ledger-policy rows.

Next use the directed-refinement seal. The comparison row supplied by
\autoref{thm:realup-finite-window-directed-refinement-seal} forces the
presentation read from $W$ and the presentation read from $W'$ through that
same finite refinement surface. Thus the two refined windows have no distinct
source boundary for a later consumer to inspect.

Now pass this common finite-window surface to the terminal seal consumer.
Apply \autoref{thm:streamname-realup-terminal-seal-boundary}. It reads the
finite StreamName-to-RegSeqRat packet as the selected classifier rows, bundled
unary and membership rows, pointwise rational classifier rows, endpoint and
denominator rows, the accepted RegSeqRat spine, the dyadic source rows, and
the finite $\Cont$ and $\hsame$ transport ledger.

Finally apply \autoref{thm:streamname-realup-terminal-seal-scope-readback}.
The scope readback keeps the source $\Pkg$ rows and
$\mathsf{RealLedgerPolicy}$ boundary attached to the same packet and excludes
all rows outside it. Hence no terminal consumer can read quotient data,
completion data, host indices, host functions, $\mathsf{Classical}$ data,
metric-completeness certificates, arithmetic or order rows, or standard
bridge rows.
\end{proof}

\begin{theorem}[RealUp terminal RegSeqRat source coverage]
\label{thm:realup-terminal-regseqrat-source-coverage}
Every terminal finite-window $\RealUp$ seal consumer factors through exactly
one $\RegSeqRat$ source surface. That source surface consists of the displayed
$\mathsf{RatStreamNameFiniteWindowClassifier}$ rows, the
$\mathsf{UnaryHistory}$ and $\mathsf{InBundle}$ rows, the pointwise
$\mathsf{RatHistoryClassifier}$ rows, the accepted $\RegSeqRat$ spine, the
DyadicRatCore observation rows, and the visible $\Cont$, $\hsame$, $\Pkg$, and
$\mathsf{RealLedgerPolicy}$ ledger rows.

Conversely, any packet carrying exactly those rows repacks into the terminal
source surface read by the seal consumer. The coverage contains no quotient
row, completion object, host index, host function, $\mathsf{Classical}$ datum,
metric-completeness row, arithmetic or order row, or standard bridge row.
\end{theorem}
\begin{proof}
Start from the terminal read inventory of
\autoref{thm:realup-terminal-refinement-seal-boundary}. It lists precisely the
finite classifier, unary-history, membership, pointwise rational, accepted
spine, dyadic-source, transport, package, and ledger rows visible to a terminal
seal consumer.

For the source side, apply the finite-window StreamName-to-RegSeqRat handoff
from \autoref{thm:realup-streamname-regseqrat-window-correspondence}
and \autoref{thm:regseqrat-seal-finite-window-source-coverage}. These identify
the StreamName finite-window classifiers and dyadic observation rows as the
same source packet consumed by the $\RegSeqRat$ seal.

Now use the tail-classifier exactness package. By
\autoref{thm:regseqrat-tail-classifier-exactness} and
\autoref{thm:regseqrat-tail-classifier-normal-form-tail-coverage}, the
accepted $\RegSeqRat$ spine is the selected tail-classifier row determined by
the displayed StreamName, dyadic, endpoint-transport, package, and
$\mathsf{RealLedgerPolicy}$ data.

It remains to check uniqueness and repacking. Apply
\autoref{thm:regseqrat-tail-classifier-pullback-uniqueness}: any terminal
candidate compatible with the displayed rows factors through the same selected
$\RegSeqRat$ source surface. Conversely, carrying those rows supplies every
entry in the inventory above, so the packet repacks into the terminal seal
source surface and leaves all excluded quotient, completion, host, Classical,
metric, arithmetic, order, and bridge rows unread.
\end{proof}

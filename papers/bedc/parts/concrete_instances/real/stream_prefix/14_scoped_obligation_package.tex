\begin{theorem}[RealUp stream-prefix scoped obligation package]
\label{thm:realup-stream-prefix-scoped-obligation-package}
The four stream-prefix obligation faces in
\autoref{thm:real-stream-prefix-carrier-obligation},
\autoref{thm:real-stream-prefix-classifier-obligation},
\autoref{thm:real-stream-prefix-stability-obligation}, and
\autoref{thm:real-stream-prefix-ledger-seal-obligation} form the scoped
RealUp stream-prefix package. Its source is bound to $\RealUp$ finite
stream-prefix observations over the $\RegSeqRat$ spine, $\StreamNameUp$ unary
observation names, $\RatUp$ endpoint rows, histories in $\mathsf{BHist}$,
$\hsame$ transport rows, selected $\mathsf{Cont}$ continuation rows, unary
observations, finite-window classifiers, dyadic endpoint ledger rows, and the
displayed $\RealUp$ seal. The package consumes no quotient, completion object,
host function, external metric primitive, or standard bridge.
\end{theorem}
\begin{proof}
Read the four named obligation theorems field by field and compare them with
the ten-field roadmap template.

\begin{enumerate}
\item Traditional target: the target is $\RealUp$ finite stream-prefix
observations. The carrier, classifier, stability, and ledger-seal faces all
refer to selected rational-observation prefixes rather than to an ambient
completion object.

\item BEDC source: the source is the $\RegSeqRat$ spine over $\StreamNameUp$
unary observations and $\RatUp$ endpoint histories in $\mathsf{BHist}$.
\autoref{thm:real-stream-prefix-carrier-obligation} reads those rows from the
certified unary-observation spine.

\item Carrier choice: the carrier face is the visible endpoint carrier data.
Apply \autoref{thm:real-stream-prefix-carrier-obligation}; it identifies the
selected prefix endpoints with the $\mathsf{RatHistoryCarrier}$ rows supplied
by the $\RegSeqRat$ spine.

\item Classifier choice: the classifier face is the pair
$\mathsf{RealStreamClassifier}$ and
$\mathsf{RealStreamPrefixClassifier}$. Apply
\autoref{thm:real-stream-prefix-classifier-obligation}; it relates global
stream rows, finite-prefix classifiers, and selected $\RatUp$ endpoint
classifier rows.

\item Stability obligations: the stability face is transport along $\hsame$,
unary-context closure, and selected continuation stability. Apply
\autoref{thm:real-stream-prefix-stability-obligation}; it preserves the
finite-prefix classifier rows and repacks them as global stream classifier
rows when all finite prefixes are present.

\item Ledger: the ledger face is the finite observation ledger together with
dyadic endpoint ledger rows. Apply
\autoref{thm:real-stream-prefix-ledger-seal-obligation}; it reads finite
windows, dyadic comparisons, regularity certificates, and the $\RealUp$ seal
from the same scoped stream-name interface.

\item Core theorem list: the package core is exactly the four obligation
faces cited in the statement. No fifth obligation theorem is required, because
the displayed carrier, classifier, stability, ledger, and seal fields exhaust
the stream-prefix boundary.

\item Exactness predicate: exactness is finite-window stream-prefix readback.
It is supplied by the classifier face through finite-prefix exactness and by
the ledger-seal face through finite-window dyadic source exhaustion.

\item Standard bridge: no standard bridge is claimed. The theorem records a
paper-side scoped package over BEDC rows only, not a bridge to classical real
analysis or to an external metric construction.

\item Status: the result is the scoped paper package for the RealUp
stream-prefix site. It collects the four local obligation faces into one
scope without adding a Lean marker or consuming data outside the displayed
BEDC source.
\end{enumerate}
\end{proof}

\begin{theorem}[RealUp scoped window tail-transport package]
\label{thm:realup-scoped-window-tail-transport-package}
For rational-observation streams $x,y:\mathbb{N}\to\Hist$, indices $n,m$, and
transported endpoint histories $u,v:\Hist$, assume the Real stream-prefix
classifier through $n$,
$$
\mathsf{RealStreamPrefixClassifier}(x,y,n),
$$
a classified finite $\RatUp$ tail of length $m$,
$$
\forall k<m,\ \mathsf{RatHistoryClassifier}(x(n+k+1),y(n+k+1)),
$$
pointwise unary anchors,
$$
\forall i,\ \mathsf{UnaryHistory}(x(i)),
$$
and transported endpoint rows
$$
\hsame(x(n+m),u),\quad \hsame(y(n+m),v).
$$
Then the transported tail window remains inside the scoped $\RealUp$
stream-prefix package. It exposes the transported endpoint $\RatUp$ classifier,
positive unary denominator rows for $u$ and $v$, unary endpoint rows, nonempty
endpoint exclusions, common e-one tail readback whenever the transported
endpoints are displayed as one-headed, finite $\mathsf{RealLedgerPolicy}$ rows,
and the dyadic endpoint source rows consumed by the $\RealUp$ seal. It consumes
no quotient, completion object, metric-completeness certificate, host function,
Classical datum, arithmetic or order primitive outside the listed rows, or
standard-bridge datum.
\end{theorem}
\begin{proof}
First package the ambient scoped source. Apply
\autoref{thm:realup-stream-prefix-scoped-obligation-package}. This fixes the
carrier, classifier, stability, ledger, and seal faces of the $\RealUp$
stream-prefix site, and it records that those faces do not read quotient,
completion, metric, host-function, or standard-bridge data.

Next transport the finite tail to the displayed endpoint boundary. Apply
\autoref{thm:real-stream-prefix-tail-extension-with-unary-anchor} to extend the
prefix classifier from $n$ to $n+m$ using the classified finite $\RatUp$ tail and
the pointwise unary anchors. Then apply
\autoref{thm:real-stream-prefix-finite-tail-transported-endpoint-denominator-package}
to the extended boundary and the two endpoint $\hsame$ rows. The result supplies
the transported endpoint $\RatUp$ classifier, the positive unary denominators,
the unary endpoint rows, and the nonempty endpoint exclusions.

Now read the window face through the scoped boundary. Apply
\autoref{thm:realup-scoped-finite-window-read-surface}. Its finite-window
readback contains the endpoint classifier rows, positive unary denominator rows,
common e-one tail readback, the finite ledger, and the dyadic source rows used by
the $\RealUp$ seal.

The rows just obtained are all rows named by the scoped stream-prefix package,
and the finite-window read surface explicitly excludes quotient, completion,
metric-completeness, Classical, arithmetic, order, and standard-bridge data
outside that boundary. Repacking these two scoped statements with the
transported finite-tail endpoint package gives the claimed transported tail
window package.
\end{proof}

\begin{theorem}[RealUp partitioned window tail-transport package]
\label{thm:realup-partitioned-window-tail-transport-package}
For rational-observation streams $x,y:\mathbb{N}\to\Hist$, an anchor $n$, a
total finite bound $w$, and a certified finite partition spine $P$ over $(n,w)$
with segments $(n_i,r_i)$, let $u_i,v_i:\Hist$ be transported histories at the
segment endpoints. Assume the Real stream-prefix classifier through the anchor,
$$
\mathsf{RealStreamPrefixClassifier}(x,y,n),
$$
the segment classifier conjunction,
$$
\bigwedge_{(n_i,r_i)\in P}
\mathsf{RealStreamWindowClassifier}(x,y,n_i,r_i),
$$
pointwise unary anchors,
$$
\forall j,\ \mathsf{UnaryHistory}(x(j)),
$$
and endpoint transport rows for every segment,
$$
\forall (n_i,r_i)\in P,
\hsame(x(n_i+r_i),u_i)\quad\text{and}\quad \hsame(y(n_i+r_i),v_i).
$$
Then every segment endpoint exposes the transported endpoint $\RatUp$
classifier, positive unary denominator rows, unary endpoint rows, nonempty
endpoint exclusions, common e-one tail readback whenever the transported
endpoints are displayed as one-headed, finite $\mathsf{RealLedgerPolicy}$ rows,
and DyadicRatCore endpoint source rows listed in
\autoref{thm:realup-scoped-window-tail-transport-package}. Moreover, the total
window presentation and the segment presentation expose the same finite ledger,
DyadicRatCore source rows, $\RegSeqRat$ spine rows, $\StreamNameUp$ request
names, $\mathsf{UnaryHistory}$ witnesses, and $\hsame$ transport rows of
\autoref{thm:realup-window-partition-read-surface}. The combined package
consumes no quotient, completion object, metric-completeness certificate,
Classical datum, arithmetic or order closure, or standard bridge.
\end{theorem}
\begin{proof}
First fix the ambient scoped boundary. Apply
\autoref{thm:realup-stream-prefix-scoped-obligation-package}; it supplies the
carrier, classifier, stability, ledger, and seal rows of the $\RealUp$
stream-prefix site from the displayed $\RegSeqRat$, $\StreamNameUp$, $\RatUp$,
$\mathsf{BHist}$, $\mathsf{UnaryHistory}$, $\mathsf{Cont}$, dyadic, and $\hsame$
source rows.

Next pass between the total and segmented windows. Apply
\autoref{thm:real-stream-window-finite-partition-fold} to the certified spine
$P$. The assumed segment classifier conjunction is the segmented presentation of
the total window $\mathsf{RealStreamWindowClassifier}(x,y,n,w)$, and the same
fold also recovers the segment conjunction from the total window when a consumer
starts from the wide observation.

Now identify the read surface. Apply
\autoref{thm:realup-window-partition-read-surface}. It says that the total
window and the partition presentation expose exactly the same finite ledger,
DyadicRatCore source rows, stream-name requests, unary-history witnesses, and
$\hsame$ transport rows, with no quotient, completion, metric-completeness,
Classical, arithmetic, order, or standard-bridge datum.

For each segment $(n_i,r_i)$ in $P$, apply
\autoref{thm:realup-scoped-window-tail-transport-package} at the segment anchor
with its endpoint histories $u_i$ and $v_i$. The segment classifier row, the
pointwise unary anchors restricted to the segment, and the two endpoint
transport rows give exactly the transported endpoint classifier, denominator,
unary-endpoint, exclusion, common-tail, finite-ledger, and dyadic source rows
named in that theorem.

Finally repack the segmentwise transported endpoint rows with the shared
partition read surface. The repacked certificate is a single finite observation
package for partitioned $\RealUp$ windows, and every row it displays has already
been read from the scoped BEDC boundary named above.
\end{proof}

\begin{theorem}[RealUp finite-window observation determinacy]
\label{thm:realup-finite-window-observation-determinacy}
For rational-observation streams $x,y:\mathbb{N}\to\Hist$, an anchor $n$, and
a finite width $w$, suppose two BEDC finite-window packages are exposed from the
same classifier
$$
\mathsf{RealStreamWindowClassifier}(x,y,n,w).
$$
Assume that both packages read the same $\StreamNameUp$ request rows, $\RatUp$
endpoint rows, DyadicRatCore source rows, $\mathsf{UnaryHistory}$ witnesses,
$\mathsf{Cont}$ ledger entries, and $\hsame$ transports named by
\autoref{thm:realup-window-partition-read-surface}. Then their visible endpoint
classifiers, positive denominator rows, finite ledger rows, dyadic source rows,
and $\RealUp$ seal rows classify together by $\hsame$. In particular, no second
public finite-window observation package with a different visible endpoint
surface is available from the same request.
\end{theorem}
\begin{proof}
First bind the common source boundary. Apply
\autoref{thm:realup-stream-prefix-scoped-obligation-package}. This fixes the
allowed $\RegSeqRat$, $\StreamNameUp$, $\RatUp$, $\mathsf{BHist}$,
$\mathsf{UnaryHistory}$, $\mathsf{Cont}$, dyadic, and $\hsame$ rows, and it
excludes quotient, completion, host-function, external metric, and
standard-bridge inputs.

Next identify the window ledger seen by the two packages. Apply
\autoref{thm:realup-window-partition-read-surface}. The total finite window and
any certified partition of it expose the same finite ledger, DyadicRatCore
source rows, stream-name requests, unary-history witnesses, and endpoint
$\hsame$ transports. Since the two packages are read from the same classifier
and the same request rows, their public ledgers are the same BEDC read surface.

For each displayed segment of a partition, apply
\autoref{thm:realup-scoped-window-tail-transport-package}. It reads the endpoint
classifiers, positive unary denominator rows, unary endpoint rows, finite
ledger entries, and dyadic endpoint source rows from the segment classifier and
the endpoint transport rows. The hypotheses identify those rows for the two
packages, so the segment endpoints classify together by $\hsame$.

Finally repack the segmentwise $\hsame$ endpoint data through
\autoref{thm:realup-partitioned-window-tail-transport-package}. The repacked
finite-window package has only the rows already named by the scoped source and
partition read surface. The same exclusion clauses remove quotient,
completion, metric-completeness, host-function, Classical, arithmetic or order
primitive, and standard-bridge data, so there is no remaining row from which a
different public endpoint package could be exposed.
\end{proof}

\begin{theorem}[RealUp finite-window common-refinement determinacy]
\label{thm:realup-finite-window-common-refinement-determinacy}
For rational-observation streams $x,y:\mathbb{N}\to\Hist$, an anchor $n$, and
a finite width $w$, suppose two certified finite-window packages expose
read surfaces for the same classifier
$$
\mathsf{RealStreamWindowClassifier}(x,y,n,w).
$$
Assume that the two packages admit a common certified partition refinement of
that classifier, and that both refined packages read the same $\StreamNameUp$
request rows, $\RegSeqRat$ spine rows, DyadicRatCore endpoint rows,
$\mathsf{UnaryHistory}$ witnesses, $\Cont$ ledger entries, and $\hsame$
transports.

Then the two refined read surfaces expose classifier-equivalent endpoint rows,
positive denominator rows, dyadic source rows, and $\RealUp$ seal rows.
Equivalently, refining the finite window through a shared certified partition
does not create a second public endpoint surface for the same observation
request.
\end{theorem}
\begin{proof}
Bind the common scoped source package with
\autoref{thm:realup-stream-prefix-scoped-obligation-package}. This fixes the
visible $\RegSeqRat$, $\StreamNameUp$, $\RatUp$, $\mathsf{BHist}$,
$\mathsf{UnaryHistory}$, $\Cont$, dyadic, and $\hsame$ rows available to both
finite-window packages.

Use \autoref{thm:realup-window-partition-read-surface} on the common
certified refinement. The theorem identifies the total classifier with its
partition read surface, so both packages are read through the same finite
ledger, request rows, unary witnesses, endpoint transports, and dyadic source
rows.

On each refined segment, apply
\autoref{thm:realup-scoped-window-tail-transport-package}. The shared segment
classifier and endpoint transports expose the same endpoint classifier rows,
positive denominator rows, common-tail rows, finite-ledger rows, and dyadic
source rows for the two packages.

Finally repack the segmentwise $\hsame$ rows through
\autoref{thm:realup-partitioned-window-tail-transport-package}. The repacked
public finite-window surfaces are built from the same partition ledger and the
same transported endpoint data, hence their endpoint, denominator, dyadic
source, and $\RealUp$ seal rows classify together by $\hsame$.
\end{proof}

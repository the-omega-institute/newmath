\begin{theorem}[RealUp Phase Real and Completion boundary]
\label{thm:realup-phase-real-completion-boundary}
The Phase Real and Completion paper boundary is exactly the composite of the
following four public packages:
\begin{enumerate}
  \item the DyadicRatCore public observation package of
  \autoref{thm:dyadicratcore-public-observation-package};
  \item the $\StreamNameUp$ public countable-name export package of
  \autoref{thm:streamname-public-interface-export-package} together with the
  finite-window interface of
  \autoref{thm:streamname-public-finite-window-interface};
  \item the $\RegSeqRat$ stream-name scoped interface of
  \autoref{thm:regseqrat-streamname-scoped-interface};
  \item the $\RealUp$ public certificate export of
  \autoref{thm:realup-public-certificate-export} after factorization through
  the $\StreamNameUp$ public interface in
  \autoref{thm:realup-public-export-through-streamname-public-interface}.
\end{enumerate}
Equivalently, its roadmap ten-field row has traditional target regular real
completion by dyadic rational streams; BEDC source the DyadicRatCore,
$\StreamNameUp$, $\RegSeqRat$, and $\RealUp$ packages listed above; carrier the
DyadicRatCore observations, rational stream names, regular stream spine, and
$\mathsf{RealConstantHistoryCarrier}$ rows; classifier the public
$\StreamNameUp$ classifier, the transported $\RegSeqRat$ stream classifier, and
$\mathsf{RealConstantHistoryClassifier}$ with
$\mathsf{RealUnaryStreamClassifier}$; stability the $\hsame$, endpoint,
stream-name, finite-window, and unary-stream transport rows exposed by those
packages; ledger the finite dyadic observation rows and
$\mathsf{RealLedgerPolicy}$; core theorem list exactly the six displayed
source theorems; exactness finite-window and public-export exactness only up to
the named classifiers; standard bridge absent; status paper-level phase
boundary for the future namespace $\mathsf{BEDC.Derived.RealUp}$.

The boundary contains no $\RealUp$ completeness theorem, no full $\RealUp$
arithmetic or order closure, no metric-completeness certificate, no quotient
construction, no Classical datum, and no standard bridge.
\end{theorem}
\leanchecked{BEDC.Derived.RealUp.RealUp\_concrete\_to\_schema}
\begin{proof}
First collect the four public package faces. Apply
\autoref{thm:dyadicratcore-public-observation-package} for the finite dyadic
observation source. Apply
\autoref{thm:streamname-public-interface-export-package} and
\autoref{thm:streamname-public-finite-window-interface} for the public
countable-name and finite-window surfaces. Apply
\autoref{thm:regseqrat-streamname-scoped-interface} for the regular
rational-stream spine.

Next attach the $\RealUp$ export. Apply
\autoref{thm:realup-public-certificate-export} to read the scoped $\RealUp$
certificate boundary as a public downstream package. Apply
\autoref{thm:realup-public-export-through-streamname-public-interface} to
factor its countable-name and finite-window components through the public
$\StreamNameUp$ interfaces, leaving precisely the dyadic, regular-stream,
endpoint, unary-history, $\hsame$, carrier, classifier, and ledger rows named
in the statement.

Finally repack the collected rows using the roadmap ten-field template. The
traditional target, BEDC source, carrier, classifier, stability, ledger, core
theorem list, exactness, standard-bridge field, and status field are all read
from the cited public packages. Since each cited package explicitly withholds
quotient, completion, arithmetic, order, metric-completeness, Classical, and
standard-bridge data, the composite phase boundary withholds exactly those data
as well.
\end{proof}

\begin{theorem}[RealUp completion phase interface exhaustion]
\label{thm:realup-completion-phase-interface-exhaustion}
Every public consumer row for the Phase Real and Completion package of
\autoref{thm:realup-phase-real-completion-boundary} factors through exactly one
of the four displayed BEDC surfaces: DyadicRatCore public observation rows,
$\StreamNameUp$ pointwise finite-window rows, $\RegSeqRat$ regular-spine and
dyadic-tolerance rows, or the $\RealUp$ seal and readback rows. The exhaustion
is exclusive at the consumer boundary: dyadic observations are read only
through \autoref{thm:dyadicratcore-public-observation-package},
stream-name windows only through the public $\StreamNameUp$ rows of
\autoref{thm:streamname-public-interface-export-package} and
\autoref{thm:streamname-public-finite-window-interface}, regular-spine and
dyadic-tolerance data only through
\autoref{thm:regseqrat-streamname-scoped-interface}, and seal/readback data
only through the $\RealUp$ export and seal interfaces of
\autoref{thm:realup-public-certificate-export},
\autoref{thm:realup-public-export-through-streamname-public-interface},
\autoref{thm:realup-public-seal-source-determinacy}, and
\autoref{thm:regseqrat-finite-window-real-seal-interface}.

No public consumer row in this phase may read completion data, quotient
representatives, host stream equality, or a direct $\RealUp$ read that bypasses
the $\StreamNameUp$--$\RegSeqRat$ handoff.
\end{theorem}
\begin{proof}
Begin with the composite boundary of
\autoref{thm:realup-phase-real-completion-boundary}. Its four cases are the
only rows placed in the phase package, so a public consumer row must be
dyadic, stream-name finite-window, regular-spine with dyadic tolerance, or
$\RealUp$ seal/readback data.

In the dyadic case, apply
\autoref{thm:dyadicratcore-public-observation-package}. It accounts for the
public observation rows and exposes no completion, quotient, host equality, or
direct $\RealUp$ read.

In the stream-name case, apply
\autoref{thm:streamname-public-interface-export-package} together with
\autoref{thm:streamname-public-finite-window-interface}. These theorems bind
pointwise finite windows through the public $\StreamNameUp$ surface and do not
introduce regular-spine or seal rows.

In the regular-spine case, apply
\autoref{thm:regseqrat-streamname-scoped-interface} and then
\autoref{thm:regseqrat-finite-window-real-seal-interface} for the dyadic
tolerance rows that hand the finite window to the $\RealUp$ seal. This is the
only path from a stream-name read to a $\RealUp$ readback row.

In the seal/readback case, apply
\autoref{thm:realup-public-certificate-export},
\autoref{thm:realup-public-export-through-streamname-public-interface}, and
\autoref{thm:realup-public-seal-source-determinacy}. The export factors
through the public stream-name interface, and seal-source determinacy prevents
any second source from serving the same public read. The excluded completion,
quotient, host-equality, and direct-read data are absent from each cited
surface, hence absent from the exhausted consumer boundary.
\end{proof}

\begin{theorem}[RealUp completion phase selected-window determinacy]
\label{thm:realup-completion-phase-selected-window-determinacy}
Let two public consumer rows of the Phase Real and Completion package select
the same certified unary finite window from the same accepted boundary packet.
Then both selected reads factor through the same displayed route: the
DyadicRatCore public observation package, the $\StreamNameUp$ finite-window
interface, the $\RegSeqRat$ stream-name scoped interface, and the $\RealUp$
seal/readback rows.

The two selected reads are compared only by the displayed $\hsame$ and
classifier rows along this route. No quotient representative, host stream
equality, completion object, or direct $\RealUp$ bypass is available as a
second comparison path.
\end{theorem}
\begin{proof}
Apply
\autoref{thm:realup-completion-phase-interface-exhaustion} to the two public
consumer rows. Since both rows select a unary finite window, the exhaustion
places them in the stream-name branch before any seal/readback row can be
read.

Use \autoref{thm:streamname-public-finite-window-interface} to identify the
selected certified window as a public $\StreamNameUp$ finite-window read. Then
apply \autoref{thm:regseqrat-streamname-scoped-interface} and
\autoref{thm:regseqrat-finite-window-real-seal-interface} to repack that same
window as the regular-spine input consumed by the $\RealUp$ seal.

Finally apply \autoref{thm:realup-public-seal-source-determinacy}. It rules
out a second seal source for the same public read, so the two consumers can be
compared only through the displayed $\hsame$ and classifier rows. The excluded
quotient, host-equality, completion, and direct-bypass data are absent by the
exhaustion theorem.
\end{proof}

\begin{theorem}[RealUp scoped finite-window read surface]
\label{thm:realup-scoped-finite-window-read-surface}
The selected finite $\RealUp$ observation windows read exactly the
carrier/classifier, stability/transport, finite ledger, dyadic source,
$\RegSeqRat$ spine, $\StreamNameUp$, $\RatUp$, $\mathsf{UnaryHistory}$,
$\hsame$, and $\mathsf{RealLedgerPolicy}$ rows named by
\autoref{thm:realup-scoped-certificate-boundary}. They read no quotient datum,
completion datum, metric-completeness certificate, arithmetic or order
operation, Classical datum, or standard-bridge datum.
\end{theorem}
\begin{proof}
First collect the finite-window observation face. Apply
\autoref{thm:realup-finite-window-exhaustion-obligation}: every selected finite
$\RealUp$ window exposes the endpoint $\RatUp$ classifiers, unary denominator
rows, common e-one tail readback, and dyadic source rows consumed by the
$\RealUp$ seal.

Next identify the structural rows read by that observation field. Apply
\autoref{thm:real-stream-prefix-carrier-obligation} for the DyadicRatCore,
$\RegSeqRat$, $\StreamNameUp$, and $\RatUp$ carrier source. Apply
\autoref{thm:real-stream-prefix-classifier-obligation} for the constant-history
and unary stream classifier source. Apply
\autoref{thm:real-stream-prefix-stability-obligation} for pointwise $\hsame$
transport and selected-continuation stability.

Finally attach the ledger and certificate boundary. Apply
\autoref{thm:real-stream-prefix-ledger-seal-obligation} for the finite
$\mathsf{RealLedgerPolicy}$ rows and seal consumption. Then apply
\autoref{thm:realup-scoped-certificate-boundary}, which states that these rows
are exactly the scoped $\RealUp$ certificate boundary and excludes quotient,
completion, arithmetic, order, metric-completeness, Classical, and
standard-bridge data.
\end{proof}

\begin{theorem}[RealUp StreamName-RegSeqRat finite-window correspondence]
\label{thm:realup-streamname-regseqrat-window-correspondence}
A selected finite $\RealUp$ observation-window surface is equivalent, as BEDC
public data, to the pair consisting of the public $\StreamNameUp$ finite-window
interface of \autoref{thm:streamname-public-finite-window-interface} and the
$\RegSeqRat$ dyadic source exhaustion rows consumed in
\autoref{thm:realup-finite-window-exhaustion-obligation}. The pair records
exactly the finite-window stream classifier, endpoint $\RatUp$ classifiers,
unary denominator witnesses, common e-one tail readback, DyadicRatCore source
rows, $\mathsf{UnaryHistory}$ endpoints, $\hsame$ transport, and finite
$\mathsf{RealLedgerPolicy}$ ledger rows, and it records no quotient,
completion, arithmetic, order, metric-completeness, Classical, or
standard-bridge datum.
\end{theorem}
\begin{proof}
For the projection direction, first apply
\autoref{thm:realup-scoped-finite-window-read-surface}. It identifies the
selected finite $\RealUp$ read surface with the DyadicRatCore, $\RegSeqRat$,
$\StreamNameUp$, $\RatUp$, unary-history, $\hsame$, and finite ledger rows that
the scoped certificate exposes.

Then apply
\autoref{thm:realup-finite-window-export-through-streamname-interface}. The
$\StreamNameUp$ component factors through the public finite-window interface of
\autoref{thm:streamname-public-finite-window-interface}, and the remaining rows
are exactly the dyadic source, regular sequence, endpoint, unary, transport,
and ledger rows read by
\autoref{thm:realup-finite-window-exhaustion-obligation}.

For the reconstruction direction, start with the displayed pair. Repack the
$\StreamNameUp$ finite-window classifier rows through the public interface, and
repack the endpoint $\RatUp$ classifiers, unary denominator witnesses, common
e-one tail readback, and DyadicRatCore source rows through the exhaustion
obligation.

The two packets share the same finite $\mathsf{UnaryHistory}$ endpoints,
$\hsame$ transport, and $\mathsf{RealLedgerPolicy}$ readback. Combining those
shared rows gives precisely the scoped boundary named in
\autoref{thm:realup-scoped-finite-window-read-surface}, so the reconstructed
surface is the original selected finite $\RealUp$ observation-window surface
and contains no excluded datum.
\end{proof}

\end{theorem}
\begin{proof}
First identify the public stream-name finite-window surface. Apply
\autoref{thm:streamname-public-finite-window-interface}: the
$\StreamNameUp$ component available to downstream $\RealUp$ windows is the
$\mathsf{RatStreamNameFiniteWindowClassifier}$ surface together with
classifier-to-window coverage, singleton-window exhaustion, reindexing and
point-map coverage, structural laws, the selected endpoint package, and
finite-window stability fields.

Next identify the selected $\RealUp$ read surface. Apply
\autoref{thm:realup-scoped-finite-window-read-surface}; it says that selected
finite observation windows read the DyadicRatCore source, $\RegSeqRat$ spine,
$\StreamNameUp$ rows, $\RatUp$ endpoints, $\mathsf{UnaryHistory}$, $\hsame$
transport, and finite $\mathsf{RealLedgerPolicy}$ rows, while excluding quotient,
completion, arithmetic, order, metric-completeness, Classical, and
standard-bridge data.

$\mathsf{BHist}$; carrier, selected rational-observation windows; classifier,
finite-window pointwise $\RatUp$ classification; stability, $\StreamNameUp$
finite-window stability plus $\hsame$ transport; ledger, finite observation
index plus $\mathsf{RealLedgerPolicy}$; core theorem, this block; exactness,
factorization through the public finite-window interface; standard bridge, none;
status, proof target.

	Finally close directly at the finite-window surface. The $\StreamNameUp$
	component is exactly the public finite-window interface just identified, and

	therefore leaves precisely the DyadicRatCore observation rows, $\RegSeqRat$

	and finite $\mathsf{RealLedgerPolicy}$ rows, with the excluded quotient,
	completion, arithmetic, order, metric-completeness, Classical, and

	\end{proof}

\begin{theorem}[RealUp StreamName window stability consumption]
\label{thm:realup-streamname-window-stability-consumption}
Let $s,t,s',t'$ be rational observation maps carried by a finite
$\mathsf{ProbeBundle}$ window, and let the selected unary observation rows be
the endpoint rows read by the finite $\RealUp$ prefix surface. Suppose the
$\StreamNameUp$ finite-window classifier accepts the window for $(s,t)$,
suppose the packet carries bundled pointwise $\hsame$ transports from $(s,t)$
to $(s',t')$, and suppose the selected transported endpoint package of
\autoref{thm:rat-stream-name-finite-window-transported-selected-full-endpoint-package}
is present.

Then the transported selected $\RealUp$ prefix read surface has the same
finite $\mathsf{RealLedgerPolicy}$ boundary as in
\autoref{thm:realup-scoped-finite-window-read-surface}. It factors through
\autoref{thm:realup-finite-window-export-through-streamname-interface} as a
finite $\mathsf{BHist}$ read surface, with no quotient datum, completion datum,
metric-completeness certificate, Classical datum, or standard-bridge datum.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:rat-stream-name-finite-window-stability-fields} to the accepted
$\StreamNameUp$ classifier and the bundled pointwise $\hsame$ transports. This
produces the transported finite-window classifier for $(s',t')$ over the same
finite $\mathsf{ProbeBundle}$ boundary.

Next apply
\autoref{thm:rat-stream-name-finite-window-transported-selected-full-endpoint-package}
to the selected endpoint rows. The transported package supplies the positive
denominator rows, unary shape rows, zero-head exclusion rows, and one-tail
agreement rows needed by the selected endpoint surface.

Finally repack these transported rows through
\autoref{thm:realup-finite-window-export-through-streamname-interface}. The
scoped read surface of \autoref{thm:realup-scoped-finite-window-read-surface}
then supplies the unchanged finite $\mathsf{RealLedgerPolicy}$ boundary, so
the transported prefix surface is still a finite $\mathsf{BHist}$ interface
and none of the excluded quotient, completion, metric, Classical, or bridge
data is introduced.
\end{proof}

\begin{theorem}[RealUp StreamName source-exhaustion equivalence]
\label{thm:realup-streamname-source-exhaustion-equivalence}
For any selected finite $\RealUp$ observation window, the selected read surface
has exactly the $\StreamNameUp$ component supplied by
\autoref{thm:streamname-public-finite-window-interface} together with the
RegSeqRat and DyadicRatCore finite-window source rows consumed by
\autoref{thm:realup-finite-window-exhaustion-obligation}. Conversely, the
concrete RegSeqRat and DyadicRatCore rows, together with the public
$\StreamNameUp$ finite-window package, repack into the $\StreamNameUp$ component
of \autoref{thm:realup-scoped-finite-window-read-surface}.

This equivalence is internal to the selected finite $\mathsf{BHist}$ window. It
does not read a quotient datum, completion datum, metric-completeness
certificate, arithmetic or order operation, Classical datum, or standard-bridge
datum.
\end{theorem}
\begin{proof}
Project the selected $\RealUp$ read surface using
\autoref{thm:realup-scoped-finite-window-read-surface}. This gives the visible
$\StreamNameUp$ rows, the RegSeqRat spine rows, the DyadicRatCore source rows,
the $\RatUp$ endpoint rows, $\mathsf{UnaryHistory}$, $\hsame$ transport, and the
finite $\mathsf{RealLedgerPolicy}$ rows, with the excluded data absent.

Next apply \autoref{thm:realup-finite-window-export-through-streamname-interface}
to the projected stream-name component. It identifies that component with the
public $\mathsf{RatStreamNameFiniteWindowClassifier}$ package of
\autoref{thm:streamname-public-finite-window-interface}.

The remaining finite source rows are read from
\autoref{thm:realup-finite-window-exhaustion-obligation}: DyadicRatCore supplies
the selected dyadic observation rows, and RegSeqRat supplies the spine rows used
by the finite window. These rows are concrete $\mathsf{BHist}$ entries rather
than quotient or completion witnesses.

For the converse direction, start with the public $\StreamNameUp$ finite-window
package and the concrete RegSeqRat and DyadicRatCore rows just described. The
public package reconstructs the same selected stream-name component, while the
source rows provide exactly the surrounding finite-window inputs required by
\autoref{thm:realup-scoped-finite-window-read-surface}. Repacking these rows
therefore recovers the selected $\StreamNameUp$ component inside the same
$\RealUp$ read surface, and no excluded datum is introduced.
\end{proof}

\begin{theorem}[RealUp finite-window constant-stream seal factorization]
\label{thm:realup-finite-window-constant-stream-seal-factorization}

component is generated by the constant rational streams
$\mathsf{RatConstStream}_d$ and $\mathsf{RatConstStream}_e$. A displayed
$\mathsf{RatHistoryClassifier}(d,e)$ together with the constant stream-name

\autoref{thm:streamname-public-finite-window-interface}. After this

observation rows, RegSeqRat spine rows, $\RatUp$ endpoint rows,
$\mathsf{UnaryHistory}$ rows, $\hsame$ transport rows, and finite
$\mathsf{RealLedgerPolicy}$ rows recorded by the selected $\RealUp$ window.

Conversely, those DyadicRatCore, RegSeqRat, $\RatUp$, unary-history,
$\hsame$, and finite $\mathsf{RealLedgerPolicy}$ rows, together with the
constant stream-name package for
$\mathsf{RatHistoryClassifier}(d,e)$, repack into the selected finite
$\RealUp$ read surface. The factorization contains no quotient row, completion
row, metric-completeness certificate, Classical datum, or standard-bridge
datum.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:rat-stream-name-constant-bridge-package-exactness}. It converts
the displayed $\mathsf{RatHistoryClassifier}(d,e)$ into the constant
stream-name package for $\mathsf{RatConstStream}_d$ and
$\mathsf{RatConstStream}_e$ without replacing the histories by a quotient or
standard model.

Next restrict this package to the finite window $W$ by
\autoref{thm:rat-stream-name-constant-finite-window-exactness}. The resulting
finite constant-stream rows are exactly the rows accepted by the public
$\StreamNameUp$ finite-window interface of
\autoref{thm:streamname-public-finite-window-interface}.

Apply
\autoref{thm:realup-finite-window-export-through-streamname-interface} to the
selected $\RealUp$ observation window. Its stream-name component therefore
factors through the public interface, while
\autoref{thm:realup-streamname-source-exhaustion-equivalence} identifies the
unconsumed finite source rows as precisely the DyadicRatCore observations,
RegSeqRat spine, $\RatUp$ endpoints, unary-history rows, $\hsame$ transports,
and finite $\mathsf{RealLedgerPolicy}$ rows.

For the converse direction, start with this same finite list of remaining
rows and the constant stream-name package. The public interface reconstructs
the selected $\StreamNameUp$ component, and the source-exhaustion equivalence
supplies exactly the surrounding RegSeqRat, DyadicRatCore, $\RatUp$,
unary-history, transport, and ledger rows needed by the scoped $\RealUp$ read
surface. Repacking these displayed rows recovers the selected finite surface,
again without quotient, completion, metric-completeness, Classical, or
standard-bridge data.
\end{proof}

\begin{theorem}[RealUp finite-window StreamName readback package]
\label{thm:realup-finite-window-streamname-readback-package}
Let $W$ be a certified finite $\mathsf{ProbeBundle}$ window and let a public
$\RealUp$ finite-window classifier read be accepted over a $\StreamNameUp$
source. Then there is a displayed $\mathsf{BHist}$ packet whose rows are exactly
the unary schedule for $W$, the $\RatUp$ observation endpoints visible in the
window, the $\StreamNameUp$ finite-window classifier row, the $\RealUp$ scoped
window row, the $\Cont$ and $\hsame$ transport ledger connecting the stream
read to the real read, and the $\Pkg$ provenance of the selected source rows.
Every public $\RealUp$ finite-window classifier read through this
$\StreamNameUp$ source factors through that packet.

The packet contains no quotient row, completion row, metric-completeness
certificate, global Cauchy modulus, arithmetic or order operation, Classical
datum, or standard bridge. Its endpoint reads are exactly the displayed
$\RatUp$ observations and its classifier reads are exactly the displayed
$\StreamNameUp$ and scoped $\RealUp$ finite-window rows.
\end{theorem}
\begin{proof}
Begin with the accepted finite-window $\RealUp$ read and project its
$\StreamNameUp$ component. By
\autoref{thm:realup-finite-window-export-through-streamname-interface}, this
component factors through the public $\StreamNameUp$ finite-window interface,
so the stream-name classifier row is displayed as a finite $\mathsf{BHist}$
row rather than as an ambient stream equality.

Next use
\autoref{thm:realup-streamname-window-stability-consumption}. It transports
the projected stream-name row across the selected finite window and records
the required $\Cont$ and $\hsame$ ledger entries. These entries are the only
transport rows consumed by the scoped $\RealUp$ classifier read.

For constant rational endpoints, apply
\autoref{thm:realup-finite-window-constant-stream-seal-factorization}. It
identifies the endpoint part of the read with the displayed $\RatUp$
observation rows, the unary schedule, and the surrounding finite
$\mathsf{RealLedgerPolicy}$ rows. For non-constant accepted stream-name
sources, the same packet shape is obtained by reading the $\RatUp$ endpoints
already exposed by the finite StreamName window.

Finally apply
\autoref{thm:realup-finite-window-common-refinement-determinacy}. Any two
displayed packets supporting the same public $\RealUp$ classifier read admit
the same common finite refinement, hence determine the same visible rows up
to the recorded $\hsame$ transports. This gives the announced readback packet
and shows that every public finite-window classifier read factors through it.
\end{proof}

\begin{theorem}[RealUp StreamName-RegSeqRat tail-selector determinacy]
\label{thm:realup-streamname-regseqrat-tail-selector-determinacy}
Let $W_0$ and $W_1$ be selected finite $\RealUp$ observation-window packages.
Assume that they display the same public $\StreamNameUp$ finite-window
classifier row, the same $\RegSeqRat$ spine rows, the same DyadicRatCore
observation rows, matching $\RatUp$ endpoint classifiers, and the same finite
$\Cont/\hsame$ ledger. Then every selected tail-classifier readback extracted
from $W_0$ and every selected tail-classifier readback extracted from $W_1$
represent the same finite $\RealUp$ tail read up to the displayed $\hsame$
transport. Moreover the two packages expose the same
$\mathsf{RealLedgerPolicy}$ row.

The conclusion is scoped to the selected finite-window surface. It does not
assert equality of quotient completions, metric completions, arithmetic
operations, order operations, Classical representatives, or standard bridge
data.
\end{theorem}
\begin{proof}
Project both selected packages through
\autoref{thm:realup-finite-window-streamname-readback-package}. Each projection
has a $\StreamNameUp$ component, $\RegSeqRat$ spine rows, DyadicRatCore
observation rows, $\RatUp$ endpoint classifiers, a finite $\Cont/\hsame$ ledger,
and a surrounding $\mathsf{RealLedgerPolicy}$ row.

Identify the shared $\StreamNameUp$ component by
\autoref{thm:realup-streamname-source-exhaustion-equivalence}. Under the
hypotheses, the two finite-window classifier rows consume the same concrete
stream-name source and the same regular-sequence and dyadic observation rows.

The constant and empty boundary theorems remove non-displayed choices. Apply
\autoref{thm:realup-finite-window-constant-stream-seal-factorization} to the
constant endpoint case and
\autoref{thm:realup-empty-window-streamname-boundary} to the empty-window case.
In both cases the visible tail read is forced by the selected endpoint,
stream-name, regular-sequence, dyadic, and ledger rows already present in the
projected package.

It remains to compare the two selected tail reads. Compose the displayed
$\hsame$ transports in the common finite ledger. Since the two packages have the
same $\Cont/\hsame$ ledger and the same $\mathsf{RealLedgerPolicy}$ boundary,
the transported tail classifiers coincide up to $\hsame$, and the exposed
$\mathsf{RealLedgerPolicy}$ rows are identical as public finite-window rows.
\end{proof}

\begin{theorem}[RealUp finite-window StreamName readback uniformity]
\label{thm:realup-finite-window-streamname-readback-uniformity}
Let $W$ be a certified finite $\mathsf{ProbeBundle}$ window supporting a public
$\RealUp$ finite-window readback through a $\StreamNameUp$ source, and let
$U$ be either a certified finite subwindow of $W$ or a certified finite common
refinement of two such windows. Assume that the restriction to $U$ carries the
same public $\StreamNameUp$ finite-window row, the same $\RegSeqRat$ spine rows,
the same DyadicRatCore observation rows, and the restricted $\Cont/\hsame$
ledger displayed over $U$. Then the $\RealUp$ readback packet over $W$
restricts over $U$ to the same visible $\RatUp$ endpoint rows and to the same
finite $\mathsf{RealLedgerPolicy}$ row.

Equivalently, certified passage to a displayed finite subwindow or common
finite refinement preserves the public endpoint readback and the real-ledger
row determined by the StreamName-RegSeqRat-DyadicRatCore surface. The assertion
does not identify quotient completions, metric completions, global Cauchy
moduli, arithmetic operations, order operations, Classical representatives, or
standard bridge data.
\end{theorem}
\begin{proof}
Start with the selected $\RealUp$ read over $W$ and apply
\autoref{thm:realup-finite-window-streamname-readback-package}. This displays a
single finite $\mathsf{BHist}$ packet containing the public $\StreamNameUp$
finite-window row, the $\RegSeqRat$ spine, the DyadicRatCore observations, the
visible $\RatUp$ endpoints, the restricted $\Cont/\hsame$ ledger, the
$\Pkg$ provenance, and the finite $\mathsf{RealLedgerPolicy}$ row.

Restrict this displayed packet along the certified subwindow $U$, or along the
certified common refinement when two windows are present. The common-refinement
determinacy theorem
\autoref{thm:realup-finite-window-common-refinement-determinacy} says that any
two displayed packets with the same public rows over the refinement determine
the same finite surface up to the recorded $\hsame$ transports.

The source rows remaining after restriction are exactly the public stream-name
row, the regular-sequence spine, and the dyadic observation rows named in the
hypotheses. This is the source-exhaustion equivalence of
\autoref{thm:realup-streamname-source-exhaustion-equivalence}, while
\autoref{thm:streamname-public-finite-window-interface} identifies the
restricted stream-name row as the same public finite-window interface row
rather than an ambient stream equality.

With the stream-name source and the restricted transport ledger fixed, the
readback package has no remaining displayed choice for the $\RatUp$ endpoints
or for the finite $\mathsf{RealLedgerPolicy}$ row. Hence the packet over $W$
restricts to the same visible endpoint rows and the same real-ledger row over
$U$.
\end{proof}

\begin{theorem}[RealUp scoped finite-window read surface]
\label{thm:realup-scoped-finite-window-read-surface}
The selected finite $\RealUp$ observation windows read exactly the
carrier/classifier, stability/transport, finite ledger, dyadic source,
$\RegSeqRat$ spine, $\StreamNameUp$, $\RatUp$, $\mathsf{UnaryHistory}$,
$\hsame$, and $\mathsf{RealLedgerPolicy}$ rows named by
\autoref{thm:realup-scoped-certificate-boundary}. They read no quotient datum,
completion datum, metric-completeness certificate, arithmetic or order
operation, Classical datum, or standard-bridge datum.
\end{theorem}
\begin{proof}
First collect the finite-window observation face. Apply
\autoref{thm:realup-finite-window-exhaustion-obligation}: every selected finite
$\RealUp$ window exposes the endpoint $\RatUp$ classifiers, unary denominator
rows, common e-one tail readback, and dyadic source rows consumed by the
$\RealUp$ seal.

Next identify the structural rows read by that observation field. Apply
\autoref{thm:real-stream-prefix-carrier-obligation} for the DyadicRatCore,
$\RegSeqRat$, $\StreamNameUp$, and $\RatUp$ carrier source. Apply
\autoref{thm:real-stream-prefix-classifier-obligation} for the constant-history
and unary stream classifier source. Apply
\autoref{thm:real-stream-prefix-stability-obligation} for pointwise $\hsame$
transport and selected-continuation stability.

Finally attach the ledger and certificate boundary. Apply
\autoref{thm:real-stream-prefix-ledger-seal-obligation} for the finite
$\mathsf{RealLedgerPolicy}$ rows and seal consumption. Then apply
\autoref{thm:realup-scoped-certificate-boundary}, which states that these rows
are exactly the scoped $\RealUp$ certificate boundary and excludes quotient,
completion, arithmetic, order, metric-completeness, Classical, and
standard-bridge data.
\end{proof}

\begin{theorem}[RealUp finite-window export through StreamNameUp interface]
\label{thm:realup-finite-window-export-through-streamname-interface}
Every selected finite $\RealUp$ observation-window read surface of
\autoref{thm:realup-scoped-finite-window-read-surface} factors its
stream-name component through the public $\StreamNameUp$ finite-window interface
of \autoref{thm:streamname-public-finite-window-interface}. Its stream-name
component is exactly the $\mathsf{RatStreamNameFiniteWindowClassifier}$ surface
with classifier-to-window coverage, singleton-window exhaustion, reindexing and
point-map coverage, structural laws, the selected endpoint package, and
finite-window stability fields.

After this factorization, the remaining $\RealUp$ rows are only the
DyadicRatCore observation rows, $\RegSeqRat$ spine rows, $\RatUp$ endpoint rows,
$\mathsf{UnaryHistory}$, $\hsame$ transport, and
$\mathsf{RealLedgerPolicy}$. The exported finite-window surface contains no
quotient datum, completion datum, arithmetic or order operation,
metric-completeness certificate, Classical datum, or standard bridge.
\end{theorem}
\begin{proof}
First identify the public stream-name finite-window surface. Apply
\autoref{thm:streamname-public-finite-window-interface}: the
$\StreamNameUp$ component available to downstream $\RealUp$ windows is the
$\mathsf{RatStreamNameFiniteWindowClassifier}$ surface together with
classifier-to-window coverage, singleton-window exhaustion, reindexing and
point-map coverage, structural laws, the selected endpoint package, and
finite-window stability fields.

Next identify the selected $\RealUp$ read surface. Apply
\autoref{thm:realup-scoped-finite-window-read-surface}; it says that selected
finite observation windows read the DyadicRatCore source, $\RegSeqRat$ spine,
$\StreamNameUp$ rows, $\RatUp$ endpoints, $\mathsf{UnaryHistory}$, $\hsame$
transport, and finite $\mathsf{RealLedgerPolicy}$ rows, while excluding quotient,
completion, arithmetic, order, metric-completeness, Classical, and
standard-bridge data.

The ten-field fit is: traditional target, finite $\RealUp$ observation export;
BEDC source, DyadicRatCore plus $\StreamNameUp$, $\RegSeqRat$, and $\RatUp$ over
$\mathsf{BHist}$; carrier, selected rational-observation windows; classifier,
finite-window pointwise $\RatUp$ classification; stability, $\StreamNameUp$
finite-window stability plus $\hsame$ transport; ledger, finite observation
index plus $\mathsf{RealLedgerPolicy}$; core theorem, this block; exactness,
factorization through the public finite-window interface; standard bridge, none;
status, proof target.

	Finally close directly at the finite-window surface. The $\StreamNameUp$
	component is exactly the public finite-window interface just identified, and
	\autoref{thm:realup-scoped-finite-window-read-surface} supplies the whole local
	read surface in which that component sits. Removing the $\StreamNameUp$ component
	therefore leaves precisely the DyadicRatCore observation rows, $\RegSeqRat$
	spine rows, $\RatUp$ endpoint rows, $\mathsf{UnaryHistory}$, $\hsame$ transport,
	and finite $\mathsf{RealLedgerPolicy}$ rows, with the excluded quotient,
	completion, arithmetic, order, metric-completeness, Classical, and
	standard-bridge data still absent.
	\end{proof}

\begin{theorem}[RealUp StreamName source-exhaustion equivalence]
\label{thm:realup-streamname-source-exhaustion-equivalence}
For any selected finite $\RealUp$ observation window, the selected read surface
has exactly the $\StreamNameUp$ component supplied by
\autoref{thm:streamname-public-finite-window-interface} together with the
RegSeqRat and DyadicRatCore finite-window source rows consumed by
\autoref{thm:realup-finite-window-exhaustion-obligation}. Conversely, the
concrete RegSeqRat and DyadicRatCore rows, together with the public
$\StreamNameUp$ finite-window package, repack into the $\StreamNameUp$ component
of \autoref{thm:realup-scoped-finite-window-read-surface}.

This equivalence is internal to the selected finite $\mathsf{BHist}$ window. It
does not read a quotient datum, completion datum, metric-completeness
certificate, arithmetic or order operation, Classical datum, or standard-bridge
datum.
\end{theorem}
\begin{proof}
Project the selected $\RealUp$ read surface using
\autoref{thm:realup-scoped-finite-window-read-surface}. This gives the visible
$\StreamNameUp$ rows, the RegSeqRat spine rows, the DyadicRatCore source rows,
the $\RatUp$ endpoint rows, $\mathsf{UnaryHistory}$, $\hsame$ transport, and the
finite $\mathsf{RealLedgerPolicy}$ rows, with the excluded data absent.

Next apply \autoref{thm:realup-finite-window-export-through-streamname-interface}
to the projected stream-name component. It identifies that component with the
public $\mathsf{RatStreamNameFiniteWindowClassifier}$ package of
\autoref{thm:streamname-public-finite-window-interface}.

The remaining finite source rows are read from
\autoref{thm:realup-finite-window-exhaustion-obligation}: DyadicRatCore supplies
the selected dyadic observation rows, and RegSeqRat supplies the spine rows used
by the finite window. These rows are concrete $\mathsf{BHist}$ entries rather
than quotient or completion witnesses.

For the converse direction, start with the public $\StreamNameUp$ finite-window
package and the concrete RegSeqRat and DyadicRatCore rows just described. The
public package reconstructs the same selected stream-name component, while the
source rows provide exactly the surrounding finite-window inputs required by
\autoref{thm:realup-scoped-finite-window-read-surface}. Repacking these rows
therefore recovers the selected $\StreamNameUp$ component inside the same
$\RealUp$ read surface, and no excluded datum is introduced.
\end{proof}

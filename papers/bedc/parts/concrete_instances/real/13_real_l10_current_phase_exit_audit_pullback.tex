\begin{theorem}[Real L10 four-face status lookup]
\label{thm:real-l10-four-face-status-lookup}
Any accepted current Phase Real and Completion exit read through
$$
\DyadicRatCoreUp \to \StreamNameUp \to \RegSeqRatUp \to \RealUp
$$
reads four separate local closurestatus records. Each face contributes exactly
the closurestatus, formalstatus, leantarget, and bridgestatus rows declared in
its own local closurestatus block. The lookup cites those rows objectwise and
does not cache their token values in the Real L10 exit audit section.

These four entries are objectwise face records. They are not an aggregate
formalstatus token, an aggregate bridge row, or a route-level closurestatus
record for the composite exit.
\end{theorem}
\begin{proof}
The four faces are the Real L10 faces named by
\autoref{def:real-l10-dependency-lattice}. The local records are read from the
same concrete chapters that supply the face data:
\autoref{thm:dyadicratcore-standard-source-bridge} for
$\DyadicRatCoreUp$, \autoref{thm:streamname-public-interface-export-package},
\autoref{thm:streamname-standard-sequence-bridge}, and
\autoref{thm:streamname-scoped-interface-package} for $\StreamNameUp$,
\autoref{thm:regseqrat-real-seal-handoff} for $\RegSeqRatUp$, and
\autoref{thm:realup-phase-real-completion-boundary} for $\RealUp$.

The cited theorem anchors locate the four local chapters, but this theorem does
not copy their token values. The local closurestatus block for each face is the
sole source for its closurestatus, formalstatus, leantarget, and bridgestatus
rows.

By the two-axis convention of
\autoref{sec:roadmap-closure-and-verification-axes}, a closure entry and a
verification entry remain attached to the object whose closurestatus block
contains them. Reading the four faces therefore produces four objectwise face
records and does not create a fifth formalstatus token, route-level bridge row,
or composite closurestatus record.
\end{proof}

$\RealUp$ completeness claim, no quotient stream, no selected limit, and no
metric-completeness datum. Its only admissible audit read is the displayed

corresponding local closure record.
\end{theorem}
\begin{proof}

\autoref{thm:real-l10-source-route-nonescape} and
\autoref{thm:real-l10-dependency-lattice-interface-totality} to confine every
accepted current Phase Real and Completion interface read to the displayed
DyadicRatCore, StreamName, RegSeqRat, and local Real rows.

Next apply \autoref{thm:real-l10-four-face-exactness}. It identifies
acceptance with the finite packet over exactly those four faces, carried only
by the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and local $\NameCert$
coordinates. The StreamName exit correspondence,
\autoref{thm:real-l10-streamname-exit-route-correspondence}, names the same

It remains to read the audit status. Apply
\autoref{thm:real-l10-four-face-status-lookup}. It reads the DyadicRatCore,
StreamName, RegSeqRat, and RealUp closurestatus entries as four objectwise face
records and excludes any aggregate formalstatus token or aggregate bridge row.

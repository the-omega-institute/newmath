\label{thm:real-separability-l10-cauchy-net-dependency}
Every $\RealSeparabilityUp$ countable dense read that is presented to an L10 Cauchy-net consumer first opens the dyadic tolerance rows of \autoref{ch:concrete-instances-dyadicratcore-namecert}, the finite $\StreamNameUp$ windows of \autoref{ch:concrete-instances-streamname-namecert}, and the $\RegSeqRatUp$ readback rows of \autoref{ch:concrete-instances-regseqrat-namecert}. \end{theorem}
\begin{proof}
Project the separability source route. A dense-read consumer sees rational or dyadic candidates only through displayed finite windows and regular rational readback, so the dyadic ledger, stream window, and RegSeqRat rows are upstream of the Real-facing density claim.

The Real seal is terminal. It receives the inspected dense candidate after the finite rows have been displayed; it does not create a host dense subset, selected nearest point, quotient equality, or ambient completion coordinate.

Thus an L10 Cauchy-net consumer reads $\RealSeparabilityUp$ through the same DyadicRatCore, StreamName, RegSeqRat, and Real rows before any countable dense handoff is public.
\end{proof}

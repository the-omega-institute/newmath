\begin{theorem}[Real dyadic RegSeq cross-reference lattice]
\label{thm:real-dyadic-regseq-crossref-lattice}
Every accepted $\RealUp$ current-phase name read factors through four displayed faces:
\begin{enumerate}
\item the $\DyadicRatCoreUp$ signed-mantissa and tolerance rows of \autoref{ch:concrete-instances-dyadicratcore-namecert};
\item the $\StreamNameUp$ finite observation schedule of \autoref{ch:concrete-instances-streamname-namecert};
\item the $\RegSeqRatUp$ regularity and classifier rows of \autoref{ch:concrete-instances-regseqrat-namecert};
\item the local $\RealUp$ seal of \autoref{ch:concrete-instances-real-namecert}.
\end{enumerate}
The cross-reference lattice is the same finite route read by the Real L10 exit audit pullback. It explicitly refuses quotient streams, selected limits, host real equality, ambient $\RealUp$ completeness, metric-completeness data, countable-choice selectors, and any standard bridge row not displayed by the four faces.
\end{theorem}
\begin{proof}
Start with the Real L10 exit audit pullback of \autoref{thm:real-l10-current-phase-exit-audit-pullback}. It identifies each accepted current Phase Real and Completion exit read with the product of the DyadicRatCore, StreamName, RegSeqRat, and local Real face records.

Next use the ordered lattice in \autoref{def:real-l10-dependency-lattice} and the source-route non-escape theorem \autoref{thm:real-l10-source-route-nonescape}. These keep the RealUp current-phase name read on the dyadic observation rows, the finite StreamName window rows, the RegSeqRat regular-rational rows, and the local Real seal.

Now attach the RegSeqRat scope triad. By \autoref{thm:regseqrat-streamname-real-scope-triad}, every public RealUp seal read through RegSeqRat is placed behind the visible StreamName schedule, the DyadicRatCore endpoint and tolerance rows, and the RegSeqRat classifier surface before the seal row is consumed.

Finally combine the non-escape clauses. The Real L10 pullback contributes no fifth formal token or completion row, and the RegSeqRat scope triad contributes no quotient stream, selected limit, host equality, completion object, ambient completeness datum, or second seal coordinate. Therefore the current-phase RealUp name read is exactly the four-face cross-reference lattice and exports none of the refused data.
\end{proof}

\begin{theorem}[Real L10 four-face phase-audit lock]
\label{thm:real-l10-four-face-phase-audit-lock}
An accepted current Phase Real and Completion audit read of $\RealUp$ is equivalent, as a public BEDC route, to the four displayed faces $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, and the local $\RealUp$ seal, together with their $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows. The route refuses quotient streams, host real equality, selected limits, ambient $\RealUp$ completeness, metric completeness, countable choice, and standard bridge rows not displayed by those four faces.
\end{theorem}
\begin{proof}
For the forward direction, project the four faces using \autoref{thm:real-dyadic-regseq-crossref-lattice}. Every accepted current-phase name read already factors through the DyadicRatCore signed-mantissa and tolerance rows, the StreamName finite observation schedule, the RegSeqRat regularity and classifier rows, and the local Real seal.

Next use the Real L10 current phase exit audit pullback, \autoref{thm:real-l10-current-phase-exit-audit-pullback}. It identifies the Phase Real and Completion audit read with the same four-face product route, so no additional completion-facing source is needed for the forward projection.

For the reverse direction, assemble the public route from the four sibling rows and the Real L10 dependency lattice. The ordered lattice of \autoref{def:real-l10-dependency-lattice}, together with \autoref{thm:real-l10-source-route-nonescape} and \autoref{thm:regseqrat-streamname-real-scope-triad}, repacks the four displayed faces as the current Phase Real and Completion audit read.

Finally combine the exclusion clauses. The cross-reference lattice, L10 nonescape row, and RegSeqRat scope triad do not expose quotient streams, host real equality, selected limits, ambient Real completeness, metric completeness, countable choice, or standard bridge rows, so the public audit route is locked to exactly the four displayed faces and their structural support.
\end{proof}

\begin{theorem}[Real L10 finite-cover face-pasting equivalence]
\label{thm:real-l10-finite-cover-face-pasting-equivalence}
Let $\{\mathcal{W}_i\}_{i\in I}$ be a finite cover of certified $\StreamNameUp$ windows. Assume that every pairwise overlap and every triple overlap satisfies the StreamName-RegSeqRat pasted source and seal hypotheses of \autoref{thm:streamname-regseqrat-finite-cover-pasted-source-exhaustion} and \autoref{thm:streamname-regseqrat-finite-cover-pasted-seal-exhaustion}. Assume also that all local rows consume the same displayed $\DyadicRatCoreUp$ endpoint, radius, tolerance, and finite-distance ledger, and that their $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows agree on all displayed overlaps.

Then the induced local $\RegSeqRatUp$ sections paste to exactly one accepted current Phase Real and Completion $\RealUp$ exit certificate over the finite union if and only if every local member exposes the same four visible faces
$$
\DyadicRatCoreUp,\quad \StreamNameUp,\quad \RegSeqRatUp,\quad \RealUp
$$
over the same $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows.

The equivalence is only a finite-cover current-phase exit certificate. It refuses quotient stream equality, selected limits, host real equality, countable choice, metric-completeness data, aggregate formalstatus rows, aggregate bridge rows, and ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
For the forward direction, suppose the local $\RegSeqRatUp$ sections paste to one accepted current Phase Real and Completion $\RealUp$ exit certificate. Project that certificate through \autoref{thm:real-dyadic-regseq-crossref-lattice}. The projection exposes only the DyadicRatCore ledger, the StreamName finite windows, the RegSeqRat local sections, and the local Real seal.

Now restrict this four-face packet to each member $\mathcal{W}_i$ of the finite cover. The phase-audit lock of \autoref{thm:real-l10-four-face-phase-audit-lock} keeps the restriction on the same $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows. Hence every local member exposes the same four visible faces over the same displayed support ledger.

For the reverse direction, assume each local member exposes the same four visible faces over the same support rows. First paste the StreamName and finite-window Real source surface using \autoref{thm:streamname-regseqrat-finite-cover-pasted-source-exhaustion}. Then paste the finite-window Real seal rows using \autoref{thm:streamname-regseqrat-finite-cover-pasted-seal-exhaustion}. These two pasting steps use only the certified pairwise overlaps, triple-overlap cocycles, and shared DyadicRatCore ledger.

The pasted source and seal rows now form a finite cover of compatible local $\RegSeqRatUp$ terminal sections. Apply \autoref{thm:regseqrat-four-face-local-gluing-real-seal-coverage} to obtain an accepted Real seal from the local four-face packets, and apply \autoref{thm:regseqrat-four-face-gluing-uniqueness} to identify any two such pasted seals as the same BEDC-visible four-face packet.

Finally read the unique pasted four-face packet as the current Phase Real and Completion exit certificate by \autoref{thm:real-l10-four-face-phase-audit-lock}. The construction has no coordinate for quotient stream equality, selected limits, host real equality, countable choice, metric-completeness data, aggregate formalstatus rows, aggregate bridge rows, or ambient Real completeness, so the certificate is exactly the finite-cover current-phase exit surface described in the statement.
\end{proof}

\begin{theorem}[RealUp L10 input factorization]
\label{thm:realup-l10-input-factorization}
Every public $\RealUp$ seal read or $\RealUp$ classifier read that is usable as Phase Real and Completion data factors through the L10 four-object input stack
$$
\DyadicRatCoreUp \to \StreamNameUp \to \RegSeqRatUp \to \RealUp .
$$
The factorization first reads the $\DyadicRatCoreUp$ observation rows of \autoref{ch:concrete-instances-dyadicratcore-namecert}, then the $\StreamNameUp$ finite-window rows of \autoref{ch:concrete-instances-streamname-namecert}, then the $\RegSeqRatUp$ regular readback rows of \autoref{ch:concrete-instances-regseqrat-namecert}, and only then the local $\RealUp$ seal or classifier surface. The visible route is transported by the displayed $\hsame$ rows, replayed by the displayed $\Cont$ rows, packaged by $\Pkg$ provenance, and named by the local $\NameCert$ rows before it is accepted as $\RealUp$ data.

The factorization exports no quotient stream, host real equality, selected limit, countable choice, or ambient completeness coordinate.
\end{theorem}
\begin{proof}
Start with the Real L10 dependency lattice of \autoref{def:real-l10-dependency-lattice}. It orders the public input surface by dyadic observations, finite StreamName windows, regular-rational readback, and the local Real seal rows.

Apply the source-route and interface-totality theorems, \autoref{thm:real-l10-source-route-nonescape} and \autoref{thm:real-l10-dependency-lattice-interface-totality}. They place every Phase Real and Completion interface read inside the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ route over the same four faces.

The sibling route theorem \autoref{thm:real-l10-open-phase-sibling-route} supplies the cross-chapter handoff through \autoref{ch:concrete-instances-dyadicratcore-namecert}, \autoref{ch:concrete-instances-streamname-namecert}, and \autoref{ch:concrete-instances-regseqrat-namecert}. Thus a public Real seal read or classifier read reaches the local $\RealUp$ surface only after those sibling rows have been displayed.

Finally use the terminal route theorems \autoref{thm:real-dyadic-stream-regseq-route} and \autoref{thm:real-terminal-four-face-open-phase-exit}. They exhaust the public exit by the four-object route and the structural rows already listed. Since none of those cited surfaces contains a quotient stream, host real equality, selected limit, countable-choice selector, or ambient completeness coordinate, no such coordinate can be exported by the accepted $\RealUp$ read.
\end{proof}

\begin{theorem}[RealUp L10 downstream interface non-escape]
\label{thm:realup-l10-downstream-interface-nonescape}
Every downstream $\MetricSpaceUp$, $\RealAlgOrderUp$, $\CompleteMetricUp$, $\CompactMetricUp$, or $\ContinuousMapUp$ consumer that uses a public $\RealUp$ seal row or public $\RealUp$ classifier row must read that row through the same four-object route
$$
\DyadicRatCoreUp \to \StreamNameUp \to \RegSeqRatUp \to \RealUp .
$$
The admissible downstream packet consists only of the displayed $\BHist$ histories, componentwise $\hsame$ transports, displayed $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ rows attached to those four faces.

The packet exports no quotient stream equality, host real equality, selected limit, countable-choice selector, ambient completeness row, or metric-completeness row.
\end{theorem}
\begin{proof}
Begin with \autoref{thm:realup-l10-input-factorization}. It already forces every public $\RealUp$ seal read and classifier read usable as Phase Real and Completion data through $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, and $\RealUp$, with only the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows.

Now pass to the five downstream faces. A $\MetricSpaceUp$, $\RealAlgOrderUp$, $\CompleteMetricUp$, $\CompactMetricUp$, or $\ContinuousMapUp$ consumer can use the $\RealUp$ row only as the public seal or classifier surface supplied by \autoref{ch:concrete-instances-real-namecert}. The sibling surfaces \autoref{ch:concrete-instances-metric-namecert}, \autoref{ch:concrete-instances-realalgorder-namecert}, \autoref{ch:concrete-instances-completemetric-namecert}, \autoref{ch:concrete-instances-compactmetric-namecert}, and \autoref{ch:concrete-instances-continuousmap-namecert} therefore inherit the same Real route rather than a separate analytic entrance.

Finally apply \autoref{thm:real-name-seal-no-completeness}. The public Real seal exports no quotient stream, selected limit, host real equality, host sequence equality, or ambient completeness theorem; the L10 input theorem also excludes countable choice from the route. Since the downstream consumer receives only that public Real row together with the displayed structural support, it cannot acquire quotient stream equality, host real equality, selected limits, countable choice, ambient completeness, or metric-completeness rows from the Real input boundary.
\end{proof}

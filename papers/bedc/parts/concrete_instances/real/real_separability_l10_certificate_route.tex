\begin{theorem}[Real separability L10 certificate route]
\label{thm:real-separability-l10-certificate-route}
Every $\RealSeparabilityUp$ density read that consumes the $\RealUp$ seal factors through the displayed L10 route
$$
\RegSeqRatUp \longmapsto \StreamNameUp \longmapsto \DyadicRatCoreUp \longmapsto \RatUp \longmapsto \CountableDenseSubsetUp \longmapsto \RealUp.
$$
The route is grounded in \autoref{ch:concrete-instances-regseqrat-namecert}, \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-dyadicratcore-namecert}, \autoref{ch:concrete-instances-rat-namecert}, \autoref{ch:concrete-instances-countable-dense-subset-namecert}, and \autoref{ch:concrete-instances-real-namecert}. Thus a separability consumer receives regular-rational readback, finite stream windows, dyadic tolerance rows, rational endpoint rows, countable dense subset naming, and the terminal Real seal as displayed sibling data, with no host dense subset, selected nearest point, quotient real equality, countable-choice enumeration, or ambient completeness coordinate.
\end{theorem}
\begin{proof}
Read the $\RealSeparabilityUp$ consumer as a request for a displayed density route. The regular-rational row supplies the source readback, while the stream row supplies the finite windows on which the density request is inspected.

The dyadic and rational rows then expose the tolerance and endpoint data used by the dense subset certificate. The $\CountableDenseSubsetUp$ row names the countable dense boundary as a sibling certificate rather than as an external host enumeration.

Only after those rows are displayed can the consumer reach the $\RealUp$ seal. Since the route has no coordinate for a host dense subset, selected nearest point, quotient equality, choice enumeration, or ambient completeness, none of those objects can be exported by the separability packet.
\end{proof}

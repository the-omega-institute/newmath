\begin{theorem}[RealUp StreamName RegSeqRat source lattice]
\label{thm:real-streamname-regseqrat-source-lattice}
Every accepted $\RealUp$ source-lattice read used by a Phase Real or
Completion consumer factors through the $\StreamNameUp$ finite-window
schedule of \autoref{ch:concrete-instances-streamname-namecert}, the
$\RegSeqRatUp$ regular readback row of
\autoref{ch:concrete-instances-regseqrat-namecert}, and the
$\DyadicRatCoreUp$ tolerance ledger of
\autoref{ch:concrete-instances-dyadicratcore-namecert} before the Real seal
is public. The read exposes only the displayed $\BHist$, $\hsame$, $\Cont$,
$\Pkg$, and $\NameCert$ coordinates carried by those sibling surfaces and the
local $\RealUp$ seal package.

In particular, the accepted source lattice contains no quotient real equality,
selected representative, ambient completeness coordinate, or direct public
Real seal edge bypassing the StreamName, RegSeqRat, and DyadicRatCore rows.
\end{theorem}
\begin{proof}
Start with an accepted $\RealUp$ source-lattice read. The local Real boundary
only accepts the finite-window package already routed through the displayed
stream, regular-rational, dyadic-tolerance, transport, continuation,
provenance, and naming rows.

First read the source schedule through
\autoref{ch:concrete-instances-streamname-namecert}. This fixes the visible
finite-window row and its unary-history observations before any regular
readback or Real seal row can be consumed.

Next read the regular-rational spine through
\autoref{ch:concrete-instances-regseqrat-namecert}. The RegSeqRat row uses the
same displayed window and supplies the regular readback surface on the common
$\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ route.

Finally attach the tolerance ledger through
\autoref{ch:concrete-instances-dyadicratcore-namecert}. The dyadic row is the
only tolerance surface available to the Real seal package, so the public Real
seal is reached only after the StreamName schedule, RegSeqRat readback, and
DyadicRatCore ledger have all been displayed.

The composed source lattice therefore has no remaining coordinate from which
to read quotient real equality, a selected representative, ambient
completeness, or a direct seal edge around the sibling surfaces.
\end{proof}

\begin{theorem}[Real StreamName RegSeqRat terminal budget stability]
\label{thm:real-streamname-regseqrat-terminal-budget-stability}
Let a terminal $\RealUp$ seal-facing consumer read its source as a
common-refinement $\StreamNameUp$ window, let its regularity witness be the
$\RegSeqRatUp$ readback row on that same window, and let its precision request
be served through $\RealObservationBudgetUp$. Then restricting the request to a
smaller finite observation budget, or enlarging it to a larger finite
observation budget, preserves the same public finite source surface:
$$
\StreamNameUp,\quad \RegSeqRatUp,\quad \DyadicRatCoreUp,\quad \RealUp,\quad
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert .
$$
The budget operation may change only which displayed finite observations are
queried. It cannot add quotient-real equality, host sequence equality, a
selected host limit, ambient Real completeness, an independent Real carrier, or
metric-completeness data.
\end{theorem}
\begin{proof}
Start with the common-refinement source. The consumer lock
\autoref{thm:real-streamname-regseqrat-common-refinement-consumer-lock} fixes
the terminal read on the shared StreamName, RegSeqRat, DyadicRatCore, and Real
seal rows before the seal-facing consumer can inspect the packet.

Now attach the observation-budget route. The StreamName Real
observation-budget factorization
\autoref{thm:streamname-real-observation-budget-factorization} routes every
accepted $\RealObservationBudgetUp$ precision request through the same visible
finite-window, dyadic, regular readback, and Real seal chain.

Finally use the carrier and non-escape rows of
\autoref{ch:concrete-instances-real-observation-budget-namecert}. They make
budget restriction or enlargement a change of displayed finite request rows,
not a change of source carrier. Since the cited routes expose only the finite
BEDC rows listed in the statement, none of the excluded analytic or host-level
objects can enter the terminal seal-facing surface.
\end{proof}

\begin{theorem}[Real StreamName RegSeqRat Dyadic source exhaustion]
\label{thm:real-streamname-regseqrat-dyadic-source-exhaustion}
Every Phase Real and Completion $\RealUp$ source read of the accepted
dependency lattice factors through exactly the displayed
$\StreamNameUp$ finite-window row, the $\RegSeqRatUp$ readback row, the
$\DyadicRatCoreUp$ tolerance ledger, and the local $\RealUp$ seal row. The
factorization exposes only the finite packet coordinates already carried by
those four surfaces and their shared $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and
$\NameCert$ routes.

Consequently the completion-facing source route contains no quotient stream,
selected host limit, ambient completeness theorem, or hidden endpoint equality.
\end{theorem}
\begin{proof}
Begin with the accepted source-lattice route of
\autoref{thm:real-streamname-regseqrat-source-lattice}. It already decomposes
the Real source read into the StreamName finite-window schedule, the
RegSeqRat regular readback row, and the DyadicRatCore tolerance ledger before
the local Real seal is public.

Next apply the terminal budget stability statement
\autoref{thm:real-streamname-regseqrat-terminal-budget-stability}. Changing
the finite observation budget changes only which displayed source observations
are queried; it does not add a carrier row outside the StreamName, RegSeqRat,
DyadicRatCore, and Real seal packet.

It remains to exhaust the packet coordinates. The nearby source-route clauses
give the visible $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows for
each of the four surfaces, while the Real seal row is reached only after those
sibling rows have been displayed.

Thus every completion-facing source read is a coordinate projection of that
finite packet. Since the packet has no coordinate for quotient streams,
selected host limits, ambient completeness theorems, or hidden endpoint
equalities, none of those objects can be read from the accepted source route.
\end{proof}

\chapter{A Concrete Naming Certificate for $\LengthSpaceUp$}
\label{ch:concrete-instances-length-space-namecert}
\origin{human}

Following Burago--Burago--Ivanov and Bridson--Haefliger on length spaces, the $\LengthSpaceUp$ packet records the finite path-length ledger by which a metric distance is witnessed through displayed interval paths and displayed segment sums. It sits over the metric source of \autoref{ch:concrete-instances-metric-namecert}, the continuous-map path surface of \autoref{ch:concrete-instances-continuousmap-namecert}, the interval address space of \autoref{ch:concrete-instances-interval-namecert}, the regular rational readback of \autoref{ch:concrete-instances-regseqrat-namecert}, and the terminal real-name seal of \autoref{ch:concrete-instances-real-namecert}. The packet is a finite ledger for length reads; it is not a geodesic existence theorem, a host path space, a quotient path-space construction, an open-cover compactness result, a $\RealUp$ completeness theorem, or a selected-minimizer principle.

\begin{definition}[LengthSpace finite path-length carrier]
\label{def:length-space-carrier}
A $\LengthSpaceUp$ carrier is a finite $\BHist$ packet
$$
L=(M,I,A,D,S,R,T,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $M$ is the metric source row, recording the accepted $\MetricUp$ carrier, the endpoint identifiers, and the finite source-distance request consumed by the length ledger.
\item $I$ is the interval path row, recording a finite $\IntervalUp$ address family and the displayed $\ContinuousMapUp$ path read used to connect the two endpoint identifiers.
\item $A$ is the partition ledger, recording a finite ordered partition of the displayed interval addresses, including adjacent address pairs and endpoint coverage.
\item $D$ is the segment-distance ledger, recording the $\MetricUp$ distance read for each adjacent pair in $A$ after the path row $I$ has supplied its two point reads.
\item $S$ is the segment-sum row, recording the finite rational sum of the entries in $D$ and the tolerance split used to compare it with the source-distance request in $M$.
\item $R$ is the $\RegSeqRatUp$ readback row, exposing the finite segment sums as regular rational observations before any terminal real seal is consumed.
\item $T$ is the $\RealUp$ length-readback row, sealing the regular rational observations from $R$ as a displayed real-name length value for this finite path ledger.
\item $C$ records componentwise $\hsame$ transport for the metric source, interval path row, partition ledger, segment-distance ledger, regular rational readback, real length readback, replay, provenance, and local naming rows.
\item $P$ is the displayed $\Cont$ replay and $\Pkg$ provenance over $\LengthSpaceUp$, $\MetricUp$, $\ContinuousMapUp$, $\IntervalUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\LengthSpaceUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $M,I,A,D,S,R,T,C,P,N$. It has no coordinate for geodesic existence, host paths, quotient path spaces, open-cover compactness, ambient $\RealUp$ completeness, or selected minimizers.
\end{definition}

\begin{theorem}[LengthSpace path ledger stability]
\label{thm:length-space-path-ledger-stability}
Let
$$
L=(M,I,A,D,S,R,T,C,P,N)
$$
be an accepted $\LengthSpaceUp$ carrier. Every endpoint-distance or length-read consumer of $L$ factors through the displayed finite route
$$
M,I,A,D,S \longmapsto R,T \longmapsto C,P,N .
$$
Consequently componentwise $\hsame$ transport of $M,I,A,D,S,R,T,C,P,N$ preserves the same metric source, interval path row, partition ledger, segment-distance ledger, $\RegSeqRatUp$ readback, and $\RealUp$ length seal, with no additional route through host paths, quotient path spaces, open-cover compactness, real completeness, geodesic selection, or minimizing-path choice.
\end{theorem}
\begin{proof}
Carrier projection gives the ordered rows $M,I,A,D,S,R,T,C,P,N$. A consumer begins with the metric source row $M$, so the endpoint-distance request is tied to a displayed $\MetricUp$ source before any path-length read is available.

The path data are finite. The interval path row $I$ supplies the displayed $\ContinuousMapUp$ reads over $\IntervalUp$, and the partition ledger $A$ lists only finitely many adjacent interval addresses. Thus every segment request is bounded by the displayed partition.

The segment-distance ledger $D$ reads each adjacent pair through the metric source, and $S$ folds those entries into one finite segment sum. The readback rows $R$ and $T$ expose that sum first as $\RegSeqRatUp$ data and then as a terminal $\RealUp$ length seal.

Transport and replay are structural. Componentwise $\hsame$ in $C$ preserves each listed row, while the displayed provenance in $P$ and the local name in $N$ replay the same finite route.

The excluded objects are absent from the carrier inventory. Since no row stores host paths, quotient path spaces, open covers, ambient completeness, geodesic witnesses, or minimizing-path choices, no consumer can obtain them from the transported packet.
\end{proof}

\begin{theorem}[LengthSpace NameCert obligation surface]
\label{thm:length-space-namecert-obligation-surface}
For an accepted $\LengthSpaceUp$ carrier
$$
L=(M,I,A,D,S,R,T,C,P,N),
$$
the local $\NameCert_{\LengthSpaceUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the metric source, interval path row, partition ledger, segment-distance ledger, segment-sum row, regular rational readback, real length readback, transport, replay, provenance, and local naming rows;
\item path-source exactness, requiring every displayed path read to pass through $M$ and then $I$ before a segment endpoint can be inspected;
\item partition exactness, requiring $A$ to list the finite ordered interval addresses and adjacent pairs consumed by the segment-distance ledger;
\item segment-distance exactness, requiring $D$ to read each adjacent pair through the same $\MetricUp$ source and no other distance object;
\item length-readback exactness, requiring $S,R,T$ to expose the finite segment sum through $\RegSeqRatUp$ and $\RealUp$ before any consumer treats it as a length value;
\item structural exhaustion, requiring $C,P,N$ to transport, replay, source, and name only the displayed route $M,I,A,D,S,R,T$.
\end{enumerate}
No NameCert obligation is stored outside those rows, and none asserts geodesic existence, host paths, quotient path spaces, open-cover compactness, $\RealUp$ completeness, or selected minimizers.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:length-space-carrier}. The accepted packet displays the metric source, interval path row, partition ledger, segment-distance ledger, segment sum, readback rows, and structural rows.

Path-source exactness is read from $M$ and $I$. A path endpoint can be inspected only after the source metric row and the displayed interval path row have both been opened.

Partition exactness is the role of $A$. It supplies the finite ordered interval addresses and adjacent pairs consumed downstream, so the segment-distance ledger has no unlisted path coordinate.

Segment-distance exactness is read from $D$. Each adjacent pair is measured through the same metric source row $M$, and no additional distance object is present.

Length-readback exactness is read from $S,R,T$. The finite segment distances are summed in $S$, exposed as regular rational observations in $R$, and sealed as a real-name length value in $T$.

Finally $C,P,N$ add no analytic coordinate. They preserve, replay, source, and name the same finite route, so the excluded geodesic, host-path, quotient, compactness, completeness, and minimizer data have no row from which a consumer could read them.
\end{proof}

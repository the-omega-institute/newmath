\chapter{A Concrete Naming Certificate for $\CauchyCompletionRecursorUp$}
\label{ch:concrete-instances-cauchy-completion-recursor-namecert}
\origin{human}

Following Bishop and Bridges on constructive completion and uniformly continuous extension, the $\CauchyCompletionRecursorUp$ packet records the elimination principle by which a completion-facing consumer reads a completed object only through a displayed dense source, a modulus-controlled map, an extension handoff, and a uniqueness row. It sits after the regular completion carrier of \autoref{ch:concrete-instances-bishop-regular-cauchy-completion-namecert}, the source admission row of \autoref{ch:concrete-instances-cauchycompletionunit-namecert}, the uniformly continuous extension packet of \autoref{ch:concrete-instances-uniform-continuous-extension-namecert}, and the terminal Real seal of \autoref{ch:concrete-instances-real-namecert}. The recursor is an elimination route for displayed completion data; it is not a selected limit, quotient stream, countable-choice principle, host real equality, or ambient $\RealUp$ completeness theorem.

\begin{definition}[CauchyCompletionRecursor carrier]
\label{def:cauchy-completion-recursor-carrier}
A $\CauchyCompletionRecursorUp$ carrier is a finite $\BHist$ packet
$$
R=(S,T,f,m,E,U,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the dense source row, read through the Bishop regular Cauchy completion carrier and the $\CauchyCompletionUnitUp$ source-admission route.
\item $T$ is the completion target row, exposing only the completion-facing boundary consumed by the recursor.
\item $f$ is the source map row, visible only on the displayed dense source $S$.
\item $m$ is the uniform modulus row controlling the source map before any completion-facing read is admitted.
\item $E$ is the extension handoff row supplied by the uniformly continuous extension route.
\item $U$ is the uniqueness row, requiring any two admitted extension reads with the same source restriction and modulus ledger to agree through the displayed completion and Real seal rows.
\item $H$ records componentwise $\hsame$ transport for the dense source, target, source map, modulus, extension handoff, uniqueness row, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a completion-facing request through $S,f,m,E,U$ and then to the terminal Real seal.
\item $P$ is the $\Pkg$ provenance over $\CauchyCompletionRecursorUp$, $\BishopRegularCauchyCompletionUp$, $\CauchyCompletionUnitUp$, $\UniformContinuousExtensionUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CauchyCompletionRecursorUp}$ row.
\end{enumerate}
The carrier compares accepted packets only through these rows. It has no coordinate for selected limits, quotient stream representatives, countable choice, host real equality, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[CauchyCompletionRecursor NameCert obligations]
\label{thm:cauchy-completion-recursor-namecert-obligations}
For every accepted $\CauchyCompletionRecursorUp$ carrier $R=(S,T,f,m,E,U,H,C,P,N)$, the local $\NameCert_{\CauchyCompletionRecursorUp}$ surface consists exactly of the following finite obligations:
\begin{enumerate}
\item carrier admission for the dense source, completion target, source map, modulus ledger, extension handoff, uniqueness row, transport, replay, provenance, and local naming rows;
\item source restriction, requiring $f$ to be read only on the displayed dense source $S$ admitted by the $\CauchyCompletionUnitUp$ route;
\item modulus preservation, requiring every extension read to preserve the same uniform modulus row $m$ before the target row $T$ is consumed;
\item extension handoff, requiring $E$ to be supplied through the uniformly continuous extension route and not by a selected limit or host completion theorem;
\item uniqueness, requiring $U$ to identify any two admitted extension reads with the same source restriction, modulus ledger, and completion target route;
\item ledger non-escape, requiring $H,C,P,N$ to replay and name only the displayed rows and to export no selected limit, quotient stream, countable-choice selector, host real equality, or ambient $\RealUp$ completeness row.
\end{enumerate}
No local NameCert obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cauchy-completion-recursor-carrier}. The visible packet consists exactly of the dense source row $S$, target row $T$, source map row $f$, modulus row $m$, extension handoff $E$, uniqueness row $U$, and structural rows $H,C,P,N$.

Source restriction follows from the placement of $S$ before $f$. The source map can be inspected only after the dense source has been displayed through the Bishop regular completion and $\CauchyCompletionUnitUp$ routes.

Modulus preservation and extension handoff are read through $m$ and $E$. The uniformly continuous extension route consumes the same modulus ledger before it exposes an extension row, so no selected limit or ambient completion theorem can replace the displayed handoff.

Uniqueness is exactly the role of $U$. Once the source restriction, modulus ledger, target row, and extension handoff agree, the local uniqueness row identifies the two recursor reads through the displayed completion-facing route.

Finally, $H,C,P,N$ exhaust the ledger. They transport, replay, source, and name the displayed rows only, so there is no carrier coordinate from which quotient streams, selected limits, countable choice, host real equality, or ambient Real completeness can be exported.
\end{proof}

\begin{theorem}[CauchyCompletionRecursor universal route]
\label{thm:cauchy-completion-recursor-universal-route}
Every completion-facing consumer of an accepted $\CauchyCompletionRecursorUp$ packet factors through the finite route
$$
\BishopRegularCauchyCompletionUp \longmapsto \CauchyCompletionUnitUp \longmapsto \UniformContinuousExtensionUp \longmapsto \RealUp .
$$
More explicitly, the consumer first reads the displayed dense source $S$, then the source map $f$ and modulus ledger $m$, then the extension handoff $E$, then the uniqueness row $U$, and only then the terminal Real seal. The route contains no selected limit, countable choice, quotient stream, host real equality, or ambient $\RealUp$ completeness row.
\end{theorem}
\begin{proof}
Start with an accepted packet $R=(S,T,f,m,E,U,H,C,P,N)$. The dense source row $S$ is the only source coordinate available to the consumer, and it is admitted through the Bishop regular completion and $\CauchyCompletionUnitUp$ routes.

Replay the continuation row $C$. A completion-facing request reaches $f$ only after $S$ is displayed, and it reaches $E$ only after the modulus ledger $m$ has been read. Thus the extension handoff is modulus-controlled rather than limit-selected.

The uniqueness row $U$ is read before the terminal Real seal. It forces any two admitted extension reads with the same source restriction, modulus ledger, and target route to share the same completion-facing output row.

The structural rows $H,C,P,N$ preserve this order under transport, replay, provenance, and naming. Since the carrier has no coordinate for selected limits, countable choice, quotient streams, host real equality, or ambient Real completeness, no completion-facing consumer can export any of those data.
\end{proof}

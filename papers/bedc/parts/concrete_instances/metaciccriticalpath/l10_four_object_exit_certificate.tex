\begin{theorem}[MetaCIC critical-path L10 four-object exit certificate]
\label{thm:metacic-critical-path-l10-four-object-exit-certificate}
For every accepted $\MetaCICCriticalPathUp$ packet $K=(S,N,O,U,D,H,C,P,L)$, the current Phase Real and Completion exit surface is reached exactly by displaying four finite source rows attached to $K$: the $\DyadicRatCoreUp$ finite dyadic budget, the $\StreamNameUp$ finite schedule row, the $\RegSeqRatUp$ regular-rational readback row, and the $\RealUp$ terminal seal row. Each displayed row is carried through finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ provenance, as synchronized by Theorem~\ref{thm:metacic-critical-path-l10-entry-budget-synchronization}, filtered by Theorem~\ref{thm:metacic-critical-path-real-completion-frontier-filter}, arranged by Theorem~\ref{thm:metacic-critical-path-phase-real-dependency-diamond}, and kept source-owner separated by Theorem~\ref{thm:metacic-critical-path-four-source-owner-noncollapse}. No additional exit coordinate is supplied by quotient stream equality, selected host limits, host real equality, trichotomy, open-cover compactness, countable choice, or completed-real closure.
\end{theorem}
\begin{proof}
First take the synchronized L10 entry surface. Theorem~\ref{thm:metacic-critical-path-l10-entry-budget-synchronization} requires the dyadic budget, StreamName schedule, RegSeqRat readback, and Real seal rows to be displayed through finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ provenance before a Phase Real or Completion consumer can enter the route.

Next pass to the Completion-facing frontier. Theorem~\ref{thm:metacic-critical-path-real-completion-frontier-filter} admits only claims backed by those four displayed rows, so every accepted exit certificate still reads from the same finite packet surface rather than from quotient stream equality, selected host limits, host real equality, trichotomy, open-cover compactness, countable choice, or completed-real data.

The dependency shape is then fixed by the diamond. Theorem~\ref{thm:metacic-critical-path-phase-real-dependency-diamond} says that a public Phase Real or Completion entry factors through exactly the four displayed source rows and recovers no other public entry row.

Finally retain the four owners separately. Theorem~\ref{thm:metacic-critical-path-four-source-owner-noncollapse} refuses any collapse of the dyadic budget owner, StreamName schedule owner, RegSeqRat readback owner, and Real seal owner into an aggregate route token. The four-object certificate is therefore precisely the current finite exit surface, and every listed host, choice, compactness, quotient, or completed-real principle remains outside the accepted packet route.
\end{proof}

$\KernelMorphismUp$ supplies source-target graph admission and
classifier-lift rows, while $\CertificateCompilerUp$ adds target landing,

\begin{theorem}[CertificateCompiler target endpoint route]
\label{thm:certificate-compiler-target-endpoint-route}
Let
$$
K=(S,T,G,E,C,H,P,N)
$$
be an accepted $\CertificateCompilerUp$ packet. Every route by which $K$
reaches the target certificate $T$ is exhausted by the source $\NameCert$ row
$S$, the displayed compiler graph $G$, the target landing row $E$, the
classifier transport row $H$, the continuation row $C$, the $\Pkg$
provenance row $P$, and the local $\NameCert$ row $N$. In particular, target
arrival is a finite $\BHist$ read through $S,G,E,H,C,P,N$, not a read of host
compiler semantics, optimiser correctness, ambient program equality, or
unrecorded target behavior.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:certificate-compiler-carrier}. The
accepted packet exposes exactly the source and target naming rows, graph,
landing, continuation, componentwise $\hsame$ transport, package provenance,
and local naming row.

Endpoint availability is supplied by
\autoref{thm:certificate-compiler-endpoint-certificate-availability}, so the
target certificate is already one of the displayed rows. Edge-classifier
exactness from \autoref{thm:certificate-compiler-edge-classifier-exactness}
then forces each graph edge to land through $E$ and to transport classifier
data only through $H$ and the paired $\Cont$ route in $C$.

Finally apply the public consumer boundary of
\autoref{thm:certificate-compiler-public-consumer-boundary}. A downstream
read can consume only the displayed graph, landing, classifier transport,
continuation, package, and local naming rows. Thus every target endpoint route
is the stated finite $\BHist$ route through $S,G,E,H,C,P,N$.
\end{proof}
\leanchecked{BEDC.Derived.CertificateCompilerUp.CertificateCompilerCarrier\_target\_endpoint\_route}

\begin{theorem}[CertificateCompiler identity composite public boundary]
\label{thm:certificate-compiler-identity-composite-public-boundary}
For accepted $\CertificateCompilerUp$ packets, identity rows, binary
composites, and triple composites remain inside the same public row surface
$S,T,G,E,C,H,P,N$. The identity packet exposes only the diagonal rows of its
local $\NameCert$ input, a binary composite exposes only the two displayed
compiler packets and their shared middle landing, and a triple composite
exposes only the three displayed graph edges, the two middle landing rows,
the ordered $\hsame$ transports, the composed $\Cont$ route, the $\Pkg$
provenance, and the local $\NameCert$ rows.
\end{theorem}
\begin{proof}
For identities, apply
\autoref{thm:certificate-compiler-identity-ledger}. The diagonal packet reads
the carrier, classifier, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows
already displayed by the local certificate.

For binary composites, use
\autoref{thm:certificate-compiler-composition-stability} together with
\autoref{thm:certificate-compiler-identity-composite-ledger-exhaustion}. The
composite graph is formed from displayed graph edges, and its landing,
classifier, continuation, package, and local naming rows are projected from
the two input packets and their middle $\NameCert$ row.

For triple composites, apply
\autoref{thm:certificate-compiler-triple-composite-associativity}. The two
associations expose the same three graph edges, two middle landing matches,
ordered classifier transports, and continuation endpoint.

The left and right unit laws,
\autoref{thm:certificate-compiler-left-identity-unit} and
\autoref{thm:certificate-compiler-right-identity-unit}, identify identity
composites with the original packet surface. Combining these rows gives the
same public boundary $S,T,G,E,C,H,P,N$ for identity, binary, and triple
compiler packets.
\end{proof}

\begin{theorem}[CertificateCompiler downstream stage unblock package]
\label{thm:certificate-compiler-downstream-stage-unblock-package}
The target endpoint route of
\autoref{thm:certificate-compiler-target-endpoint-route} and the
identity/composite boundary of
\autoref{thm:certificate-compiler-identity-composite-public-boundary} supply
the public package consumed by $\CertificatePipelineStageBoundaryUp$,
$\KernelMorphismUp$, and $\DecompilerRightInverseUp$. The package consists
only of finite $\BHist$ rows, displayed $\hsame$ transport, $\Cont$ routes,
$\Pkg$ provenance, and local $\NameCert$ rows over
$K=(S,T,G,E,C,H,P,N)$; it does not export host compiler semantics, optimiser
correctness, decompiler correctness, ambient program equality, or target
behavior absent from the packet.
\end{theorem}
\begin{proof}
First build the endpoint part of the package. By
\autoref{thm:certificate-compiler-target-endpoint-route}, every accepted
target route reads the source naming row, graph row, landing row, classifier
transport row, continuation row, package provenance, and local naming row.

Next build the compositional part. By
\autoref{thm:certificate-compiler-identity-composite-public-boundary},
identities, binary composites, and triple composites keep the same public
row surface and do not add a host compiler coordinate.

Now combine these with
\autoref{thm:certificate-compiler-source-target-landing-exhaustion} and
\autoref{thm:certificate-compiler-finite-route-non-escape}. The source and
target endpoints are exhausted by the displayed packet rows, and every finite
route stays inside $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$
coordinates.

The consumers named in the statement require exactly this public route
surface: pipeline stages read adjacent compiler boundaries, kernel morphisms
read source-to-target graph preservation, and decompiler right-inverse
packets read the displayed compiler/decompiler interface. Since all three
reads factor through the same finite package, none receives a host compiler
semantics or optimiser-correctness theorem.
\end{proof}

\begin{theorem}[CertificateCompiler target endpoint determinacy]
\label{thm:certificate-compiler-target-endpoint-determinacy}
Let $K=(S,T,G,E,C,H,P,N)$ be an accepted $\CertificateCompilerUp$ packet. Once
the source row $S$, a displayed graph edge in $G$, its landing row $E$, the
classifier transport row $H$, and the continuation row $C$ are fixed, the
target endpoint route to $T$ is determined as the finite route through
$S,G,E,H,C,P,N$. No second endpoint route can be obtained from host compiler
semantics, optimiser correctness, ambient program equality, or an unrecorded
target behavior.
\end{theorem}
\begin{proof}
Start from \autoref{thm:certificate-compiler-target-endpoint-route}. It
identifies every target arrival with the displayed source, graph, landing,
classifier, continuation, package, and local naming rows.

Use \autoref{thm:certificate-compiler-edge-classifier-exactness}. With the
source row, graph edge, landing row, classifier transport, and continuation
row fixed, the edge has no further classifier-facing or endpoint-facing
coordinate to vary.

Finally apply \autoref{thm:certificate-compiler-finite-route-non-escape}. The
carrier exports no external compiler semantics or target behavior, so the
fixed displayed rows determine the endpoint route.
\end{proof}

\begin{theorem}[CertificateCompiler route composition exactness]
\label{thm:certificate-compiler-route-composition-exactness}
For accepted $\CertificateCompilerUp$ packets, the identity route, every
binary composite route, and every triple composite route use only the
displayed graph rows, landing rows, classifier transport rows, and $\Cont$
rows, together with the visible $\Pkg$ and local $\NameCert$ provenance. These
routes introduce no hidden program equality, optimiser correctness theorem,
ambient compiler semantics, or target behavior outside the finite packet.
\end{theorem}
\begin{proof}
For the identity and binary cases, apply
\autoref{thm:certificate-compiler-identity-composite-public-boundary}. The
identity packet reads the diagonal local naming rows, and the binary composite
reads the two displayed compiler packets and their shared middle landing.

For the triple case, use the same public boundary together with
\autoref{thm:certificate-compiler-edge-classifier-exactness}. The three graph
edges, two middle landing rows, ordered classifier transports, and composed
$\Cont$ route are exactly the rows visible to the consumer.

The finite non-escape theorem
\autoref{thm:certificate-compiler-finite-route-non-escape} excludes any
additional semantic coordinate. Hence all identity, binary, and triple
composition routes are exact over the displayed rows.
\end{proof}

\begin{theorem}[CertificateCompiler ledger cut elimination]
\label{thm:certificate-compiler-ledger-cut-elimination}
Let a downstream consumer read the public package supplied by
\autoref{thm:certificate-compiler-downstream-stage-unblock-package}. An
intermediate package row may be cut from that consumer route only when the
same displayed endpoint rows and the same identity, binary, or triple
composite rows remain preserved in the resulting route. A cut that changes
the endpoint route, changes the composite route, or replaces either route by
host compiler semantics, decompiler correctness, ambient program equality, or
unrecorded target behavior is not a valid $\CertificateCompilerUp$ consumer
read.
\end{theorem}
\begin{proof}
First read the downstream package through
\autoref{thm:certificate-compiler-downstream-stage-unblock-package}. The
consumer-visible packet consists only of finite $\BHist$ rows, displayed
$\hsame$ transport, $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$
rows.

Endpoint preservation is forced by
\autoref{thm:certificate-compiler-target-endpoint-route}. If a cut removes an
intermediate package row but keeps the same source, graph, landing,
classifier, continuation, provenance, and local naming rows, the target route
is still the displayed finite route.

Composite preservation is forced by
\autoref{thm:certificate-compiler-identity-composite-public-boundary} and
\autoref{thm:certificate-compiler-route-composition-exactness}. Identity,
binary, and triple composites remain valid exactly when their displayed graph,
landing, classifier, and continuation rows are unchanged.

Finally use \autoref{thm:certificate-compiler-finite-route-non-escape}. Any
cut that replaces a displayed row by host semantics, decompiler correctness,
ambient equality, or unrecorded target behavior leaves the finite packet
surface and is therefore unavailable to downstream consumers.
\end{proof}

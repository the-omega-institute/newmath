\section{The certificate}

\begin{definition}[TotalOrder source specification]
\label{def:totalorder-source-specification}
The source is a certified poset carrier plus apartness-aware trichotomy.
\end{definition}

\begin{definition}[TotalOrder pattern specification]
\label{def:totalorder-pattern-specification}
The pattern is comparison formation producing one of three certified comparison cases.
\end{definition}

\begin{definition}[TotalOrder classifier specification]
\label{def:totalorder-classifier-specification}
The classifier is the poset carrier classifier, with apartness stable under classifier replacement.
\end{definition}

\begin{definition}[TotalOrder stability certificate]
\label{def:totalorder-stability-certificate}
The stability certificate consists of:
\begin{enumerate}
  \item all poset stability fields
  \item strict-order-to-apartness field
  \item trichotomy as explicit comparison data
  \item comparison stability under classifier replacement
  \item transitivity of strict comparison where supplied
\end{enumerate}
\end{definition}

\begin{definition}[TotalOrder ledger policy]
\label{def:totalorder-ledger-policy}
Record the comparison certificate selected for each pair.
\end{definition}

\begin{definition}[\(\TotalOrderUp\) certificate obligations]
\label{def:totalorder-certificate-obligations}
The carrier, source, pattern, classifier, stability, and ledger data listed above are the obligations required for the interface name \(\TotalOrderUp\).
\end{definition}

\begin{theorem}[Concrete singleton history $\TotalOrderUp$ naming certificate]
\label{thm:concrete-singleton-history-totalorder-namecert}
The concrete tuple of
\autoref{def:concrete-singleton-history-totalorder-instance} supplies the
source, pattern, classifier, stability, ledger, and exactness rows for a
$\NameCert$ licensing $\TotalOrderUp$ over the singleton carrier $A_{\emp}$,
classifier $\rho_{\emp}$, order $\le_{\emp}$, and comparison branches
$\mathsf{lt}_{\emp}:=\bot$, $\mathsf{same}_{\emp}:=\rho_{\emp}$, and
$\mathsf{gt}_{\emp}:=\bot$.
\end{theorem}
\begin{proof}
The carrier row is the singleton $\mathsf{BHist}$ source. Apply
\autoref{thm:singleton-history-totalorder-laws} to obtain the carried endpoint
facts, classifier stability, the $\POSetUp$ fields, trichotomy, and comparison
stability for the displayed branches.

The exactness row is the underlying partial-order exactness supplied by
\autoref{thm:singleton-empty-poset-laws}: on carried histories,
$\rho_{\emp}(h,k)$ is equivalent to the two opposing prefix comparisons
$h\le_{\emp}k$ and $k\le_{\emp}h$.

The strict rows contribute no extra witnesses. Both strict comparison branches
are empty by definition, so strict-to-apartness and strict transitivity are
vacuous.

It remains to assemble the public certificate. Repack the source as
$A_{\emp}$, the pattern as the three-way comparison formation, the classifier
as $\rho_{\emp}$, the stability data as the fields just cited, and the ledger
as the selected same-branch comparison certificate.
\end{proof}

The per-object template is: traditional target singleton constructive total
order; BEDC source singleton $\mathsf{BHist}$; carrier $A_{\emp}$; classifier
$\rho_{\emp}$; stability obligations from
\autoref{thm:concrete-singleton-history-totalorder-namecert}; ledger the
selected same-branch comparison certificate; exactness singleton classifier
iff opposing prefix comparisons; standard bridge singleton total order; status
$\mathsf{ProofTarget}$.

\paragraph{Primitive boundary.}
\(\TotalOrderUp\) is not a BEDC primitive. It is admitted only when those obligations are supplied.

\begin{remark}[Zero-axiom encoding note]
\label{rem:totalorder-zero-axiom-note}
LEM-style totality is expressed as decidable comparison data and apartness.
\end{remark}

\chapter{A Concrete Naming Certificate for $\ModelPredictiveControlUp$}
\label{ch:concrete-instances-modelpredictivecontrol-namecert}

The $\ModelPredictiveControlUp$ interface records the finite receding-horizon
surface used by control packets that repeatedly solve a displayed finite
prediction problem and export only the first control row. The reference point
is Rawlings, Mayne, and Diehl, \emph{Model Predictive Control: Theory,
Computation, and Design} (2017), together with Boyd and Vandenberghe,
\emph{Convex Optimization} (2004) for finite convex constraint rows. The BEDC
certificate below names the finite rows of that surface over
$\StateSpaceModelUp$, $\DynSystemUp$, $\MatrixUp$, $\VecSpaceUp$, $\RealUp$,
$\LQRUp$, $\RiccatiUp$, $\ConvexSetUp$, $\PolytopeUp$,
$\ControlObservabilityUp$, and $\ControlControllabilityUp$. It does not assert
infinite-horizon optimality, analytic convergence, stochastic control,
recursive feasibility beyond the displayed terminal rows, or host optimizer
semantics.

\begin{definition}[Model predictive control $\mathsf{BHist}$ carrier]
\label{def:modelpredictivecontrol-bhist-carrier}
A $\ModelPredictiveControlUp$ carrier is a finite $\mathsf{BHist}$ packet
$$
M=(X,U,N,A,Q,W,J,T,O,F,H,R,P,C)
$$
whose visible rows are:
\begin{enumerate}
\item a current-state row $X$ read through $\StateSpaceModelUp$ and
$\VecSpaceUp$;
\item an input row $U$ read through $\VecSpaceUp$ and the displayed control
dependency certificates;
\item a finite prediction-horizon row $N$ read as a unary length bound;
\item a dynamics-transition row $A$ read through $\DynSystemUp$,
$\StateSpaceModelUp$, $\MatrixUp$, and $\VecSpaceUp$;
\item a quadratic-cost row $Q$ read through $\LQRUp$ and $\RiccatiUp$;
\item a constraint-window row $W$ naming the displayed admissible state and
control rows through $\ConvexSetUp$ and $\PolytopeUp$ for each horizon index;
\item a stage-cost ledger $J$ read through $\RealUp$, $\MatrixUp$,
$\VecSpaceUp$, and the finite-cost face of $\LQRUp$;
\item a terminal-cost or terminal-region row $T$ read through $\LQRUp$,
$\RiccatiUp$, $\ConvexSetUp$, and $\PolytopeUp$ when those rows are present;
\item an optimizer-candidate row $O$ containing only a displayed finite control
sequence candidate and its finite rollout ledger;
\item a first-control extraction row $F$ selecting the initial control entry
from $O$;
\item componentwise $\hsame$ transports $H$ for the state, horizon,
transition, window, cost, terminal, candidate, first-control, and provenance
rows;
\item $\Cont$ rollout routes $R$ for reading the finite prediction recursion,
stage-cost accumulation, terminal read, and first-control extraction;
\item a $\mathsf{Pkg}$ provenance row $P$ naming the consumed state-space,
dynamical, matrix, vector-space, real, LQR, Riccati, convex-set, polytope,
observability, and controllability certificates;
\item a local $\NameCert$ provenance row $C$ for the finite receding-horizon
surface.
\end{enumerate}
Two accepted packets classify together only when these displayed rows
transport componentwise by $H$ and every rollout, cost, terminal, and
first-control read follows one of the recorded $\Cont$ routes in $R$.
\end{definition}

\begin{theorem}[Model predictive control finite-horizon obligation]
\label{thm:modelpredictivecontrol-finite-horizon-obligation}
Every accepted $\ModelPredictiveControlUp$ packet exposes its prediction
ledger as exactly the finite row family consisting of the current-state row,
the input row, the unary horizon row, the displayed dynamics-transition row,
the quadratic-cost rows, the rollout $\Cont$ routes, and the provenance rows
of \autoref{def:modelpredictivecontrol-bhist-carrier}. The prediction surface
exports no host trajectory space, no infinite rollout, and no unlogged model
semantics.
\end{theorem}
\begin{proof}
Project the accepted carrier of
\autoref{def:modelpredictivecontrol-bhist-carrier}. The only coordinates that
can generate prediction rows are $X$, $U$, $N$, $A$, $Q$, $R$, $P$, and the
transport data $H$.

Read the rollout one horizon index at a time. Each step follows a recorded
$\Cont$ route from the displayed state and transition rows to the next
displayed state row, so the ledger is finite and indexed by $N$.

Transporting the packet replaces each displayed coordinate by its $\hsame$
match and reuses the same rollout routes. Since the carrier has no coordinate
for a host trajectory space or an infinite stream, the prediction ledger is
exhausted by these finite rows.
\end{proof}

\begin{theorem}[Model predictive control constraint ledger]
\label{thm:modelpredictivecontrol-constraint-ledger}
For an accepted $\ModelPredictiveControlUp$ packet, every admissibility read
is the constraint-window row $W$ together with the horizon index, state and
control rows, rollout routes, convex and polytopal provenance, and terminal
row recorded in \autoref{def:modelpredictivecontrol-bhist-carrier}. The packet
exports no ambient feasible set, no compactness assertion, and no
recursive-feasibility theorem beyond the displayed terminal-cost or
terminal-region row.
\end{theorem}
\begin{proof}
Project the constraint coordinate $W$ from the carrier. Its entries are
indexed by the same finite horizon row $N$ used by the prediction ledger, and
they refer only to the displayed state and control rows through the consumed
$\ConvexSetUp$ and $\PolytopeUp$ certificates.

Use the rollout routes to align each constraint entry with the corresponding
predicted state and candidate control row. The admissibility read is therefore
a row projection from $W$, not an external feasible-set predicate.

The terminal coordinate $T$ may record a terminal cost or terminal region row,
but it is another displayed finite row. No coordinate supplies compactness,
global feasibility, or recursive feasibility beyond this displayed terminal
surface.
\end{proof}

\begin{theorem}[Model predictive control receding-horizon boundary]
\label{thm:modelpredictivecontrol-receding-horizon-boundary}
The control exported by an accepted $\ModelPredictiveControlUp$ packet is
exactly the first-control extraction row $F$ applied to the displayed
optimizer-candidate row $O$. The stage-cost ledger $J$, terminal row $T$,
constraint window $W$, rollout routes $R$, $\hsame$ transports $H$, and
provenance rows $P$ and $C$ certify the finite candidate surface, but they do
not assert that $O$ is a globally optimal host solution.
\end{theorem}
\begin{proof}
Project the candidate row $O$. It contains a displayed finite control sequence
candidate together with the rollout and cost rows that make the candidate
visible inside the packet.

Apply the extraction row $F$. This row selects the initial control entry from
the displayed finite sequence and exports that row as the receding-horizon
control read.

The cost, constraint, terminal, transport, continuation, and provenance rows
certify how the candidate is displayed. None of these rows contains a host
minimizer, a global optimality proof, or an external solver state, so the
exported control row is precisely the first displayed entry of $O$.
\end{proof}

\begin{theorem}[Model predictive control consumer boundary]
\label{thm:modelpredictivecontrol-consumer-boundary}
Every downstream reader of an accepted $\ModelPredictiveControlUp$ packet sees
only the finite state, input, horizon, dynamics, quadratic-cost, constraint,
rollout, terminal, first-control, transport, continuation, and provenance rows
of \autoref{def:modelpredictivecontrol-bhist-carrier}. A consumer cannot read
an infinite-horizon optimum, analytic convergence proof, stochastic controller,
or host optimizer semantics from the packet.
\end{theorem}
\begin{proof}
Project an arbitrary downstream read of the accepted packet. The carrier
provides only the displayed coordinates $X$, $U$, $N$, $A$, $Q$, $W$, $J$,
$T$, $O$, $F$, $H$, $R$, $P$, and $C$.

The prediction, constraint, and receding-horizon theorems identify the only
control-facing rows: finite rollout, finite admissibility, and first-control
extraction. Each one is reached by a recorded $\Cont$ route and transported
componentwise by $\hsame$.

Since the packet has no coordinate for infinite horizons, analytic convergence
data, stochastic laws, or host optimizer semantics, no consumer read can export
those objects.
\end{proof}

\begin{theorem}[Model predictive control NameCert obligation surface]
\label{thm:modelpredictivecontrol-namecert-obligation-surface}
The local $\NameCert_{\ModelPredictiveControlUp}$ obligation surface consists
of five finite rows:
\begin{enumerate}
\item carrier habitation: the finite packet of
\autoref{def:modelpredictivecontrol-bhist-carrier};
\item classifier transport: the componentwise $\hsame$ rows for the carrier
coordinates and the $\Cont$ routes that preserve the displayed reads;
\item finite horizon rollout: the finite prediction rows of
\autoref{thm:modelpredictivecontrol-finite-horizon-obligation};
\item constraint ledger exactness: the admissibility rows of
\autoref{thm:modelpredictivecontrol-constraint-ledger};
\item consumer boundary: the $\mathsf{Pkg}$ and $\NameCert$ rows of
\autoref{thm:modelpredictivecontrol-consumer-boundary} naming
$\StateSpaceModelUp$, $\DynSystemUp$, $\MatrixUp$, $\VecSpaceUp$, $\RealUp$,
$\LQRUp$, $\RiccatiUp$, $\ConvexSetUp$, $\PolytopeUp$,
$\ControlObservabilityUp$, and $\ControlControllabilityUp$ as the only
imported certificate surfaces.
\end{enumerate}
These rows form the finite seed obligation surface for naming receding-horizon
model predictive control data in BEDC.
\end{theorem}
\begin{proof}
The carrier row fixes every coordinate of the packet. Its displayed state,
horizon, transition, window, cost, terminal, candidate, extraction, transport,
continuation, package, and local name rows are all finite $\mathsf{BHist}$
data.

The finite-horizon, constraint-ledger, receding-horizon, and consumer-boundary
theorems identify the only rollout, admissibility, exported-control, and
downstream-consumer reads. Each read is a projection from the same finite
packet and follows one of the recorded $\Cont$ routes.

The provenance row then lists exactly the consumed control and algebra
certificates. Since no extra coordinate carries host optimization, analytic
minimizers, compactness, infinite horizons, or unlogged feasibility data, the
five rows exhaust the local $\NameCert_{\ModelPredictiveControlUp}$ surface.
\end{proof}

\begin{theorem}[Model predictive control scoped kernel support]
\label{thm:modelpredictivecontrol-scoped-kernel-support}
The finite MPC carrier, finite-horizon obligation, constraint ledger,
receding-horizon boundary, consumer boundary, and $\NameCert$ obligation
surface are scoped exactly by the displayed $\mathsf{BHist}$ packet rows,
$\hsame$ transport rows, $\Cont$ rollout routes, $\mathsf{Pkg}$ provenance,
$\NameCert$ provenance, and the cited $\StateSpaceModelUp$, $\DynSystemUp$,
$\MatrixUp$, $\VecSpaceUp$, $\RealUp$, $\LQRUp$, $\RiccatiUp$,
$\ConvexSetUp$, $\PolytopeUp$, $\ControlObservabilityUp$, and
$\ControlControllabilityUp$ sources. The scoped support exports no host
optimizer semantics, infinite-horizon optimality, stochastic control, or
unlogged feasibility data.
\end{theorem}
\begin{proof}
Project the carrier coordinates
$M=(X,U,N,A,Q,W,J,T,O,F,H,R,P,C)$ from
\autoref{def:modelpredictivecontrol-bhist-carrier}. These coordinates are the
only state, input, horizon, transition, cost, constraint, terminal, candidate,
first-control, transport, continuation, package, and naming rows in the
packet.

Apply
\autoref{thm:modelpredictivecontrol-finite-horizon-obligation},
\autoref{thm:modelpredictivecontrol-constraint-ledger},
\autoref{thm:modelpredictivecontrol-receding-horizon-boundary}, and
\autoref{thm:modelpredictivecontrol-consumer-boundary}. They identify the
prediction rows, admissibility rows, first-control read, and downstream
consumer reads as projections from the same finite carrier, transported by
$\hsame$ and routed by the displayed $\Cont$ rollout data.

Finally repack the five rows of
\autoref{thm:modelpredictivecontrol-namecert-obligation-surface}. The
$\mathsf{Pkg}$ row names exactly the listed control, algebra, convexity, and
real-number sources, while the local $\NameCert$ row records the
receding-horizon surface. Coordinate exhaustion then excludes host optimizer
semantics, infinite-horizon optimality, stochastic control, and unlogged
feasibility data.
\end{proof}

\begin{theorem}[Model predictive control finite-horizon prefix restriction]
\label{thm:modelpredictivecontrol-finite-horizon-prefix-restriction}
Let $M=(X,U,N,A,Q,W,J,T,O,F,H,R,P,C)$ be an accepted
$\ModelPredictiveControlUp$ packet, and let $m\leq N$ be a certified unary
prefix horizon. Form the prefix packet $M_{\leq m}$ by replacing the horizon
row $N$ by $m$, retaining the unindexed state, input, transition, quadratic
cost, package, and local name-certificate rows, and retaining exactly the
prediction, constraint-window, stage-cost, terminal, optimizer-candidate,
first-control, $\hsame$, and $\Cont$ rows whose horizon addresses lie in the
prefix horizon $m$. Then $M_{\leq m}$ is an accepted
$\ModelPredictiveControlUp$ packet over horizon $m$.
\end{theorem}
\begin{proof}
Unfold the carrier of \autoref{def:modelpredictivecontrol-bhist-carrier}. Every
coordinate required by a $\ModelPredictiveControlUp$ packet is one of
$$
X,U,N,A,Q,W,J,T,O,F,H,R,P,C.
$$
After prefix restriction, the rows $X$, $U$, $A$, $Q$, $P$, and $C$ are the same
displayed rows, the horizon coordinate is the certified prefix $m$, and every
indexed row in $W$, $J$, $T$, $O$, $H$, and $R$ is a retained row whose address
already occurred under the original horizon $N$.

The prediction ledger is stable under this operation. By
\autoref{thm:modelpredictivecontrol-finite-horizon-obligation}, an accepted MPC
packet exposes its prediction surface exactly through the current-state, input,
unary horizon, dynamics-transition, quadratic-cost, rollout $\Cont$, transport,
and provenance rows. Restricting from $N$ to $m$ keeps precisely those rows at
the prefix addresses and changes no rollout endpoint outside the retained
prefix. Thus the prediction ledger of $M_{\leq m}$ is the finite prediction
ledger over $m$.

The admissibility, cost, and terminal reads are also prefix-local. By
\autoref{thm:modelpredictivecontrol-constraint-ledger}, every admissibility read
comes from the constraint-window row $W$ together with the same horizon index,
state and control rows, rollout routes, convex and polytopal provenance, and
terminal row. Since the prefix packet keeps exactly the rows indexed by the
certified prefix horizon, each retained constraint, stage-cost, and terminal
read has the same source row and the same recorded route as it had in $M$.
No ambient feasible set or external compactness assertion is used to rebuild
acceptance.

For the receding-horizon export, use
\autoref{thm:modelpredictivecontrol-receding-horizon-boundary}. The exported
control is exactly the first-control extraction row $F$ applied to the displayed
optimizer-candidate row $O$. The prefix packet retains that first-control row
and the initial candidate entry, so the receding-horizon boundary remains the
same boundary read inside the shorter packet.

It remains to check transport and provenance. The componentwise $\hsame$ rows
and the $\Cont$ routes retained in $M_{\leq m}$ have the same endpoints after
restriction, because both endpoints are prefix rows. The $\Pkg$ row still names
the same consumed $\StateSpaceModelUp$, $\DynSystemUp$, $\MatrixUp$,
$\VecSpaceUp$, $\RealUp$, $\LQRUp$, $\RiccatiUp$, $\ConvexSetUp$,
$\PolytopeUp$, $\ControlObservabilityUp$, and
$\ControlControllabilityUp$ certificates, and the local $\NameCert$ row still
names the finite receding-horizon surface. Repacking these retained carrier,
prediction, constraint, first-control, transport, continuation, package, and
name-certificate rows gives the accepted $\ModelPredictiveControlUp$ packet over
$m$.
\end{proof}

\begin{theorem}[Model predictive control finite rollout bridge boundary]
\label{thm:modelpredictivecontrol-finite-rollout-bridge-boundary}
For every finite public $\ModelPredictiveControlUp$ packet displayed by the
carrier, rollout, constraint, prefix, package, and $\NameCert$ rows of this
chapter, there is a bridge boundary from that packet to the abstract
receding-horizon control interface. The finite-horizon prefix restriction
supplies the source row, the constraint ledger supplies hidden constraint
provenance, and the consumer boundary transports the exported packet through
$\Cont$ and $\hsame$ without exposing host optimization primitives.
\end{theorem}
\begin{proof}
Begin with the finite source packet. Apply
\autoref{thm:modelpredictivecontrol-finite-horizon-prefix-restriction} to
replace an accepted horizon by any certified prefix while retaining exactly the
displayed current-state, input, rollout, first-control, package, and local
$\NameCert$ rows at prefix addresses.

Next record the hidden feasibility side. By
\autoref{thm:modelpredictivecontrol-constraint-ledger}, every constraint-window,
admissibility, stage-cost, and terminal read used by the prefix packet is
accounted for by the displayed ledger rows and the named convex, polytopal, and
terminal provenance. Thus the bridge source carries constraint provenance as
package data rather than as an ambient host feasible set.

Now export the control boundary. Apply
\autoref{thm:modelpredictivecontrol-receding-horizon-boundary} to identify the
consumer-visible control as the first-control extraction from the displayed
optimizer-candidate row. Then apply
\autoref{thm:modelpredictivecontrol-consumer-boundary} to move this packet
through the $\Cont$ routes and componentwise $\hsame$ rows.

The resulting boundary exposes only the finite MPC public packet: carrier rows,
rollout rows, constraint provenance, prefix horizon, package provenance, and the
local $\NameCert$ row. Since neither cited boundary has a constructor field for
host optimizer semantics, infinite-horizon optimality, stochastic control, or
unlogged feasibility data, the transported packet is a bridge boundary to the
abstract receding-horizon interface and not an exposure of host optimization
primitives.
\end{proof}

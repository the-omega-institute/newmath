\chapter{A Concrete Naming Certificate for $\ModelPredictiveControlUp$}
\label{ch:concrete-instances-modelpredictivecontrol-namecert}

The $\ModelPredictiveControlUp$ interface records the finite receding-horizon
surface used by control packets that repeatedly solve a displayed finite
prediction problem and export only the first control row. The reference point
is Rawlings, Mayne, and Diehl, \emph{Model Predictive Control: Theory,
Computation, and Design} (2017). The BEDC certificate below names the finite
rows of that surface over $\StateSpaceModelUp$, $\DynSystemUp$, $\MatrixUp$,
$\VecSpaceUp$, $\RealUp$, $\LQRUp$, $\ControlObservabilityUp$,
$\ControlControllabilityUp$, $\KalmanFilterUp$, and
$\LyapunovStabilityUp$. It does not assert infinite-horizon optimality,
analytic optimizer existence, compactness, recursive feasibility beyond the
displayed terminal rows, or a host convex optimization solver.

\begin{definition}[MPC finite-horizon carrier]
\label{def:mpc-finite-horizon-carrier}
A $\ModelPredictiveControlUp$ carrier is a finite $\mathsf{BHist}$ packet
$$
M=(X,N,A,W,J,T,O,F,H,R,P,C)
$$
whose visible rows are:
\begin{enumerate}
\item a current-state row $X$ read through $\StateSpaceModelUp$ and
$\VecSpaceUp$;
\item a finite prediction-horizon row $N$ read as a unary length bound;
\item a dynamics-transition row $A$ read through $\DynSystemUp$,
$\StateSpaceModelUp$, $\MatrixUp$, and $\VecSpaceUp$;
\item a constraint-window row $W$ naming the displayed admissible state and
control rows for each horizon index;
\item a stage-cost ledger $J$ read through $\RealUp$, $\MatrixUp$,
$\VecSpaceUp$, and the finite-cost face of $\LQRUp$;
\item a terminal-cost or terminal-region row $T$ read through $\LQRUp$ and
$\LyapunovStabilityUp$ when those rows are present;
\item an optimizer-candidate row $O$ containing only a displayed finite control
sequence candidate and its finite rollout ledger;
\item a first-control extraction row $F$ selecting the initial control entry
from $O$;
\item componentwise $\hsame$ transports $H$ for the state, horizon,
transition, window, cost, terminal, candidate, first-control, and provenance
rows;
\item $\Cont$ rollout routes $R$ for reading the finite prediction recursion,
stage-cost accumulation, terminal read, and first-control extraction;
\item a $\mathsf{Pkg}$ provenance row $P$ naming the consumed state-space,
dynamical, matrix, vector-space, real, LQR, observability, controllability,
Kalman-filter, and Lyapunov-stability certificates;
\item a local $\NameCert$ provenance row $C$ for the finite receding-horizon
surface.
\end{enumerate}
Two accepted packets classify together only when these displayed rows
transport componentwise by $H$ and every rollout, cost, terminal, and
first-control read follows one of the recorded $\Cont$ routes in $R$.
\end{definition}

\begin{theorem}[MPC prediction ledger obligation]
\label{thm:mpc-prediction-ledger-obligation}
Every accepted $\ModelPredictiveControlUp$ packet exposes its prediction
ledger as exactly the finite row family consisting of the current-state row,
the finite horizon row, the displayed dynamics-transition row, the rollout
$\Cont$ routes, and the provenance rows of
\autoref{def:mpc-finite-horizon-carrier}. The prediction surface exports no
host trajectory space, no infinite rollout, and no unlogged model semantics.
\end{theorem}
\begin{proof}
Project the accepted carrier of \autoref{def:mpc-finite-horizon-carrier}. The
only coordinates that can generate prediction rows are $X$, $N$, $A$, $R$,
$P$, and the transport data $H$.

Read the rollout one horizon index at a time. Each step follows a recorded
$\Cont$ route from the displayed state and transition rows to the next
displayed state row, so the ledger is finite and indexed by $N$.

Transporting the packet replaces each displayed coordinate by its $\hsame$
match and reuses the same rollout routes. Since the carrier has no coordinate
for a host trajectory space or an infinite stream, the prediction ledger is
exhausted by these finite rows.
\end{proof}

\begin{theorem}[MPC constraint-window obligation]
\label{thm:mpc-constraint-window-obligation}
For an accepted $\ModelPredictiveControlUp$ packet, every admissibility read
is the constraint-window row $W$ together with the horizon index, state and
control rows, rollout routes, and provenance recorded in
\autoref{def:mpc-finite-horizon-carrier}. The packet exports no ambient
feasible set, no compactness assertion, and no recursive-feasibility theorem
beyond the displayed terminal-cost or terminal-region row.
\end{theorem}
\begin{proof}
Project the constraint coordinate $W$ from the carrier. Its entries are
indexed by the same finite horizon row $N$ used by the prediction ledger, and
they refer only to the displayed state and control rows.

Use the rollout routes to align each constraint entry with the corresponding
predicted state and candidate control row. The admissibility read is therefore
a row projection from $W$, not an external feasible-set predicate.

The terminal coordinate $T$ may record a terminal cost or terminal region row,
but it is another displayed finite row. No coordinate supplies compactness,
global feasibility, or recursive feasibility beyond this displayed terminal
surface.
\end{proof}

\begin{theorem}[MPC receding-horizon control row]
\label{thm:mpc-receding-horizon-control-row}
The control exported by an accepted $\ModelPredictiveControlUp$ packet is
exactly the first-control extraction row $F$ applied to the displayed
optimizer-candidate row $O$. The stage-cost ledger $J$, terminal row $T$,
constraint window $W$, rollout routes $R$, $\hsame$ transports $H$, and
provenance rows $P$ and $C$ certify the finite candidate surface, but they do
not assert that $O$ is a globally optimal host solution.
\end{theorem}
\begin{proof}
Project the candidate row $O$. It contains a displayed finite control sequence
candidate together with the rollout and cost rows that make the candidate
visible inside the packet.

Apply the extraction row $F$. This row selects the initial control entry from
the displayed finite sequence and exports that row as the receding-horizon
control read.

The cost, constraint, terminal, transport, continuation, and provenance rows
certify how the candidate is displayed. None of these rows contains a host
minimizer, a global optimality proof, or an external solver state, so the
exported control row is precisely the first displayed entry of $O$.
\end{proof}

\begin{theorem}[MPC NameCert obligation surface]
\label{thm:mpc-namecert-obligation-surface}
The local $\NameCert_{\ModelPredictiveControlUp}$ obligation surface consists
of five finite rows:
\begin{enumerate}
\item carrier habitation: the finite packet of
\autoref{def:mpc-finite-horizon-carrier};
\item prediction ledger: the finite rollout rows of
\autoref{thm:mpc-prediction-ledger-obligation};
\item constraint-window ledger: the admissibility rows of
\autoref{thm:mpc-constraint-window-obligation};
\item control extraction: the first-control row of
\autoref{thm:mpc-receding-horizon-control-row};
\item provenance boundary: the $\mathsf{Pkg}$ and $\NameCert$ rows naming
$\StateSpaceModelUp$, $\DynSystemUp$, $\MatrixUp$, $\VecSpaceUp$, $\RealUp$,
$\LQRUp$, $\ControlObservabilityUp$, $\ControlControllabilityUp$,
$\KalmanFilterUp$, and $\LyapunovStabilityUp$ as the only imported certificate
surfaces.
\end{enumerate}
These rows form the finite seed obligation surface for naming receding-horizon
model predictive control data in BEDC.
\end{theorem}
\begin{proof}
The carrier row fixes every coordinate of the packet. Its displayed state,
horizon, transition, window, cost, terminal, candidate, extraction, transport,
continuation, package, and local name rows are all finite $\mathsf{BHist}$
data.

The prediction, constraint-window, and control-extraction theorems identify
the only rollout, admissibility, and exported-control reads. Each read is a
projection from the same finite packet and follows one of the recorded
$\Cont$ routes.

The provenance row then lists exactly the consumed control and algebra
certificates. Since no extra coordinate carries host optimization, analytic
minimizers, compactness, infinite horizons, or unlogged feasibility data, the
five rows exhaust the local $\NameCert_{\ModelPredictiveControlUp}$ surface.
\end{proof}

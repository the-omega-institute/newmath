\chapter{A Concrete Naming Certificate for $\PermutationUp$}
\label{ch:concrete-instances-permutation-namecert}

The permutation interface $\PermutationUp$ packages the carrier of bijections of a $\NameCert_{\FinSetUp}$ (\autoref{ch:concrete-instances-finset-namecert}) onto itself. It supplies the underlying carrier for $\NameCert_{\SymGroupUp}$ (\autoref{ch:concrete-instances-symgroup-namecert}) and the action data for combinatorial group-theoretic certificates.

\section{Carrier definition}
\label{sec:permutation-carrier}

\begin{definition}[Permutation bijection source row]
\label{def:permutation-bijection-source-row}
Fix a finite-set source certificate $\NameCert_{\FinSetUp}$ with carrier row $F$, classifier $\sim_F$, finite-spine endpoint row $xs$, and enumeration ledger as in \autoref{ch:concrete-instances-finset-namecert}. A BEDC-facing $\PermutationUp$ source row over $F$ is the $\mathsf{BHist}$-visible package
$$
(\mathsf{src},\mathsf{tgt},G,G^{-1},C,A,L)
$$
with the following rows.
\begin{enumerate}
\item $\mathsf{src}$ and $\mathsf{tgt}$ are the two displayed endpoint rows for the source and target copies of the same carried $\FinSetUp$ object.
\item $G$ is a bijection graph/probe row: for every source endpoint read by the $\mathsf{ProbeBundle}$, the graph supplies exactly one target endpoint, and both endpoints are classified by the fixed $\FinSetUp$ carrier.
\item $G^{-1}$ is the inverse graph row with the same readback discipline in the opposite direction.
\item $C$ is the composition row witnessing that $G^{-1}\circ G$ and $G\circ G^{-1}$ transport to the corresponding identity graph rows by $\hsame$ and by the package provenance attached to the two endpoint rows.
\item $A$ is the action row sending a carried source endpoint through $G$ to its carried target endpoint.
\item $L$ is the ledger containing the hidden enumeration witness for the chosen finite spine, the hidden order witness used to name positions, the $\Pkg$ provenance for the two endpoint copies, and the probe-read entries that justify graph coverage.
\end{enumerate}
The only visible carrier fields are the two $\FinSetUp$ endpoint rows, the graph, inverse graph, composition row, action row, and ledger boundary. Enumeration and order witnesses are not exposed as host data; they are consumed through $L$.
\end{definition}

\section{The certificate}
\label{sec:permutation-certificate}

\begin{theorem}[Permutation carrier obligation]
\label{thm:permutation-carrier-obligation}
A $\PermutationUp$ endpoint over the finite-set source row of \autoref{def:permutation-bijection-source-row} is exactly a $\mathsf{BHist}$ row exposing the $\FinSetUp$ carrier, the bijection graph $G$, the inverse graph $G^{-1}$, the action row $A$, the $\Pkg$ provenance for the source and target copies, and the $\mathsf{ProbeBundle}$ reads used to certify graph coverage. No endpoint may expose a host enumeration, a host order, or a bare host function outside these rows.
\end{theorem}
\leanchecked{BEDC.Derived.PermutationUp.PermutationBijectionSourceRow\_carrier\_surface}
\begin{proof}
Unpack the source row. The two endpoint copies are carried by the displayed $\FinSetUp$ rows, and every graph edge is read through the $\mathsf{ProbeBundle}$ against those carriers.

The forward reading of a $\PermutationUp$ endpoint therefore projects the finite-set carrier, the graph, the inverse graph, the action row, and the package provenance recorded in the ledger. The hidden enumeration and order witnesses occur only as ledger entries, so projecting an endpoint cannot reveal them as carrier fields.

Conversely, the listed rows repack into a $\mathsf{BHist}$ endpoint because the graph and inverse graph provide the probe-read coverage in both directions, while the action row gives the endpoint-level readback of the graph. Thus the displayed inventory is exact.
\end{proof}

\begin{theorem}[Permutation classifier, composition, and inverse obligation]
\label{thm:permutation-classifier-composition-inverse-obligation}
Classifier transport for $\PermutationUp$ is componentwise $\FinSetUp$ transport of the source and target endpoint rows together with $\hsame$ and $\Pkg$ transport of the graph row, inverse graph row, and composition row. The obligation is to prove that these transports preserve the two identity-composition witnesses
$$
G^{-1}\circ G \sim_{\mathsf{Graph}} \mathsf{id}_{\mathsf{src}}
$$
and
$$
G\circ G^{-1} \sim_{\mathsf{Graph}} \mathsf{id}_{\mathsf{tgt}},
$$
without asserting equality of host functions or equality of hidden enumerations.
\end{theorem}
\begin{proof}
Transport starts with the two finite-set endpoint rows. Applying the $\FinSetUp$ classifier moves source endpoints to source endpoints and target endpoints to target endpoints while preserving the carried finite-set membership relation.

The graph, inverse graph, and composition rows are then transported by the $\hsame$ evidence recorded in the package ledger. This keeps the graph-read and inverse-read rows attached to the same $\Pkg$ provenance rather than replacing them by a host-level function comparison.

It remains to read the two composition witnesses after transport. Since composition is itself a displayed row in \autoref{def:permutation-bijection-source-row}, the transported graph followed by the transported inverse graph is classified by the transported identity row on the corresponding finite-set endpoint copy. The same argument in the opposite direction gives the target identity row.
\end{proof}

\begin{theorem}[Permutation action and ledger exactness obligation]
\label{thm:permutation-action-ledger-exactness-obligation}
The action row $A$ and the ledger $L$ of \autoref{def:permutation-bijection-source-row} supply the exact downstream surface consumed by $\SymGroupUp$: coverage says every carried source endpoint has a graph image and every carried target endpoint has an inverse-graph preimage; soundness says every action output is carried by the target $\FinSetUp$ endpoint row and classified by the graph read; completeness says any graph read with the same source endpoint is classified by the action output. These three clauses, together with the composition and inverse rows, are the full $\PermutationUp$ ledger boundary available to $\SymGroupUp$.
\end{theorem}
\begin{proof}
Coverage is the graph and inverse-graph exactness recorded in the ledger: each source probe read produces a target graph read, and each target probe read produces an inverse-graph read. The hidden enumeration and order witnesses justify the finite search but remain inside $L$.

Soundness follows by projecting the action row. When $A$ sends a source endpoint through $G$, the graph row supplies the same target endpoint as a carried $\FinSetUp$ target row, and the classifier witness is read from the probe entry.

Completeness uses the uniqueness field of the graph/probe row. Any second graph read with the same carried source endpoint is classified by the target endpoint returned by $A$. Pairing this with the two identity-composition witnesses gives exactly the permutation face needed by $\SymGroupUp$, with no additional carrier, classifier, or host-function field.
\end{proof}

It sits downstream of the regular extraction surface of \autoref{ch:concrete-instances-regular-cauchy-diagonal-extraction-namecert}, uses the tail-basis route of \autoref{ch:concrete-instances-cauchy-tail-basis-namecert}, reuses the diagonal-limit seal of \autoref{ch:concrete-instances-cauchy-diagonal-limit-namecert}, and keeps the endpoint comparison boundary aligned with \autoref{ch:concrete-instances-cauchydifferencecriterion-namecert}.

It records rows, budgets, ledgers, and local naming for constructive diagonal passage through regular Cauchy tails while excluding quotient stream equality, host selected limits, countable choice, trichotomy, ambient $\RealUp$ completeness, Lean verification, and bridge credentials.

\begin{definition}[Cauchy diagonal argument carrier]
\label{def:cauchy-diagonal-argument-carrier}
A $\CauchyDiagonalArgumentUp$ carrier is a finite $\BHist$ packet
$$
A=(R,S,T,B,L,W,Q,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $R$ is the regular-Cauchy row family, listing the finitely inspected $\RegSeqRatUp$ source rows and their admissible tail windows.
\item $S$ is the diagonal index-selection row, recording the selected index for each visible precision request without importing a countable-choice schedule.
\item $T$ is the cofinal tail-budget row, displaying the tail threshold that simultaneously supports the selected row, the diagonal window, and every downstream readback.
\item $B$ is the $\CauchyTailBasisUp$ basis row used to compare the selected tail with sibling tail requests.
\item $L$ is the $\DyadicRatCoreUp$ error ledger, storing dyadic radius, slack, and comparison entries for the selected row and cofinal tail.
\item $W$ is the finite $\StreamNameUp$ window row read from the selected tail.
\item $Q$ is the $\RegSeqRatUp$ diagonal readback row assembled from $R,S,T,L,W$.
\item $E$ is the terminal $\RealUp$ seal row, visible only after the row family, selected index, cofinal tail budget, basis comparison, dyadic ledger, window, and readback rows have been displayed.
\item $H$ records componentwise $\hsame$ transport for the row family, index selection, tail budget, basis row, dyadic ledger, finite window, diagonal readback, Real seal, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from each precision request through $R,S,T,B,L,W,Q,E,H,P,N$.
\item $P$ is the $\Pkg$ provenance over $\CauchyDiagonalArgumentUp$, $\RegularCauchyDiagonalExtractionUp$, $\CauchyTailBasisUp$, $\CauchyDiagonalLimitUp$, $\CauchyDifferenceCriterionUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CauchyDiagonalArgumentUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $R,S,T,B,L,W,Q,E,H,C,P,N$. It contains no quotient stream equality, detached limit selector, countable-choice row, trichotomy row, ambient $\RealUp$ completeness datum, Lean credential, or standard bridge.
\end{definition}

\begin{theorem}[Cauchy diagonal argument tail selection]
\label{thm:cauchy-diagonal-argument-tail-selection}
Let $A=(R,S,T,B,L,W,Q,E,H,C,P,N)$ be an accepted $\CauchyDiagonalArgumentUp$ carrier. Every Real-facing precision request that enters the packet factors through one displayed diagonal index in $S$ and one cofinal tail budget in $T$ before it can read $W$, $Q$, or $E$. The selected route is
$$
R,S,T,B,L \longmapsto W \longmapsto Q \longmapsto E .
$$
Thus the diagonal selection is a finite NameCert route over regular Cauchy tails, not a host-selected limit, quotient stream, countable-choice selector, or ambient completeness witness.
\end{theorem}
\begin{proof}
Begin with an accepted carrier and a Real-facing precision request. Carrier admission provides the row family $R$ as the only source of regular Cauchy data, so the request cannot inspect a stream window or Real seal before a source row has been displayed.

The diagonal index is then read from $S$. This row stores the visible selected index for the request as packet data, and it has no coordinate for an external choice function or detached limit selector.

The cofinal tail budget $T$ is read next. It is the row that binds the selected index to the tail threshold used by the basis row $B$ and the dyadic ledger $L$, so every downstream window remains tied to the same finite request.

Finally $W,Q,E$ are downstream of the selected budget. The structural rows $H,C,P,N$ transport, replay, source, and name the same finite route, so the consumer receives the displayed diagonal passage and no quotient stream, host limit, countable-choice row, trichotomy principle, or ambient completeness datum.
\end{proof}

\begin{theorem}[Cauchy diagonal argument NameCert obligations]
\label{thm:cauchy-diagonal-argument-namecert-obligations}
For every accepted $\CauchyDiagonalArgumentUp$ carrier $A=(R,S,T,B,L,W,Q,E,H,C,P,N)$, the local $\NameCert_{\CauchyDiagonalArgumentUp}$ surface consists exactly of the following five finite obligation rows:
\begin{enumerate}
\item carrier admission for the regular-Cauchy row family, diagonal index-selection row, cofinal tail-budget row, CauchyTailBasis row, dyadic error ledger, finite StreamName window, RegSeqRat diagonal readback, terminal Real seal, transport, replay, provenance, and local naming rows;
\item index-selection exactness, requiring $S$ to be the only row that supplies the displayed diagonal index for a visible precision request;
\item cofinal tail-budget exactness, requiring $T$ and $B$ to bind that selected index to a common tail before any readback is exposed;
\item dyadic error-ledger exactness, requiring $L$ to store precisely the radius, slack, and comparison entries consumed by $W$ and $Q$;
\item Real seal non-escape, requiring $E,H,C,P,N$ to export only the displayed diagonal route and no quotient stream equality, detached limit selector, countable choice, trichotomy, ambient $\RealUp$ completeness, Lean verification, or bridge credential.
\end{enumerate}
No Cauchy diagonal argument obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cauchy-diagonal-argument-carrier}. The accepted packet displays exactly the row family, index-selection row, cofinal tail budget, basis comparison, dyadic ledger, finite window, diagonal readback, Real seal, and structural rows.

Index-selection exactness follows by inspecting $S$. Since no other coordinate supplies a selected diagonal index, every visible precision request uses the displayed index row.

Cofinal tail-budget exactness follows by inspecting $T$ and $B$. The tail threshold is recorded before $W,Q$, or $E$ is visible, and the basis row compares it with sibling tail requests through the same finite carrier.

The dyadic ledger clause is the projection to $L$. The ledger stores the radius, slack, and comparison entries used by the finite window $W$ and the regular rational readback $Q$, so the error budget cannot be replaced by a detached representative.

Finally inspect $E,H,C,P,N$. These rows seal, transport, replay, source, and name the displayed route only; because the carrier has no quotient stream, selected host limit, countable-choice row, trichotomy principle, ambient completeness datum, Lean credential, or bridge credential, the listed five rows exhaust the local NameCert surface.
\end{proof}

The $\CandidateSetNormalizationUp$ packet realizes the candidate-set reading of \autoref{ch:metacic-normalization-audit-map} as finite NameCert data. It records candidate evidence, closed-term projection, Pi and application adequacy, no-infinite-reduction, obstruction, transport, continuation, provenance, and local naming rows. The packet is deliberately a candidate-set surface: it does not claim unconditional strong normalization for every closed typed MetaCIC term.

\begin{definition}[MetaCIC candidate-set normalization carrier]
\label{def:metacic-candidate-set-normalization-carrier}
A $\CandidateSetNormalizationUp$ carrier is a finite $\BHist$ packet
$$
S=(K,T,A,I,O,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $K$ is the finite candidate-set evidence row, including candidate membership, soundness, and beta-closure evidence for the accepted read.
\item $T$ is the closed typed term projection row, recording the empty-context typed term whose normalization read is supplied by $K$.
\item $A$ is the application and Pi adequacy row, tying application reads and binder reads back to the same candidate-set evidence.
\item $I$ is the no-infinite-reduction row obtained from the candidate-set strong-normalization route.
\item $O$ is the obstruction row recording that a closed-term normalization read is refused when $K$ is absent.
\item $H$ records componentwise $\hsame$ transport for $K,T,A,I,O$ and the local naming row.
\item $C$ records displayed $\Cont$ replay from candidate-set evidence to strong normalization, closed-term projection, application and Pi adequacy, and no-infinite-reduction reads.
\item $P$ is the $\Pkg$ provenance over $\CandidateSetNormalizationUp$, $\MetaCICRedexFrontierUp$, $\MetaCICNormalizationFrontierUp$, $\BoundedNormalEqualityCheckerUp$, $\MetaCICOpenProblemLedgerUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.
\item $N$ is the local $\NameCert_{\CandidateSetNormalizationUp}$ row.
\end{enumerate}
The classifier compares accepted carriers only through $K,T,A,I,O,H,C,P,N$. It contains no coordinate for evidence-free closed-term normalization, componentwise application normalization, host reduction normalization, quotient reduction equality, subject-reduction discharge, or a global normalization oracle.
\end{definition}

\begin{theorem}[MetaCIC candidate-set normalization obligations]
\label{thm:metacic-candidate-set-normalization-obligations}
For every accepted $\CandidateSetNormalizationUp$ carrier $S=(K,T,A,I,O,H,C,P,N)$, the local $\NameCert_{\CandidateSetNormalizationUp}$ surface consists exactly of finite candidate-set evidence through $K$; closed-term projection through $T$; application and Pi adequacy through $A$; no-infinite-reduction through $I$; obstruction when the candidate-set row is absent through $O$; componentwise transport through $H$; displayed replay through $C$; provenance through $P$; local naming through $N$; and non-escape from unconditional MetaCIC strong normalization.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:metacic-candidate-set-normalization-carrier}. The tuple displays the candidate-set evidence, closed typed term projection, adequacy row, no-infinite-reduction row, obstruction row, transport, replay, provenance, and local name.

The positive route begins at $K$. Candidate-set membership, soundness, and beta-closure are read before $T$, $A$, or $I$ can export a normalization-facing conclusion.

The closed-term projection is $T$, and the application and Pi adequacy route is $A$. Both rows consume the same candidate-set evidence rather than a componentwise strong-normalization shortcut.

The no-infinite-reduction row $I$ is downstream of that evidence. If $K$ is absent, $O$ records the obstruction, so the packet refuses an evidence-free theorem over all closed typed terms.

Finally $H,C,P,N$ transport, replay, source, and name the same finite branch. Since no global normalization oracle or quotient reduction equality is present in the carrier, the listed rows exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[MetaCIC candidate-set normalization idempotence]
\label{thm:metacic-candidate-set-normalization-idempotence}
Let $S=(K,T,A,I,O,H,C,P,N)$ be an accepted $\CandidateSetNormalizationUp$ carrier whose candidate-set packet has already been normalized through the displayed frontier-dependency rows. Applying the same normalization route again exposes the same accepted candidate rows, the same closed-term projection, the same application and Pi adequacy rows, and no additional spectral-trigger row.
\end{theorem}
\begin{proof}
Use the carrier and obligation surface of \autoref{def:metacic-candidate-set-normalization-carrier} and \autoref{thm:metacic-candidate-set-normalization-obligations}. The first normalization pass can read only $K,T,A,I,O,H,C,P,N$, and the accepted candidate-set evidence is already attached to the displayed no-infinite-reduction and obstruction rows.

The frontier-dependency rows of \autoref{ch:concrete-instances-metacic-frontier-dependency-matrix-namecert} determine which candidate rows may be exposed before the closed-term projection and adequacy rows are replayed. Running the same route again therefore reuses the same dependency rows rather than introducing a new candidate branch.

Finally compare with the spectral-trigger surface of \autoref{ch:concrete-instances-wobble-quotient-spectral-trigger-namecert}. A trigger row would have to be present as an additional obstruction or replay coordinate, but the $\CandidateSetNormalizationUp$ classifier contains no such coordinate beyond $O$ and $C$. The second pass is therefore idempotent on accepted candidate rows and creates no further spectral-trigger row.
\end{proof}

\chapter{A Concrete Naming Certificate for \(\TotalOrderUp\)}
\label{ch:concrete-instances-totalorder-namecert}

The total order interface \(\TotalOrderUp\) extends \autoref{ch:concrete-instances-poset-namecert} with the trichotomy field, expressed constructively: for any \(a, b\) in the carrier, the carrier's apartness predicate decides \(a < b\), \(a = b\) (under the classifier), or \(a > b\). The classical formulation (LEM-style "either \(a \le b\) or \(b \le a\)") is replaced by decidable comparison so that the total-order certificate remains 0-axiom.

\section{Carrier definition}

\begin{definition}[TotalOrder carrier]
\label{def:totalorder-carrier}
Use a poset setup extended with constructive comparison data returning less, same, or greater.
\end{definition}

\section{Concrete singleton history total order instance}
\label{sec:totalorder-concrete-singleton-history-instance}

\begin{definition}[Concrete singleton history total order instance]
\label{def:concrete-singleton-history-totalorder-instance}
Let $A_{\emp}$ be the singleton history source of
Definition~\ref{def:list-singleton-history-source} at $a_0=\emp$. Define
$$
\rho_{\emp}(h,k):=
A_{\emp}(h)\land A_{\emp}(k)\land\hsame(h,k),
$$
and let the order relation on carried endpoints be
$$
h\le_{\emp}k:=\mathsf{PreorderPrefixLE}(h,k).
$$
The constructive comparison package has three branches:
$$
\mathsf{lt}_{\emp}(h,k):=\bot,\qquad
\mathsf{same}_{\emp}(h,k):=\rho_{\emp}(h,k),\qquad
\mathsf{gt}_{\emp}(h,k):=\bot.
$$
Thus the less and greater branches are empty, while the same branch stores the
singleton classifier witness.
\end{definition}

\begin{theorem}[Singleton history total order laws]
\label{thm:singleton-history-totalorder-laws}
For carried histories $h,k:\mathsf{BHist}$ with $A_{\emp}(h)$ and
$A_{\emp}(k)$, the comparison package of
\autoref{def:concrete-singleton-history-totalorder-instance} always returns
the same branch carrying $\rho_{\emp}(h,k)$. The resulting tuple satisfies the
$\TotalOrderUp$ carrier field, the $\POSetUp$ fields, classifier stability,
strict-to-apartness, trichotomy, and comparison stability.
\end{theorem}
\begin{proof}
The carrier data are inherited from the singleton history source. The
hypotheses $A_{\emp}(h)$ and $A_{\emp}(k)$ unfold to $\hsame(h,\emp)$ and
$\hsame(k,\emp)$; symmetry and transitivity of $\hsame$ compose these endpoint
witnesses into $\hsame(h,k)$. Together with the two carrier facts, this is
exactly $\rho_{\emp}(h,k)$.

The preorder comparisons in both directions are supplied by
$\mathsf{PreorderPrefixLE\_of\_hsame}$ from the displayed
$\hsame(h,k)$ witness and its symmetric form. These witnesses give the
$\POSetUp$ order relation on the restricted singleton carrier.

Antisymmetry is the concrete prefix collapse:
$\mathsf{PreorderPrefixLE\_antisymm\_hsame}$ sends opposing prefix comparisons
to $\hsame(h,k)$, which repackages with the carrier facts as the classifier
$\rho_{\emp}(h,k)$. Classifier stability and comparison stability use the same
endpoint transport through $\emp$.

The strict branches are false by definition, so strict-to-apartness is vacuous.
Trichotomy is the same branch just constructed; no classical comparison
principle is used.
\end{proof}

The $\TotalOrderUp$ schema is represented by this concrete $\mathsf{BHist}$
carrier and explicit comparison data, giving later Lean formalization a
$\mathsf{BHist}$-anchored target rather than an abstract parameter schema.

\section{The certificate}

\begin{definition}[TotalOrder source specification]
\label{def:totalorder-source-specification}
The source is a certified poset carrier plus apartness-aware trichotomy.
\end{definition}

\begin{definition}[TotalOrder pattern specification]
\label{def:totalorder-pattern-specification}
The pattern is comparison formation producing one of three certified comparison cases.
\end{definition}

\begin{definition}[TotalOrder classifier specification]
\label{def:totalorder-classifier-specification}
The classifier is the poset carrier classifier, with apartness stable under classifier replacement.
\end{definition}

\begin{definition}[TotalOrder stability certificate]
\label{def:totalorder-stability-certificate}
The stability certificate consists of:
\begin{enumerate}
  \item all poset stability fields
  \item strict-order-to-apartness field
  \item trichotomy as explicit comparison data
  \item comparison stability under classifier replacement
  \item transitivity of strict comparison where supplied
\end{enumerate}
\end{definition}

\begin{definition}[TotalOrder ledger policy]
\label{def:totalorder-ledger-policy}
Record the comparison certificate selected for each pair.
\end{definition}

\begin{definition}[\(\TotalOrderUp\) certificate obligations]
\label{def:totalorder-certificate-obligations}
The carrier, source, pattern, classifier, stability, and ledger data listed above are the obligations required for the interface name \(\TotalOrderUp\).
\end{definition}

\begin{theorem}[Concrete singleton history $\TotalOrderUp$ naming certificate]
\label{thm:concrete-singleton-history-totalorder-namecert}
The concrete tuple of
\autoref{def:concrete-singleton-history-totalorder-instance} supplies the
source, pattern, classifier, stability, ledger, and exactness rows for a
$\NameCert$ licensing $\TotalOrderUp$ over the singleton carrier $A_{\emp}$,
classifier $\rho_{\emp}$, order $\le_{\emp}$, and comparison branches
$\mathsf{lt}_{\emp}:=\bot$, $\mathsf{same}_{\emp}:=\rho_{\emp}$, and
$\mathsf{gt}_{\emp}:=\bot$.
\end{theorem}
\begin{proof}
The carrier row is the singleton $\mathsf{BHist}$ source. Apply
\autoref{thm:singleton-history-totalorder-laws} to obtain the carried endpoint
facts, classifier stability, the $\POSetUp$ fields, trichotomy, and comparison
stability for the displayed branches.

The exactness row is the underlying partial-order exactness supplied by
\autoref{thm:singleton-empty-poset-laws}: on carried histories,
$\rho_{\emp}(h,k)$ is equivalent to the two opposing prefix comparisons
$h\le_{\emp}k$ and $k\le_{\emp}h$.

The strict rows contribute no extra witnesses. Both strict comparison branches
are empty by definition, so strict-to-apartness and strict transitivity are
vacuous.

It remains to assemble the public certificate. Repack the source as
$A_{\emp}$, the pattern as the three-way comparison formation, the classifier
as $\rho_{\emp}$, the stability data as the fields just cited, and the ledger
as the selected same-branch comparison certificate.
\end{proof}

The per-object template is: traditional target singleton constructive total
order; BEDC source singleton $\mathsf{BHist}$; carrier $A_{\emp}$; classifier
$\rho_{\emp}$; stability obligations from
\autoref{thm:concrete-singleton-history-totalorder-namecert}; ledger the
selected same-branch comparison certificate; exactness singleton classifier
iff opposing prefix comparisons; standard bridge singleton total order; status
$\mathsf{ProofTarget}$.

\paragraph{Primitive boundary.}
\(\TotalOrderUp\) is not a BEDC primitive. It is admitted only when those obligations are supplied.

\begin{remark}[Zero-axiom encoding note]
\label{rem:totalorder-zero-axiom-note}
Replaces LEM-style totality with decidable comparison data and apartness.
\end{remark}

\chapter{A Concrete Naming Certificate for $\NormedSpaceUp$}

Following Bourbaki and Bishop--Bridges constructive analysis, the $\NormedSpaceUp$ packet records the finite NameCert surface for a non-singleton normed vector space. It consumes the displayed vector-space source of \autoref{ch:concrete-instances-vecspace-namecert}, the real-valued norm interface of \autoref{ch:concrete-instances-norm-namecert}, the metric readback discipline of \autoref{ch:concrete-instances-metric-namecert}, and the completion-facing boundary of \autoref{ch:concrete-instances-completion-namecert}. The packet is an interface for later Banach, Hilbert, bounded-operator, and functional-analysis certificates; it is not the singleton $\NormUp$ instance, not a host normed-space structure, not quotient completion, and not a choice principle.

\begin{definition}[NormedSpace carrier]
\label{def:normed-space-carrier}
A $\NormedSpaceUp$ carrier is a finite $\BHist$ packet
$$
S=(V,R,N,M,Q,H,T,P,C)
$$
with the following displayed rows:
\begin{enumerate}
\item $V$ is the $\VecSpaceUp$ row, exposing the vector carrier, vector classifier, zero vector, addition, negation, and scalar action only through displayed BEDC data.
\item $R$ is the $\RealUp$ row used by all norm values, scalar absolute values, metric distances, and completion tolerances.
\item $N$ is the real-valued norm row, carrying a map from admitted vector histories to $R$-sealed real histories together with nonnegativity, zero exactness, triangle inequality, and scalar compatibility evidence.
\item $M$ is the induced $\MetricUp$ row, whose distance read between $x$ and $y$ is the norm read of the displayed difference $x-y$ through $V,N,R$.
\item $Q$ is the completion-facing row, recording Cauchy windows, tolerance requests, and handoff data for completion consumers without selecting a completed point.
\item $H$ records componentwise $\hsame$ transport for $V,R,N,M,Q$ and for the replay, provenance, and local naming rows.
\item $T$ records displayed $\Cont$ replay from every norm, metric, or completion request through the same rows $V,R,N,M,Q,H,P,C$.
\item $P$ is the $\Pkg$ provenance over $\NormedSpaceUp$, $\VecSpaceUp$, $\NormUp$, $\RealUp$, $\MetricUp$, completion-facing Cauchy data, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $C$ is the local $\NameCert_{\NormedSpaceUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through these nine rows. It has no coordinate for host normed-space equality, quotient completion, selected limits, basis choice, classical duality, or an ambient Banach or Hilbert structure.
\end{definition}

\begin{theorem}[NormedSpace NameCert obligations]
\label{thm:normed-space-namecert-obligations}
For every accepted $\NormedSpaceUp$ carrier
$$
S=(V,R,N,M,Q,H,T,P,C),
$$
the local $\NameCert_{\NormedSpaceUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the displayed vector-space, Real, norm, induced metric, completion-facing, transport, replay, provenance, and local naming rows;
\item classifier transport, requiring equality of packet reads to be checked componentwise through $V,R,N,M,Q,H,T,P,C$ and never through host extensional equality;
\item zero exactness, requiring the norm-zero classifier in $N$ to coincide with the displayed vector-zero classifier in $V$;
\item triangle inequality, requiring every norm read of a displayed sum to factor through the $R$-sealed addition and order rows already attached to $N$;
\item scalar compatibility, requiring each scalar multiplication read to pass through the scalar row of $V$, the real absolute-value row of $R$, and the norm row $N$;
\item metric handoff, requiring $M$ to read distances only as norm reads of displayed vector differences through $V,N,R$;
\item completion-facing nonescape, requiring $Q,H,T,P,C$ to preserve the same displayed route for Cauchy and completion consumers without adding quotient completions, selected limits, basis choice, choice, or host Banach/Hilbert structure.
\end{enumerate}
No $\NormedSpaceUp$ obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:normed-space-carrier}. The accepted packet lists the vector-space, Real, norm, induced metric, completion-facing, transport, replay, provenance, and local naming rows.

Classifier transport is then componentwise. A comparison of two accepted packets can inspect only the rows displayed in the carrier, so vector data are compared through $V$, real and norm data through $R,N$, metric data through $M$, completion-facing data through $Q$, and structural replay through $H,T,P,C$.

The zero, triangle, and scalar rows are the norm obligations. Zero exactness identifies a zero norm read precisely with the displayed vector-zero classifier of $V$. The triangle row reads the norm of a sum through the real addition and order data in $R$, and scalar compatibility reads scalar multiplication through the scalar row of $V$ together with the absolute-value row of $R$.

The metric row is forced by construction: $M$ has no independent distance source. Its distance read from $x$ to $y$ is the norm read of the displayed difference $x-y$, so every metric consumer must pass through $V,N,R$ before it reaches $M$.

Finally, the completion-facing row $Q$ and the structural rows $H,T,P,C$ replay the same displayed route for Cauchy windows and completion requests. Since the carrier has no quotient-completion, selected-limit, basis-choice, or host Banach/Hilbert coordinate, the listed obligations exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[NormedSpace zero exactness]
\label{thm:normed-space-zero-exactness}
Let $S=(V,R,N,M,Q,H,T,P,C)$ be an accepted $\NormedSpaceUp$ packet. The zero classifier exposed by the norm row $N$ is exact precisely along the displayed vector-zero row of $V$: a norm read may certify zero only by reading the vector zero classifier in $V$, the real zero and order rows in $R$, and the local transport and replay rows $H,T,C$. It cannot certify zero by host vector equality, quotient equality, basis coordinates, or an ambient Banach-space identity.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:normed-space-carrier}. The only vector zero datum is in $V$, and the only real zero and order data available to a norm read are in $R$.

Apply the zero-exactness clause of \autoref{thm:normed-space-namecert-obligations}. It identifies a zero norm read with the displayed vector-zero classifier rather than with a host equality relation.

The structural rows $H,T,C$ preserve this route under transport and replay, while $P$ and the local naming row keep it inside the same packet. Since no quotient equality, basis coordinate, or ambient Banach identity is a carrier row, the zero read is exact along $V,R,N$.
\end{proof}

\begin{theorem}[NormedSpace triangle and scalar compatibility]
\label{thm:normed-space-triangle-scalar-compatibility}
For every accepted $\NormedSpaceUp$ packet $S=(V,R,N,M,Q,H,T,P,C)$, the triangle and scalar-compatibility obligations are routed through the same displayed rows. A norm read of a sum must first expose vector addition in $V$, then the norm row $N$, and then the real addition and order rows in $R$. A norm read of a scalar multiple must first expose scalar action in $V$, then the scalar absolute-value read in $R$, and then the norm row $N$. Neither read may use host linearity, an external ordered field, basis expansion, quotient vectors, or a hidden continuity principle.
\end{theorem}
\begin{proof}
Start with the triangle request. The sum is not a host vector expression; it is the displayed addition operation in $V$. Once the sum row is read, the norm row $N$ supplies the norm observation and the Real row $R$ supplies the addition and order comparison.

Now consider scalar compatibility. The scalar action is again a displayed operation in $V$, while the absolute-value and real multiplication reads come from $R$. The norm row $N$ relates those two displayed sources.

Finally transport and replay the two routes through $H,T,C$ with provenance in $P$. Because the carrier contains no host linearity, external ordered field, basis expansion, quotient-vector coordinate, or hidden continuity row, the triangle and scalar obligations stay inside $V,R,N$.
\end{proof}

\begin{theorem}[NormedSpace metric nonescape]
\label{thm:normed-space-metric-nonescape}
For every accepted $\NormedSpaceUp$ packet, the metric row $M$ can read a distance between displayed vectors only as the norm read of their displayed difference through $V,N,R$. The metric consumer may then pass through $Q,H,T,P,C$ for completion-facing handoff, transport, replay, provenance, and local naming, but it cannot export an independent host metric, quotient completion, selected limit, or Banach/Hilbert structure.
\end{theorem}
\begin{proof}
Read $M$ from \autoref{def:normed-space-carrier}. Its distance data are induced data, and the carrier names no distance source except the norm row $N$ applied to a displayed vector difference from $V$.

Apply the metric handoff clause of \autoref{thm:normed-space-namecert-obligations}. It forces each distance request through $V,N,R$ before $M$ can be read by a downstream consumer.

The completion-facing row $Q$ and the structural rows $H,T,P,C$ preserve the same displayed route. Since no independent host metric, quotient completion, selected limit, or Banach/Hilbert coordinate is present, metric reads cannot escape the induced norm route.
\end{proof}

\label{thm:normed-space-completion-handoff}

$$

\begin{proof}
Start with a downstream consumer request. A vector operation can be answered only by projecting the displayed vector-space row $V$, while a norm or scalar-size request must read $R$ and $N$ before any real-valued comparison is available.

Metric requests are not separate host data. By \autoref{thm:normed-space-namecert-obligations}, the metric row $M$ reads distance as the norm of a displayed vector difference, so the route necessarily passes through $V,N,R$ before a metric comparison is exported.

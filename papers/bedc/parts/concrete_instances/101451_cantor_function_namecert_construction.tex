\begin{proof}

\begin{theorem}[Cantor function prefix monotonicity]
\label{thm:cantor-function-prefix-monotonicity}

\end{theorem}
\begin{proof}
The retained-prefix relation is a finite comparison inside $T$. Since the carrier admits source prefixes only through $S$ and the $\CantorSetUp$ classifier, both $t$ and $t'$ remain displayed retained-branch rows.

The output prefix attached to a retained row is read through the continuation route from $T$ to $B$. Refining the source prefix therefore exposes a longer finite binary ledger entry rather than a different output object.

The graph row $G$ only pairs the displayed source and output prefixes. It contains no coordinate for an ambient function, measure computation, singularity assertion, quotient real comparison, or branch selection, so the monotonic statement is exhausted by the two finite ledgers and their transport rows.

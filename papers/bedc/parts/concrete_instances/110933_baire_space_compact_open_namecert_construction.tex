\chapter{A Concrete Naming Certificate for $\BaireSpaceCompactOpenUp$}
\label{ch:concrete-instances-baire-space-compact-open-namecert}
\origin{human}

Following Kelley and Bishop--Bridges, the $\BaireSpaceCompactOpenUp$ packet records the compact-open reading of Baire-space maps as finite BEDC NameCert data. It sits after the Baire prefix carrier of \autoref{ch:concrete-instances-bairespace-namecert}, the compact-open topology surface of \autoref{ch:concrete-instances-compact-open-topology-namecert}, the continuous-map graph rows of \autoref{ch:concrete-instances-continuousmap-namecert}, and the L10 real-name route through \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, and \autoref{ch:concrete-instances-real-namecert}. The packet names only finite prefix cylinders, compact-window tests, graph readbacks, regular rational handoffs, and real seals; it does not import a host function space.

\begin{definition}[Baire-space compact-open carrier]
\label{def:baire-space-compact-open-namecert-carrier}
A $\BaireSpaceCompactOpenUp$ carrier is a finite $\BHist$ packet
$$
K=(B,O,F,W,R,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $B$ is the $\BaireSpaceUp$ prefix-cylinder row, exposing only finite natural-index windows.
\item $O$ is the $\CompactOpenTopologyUp$ subbasic-window row, recording the compact-open request as a finite list of prefix-cylinder probes and target-radius budgets.
\item $F$ is the $\ContinuousMapUp$ graph row whose source inspections all pass through $B$ and whose target inspections pass through the L10 real-name route.
\item $W$ is the $\StreamNameUp$ observation schedule used by the compact-open request and by every target readback.
\item $R$ is the $\RegSeqRatUp$ regular-rational handoff row attached to the finite target windows in $W$.
\item $E$ is the terminal $\RealUp$ seal row for the displayed target values.
\item $H$ records componentwise $\hsame$ transport for $B,O,F,W,R,E,C,P,N$.
\item $C$ records displayed $\Cont$ replay from compact-open requests through prefix windows, graph inspection, regular readback, and the real seal.
\item $P$ is the $\Pkg$ provenance over $\BaireSpaceCompactOpenUp$, $\BaireSpaceUp$, $\CompactOpenTopologyUp$, $\ContinuousMapUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\BaireSpaceCompactOpenUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $B,O,F,W,R,E,H,C,P,N$. It has no coordinate for host function equality, quotient streams, arbitrary open-cover compactness, countable choice, or ambient $\RealUp$ completeness.
\end{definition}

\begin{theorem}[Baire-space compact-open NameCert obligations]
\label{thm:baire-space-compact-open-namecert-obligations}
For every accepted $\BaireSpaceCompactOpenUp$ carrier $K=(B,O,F,W,R,E,H,C,P,N)$, the local $\NameCert_{\BaireSpaceCompactOpenUp}$ surface consists exactly of carrier admission for the Baire prefix row, compact-open subbasic-window row, continuous-map graph row, StreamName observation schedule, RegSeqRat target readback, Real seal, componentwise transport, displayed continuation replay, provenance, and local naming. Every compact-open read must inspect the prefix row $B$ and the subbasic-window row $O$ before it may consume $F,W,R,E$.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:baire-space-compact-open-namecert-carrier}. The rows $B$ and $O$ display the source prefix and compact-open request before any graph or target row is available.

The continuous-map row $F$ is therefore read only after the compact-open window has been fixed. Its target observations pass through the same $W,R,E$ route used by StreamName, RegSeqRat, and Real consumers.

Finally $H,C,P,N$ transport, replay, source, and name exactly those rows. Since the carrier contains no coordinate for host function equality, quotient streams, open-cover compactness, countable choice, or ambient completeness, the listed rows exhaust the local obligation surface.
\end{proof}

\begin{theorem}[Baire-space compact-open prefix handoff]
\label{thm:baire-space-compact-open-prefix-handoff}
Every compact-open consumer of an accepted $\BaireSpaceCompactOpenUp$ carrier factors through the finite route
$$
\BaireSpaceUp \longrightarrow \CompactOpenTopologyUp \longrightarrow \ContinuousMapUp \longrightarrow \StreamNameUp \longrightarrow \RegSeqRatUp \longrightarrow \RealUp .
$$
The handoff exports only displayed prefix cylinders, compact-open subbasic windows, graph rows, finite target observations, regular readbacks, and real seals.
\end{theorem}
\begin{proof}
Begin with the NameCert obligations of \autoref{thm:baire-space-compact-open-namecert-obligations}. They force the Baire prefix row and compact-open subbasic window to be present before any graph read is replayed.

Use the Baire prefix discipline of \autoref{ch:concrete-instances-bairespace-namecert}. It supplies finite source cylinders rather than an infinite host sequence. The compact-open row then consumes those cylinders through the topology surface of \autoref{ch:concrete-instances-compact-open-topology-namecert}.

The target side is the existing L10 route: StreamName windows feed RegSeqRat readbacks, and the Real row seals the displayed target value. Thus the public handoff is exactly the stated finite route.
\end{proof}

\begin{theorem}[Baire-space compact-open nonescape]
\label{thm:baire-space-compact-open-nonescape}
An accepted $\BaireSpaceCompactOpenUp$ packet exports no compact-open or function-space read outside $B,O,F,W,R,E,H,C,P,N$. In particular, it cannot export host function equality, quotient streams, open-cover compactness, countable choice, ambient $\RealUp$ completeness, or a completed space of all Baire-to-real functions.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:baire-space-compact-open-namecert-carrier}. The only analytic rows are the prefix row, compact-open request row, continuous-map graph row, finite observation schedule, regular readback, and real seal.

Apply the prefix handoff of \autoref{thm:baire-space-compact-open-prefix-handoff}. Every downstream read is routed through the same displayed rows before the structural rows $H,C,P,N$ can replay or name it.

The excluded objects require coordinates that are absent from the carrier. Since transport and replay preserve only displayed rows, the packet cannot leave the finite compact-open route.
\end{proof}

$$

\item $M$ is the input Cauchy modulus row, assigning displayed tail thresholds to requested dyadic tolerances.
\item $F$ is the finite normalization table, recording prefix maxima or equivalent displayed threshold joins for the finitely many tighter requests visible to the packet.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger for precision requests, slack conversions, and positivity checks consumed by $M$ and $F$.

\item $S$ is the normalized Cauchy-window seal row saying that the readback admitted after $F$ is already admitted by the input modulus row $M$ at the requested tolerance.

\begin{theorem}[Cauchy modulus normalizer NameCert obligations]
\label{thm:cauchy-modulus-normalizer-namecert-obligations}
For every accepted $\CauchyModulusNormalizerUp$ carrier
$$
Z=(X,M,F,D,W,Q,S,R,H,C,P,N),
$$
the local $\NameCert_{\CauchyModulusNormalizerUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the source, input modulus, finite normalization table, dyadic tolerance ledger, StreamName windows, RegSeqRat readback, Cauchy-window seal, Real seal, transport, replay, provenance, and local naming rows;
\item modulus-source exposure, requiring the source row $X$ and input modulus row $M$ to be displayed before the normalizer table $F$ is read;
\item finite normalization exactness, requiring $F$ to read only finitely many tolerance requests from $D$ and threshold rows from $M$;
\item window-readback order, requiring StreamName windows, RegSeqRat readback, and the Real seal to appear downstream of the normalized threshold route as $W,Q,S,R$;
\item non-escape, requiring $H,C,P,N$ to transport, replay, source, and name only the displayed normalizer rows and to export no Real completeness, quotient completion, quotient stream equality, selected limit, countable choice, trichotomy, host real equality, or bridge datum.
\end{enumerate}
No Cauchy-modulus-normalizer obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cauchy-modulus-normalizer-carrier}. The accepted packet exposes the source row, input modulus, finite normalization table, dyadic tolerance ledger, stream windows, rational readback, Cauchy-window seal, Real seal, and structural rows as one finite carrier.

Modulus-source exposure is the order of the displayed packet. A consumer cannot inspect $F$ until the source row $X$, input threshold schedule $M$, and dyadic tolerance ledger $D$ have fixed the finite requests being normalized.

Finite normalization exactness is finite because $F$ reads only the displayed tolerance requests in $D$ and their corresponding thresholds in $M$. This is the same constructive modulus-normalization discipline used by the monotone normalization packet, but here it is packaged as the normalizer boundary immediately before downstream window reads.

The window-readback order follows from the coordinates $W,Q,S,R$. StreamName observations are read after the normalized threshold, RegSeqRat readback records the rational observations, the Cauchy-window seal $S$ checks preservation against $M$, and the terminal Real seal $R$ is downstream of those rows.

Finally, $H,C,P,N$ preserve, replay, source, and name the same finite route. Since the carrier has no coordinate for Real completeness, quotient completion, quotient stream equality, selected limits, countable choice, trichotomy, host real equality, or bridge data, none of those objects can become a local NameCert obligation.
\end{proof}

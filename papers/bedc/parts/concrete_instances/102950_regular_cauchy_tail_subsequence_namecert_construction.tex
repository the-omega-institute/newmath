\begin{theorem}[Regular-Cauchy tail-subsequence NameCert obligations]
\label{thm:regular-cauchy-tail-subsequence-namecert-obligations}

$$

\item carrier admission for the source regular Cauchy row, tail-window ledger, unary cutoff, dyadic tolerance ledger, shifted regular rational readback, real seal, componentwise transport, continuation replay, provenance, and local naming rows;
\item tail cutoff exactness, requiring every retained observation to pass through the displayed unary row $k$ and the corresponding retained window in $W$;

Tail cutoff exactness is read from $k$ and $W$. A tail observation cannot be inspected until the cutoff row has been displayed and the retained source window has been named in the window ledger.

Shifted readback exactness is read from $R$. Each displayed tail index $n$ is interpreted by the finite arithmetic row $k+n$, so the packet uses a visible unary shift rather than an external subsequence function.

Tolerance preservation is the projection to $D$. The source row, retained windows, shifted readback, and real seal all consume that same dyadic ledger, so no detached modulus can alter the tail read.

Finally $H,C,P,N$ are structural rows. They preserve, replay, source, and name only the displayed finite route, leaving no row from which a consumer could extract quotient streams, selected limits, countable choice, host equality, trichotomy, ambient completeness, Lean verification, or bridge data.

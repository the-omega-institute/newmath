Following Kelley on metric topology and Bishop--Bridges constructive metric spaces, the $\MetricClosureUp$ packet names the finite closure interface by which metric tests, neighbourhood tests, located-set rows, finite StreamName windows, RegSeqRat readbacks, and Real seals expose a candidate limit as closure evidence. It sits over the metric source of \autoref{ch:concrete-instances-metric-namecert}, the topology surface of \autoref{ch:concrete-instances-topology-namecert}, the located-set interface of \autoref{ch:concrete-instances-locatedset-namecert}, the sequence-limit handoff of \autoref{ch:concrete-instances-sequentialclosure-namecert}, the finite windows of \autoref{ch:concrete-instances-streamname-namecert}, the regular rational readbacks of \autoref{ch:concrete-instances-regseqrat-namecert}, and the terminal $\RealUp$ seal of \autoref{ch:concrete-instances-real-namecert}. The packet is a named BEDC surface for metric closure reads; it is not a host topological closure operator, not an arbitrary subsequence selector, not open-cover compactness, not quotient equality, and not $\RealUp$ completeness.

\begin{definition}[MetricClosure carrier]
\label{def:metric-closure-carrier}
A $\MetricClosureUp$ carrier is a finite $\BHist$ packet
$$
K=(M,T,S,L,U,W,R,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $M$ is the ambient $\MetricUp$ row naming the finite metric comparison surface.
\item $T$ is the $\TopologyUp$ row naming the neighbourhood tests compatible with $M$.
\item $S$ is the subset or $\LocatedSetUp$ row whose admitted points are being closed.
\item $L$ is the candidate limit row, read only through the same metric and topological tests.
\item $U$ is the finite neighbourhood and metric-test ledger, pairing each displayed neighbourhood request with the matching metric tolerance request.
\item $W$ is the $\StreamNameUp$ finite-window row used to read the displayed approximating points and the candidate limit.
\item $R$ is the $\RegSeqRatUp$ handoff row for the same windows, tolerances, and metric-comparison requests.
\item $E$ is the $\RealUp$ seal reached only after $M,T,S,L,U,W,R$ have been displayed.
\item $H$ records componentwise $\hsame$ transport for the metric, topology, subset or located-set, candidate limit, test ledger, window, readback, and Real seal rows.
\item $C$ records displayed $\Cont$ replay from neighbourhood requests to metric tests, then through StreamName windows, RegSeqRat readback, and the Real seal.
\item $P$ is the $\Pkg$ provenance over $\MetricClosureUp$, $\MetricUp$, $\TopologyUp$, $\LocatedSetUp$, $\SequentialClosureUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\MetricClosureUp}$ row.
\end{enumerate}
The classifier accepts only packets whose metric row, topology row, subset or located-set row, candidate limit, paired test ledger, finite windows, rational readbacks, Real seal, transport, replay, provenance, and local naming are all displayed. It has no coordinate for a host closure operator, arbitrary subsequence choice, open-cover compactness, quotient equality, or ambient completeness of $\RealUp$.
\end{definition}

\begin{theorem}[MetricClosure NameCert obligations]
\label{thm:metric-closure-namecert-obligations}
For every accepted $\MetricClosureUp$ carrier $K=(M,T,S,L,U,W,R,E,H,C,P,N)$, the local $\NameCert_{\MetricClosureUp}$ surface consists exactly of carrier admission for the ambient metric row, the compatible topology row, the subset or located-set row, the candidate limit row, the paired neighbourhood and metric-test ledger, StreamName windows, RegSeqRat readback, RealUp sealing, componentwise $\hsame$ transport, displayed $\Cont$ replay, $\Pkg$ provenance, and local naming. Every public closure read must pass through those rows before it can consume the candidate limit.
\end{theorem}
\begin{proof}
Project the accepted carrier to the displayed rows $M,T,S,L,U,W,R,E,H,C,P,N$. The metric, topology, subset, and candidate rows give the only admissible source data for a closure read.

The ledger row $U$ then binds each neighbourhood request to a finite metric-test request. The StreamName row $W$ and RegSeqRat row $R$ provide the visible windows and rational readbacks for that same request before the Real seal $E$ is exposed.

Finally $H,C,P,N$ transport, replay, source, and name the same displayed packet. Since no row stores a host closure operator, an arbitrary subsequence, an open-cover object, quotient equality, or ambient $\RealUp$ completeness, the NameCert obligations are exhausted by the listed rows.
\end{proof}

\begin{theorem}[MetricClosure sequential-closure handoff]
\label{thm:metric-closure-sequential-closure-handoff}
Every $\SequentialClosureUp$ consumer of an accepted $\MetricClosureUp$ packet reads the candidate limit through the finite route
$$
M,T,S,L,U,W,R,E,H,C,P,N.
$$
The consumer receives a displayed metric row, a compatible topology row, a subset or located-set row, a candidate limit, paired neighbourhood and metric-test rows, finite StreamName windows, RegSeqRat readbacks, a RealUp seal, transport, replay, provenance, and local naming. It receives no ambient topological closure operator, arbitrary subsequence selector, open-cover compactness principle, quotient-equality row, or RealUp completeness theorem.
\end{theorem}
\begin{proof}
Start with a sequential-closure consumer. The carrier exposes the candidate limit $L$ only together with the common metric row $M$, topology row $T$, and source row $S$, so the consumer cannot detach the limit from the displayed metric/topology interface.

Use the test ledger $U$ to align the neighbourhood read with the metric tolerance read. The finite window row $W$, regular rational row $R$, and Real seal $E$ then give the same handoff surface used by the $\SequentialClosureUp$ packet of \autoref{ch:concrete-instances-sequentialclosure-namecert}.

The rows $H,C,P,N$ preserve that route under transport, replay, provenance, and naming. Thus the handoff is the finite metric-closure surface itself, not a stronger host closure, subsequence-choice, compactness, quotient, or completeness principle.
\end{proof}

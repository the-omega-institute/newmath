\chapter{A Concrete Naming Certificate for \(\OptionUp\)}
\label{ch:concrete-instances-option-namecert}

This chapter separates two certificate scopes. The checked BEDC object is the nullable history envelope $\mathsf{OptionHistoryCarrier}_S(h)\equiv \hsame(h,\emp)\lor S(h)$ together with the classifier endpoint relation carried by $\hsame$. The constructor-tagged optional value interface for $\OptionUp$ is a separate visible-tag certificate over a different carrier presentation; it is not a strict strengthening of the nullable envelope unless the bridge data of Definition~\ref{def:tagged-option-to-nullable-bridge-hypotheses} are supplied, with descent given by Theorem~\ref{thm:tagged-option-to-nullable-classifier-bridge}.

\section{Roadmap template status}

\begin{enumerate}
  \item \textbf{Traditional target}: optional data with one absent value and present values over a source object.
  \item \textbf{BEDC source}: any licensed source carrier $A$ equipped with its existing name certificate.
  \item \textbf{Carrier choice}: the typed $\mathsf{OptionCarrier}$ has $\mathsf{none}$ and $\mathsf{some}$ constructors, while the checked history carrier is the nullable envelope $\mathsf{OptionHistoryCarrier}_S(h)\equiv \hsame(h,\emp)\lor S(h)$.
  \item \textbf{Classifier choice}: the typed $\mathsf{OptionClassifierSpec}$ relates only matching option tags through the source classifier; $\mathsf{OptionHistoryClassifier}$ is the nullable-history classifier consisting of endpoint envelope-carrier witnesses and endpoint $\hsame$. These classifiers live over different carriers unless $\mathsf{TaggedOptionToNullableBridge}(S,\sim_S)$ supplies Definition~\ref{def:tagged-option-to-nullable-bridge-hypotheses}.
  \item \textbf{Stability obligations}: typed optional carrier transport, typed absent/present separation, lifted typed classifier symmetry and transitivity, nullable-history endpoint transport, nullable-history classifier symmetry and transitivity, and for the constructor-tagged optional-history certificate a payload coherence condition converting source-carrier $\hsame$ between middle payloads into the source classifier.
  \item \textbf{Ledger}: the typed option interface records the absent-or-present tag and present payload evidence, while the checked nullable-history envelope ledger records only the raw envelope-carrier witness and the raw-to-visible $\hsame$ link.
  \item \textbf{Core theorem list}: typed carrier cases, typed absent/present no-confusion, present inversion, typed classifier equivalence, tagged same-tag exactness, tagged mixed-tag separation, source-compatibility for the tagged semantic target, nullable-history carrier transport, nullable-history classifier transport, and the nullable-history semantic certificate.
  \item \textbf{Exactness predicate}: matching-tag exactness belongs to $\mathsf{OptionClassifierSpec}$; the tagged history classifier has same-tag exactness through visible absent and present endpoints plus mixed-tag separation, while the checked nullable-history classifier exactness is endpoint envelope membership plus endpoint $\hsame$ and carries no present constructor tag. The tagged exactness clauses do not by themselves produce nullable envelope membership for a visible present endpoint.
  \item \textbf{Standard bridge}: the checked nullable envelope has only empty-or-source endpoint readback. Ordinary $\mathsf{none}$ and $\mathsf{some}(a)$ readback belongs to the typed and tagged option interface under $\mathsf{TaggedOptionHistoryCarrier}$, $\mathsf{TaggedOptionHistoryClassifier}$, same-tag exactness, mixed-tag separation, and the tag-plus-source ledger; using that readback as nullable-envelope evidence is exactly the descent of Theorem~\ref{thm:tagged-option-to-nullable-classifier-bridge}.
  \item \textbf{Status}: Theorem~\ref{thm:concrete-option-namecert} is the checked nullable empty-or-source envelope only; Theorem~\ref{thm:tagged-option-history-semantic-certificate} is the Phase Finite Data public-object $\mathsf{ProofTarget}$ for constructor-tagged $\OptionUp$ under a source $\NameCert$ and $\mathsf{TaggedOptionSourceHsameCompatible}$; comparison with the nullable carrier presentation uses Definition~\ref{def:tagged-option-to-nullable-bridge-hypotheses}.
\end{enumerate}

\section{Carrier definition}

\begin{definition}[Option carrier]
\label{def:option-carrier}
For a licensed source carrier \(A\), define an inductive carrier with constructors \(\mathsf{none}\) and \(\mathsf{some}(a)\).
\end{definition}

The checked BEDC-history carrier used below is the nullable semantic carrier $\mathsf{OptionHistoryCarrier}_S(h)\equiv \hsame(h,\emp)\lor S(h)$, not a tagged history carrier.

\section{The certificate}

\begin{definition}[Option source specification]
\label{def:option-source-specification}
There are two source scopes. For the checked nullable-history certificate, the
source input is a BEDC history predicate $S:\mathsf{BHist}\to\mathsf{Prop}$ and
$\mathsf{OptionHistoryCarrier}_S(h)$ is exactly the empty-or-source envelope
$\hsame(h,\emp)\lor S(h)$. A source $\NameCert$ belongs only to the typed and
constructor-tagged option interface used later in
Theorem~\ref{thm:tagged-option-history-semantic-certificate}; it is not part of
the checked nullable-envelope definition displayed here.
\end{definition}

\begin{theorem}[Option history carrier transport]
\label{thm:option-history-carrier-hsame-transport}
If $h$ and $k$ are internally history-same, membership of $h$ in
$\mathsf{OptionHistoryCarrier}_S$ transports to membership of $k$ in the same
carrier. In the source branch this does not assume that an arbitrary source classifier is stable under $\hsame$; it uses the checked identity characterization of history-sameness on $\mathsf{BHist}$,
$\mathsf{BEDC.FKernel.Hist.hsame\_iff\_eq}$.
\end{theorem}
\begin{proof}
Invert the optional carrier witness. In the absent branch, use only symmetry and
transitivity of $\hsame$: reverse the displayed $\hsame(h,k)$ link and compose
it with the stored $\hsame(h,\emp)$ witness. In the source branch, convert the displayed $\hsame(h,k)$ witness via
$\mathsf{BEDC.FKernel.Hist.hsame\_iff\_eq}$, rewrite the endpoint, and transport the stored predicate witness from $S(h)$ to $S(k)$. The exported nullable classifier remains exactly
$\mathsf{OptionHistoryCarrier}_S(h)\land\mathsf{OptionHistoryCarrier}_S(k)\land\hsame(h,k)$,
so this proof does not export identity reflection as a public equality principle for $\OptionUp$.
\end{proof}
\leanchecked{BEDC.Derived.OptionUp.OptionHistoryCarrier\_hsame\_transport}

\begin{theorem}[Option history carrier unary prefix closure]
\label{thm:option-history-carrier-unary-prefix-closed}
Let $p,h:\mathsf{BHist}$. If $p$ is a unary history and
$\mathsf{OptionHistoryCarrier}_{\mathsf{UnaryHistory}}(h)$ holds, then
$$
\mathsf{OptionHistoryCarrier}_{\mathsf{UnaryHistory}}(\mathsf{append}(p,h))
$$
holds.
\end{theorem}
\begin{proof}
Split the nullable carrier witness for $h$. In the absent branch, transport
the empty unary-history witness along $\hsame(\emp,h)$ and close by unary
closure under append with the prefix $p$. In the source branch, the source
witness is already a unary-history witness for $h$, and unary append closure
again gives the carrier witness for $\mathsf{append}(p,h)$.
\end{proof}

\begin{theorem}[Option history classifier unary prefix closure]
\label{thm:option-history-classifier-unary-prefix-closed}
Let $p,h,k:\mathsf{BHist}$. If $p$ is unary and
$\mathsf{OptionHistoryClassifier}_{\mathsf{UnaryHistory}}(h,k)$ holds, then
the nullable option classifier is preserved by placing $p$ on the left:
$$
\mathsf{OptionHistoryClassifier}_{\mathsf{UnaryHistory}}
  (\mathsf{append}(p,h),\mathsf{append}(p,k)).
$$
\end{theorem}
\begin{proof}
Unpack the nullable classifier into the two endpoint carrier witnesses and the
stored $\hsame(h,k)$ link. Apply the unary-prefix carrier closure to both
endpoints. The remaining classifier field is the prefixed endpoint sameness,
obtained by eliminating the $\hsame(h,k)$ witness.
\end{proof}

\begin{definition}[Option pattern specification]
\label{def:option-pattern-specification}
The pattern is absent-or-present generation: one null constructor and one wrapper constructor over source witnesses.
\end{definition}

\begin{definition}[Option classifier specification]
\label{def:option-classifier-specification}
The typed classifier relates \(\mathsf{none}\) only to \(\mathsf{none}\), and relates \(\mathsf{some}(a)\) to \(\mathsf{some}(a')\) exactly when the source classifier relates \(a\) and \(a'\). The checked history-level classifier for $\mathsf{OptionHistoryCarrier}$ is the nullable classifier: both endpoints lie in the empty-or-source envelope, and the endpoints are related by $\hsame$.
\end{definition}

\begin{theorem}[Option history carrier source monotonicity]
\label{thm:option-history-carrier-source-monotonicity}
Let $S,T:\mathsf{BHist}\to\mathsf{Prop}$ be source predicates. If
$\forall h.\; S(h)\to T(h)$, then for every $h:\mathsf{BHist}$,
$\mathsf{OptionHistoryCarrier}_S(h)$ implies
$\mathsf{OptionHistoryCarrier}_T(h)$. Under the same hypothesis, for every
$h,k:\mathsf{BHist}$, $\mathsf{OptionHistoryClassifier}_S(h,k)$ implies
$\mathsf{OptionHistoryClassifier}_T(h,k)$.
\end{theorem}
\begin{proof}
For the carrier clause, split the nullable carrier witness. The absent branch
already supplies the same $\hsame(h,\emp)$ witness for the target envelope. In
the source branch, apply the displayed implication to the stored $S(h)$ witness
and obtain the required $T(h)$ witness.

For the classifier clause, unfold the nullable classifier data: both endpoints
come with carrier witnesses and the classifier stores the same endpoint
$\hsame(h,k)$ witness. Apply the carrier clause to the two endpoint carrier
witnesses, and keep the stored $\hsame(h,k)$ witness unchanged.
\end{proof}
\leanchecked{BEDC.Derived.OptionUp.OptionHistorySource\_monotonicity}

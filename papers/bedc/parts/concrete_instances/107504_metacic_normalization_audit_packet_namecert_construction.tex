\begin{definition}[MetaCIC normalization audit packet carrier]
\label{def:metacic-normalization-audit-packet-carrier}
A $\MetaCICNormalizationAuditPacketUp$ carrier is a finite $\BHist$ packet
$$
M=(T,K,A,S,I,P,F,H,C,G,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $T$ is the closed typed term row, exposing the term, type, empty-context read, and closedness data.
\item $K$ is the candidate-evidence row, carrying $\mathsf{Candidate}$, closed-candidate, or candidate-set membership evidence for the displayed term.
\item $A$ is the application adequacy row, recording the function-side Pi evidence, argument evidence, and application-candidate handoff used when the term route contains an application.
\item $S$ is the strong-normalization projection row, reading SN only through the candidate or candidate-set evidence displayed in $K$.
\item $I$ is the no-infinite-reduction row, projecting the same candidate-mediated SN evidence to the absence of infinite reduction chains.
\item $P$ is the Pi-candidate adequacy row, recording the Pi route used by dependent function cases without replacing $K$.
\item $F$ is the normalization-frontier refusal row, kept visible whenever a requested citation would erase candidate evidence, subject-reduction boundaries, or confluence boundaries.
\item $H$ records componentwise $\hsame$ transport for the closed typed term, candidate evidence, application adequacy, SN projection, no-infinite-reduction projection, Pi adequacy, frontier, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a normalization-audit request through $T,K,A,S,I,P,F,H,G,N$.

\item $N$ is the local $\NameCert_{\MetaCICNormalizationAuditPacketUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $T,K,A,S,I,P,F,H,C,G,N$. No row stores a theorem that every closed typed MetaCIC term normalizes without candidate or candidate-set evidence.
\end{definition}

\begin{theorem}[MetaCIC normalization audit packet NameCert obligations]
\label{thm:metacic-normalization-audit-packet-namecert-obligations}

\end{theorem}
\begin{proof}
Project the carrier. The only term source is $T$, and the only evidence source for a public SN read is the candidate row $K$.

Read $A$ and $P$ as adequacy rows for application and Pi cases. They refine the same candidate-mediated route and do not create an evidence-free normalization source.

The SN row $S$ is downstream of $K$. The no-infinite-reduction row $I$ is downstream of $S$, so absence of infinite reduction chains is cited only through the same candidate or candidate-set evidence.

The frontier row $F$ records requests that would erase the evidence boundary. Transport, replay, provenance, and local naming through $H,C,G,N$ preserve these rows without adding subject reduction, confluence, quotient syntax equality, host evaluator soundness, or closed consistency.
\end{proof}

\begin{theorem}[MetaCIC normalization audit packet candidate handoff]
\label{thm:metacic-normalization-audit-packet-candidate-handoff}
Every public strong-normalization or no-infinite-reduction read from an accepted $\MetaCICNormalizationAuditPacketUp$ carrier factors through the finite route
$$
T \longmapsto K \longmapsto (A,P) \longmapsto S \longmapsto I \longmapsto F .
$$
In particular, a public SN read must pass through candidate or candidate-set evidence before the closed typed term can be read as strongly normalizing, and the exported packet is not an unconditional normalization theorem for all closed typed MetaCIC terms.
\end{theorem}
\begin{proof}
Start with a public SN request. The term row $T$ displays the closed typed term, but the carrier does not allow $T$ alone to be read as normalized.

The request must next pass through $K$. Candidate, closed-candidate, or candidate-set evidence is the only input accepted by the projection row $S$, while $A$ and $P$ supply only the application and Pi adequacy data required by that same route.

The no-infinite-reduction row $I$ reuses the output of $S$. If a consumer asks for evidence-free normalization, unrestricted subject reduction, global confluence, quotient syntax equality, host evaluation soundness, or closed consistency, the frontier row $F$ keeps that request outside the packet.

Thus the handoff exports candidate-mediated SN and no-infinite-reduction evidence, not a theorem that every closed typed MetaCIC term normalizes without the displayed candidate or candidate-set rows.
\end{proof}

\end{theorem}
\begin{proof}

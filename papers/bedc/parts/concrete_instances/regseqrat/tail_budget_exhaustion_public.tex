\begin{theorem}[RegSeqRat tail-budget exhaustion]
\label{thm:regseqrat-tail-budget-exhaustion}
Every finite dyadic request admitted by the $\RegSeqRatUp$ tail selector is exhausted by a displayed tail budget and cannot escape the $\RealUp$ seal route. The displayed budget contains the StreamName window row, DyadicRatCore tolerance row, RegSeqRat regularity row, selected tail row, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows.

Thus no admitted finite dyadic request can be consumed as a Real-facing request without passing through the same tail-budget packet and the same public seal boundary.
\end{theorem}
\begin{proof}
Begin with the tail-budget handoff of \autoref{thm:regseqrat-tail-budget-meet-handoff}. It names the finite StreamName window, DyadicRatCore observation rows, RegSeqRat regularity rows, and local support rows that any Real-facing tail-budget request must display.

Apply the tail selector Real-seal exhaustion theorem, \autoref{thm:regseqrat-tail-budget-terminal-real-seal}. The selector cannot export a finite dyadic request to the Real seal unless the displayed budget rows already carry that request.

Now use the non-escape result \autoref{thm:regseqrat-tail-budget-nonescape}. It removes quotient sequence equality, selected limits, countable choice, and ambient completeness from the tail-budget consumer route.

The remaining data are exactly the displayed tail budget and the public RealUp seal route. Therefore every admitted finite dyadic request is exhausted by that budget and cannot escape the seal boundary.
\end{proof}

\begin{theorem}[RegSeqRat tail-budget exit status product]
\label{thm:regseqrat-tail-budget-exit-status-product}
Every accepted finite dyadic request admitted by the $\RegSeqRatUp$ tail selector and consumed as a Phase Real and Completion endpoint is audit-readable only as the componentwise status product of four local face records: the $\DyadicRatCoreUp$ tolerance ledger, the $\StreamNameUp$ finite-window row, the $\RegSeqRatUp$ regularity and classifier row, and the $\RealUp$ seal row. Each face contributes its own $\mathsf{closurestatus}$ and $\mathsf{formalstatus}$ record, while the shared transport is only the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support.

The exit read contains no quotient stream, host equality, selected limit, countable-choice principle, ambient completeness datum, metric-completion datum, bridge row, aggregate $\mathsf{formalstatus}$ token, or promotion from one status axis to the other.
\end{theorem}
\begin{proof}
Start with \autoref{thm:regseqrat-tail-budget-exhaustion}. It puts every admitted finite dyadic request on the displayed tail-budget packet and keeps the Real-facing consumer inside the same public $\RealUp$ seal boundary.

Next apply \autoref{thm:regseqrat-four-face-terminal-pullback} and \autoref{thm:regseqrat-dyadic-tail-real-seal-status-exhaustion}. These results identify the terminal read as the four-face ledger consisting of the DyadicRatCore tolerance face, StreamName finite-window face, RegSeqRat regularity and classifier face, and terminal Real seal face, transported only by the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and local $\NameCert$ rows.

Finally read the ledger through the two-axis discipline of \autoref{ch:closure-and-verification-discipline}. A face's closure record and verification record remain local to that face, so their product is only a componentwise audit surface. It cannot add a quotient stream, host equality, selected limit, countable-choice principle, completeness datum, bridge row, aggregate verification token, or implication from one axis to the other.
\end{proof}

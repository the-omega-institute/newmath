The zero-source gluing theorem therefore has no compatible local zero packet from which to recover the same boundary on $G$.

If the deleted row is one of the shared support rows, first apply \autoref{thm:regseqrat-audit-threshold-support-removal-descent} to transport the support loss to the glued RegSeqRat audit-threshold read. Then apply \autoref{thm:streamname-finite-cover-support-removal-global-obstruction} on the StreamName--DyadicRatCore--RegSeqRat--Real route. The altered cover no longer carries the global support needed to present the same zero-source seal-visible boundary.

It remains to exclude external substitutes. Apply \autoref{thm:regular-cauchy-zero-terminal-consumer-uniqueness} to identify any alleged presentation of the same glued boundary with the terminal zero-source consumer. Then use \autoref{thm:regular-cauchy-zero-real-seal-nonescape} and \autoref{thm:regular-cauchy-zero-downstream-handoff} to rule out quotient streams, host real equality, selected limits, nonzero witnesses, and ambient completeness as alternate witnesses.

The countable-choice, metric-completeness, aggregate status, and bridge rows are also outside the finite source packet. By \autoref{thm:regseqrat-tail-budget-support-minimality} and the finite-cover gluing theorem, none of them is among the four displayed zero faces or the shared $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support. Thus the altered finite cover cannot present the same glued $\RegularCauchyZeroUp$ zero-source boundary.
\end{proof}

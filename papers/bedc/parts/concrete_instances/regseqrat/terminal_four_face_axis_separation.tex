\section{RegSeqRat Terminal Four-Face Axis Separation}
\label{sec:regseqrat-terminal-four-face-axis-separation}

\begin{theorem}[RegSeqRat terminal four-face axis separation]
\label{thm:regseqrat-terminal-four-face-axis-separation}
Every $\RegSeqRatUp$ terminal four-face read through
$$
\StreamNameUp,\quad \DyadicRatCoreUp,\quad \RegSeqRatUp,\quad \RealUp
$$
supplies only the paper-side finite boundary whose support rows are
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
The read cannot raise a formal-status token for any of the four faces, create
a standard bridge row, assert ambient $\RealUp$ completeness, introduce
quotient stream equality, or supply metric-completeness data.

Thus downstream Phase Real and Completion consumers may use the terminal
four-face packet only as a finite manuscript boundary. Verification status,
bridge status, quotient identification, and completeness data must still be
read from the relevant face's own closurestatus, Lean target, and bridge
boundary.
\end{theorem}
\begin{proof}
Start with \autoref{thm:regseqrat-terminal-four-face-correspondence}. It
identifies terminal RegSeqRat agreement exactly with the visible four-face
inventory and transports that agreement only through the displayed $\BHist$,
$\hsame$, $\Cont$, $\Pkg$, and local $\NameCert$ rows.

Next apply \autoref{thm:regseqrat-four-face-hub-sibling-route}. The route is
not an isolated RegSeqRat coordinate: it passes through the DyadicRatCore,
StreamName, and Real sibling chapters before it is consumed as a Real-facing
handoff, while retaining the same finite support rows.

Now invoke \autoref{thm:streamname-real-completion-four-object-exit-route}.
It reads the Phase Real and Completion exit packet as exactly the four faces
$\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$, with no
quotient stream equality, host sequence equality, selected limit, ambient
$\RealUp$ completeness theorem, metric-completeness datum, countable-choice
row, or standard bridge row.

Finally use \autoref{thm:real-regseq-streamname-cauchy-completion-interface}.
The Real-facing completion interface consumes the same finite window,
regular-rational readback, dyadic observation ledger, seal row, componentwise
$\hsame$ transport, displayed $\Cont$ route, package provenance, and local
$\NameCert$ row. Since the cited routes contain no verification-axis token,
standard bridge row, ambient completeness principle, quotient stream
equality, or metric-completeness coordinate, the terminal four-face read
separates the paper-side boundary from those independent axes.
\end{proof}

\begin{theorem}[RegSeqRat local-threshold exit descent]
\label{thm:regseqrat-local-threshold-exit-descent}
Let $\mathcal E$ be an accepted objectwise Phase $\RealUp$ and Completion exit-threshold read whose public classifier coordinate is read on the same visible terminal packet. Suppose the visible source of $\mathcal E$ is an accepted $\RegSeqRatUp$ terminal four-face packet with rows over $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and the local $\RealUp$ seal. Then the exit-threshold coordinate descends to that same finite source record.

The descended record carries exactly the $\StreamNameUp$ finite-window rows, the $\DyadicRatCoreUp$ endpoint, radius, distance, and regularity rows, the $\RegSeqRatUp$ classifier and common-tail rows, the local $\RealUp$ seal rows, and the support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
The descent uses only the displayed finite $\BHist$ packet, componentwise $\hsame$ transport, displayed $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ support. It exports no quotient streams, selected limits, host real equality, countable-choice row, ambient $\RealUp$ completeness, metric-completeness data, aggregate $\formalstatus$ row, or aggregate bridge row.
\end{theorem}
\begin{proof}
First read the threshold and classifier coordinates from one visible packet. Apply \autoref{thm:real-l10-exit-threshold-classifier-coherence}. The objectwise exit-threshold read and the public classifier coordinate are two readings of the same finite four-face record over $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, local $\RealUp$, and the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.

Next identify the terminal pullback. Apply \autoref{thm:real-l10-terminal-pullback-uniqueness} to the accepted Phase Real and Completion terminal read. The terminal boundary has only one pullback packet with those four visible faces and those support rows, so the exit-threshold coordinate cannot be read from a second completion-facing source.

Now descend through the RegSeqRat side. Use \autoref{thm:regseqrat-terminal-four-face-source-uniqueness}. Since the visible source is an accepted $\RegSeqRatUp$ terminal four-face packet and the StreamName, DyadicRatCore, RegSeqRat, RealUp, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and local $\NameCert$ rows agree with the terminal pullback, the RegSeqRat source section is exactly the same finite record.

Finally compare the common-tail boundary rows. The common-tail bridge and terminal section results, \autoref{thm:regseqrat-common-tail-real-boundary-equivalence}, \autoref{thm:regseqrat-common-tail-terminal-section-cover}, and \autoref{thm:regseqrat-common-tail-terminal-four-face-sheaf-exactness}, keep the RegSeqRat regularity and common-tail rows on the same four-face surface. The four-face exactness theorem \autoref{thm:regseqrat-real-terminal-four-face-exactness} then reads the local $\RealUp$ seal from that displayed packet.

Thus the threshold coordinate descends to the same finite record over $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$. None of the cited packets contains a coordinate for quotient streams, selected limits, host real equality, countable choice, ambient $\RealUp$ completeness, metric completeness, aggregate verification status, or bridge rows, so no such data can appear in the descended source.
\end{proof}

\begin{theorem}[RegSeqRat local-threshold four-face sufficiency]
\label{thm:regseqrat-local-threshold-four-face-sufficiency}
Let $\mathcal R$ be a finite $\RegSeqRatUp$ terminal source record. Suppose $\mathcal R$ carries the $\StreamNameUp$ finite-window rows, the $\DyadicRatCoreUp$ endpoint, radius, distance, and regularity rows, the $\RegSeqRatUp$ classifier and common-tail rows, the local $\RealUp$ seal rows, and the displayed support
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
If these rows satisfy the four-face exactness criterion of \autoref{thm:regseqrat-real-terminal-four-face-exactness}, then the objectwise Phase $\RealUp$ and Completion exit-threshold coordinate is accepted on the same record $\mathcal R$.

The accepted coordinate exports no quotient stream, selected limit, host real equality, countable-choice row, metric-completeness data, aggregate $\formalstatus$ row, aggregate bridge row, or ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
Begin with the displayed four-face data on $\mathcal R$. Apply \autoref{thm:regseqrat-real-terminal-four-face-exactness}. The StreamName finite window, DyadicRatCore endpoint and radius ledger, RegSeqRat classifier and common-tail face, and local Real rows therefore present a Real terminal seal on exactly the same finite source record.

Next read the classifier and threshold coordinates from that terminal seal. Apply \autoref{thm:real-l10-exit-threshold-classifier-coherence}. The public classifier coordinate and the objectwise exit-threshold coordinate are two visible readings of the same four-face packet, with support only from the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.

Finally compare the accepted threshold read with the original RegSeqRat source. Use \autoref{thm:regseqrat-local-threshold-exit-descent}. Since the visible source already is $\mathcal R$ and the terminal four-face exactness step supplies the local Real seal on $\mathcal R$, the descended source of the accepted exit-threshold coordinate is $\mathcal R$ itself.

The construction has used only the finite rows named in the statement. No cited component introduces quotient streams, selected limits, host real equality, countable choice, metric-completeness structure, aggregate verification status, aggregate bridge data, or ambient $\RealUp$ completeness, so the accepted coordinate contains none of those excluded fields.
\end{proof}

\begin{theorem}[RegSeqRat local-threshold terminal-window fusion]
\label{thm:regseqrat-local-threshold-terminal-window-fusion}
Let $T$ be a finite $\RegSeqRatUp$ terminal source presented to the Phase $\RealUp$ and Completion exit. An accepted local-threshold coordinate for the Phase $\RealUp$ and Completion read exists on $T$ if and only if $T$ carries one fused terminal-window packet whose visible faces are
$$
\StreamNameUp,\quad \DyadicRatCoreUp,\quad \RegSeqRatUp,\quad \RealUp
$$
and whose support is only
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
Under this equivalence, the local-threshold coordinate and the fused terminal-window packet determine the same BEDC-visible terminal source.

The fused packet refuses quotient streams, host real equality, selected limits, countable choice, metric-completeness data, aggregate status rows, standard bridge rows, and ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
For the forward direction, start with an accepted Phase $\RealUp$ and Completion local-threshold coordinate on the finite source $T$. Apply \autoref{thm:regseqrat-local-threshold-exit-descent}. The coordinate descends to the same RegSeqRat terminal source and exposes only the StreamName, DyadicRatCore, RegSeqRat, local Real, and displayed support rows named in the statement.

Next apply \autoref{thm:regseqrat-real-terminal-four-face-exactness}. The descended source is therefore the exact four-face terminal surface over $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$. Then use \autoref{thm:streamname-terminal-window-fusion-source-exhaustion} to repack that surface as one fused terminal-window packet without changing the source $T$.

Finally apply \autoref{thm:dyadicratcore-l10-exit-namecert-interface}. It identifies the fused four-face packet with the local $\NameCert$ interface consumed by the L10 exit, so the packet carries precisely the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows.

For the reverse direction, assume $T$ carries one fused terminal-window packet over the four displayed faces and the five displayed support rows. The source-exhaustion theorem \autoref{thm:streamname-terminal-window-fusion-source-exhaustion} reads this packet as the same public finite-request section, and \autoref{thm:dyadicratcore-l10-exit-namecert-interface} reads it as the local $\NameCert$ interface for the Phase $\RealUp$ and Completion exit.

Now apply \autoref{thm:regseqrat-real-terminal-four-face-exactness} to obtain the local $\RealUp$ terminal seal on $T$. With that exact four-face surface in place, \autoref{thm:regseqrat-local-threshold-four-face-sufficiency} supplies the accepted local-threshold coordinate on the same source.

The non-escape clause is inherited in both directions. The descent theorem, the four-face sufficiency theorem, the RegSeqRat terminal exactness theorem, the StreamName source-exhaustion theorem, and the DyadicRatCore local $\NameCert$ interface all expose only finite $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support. None supplies a coordinate for quotient streams, host real equality, selected limits, countable choice, metric-completeness data, aggregate status rows, standard bridge rows, or ambient $\RealUp$ completeness, so the equivalence cannot export any of those fields.
\end{proof}

\begin{theorem}[RegSeqRat four-face exit-threshold equivalence]
\label{thm:regseqrat-four-face-exit-threshold-equivalence}
Let $\mathcal R$ be a finite $\RegSeqRatUp$ terminal source record whose visible rows are exactly the $\StreamNameUp$ finite-window rows, the $\DyadicRatCoreUp$ endpoint, radius, distance, and regularity rows, the $\RegSeqRatUp$ classifier and common-tail rows, the local $\RealUp$ seal rows, the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support, and no other completion-facing coordinates. Then $\mathcal R$ satisfies the real terminal four-face exactness criterion of \autoref{thm:regseqrat-real-terminal-four-face-exactness} if and only if the objectwise Phase $\RealUp$ and Completion exit-threshold coordinate is accepted on the same record $\mathcal R$.

The equivalence is terminal over the displayed four faces: it exports no quotient stream, selected limit, host real equality, countable-choice row, metric-completeness row, aggregate $\formalstatus$ row, aggregate bridge row, or ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
For the forward direction, assume the displayed rows of $\mathcal R$ satisfy the four-face exactness criterion. Apply \autoref{thm:regseqrat-local-threshold-four-face-sufficiency}. Since the source record already has exactly the StreamName finite-window rows, DyadicRatCore endpoint, radius, distance, and regularity rows, RegSeqRat classifier and common-tail rows, local Real seal rows, and the displayed support rows, the objectwise Phase $\RealUp$ and Completion exit-threshold coordinate is accepted on $\mathcal R$ itself.

For the reverse direction, assume the objectwise Phase $\RealUp$ and Completion exit-threshold coordinate is accepted on $\mathcal R$. Apply \autoref{thm:regseqrat-local-threshold-exit-descent} to descend that accepted coordinate to the same finite RegSeqRat terminal source. The descent identifies the accepted source with the four visible faces and the same $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support carried by $\mathcal R$.

It remains to read the descended source as the terminal four-face record. Use \autoref{thm:real-l10-current-phase-exit-local-threshold} to identify the accepted exit coordinate with the objectwise product of the $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, and local $\RealUp$ threshold records. Then apply \autoref{thm:regseqrat-real-terminal-four-face-exactness} to read those four rows as the RegSeqRat real terminal four-face exactness certificate on $\mathcal R$.

All steps are performed on the finite source record named in the statement. The cited criteria mention only the displayed four faces and support rows, so quotient streams, selected limits, countable choice, metric-completeness rows, aggregate verification or bridge rows, and ambient $\RealUp$ completeness are excluded from both directions.
\end{proof}

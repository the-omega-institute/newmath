\begin{definition}[Uniform entourage regular-name carrier]
\label{def:uniform-entourage-regular-name-carrier}
A $\UniformEntourageRegularNameUp$ carrier is a finite $\BHist$ packet
$$
E=(U,F,W,R,Q,T,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $U$ is the $\UniformSpaceUp$ entourage-data row, listing the finite entourage requests, refinement witnesses, diagonal witnesses, and composition witnesses available to the route.
\item $F$ is the $\CauchyFilterUp$ request row, listing the finite filter requests and stability witnesses whose Cauchy behavior is tested against the entourage data in $U$.
\item $W$ is the $\StreamNameUp$ finite-window row opened by the displayed entourage and Cauchy-filter requests.
\item $R$ is the $\RegSeqRatUp$ readback row obtained from the same $U,F,W$ data.
\item $Q$ is the $\CauchyPrecompletionUp$ handoff row that accepts the displayed regular rational name before quotient completion or representative selection.
\item $T$ is the $\RealUp$ consumer row, naming only downstream real-name readers of $Q$ and $R$ without importing ambient completeness.
\item $H$ records componentwise $\hsame$ transport for the uniform, Cauchy-filter, stream-window, regular-readback, precompletion, Real-consumer, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from an entourage-regular-name request through $U,F,W,R,Q,T,H,P,N$.

\item $N$ is the local $\NameCert_{\UniformEntourageRegularNameUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $U,F,W,R,Q,T,H,C,P,N$ and the displayed routes among those rows. It contains no quotient completion row, selected representative, countable-choice schedule, host real equality, ambient $\RealUp$ completeness datum, open-cover compactness datum, Lean verification credential, or standard bridge.
\end{definition}

\begin{theorem}[Uniform entourage regular-name NameCert obligations]
\label{thm:uniform-entourage-regular-name-namecert-obligations}
For every accepted $\UniformEntourageRegularNameUp$ carrier
$$
E=(U,F,W,R,Q,T,H,C,P,N),
$$
the local $\NameCert_{\UniformEntourageRegularNameUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the uniform entourage data, Cauchy-filter requests, finite stream windows, regular rational readback, precompletion handoff, Real-consumer row, transport, replay, provenance, and local naming row;
\item entourage locality, requiring every regular-name request to read only the displayed $\UniformSpaceUp$ entourages in $U$ before it can inspect a stream window;
\item Cauchy-filter compatibility, requiring the displayed $\CauchyFilterUp$ row $F$ to be the only filter source paired with the entourage tests in $U$;
\item finite-window readback, requiring $W$ and $R$ to expose a $\StreamNameUp$ window and $\RegSeqRatUp$ readback attached to the same $U,F$ route;
\item completion-facing handoff, requiring $Q$ and $T$ to name only the $\CauchyPrecompletionUp$ and $\RealUp$ consumers of that regular name;
\item non-escape, requiring $H,C,P,N$ to transport, replay, source, and name no row outside $U,F,W,R,Q,T,H,C,P,N$.
\end{enumerate}
No local obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:uniform-entourage-regular-name-carrier}. The accepted packet displays exactly the uniform, filter, window, readback, handoff, consumer, and structural rows.

Entourage locality is forced by the first coordinate $U$. Any regular-name request starts by reading the finite entourage data already accepted by the $\UniformSpaceUp$ row, so no external uniform structure can be inspected.

Cauchy-filter compatibility is the role of $F$. The packet pairs the finite $\CauchyFilterUp$ requests with the same entourage data, and there is no second filter coordinate from which another Cauchy source could be read.

The finite-window and readback obligations are the rows $W$ and $R$. They expose the $\StreamNameUp$ windows and $\RegSeqRatUp$ regular rational name attached to $U,F$, not to a selected representative or host real.

The rows $Q$ and $T$ only hand the displayed regular name to $\CauchyPrecompletionUp$ and $\RealUp$ consumers. Finally $H,C,P,N$ preserve, replay, source, and name the same finite carrier, so the listed rows exhaust the local certificate surface.
\end{proof}

\begin{theorem}[Uniform entourage RegSeqRat handoff]
\label{thm:uniform-entourage-regular-name-regseqrat-handoff}
Every regular rational name exported by an accepted $\UniformEntourageRegularNameUp$ carrier
$$
E=(U,F,W,R,Q,T,H,C,P,N)
$$
factors through the displayed route
$$
U,F \longmapsto W \longmapsto R
$$
before it reaches precompletion or Real-facing consumers. Consequently the packet hands uniform Cauchy data to $\RegSeqRatUp$ without selecting representatives, invoking countable choice, or asserting host real equality.
\end{theorem}
\begin{proof}
Begin with an exported regular rational name. The carrier exposes the uniform entourage row $U$ and the Cauchy-filter request row $F$ before any stream observation is available.

The stream-window row $W$ is therefore opened only by the displayed $U,F$ data. It contains the finite observations permitted by the route and no selected infinite representative.

The readback row $R$ repacks those observations as the $\RegSeqRatUp$ name attached to the same route. Thus the exported regular rational name is generated by the displayed entourage and filter data rather than by quotient completion or host equality.

Rows $Q,T,H,C,P,N$ appear only after this readback. They hand the name to precompletion and Real-facing consumers, then preserve and name the route without adding countable choice or representative selection.
\end{proof}

It is a NameCert surface for an inverse-reading route, not a host inverse function, real trichotomy, quotient real equality, selected representative, or ambient $\RealUp$ completeness principle. The packet consumes the sign and interval refinement discipline of \autoref{ch:concrete-instances-intermediate-value-bisection-namecert}, the monotone limit route of \autoref{ch:concrete-instances-bishop-monotone-real-limit-namecert}, the local slope and image controls of \autoref{ch:concrete-instances-constructive-mean-value-namecert} and \autoref{ch:concrete-instances-mean-value-theorem-namecert}, and the real-name readback lattice of \autoref{ch:concrete-instances-real-namecert}, \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, and \autoref{ch:concrete-instances-dyadicratcore-namecert}.

\begin{definition}[Monotone inverse carrier]
\label{def:monotone-inverse-carrier}
A $\MonotoneInverseUp$ carrier is a finite $\BHist$ packet
$$
I=(A,T,M,B,W,R,D,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $A$ is the source interval row. It records the input endpoint windows, endpoint order evidence, located subinterval identity, and the displayed domain restriction on which the map is inspected.
\item $T$ is the target interval row. It records the target real-name window, endpoint-bracket data, and the finite assertion that the target request lies between the displayed source endpoint images.
\item $M$ is the strict monotonicity evidence row. It stores the finite comparison route for source windows and the orientation flag, increasing or decreasing, used by every inverse read.
\item $B$ is the finite bisection schedule. It records branch prefixes, midpoint image comparisons, retained source intervals, and the handoff to the intermediate-value bisection surface.
\item $W$ is the $\StreamNameUp$ window row for source endpoints, target requests, midpoint images, and retained source windows.
\item $R$ is the $\RegSeqRatUp$ readback row. It records the regular rational approximants extracted from $W$ before any real-facing seal is inspected.
\item $D$ is the $\DyadicRatCoreUp$ tolerance row. It records requested precisions, dyadic interval widths, branch-depth budgets, image-comparison tolerances, and retained-window radii.
\item $E$ is the $\RealUp$ inverse seal. It is reached only after the source interval, target interval, monotonicity evidence, bisection schedule, stream windows, regular readback, and dyadic tolerance rows have been displayed.
\item $H$ records componentwise $\hsame$ transport for the interval rows, monotonicity row, bisection schedule, stream windows, regular readbacks, dyadic tolerances, inverse seal, replay, provenance, and local naming rows.
\item $C$ records displayed $\Cont$ replay from a target request through interval inspection, monotonicity comparison, bisection refinement, stream-window exposure, regular readback, dyadic tolerance accounting, and inverse-seal consumption.
\item $P$ is the $\Pkg$ provenance over $\MonotoneInverseUp$, $\IntermediateValueBisectionUp$, $\BishopMonotoneRealLimitUp$, $\ConstructiveMeanValueUp$, $\MeanValueTheoremUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\MonotoneInverseUp}$ row.
\end{enumerate}
The classifier compares accepted carriers only through $A,T,M,B,W,R,D,E,H,C,P,N$. It has no coordinate for a host inverse function, real trichotomy, quotient real equality, selected representative, ambient $\RealUp$ completeness, or an unrecorded comparison of real values.
\end{definition}

\begin{theorem}[Monotone inverse NameCert obligations]
\label{thm:monotone-inverse-namecert-obligations}
For every accepted $\MonotoneInverseUp$ carrier
$$
I=(A,T,M,B,W,R,D,E,H,C,P,N),
$$
the local $\NameCert_{\MonotoneInverseUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the source interval, target interval, strict monotonicity evidence, finite bisection schedule, stream windows, regular readbacks, dyadic tolerance rows, inverse seal, transport, replay, provenance, and local naming;
\item interval discipline, requiring $A$ and $T$ to expose the source bracket and target-in-image bracket before any branch comparison or real-facing inverse row is available;
\item monotonicity discipline, requiring $M$ to orient every midpoint comparison and retained-interval decision before the bisection schedule can be read;
\item bisection handoff discipline, requiring $B,W,R,D$ to display branch prefixes, finite windows, regular rational readbacks, and dyadic tolerance accounting before the inverse seal is inspected;
\item structural nonescape, requiring $E,H,C,P,N$ to export only the displayed route and no host inverse function, real trichotomy, quotient real equality, selected representative, ambient $\RealUp$ completeness, or unrecorded real comparison.
\end{enumerate}
No MonotoneInverse obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:monotone-inverse-carrier}. The accepted packet displays exactly the source interval, target interval, strict monotonicity evidence, bisection schedule, stream windows, regular readbacks, dyadic tolerances, inverse seal, and structural rows.

The interval discipline is the entry point. The source row $A$ fixes the domain bracket, while the target row $T$ fixes the requested value as a displayed bracket between endpoint image rows. Without those two rows there is no bisection request to orient.

The monotonicity discipline supplies the orientation. The row $M$ is the only stored evidence that source-order comparisons determine image-order comparisons, so each branch decision in $B$ must read $M$ before retaining a half-interval.

The bisection handoff is then forced by the packet order. The schedule $B$ produces finite retained source windows, $W$ exposes those windows as StreamName data, $R$ reads them as regular rational approximants, and $D$ records the dyadic tolerance budget used by the requested inverse precision.

Finally $E,H,C,P,N$ seal, transport, replay, source, and name the same finite route. Since the excluded analytic objects are absent from the carrier, no local obligation can export them.
\end{proof}

\begin{theorem}[Monotone inverse bisection handoff]
\label{thm:monotone-inverse-bisection-handoff}
Every inverse read of an accepted $\MonotoneInverseUp$ carrier factors through the displayed route
$$
A,T,M \longmapsto B \longmapsto W,R,D \longmapsto E,H,C,P,N
$$
before any $\RealUp$ inverse seal is public. The route first reads the source and target interval rows, then the strict monotonicity orientation, then the finite bisection schedule, then the StreamName windows, RegSeqRat readbacks, and DyadicRatCore tolerance rows, and only then the terminal real-name seal. It exports a finite inverse-read packet rather than a host inverse function or an ambient completeness argument.
\end{theorem}
\begin{proof}
Start with an accepted packet $I=(A,T,M,B,W,R,D,E,H,C,P,N)$. The only source-domain and target-request coordinates are $A$ and $T$, so every inverse read begins by opening those displayed interval rows.

Next use strict monotonicity. The row $M$ supplies the orientation that turns each midpoint image comparison into a retained source half-interval. The bisection schedule $B$ is therefore downstream of $M$, and it cannot be read as an unordered search or as real trichotomy.

The readback stage is finite. The branch prefix in $B$ exposes retained windows through $W$, the regular rational approximants are read through $R$, and the requested precision is accounted for through $D$. These are exactly the StreamName, RegSeqRat, and DyadicRatCore rows available before sealing.

The $\RealUp$ inverse seal $E$ is terminal. The transport, replay, provenance, and local naming rows $H,C,P,N$ preserve and present the same route. Since no carrier coordinate stores a host inverse, quotient equality, selected representative, or ambient completeness witness, every inverse read factors through the displayed bisection and monotonicity route before the seal.
\end{proof}

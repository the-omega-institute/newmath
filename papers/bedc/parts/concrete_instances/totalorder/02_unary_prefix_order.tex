\section{Concrete unary prefix total order instance}
\label{sec:totalorder-concrete-unary-prefix-instance}

\begin{definition}[Concrete unary prefix total order instance]
\label{def:concrete-unary-prefix-totalorder-instance}
Let the carrier be the concrete unary-history preorder carrier:
$$
\mathsf{Carrier}_{\mathsf{u}}:=\mathsf{PreorderCarrier}
=\mathsf{UnaryHistory}.
$$
Let the carrier classifier be
$$
\rho_{\mathsf{u}}(h,k):=
\mathsf{PreorderCarrierClassifierSpec}(\mathsf{PreorderCarrier})(h,k),
$$
and let the order relation be the concrete prefix relation
$$
h\le_{\mathsf{u}}k:=\mathsf{PreorderPrefixLE}(h,k).
$$
The constructive comparison package has the three branches
$$
\mathsf{lt}_{\mathsf{u}}(h,k):=\mathsf{NatUnaryStrictPrefix}(h,k),
\qquad
\mathsf{same}_{\mathsf{u}}(h,k):=\rho_{\mathsf{u}}(h,k),
\qquad
\mathsf{gt}_{\mathsf{u}}(h,k):=\mathsf{NatUnaryStrictPrefix}(k,h).
$$
Thus strict-left means that $k$ extends $h$ by a nonempty unary tail, same is
the carrier classifier, and strict-right is the opposite strict unary prefix.
\end{definition}

\begin{theorem}[Concrete unary prefix total order laws]
\label{thm:concrete-unary-prefix-totalorder-laws}
For carried unary histories $h,k$ in
$\mathsf{Carrier}_{\mathsf{u}}$, the branches of
\autoref{def:concrete-unary-prefix-totalorder-instance} supply constructive
total comparison. Each strict branch implies carrier-classifier apartness, the
same branch satisfies
$$
\mathsf{same}_{\mathsf{u}}(h,k)\Longleftrightarrow
(\mathsf{PreorderPrefixLE}(h,k)\land\mathsf{PreorderPrefixLE}(k,h)),
$$
and the inherited preorder and antisymmetry fields assemble these data into a
concrete $\TotalOrderUp$ law package over unary histories. The strict comparison
row is the isolated branch transitivity of
\autoref{thm:totalorder-strict-branch-transitivity}.
\end{theorem}
\begin{proof}
Start from the total unary prefix comparison of
\autoref{thm:nat-unary-prefix-total}. For carried histories $h$ and $k$, this
returns a unary continuation from $h$ to $k$ or one from $k$ to $h$.

Split the selected unary tail by the unary-generation cases. A nonempty tail
chooses the corresponding strict branch
$\mathsf{NatUnaryStrictPrefix}(h,k)$ or
$\mathsf{NatUnaryStrictPrefix}(k,h)$. An empty tail collapses the continuation
to $\hsame(h,k)$, which repackages with the two carrier witnesses as
$\rho_{\mathsf{u}}(h,k)$ and therefore chooses the same branch.

The strict branches are apart from the same branch. If a strict-left witness and
the carrier classifier both held, the classifier exactness of
\autoref{thm:poset-prefix-classifier-exactness} would give the reverse prefix
comparison as well; splitting that reverse continuation into empty or nonempty
form either collapses the strict tail to $\emp$ or gives opposing strict
prefixes, both ruled out by \autoref{thm:nat-unary-strict-prefix-asymm}. The
strict-right case is symmetric.

The displayed equivalence for the same branch is exactly the prefix classifier
exactness theorem \autoref{thm:poset-prefix-classifier-exactness} after
unfolding $\mathsf{same}_{\mathsf{u}}$. Relation stability, reflexivity, and
transitivity are inherited from
\autoref{thm:concrete-unary-prefix-preorder-laws}, while antisymmetry is the
same exactness field of \autoref{thm:poset-prefix-classifier-exactness}.

For strict-comparison transitivity, use
\autoref{thm:totalorder-strict-branch-transitivity}. It applies the concrete
strict unary-prefix transitivity field to the strict-left branch and the same
argument with reversed endpoints to the strict-right branch. These fields repack
into the concrete $\TotalOrderUp$ law package.
\end{proof}

\begin{theorem}[Concrete unary prefix total-order comparison partition]
\label{thm:concrete-unary-prefix-totalorder-comparison-partition}
For carried unary histories $h,k$ in
$\mathsf{Carrier}_{\mathsf{u}}$, exactly one of the three concrete branches
$$
\mathsf{NatUnaryStrictPrefix}(h,k),
\qquad
\rho_{\mathsf{u}}(h,k),
\qquad
\mathsf{NatUnaryStrictPrefix}(k,h)
$$
supplies the comparison branch. The middle branch is exact:
$$
\rho_{\mathsf{u}}(h,k)\Longleftrightarrow
(\mathsf{PreorderPrefixLE}(h,k)\land\mathsf{PreorderPrefixLE}(k,h)).
$$
The strict-left and strict-right branches cannot both hold, and neither strict
branch is compatible with $\rho_{\mathsf{u}}(h,k)$.

The per-object fields are: traditional target constructive total-order branch
partition; BEDC source $\mathsf{UnaryHistory}$, $\mathsf{PreorderPrefixLE}$, and
$\mathsf{NatUnaryStrictPrefix}$ over $\mathsf{BHist}$; carrier
$\mathsf{Carrier}_{\mathsf{u}}$; classifier $\rho_{\mathsf{u}}$; stability
obligations branch totality and incompatibility; ledger selected comparison
branch; core theorem this partition; exactness the same-branch equivalence;
standard bridge deferred; status $\mathsf{ProofTarget}$.
\end{theorem}
\begin{proof}
Use the total unary prefix comparison of
\autoref{thm:nat-unary-prefix-total}. For carried histories $h$ and $k$, it gives
a unary continuation in one direction, and the unary tail is then split into
empty and nonempty cases.

If the selected tail is nonempty from $h$ to $k$, the selected branch is
$\mathsf{NatUnaryStrictPrefix}(h,k)$. If the selected tail is nonempty from $k$
to $h$, the selected branch is $\mathsf{NatUnaryStrictPrefix}(k,h)$.

If the selected tail is empty, the continuation collapses to the same-history
case. The carrier classifier is exactly the same-history branch, and
\autoref{thm:poset-prefix-classifier-exactness} identifies it with the pair of
prefix comparisons in both directions.

The two strict branches are incompatible by
\autoref{thm:nat-unary-strict-prefix-asymm}. A strict branch together with
$\rho_{\mathsf{u}}(h,k)$ would, by
\autoref{thm:poset-prefix-classifier-exactness}, also provide the opposite
prefix comparison; splitting that opposite comparison yields either an empty
tail contradicting strictness or the opposing strict branch, again ruled out by
\autoref{thm:nat-unary-strict-prefix-asymm}. The strict-right case is symmetric.

These cases give total coverage, the exact middle branch, and pairwise
incompatibility, so the three branches form the concrete constructive
partition.
\end{proof}

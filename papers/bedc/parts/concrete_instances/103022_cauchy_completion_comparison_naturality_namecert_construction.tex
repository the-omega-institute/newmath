\label{def:cauchy-completion-comparison-naturality-carrier}

\item $U$ is the $\CauchyCompletionUnitUp$ route row for source admission into the completion-facing presentation.
\item $V$ is the $\CauchyCompletionCounitUp$ route row for returning an already completed presentation to its displayed completion consumer.
\item $F$ is the $\CompletionReflectionUp$ witness row, recording that both the unit-facing and counit-facing reads are attached to the same reflection boundary.

The classifier compares accepted packets only through $Q,U,V,F,R,S,D,E,H,C,P,N$. It has no coordinate for quotient completion equality, selected representatives, host function equality, countable choice, ambient $\RealUp$ completeness, Lean verification, or bridge checking.

\begin{theorem}[Cauchy completion comparison naturality obligations]
\label{thm:cauchy-completion-comparison-naturality-namecert-obligations}
For every accepted $\CauchyCompletionComparisonNaturalityUp$ carrier
$$
M=(Q,U,V,F,R,S,D,E,H,C,P,N),
$$
the local $\NameCert_{\CauchyCompletionComparisonNaturalityUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier admission for the completion-comparison map, unit route, counit route, reflection witness, StreamName window, DyadicRatCore tolerance, RegSeqRat readback, Real seal, transport, replay, provenance, and local naming row;
\item comparison-map exposure, requiring $Q$ to display the two completion presentations before the unit square or counit square is inspected;
\item unit-square routing, requiring the source-side read to pass through $U$ and the same $Q,F,S,D,R,E$ rows before any Real-facing seal is public;
\item counit-square routing, requiring the completed-side read to pass through $V$ and the same $Q,F,S,D,R,E$ rows before any completion-facing consumer receives a seal;
\item reflection stability, requiring $F$ to be the only witness row tying the unit and counit reads to the shared completion boundary;
\item structural exhaustion, requiring $H,C,P,N$ to transport, replay, source, and name only the listed rows.
\end{enumerate}
No obligation is stored outside these rows, and none exports quotient completion equality, selected representatives, host function equality, countable choice, ambient $\RealUp$ completeness, Lean verification, or bridge checking.
\end{theorem}
\begin{proof}
Carrier admission is projection from \autoref{def:cauchy-completion-comparison-naturality-carrier}. The accepted tuple displays exactly the completion-comparison map, unit route, counit route, reflection witness, analytic readback rows, and structural rows.

Comparison-map exposure is read from $Q$. Since $Q$ occurs before the unit and counit routes are inspected, both naturality checks compare the same two completion presentations.

The unit-square obligation is read from $U$. A source-side consumer first enters through the completion unit and can reach the Real seal only by passing through the displayed comparison, reflection, window, tolerance, and readback rows.

The counit-square obligation is read from $V$. A completed-side consumer returns through the counit route and is compared against the same $Q,F,S,D,R,E$ route rather than against a separate representative of a completion.

Reflection stability is supplied by $F$, while $H,C,P,N$ add only componentwise transport, replay, provenance, and local naming. Since the carrier contains no coordinate for quotient equality, selected representatives, host equality, choice, ambient completeness, Lean verification, or bridge checking, the listed rows exhaust the local NameCert surface.
\end{proof}

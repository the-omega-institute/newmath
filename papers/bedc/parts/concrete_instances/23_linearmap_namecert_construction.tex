\chapter{A Concrete Naming Certificate for \(\LinearMapUp\)}
\label{ch:concrete-instances-linearmap-namecert}

The linear map interface \(\LinearMapUp\) has certificate obligations between two vector spaces over the same scalar field from \autoref{ch:concrete-instances-vecspace-namecert}. The carrier is the predicate "\(f : V \to W\) preserves the additive structure and commutes with scalar multiplication"; the classifier is pointwise carrier sameness on the codomain restricted to maps satisfying linearity. It is not a primitive function space; it is admitted only as a derived predicate over function pairs.

\section{Carrier definition}

\begin{definition}[LinearMap carrier]
\label{def:linearmap-carrier}
Use a structure containing a function \(f:V\to W\) plus explicit proofs of additivity and scalar compatibility.
\end{definition}

\section{Concrete singleton empty-history linear map instance}
\label{sec:linearmap-concrete-singleton-instance}

\begin{definition}[Concrete singleton empty-history linear map instance]
\label{def:singleton-empty-history-linearmap-instance}
Let the scalar carrier, source vector carrier, and target vector carrier all be
the singleton history source $A_{\emp}$ of
Definition~\ref{def:list-singleton-history-source} at $a_0=\emp$. Use the
shared endpoint classifier $\rho_{\emp}$ from
Definition~\ref{def:singleton-empty-history-vecspace-instance}. Let the public
map names be $\mathsf{BHist}$ codes and define the map-token carrier by
$$
L_{\emp}(f)\Longleftrightarrow A_{\emp}(f).
$$
The map classifier is
$$
\lambda_{\emp}(f,g):=
L_{\emp}(f)\land L_{\emp}(g)\land\hsame(f,g).
$$
The pointwise evaluation is
$$
\mathsf{eval}_{\emp}(f,x):=\emp.
$$
Equivalently, use the
constant singleton graph
$$
\mathsf{Eval}_{\emp}(f,v,w):=
L_{\emp}(f)\land A_{\emp}(v)\land A_{\emp}(w)\land\hsame(w,\emp),
$$
with the same chosen endpoint evaluation. Set the identity map code, zero-map
code, and composite map code to
$$
\mathsf{id}_{\emp}:=\emp,\qquad
0_L:=\emp,\qquad
\mathsf{comp}_{\emp}(g,f):=\emp.
$$
The certificate fit has ten fields:
\begin{enumerate}
  \item Traditional target: the unique linear map between singleton vector
  spaces.
  \item BEDC source: $\mathsf{BHist}$.
  \item Carrier choice: the map-token carrier $L_{\emp}$.
  \item Classifier choice: pointwise $\rho_{\emp}$ through
  $\mathsf{eval}_{\emp}$.
  \item Stability obligations: zero preservation, additivity, scalar
  compatibility, identity, and composition.
  \item Ledger: the map token together with the singleton evaluation endpoint.
  \item Core theorem list:
  Theorem~\ref{thm:singleton-empty-history-linearmap-laws}.
  \item Exactness predicate: singleton endpoint classification.
  \item Standard bridge: the one-map hom-set between singleton vector spaces.
  \item Status: $\mathsf{ProofTarget}$.
\end{enumerate}
\end{definition}

\begin{theorem}[Singleton empty-history linear map laws]
\label{thm:singleton-empty-history-linearmap-laws}
For the singleton vector spaces of
Definition~\ref{def:singleton-empty-history-vecspace-instance}, the data
$$
(A_{\emp},\rho_{\emp},L_{\emp},\lambda_{\emp},
\mathsf{eval}_{\emp},\mathsf{id}_{\emp},0_L,\mathsf{comp}_{\emp})
$$
satisfy the concrete $\LinearMapUp$ law package:
\begin{enumerate}
  \item $L_{\emp}(\emp)$, so the linear-map carrier is inhabited, and $0_L$ is
  carried.
  \item For all map codes $f,g$,
  $$
  \lambda_{\emp}(f,g)\Longleftrightarrow
  L_{\emp}(f)\land L_{\emp}(g)\land\hsame(f,g).
  $$
  \item Evaluation respects map and vector classification:
  $\lambda_{\emp}(f,g)$ and $\rho_{\emp}(v,w)$ imply
  $\rho_{\emp}(\mathsf{eval}_{\emp}(f,v),\mathsf{eval}_{\emp}(g,w))$.
  \item Each carried map preserves zero, vector addition, and scalar
  multiplication under $\rho_{\emp}$.
  \item $\mathsf{id}_{\emp}$ is carried, and
  $\mathsf{comp}_{\emp}(g,f)$ is carried whenever $f$ and $g$ are carried.
  \item Left identity, right identity, and associativity of composition hold
  under $\lambda_{\emp}$.
  \item The pointwise classifier induced by $\rho_{\emp}$ is reflexive and
  transitive on admitted map tokens.
\end{enumerate}
\end{theorem}
\begin{proof}
The domain and codomain vector-space fields are exactly the singleton
empty-history fields of
Theorem~\ref{thm:singleton-empty-history-vecspace-laws}. Thus every scalar
code, vector code, vector operation endpoint, and scalar-action endpoint is
classified at the singleton endpoint $\emp$.

Carrier inhabitance follows from the singleton carrier witness
$A_{\emp}(\emp)$. The same witness carries $0_L$, $\mathsf{id}_{\emp}$, and
every composite endpoint, since all three codes are definitionally $\emp$.
The classifier statement is the displayed definition of $\lambda_{\emp}$, with
symmetry and transitivity of $\hsame$ available for the usual classifier
symmetry and transitivity clauses.

For evaluation respect, both chosen endpoints
$\mathsf{eval}_{\emp}(f,v)$ and $\mathsf{eval}_{\emp}(g,w)$ reduce to $\emp$.
The singleton carrier witnesses for $\emp$ and reflexivity of $\hsame$ assemble
the required $\rho_{\emp}$ witness, independent of the particular carried map
and vector names.

For zero preservation, both the source zero endpoint evaluated by the map and
the target zero endpoint reduce to $\emp$. The vector-space theorem supplies
the singleton zero endpoint on the displayed carrier, and $\hsame(\emp,\emp)$
supplies the comparison.

For additivity, the source side
$\mathsf{eval}_{\emp}(f,\mathsf{add}(v,w))$ and the target side
$\mathsf{add}(\mathsf{eval}_{\emp}(f,v),\mathsf{eval}_{\emp}(f,w))$ both
reduce to the singleton endpoint. The vector-space theorem supplies closure of the
displayed addition endpoint, and $\hsame$ reflexivity supplies the classifier
component.

For scalar compatibility,
$\mathsf{eval}_{\emp}(f,\mathsf{smul}(r,v))$ and
$\mathsf{smul}(r,\mathsf{eval}_{\emp}(f,v))$ again reduce to $\emp$. Closure and
classification are the singleton vector-space closure witness and
$\hsame(\emp,\emp)$.

The identity code and every composite code are definitionally $\emp$, so
$L_{\emp}(\mathsf{id}_{\emp})$ and
$L_{\emp}(\mathsf{comp}_{\emp}(g,f))$ follow from the singleton carrier
witness $A_{\emp}(\emp)$. The left and right identity comparisons reduce to
comparing carried map codes with $\emp$; the carrier witnesses $L_{\emp}(f)$ or
$L_{\emp}(g)$ provide the needed endpoint links, using symmetry when the
displayed direction is reversed.

For associativity, both composite codes
$\mathsf{comp}_{\emp}(h,\mathsf{comp}_{\emp}(g,f))$ and
$\mathsf{comp}_{\emp}(\mathsf{comp}_{\emp}(h,g),f)$ are $\emp$. The two carrier
witnesses and reflexivity of $\hsame$ give the final
$\lambda_{\emp}$ comparison.

The pointwise classifier laws are endpoint laws. Reflexivity compares
$\mathsf{eval}_{\emp}(f,x)$ with itself, and transitivity composes two
comparisons between endpoints that are all $\emp$. The required
$\rho_{\emp}$ witnesses are therefore the singleton endpoint classification
witnesses already used by the vector-space package.
\end{proof}

The $\LinearMapUp$ schema is instantiated by this concrete $\mathsf{BHist}$
carrier and map-token evaluation, so the Lean formalization can target
$\mathsf{BEDC.Derived.LinearMapUp}$ rather than an abstract function-space
echo.

\section{The certificate}

\begin{definition}[LinearMap source specification]
\label{def:linearmap-source-specification}
The source is a pair of vector spaces over the same certified field.
\end{definition}

\begin{definition}[LinearMap pattern specification]
\label{def:linearmap-pattern-specification}
The pattern is function-with-law formation: map data together with linearity witnesses.
\end{definition}

\begin{definition}[LinearMap classifier specification]
\label{def:linearmap-classifier-specification}
The classifier relates two linear maps when their outputs are codomain-classified the same for every input witness.
\end{definition}

\begin{definition}[LinearMap stability certificate]
\label{def:linearmap-stability-certificate}
The stability certificate consists of:
\begin{enumerate}
  \item pointwise classifier reflexivity
  \item pointwise classifier transitivity
  \item preservation of zero derived from linearity
  \item additivity preservation
  \item scalar-compatibility preservation
  \item closure under identity and composition
\end{enumerate}
\end{definition}

\begin{definition}[LinearMap ledger policy]
\label{def:linearmap-ledger-policy}
Record the function identifier, its linearity witnesses, and codomain ledgers for pointwise comparisons.
\end{definition}

\begin{definition}[\(\LinearMapUp\) certificate obligations]
\label{def:linearmap-certificate-obligations}
The carrier, source, pattern, classifier, stability, and ledger data listed above are the obligations required for the interface name \(\LinearMapUp\).
\end{definition}

\paragraph{Primitive boundary.}
\(\LinearMapUp\) is not a BEDC primitive. It is admitted only when those obligations are supplied.

\begin{remark}[Zero-axiom encoding note]
\label{rem:linearmap-zero-axiom-note}
Avoids funext by using pointwise classifier sameness instead of equality of functions.
\end{remark}

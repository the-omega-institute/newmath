\chapter{A Concrete Naming Certificate for \(\LinearMapUp\)}
\label{ch:concrete-instances-linearmap-namecert}

The linear map interface \(\LinearMapUp\) has certificate obligations between two vector spaces over the same scalar field from \autoref{ch:concrete-instances-vecspace-namecert}. The carrier is the predicate "\(f : V \to W\) preserves the additive structure and commutes with scalar multiplication"; the classifier is pointwise carrier sameness on the codomain restricted to maps satisfying linearity. It is not a primitive function space; it is admitted only as a derived predicate over function pairs.

\section{Carrier definition}

\begin{definition}[LinearMap carrier]
\label{def:linearmap-carrier}
Use a structure containing a function \(f:V\to W\) plus explicit proofs of additivity and scalar compatibility.
\end{definition}

\section{Concrete singleton empty-history linear map instance}
\label{sec:linearmap-concrete-singleton-instance}

\begin{definition}[Concrete singleton empty-history linear map instance]
\label{def:singleton-empty-history-linearmap-instance}
Use the singleton empty-history vector space of
Definition~\ref{def:singleton-empty-history-vecspace-instance} as both source
and target. Its scalar, source-vector, and target-vector carriers are the
singleton history source $A_{\emp}$ of
Definition~\ref{def:list-singleton-history-source} at $a_0=\emp$. The map-code
carrier is the same existing singleton predicate
$$
A_{\emp}:\mathsf{BHist}\to\mathsf{Prop},
$$
and the map-code classifier is the existing singleton classifier
$\rho_{\emp}$. The realized action is the constant endpoint action
$$
\Lambda_{\emp}(f,x):=\emp
$$
for every admitted map code $f$ and singleton vector input $x$. The same
endpoint action may be displayed as
$$
F_{\emp}(h):=\emp.
$$
The linear-map carrier on public map codes and its pointwise map classifier
are
$$
L_{\emp}(f):=\hsame(f,\emp),
\qquad
\lambda_{\emp}(f,g):=
L_{\emp}(f)\land L_{\emp}(g)\land\hsame(f,g).
$$
The pointwise evaluation is
$$
\mathsf{ev}_{\emp}(f,h):=F_{\emp}(h).
$$
The identity, zero-map, and composition code are
$$
\mathsf{id}_{\emp}:=\emp,\qquad
\mathsf{zeroMap}_{\emp}:=\emp,\qquad
\mathsf{comp}_{\emp}(g,f):=\emp.
$$

This certificate has ten fields:
\begin{enumerate}
  \item Traditional target: the terminal one-map linear map space between the
  singleton vector space and itself.
  \item BEDC source: $\mathsf{BHist}$.
  \item Carrier choice: $A_{\emp}$ for source vectors, target vectors, and
  map codes.
  \item Classifier choice: $\rho_{\emp}$ for endpoint and map-code equality.
  \item Stability obligations: output closure, pointwise classifier
  congruence, additivity, scalar compatibility, zero preservation, identity
  closure, and composition closure.
  \item Ledger: singleton map-code provenance for $F_{\emp}$,
  $\mathsf{id}_{\emp}$, $\mathsf{zeroMap}_{\emp}$, and
  $\mathsf{comp}_{\emp}$.
  \item Core theorem list: Theorem~\ref{thm:singleton-empty-history-linearmap-laws}.
  \item Exactness predicate: singleton endpoint classification.
  \item Standard bridge: the terminal one-map linear hom space.
  \item Status: $\mathsf{ProofTarget}$.
\end{enumerate}
\end{definition}

\begin{theorem}[Singleton empty-history linear map laws]
\label{thm:singleton-empty-history-linearmap-laws}
The tuple
$$
(A_{\emp},\rho_{\emp},\Lambda_{\emp},\emp,\mathsf{comp}_{\emp})
$$
satisfies the $\LinearMapUp$ schema over the singleton vector space, where the
displayed empty-history code supplies the identity and zero-map slots.
\end{theorem}
\begin{proof}
The source and target vector-space laws are those of
Theorem~\ref{thm:singleton-empty-history-vecspace-laws}. They supply the
singleton scalar laws, vector closure, additive laws, scalar-action closure,
and scalar-action congruence for the same $A_{\emp}$ carrier and
$\rho_{\emp}$ classifier.

Output closure is $A_{\emp}(\emp)$, since
$\Lambda_{\emp}(f,x)$ is definitionally $\emp$. Pointwise classifier
congruence is immediate because
$\Lambda_{\emp}(f,x)$ and $\Lambda_{\emp}(g,y)$ are both $\emp$, so the
required endpoint comparison is the singleton $\rho_{\emp}$ witness.

For additivity,
$\Lambda_{\emp}(f,\mathsf{add}(x,y))$ and
$\mathsf{add}(\Lambda_{\emp}(f,x),\Lambda_{\emp}(f,y))$ are both classified
with $\emp$. The vector-space theorem supplies closure for the displayed
addition endpoint, and the singleton classifier supplies the endpoint
comparison.

For scalar compatibility,
$\Lambda_{\emp}(f,\mathsf{smul}(r,x))$ and
$\mathsf{smul}(r,\Lambda_{\emp}(f,x))$ are again classified with $\emp$.
Scalar-action closure and congruence come from the singleton vector-space
laws, so no additional map-specific datum is needed.

Zero preservation compares $\Lambda_{\emp}(f,0)$ with the singleton vector
zero. Both sides are $\emp$, and the required classification is the same
$\rho_{\emp}$ endpoint witness.

The identity and zero-map codes are $\emp$, and their action on any carried
input $x$ reduces to $\emp$. The input carrier witness $A_{\emp}(x)$ together
with $\rho_{\emp}$ gives the required singleton endpoint classification.

For composition closure, every composite map code and every realized output
reduce to $\emp$. The carrier witness for $\emp$ and reflexivity inside
$\rho_{\emp}$ close the composition law and the comparison of the two
evaluation orders.
\end{proof}
\leanchecked{BEDC.Derived.LinearMapUp.LinearMapSingleton\_empty\_history\_laws}

\leanchecked{BEDC.Derived.LinearMapUp.LinearMapSingleton\_laws}

\begin{theorem}[Singleton empty-history linear map public-code exactness]
\label{thm:singleton-empty-history-linearmap-public-code-exactness}
For the concrete instance of
Definition~\ref{def:singleton-empty-history-linearmap-instance}, for all
public map codes $f,g$ and vector code $x$, if $L_{\emp}(f)$,
$L_{\emp}(g)$, and $A_{\emp}(x)$, then
$$
\lambda_{\emp}(f,\mathsf{id}_{\emp})\land
\lambda_{\emp}(\mathsf{zeroMap}_{\emp},\mathsf{id}_{\emp})\land
\lambda_{\emp}(\mathsf{comp}_{\emp}(g,f),\mathsf{id}_{\emp})\land
\rho_{\emp}(\mathsf{ev}_{\emp}(f,x),x).
$$
Its per-object fit is: traditional target terminal one-map hom exactness;
BEDC source $\mathsf{BHist}$ over the singleton vector-space instance;
carrier $A_{\emp}$ for vectors and map codes; classifier $\rho_{\emp}$ and
$\lambda_{\emp}$; stability obligations public-code exactness, evaluation
exactness, and composition collapse; ledger singleton map-code provenance;
core theorem this block; exactness predicate the displayed classifier
collapse; standard bridge terminal linear hom; status $\mathsf{ProofTarget}$.
\end{theorem}
\begin{proof}
The three public map-code endpoints
$\mathsf{id}_{\emp}$, $\mathsf{zeroMap}_{\emp}$, and
$\mathsf{comp}_{\emp}(g,f)$ reduce to $\emp$. The hypotheses
$L_{\emp}(f)$ and $L_{\emp}(g)$ give the nontrivial singleton witnesses for
the two displayed input map codes.

The classifier component of each map-code comparison is
$\hsame(\emp,\emp)$. Since both advertised comparison partners reduce to the
empty history, the reflexive singleton classifier closes the identity,
zero-map, and composite-code clauses.

For evaluation, $\mathsf{ev}_{\emp}(f,x)$ is $F_{\emp}(x)$ and therefore
reduces to $\emp$. The carrier hypothesis $A_{\emp}(x)$ supplies the endpoint
witness for the input vector, and $\rho_{\emp}$ supplies the reflexive
comparison between the realized endpoint and $x$ in the singleton carrier.
\end{proof}
\leanchecked{BEDC.Derived.LinearMapUp.LinearMapSingleton\_public\_empty\_code\_exactness}

\begin{theorem}[Singleton linear-map classifier append split empty]
\label{thm:linearmap-singleton-classifier-append-split-empty}
For all histories $p,q,h:\mathsf{BHist}$,
$$
\lambda_{\emp}(\mathsf{append}(p,q),h)
$$
holds exactly when $L_{\emp}(p)$, $L_{\emp}(q)$, and $L_{\emp}(h)$ all hold.
\end{theorem}
\begin{proof}
The forward direction reads the first carrier field of
$\lambda_{\emp}(\mathsf{append}(p,q),h)$ as
$\hsame(\mathsf{append}(p,q),\emp)$. Empty-result inversion for BEDC append
splits this into $p\hsame\emp$ and $q\hsame\emp$, while the second classifier
field gives $h\hsame\emp$.

Conversely, the three singleton carrier witnesses show that
$\mathsf{append}(p,q)$ is empty and that $h$ is empty. These are the two
carrier fields of $\lambda_{\emp}$, and composing the empty endpoint
comparison with the symmetry of the $h$-carrier gives the classifier
comparison between $\mathsf{append}(p,q)$ and $h$.
\end{proof}
\leanchecked{BEDC.Derived.LinearMapUp.LinearMapSingletonClassifier\_append\_split\_empty\_iff}

\begin{theorem}[Singleton linear map composition associativity empty classifier]
\label{thm:linearmap-singleton-comp-assoc-empty-classifier}
For singleton linear map codes $f,g,h$, if each code lies in
$L_{\emp}$, then both associations of the composite code classify with
$\emp$, and the two associated composite codes classify with each other:
$$
\lambda_{\emp}(\mathsf{comp}_{\emp}(h,\mathsf{comp}_{\emp}(g,f)),\emp)
\land
\lambda_{\emp}(\mathsf{comp}_{\emp}(\mathsf{comp}_{\emp}(h,g),f),\emp)
\land
\lambda_{\emp}(\mathsf{comp}_{\emp}(h,\mathsf{comp}_{\emp}(g,f)),
  \mathsf{comp}_{\emp}(\mathsf{comp}_{\emp}(h,g),f)).
$$
\end{theorem}
\begin{proof}
The singleton composition code is definitionally $\emp$ for every pair of
inputs. Therefore both associated composites reduce to $\emp$, and each
displayed classifier is the reflexive singleton map-code classifier at the
empty history.
\end{proof}
\leanchecked{BEDC.Derived.LinearMapUp.LinearMapSingleton\_comp\_assoc\_empty\_classifier}

\begin{theorem}[Singleton linear-map classifier append left cancellation]
\label{thm:singleton-linearmap-classifier-append-left-cancel}
For all $P,Q,R:\mathsf{BHist}$, if $L_{\emp}(P)$, then
$$
\lambda_{\emp}(\mathsf{append}(P,Q),\mathsf{append}(P,R))
\Longleftrightarrow
\lambda_{\emp}(Q,R).
$$
\end{theorem}
\begin{proof}
For the forward direction, unfold the singleton classifier. The carrier
clauses say that both appended histories are classified with $\emp$.
Applying the empty-append inversion to each carrier clause gives
$Q\hsame\emp$ and $R\hsame\emp$. These two endpoint witnesses compose, with
symmetry on the right one, to give $Q\hsame R$.

For the reverse direction, the hypothesis $L_{\emp}(P)$ identifies the left
prefix with $\emp$. The two appended histories therefore reduce through the
left-empty continuation law to $Q$ and $R$. The carrier clauses and classifier
comparison supplied by $\lambda_{\emp}(Q,R)$ transport along those reductions
to rebuild the prefixed singleton classifier.
\end{proof}
\leanchecked{BEDC.Derived.LinearMapUp.LinearMapSingletonClassifier\_append\_left\_cancel\_iff}

The $\LinearMapUp$ schema is instantiated by this concrete $\mathsf{BHist}$
carrier and singleton action, so later Lean work can target BHist-anchored
linear-map laws instead of only abstract function fields.

\section{The certificate}

\begin{definition}[LinearMap source specification]
\label{def:linearmap-source-specification}
The source is a pair of vector spaces over the same certified field.
\end{definition}

\begin{definition}[LinearMap pattern specification]
\label{def:linearmap-pattern-specification}
The pattern is function-with-law formation: map data together with linearity witnesses.
\end{definition}

\begin{definition}[LinearMap classifier specification]
\label{def:linearmap-classifier-specification}
The classifier relates two linear maps when their outputs are codomain-classified the same for every input witness.
\end{definition}

\begin{definition}[LinearMap stability certificate]
\label{def:linearmap-stability-certificate}
The stability certificate consists of:
\begin{enumerate}
  \item pointwise classifier reflexivity
  \item pointwise classifier transitivity
  \item preservation of zero derived from linearity
  \item additivity preservation
  \item scalar-compatibility preservation
  \item closure under identity and composition
\end{enumerate}
\end{definition}

\begin{definition}[LinearMap ledger policy]
\label{def:linearmap-ledger-policy}
Record the function identifier, its linearity witnesses, and codomain ledgers for pointwise comparisons.
\end{definition}

\begin{definition}[\(\LinearMapUp\) certificate obligations]
\label{def:linearmap-certificate-obligations}
The carrier, source, pattern, classifier, stability, and ledger data listed above are the obligations required for the interface name \(\LinearMapUp\).
\end{definition}

\paragraph{Primitive boundary.}
\(\LinearMapUp\) is not a BEDC primitive. It is admitted only when those obligations are supplied.

\begin{remark}[Zero-axiom encoding note]
\label{rem:linearmap-zero-axiom-note}
Avoids funext by using pointwise classifier sameness instead of equality of functions.
\end{remark}

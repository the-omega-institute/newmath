\begin{theorem}[CauchyFilterCompletion window cofinality]
\label{thm:cauchy-filter-completion-window-cofinality}
Let $F=(C,W,T,R,L,H,K,P,N)$ be an accepted $\CauchyFilterCompletionUp$ carrier. Every completion-facing tolerance request that reaches the terminal $\RealUp$ seal has a displayed cofinal window route through the same finite rows $C,W,T,R,L,H,K,P,N$: the request first reads the $\CauchyFilterUp$ source row $C$, then a $\StreamNameUp$ window in $W$, then the $\DyadicRatCoreUp$ tolerance ledger $T$, then the $\RegSeqRatUp$ readback row $R$, and only then the seal row $L$ with the structural rows $H,K,P,N$.
\end{theorem}
\begin{proof}
Start with the carrier of \autoref{def:cauchy-filter-completion-carrier}. It displays $C$ as the only Cauchy-filter source and $W$ as the only finite observation-window family.

Apply \autoref{thm:cauchy-filter-completion-window-tolerance-totality}. This forces every accepted readback to pass through $W$ and $T$ before $R$ is available.

Finally use the Real seal handoff of \autoref{thm:cauchy-filter-completion-real-seal-handoff}. The terminal seal remains downstream of $C,W,T,R$ and upstream of $H,K,P,N$, so every completion-facing tolerance request has the displayed cofinal route.
\end{proof}

\begin{theorem}[CauchyFilterCompletion source exactness]
\label{thm:cauchy-filter-completion-source-exactness}
Let $F=(C,W,T,R,L,H,K,P,N)$ be an accepted $\CauchyFilterCompletionUp$ carrier. The source of any accepted completion read is exactly the displayed L10 route
$$
\CauchyFilterUp \longmapsto \StreamNameUp \longmapsto \DyadicRatCoreUp \longmapsto \RegSeqRatUp \longmapsto \RealUp
$$
carried by $C,W,T,R,L$ and preserved by the $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows $H,K,P,N$. No read can replace this source by a quotient filter, selected limit, detached stream, free tolerance, host real equality, countable-choice selector, or ambient completeness witness.
\end{theorem}
\begin{proof}
Project $F$ and apply \autoref{thm:cauchy-filter-completion-window-cofinality}. The cofinal window route already identifies the source side as $C,W,T,R$ before the terminal seal $L$ is read.

Use \autoref{thm:cauchy-filter-completion-filter-base-admission} to pin $C$ to the displayed $\CauchyFilterUp/\FilterBaseUp$ source and \autoref{thm:cauchy-filter-completion-real-seal-uniqueness-route} to pin every terminal read to $L$.

The structural rows then preserve the same route. Componentwise $\hsame$ transport is stored in $H$, displayed $\Cont$ replay is stored in $K$, and $P,N$ source and name only this packet. Since the carrier has no row for the excluded objects, the accepted source is exact.
\end{proof}

\begin{theorem}[CauchyFilterCompletion ledger exhaustion]
\label{thm:cauchy-filter-completion-ledger-exhaustion}
For an accepted $\CauchyFilterCompletionUp$ carrier $F=(C,W,T,R,L,H,K,P,N)$, every completion, filter-limit, metric-completion, or uniform-completion consumer read is exhausted by the displayed ledger rows $C,W,T,R,L,H,K,P,N$. In particular, the L10 source route through $\CauchyFilterUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$ is the whole public source ledger; there is no additional quotient-filter row, selected-limit row, detached completion object, host equality row, countable-choice row, or ambient-completeness row available to any consumer.
\end{theorem}
\begin{proof}
Begin with source exactness from \autoref{thm:cauchy-filter-completion-source-exactness}. It confines every accepted source read to $C,W,T,R,L,H,K,P,N$.

For filter-limit consumers, use \autoref{thm:cauchy-filter-completion-filter-convergence-handoff}. For metric-facing and uniform-facing consumers, use \autoref{thm:cauchy-filter-completion-metric-uniform-compatibility} and \autoref{thm:cauchy-filter-completion-completion-handoff}. These theorems all reuse the same displayed route rather than adding a second consumer-specific source.

Finally apply \autoref{thm:cauchy-filter-completion-non-choice-boundary}. The refused quotient, selector, choice, host-equality, and completeness objects have no carrier coordinate. Thus the ledger is exhausted by the displayed L10 source route and the structural rows of $F$.
\end{proof}

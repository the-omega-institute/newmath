\chapter{A Concrete Naming Certificate for $\DyadicSubdivisionUp$}
\label{ch:concrete-instances-dyadicsubdivision-namecert}
\origin{human}

The $\DyadicSubdivisionUp$ interface names a finite subdivision certificate for one rational interval into dyadic mesh cells with validated enclosure rows attached. In the Moore--Kearfott--Cloud interval-analysis reading, subdivision refines an enclosure into finitely many checked sub-enclosures; in BEDC the object is only a finite $\mathsf{BHist}$ ledger over $\DyadicRatCoreUp$, $\RationalIntervalUp$, $\DyadicMeshUp$, $\ValidatedNumericsUp$, $\RealUp$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$.

The unary bisection route used below is read through the sibling \autoref{ch:concrete-instances-dyadicbisection-namecert} before being repacked as an ordered subdivision ledger. $\DyadicSubdivisionUp$ consumes $\DyadicBisectionUp$ only as a finite route source; it does not import an infinite refinement tree.

\begin{definition}[DyadicSubdivision source]
\label{def:dyadic-subdivision-source}
A $\DyadicSubdivisionUp$ source is a finite $\mathsf{BHist}$ packet
$$
S=(I,L,C,M,V,P,N)
$$
with the following visible rows:
\begin{enumerate}
\item $I$ is the parent $\RationalIntervalUp$ row whose endpoint and order data are the interval being subdivided;
\item $L$ is the unary dyadic level row used to read every child cell at one displayed mesh precision;
\item $C$ is a finite ordered cell-list row, with each entry carrying its displayed position inside the parent interval;
\item $M$ is the finite family of $\DyadicMeshUp$ cell packets indexed by $C$;
\item $V$ is the finite family of $\ValidatedNumericsUp$ containment rows attached to the same cells;
\item $P$ is the $\Pkg$ provenance row naming $\DyadicRatCoreUp$, $\RationalIntervalUp$, $\DyadicMeshUp$, $\ValidatedNumericsUp$, and $\RealUp$;
\item $N$ is the local $\NameCert_{\DyadicSubdivisionUp}$ row.
\end{enumerate}
The classifier compares accepted subdivision packets by transporting the parent interval, level row, ordered cell list, mesh cells, containment rows, package provenance, and local name row componentwise. The carrier has no coordinate for a host interval tree, infinite refinement process, solver state, or completed real subset.
\end{definition}

\item exactness: every child boundary and containment read is one of the $\DyadicMeshUp$ or $\ValidatedNumericsUp$ rows indexed by the ordered list $C$;

\begin{theorem}[DyadicSubdivision refinement stability]
\label{thm:dyadic-subdivision-refinement-stability}

\end{theorem}
\leanchecked{BEDC.Derived.DyadicSubdivisionUp.DyadicSubdivisionPacket\_refinement\_stability}

For each entry of the ordered list $C$, use the $\DyadicMeshUp$ refinement route to replace the displayed cell by a finer finite family. The endpoint comparison is still through the transported $\RationalIntervalUp$ boundary rows, so no host interval or completed real subset is introduced.

Transport the validated containment rows through the same cell routes. Repacking the parent row, refined level data, refined ordered cell ledger, refined mesh entries, transported containment entries, provenance, and local certificate gives exactly the carrier of \autoref{def:dyadic-subdivision-source}.
\end{proof}

Each child coverage object is then projected from the matching $\DyadicMeshUp$ entry in $M$. Its rational interval and dyadic endpoint ledger are already the finite coverage surface supplied by the mesh certificate.

\begin{theorem}[DyadicSubdivision refinement composition]
\label{thm:dyadic-subdivision-refinement-composition}
Let $S=(I,L,C,M,V,P,N)$ be an accepted $\DyadicSubdivisionUp$ packet. Suppose $S'$ is obtained from $S$ by displayed finite $\DyadicMeshUp$ refinement routes over the same parent $\RationalIntervalUp$ row $I$, and suppose $S''$ is obtained from $S'$ by a second displayed finite family of $\DyadicMeshUp$ refinement routes over the same parent row. Then $S''$ is an accepted $\DyadicSubdivisionUp$ packet for $I$.

The mesh coverage and validated enclosure handoff exported by $S''$ are the composed finite ledger: first the refined child rows from $S$ to $S'$, then the refined child rows from $S'$ to $S''$, together with the transported $\ValidatedNumericsUp$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows. \end{theorem}
\leanchecked{BEDC.Derived.DyadicSubdivisionUp.DyadicSubdivisionPacket\_refinement\_composition}
\begin{proof}

Apply refinement stability again to the second displayed refinement family. Since its routes are also over the same parent row, the second application yields an accepted packet $S''$ whose parent interval and provenance surface remain the displayed rows of the original packet.

It remains to identify the consumer surface. Mesh coverage reads only the finite ordered cell list and the $\DyadicMeshUp$ packets; validated enclosure handoff reads the matching $\ValidatedNumericsUp$ rows along the same routes.

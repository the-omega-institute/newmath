\chapter{A Concrete Naming Certificate for $\CompletionIdempotenceUp$}
\label{ch:concrete-instances-completion-idempotence-namecert}
\origin{human}

Following Bishop and Bridges constructive analysis, the $\CompletionIdempotenceUp$ packet records the finite reflector surface by which a displayed completion request over an already completed name is read back through the same completion boundary. It sits over the Cauchy completion functor of \autoref{ch:concrete-instances-cauchycompletionfunctor-namecert}, the metric completion surface of \autoref{ch:concrete-instances-metriccompletion-namecert}, the separated completion seal of \autoref{ch:concrete-instances-separatedcompletion-namecert}, and the uniform completion route of \autoref{ch:concrete-instances-uniformcompletion-namecert}. The packet is a reuse certificate for completed names; it does not construct a second quotient object, select representatives, assert host equality, invoke choice, or claim an ambient completeness theorem.

\begin{definition}[Completion idempotence carrier]
\label{def:completion-idempotence-carrier}
A $\CompletionIdempotenceUp$ carrier is a finite $\BHist$ packet
$$
I=(F,R,Q,W,D,G,A,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $F$ is the first completion seal, naming the completed boundary already available to downstream consumers.
\item $R$ is the second completion request row, recording that a consumer asks to complete the already sealed boundary.
\item $Q$ is the Cauchy readback row, requiring every second request to read the same finite Cauchy evidence used by $F$.
\item $W$ is the $\StreamNameUp$ window row exposing the finite windows through which the repeated completion request is observed.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger tying the first seal and the second request to the same dyadic comparison budget.
\item $G$ is the $\RegSeqRatUp$ handoff row, keeping regular rational readback attached to the displayed Cauchy data.
\item $A$ is the classifier transport row, recording componentwise $\hsame$ movement for the first seal, second request, Cauchy readback, window, tolerance, and handoff rows.
\item $C$ is the displayed $\Cont$ replay row from the second completion request through $R,Q,W,D,G,A$ back to the first completion seal.
\item $P$ is the $\Pkg$ provenance over $\CompletionIdempotenceUp$, $\CauchyCompletionFunctorUp$, $\MetricCompletionUp$, $\SeparatedCompletionUp$, $\UniformCompletionUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CompletionIdempotenceUp}$ row.
\end{enumerate}
The carrier contains no coordinate for a second completion object, quotient class, selected representative, host equality, choice principle, detached Cauchy witness, or unrecorded completeness assertion.
\end{definition}

\begin{theorem}[CompletionIdempotence NameCert obligations]
\label{thm:completion-idempotence-namecert-obligations}
For every accepted $\CompletionIdempotenceUp$ carrier $I=(F,R,Q,W,D,G,A,C,P,N)$, the local $\NameCert_{\CompletionIdempotenceUp}$ surface consists exactly of these finite obligation rows:
\begin{enumerate}
\item carrier admission for the first completion seal, second completion request, Cauchy readback, StreamName windows, DyadicRatCore tolerance ledger, RegSeqRat handoff, classifier transport, continuation replay, provenance, and local naming row;
\item first-seal visibility, requiring $F$ to be read before the second request is admitted;
\item repeated-request exactness, requiring $R$ and $Q$ to reuse the Cauchy data already attached to the first seal;
\item window and tolerance stability, requiring $W,D,G$ to keep finite observation windows, dyadic tolerances, and regular rational handoff on the same displayed route;
\item classifier and replay closure, requiring $A,C,P,N$ to transport, replay, source, and name only the displayed rows;
\item nonescape, excluding a second completion object, quotient class, selected representative, host equality, choice principle, detached Cauchy witness, and unrecorded completeness assertion.
\end{enumerate}
No completion idempotence obligation is stored outside these rows.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:completion-idempotence-carrier}. The only rows available to the local certificate are $F,R,Q,W,D,G,A,C,P,N$.

The first-seal and repeated-request obligations follow by reading $F$ before $R$. The Cauchy readback row $Q$ attaches the second request to the same displayed evidence rather than to a fresh completion object.

The window, tolerance, and handoff obligations are carried by $W,D,G$. They keep the observed StreamName windows, DyadicRatCore tolerances, and RegSeqRat readback on the route already named by the first seal.

The structural rows $A,C,P,N$ transport, replay, source, and name the displayed packet. Since every excluded object is absent from the tuple, the certificate cannot export a second completion object, quotient class, selected representative, host equality, choice principle, detached Cauchy witness, or unrecorded completeness assertion.
\end{proof}

\begin{theorem}[Completion idempotence stable reflection]
\label{thm:completion-idempotence-stable-reflection}
Let $I=(F,R,Q,W,D,G,A,C,P,N)$ be an accepted $\CompletionIdempotenceUp$ carrier. Every downstream completion consumer that reads the second request factors through the finite route
$$
F \longmapsto R,Q \longmapsto W,D,G \longmapsto A,C,P,N .
$$
Thus a completed name can be reused through its first completion seal, Cauchy readback, finite StreamName windows, DyadicRatCore tolerance ledger, RegSeqRat handoff, classifier transport, continuation replay, provenance, and local NameCert row, without constructing or naming a second completion object.
\end{theorem}
\begin{proof}
Start from an accepted repeated-completion read. Carrier admission exposes the first seal $F$ and the second request $R$ as rows of the same finite packet.

The Cauchy readback row $Q$ ties the second request to the evidence already carried by $F$. The rows $W,D,G$ then preserve finite observation windows, dyadic tolerances, and regular rational handoff along that same route.

Finally the classifier transport $A$, continuation replay $C$, provenance $P$, and local naming row $N$ package the route for downstream consumers. None of these rows adds a second completion object, so the reflection is stable by exhaustion of the displayed carrier.
\end{proof}

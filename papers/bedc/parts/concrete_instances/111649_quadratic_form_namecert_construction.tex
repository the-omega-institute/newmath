\chapter{A Concrete Naming Certificate for $\QuadraticFormUp$}
\label{ch:concrete-instances-quadratic-form-namecert}
\origin{human}

Following Bourbaki and Serre on quadratic forms, the $\QuadraticFormUp$ packet records a scalar-valued quadratic law on a displayed module or vector-space carrier as finite BEDC NameCert data. It sits between the scalar field of \autoref{ch:concrete-instances-field-namecert}, the module and vector-space source rows of \autoref{ch:concrete-instances-module-namecert} and \autoref{ch:concrete-instances-vecspace-namecert}, the bilinear-form pairing surface of \autoref{ch:concrete-instances-bilinform-namecert}, and the real-facing scalar seal of \autoref{ch:concrete-instances-real-namecert}. Downstream consumers such as \autoref{ch:concrete-instances-sylvester-inertia-namecert}, \autoref{ch:concrete-instances-clifford-namecert}, and \autoref{ch:concrete-instances-semidefinitecone-namecert} may read the same finite quadratic packet instead of redeclaring diagonal scalar data.

\begin{definition}[Quadratic form carrier]
\label{def:quadratic-form-carrier}
A $\QuadraticFormUp$ carrier is a finite $\BHist$ packet
$$
Q=(F,V,B,q,H,P,T,C,S,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $F$ is the scalar $\FieldUp$ source row, with the $\RealUp$ seal recorded when the scalar endpoint is real-valued.
\item $V$ is the $\ModuleUp$ or $\VecSpaceUp$ carrier row whose displayed addition, zero, inverse, and scalar-action rows are the only vector endpoints admitted by the packet.
\item $B$ is the $\BilinFormUp$ source row that supplies the visible polarization candidate used by quadratic consumers.
\item $q$ is the scalar endpoint row assigning each accepted vector endpoint a carried scalar value in the source named by $F$.
\item $H$ records the homogeneity ledger $q(a\cdot x)=a^{2}q(x)$ over displayed scalar-action rows and scalar-square rows.
\item $P$ records the polarization handoff row, reading the cross-term endpoint through the bilinear source $B$ and the displayed scalar expression determined by $q(x+y)$, $q(x)$, and $q(y)$.
\item $T$ records componentwise $\hsame$ transport for scalar field data, module or vector-space data, bilinear-form source rows, quadratic endpoint rows, homogeneity rows, polarization rows, continuation, provenance, and local naming.
\item $C$ records displayed $\Cont$ replay from the source carrier through the quadratic endpoint, homogeneity ledger, polarization handoff, and local certificate.
\item $S$ is the $\Pkg$ provenance over $\QuadraticFormUp$, $\FieldUp$, $\ModuleUp$, $\VecSpaceUp$, $\BilinFormUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\QuadraticFormUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $F,V,B,q,H,P,T,C,S,N$. It has no coordinate for a host function on an ambient vector space, quotient equality of quadratic maps, a selected basis, diagonalization, inertia, positivity, Clifford quotient data, or bridge checking.
\end{definition}

\begin{theorem}[Quadratic form polarization boundary]
\label{thm:quadratic-form-polarization-boundary}
For every accepted $\QuadraticFormUp$ carrier $Q=(F,V,B,q,H,P,T,C,S,N)$, the polarization row $P$ is the only public route from scalar quadratic values to a bilinear-form read. A consumer may inspect the displayed cross-term package through the $\BilinFormUp$ source row $B$ and the scalar endpoints named by $q$, but it receives no host bilinear map, no quotient equality of forms, no selected coordinate matrix, and no diagonal basis.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:quadratic-form-carrier}. The only row that mentions both the quadratic endpoint $q$ and a bilinear output is the polarization handoff $P$.

The handoff first reads the admitted vector endpoints through $V$ and their scalar outputs through $q$. It then compares the cross-term endpoint with the bilinear source $B$ inside the same scalar field row $F$, so the comparison is a displayed BEDC package rather than an external equality of functions.

The excluded objects are absent from the carrier. Since no row stores a host bilinear map, quotient equality of forms, selected coordinates, or a diagonal basis, the polarization read cannot export them.
\end{proof}

\begin{theorem}[Quadratic form NameCert obligations]
\label{thm:quadratic-form-namecert-obligations}
For every accepted $\QuadraticFormUp$ carrier $Q=(F,V,B,q,H,P,T,C,S,N)$, the local $\NameCert_{\QuadraticFormUp}$ surface consists of carrier admission for the scalar field, module or vector-space source, bilinear-form source, quadratic endpoint, homogeneity ledger, polarization handoff, componentwise $\hsame$ transport, displayed $\Cont$ replay, $\Pkg$ provenance, and local naming. The exactness boundary says that every public quadratic-form read is generated by these rows and that every downstream consumer must factor through the displayed source certificates before reading scalar quadratic data.
\end{theorem}
\begin{proof}
Read the carrier rows in the displayed order. The scalar field row $F$, vector or module carrier row $V$, bilinear source $B$, and scalar endpoint row $q$ determine the admitted quadratic surface.

The homogeneity obligation is the ledger $H$. It uses only the scalar-action rows from $V$ and the scalar-square rows from $F$, so it remains inside the displayed module or vector-space source.

The polarization obligation is the row $P$ and is bounded by \autoref{thm:quadratic-form-polarization-boundary}. Finally $T,C,S,N$ supply transport, replay, provenance, and local naming. These projections exhaust the local NameCert surface, so no hidden positivity, inertia, diagonalization, Clifford quotient, host function equality, or bridge row is available.
\end{proof}

\begin{theorem}[Quadratic form bilinear-form handoff]
\label{thm:quadratic-form-bilinear-form-handoff}
Every accepted $\QuadraticFormUp$ packet exposes its bilinear-form interface through the route
$$
\BilinFormUp,\ModuleUp,\VecSpaceUp,\FieldUp,\RealUp,\BHist,\hsame,\Cont,\Pkg,\NameCert .
$$
The route sends diagonal reads to the scalar endpoint $q$ and cross-term reads to the polarization row $P$. Consequently $\SylvesterInertiaUp$, $\CliffordUp$, and $\SemidefiniteConeUp$ consumers may cite the same quadratic packet when they need scalar diagonal data, bilinear polarization data, and source-carrier transport.
\end{theorem}
\begin{proof}
Start from a public consumer read. By \autoref{def:quadratic-form-carrier}, diagonal scalar data can be read only from $q$, and cross-term data can be read only from $P$ together with the bilinear source $B$.

The route first fixes the scalar source $F$ and the vector or module source $V$. It then reads $B$ for the bilinear-form side and reads $q,H,P$ for the quadratic endpoint, homogeneity, and polarization rows.

Apply the NameCert obligation surface of \autoref{thm:quadratic-form-namecert-obligations}. The consumer receives exactly the finite rows needed by the inertia, Clifford, and semidefinite-cone chapters: scalar diagonal values, bilinear handoff, carrier transport, continuation replay, provenance, and local naming. Since selected bases, diagonalization, positivity decisions, Clifford quotients, and optimization cones are not carrier coordinates, they remain obligations of the downstream chapters rather than hidden exports of $\QuadraticFormUp$.
\end{proof}

\begin{theorem}[ContinuationMonad root formal target surface]
\label{thm:continuation-monad-root-formal-target-surface}
For an accepted $\ContinuationMonadUp$ packet $Z=(A,B,C,f,g,u,H,K,L,N)$, the formal target surface available at the root consists exactly of the finite rows $A,B,C,f,g,u,H,K,L,N$ together with their displayed $\BHist$, $\Cont$, $\hsame$, $\Pkg$, $\CategoryUp$ hom-carrier, $\mathsf{ContinuationMorphism}$, and local $\NameCert$ readings. Downstream formal targets may consume unit, bind, generator-refusal, and Category hom data only through that surface, not through quotient Kleisli arrows, host-function composition, ambient category completion, or unlisted continuation generators.
\end{theorem}
\leanchecked{BEDC.Derived.ContinuationMonadUp.ContinuationMonadCarrier\_root\_formal\_target\_surface}
\begin{proof}
Use the root unit-bind exhaustion theorem to account for the unit and bind reads. It restricts those reads to the displayed source histories, continuation tails, unit tail, route rows, ledger rows, transport row, provenance, and local naming row.

Use the root generator-category lock for the generator-refusal and Category hom faces. It ties those faces to the same $A,B,C,f,g,u,H,K,L,N$ route and prevents a separate quotient or ambient category object from entering the surface.

Combine the two readings with the root downstream unblock package. The package is already the complete downstream surface, so the formal target surface is the same finite row list with no additional Kleisli, host-composition, completion, or generator coordinates.
\end{proof}

\begin{theorem}[ContinuationMonad root route formal handoff]
\label{thm:continuation-monad-root-route-formal-handoff}

\end{theorem}
\leanchecked{BEDC.Derived.ContinuationMonadUp.ContinuationMonadCarrier\_root\_route\_formal\_handoff}
\begin{proof}
First apply root unit-bind exhaustion. It makes the public unit and bind entries read $u,f,g,K,L,H,N$ over the source histories $A,B,C$ rather than a host monad operation.

Next apply the generator-category lock. The same finite route carries generator refusal and $\CategoryUp$ hom readings, so a downstream target cannot split the generator ledger from the Category hom row.

Now use bind readback exhaustion. It states that every bind readback is already exhausted by the displayed packet rows and cannot be supplied by a quotient Kleisli arrow, host function, unlisted generator, or external evaluator.

The root downstream unblock package then repacks these rows as the formal handoff. Since all consumer-visible data has already been routed through the finite packet, the refused host and quotient readings have no handoff coordinate.
\end{proof}

\begin{theorem}[ContinuationMonad root formal downstream coverage]
\label{thm:continuation-monad-root-formal-downstream-coverage}
For an accepted $\ContinuationMonadUp$ packet $Z=(A,B,C,f,g,u,H,K,L,N)$, every accepted local consumer of the root formal target is exhausted by the finite downstream coverage surface
$$
A,B,C,f,g,u,H,K,L,N.
$$
The coverage surface consists of the displayed $\BHist$ source histories, $\Cont$ continuation tails, componentwise $\hsame$ transports, $\Pkg$ provenance, the $\CategoryUp$ hom-carrier row, the ContinuationMorphism row, the local $\NameCert$ row, and the ledger and readback rows named by the root formal route. Unit and bind consumers factor through the unit-bind and bind-readback rows, generator and Category consumers factor through the generator-category rows, and local naming or handoff consumers factor through the root formal handoff rows. Quotient Kleisli arrows, host-function composition, ambient category completion, external monad evaluators, and unlisted continuation-generator rules remain outside the packet.
\end{theorem}
\begin{proof}
Start with \autoref{thm:continuation-monad-root-route-formal-handoff}. It states that every downstream formal target reading the root route receives $A,B,C,f,g,u,H,K,L,N$ through root unit-bind exhaustion, generator-category lock, the root downstream unblock package, and bind readback exhaustion. Thus the accepted local consumers are already sorted into the unit-bind face, the generator-Category face, and the bind-readback or handoff face.

Apply \autoref{thm:continuation-monad-root-formal-route-saturation}. The saturation theorem identifies the exported formal route with the same finite rows and lists the only readings exported by that route: $\BHist$, $\Cont$, $\hsame$, $\Pkg$, the $\CategoryUp$ hom-carrier, the ContinuationMorphism row, and local $\NameCert$. Therefore any consumer-facing use of the formal target must factor through one of those displayed rows.

It remains to check that no local consumer can enter through an accounting side door. The ledger side is closed by \autoref{thm:continuation-monad-root-ledger-saturation-boundary}, which records no extra entry beyond $A,B,C,f,g,u,H,K,L,N$. The coverage side is closed by \autoref{thm:continuation-monad-root-formal-audit-factorization} and \autoref{thm:continuation-monad-root-formal-route-budget-exhaustion}, which route audit, naming, Category, unit, bind, and generator-facing reads through the same packet and exhaust the route budget. Finally \autoref{thm:continuation-monad-root-formal-namecert-consumer-lock} transfers that exhaustion to local formal consumers. Since none of these rows contains a coordinate for quotient Kleisli arrows, host-function composition, ambient category completion, external monad evaluators, or unlisted continuation-generator rules, all excluded alternatives remain outside the BEDC packet.
\end{proof}

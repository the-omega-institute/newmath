\label{def:gap-closure-boundary-hub-root-inventory}

\begin{theorem}[GapClosureBoundary hub root readiness]
\label{thm:gap-closure-boundary-hub-root-readiness}

\end{theorem}
\begin{proof}
Use the carrier obligations of \autoref{thm:gap-closure-boundary-namecert-obligations} to fix the shared packet $G,S,R,H,C,P,N$ before any listed hard-problem consumer reads the boundary.

Apply source-route totality and refusal-route exactness, \autoref{thm:gap-closure-boundary-source-route-totality} and \autoref{thm:gap-closure-boundary-refusal-route-exactness}. They give the two displayed branches $G \longrightarrow S$ and $G \longrightarrow R$ over the same structural rows.

Finally apply the root-unblock and consumer-exactness packages, \autoref{thm:gap-closure-boundary-root-unblock-package} and \autoref{thm:gap-closure-boundary-consumer-exactness-package}. These packages identify the five listed consumers as readers of the same finite source/refusal surface and exclude the hidden rows named in the statement.
\end{proof}

\begin{theorem}[GapClosureBoundary hub unblock surface]
\label{thm:gap-closure-boundary-hub-unblock-surface}
Every downstream consumer named by the $\GapClosureBoundaryUp$ root package reads only the displayed source/refusal split through $G,S,R,H,C,P,N$. A supported read passes through $G \longrightarrow S$ with $H,C,P,N$, an unsupported closure request passes through $G \longrightarrow R$ with $H,C,P,N$, and no downstream consumer receives a third route outside the hub root inventory.
\end{theorem}
\begin{proof}
Begin with the hub root inventory of \autoref{def:gap-closure-boundary-hub-root-inventory}. It names the carrier, source route, refusal route, structural rows, and consumer-exactness row used by the hub.

For supported reads, use \autoref{thm:gap-closure-boundary-source-route-totality}. For unsupported closure requests, use \autoref{thm:gap-closure-boundary-refusal-route-exactness}. Both routes are expressed through the same coordinates $G,S,R,H,C,P,N$.

The consumer-exactness package of \autoref{thm:gap-closure-boundary-consumer-exactness-package} then excludes any off-inventory route. Hence the hub exposes exactly the displayed split and no additional downstream surface.
\end{proof}

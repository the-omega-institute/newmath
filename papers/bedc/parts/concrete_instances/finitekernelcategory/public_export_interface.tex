\begin{theorem}[Finite-kernel category public export interface]
\label{thm:finite-kernel-category-public-export-interface}
Let
$$
K=(O,M,I,Q,A,U,H,C,P,N)
$$
be an accepted $\FiniteKernelCategoryUp$ carrier. The public export of $K$ is
exactly the displayed row family $O,M,I,Q,A,U,H,C,P,N$. It is read first as the
$\CategoryUp$ interface carried by the finite kernel package, and then as the
required source boundary for every downstream $\FunctorUp$ and $\NatTransUp$
consumer. No quotient morphism, host arrow equality, functor-category object,
or external category library is part of that public export.
\end{theorem}
\begin{proof}
Start with the category interface handoff,
\autoref{thm:finite-kernel-category-category-interface-handoff}. It identifies
the displayed rows $O,M,I,Q,A,U,H,C,P,N$ as the $\CategoryUp$ interface of the
accepted finite-kernel carrier.

Next apply the category, functor, and natural-transformation row readback,
\autoref{thm:finite-kernel-category-category-functor-nattrans-row-readback}.
It makes the same row family the source boundary read by $\FunctorUp$ and
$\NatTransUp$ consumers after the category interface has been exposed.

The downstream scope is fixed by
\autoref{thm:finite-kernel-category-nattrans-consumer-scope}. It permits those
consumers to read only the complete displayed package together with the
existing TasteGate marker.

Finally apply the bridge consumer nonescape boundary,
\autoref{thm:finite-kernel-category-bridge-consumer-nonescape}. Since bridge
consumers cannot extract quotient morphisms, host arrow equality,
functor-category objects, or an external category library, the displayed row
family is the whole public export.
\end{proof}

\chapter{A Concrete Naming Certificate for $\DenseEmbeddingUp$}
\label{ch:concrete-instances-dense-embedding-namecert}
\origin{human}

Following Kelley and Bourbaki on dense embeddings, the $\DenseEmbeddingUp$ packet records the finite BEDC surface by which a source metric carrier is read inside a target metric completion through a dense image-window and an approximation ledger. It sits before the specialized metric-completion embedding packet of \autoref{ch:concrete-instances-metric-completion-dense-embedding-namecert}, and it supplies a canonical source route for $\RealUp$, $\RegSeqRatUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, and $\CauchyCompletenessWitnessUp$ consumers in \autoref{ch:concrete-instances-real-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, \autoref{ch:concrete-instances-streamname-namecert}, \autoref{ch:concrete-instances-dyadicratcore-namecert}, and \autoref{ch:concrete-instances-cauchy-completeness-witness-namecert}. The packet names dense-image approximation as finite certificate data; it does not import a host embedding theorem, quotient representatives, selected limits, countable choice, or ambient completion completeness.

\begin{definition}[Dense embedding carrier]\label{def:dense-embedding-carrier}
A $\DenseEmbeddingUp$ carrier is a finite $\BHist$ packet
$$
D=(S,T,C,I,W,A,E,H,K,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $S$ is the source metric row, including the finite carrier, distance readings, and source-side comparison windows.
\item $T$ is the target metric row, read only through displayed target-distance comparisons.
\item $C$ is the target completion row, carrying the completion-facing names used by the embedding read.
\item $I$ is the source-to-target image row. It records the embedded image of inspected source points without choosing quotient representatives or host limits.
\item $W$ is the dense image-window row. Each density request opens a finite target window, a source witness window, and the target-distance tolerance being tested.
\item $A$ is the approximation ledger. It records that the witness in $S$ reaches the target request through $I,W,T,C$ at the displayed tolerance.
\item $E$ is the completion handoff row exposed to completion-facing consumers after $S,T,C,I,W,A$ have already been read.
\item $H$ stores componentwise $\hsame$ transport for the source metric, target metric, completion, image, dense window, approximation ledger, handoff, continuation, provenance, and local naming rows.
\item $K$ stores displayed $\Cont$ replay from source observations to image observations, from image observations to dense windows, and from dense windows to the completion handoff row.
\item $P$ is the $\Pkg$ provenance over $\DenseEmbeddingUp$, $\MetricCompletionDenseEmbeddingUp$, $\RealUp$, $\RegSeqRatUp$, $\StreamNameUp$, $\DyadicRatCoreUp$, $\CauchyCompletenessWitnessUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\DenseEmbeddingUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through $S,T,C,I,W,A,E,H,K,P,N$. It has no coordinate for host topological density, quotient completion equality, a selected limit, a global inverse, countable choice, or ambient completeness of the target completion.
\end{definition}

\begin{theorem}[Dense embedding NameCert obligations]\label{thm:dense-embedding-namecert-obligations}
For every accepted $\DenseEmbeddingUp$ carrier $D=(S,T,C,I,W,A,E,H,K,P,N)$, the local $\NameCert_{\DenseEmbeddingUp}$ surface consists exactly of carrier admission for the source metric, target metric, target completion, source-to-target image, dense image-window, approximation ledger, completion handoff, componentwise $\hsame$ transport, displayed $\Cont$ replay, $\Pkg$ provenance, and local naming rows. Every density read must first expose $S,T,C,I,W,A$ before the handoff row $E$ can be consumed.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:dense-embedding-carrier}. The source metric, target metric, completion row, image row, dense window, and approximation ledger are exactly $S,T,C,I,W,A$, so a density request has no other analytic coordinates to inspect.

The handoff row $E$ is downstream of those six rows. Hence completion-facing consumers first see the displayed source witness, its image, the finite target window, and the approximation ledger before receiving any completion data.

The structural rows $H,K,P,N$ only transport, replay, source, and name that same finite route. Since the carrier excludes host density, quotient representatives, selected limits, countable choice, global inverses, and ambient completeness, the listed rows exhaust the local NameCert surface.
\end{proof}

\begin{theorem}[Dense embedding completion handoff]\label{thm:dense-embedding-completion-handoff}
Every $\MetricCompletionDenseEmbeddingUp$, $\RealUp$, or $\CauchyCompletenessWitnessUp$ consumer of an accepted $\DenseEmbeddingUp$ packet factors through the finite route
$$
S,T,C,I\longmapsto W\longmapsto A\longmapsto E
$$
together with $H,K,P,N$. The route may be read through the $\RegSeqRatUp$, $\StreamNameUp$, and $\DyadicRatCoreUp$ windows named by the packet, but it exports no selected completion limit, quotient representative, ambient density theorem, countable-choice selector, or host completeness principle.
\end{theorem}
\begin{proof}
Start with the NameCert obligations of \autoref{thm:dense-embedding-namecert-obligations}. They place the source metric row, target metric row, target completion row, and image row before any dense-window or approximation-ledger read is available.

The dense image-window $W$ opens only the finite source and target observations requested by the consumer. The approximation ledger $A$ then records that this displayed source witness reaches the requested target window through the image row $I$ at the stated tolerance.

Only after that ledger is read does the completion handoff row $E$ become available. The rows $H,K,P,N$ preserve and name the same route, so specialized completion, real, and Cauchy-completeness consumers receive finite dense-embedding data rather than host completion structure.
\end{proof}

\begin{theorem}[CauchySequenceAlgebra NameCert obligations]
\label{thm:cauchy-sequence-algebra-namecert-obligations}

$$

$$
the local $\NameCert_{\CauchySequenceAlgebraUp}$ surface consists exactly of the following finite obligation rows:
\begin{enumerate}
\item carrier exposure for the two $\RegSeqRatUp$ sources, synchronized $\StreamNameUp$ windows, $\DyadicRatCoreUp$ operation budgets, $\RegularCauchyAdditionUp$ route, $\RegularCauchyProductUp$ route, $\RealUp$ seal, transport, replay, provenance, and local naming;
\item source-window synchronization, requiring every operation request to inspect $X$ and $Y$ through the same displayed window row $W$ before any endpoint arithmetic is read;
\item dyadic-budget exactness, requiring every addition or product endpoint row to consume only the displayed tolerance, slack, comparison, and magnitude-budget entries of $D$;
\item operation-route separation, requiring $A$ to carry only the addition route and $M$ to carry only the product route, while both routes consume the same source, window, and dyadic-budget rows;
\item Real seal handoff, requiring $S$ to be exposed only after the selected operation route has produced its $\RegSeqRatUp$ readback;

\end{theorem}
\begin{proof}

Source-window synchronization is forced by the single window coordinate $W$. Both source reads must pass through that coordinate before the dyadic operation budget is available, so the carrier has no second window service for either input.

Dyadic-budget exactness and operation-route separation are read from $D,A,M$.

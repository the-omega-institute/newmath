\label{thm:cauchy-remainder-estimate-namecert-obligations}

$$

\item carrier admission for the source, dyadic tolerance ledger, finite threshold, admitted windows, rational readback, boundary row, transport, replay, provenance, and local naming rows;
\item tolerance exposure, requiring each remainder request to read its bound and slack data from $D$ before any tail observation is opened;
\item threshold exactness, requiring $T$ to be the only finite index gate through which $W$ may expose tail observations;
\item readback alignment, requiring $R$ to compare only the observations admitted by $W$ against the same tolerance request from $D$;

The request then reads $T,W,R$ in order. The threshold row supplies the finite tail gate, the window row exposes only observations admitted by that gate, and the readback row compares those observations against the same dyadic tolerance data.

Finally, $B$ determines whether the estimate is consumed by a Real seal route or a Specker refusal route, while $H,C,P,N$ transport, replay, source, and name the same finite data. Since the excluded principles do not occur as carrier coordinates, the listed rows exhaust the local NameCert surface.

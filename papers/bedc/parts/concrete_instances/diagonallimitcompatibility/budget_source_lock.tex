\begin{proof}

Finally use \autoref{thm:diagonal-limit-compatibility-seal-source-synchronization}. The seal route is synchronized with the same source rows, so no $\CauchyLimitSealUp$ read can occur before those budget rows have been exposed.

The $\CauchyLimitSealUp$ read and the $\CauchySealBudgetSynchronizerUp$ handoff occur only after that same prefix, so the handoff is a route through the displayed budget rows rather than through a separate selected limit or quotient stream.

Finally use \autoref{thm:diagonal-limit-compatibility-synchronized-budget-terminality}.

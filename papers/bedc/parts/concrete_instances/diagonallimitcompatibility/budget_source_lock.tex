\begin{theorem}[Diagonal-limit compatibility budget source lock]
\label{thm:diagonal-limit-compatibility-budget-source-lock}
Every accepted $\DiagonalLimitCompatibilityUp$ packet reads its $\CauchyDiagonalBudgetUp$, $\RealDiagonalMeshBudgetUp$, $\RegularCauchyTriangleBoundUp$, and $\StreamNameUp$ rows before any seal route is available. The first seal-facing consumer must therefore pass through the displayed budget source, mesh budget, triangle bound, and stream-window rows of the finite compatibility packet.
\end{theorem}
\leanchecked{BEDC.Derived.DiagonallimitcompatibilityUp.DiagonalLimitCompatibilityBudgetSourceLock}
\begin{proof}
Start with the carrier inventory in \autoref{def:diagonal-limit-compatibility-carrier}. The packet names the diagonal source, triangle estimate, stream window, regular readback, Real seal row, transport rows, continuation routes, package provenance, and local certificate in one finite surface.

Apply \autoref{thm:diagonal-limit-compatibility-source-ledger}, \autoref{thm:diagonal-limit-compatibility-root-dependency-triad}, and \autoref{thm:diagonal-limit-compatibility-root-budget-source}. These anchors identify the budget source with the displayed diagonal, mesh, triangle, and stream rows rather than with a hidden analytic input.

Finally use \autoref{thm:diagonal-limit-compatibility-seal-source-synchronization}. The seal route is synchronized with the same source rows, so no $\CauchyLimitSealUp$ read can occur before those budget rows have been exposed.
\end{proof}

\begin{theorem}[Diagonal-limit compatibility seal-budget route]
\label{thm:diagonal-limit-compatibility-seal-budget-route}
The $\CauchyLimitSealUp$ and $\CauchySealBudgetSynchronizerUp$ handoff exported by an accepted $\DiagonalLimitCompatibilityUp$ packet is a displayed finite route through the budget rows of \autoref{thm:diagonal-limit-compatibility-budget-source-lock}. \end{theorem}
\leanchecked{BEDC.Derived.DiagonallimitcompatibilityUp.DiagonalLimitCompatibilitySealBudgetRoute}
\begin{proof}
The source lock fixes the budget-facing prefix. By \autoref{thm:diagonal-limit-compatibility-budget-source-lock}, the route must first read the diagonal budget, mesh budget, triangle bound, and stream-window rows.

Now apply \autoref{thm:diagonal-limit-compatibility-seal-route-compatibility}, \autoref{thm:diagonal-limit-compatibility-root-seal-boundary}, and \autoref{thm:diagonal-limit-compatibility-seal-budget-synchronizer-handoff}. The non-escape clause is \autoref{thm:diagonal-limit-compatibility-real-seal-budget-nonescape}. Since the packet carries no selected limit, quotient stream, host equality coordinate, or ambient completeness input, the displayed seal-budget route exhausts the available handoff.
\end{proof}

Apply \autoref{thm:diagonal-limit-compatibility-completion-consumer-nonescape} and \autoref{thm:diagonal-limit-compatibility-root-consumer-nonescape}; together they exhaust the consumer surface by the displayed packet rows, with no quotient stream, selected limit, host equality, or ambient completeness coordinate.

\begin{theorem}[Diagonal-limit compatibility budget-source readback totality]
\label{thm:diagonal-limit-compatibility-budget-source-readback-totality}
Every $\RegSeqRatUp$, $\RealUp$, or $\CompletionUp$ consumer of an accepted $\DiagonalLimitCompatibilityUp$ packet first reads the displayed $\CauchyDiagonalBudgetUp$, $\RealDiagonalMeshBudgetUp$, $\RegularCauchyTriangleBoundUp$, and $\StreamNameUp$ rows. The consumer surface has no alternate route through quotient streams, selected limits, host equality, or ambient completeness.
\end{theorem}
\begin{proof}
The budget-source lock \autoref{thm:diagonal-limit-compatibility-budget-source-lock} fixes the first visible prefix of the packet. Thus any consumer that reaches the regular readback, Real row, or completion row has already read the displayed diagonal budget, mesh budget, triangle bound, and stream-window rows.

Next use the seal-budget route of \autoref{thm:diagonal-limit-compatibility-seal-budget-route}. It places the $\CauchyLimitSealUp$ read and the $\CauchySealBudgetSynchronizerUp$ handoff downstream of that same finite budget prefix, so the seal-facing part of the consumer cannot bypass the displayed source rows.

Finally apply \autoref{thm:diagonal-limit-compatibility-completion-consumer-gate}. Its consumer gate covers the $\RealUp$ and $\CompletionUp$ exits, and the regular readback row is one of the displayed rows feeding those exits. The non-escape clauses exclude quotient streams, selected limits, host equality, and ambient completeness, so the budget-source readback prefix is total for all listed consumers.
\end{proof}

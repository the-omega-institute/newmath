\begin{theorem}[DiagonalLimitCompatibility root selector budget lock]
\label{thm:diagonal-limit-compatibility-root-selector-budget-lock}
Let
$$
K=(R,T,S,D,W,Q,L,H,C,P,N)
$$
be an accepted $\DiagonalLimitCompatibilityUp$ packet. Every accepted root-selector packet routes the displayed source budget through the selector row, the seal-synchronizer handoff, the $\BHist$ inventory, the componentwise $\hsame$ transports, the displayed $\Cont$ routes, the $\Pkg$ provenance row, and the local $\NameCert$ row before any diagonal-limit read reaches $D,W,Q,L$. No selector read can bypass that route by a host limit, quotient stream, host real equality, chosen completion point, or ambient completeness principle.
\end{theorem}
\begin{proof}
Project the accepted packet through \autoref{def:diagonal-limit-compatibility-carrier}. The carrier exposes the diagonal source, triangle and selector-facing budget rows, dyadic ledger, stream window, regular readback, Real seal, transport rows, continuation routes, package provenance, and local name as one finite $\BHist$ surface.

Apply \autoref{thm:diagonal-limit-compatibility-selector-budget-source-lock}. It identifies the selector-budget source with the displayed diagonal route, window row, dyadic ledger, regular readback, Real seal, and structural rows $H,C,P,N$.

Next use \autoref{thm:diagonal-limit-compatibility-seal-synchronizer-budget-handoff}. The synchronizer is read downstream of the same selector-budget prefix, so the root selector cannot obtain an independent seal source before the diagonal-limit read.

Finally apply \autoref{thm:diagonal-limit-compatibility-root-route-budget-package} and \autoref{thm:diagonal-limit-compatibility-root-route-nonescape}. They place the diagonal-limit consumer on the displayed $\BHist/\hsame/\Cont/\Pkg/\NameCert$ package and remove every route through host limits, quotient streams, host equality, chosen completion points, or ambient completeness.
\end{proof}

\begin{theorem}[DiagonalLimitCompatibility seal budget route exhaustion]
\label{thm:diagonal-limit-compatibility-seal-budget-route-exhaustion}
Let
$$
K=(R,T,S,D,W,Q,L,H,C,P,N)
$$
be an accepted $\DiagonalLimitCompatibilityUp$ packet. Every $\RealUp$ or $\CompletionUp$ seal consumer that reads the selector-budget route factors through the same public route
$$
R,T,S,D,W,Q,L,H,C,P,N .
$$
The consumer receives the source budget, selector handoff, seal synchronizer, dyadic ledger, stream window, regular readback, Real seal, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows, and it cannot export quotient streams, selected limits, host equality, or ambient completeness.
\end{theorem}
\begin{proof}
Start with \autoref{thm:diagonal-limit-compatibility-root-selector-budget-lock}. It fixes the selector-budget route before a diagonal-limit or seal-facing consumer can inspect the readback rows.

Use \autoref{thm:diagonal-limit-compatibility-completion-consumer-budget-nonescape}. Every completion-facing or Real-facing consumer that uses the selector-budget route is exhausted by $R,T,S,D,W,Q,L,H,C,P,N$.

Now apply \autoref{thm:diagonal-limit-compatibility-real-seal-budget-nonescape}. The inserted synchronizer and tail-schedule route remain part of the same finite compatibility surface, so no second seal route is available.

The exclusion clause follows from \autoref{thm:diagonal-limit-compatibility-root-route-consumer-exactness} and \autoref{thm:diagonal-limit-compatibility-root-route-nonescape}. The public consumer surface contains only the listed packet rows and has no coordinate for quotient streams, selected limits, host equality, or ambient completeness.
\end{proof}

\begin{theorem}[DiagonalLimitCompatibility selector-lock seal route exhaustion]
\label{thm:diagonal-limit-compatibility-selector-lock-seal-route-exhaustion}
Let
$$
K=(R,T,S,D,W,Q,L,H,C,P,N)
$$
be an accepted $\DiagonalLimitCompatibilityUp$ packet. The route exports no quotient stream, selected host limit, host real equality, independent seal object, countable-choice row, or ambient completeness principle.
\end{theorem}
\begin{proof}
First combine \autoref{thm:diagonal-limit-compatibility-root-selector-budget-lock} with \autoref{thm:diagonal-limit-compatibility-seal-budget-route-exhaustion}. The first theorem fixes the upstream selector-budget prefix, and the second exhausts every $\RealUp$ or $\CompletionUp$ seal consumer by the same finite packet $R,T,S,D,W,Q,L,H,C,P,N$.

Next insert the synchronizer with \autoref{thm:diagonal-limit-compatibility-seal-budget-synchronizer-handoff}. It forces the $\CauchySealBudgetSynchronizerUp$ request, seal, budget, selector, compatibility, transport, route, provenance, and local naming rows to be read before $Q$ and $L$.

Finally apply \autoref{thm:diagonal-limit-compatibility-real-seal-budget-nonescape}. The composed route exports only the displayed compatibility rows, synchronizer rows, tail-schedule rows, regular-rational readback, and $\RealUp$ seal, so no quotient stream, selected host limit, host equality, independent seal object, countable-choice row, or ambient completeness coordinate can escape it.
\end{proof}

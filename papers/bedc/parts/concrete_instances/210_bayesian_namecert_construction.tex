\begin{theorem}[Bayesian posterior source obligation]
\label{thm:bayesian-posterior-source-obligation}
Let $\mathcal{B}$ be a Bayesian update packet over
$\NameCert_{\DistributionUp}$ and $\NameCert_{\CondExpUp}$. The packet exposes,
as $\mathsf{BHist}$ data, the prior distribution row $\pi$, the likelihood row
$\lambda$, the evidence row $e$, the posterior endpoint $\omega$, the classifier
comparison $\hsame(\mathsf{Bayes}(\pi,\lambda,e),\omega)$, and the
normalisation row $\Cont(\pi,\lambda,e;\omega)$. These are the only public
source obligations of the Bayesian posterior surface; no host probability
kernel, measurable-space quotient, or external conditional-density equality is
exported by the certificate.
\end{theorem}
\leanchecked{BEDC.Derived.BayesianUp.BayesianPosteriorSurface\_source\_obligation}
\begin{proof}
Project the displayed update packet. Its $\DistributionUp$ source supplies the
prior distribution row and the posterior endpoint as distribution-side
$\mathsf{BHist}$ witnesses. Its $\CondExpUp$ source supplies the conditioning
row used to read the likelihood and evidence ledger.

The classifier projection is the displayed history-sameness row comparing the
Bayesian update expression with the posterior endpoint. The continuation
projection is the normalisation row, recording that the prior, likelihood, and
evidence data are continued to the visible posterior endpoint.

Repacking these projected rows gives exactly the stated public surface. Since
the packet inventory consists only of the two dependency certificates, the
evidence ledger, the posterior endpoint, the classifier row, the continuation
row, and provenance, there is no field from which a host probability kernel or
external conditional-density equality can be read.
\end{proof}

\begin{theorem}[Bayesian update ledger exactness]
\label{thm:bayesian-update-ledger-exactness}
For any accepted Bayesian update packet, every public posterior read factors
through the displayed $\DistributionUp$ source, the $\CondExpUp$ conditioning
row, the evidence ledger, the posterior endpoint, and the provenance row. A
consumer can therefore read the posterior only by projecting those rows and
repacking them into the displayed classifier and normalisation obligations.
\end{theorem}
\leanchecked{BEDC.Derived.BayesianUp.BayesianPosteriorSurface\_update\_ledger\_exactness}
\begin{proof}
Unpack the accepted packet into its dependency rows. The $\DistributionUp$
component is the sole source for the prior distribution row and the posterior
endpoint. The $\CondExpUp$ component is the sole source for the conditioning row
that binds the likelihood to the evidence ledger.

The public posterior read then has only two operations available. First, project
the classifier comparison $\hsame(\mathsf{Bayes}(\pi,\lambda,e),\omega)$ from
the displayed packet rows. Second, project the continuation row
$\Cont(\pi,\lambda,e;\omega)$ that records the normalised endpoint.

Finally, the provenance row records the $\Pkg$ origin of the distribution,
conditioning, evidence, endpoint, classifier, and normalisation rows. Repacking
only these projected rows reconstructs the public posterior read, while any
read requiring a host kernel, an unstated probability space, or an external
density equality has no source field in the packet.
\end{proof}

\begin{definition}[Cauchy reflective embedding carrier]
\label{def:cauchy-reflective-embedding-carrier}

$$

\item $S$ is the $\StreamNameUp$ window row through which the Cauchy data are observed before any real seal is read.
\item $Q$ is the $\RegSeqRatUp$ readback row that turns the displayed windows into regular rational data.
\item $D$ is the $\DyadicRatCoreUp$ tolerance ledger controlling each requested comparison window.

\begin{theorem}[Cauchy reflective embedding NameCert obligations]
\label{thm:cauchy-reflective-embedding-namecert-obligations}

\end{theorem}
\begin{proof}

The finite observation route is ordered. A consumer must read the window row $S$ before the regular readback $Q$ and dyadic ledger $D$ are available, and the Real seal $E$ is downstream of those displayed rows.

The completion-facing handoff is equally explicit. The row $J$ is the only coordinate that connects the reflected Real-facing data to the source-to-completion entry route of \autoref{ch:concrete-instances-cauchy-completion-embedding-namecert}.

The rows $H,T,P,N$ preserve and name the same route. Since no carrier coordinate contains quotient equality, selected limits, ambient completeness, countable choice, host equality, or bridge credentials, the listed rows exhaust the local NameCert surface.

\begin{proof}

The readback and tolerance rows $Q,D$ are the only rows answering precision requests. Thus the terminal seal $E$ is reached through the displayed regular-rational and dyadic route, not through an ambient completeness theorem.

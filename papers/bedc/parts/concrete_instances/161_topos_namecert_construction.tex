\chapter{A Concrete Naming Certificate for $\ToposUp$}
\label{ch:concrete-instances-topos-namecert}

The topos interface $\ToposUp$ packages a category with finite limits, exponentials, and a subobject classifier. It depends on $\NameCert_{\CategoryUp}$ (\autoref{ch:concrete-instances-category-namecert}) and $\NameCert_{\SheafUp}$ (\autoref{ch:concrete-instances-sheaf-namecert}), and is the substrate for sheaf-of-sets cohomology and internal logic.

\section{Carrier definition}
\label{sec:topos-carrier}

\begin{definition}[Topos finite carrier]
\label{def:topos-carrier}
A $\ToposUp$ carrier is a finite $\mathsf{BHist}$ packet displaying a $\NameCert_{\CategoryUp}$ row, a $\NameCert_{\SheafUp}$ row, finite-limit rows, exponential rows, a subobject-classifier row, componentwise $\hsame$ comparisons, $\Cont$ rows connecting those structures to the public readback, and $\mathsf{Pkg}$ provenance for the category and sheaf sources. The carrier exports no ambient universe of sets, no host internal language, and no unlogged categorical object.
\end{definition}

\begin{theorem}[Topos carrier obligation]
\label{thm:topos-carrier-obligation}
Every accepted $\ToposUp$ packet is generated by the finite $\mathsf{BHist}$ carrier of \autoref{def:topos-carrier}: its $\NameCert_{\CategoryUp}$ row, $\NameCert_{\SheafUp}$ row, finite-limit rows, exponential rows, subobject-classifier row, $\hsame$ comparisons, $\Cont$ ledger, and $\mathsf{Pkg}$ provenance are all displayed. These are the only rows exported to downstream categorical or internal-logic consumers.
\end{theorem}
\begin{proof}
Project the carrier. It records the category and sheaf sources as explicit certificate rows and attaches the finite-limit, exponential, and subobject-classifier data to those sources.

The accepted classifier is componentwise over the displayed $\hsame$ rows. The continuation ledger connects each structural row to the public packet readback, while the package rows record source provenance.

Since the $\mathsf{BHist}$ packet has no ambient set universe or host internal-language coordinate, the carrier exports exactly the displayed finite rows.
\end{proof}

\begin{theorem}[Topos finite-limit exponential scope]
\label{thm:topos-finite-limit-exponential-scope}
The finite-limit and exponential data of a $\ToposUp$ packet are read only from the displayed $\mathsf{BHist}$ rows, the $\NameCert_{\CategoryUp}$ dependency row, the $\NameCert_{\SheafUp}$ dependency row, componentwise $\hsame$ transport, the $\Cont$ ledger, and $\mathsf{Pkg}$ provenance. The packet does not import an ambient complete category, external cartesian-closed structure, or host set-theoretic semantics.
\end{theorem}
\begin{proof}
Read a finite-limit or exponential coordinate from the carrier. Its source is the displayed category row, with sheaf provenance available only through the named dependency certificate.

Transport is componentwise. The $\hsame$ rows move finite-limit, exponential, continuation, and provenance coordinates together without changing the packet shape.

Thus the finite-limit and exponential scope is exactly the displayed BEDC surface, not an ambient categorical universe.
\end{proof}

\section{The certificate}
\label{sec:topos-certificate}

\begin{theorem}[Topos subobject-classifier ledger exactness]
\label{thm:topos-subobject-classifier-ledger-exactness}
Every subobject-classifier readback in an accepted $\ToposUp$ packet is determined by the displayed subobject-classifier row, finite-limit rows, exponential rows, $\NameCert_{\CategoryUp}$ and $\NameCert_{\SheafUp}$ provenance rows, componentwise $\hsame$ transport, the $\Cont$ ledger, and $\mathsf{Pkg}$ data. It exposes no host truth-value object or unlogged internal predicate.
\end{theorem}
\begin{proof}
Project the subobject-classifier coordinate. It is stored as a finite row in the same packet as the finite-limit and exponential coordinates.

The dependency route is visible: the category and sheaf certificates provide provenance, while the continuation ledger carries the classifier row to the public readback. Componentwise $\hsame$ transport preserves this route.

No row in the carrier names a host truth-value object or external predicate semantics, so the subobject-classifier readback is exactly the displayed ledger entry.
\end{proof}

\begin{theorem}[Topos certificate boundary]
\label{thm:topos-certificate-boundary}
Every downstream internal-logic consumer of $\ToposUp$ reads only the displayed finite-limit, exponential, subobject-classifier, $\NameCert_{\CategoryUp}$, $\NameCert_{\SheafUp}$, $\mathsf{BHist}$, $\hsame$, $\Cont$, $\mathsf{Pkg}$, and provenance rows of \autoref{def:topos-carrier}. The certificate boundary excludes host set semantics, unlogged predicates, and any internal-language object not displayed in the packet.
\end{theorem}
\begin{proof}
Start from an internal-logic readback. The carrier obligation theorem confines it to the finite packet rows.

Use the finite-limit exponential theorem for the structural categorical coordinates and the subobject-classifier theorem for predicate-like readback. These account for the category and sheaf dependency rows, the continuation ledger, and the provenance data.

With no remaining coordinate for host semantics or an external internal-language object, the consumer can read only the displayed certificate rows.
\end{proof}

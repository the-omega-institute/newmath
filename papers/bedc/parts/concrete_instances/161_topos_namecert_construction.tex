\chapter{A Concrete Naming Certificate for $\ToposUp$}
\label{ch:concrete-instances-topos-namecert}

The topos interface $\ToposUp$ packages a category with finite limits, exponentials, and a subobject classifier. It depends on $\NameCert_{\CategoryUp}$ (\autoref{ch:concrete-instances-category-namecert}) and $\NameCert_{\SheafUp}$ (\autoref{ch:concrete-instances-sheaf-namecert}), and is the substrate for sheaf-of-sets cohomology and internal logic.

\section{Carrier definition}
\label{sec:topos-carrier}

\begin{definition}[Topos finite carrier]
\label{def:topos-carrier}
A $\ToposUp$ carrier is a finite $\mathsf{BHist}$ packet displaying a $\NameCert_{\CategoryUp}$ row, a $\NameCert_{\SheafUp}$ row, finite-limit rows, exponential rows, a subobject-classifier row, componentwise $\hsame$ comparisons, $\Cont$ rows connecting those structures to the public readback, and $\mathsf{Pkg}$ provenance for the category and sheaf sources. The carrier exports no ambient universe of sets, no host internal language, and no unlogged categorical object.
\end{definition}

\begin{theorem}[Topos carrier obligation]
\label{thm:topos-carrier-obligation}
Every accepted $\ToposUp$ packet is generated by the finite $\mathsf{BHist}$ carrier of \autoref{def:topos-carrier}: its $\NameCert_{\CategoryUp}$ row, $\NameCert_{\SheafUp}$ row, finite-limit rows, exponential rows, subobject-classifier row, $\hsame$ comparisons, $\Cont$ ledger, and $\mathsf{Pkg}$ provenance are all displayed. These are the only rows exported to downstream categorical or internal-logic consumers.
\end{theorem}
\leanchecked{BEDC.Derived.ToposUp.ToposFiniteCarrier\_obligation\_surface}
\begin{proof}
Project the carrier. It records the category and sheaf sources as explicit certificate rows and attaches the finite-limit, exponential, and subobject-classifier data to those sources.

The accepted classifier is componentwise over the displayed $\hsame$ rows. The continuation ledger connects each structural row to the public packet readback, while the package rows record source provenance.

Since the $\mathsf{BHist}$ packet has no ambient set universe or host internal-language coordinate, the carrier exports exactly the displayed finite rows.
\end{proof}

\begin{theorem}[Topos finite-limit exponential scope]
\label{thm:topos-finite-limit-exponential-scope}
The finite-limit and exponential data of a $\ToposUp$ packet are read only from the displayed $\mathsf{BHist}$ rows, the $\NameCert_{\CategoryUp}$ dependency row, the $\NameCert_{\SheafUp}$ dependency row, componentwise $\hsame$ transport, the $\Cont$ ledger, and $\mathsf{Pkg}$ provenance. The packet does not import an ambient complete category, external cartesian-closed structure, or host set-theoretic semantics.
\end{theorem}
\begin{proof}
Read a finite-limit or exponential coordinate from the carrier. Its source is the displayed category row, with sheaf provenance available only through the named dependency certificate.

Transport is componentwise. The $\hsame$ rows move finite-limit, exponential, continuation, and provenance coordinates together without changing the packet shape.

Thus the finite-limit and exponential scope is exactly the displayed BEDC surface, not an ambient categorical universe.
\end{proof}
\leanchecked{BEDC.Derived.ToposUp.ToposFiniteCarrier\_finite\_limit\_exponential\_scope}

\section{The certificate}
\label{sec:topos-certificate}

\begin{theorem}[Topos subobject-classifier ledger exactness]
\label{thm:topos-subobject-classifier-ledger-exactness}
Every subobject-classifier readback in an accepted $\ToposUp$ packet is determined by the displayed subobject-classifier row, finite-limit rows, exponential rows, $\NameCert_{\CategoryUp}$ and $\NameCert_{\SheafUp}$ provenance rows, componentwise $\hsame$ transport, the $\Cont$ ledger, and $\mathsf{Pkg}$ data. It exposes no host truth-value object or unlogged internal predicate.
\end{theorem}
\leanchecked{BEDC.Derived.ToposUp.ToposSubobjectClassifierLedger\_exactness}
\begin{proof}
Project the subobject-classifier coordinate. It is stored as a finite row in the same packet as the finite-limit and exponential coordinates.

The dependency route is visible: the category and sheaf certificates provide provenance, while the continuation ledger carries the classifier row to the public readback. Componentwise $\hsame$ transport preserves this route.

No row in the carrier names a host truth-value object or external predicate semantics, so the subobject-classifier readback is exactly the displayed ledger entry.
\end{proof}

\begin{theorem}[Topos subobject-classifier pullback boundary]
\label{thm:topos-subobject-classifier-pullback-boundary}
Any pullback read of a displayed subobject-classifier row in a $\ToposUp$
packet factors only through the $\CategoryUp$ site row, the $\SheafUp$
cover/gluing row, finite-limit rows, exponential rows, the displayed
subobject-classifier row, componentwise $\hsame$, finite $\Cont$, and
$\mathsf{Pkg}$ provenance. It exposes no host truth values, ambient set
semantics, or unlogged predicate extensions.
\end{theorem}
\leanchecked{BEDC.Derived.ToposUp.ToposSubobjectClassifierLedger\_pullback\_boundary}
\leanvariant{BEDC.Derived.ToposUp.ToposSubobjectClassifierLedger\_pullback\_boundary\_certificate}
\leanvariant{BEDC.Derived.ToposUp.ToposFiniteCarrier\_subobject\_classifier\_pullback\_boundary}
\leanvariant{BEDC.Derived.ToposUp.ToposSubobjectClassifier\_pullback\_boundary}
\begin{proof}
Start with \autoref{thm:topos-finite-limit-exponential-scope}. It confines the
structural part of any pullback request to the displayed category and sheaf
source rows, finite-limit rows, exponential rows, componentwise transport,
continuation, and package provenance.

Combine this with
\autoref{thm:topos-subobject-classifier-ledger-exactness}. The classifier row
itself is a displayed finite ledger entry in the same packet and carries no
host truth-value object or external predicate semantics.

Repack the pullback request as one of the finite-limit rows already present in
the carrier, with the displayed subobject-classifier row as its classified
input. The repacking uses only the visible $\hsame$, $\Cont$, and
$\mathsf{Pkg}$ routes, so no ambient set semantics or unlogged predicate
extension is introduced.
\end{proof}

\begin{theorem}[Topos site-sheaf classifier obligation]
\label{thm:topos-site-sheaf-classifier-obligation}
Any subobject-classifier readback in a $\ToposUp$ packet factors through the
displayed $\CategoryUp$ site row, the $\SheafUp$ cover/gluing row, finite-limit
and exponential rows, componentwise $\hsame$, finite $\Cont$, and
$\mathsf{Pkg}$ provenance. It exports no host truth-value object or ambient set
semantics.
\end{theorem}
\leanchecked{BEDC.Derived.ToposUp.ToposFiniteCarrier\_site\_sheaf\_classifier\_obligation}
\begin{proof}
Project the carrier. The packet stores the subobject-classifier row together
with the displayed category and sheaf source rows, so every classifier readback
starts from those finite $\mathsf{BHist}$ coordinates.

Use \autoref{thm:topos-finite-limit-exponential-scope} for the structural
categorical rows. The finite-limit and exponential data are already routed
through the $\CategoryUp$ and $\SheafUp$ dependencies, componentwise $\hsame$,
finite $\Cont$, and $\mathsf{Pkg}$ provenance.

Use \autoref{thm:topos-subobject-classifier-ledger-exactness} for the
predicate-like classifier row. Repacking these dependency routes gives a finite
BEDC obligation surface, and no remaining coordinate can supply a host
truth-value object or ambient set semantics.
\end{proof}

\begin{theorem}[Topos certificate boundary]
\label{thm:topos-certificate-boundary}
Every downstream internal-logic consumer of $\ToposUp$ reads only the displayed finite-limit, exponential, subobject-classifier, $\NameCert_{\CategoryUp}$, $\NameCert_{\SheafUp}$, $\mathsf{BHist}$, $\hsame$, $\Cont$, $\mathsf{Pkg}$, and provenance rows of \autoref{def:topos-carrier}. The certificate boundary excludes host set semantics, unlogged predicates, and any internal-language object not displayed in the packet.
\end{theorem}
\leanchecked{BEDC.Derived.ToposUp.ToposFiniteCarrier\_certificate\_boundary}
\leanvariant{BEDC.Derived.ToposUp.ToposFiniteCarrier\_classified\_boundary\_transport}
\begin{proof}
Start from an internal-logic readback. The carrier obligation theorem confines it to the finite packet rows.

Use the finite-limit exponential theorem for the structural categorical coordinates and the subobject-classifier theorem for predicate-like readback. These account for the category and sheaf dependency rows, the continuation ledger, and the provenance data.

With no remaining coordinate for host semantics or an external internal-language object, the consumer can read only the displayed certificate rows.
\end{proof}

\begin{theorem}[Topos NameCert obligation surface]
\label{thm:topos-namecert-obligation-surface}
The public $\NameCert_{\ToposUp}$ fields determined by \autoref{def:topos-carrier} are exactly the displayed $\CategoryUp$ and $\SheafUp$ source rows, finite-limit rows, exponential rows, subobject-classifier row, componentwise $\hsame$ transport, finite $\Cont$ ledger, $\mathsf{Pkg}$ provenance, and the consumer boundary of \autoref{thm:topos-certificate-boundary}. The obligation surface excludes host set semantics, an ambient universe, unlogged predicates, and any external internal language.
\end{theorem}
\leanchecked{BEDC.Derived.ToposUp.ToposFiniteCarrier\_namecert\_obligation\_surface}
\begin{proof}
Project the carrier obligation. It supplies the displayed $\CategoryUp$ and $\SheafUp$ source rows, the finite-limit and exponential rows, the subobject-classifier row, componentwise $\hsame$ transport, finite $\Cont$, and $\mathsf{Pkg}$ provenance.

Apply \autoref{thm:topos-finite-limit-exponential-scope}. The structural reads remain inside the finite carrier rows and cannot import an ambient complete category, an external cartesian-closed structure, or host set-theoretic semantics.

Apply \autoref{thm:topos-subobject-classifier-ledger-exactness} and \autoref{thm:topos-subobject-classifier-pullback-boundary}. The classifier readback is the displayed finite ledger entry, and pullback access to it factors only through the same $\CategoryUp$, $\SheafUp$, finite-limit, exponential, $\hsame$, $\Cont$, and $\mathsf{Pkg}$ routes.

Finally apply \autoref{thm:topos-certificate-boundary}. It exhausts downstream internal-logic reads by the listed certificate rows, leaving no coordinate for host set semantics, an ambient universe, unlogged predicates, or an external internal language.
\end{proof}

\begin{theorem}[Topos public certificate export]
\label{thm:topos-public-certificate-export}
Downstream internal-logic and sheaf consumers of $\ToposUp$ may read exactly
the finite $\mathsf{BHist}$ packet rows listed in
\autoref{thm:topos-namecert-obligation-surface}: the $\CategoryUp$ row, the
$\SheafUp$ row, finite-limit rows, exponential rows, subobject-classifier row,
componentwise $\hsame$, finite $\Cont$, $\mathsf{Pkg}$ provenance, and the
displayed provenance rows.

The public certificate exports no host set semantics, ambient universe,
unlogged predicate, or external internal-language object.
\end{theorem}
\begin{proof}
Start with \autoref{thm:topos-carrier-obligation}. It places every consumer
read inside the displayed finite $\mathsf{BHist}$ packet with the category,
sheaf, finite-limit, exponential, subobject-classifier, classifier, continuation,
and package rows.

Apply \autoref{thm:topos-finite-limit-exponential-scope} for the structural
rows, and apply \autoref{thm:topos-subobject-classifier-ledger-exactness} for
the predicate-like row. The pullback boundary of
\autoref{thm:topos-subobject-classifier-pullback-boundary} keeps classifier
access on the same finite ledger.

Finally use \autoref{thm:topos-certificate-boundary} and
\autoref{thm:topos-namecert-obligation-surface}. Together they repack the
obligation surface as the public certificate export and leave no coordinate for
host set semantics, an ambient universe, an unlogged predicate, or an external
internal-language object.
\end{proof}

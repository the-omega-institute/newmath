Following Bishop and Bridges' constructive metric-space treatment, the $\PointwiseStreamMetricUp$ packet records the standard pointwise distance interface for certified streams as finite BEDC data. It sits after the stream-name surface of \autoref{ch:concrete-instances-streamname-namecert}, the regular-rational readback of \autoref{ch:concrete-instances-regseqrat-namecert}, the terminal real-name seal of \autoref{ch:concrete-instances-real-namecert}, and the metric-space carrier discipline of \autoref{ch:concrete-instances-metric-namecert}. The packet exposes pointwise metric data for completion and continuous-map consumers without host functions, quotient stream equality, selected representatives, or choice.

\begin{definition}[Pointwise stream metric carrier]
\label{def:pointwise-stream-metric-carrier}
A $\PointwiseStreamMetricUp$ carrier is a finite $\BHist$ packet
$$
Q=(S,W,D,R,E,M,H,C,P,N)
$$
with the following displayed coordinates:
\begin{enumerate}
\item $S$ is the certified stream source row, read through the $\StreamNameUp$ finite-window interface.
\item $W$ records the finite pointwise window schedule used when two certified streams are compared at a displayed index.
\item $D$ is the dyadic tolerance and triangle ledger governing the finite comparisons of pointwise observations.
\item $R$ is the $\RegSeqRatUp$ readback row that converts the finite rational comparison stream into a regular rational name.
\item $E$ is the terminal $\RealUp$ seal for the distance value obtained from the displayed pointwise route.
\item $M$ is the $\MetricSpaceUp$ metric-structure row, used only after the pointwise stream route has displayed $S,W,D,R,E$.
\item $H$ stores componentwise $\hsame$ transport for stream names, window schedules, dyadic ledgers, regular readbacks, real seals, metric rows, replay, provenance, and local naming rows.
\item $C$ stores displayed $\Cont$ replay from a consumer request through $S,W,D,R,E,M,H,P,N$.
\item $P$ is the $\Pkg$ provenance over $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, $\RealUp$, $\MetricSpaceUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$ rows.
\item $N$ is the local $\NameCert_{\PointwiseStreamMetricUp}$ row.
\end{enumerate}
The classifier compares accepted packets only through these coordinates. It contains no host function equality, quotient stream object, selected stream representative, countable-choice schedule, or ambient completeness assertion.
\end{definition}

\begin{theorem}[Pointwise stream metric pointwise classifier]
\label{thm:pointwise-stream-metric-pointwise-classifier}
For an accepted $\PointwiseStreamMetricUp$ carrier $Q=(S,W,D,R,E,M,H,C,P,N)$, every pointwise distance classifier read factors through the certified stream source $S$, the finite window schedule $W$, the dyadic tolerance ledger $D$, the regular-rational readback $R$, and the terminal $\RealUp$ seal $E$ before the $\MetricSpaceUp$ row $M$ may be inspected. The classifier therefore names a pointwise metric route on certified streams and does not read host functions, quotient streams, selected representatives, countable choice, or an ambient completion theorem.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:pointwise-stream-metric-carrier}. The source stream, window schedule, dyadic ledger, regular readback, Real seal, metric row, transport, replay, provenance, and local naming row are all displayed coordinates.

The pointwise route begins with $S$ and $W$. A consumer must name the stream source and the finite observation windows before any rational readback or metric row can be inspected.

The distance read then passes through $D,R,E$. The dyadic ledger bounds the finite comparisons, the regular-rational row gives the readback discipline, and the Real seal is the terminal distance value. Since no coordinate stores a host function or quotient stream, the classifier cannot export one.
\end{proof}

\begin{theorem}[Pointwise stream metric constant stream]
\label{thm:pointwise-stream-metric-constant-stream}
Let $Q=(S,W,D,R,E,M,H,C,P,N)$ be an accepted $\PointwiseStreamMetricUp$ carrier whose stream source row $S$ is displayed as a constant certified stream. Then every distance read between two displayed copies of that source is routed through the same finite window schedule $W$, dyadic zero-tolerance ledger $D$, regular-rational readback $R$, terminal Real seal $E$, and metric row $M$. The packet records the constant-stream zero-distance route as displayed NameCert data, not as host equality of functions or quotient stream identity.
\end{theorem}
\begin{proof}
Start with the displayed constant stream row in $S$. Because the stream is certified by its $\StreamNameUp$ windows, the two copies are compared only through the finite schedule $W$.

Use the dyadic ledger $D$ for the zero-tolerance comparison. The readback row $R$ and the terminal seal $E$ record the regular-rational and Real distance route generated by that finite ledger.

Finally inspect $M$ only after the pointwise route has been displayed. The metric row reads the zero-distance route as finite certificate data, while host function equality and quotient stream identity are absent from the carrier.
\end{proof}

\begin{theorem}[Pointwise stream metric reindexing stability]
\label{thm:pointwise-stream-metric-reindexing-stability}
Let $Q=(S,W,D,R,E,M,H,C,P,N)$ be an accepted $\PointwiseStreamMetricUp$ carrier. Any accepted finite reindexing of the certified stream windows transports the pointwise metric classifier only through the displayed $\hsame$ rows $H$ and the replay row $C$: the reindexed route must preserve the stream source row, window schedule, dyadic ledger, regular-rational readback, Real seal, metric row, provenance, and local naming row. It cannot add a hidden stream quotient, selected representative, or host-level function extensionality principle.
\end{theorem}
\begin{proof}
Read the transport coordinate $H$. It is componentwise: stream sources, window schedules, dyadic ledgers, readback rows, Real seals, metric rows, replay, provenance, and naming rows are transported as displayed rows.

Replay the reindexed request with $C$. Since $C$ follows the same order $S,W,D,R,E,M,H,P,N$, the pointwise classifier is preserved only by the finite route already present in the carrier.

The exclusions are an inventory check. No coordinate of $Q$ stores a stream quotient, selected representative, or host-level function extensionality principle, so reindexing stability cannot export such data.
\end{proof}

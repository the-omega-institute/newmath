\begin{definition}[Metric Cauchy-net cluster carrier]
\label{def:metric-cauchy-net-cluster-carrier}
A $\MetricCauchyNetClusterUp$ carrier is a finite $\BHist$ packet
$$
K=(D,W,M,J,S,R,T,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $D$ is the directed-net row, recording the inspected finite directed indices and the local comparison order used by the Moore--Smith route.
\item $W$ is the Cauchy-window row, listing the finite cofinal windows and the tolerance requests that must be met before any cluster read is exposed.
\item $M$ is the metric completion boundary row, connecting the displayed net windows to the $\MetricCompletionUp$ and $\CauchyNetCompletionUp$ consumer surfaces.
\item $J$ is the cluster witness row, recording only finite witness windows and not a selected cluster point.
\item $S$ is the $\StreamNameUp$ row that opens the finite observation windows used by $W$ and $J$.
\item $R$ is the $\RegSeqRatUp$ readback row for rational distance and tolerance comparisons inspected along the route.
\item $T$ is the $\DyadicRatCoreUp$ tolerance ledger attached to the same windows and readbacks.
\item $E$ is the terminal $\RealUp$ seal row, available only after the directed-net, Cauchy-window, cluster-witness, stream, regular-readback, dyadic, and metric-completion rows have been displayed.
\item $H$ records componentwise $\hsame$ transport for all displayed rows.
\item $C$ records displayed $\Cont$ replay from a completion-facing request through $D,W,M,J,S,R,T,E,H,P,N$.

\item $N$ is the local $\NameCert_{\MetricCauchyNetClusterUp}$ row.
\end{enumerate}
The carrier has no coordinate for quotient nets, chosen cluster points, choice-selected subnets, host metric completion equality, or ambient completeness.
\end{definition}

\begin{theorem}[Metric Cauchy-net cluster NameCert obligations]
\label{thm:metric-cauchy-net-cluster-namecert-obligations}
For every accepted $\MetricCauchyNetClusterUp$ carrier $K=(D,W,M,J,S,R,T,E,H,C,P,N)$, the local $\NameCert_{\MetricCauchyNetClusterUp}$ surface consists exactly of the following obligation rows:
\begin{enumerate}
\item carrier admission for the directed-net row, Cauchy-window row, metric-completion boundary, cluster witness, stream windows, regular rational readback, dyadic tolerance ledger, Real seal, transport, replay, provenance, and local naming rows;
\item directed-window stability, requiring every inspected directed index to be read through $D$ before $W$ can answer a Cauchy request;
\item cluster-witness locality, requiring $J$ to expose only finite witness windows tied to the same $D,W,S,T$ rows;
\item completion-boundary handoff, requiring $M$ to mediate every $\CauchyNetCompletionUp$ or $\MetricCompletionUp$ consumer before $E$ is read;
\item regular and dyadic accounting, requiring $R$ and $T$ to record the rational distance and tolerance ledger used by the same finite windows;
\item structural exactness through $H,C,P,N$, with no quotient-net equality, selected cluster point, countable choice, host real equality, or ambient completeness coordinate.
\end{enumerate}
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:metric-cauchy-net-cluster-carrier}. The packet exposes the directed-net row, Cauchy-window row, metric boundary, cluster witness, finite windows, regular readback, dyadic ledger, terminal seal, and the structural rows as one finite object.

The directed and Cauchy-window obligations are read first. The row $D$ fixes the finite directed comparisons, and $W$ answers tolerance requests only through those displayed comparisons and the finite stream windows $S$.

The cluster witness row $J$ is then local to the same windows. It records witness windows rather than a selected point, while $R$ and $T$ record the rational distance readback and dyadic tolerance ledger used by the window.

The completion boundary $M$ mediates the completion-facing read before the Real seal row $E$ is available. Since $H,C,P,N$ only transport, replay, source, and name the same rows, the listed obligations exhaust the NameCert surface.
\end{proof}

\label{thm:metric-cauchy-net-cluster-completion-handoff}

\begin{proof}
Begin with a completion-facing consumer. By \autoref{def:metric-cauchy-net-cluster-carrier}, the only completion boundary in the packet is the row $M$, and the only terminal real-facing seal is $E$.

Apply the obligation surface of \autoref{thm:metric-cauchy-net-cluster-namecert-obligations}. It places directed-net admission and Cauchy-window stability before any cluster witness is read, and it ties the witness row to the same finite stream windows.

The regular rational and dyadic rows then supply the metric tolerance accounting used by the finite windows. The completion boundary $M$ is downstream of that accounting, and the seal row $E$ is downstream of $M$.

Finally $H,C,P,N$ preserve and name only the displayed route. Quotient nets, selected cluster points, countable choice, host real equality, detached completion objects, and ambient completeness have no carrier coordinate, so they cannot replace the finite route.
\end{proof}

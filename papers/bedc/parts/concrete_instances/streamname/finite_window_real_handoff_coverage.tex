\begin{theorem}[StreamName finite-window Real handoff coverage]
\label{thm:streamname-finite-window-real-handoff-coverage}
Every RegSeqRat-to-$\RealUp$ handoff read mediated by $\StreamNameUp$ factors through displayed finite windows, the $\RegSeqRat$ pointwise rows, the $\RealUp$ seal boundary, componentwise $\hsame$ transport, displayed $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ rows. More explicitly, the source surface is the finite inventory of $\mathsf{RatHistoryClassifier}$ observations, $\mathsf{ProbeBundle}$ ledgers, $\mathsf{InBundle}$ membership rows, $\mathsf{UnaryHistory}$ observations, accepted $\RegSeqRat$ pointwise seal rows, finite continuation rows, transport rows, provenance rows, and naming rows already displayed by the StreamName handoff.

The coverage theorem exports no quotient stream, no host sequence equality, no ambient real completeness principle, no completion representative, and no $\RealUp$ seal read outside the finite StreamName and RegSeqRat rows.
\end{theorem}
\begin{proof}
First read the StreamName source surface. Apply \autoref{thm:streamname-real-regseq-finite-window-handoff}: a public StreamName handoff exposes only selected rational-history classifier rows, the finite $\mathsf{ProbeBundle}$ ledger, $\mathsf{InBundle}$ membership rows, and $\mathsf{UnaryHistory}$ observations.

Next refine that source pointwise. Apply \autoref{thm:streamname-regseqrat-pointwise-handoff}; the downstream RegSeqRat or finite-window Real consumer receives exactly the StreamName carrier rows, finite-window classifier rows, certified unary observations, membership rows, and pointwise $\hsame$ comparisons. No regularity or seal row is created by this refinement.

Now pass to the Real seal boundary. Apply \autoref{thm:streamname-real-seal-factorization-boundary}. It says that a finite-window $\RealUp$ seal read arising from the StreamName-to-RegSeqRat handoff factors through the StreamName point rows, accepted $\RegSeqRat$ seal rows, bundle and membership rows, unary observations, finite $\Cont$ ledger, displayed $\hsame$ transports, and $\Pkg$ provenance.

Finally repack the same finite rows with the local naming certificate. The cited handoff and factorization rows already supply the finite windows, pointwise RegSeqRat rows, Real seal boundary, transport, continuation, and provenance. The StreamName NameCert row names that same finite surface, so a consumer has no route to quotient streams, host sequence equality, ambient real completeness, completion representatives, or a Real seal read outside the displayed packet.
\end{proof}

\begin{theorem}[StreamName RegSeqRat Real window factorization]
\label{thm:streamname-regseqrat-real-window-factorization}
Suppose two $\StreamNameUp$-mediated $\RegSeqRat$-to-$\RealUp$ handoff reads use the same finite StreamName window, the same accepted $\RegSeqRat$ pointwise row, and the same $\RealUp$ seal boundary. Then the two reads have the same public source surface: the same finite packet of $\mathsf{RatHistoryClassifier}$ observations, $\mathsf{ProbeBundle}$ and $\mathsf{InBundle}$ rows, $\mathsf{UnaryHistory}$ observations, $\RegSeqRat$ pointwise seal rows, displayed $\hsame$ transports, displayed $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ rows.

This factorization exposes no second $\RealUp$ seal, no quotient stream, no host sequence equality, no selected completion representative, and no ambient completeness coordinate.
\end{theorem}
\begin{proof}
First apply \autoref{thm:streamname-finite-window-real-handoff-coverage} to both handoff reads. The two inventories are finite packets whose only public rows are the StreamName window, the RegSeqRat pointwise data, the Real seal boundary, the displayed transport and continuation routes, the provenance row, and the local naming rows.

Next apply \autoref{thm:streamname-regseqrat-pointwise-handoff} to the shared accepted pointwise row. It identifies the RegSeqRat portion of both packets with the same StreamName carrier rows, classifier observations, unary observations, membership rows, and componentwise $\hsame$ comparisons.

Now apply \autoref{thm:streamname-real-seal-factorization-boundary} to the shared Real seal boundary. The Real read factors through the same accepted RegSeqRat rows, finite bundle rows, displayed $\Cont$ routes, $\Pkg$ provenance, and transport rows; it does not create an additional Real-facing source coordinate.

Finally use \autoref{thm:streamname-real-regseq-common-window-consumer-determinacy}. Common-window determinacy identifies the two consumer reads once their finite StreamName window, RegSeqRat row, and Real seal boundary agree. Repacking those common rows under the local $\NameCert$ surface gives the stated public packet and rules out any quotient, host-equality, representative, or ambient-completeness exposure.
\end{proof}

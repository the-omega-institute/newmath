\label{thm:streamname-regularcauchyname-lattice-link}
Every $\StreamNameUp$ finite-window schedule consumed by $\RegularCauchyNameUp$ is read as the scheduled-window sibling route of \autoref{ch:concrete-instances-regularcauchyname-namecert}. It exports no host sequence, quotient-stream equality, selected limit, independent real carrier, or ambient completeness coordinate.
\end{theorem}
\begin{proof}
Start with the finite-window rows already displayed in the $\StreamNameUp$ packet. The public interface and lattice dependency rows expose scheduled windows, bundle membership, certified unary observations, pointwise $\RegSeqRatUp$ classifier data, componentwise $\hsame$ transport, displayed $\Cont$ routes, package provenance, and local naming.

The $\RegularCauchyNameUp$ handoff of \autoref{ch:concrete-instances-regularcauchyname-namecert} consumes exactly that scheduled-window surface before exposing its dyadic-radius row, regularity ledger, and $\RealUp$ seal. Hence the StreamName schedule is not a free-floating kernel row; it is the window sibling route through which the regular Cauchy name reads its source observations.

The non-escape clause is coordinate exhaustion. Neither the StreamName finite-window package nor the RegularCauchyName handoff contains a host sequence, quotient-stream equality, selected limit, independent real carrier, or ambient completeness coordinate, so the routed lattice link cannot export one.
\end{proof}

\end{proof}

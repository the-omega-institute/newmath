\section{StreamName finite-section pullback exactness}
\label{sec:streamname-finite-section-pullback-exactness}

\begin{theorem}[StreamName finite-section pullback exactness]
\label{thm:streamname-finite-section-pullback-exactness}
Let $Q$ be an accepted finite request section of a $\StreamNameUp$ packet. If
the finite windows of $Q$ agree with the $\DyadicRatCoreUp$ ledger and the
$\RegSeqRatUp$ readback used by the current $\RealUp$ seal, then $Q$
determines the same terminal four-face route
$\StreamNameUp,\DyadicRatCoreUp,\RegSeqRatUp,\RealUp$. The support of this
route is only the displayed $\BHist$ rows, componentwise $\hsame$ transports,
displayed $\Cont$ routes, $\Pkg$ provenance, and local $\NameCert$ rows. It
exports no quotient stream, selected limit, host sequence equality, ambient
completeness row, countable-choice row, or standard bridge row.
\end{theorem}
\begin{proof}
First apply \autoref{thm:streamname-finite-request-section-pullback}. The
accepted request section is already pulled back to the displayed
$\StreamNameUp$ schedule, the $\DyadicRatCoreUp$ endpoint-radius ledger, the
$\RegSeqRatUp$ regularity and readback surface, and the support rows
$\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$.

Next use \autoref{thm:streamname-open-phase-four-face-terminality}. Since the
finite windows agree with the DyadicRatCore ledger and the RegSeqRat readback
used by the current RealUp seal, terminality identifies the resulting route
with the unique four-face route through $\StreamNameUp$, $\DyadicRatCoreUp$,
$\RegSeqRatUp$, and $\RealUp$.

Now apply \autoref{thm:streamname-regseqrat-real-public-seal-no-quotient}. The
RealUp seal remains on the public finite rows and therefore cannot add a
quotient stream, selected limit, host sequence equality, or ambient
completeness coordinate to the section.

Finally use \autoref{thm:streamname-regseqrat-dyadic-real-obligation-handoff}.
It exhausts carrier, classifier, stability, ledger, and non-escape obligations
by the same four faces and the displayed support rows, leaving no
countable-choice row or standard bridge row to export.
\end{proof}

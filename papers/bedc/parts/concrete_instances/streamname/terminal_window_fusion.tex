\begin{theorem}[StreamName Real-completion terminal-window fusion]
\label{thm:streamname-real-completion-terminal-window-fusion}
Every accepted Phase Real and Completion read through
$$
\StreamNameUp,\quad \DyadicRatCoreUp,\quad \RegSeqRatUp,\quad \RealUp
$$
is determined by the fused terminal window consisting of the StreamName finite request window, the $\DyadicRatCoreUp$ tolerance ledger, the $\RegSeqRatUp$ readback, the $\RealUp$ seal, and the support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
Equivalently, two accepted reads with the same fused terminal window have the same public classifier surface and the same terminal factorization through the four Real and Completion exit faces.

The fused packet exports no quotient stream equality, no host sequence equality, no selected limit, no ambient $\RealUp$ completeness theorem, no metric-completeness datum, no countable-choice row, and no standard bridge row.
\end{theorem}
\begin{proof}
Begin with the four-object exit route of \autoref{thm:streamname-real-completion-four-object-exit-route}. It identifies every accepted current read with the four displayed faces and with only the $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ support rows.

Next apply the terminal pullback and terminal exactness results, \autoref{thm:streamname-real-completion-four-object-terminal-pullback} and \autoref{thm:streamname-real-completion-four-object-terminal-exactness}. These results say that agreement on the StreamName finite request window, the DyadicRatCore tolerance ledger, the RegSeqRat readback, the RealUp seal, and the displayed support rows is precisely the terminal boundary for public acceptance.

Now transport this terminal boundary to the classifier surface. Use \autoref{thm:streamname-real-completion-classifier-pullback}: once the fused terminal window agrees componentwise, the public classifier has no second coordinate on which to distinguish the two reads.

It remains to bind the dependency and handoff rows. The dependency exactness theorem \autoref{thm:streamname-real-completion-dependency-exactness} names the same four faces as necessary and sufficient, while \autoref{thm:streamname-real-completion-three-touchpoint-handoff-route} reads the StreamName finite-window handoff, DyadicRatCore ledger handoff, and RegSeqRat-to-Real seal as route witnesses for those existing rows.

Finally apply the no-choice route factorization, \autoref{thm:streamname-real-completion-route-no-choice-factorization}. It removes every attempted selector, quotient, completeness, metric, choice, or bridge coordinate from the packet. Therefore the fused terminal window determines the accepted Phase Real and Completion read exactly through the four displayed exit faces and the five displayed support rows.
\end{proof}

\begin{theorem}[StreamName terminal fusion common-refinement stability]
\label{thm:streamname-terminal-fusion-common-refinement-stability}
Let $\mathcal W_L$ and $\mathcal W_R$ be accepted Phase Real and Completion reads whose $\StreamNameUp$ finite windows admit a certified common refinement $\mathcal R$. Suppose the refinement transports the same $\DyadicRatCoreUp$ tolerance ledger, the same $\RegSeqRatUp$ readback, the same $\RealUp$ seal, and the same displayed support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
Then the fused terminal-window packet read through $\mathcal W_L$ is the same packet read through $\mathcal W_R$. In particular the public classifier surface is unchanged by passing to $\mathcal R$ and restricting back to either selected window.

The stability statement adds no quotient stream equality, no host sequence equality, no selected limit, no ambient $\RealUp$ completeness theorem, no metric-completeness datum, no countable-choice row, and no standard bridge row.
\end{theorem}
\begin{proof}
First read the common refinement at the StreamName boundary. Apply \autoref{thm:streamname-finite-window-common-refinement-export} and \autoref{thm:streamname-common-refinement-source-determinacy}. The refinement $\mathcal R$ supplies one finite-window classifier, one $\mathsf{ProbeBundle}$ row, the same $\mathsf{InBundle}$ membership rows, the same selected histories, and the same displayed $\hsame/\Cont/\Pkg$ ledger rows for both selected windows.

Next pass from the source tuple to the Real and Completion exit packet. Use \autoref{thm:streamname-common-refinement-public-exactness}: every downstream $\RegSeqRat$ handoff and every finite-window $\RealUp$ handoff read from $\mathcal R$ factors through that common source tuple. The hypotheses identify the $\DyadicRatCoreUp$ tolerance ledger, $\RegSeqRatUp$ readback, $\RealUp$ seal, and local $\NameCert$ rows as the same transported data on the two restrictions.

Now apply the terminal pullback theorem \autoref{thm:streamname-real-completion-four-object-terminal-pullback}. Since the four faces $\StreamNameUp$, $\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$ and the displayed support rows agree after transport along $\mathcal R$, the two accepted reads determine the same terminal finite BEDC pullback. The terminal-window fusion theorem \autoref{thm:streamname-real-completion-terminal-window-fusion} repacks this pullback as one fused terminal-window packet.

Finally transport the equality of terminal packets to the public classifier surface by \autoref{thm:streamname-real-completion-classifier-pullback}. The no-choice route factorization \autoref{thm:streamname-real-completion-route-no-choice-factorization} removes every selector, quotient, completeness, metric, choice, and bridge coordinate, so a common refinement cannot create a second public presentation of the same terminal route.
\end{proof}

\begin{theorem}[StreamName terminal fusion common-tail Real boundary pullback]
\label{thm:streamname-terminal-fusion-common-tail-real-boundary-pullback}
Let $\mathcal W$ be an accepted Phase Real and Completion read, and let its fused terminal window be transported through a $\RegSeqRatUp$ common-tail window packet with common refinement $W_\ast$. The transported fused terminal packet determines exactly the same public $\RealUp$ seal boundary as the common-tail Real boundary equivalence of \autoref{thm:regseqrat-common-tail-real-boundary-equivalence}.

Equivalently, two such transported reads have the same public classifier surface if and only if they agree on the $\StreamNameUp$ finite window, the $\DyadicRatCoreUp$ endpoint, radius, and distance ledger, the $\RegSeqRatUp$ classifier and regularity rows, the $\RealUp$ seal-source rows, and the displayed rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
No quotient stream equality, host sequence equality, selected limit, ambient $\RealUp$ completeness theorem, metric-completeness datum, countable-choice row, or standard bridge row is part of this pullback boundary.
\end{theorem}
\begin{proof}
First use \autoref{thm:streamname-terminal-fusion-common-refinement-stability}. The common refinement makes the selected $\StreamNameUp$ finite windows one fused terminal packet after the shared $\DyadicRatCoreUp$ endpoint, radius, and distance ledger, the $\RegSeqRatUp$ readback rows, the $\RealUp$ seal, and the displayed support rows have been transported through $W_\ast$.

Next apply \autoref{thm:regseqrat-common-tail-real-boundary-equivalence}. The common-tail packet and the finite-request $\RealUp$ seal boundary determine one another exactly on the StreamName schedule and refinement rows, the dyadic endpoint-radius-distance ledger, the RegSeqRat classifier and regularity rows, the Real seal-source rows, and the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ provenance.

Now compare this boundary with the four-face public interface. By \autoref{thm:real-l10-four-face-finite-pullback-public-interface}, those same finite rows form the public pullback surface over $\DyadicRatCoreUp$, $\StreamNameUp$, $\RegSeqRatUp$, and $\RealUp$. Thus the common-tail presentation cannot add or remove a public row from the fused terminal presentation.

Finally transport the shared four-face surface to the public classifier by \autoref{thm:streamname-real-completion-classifier-pullback}. Agreement on the displayed rows is therefore sufficient for public classifier agreement, and any public classifier agreement reads back to the same displayed rows through the common-tail Real boundary equivalence. The no-choice factorization of \autoref{thm:streamname-real-completion-route-no-choice-factorization} excludes selectors, quotient stream equality, host sequence equality, chosen limits, completeness, metric-completeness data, countable choice, and bridge rows from serving as another boundary coordinate.
\end{proof}

\begin{theorem}[StreamName terminal fusion finite-request section exactness]
\label{thm:streamname-terminal-fusion-finite-request-section-exactness}
Let $\mathcal W$ be an accepted fused terminal-window packet for the
$\StreamNameUp,\DyadicRatCoreUp,\RegSeqRatUp,\RealUp$ route, and view its
terminal request as a finite-request section. Then the displayed rows of
$\mathcal W$ are exactly the finite-request section pullback rows: the
$\StreamNameUp$ finite window, the $\DyadicRatCoreUp$ endpoint, radius, and
distance ledger, the $\RegSeqRatUp$ classifier, regularity, and readback rows,
the $\RealUp$ seal-source rows, and the support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$

Conversely, any finite-request section carrying precisely those displayed rows
determines the same fused terminal-window packet on the public terminal
four-face route.

This identification exports no quotient stream equality, no host sequence
equality, no selected limit, no ambient completeness row, no
metric-completeness data, no countable-choice row, and no bridge row.
\end{theorem}
\begin{proof}
For the forward direction, first read $\mathcal W$ through
\autoref{thm:streamname-real-completion-terminal-window-fusion}. The fused
terminal packet is exhausted by the finite StreamName window, the DyadicRatCore
ledger, the RegSeqRat readback, the Real seal, and the displayed
$\BHist/\hsame/\Cont/\Pkg/\NameCert$ support rows.

Next view the same request as a finite-request section and apply
\autoref{thm:streamname-finite-request-section-pullback}. This identifies the
section with the same StreamName finite schedule, DyadicRatCore endpoint,
radius, and distance ledger, RegSeqRat classifier and regularity/readback
rows, Real seal-source rows, and support provenance.

For the reverse direction, start with a finite-request section carrying those
rows. Apply
\autoref{thm:streamname-finite-request-section-classifier-pullback} to recover
the public carrier and classifier verdict on the terminal four-face route.
Then apply
\autoref{thm:streamname-terminal-fusion-common-tail-real-boundary-pullback} to
repack the same row family as the common-tail Real boundary of one fused
terminal packet.

Finally use \autoref{thm:streamname-finite-request-section-nonescape}. It
removes quotient stream equality, host sequence equality, selected limits,
ambient completeness, metric-completeness data, countable choice, and bridge
rows from the section surface. Hence the finite-request section and the fused
terminal-window packet are interchangeable exactly on the displayed BEDC-visible
L10 rows.
\end{proof}

\begin{theorem}[StreamName terminal fusion finite-cover cocycle exactness]
\label{thm:streamname-terminal-fusion-finite-cover-cocycle-exactness}

Suppose the pairwise windows $\mathcal W_{ij}$ and triple windows $\mathcal W_{ijk}$ carry the pasted-source hypotheses for finite StreamName--$\RegSeqRatUp$ covers, each local terminal fused packet transports through a $\RegSeqRatUp$ common-tail packet, and every local handoff row consumes the same $\DyadicRatCoreUp$ endpoint, radius, and distance ledger together with the same $\RealUp$ seal-source row.

Then the family pastes to one terminal fused packet exactly when the pairwise restrictions expose the same common-tail Real boundary. The pasted packet has precisely the visible rows
$$
\StreamNameUp,\quad \DyadicRatCoreUp,\quad \RegSeqRatUp,\quad \RealUp
$$
where the $\StreamNameUp$ rows are the given finite windows, the $\DyadicRatCoreUp$ rows are the endpoint, radius, and distance ledger, the $\RegSeqRatUp$ rows are the common-tail classifier and regularity rows, and the $\RealUp$ row is the seal-source row. Its only support provenance is
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$

The exactness claim exports no quotient stream equality, no host sequence equality, no selected limit, no ambient $\RealUp$ completeness theorem, no metric-completeness datum, no countable-choice row, and no standard bridge row.
\end{theorem}
\begin{proof}
First build the finite StreamName source surface. Apply \autoref{thm:streamname-regseqrat-finite-cover-pasted-source-exhaustion} to the certified windows and their pairwise and triple overlap data. This produces one pasted finite source whose support rows remain the displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ provenance.

Next transport every local fused packet through the common-tail boundary. By \autoref{thm:regseqrat-common-tail-real-boundary-equivalence}, the $\RegSeqRatUp$ common-tail classifier and regularity rows expose exactly the same Real seal-source boundary as the dyadic endpoint, radius, and distance ledger carried by the local handoff rows.

Now apply \autoref{thm:real-finite-cover-seal-pasting-criterion}. The common-tail Real boundary is therefore the cocycle condition for pasting the local $\RealUp$ seal-source rows across the finite cover, and the shared $\DyadicRatCoreUp$ ledger prevents a second radius or distance coordinate from entering the seal.

Finally use \autoref{thm:streamname-terminal-fusion-common-tail-real-boundary-pullback}. It identifies agreement of the pairwise common-tail Real boundaries with agreement of the terminal fused StreamName packet on the four visible faces and the five support rows. Hence the local terminal fused packets paste to one terminal fused packet exactly in the stated case, while quotient stream equality, host sequence equality, chosen limits, completeness, metric-completeness data, countable choice, and bridge rows remain outside the boundary.
\end{proof}

$\DyadicRatCoreUp$ endpoint, radius, and distance ledger, the same
$\RegSeqRatUp$ common-tail classifier and regularity rows, and the same
$\RealUp$ seal-source row. Then pairwise common-tail $\RealUp$ boundary
agreement is equivalent to descent of the local fused terminal packets to one
finite-request section.

The descended section has exactly the visible rows consisting of the
$\StreamNameUp$ windows, the $\DyadicRatCoreUp$ endpoint, radius, and distance
ledger, the $\RegSeqRatUp$ common-tail classifier and regularity rows, the
$\RealUp$ seal-source row, and the support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
It exports no quotient stream equality, no host sequence equality, no selected
limit, no countable-choice row, no ambient $\RealUp$ completeness theorem, no
metric-completeness datum, and no standard bridge row.
\end{theorem}
\begin{proof}
For the descent direction, apply
\autoref{thm:streamname-terminal-fusion-finite-cover-cocycle-exactness}. The
pairwise common-tail $\RealUp$ boundary agreement is exactly the cocycle
condition for the certified local terminal packets to paste to one fused
terminal packet over the finite family.

Next read the pasted terminal packet as a finite request. Use
\autoref{thm:streamname-finite-request-section-pullback}. The pullback exposes
precisely the StreamName windows, the DyadicRatCore endpoint-radius-distance
ledger, the RegSeqRat classifier and regularity rows, the Real seal-source
row, and the displayed $\BHist/\hsame/\Cont/\Pkg/\NameCert$ support rows.
Thus the pasted packet descends to one finite-request section with no further

For the reverse direction, start with such a finite-request section. The

\autoref{thm:regseqrat-common-tail-real-boundary-equivalence} identifies those
rows with the common-tail $\RealUp$ boundary read by the RegSeqRat classifier
and regularity data.

Finally apply \autoref{thm:real-finite-cover-seal-pasting-criterion} and then
\autoref{thm:streamname-terminal-fusion-common-tail-real-boundary-pullback}.
The first theorem reads the shared boundary as the finite-cover seal pasting
criterion, while the second reads it back as agreement of the local fused

completeness, metric-completeness, and bridge rows therefore cannot be part of

\end{proof}

\begin{theorem}[StreamName terminal fusion refinement invariance]
\label{thm:streamname-terminal-fusion-refinement-invariance}
Let $\{\mathcal W_i\}_{i\in I}$ be a finite family of certified
$\StreamNameUp$ windows satisfying the hypotheses of
\autoref{thm:streamname-terminal-fusion-cocycle-section-descent}. Suppose
$\{\mathcal R_a\}_{a\in A}$ is a certified finite refinement of that family
whose restrictions preserve the same $\DyadicRatCoreUp$ endpoint, radius, and
distance ledger, the same $\RegSeqRatUp$ common-tail classifier and regularity
rows, the same $\RealUp$ seal-source row, and the same support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
Then the finite-request section descended from the refined family is the same
section descended from the original family. Consequently the public terminal
fused packet over the four faces
$$
\StreamNameUp,\quad \DyadicRatCoreUp,\quad \RegSeqRatUp,\quad \RealUp
$$
is unchanged by passing to the certified refinement and restricting back to the
original cover.

The invariant packet exports no quotient stream equality, no host sequence
equality, no selected limit, no countable-choice row, no ambient $\RealUp$
completeness theorem, no metric-completeness datum, and no standard bridge row.
\end{theorem}
\begin{proof}
First compare each original window with its refined restrictions. Apply
\autoref{thm:streamname-terminal-fusion-common-refinement-stability}. The
preserved endpoint-radius-distance ledger, common-tail classifier and
regularity rows, seal-source row, and displayed support rows make every
restriction read the same terminal fused packet as the original window from
which it is certified.

Next verify the cocycle on the refined cover. Use
\autoref{thm:regseqrat-common-tail-triple-overlap-cocycle} to identify the
triple-overlap RegSeqRat common-tail rows, and then apply
\autoref{thm:streamname-terminal-fusion-finite-cover-cocycle-exactness}. The
pairwise common-tail $\RealUp$ boundary condition for the refined windows is
therefore the same finite-cover cocycle condition as for the original family.

Now descend both finite covers. By
\autoref{thm:streamname-terminal-fusion-cocycle-section-descent}, the original
family and the certified refinement each produce one finite-request section
carrying exactly the displayed $\StreamNameUp$, $\DyadicRatCoreUp$,
$\RegSeqRatUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and
$\NameCert$ rows.

Finally restrict the refined section back along the refinement maps. The
common-refinement stability already identifies every restricted local terminal
packet with its original local packet, while the finite-cover cocycle
exactness identifies the pasted packet by precisely those rows. Hence the two
descended sections are the same finite-request section, and the public
terminal fused packet is invariant under the certified refinement. Since all
steps read only the visible four faces and the five support rows, quotient
stream equality, host sequence equality, selected limits, choice,
completeness, metric-completeness data, and bridge rows remain outside the
interface.
\end{proof}

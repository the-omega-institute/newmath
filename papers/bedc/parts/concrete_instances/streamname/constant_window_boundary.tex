\begin{theorem}[StreamName constant subwindow exactness]
\label{thm:streamname-constant-window-subwindow-exactness}
Let $a,b:\Hist$ be rational-history endpoints and let
$I\subseteq J$ be inhabited finite unary observation windows. For the
constant rational stream window package on $J$, the restricted package on
$I$ recovers exactly the same $\mathsf{RatHistoryClassifier}(a,b)$ row. No
extra stream classifier, completion witness, or host equality row is read by
the subwindow.
\end{theorem}
\begin{proof}
Project the finite constant stream window on $J$ through
\autoref{thm:rat-stream-name-finite-window-subwindow-restriction}. This gives
the same constant-stream carrier and classifier rows over the inhabited
subwindow $I$.

Apply
\autoref{thm:rat-stream-name-constant-finite-window-exactness} to the
restricted window. Because every observation in the window reads the same two
constant endpoints, the window classifier collapses to the single underlying
$\mathsf{RatHistoryClassifier}(a,b)$ row.

Finally use
\autoref{thm:streamname-public-finite-window-interface} to identify the
subwindow package as a public $\StreamNameUp$ finite-window interface. The
interface contributes only the displayed carrier, classifier, and ledger rows,
so no additional comparison or completion row is introduced.
\end{proof}

\begin{theorem}[StreamName constant RealUp boundary]
\label{thm:streamname-constant-window-real-boundary}
A finite-window $\RealUp$ consumer of the constant rational stream package may
read only the $\RatUp$ point classifier row
$\mathsf{RatHistoryClassifier}(a,b)$ and the finite $\mathsf{ProbeBundle}$ ledger
carried by the $\StreamNameUp$ window. It does not obtain a completeness
claim, a Cauchy-modulus row, or any equality beyond the displayed rational
point classifier.
\end{theorem}
\begin{proof}
Start with the public finite-window interface of
\autoref{thm:streamname-public-finite-window-interface}. It exposes the
window carrier, classifier, and finite $\mathsf{ProbeBundle}$ ledger as the only rows
available to a downstream consumer.

For a constant rational stream window, apply
\autoref{thm:streamname-constant-window-subwindow-exactness}. The classifier
read by any inhabited subwindow is exactly the point classifier
$\mathsf{RatHistoryClassifier}(a,b)$.

Thus a $\RealUp$ finite-window consumer can depend on the rational point row
and on the ledger naming the observed probes. The cited interface contains no
regularity, convergence, or completion field, so those obligations remain
outside this boundary.
\end{proof}

\begin{theorem}[StreamName constant RegSeqRat boundary]
\label{thm:streamname-constant-window-regseq-boundary}
A $\RegSeqRat$ construction may consume the same constant $\StreamNameUp$
finite-window evidence as a source of rational point-classifier and
$\mathsf{ProbeBundle}$ ledger data, while the regularity rows of $\RegSeqRat$ remain
separate obligations not supplied by the constant-window package.
\end{theorem}
\begin{proof}
Read the finite-window evidence through
\autoref{thm:streamname-public-finite-window-interface}. The rows available at
the boundary are the carried observations, the stream-name classifier, and the
finite ledger.

For constant rational streams, combine
\autoref{thm:rat-stream-name-finite-window-subwindow-restriction} with
\autoref{thm:rat-stream-name-constant-finite-window-exactness}. Any inhabited
subwindow keeps the same endpoints and therefore recovers the same
$\mathsf{RatHistoryClassifier}(a,b)$ row.

Repack these rows as the StreamName input to $\RegSeqRat$. Since none of the
cited window theorems asserts a regularity modulus, bounded variation row, or
sequence-comparison law, those rows must be supplied by the $\RegSeqRat$
certificate itself.
\end{proof}

\begin{theorem}[StreamName constant window overlap exactness]
\label{thm:streamname-constant-window-overlap-exactness}
Let $a,b:\Hist$ be rational-history endpoints, and let $I,J$ be inhabited
finite unary windows for constant rational streams over the same BEDC source
$\mathsf{BHist}$. Suppose $K$ is a certified common subwindow of $I$ and $J$,
with every probe in $K$ carried by $\mathsf{UnaryHistory}$ and by the finite
$\mathsf{ProbeBundle}$ ledgers of both windows. Then the restrictions of the two
constant-window packages to $K$ expose the same
$\mathsf{RatHistoryClassifier}(a,b)$ row, with the StreamName comparison read
through the displayed $\mathsf{hsame}$ rows on the overlap.

Consequently, any $\RealUp$ finite-window read surface built from the two
restricted packages has the same $\StreamNameUp$ component on $K$. The overlap
surface contains only the public $\StreamNameUp$ finite-window interface rows
over $\mathsf{BHist}$, $\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$,
$\mathsf{hsame}$, and $\mathsf{RatHistoryClassifier}(a,b)$; it does not use a
quotient, completion row, host index equality, or hidden stream datum.
\end{theorem}
\leanchecked{BEDC.Derived.StreamNameUp.StreamNameConstantWindow\_overlap\_exactness}
\begin{proof}
Restrict the $I$-window package to the certified overlap $K$. By
\autoref{thm:streamname-constant-window-subwindow-exactness}, the restricted
package recovers exactly the row $\mathsf{RatHistoryClassifier}(a,b)$, because
the ambient stream is constant and every probe of $K$ is an inhabited unary
observation.

Repeat the same restriction for the $J$-window package. The second application
of \autoref{thm:streamname-constant-window-subwindow-exactness} yields the same
endpoint row $\mathsf{RatHistoryClassifier}(a,b)$ on the same
$\mathsf{ProbeBundle}$ overlap ledger. The displayed $\mathsf{hsame}$ clauses on
$K$ therefore identify the two public StreamName components without comparing
host streams.

Now read both packages as possible inputs to a finite-window $\RealUp$ consumer.
By \autoref{thm:streamname-constant-window-real-boundary}, such a consumer may
read only the rational point-classifier row and the finite probe ledger from the
constant $\StreamNameUp$ window.

Finally apply
\autoref{thm:realup-finite-window-exhaustion-overlap-closure}. Since both
restricted read surfaces present the same overlap rows just listed, the RealUp
finite-window surface repacks the same StreamName component over $K$. No
quotient, completion witness, host index, or hidden stream equality is added in
that repacking.
\end{proof}

\begin{theorem}[StreamName constant window union exactness]
\label{thm:streamname-constant-window-union-exactness}
Let $a,b:\Hist$ be rational-history endpoints, and let $I,J$ be inhabited
finite unary constant rational $\StreamNameUp$ windows over the same BEDC
source $\mathsf{BHist}$. Suppose both windows expose the row
$\mathsf{RatHistoryClassifier}(a,b)$. Then the finite union window represented
by $\mathsf{bundleAppend}(I,J)$ exposes the same classifier row.

Every selected observation of $\mathsf{bundleAppend}(I,J)$ restricts back to one
of the original constant-window rows. A finite-window $\RealUp$ or
$\RegSeqRat$ consumer of this union window reads only $\StreamNameUp$,
$\mathsf{ProbeBundle}$, $\mathsf{UnaryHistory}$, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RatHistoryClassifier}(a,b)$. It receives no completion row, quotient,
host stream equality, or hidden selector.
\end{theorem}
\begin{proof}
Apply \autoref{thm:streamname-constant-window-subwindow-exactness} to the
inhabited window $I$. Its selected constant observations all read the displayed
row $\mathsf{RatHistoryClassifier}(a,b)$, and the subwindow proof uses only the
public finite-window StreamName rows.

Apply the same subwindow exactness theorem to $J$. Since the endpoints are the
same $a,b$, the second package exposes the identical rational-history
classifier row, again without a host stream comparison.

Use the finite-window structural laws recorded by
\autoref{thm:streamname-public-finite-window-interface}. The
$\mathsf{bundleAppend}$ membership law places every selected observation of the
union bundle in either the $I$ branch or the $J$ branch, and each branch has
already been reduced to the same constant-window classifier row.

Finally apply \autoref{thm:streamname-constant-window-real-boundary} and
\autoref{thm:streamname-constant-window-regseq-boundary}. These boundaries say
that downstream $\RealUp$ and $\RegSeqRat$ finite-window consumers can read only
the listed StreamName, probe-bundle, unary-history, transport, continuation,
package, and rational-history classifier rows. The appended union therefore
adds no completion, quotient, host stream equality, or hidden selector.
\end{proof}

\begin{theorem}[StreamName constant union RegSeqRat Real readback preservation]
\label{thm:streamname-constant-union-regseq-real-readback-preservation}
Let $a,b:\Hist$ be rational-history endpoints, and let $I,J$ be inhabited
finite unary constant rational windows over the same $\mathsf{BHist}$ source.
Suppose both windows expose the row $\mathsf{RatHistoryClassifier}(a,b)$. Then
the union window $\mathsf{bundleAppend}(I,J)$ preserves the same selected
$\RegSeqRatUp$ tail read and the same finite-window $\RealUp$ readback surface.

The preserved readback surface is exactly the rows $\StreamNameUp$,
$\mathsf{ProbeBundle}$, $\mathsf{UnaryHistory}$, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RatHistoryClassifier}(a,b)$. It contains no completion row, quotient,
host stream equality, hidden selector, or selected real limit.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:streamname-constant-window-union-exactness}. The appended bundle
has no classifier branch except the common row
$\mathsf{RatHistoryClassifier}(a,b)$, because every selected observation comes
from the $I$ side or the $J$ side and both sides expose the same endpoint row.

Next use \autoref{thm:streamname-constant-window-regseq-boundary}. The common
classifier row and finite $\mathsf{ProbeBundle}$ ledger repack as the
$\RegSeqRatUp$ finite-window input, while the regularity and tail obligations
remain external to the constant-window package.

Then apply \autoref{thm:streamname-constant-window-real-boundary}. The
$\RealUp$ readback uses the same rational point-classifier row and the same
finite probe ledger already read by the $\RegSeqRatUp$ boundary. It receives no
completion field or selected limit from the appended window.

Finally exhaust the coordinates of the public StreamName boundary. The only rows
used in the three cited results are $\StreamNameUp$, $\mathsf{ProbeBundle}$,
$\mathsf{UnaryHistory}$, $\hsame$, $\Cont$, $\Pkg$, and
$\mathsf{RatHistoryClassifier}(a,b)$, so the union cannot contribute a quotient,
host stream equality, hidden selector, completion row, or selected real limit.
\end{proof}

\begin{theorem}[StreamName constant directed-overlap exactness]
\label{thm:streamname-constant-window-directed-overlap-exactness}
Let $a,b:\Hist$ be rational-history endpoints, and let
$\{I_i\}_{i\in I}$ be a directed finite family of inhabited constant rational
$\StreamNameUp$ windows over the same $\mathsf{BHist}$ source. For every
certified common overlap in the directed family, the restricted windows expose
the same $\mathsf{RatHistoryClassifier}(a,b)$ row, with comparison read only
through the displayed $\mathsf{hsame}$ rows and the finite
$\mathsf{ProbeBundle}$ ledgers.

Consequently, any $\RegSeqRat$ or $\RealUp$ finite-window consumer of the
directed family reads only that single rational-history classifier row together
with the finite $\mathsf{ProbeBundle}$ ledger. The directed system supplies no
quotient row, completion row, host stream equality, hidden regularity modulus,
or external finite-window selector.
\end{theorem}
\begin{proof}
Start with one certified common overlap in the directed family. By
\autoref{thm:streamname-constant-window-overlap-exactness}, the two restricted
constant-window packages on that overlap expose the same
$\mathsf{RatHistoryClassifier}(a,b)$ row and the same finite probe ledger.

Now pass along the directed finite family. Iterating the overlap exactness
statement over the directed overlaps identifies each exposed constant-window
classifier with the same endpoint row. Directedness ensures that every pair of
member windows is compared inside a certified common window already carrying
the displayed $\mathsf{hsame}$ and $\mathsf{ProbeBundle}$ rows.

For a $\RegSeqRat$ consumer, apply
\autoref{thm:streamname-constant-window-regseq-boundary}. The consumer may use
the constant StreamName row and the finite probe ledger, while all regularity
obligations remain external to the constant-window boundary.

For a $\RealUp$ consumer, apply
\autoref{thm:streamname-constant-window-real-boundary}. It reads the same
rational-history classifier row and the same finite probe ledger, with no
completion or host equality data added by the directed overlap system.
\end{proof}

\begin{theorem}[StreamName constant directed RegSeqRat Real tail boundary]
\label{thm:streamname-constant-window-directed-regseq-real-tail-boundary}
Let $a,b:\Hist$ be rational-history endpoints, and let
$\{I_i\}_{i\in I}$ be a directed finite family of inhabited constant rational

$\mathsf{RatHistoryClassifier}(a,b)$ row, the same finite
$\mathsf{ProbeBundle}$ ledger, and the displayed $\mathsf{UnaryHistory}$,

factoring through that $\RegSeqRat$ surface, obtains one selected
$\RegSeqRat$ tail-classifier row. The handoff surface is exhausted by the
$\StreamNameUp$ finite-window rows, the selected $\RegSeqRat$ spine,
$\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$, $\hsame$, $\Cont$, $\Pkg$,
and the rational classifier $\mathsf{RatHistoryClassifier}(a,b)$. It does not
read completion, quotient, host stream equality, order or arithmetic rows, or
standard bridge rows.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:streamname-constant-window-directed-overlap-exactness}. The
directed family presents one rational-history classifier row on every
certified overlap, and the comparison uses only the displayed $\hsame$ rows
and finite $\mathsf{ProbeBundle}$ ledgers.

Next use \autoref{thm:streamname-constant-window-regseq-boundary}. It repacks
the constant $\StreamNameUp$ evidence as the finite StreamName input that a
$\RegSeqRat$ construction may consume, while leaving regularity rows as
$\RegSeqRat$ obligations rather than StreamName data.

For a finite-window $\RealUp$ consumer, use
\autoref{thm:streamname-constant-window-real-boundary}. The RealUp side can
read the same rational classifier and the same finite probe ledger, but it
does not add completion or host-equality information to the window surface.

Finally apply \autoref{thm:regseqrat-tail-classifier-exactness}. The matching

and shared $\Cont/\hsame$ ledger determine a single selected tail-classifier

\begin{theorem}[StreamName constant directed RealUp source uniqueness]
\label{thm:streamname-constant-window-directed-realup-source-uniqueness}
Let $a,b:\Hist$ be rational-history endpoints, and let
$\{I_i\}_{i\in I}$ and $\{J_j\}_{j\in J}$ be two directed finite families of
inhabited constant rational $\StreamNameUp$ windows over the same
$\mathsf{BHist}$ source. Suppose a finite-window $\RealUp$ consumer of the
first family and a finite-window $\RealUp$ consumer of the second family overlap
on a certified common directed subwindow $K$, with the displayed
$\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$, $\hsame$, $\Cont$, and $\Pkg$
rows inherited from both directed systems.

Then the public $\StreamNameUp$ component exposed to the two $\RealUp$
consumers on $K$ is uniquely the constant rational stream row determined by
$\mathsf{RatHistoryClassifier}(a,b)$ and the finite $\mathsf{ProbeBundle}$
ledger on $K$. No alternative $\RegSeqRat$ source row, $\RealUp$ source row, or
host stream row can be read at that overlap without first passing through the
same $\StreamNameUp$ finite-window carrier and classifier.
\end{theorem}
\begin{proof}
Restrict both directed constant-window families to the common directed
subwindow $K$. The restricted packages still carry the same
$\mathsf{BHist}$ source, the inherited unary observation rows, and the finite
$\mathsf{ProbeBundle}$ ledgers displayed by the two original systems.

Apply
\autoref{thm:streamname-constant-window-directed-overlap-exactness} to the
restricted directed families. It identifies the public StreamName component on
$K$ with the single rational-history classifier row
$\mathsf{RatHistoryClassifier}(a,b)$, using only the displayed $\hsame$ rows
and the finite probe ledgers.

Next use \autoref{thm:streamname-constant-window-real-boundary}. A
finite-window $\RealUp$ consumer may inspect only that StreamName carrier,
classifier, and ledger surface. Thus the consumer has no independent access to
a completion row, host stream equality, or an external source selector.

Finally apply
\autoref{thm:streamname-constant-window-directed-regseq-real-tail-boundary}.
Any $\RegSeqRat$ tail seen by the $\RealUp$ side is selected through the same
constant StreamName finite-window rows and the displayed $\Cont/\hsame$ ledger.
Therefore every available RealUp source row factors through the unique
StreamName component on $K$.
\end{proof}

\begin{theorem}[StreamName constant directed RealUp seal classifier determinacy]
\label{thm:streamname-constant-directed-realup-seal-classifier-determinacy}
Let $a,b:\Hist$ be rational-history endpoints, and let
$\{I_i\}_{i\in I}$ and $\{J_j\}_{j\in J}$ be directed finite inhabited
constant rational $\StreamNameUp$ window families over those endpoints.
Assume that their directed overlap rows expose the same
$\mathsf{RatHistoryClassifier}(a,b)$ row, the same finite
$\mathsf{ProbeBundle}$ ledger, and the displayed $\mathsf{UnaryHistory}$,
$\hsame$, $\Cont$, and $\Pkg$ rows. Let two finite-window $\RealUp$ seals
consume these families through $\RegSeqRat$ stream-certificate spines
compatible with the displayed constant-window rows.

Then the two seals export the same $\RealUp$ observation classifier row. The
classifier factors through the constant $\StreamNameUp$ finite-window package,
the $\RegSeqRat$ constant stream spine, and the displayed seal handoff. It
reads no completion, quotient, host stream equality, hidden selector,
arithmetic or order package, or standard bridge.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:streamname-constant-window-directed-realup-source-uniqueness}.
The two seals see the same public $\StreamNameUp$ source on every common
directed subwindow, namely the constant rational stream row determined by
$\mathsf{RatHistoryClassifier}(a,b)$ and the shared finite
$\mathsf{ProbeBundle}$ ledger.

Next use \autoref{thm:regseqrat-reindexed-constant-stream-spine}. The common
constant StreamName row supplies compatible $\RegSeqRat$ stream-certificate
spines for the reindexed constant streams. Apply
\autoref{thm:regseqrat-constant-stream-seal-classifier-package} to those
spines and to the displayed seal classifier row; the resulting package is the
RealUp observation classifier exported by each finite-window seal.

Finally use
\autoref{thm:streamname-constant-window-directed-regseq-real-tail-boundary}.
That boundary keeps the selected $\RegSeqRat$ tail and the RealUp handoff
inside the displayed $\mathsf{UnaryHistory}$, $\mathsf{ProbeBundle}$,
$\hsame$, $\Cont$, $\Pkg$, and rational-classifier rows. Since both seals
consume exactly that finite surface, no completion, quotient, host stream
equality, hidden selector, arithmetic or order package, or standard bridge can
distinguish their exported classifier rows.
\end{proof}

\section{StreamName finite-request section packet}
\label{sec:streamname-finite-request-section-packet}

\begin{theorem}[StreamName Real endpoint finite-request section]
\label{thm:streamname-real-endpoint-finite-request-section}
Every finite $\StreamNameUp$ window request that is displayed to a $\RealUp$
endpoint row factors through a section packet whose visible rows are exactly
the endpoint rows, the certified finite-window observations, the
$\RegSeqRatUp$ readback rows attached to those observations, the
$\DyadicRatCoreUp$ budget rows used by the read, the componentwise $\hsame$
transport, the $\Cont$ replay route, $\Pkg$ provenance, and the local
$\NameCert$ rows. The section has no coordinate for quotient stream equality,
host sequence equality, selected limits, host real equality, or ambient
completion.
\end{theorem}
\begin{proof}
Start with the finite observation packet of
\autoref{thm:streamname-finite-observation-section}. It projects a displayed
window section to the membership, unary-history, rational-history classifier,
$\hsame$, $\Cont$, $\Pkg$, and local $\NameCert$ rows.

Next apply the Real exit handoff of
\autoref{thm:streamname-finite-observation-real-exit-handoff}. The endpoint
consumer reads that same section through $\StreamNameUp$ before the downstream
$\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$ faces are consumed.

Finally use
\autoref{thm:streamname-regseqrat-real-budget-factorization}. It adds exactly
the RegSeqRat readback face, the DyadicRatCore budget face, and the RealUp
seal face to the StreamName section. Since the cited packets contain no
quotient stream, host sequence, selected-limit, host-real, or ambient
completion coordinate, none can appear in the endpoint request.
\end{proof}

\begin{theorem}[StreamName RegSeqRat window-budget section]
\label{thm:streamname-regseqrat-window-budget-section}
Every finite $\StreamNameUp$ window request consumed by a $\RegSeqRatUp$
readback and then by a $\RealUp$ endpoint seal is budgeted by one displayed
section packet. The packet displays the finite endpoint rows, RegSeqRat
readback rows, DyadicRatCore budget rows, StreamName window rows,
componentwise $\hsame$ transport, $\Cont$ replay, $\Pkg$ provenance, and
local $\NameCert$ rows, and every downstream read is classified by those rows.
It exports no quotient stream equality, no host sequence equality, no selected
limit, no second dyadic source, and no ambient completion datum.
\end{theorem}
\begin{proof}
First read the request through
\autoref{thm:streamname-tail-compatible-window-request}. The finite window is
determined by its certified unary observation, bundle membership, tail route,
window budget, and pointwise RegSeqRat handoff.

Next apply \autoref{thm:streamname-tail-compatible-window-coverage}. It forces
every accepted downstream RegSeqRat or RealUp read to factor through that
request together with the displayed $\hsame$, $\Cont$, $\Pkg$, and
$\NameCert$ support rows.

Now use \autoref{thm:streamname-regseqrat-real-budget-route-exactness}. The
accepted route is exactly the four finite faces $\StreamNameUp$,
$\DyadicRatCoreUp$, $\RegSeqRatUp$, and $\RealUp$ with the same support rows,
so the endpoint, readback, and budget data are one section packet rather than
separate routes.

The non-escape clauses in
\autoref{thm:streamname-tail-compatible-window-nonescape} and
\autoref{thm:streamname-regseqrat-real-budget-nonescape} remove quotient
streams, host sequence equality, selected limits, second dyadic sources, and
ambient completion data from the packet.
\end{proof}

\begin{theorem}[StreamName Real seal request section nonescape]
\label{thm:streamname-real-seal-request-section-nonescape}
Every finite StreamName request that reaches a $\RealUp$ seal through
$\RegSeqRatUp$ and $\DyadicRatCoreUp$ is exhausted by the finite-request
section packet of
\autoref{thm:streamname-real-endpoint-finite-request-section} and
\autoref{thm:streamname-regseqrat-window-budget-section}. Thus the Real seal
can replay only endpoint rows, RegSeqRat readback rows, DyadicRatCore budget
rows, StreamName window rows, $\hsame$ transport, $\Cont$ routes, $\Pkg$
provenance, and local $\NameCert$ rows. It cannot obtain quotient equality of
streams, quotient equality of Cauchy presentations, selected limits, host real
equality, metric-completeness data, or ambient completion.
\end{theorem}
\begin{proof}
Begin with
\autoref{thm:streamname-real-endpoint-finite-request-section}. It fixes the
Real endpoint read as a finite StreamName section together with the RegSeqRat,
DyadicRatCore, RealUp, and BEDC support rows.

Then apply \autoref{thm:streamname-regseqrat-window-budget-section}. The
RegSeqRat readback and DyadicRatCore budget rows are the same section packet
used by the endpoint seal, so the Real seal does not receive an independent
readback source or a second budget source.

Finally compose with
\autoref{thm:streamname-regseqrat-real-budget-nonescape}. The four-face
budget boundary contains no coordinate for quotient stream equality, quotient
Cauchy-presentation equality, selected limits, host real equality,
metric-completeness data, or ambient completion. The finite-request section
therefore reaches the Real seal only through the displayed rows.
\end{proof}

\begin{theorem}[StreamName finite-request terminal coherence]
\label{thm:streamname-finite-request-terminal-coherence}
Any finite $\StreamNameUp$ request that is read both as a $\RealUp$ endpoint
finite-request section and as a $\RegSeqRatUp$ window-budget section
determines a unique terminal public row. That row is the four-face
Real-and-Completion packet
$$
\StreamNameUp,\quad \DyadicRatCoreUp,\quad \RegSeqRatUp,\quad \RealUp
$$
together with only the displayed support rows
$$
\BHist,\quad \hsame,\quad \Cont,\quad \Pkg,\quad \NameCert.
$$
The same packet is therefore the terminal exit route for the request, and it
contains no quotient stream equality, no quotient equality of Cauchy
presentations, no host sequence equality, no selected limit, no host real
equality, no ambient or metric-completeness datum, no countable-choice row, no
Classical row, and no standard bridge row.
\end{theorem}
\begin{proof}
First apply
\autoref{thm:streamname-real-endpoint-finite-request-section}. The endpoint
reading fixes the finite request as a $\RealUp$ section with the StreamName
window, RegSeqRat readback, DyadicRatCore budget, RealUp seal, and the
displayed $\BHist$, $\hsame$, $\Cont$, $\Pkg$, and $\NameCert$ rows.

Next apply \autoref{thm:streamname-regseqrat-window-budget-section}. The
RegSeqRat window-budget reading uses the same finite section packet, so the
readback and budget faces are not second routes parallel to the endpoint
section.

Now compose with
\autoref{thm:streamname-real-seal-request-section-nonescape}. It removes every
Real-seal escape coordinate outside the displayed section rows. Invoke
\autoref{thm:streamname-real-completion-four-object-exit-route} to identify
those rows with the four-object Real-and-Completion exit packet.

Finally use \autoref{thm:streamname-open-phase-four-face-terminality}. Once
the StreamName window, DyadicRatCore ledger, RegSeqRat readback, RealUp seal,
and displayed support rows agree, there is only one terminal finite BEDC route
for the request. The exclusions are exactly the non-escape exclusions of
\autoref{thm:streamname-regseqrat-real-budget-nonescape}, so no quotient,
selected-limit, ambient-completeness, choice, Classical, or bridge coordinate
can replace the terminal four-face packet.
\end{proof}

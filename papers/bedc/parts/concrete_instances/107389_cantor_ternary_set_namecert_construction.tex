\chapter{A Concrete Naming Certificate for $\CantorTernarySetUp$}
\label{ch:concrete-instances-cantor-ternary-set-namecert}
\origin{human}

Following Cantor's middle-third construction and Bishop--Bridges constructive analysis, the $\CantorTernarySetUp$ packet records the traditional ternary description of the Cantor set as finite BEDC data. It refines the retained-prefix and deleted-gap surface of \autoref{ch:concrete-instances-cantorset-namecert} by making the inspected base-three windows explicit, and it routes every real-facing read through \autoref{ch:concrete-instances-real-namecert}, \autoref{ch:concrete-instances-regseqrat-namecert}, \autoref{ch:concrete-instances-streamname-namecert}, and \autoref{ch:concrete-instances-dyadicratcore-namecert}. The packet names only finite ternary windows, triadic endpoint ledgers, gap checks, regular-rational readback, and the terminal Real seal; it does not assert host powersets, infinite branch choice, quotient real equality, measure, compactness, or ambient $\RealUp$ completeness.

\begin{definition}[Cantor ternary set carrier]
\label{def:cantor-ternary-set-carrier}
A $\CantorTernarySetUp$ carrier is a finite $\BHist$ packet
$$
T=(W,D,V,G,I,R,E,H,C,P,N)
$$
with the following displayed rows:
\begin{enumerate}
\item $W$ is the $\StreamNameUp$ window row selecting the finite base-three digit positions currently inspected.
\item $D$ is the ternary digit ledger for $W$, and every accepted digit row is tagged as a retained digit $0$ or $2$.
\item $V$ is the triadic place-value row, evaluating the inspected prefix as a rational with denominator a displayed power of $3$.
\item $G$ is the deleted-middle gap ledger, listing the middle-third intervals excluded by every inspected non-retained alternative at the displayed depths.
\item $I$ is the retained interval row inherited from $\CantorSetUp$, with endpoints computed from the same triadic prefix and checked against the gap ledger.
\item $R$ is the $\RegSeqRatUp$ readback row reached from $W,D,V,G,I$.
\item $E$ is the $\RealUp$ seal row exposed only after the regular-rational readback has been displayed.
\item $H$ records componentwise $\hsame$ transport for the window, digit, triadic value, gap, interval, readback, Real seal, continuation, provenance, and local name rows.
\item $C$ records displayed $\Cont$ replay from window requests through the digit ledger, triadic value ledger, gap ledger, retained interval, regular readback, and Real seal.
\item $P$ is the $\Pkg$ provenance over $\CantorTernarySetUp$, $\CantorSetUp$, $\StreamNameUp$, $\RegSeqRatUp$, $\DyadicRatCoreUp$, $\RealUp$, $\BHist$, $\hsame$, $\Cont$, and $\NameCert$.
\item $N$ is the local $\NameCert_{\CantorTernarySetUp}$ row.
\end{enumerate}
The classifier accepts a packet only when $W,D,V,G,I,R,E,H,C,P,N$ are all displayed for the same finite triadic window, the digit ledger uses only retained digits, the gap ledger refuses the corresponding middle thirds, and every $\RealUp$ consumer enters through $R$ and $E$. It has no coordinate for a host subset of $\RealUp$, selected infinite ternary stream, quotient real equality, endpoint convention selector, measure, open-cover compactness, countable choice, or ambient completeness.
\end{definition}

\begin{theorem}[Cantor ternary set NameCert obligations]
\label{thm:cantor-ternary-set-namecert-obligations}
For an accepted $\CantorTernarySetUp$ carrier
$$
T=(W,D,V,G,I,R,E,H,C,P,N),
$$
the local $\NameCert_{\CantorTernarySetUp}$ surface consists exactly of carrier admission, finite ternary-window classification, retained-digit checking, triadic place-value evaluation, middle-third gap exclusion, retained-interval compatibility, $\RegSeqRatUp$ readback, $\RealUp$ seal handoff, componentwise transport, continuation replay, provenance, and local naming through the displayed rows $W,D,V,G,I,R,E,H,C,P,N$.
\end{theorem}
\begin{proof}
Carrier admission projects the finite tuple of \autoref{def:cantor-ternary-set-carrier}. The window row $W$ fixes the inspected positions, and the digit ledger $D$ is the only source of ternary digit information for those positions.

The retained-digit and triadic-value checks are then internal to the same finite prefix. The row $V$ evaluates exactly the inspected prefix, while the gap row $G$ records the deleted middle thirds rejected by replacing a retained digit with the middle alternative.

The retained interval row $I$ is inherited from the $\CantorSetUp$ surface and is therefore tied to the same prefix and gap ledger. The regular-rational row $R$ and Real seal row $E$ are downstream of that finite data.

Finally, $H,C,P,N$ transport, replay, source, and name the same packet. Since no carrier coordinate stores host subsets, infinite streams, quotient equality, endpoint selectors, measure, compactness, choice, or ambient completeness, the displayed rows exhaust the local NameCert obligations.
\end{proof}

\begin{theorem}[Cantor ternary set triadic nonescape]
\label{thm:cantor-ternary-set-triadic-nonescape}
Every $\RealUp$-facing read of an accepted $\CantorTernarySetUp$ carrier remains on the finite triadic route
$$
W,D,V,G,I\longmapsto R\longmapsto E
$$
together with $H,C,P,N$. A consumer may inspect retained ternary windows, deleted middle thirds, triadic endpoint values, regular-rational readback, and the terminal Real seal, but it cannot extract a selected infinite ternary branch, a host real subset, quotient equality of real representatives, a measure row, countable choice, open-cover compactness, or ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
Start with an accepted carrier. Any real-facing consumer must first enter through the displayed window $W$, because no digit row outside that finite window is part of the packet.

The digit ledger $D$, triadic value row $V$, and gap ledger $G$ determine the retained middle-third decision at the inspected depths. The interval row $I$ packages the same decision as a $\CantorSetUp$ retained-prefix row, so there is no separate branch or endpoint source.

The handoff to real data is forced through $R$ and $E$. The regular-rational row $R$ is reached only after the finite triadic rows have been displayed, and the Real seal $E$ is downstream of $R$.

The structural rows $H,C,P,N$ preserve and name this same route. Thus every exported read remains on the finite triadic packet, and the excluded host subset, infinite branch, quotient equality, measure, choice, compactness, and completeness data are not available to the consumer.
\end{proof}

\begin{theorem}[Cantor ternary set CantorSet dependency]
\label{thm:cantor-ternary-set-cantor-set-dependency}
Every accepted $\CantorTernarySetUp$ packet factors through the retained-prefix and deleted-gap surface of \autoref{ch:concrete-instances-cantorset-namecert} before any $\RealUp$-facing seal is read. After that CantorSet row is displayed, the remaining route uses $\RegSeqRatUp$, $\StreamNameUp$, and $\DyadicRatCoreUp$ rows to reach the terminal real-name boundary, without asserting compactness, measure, host powersets, or ambient $\RealUp$ completeness.
\end{theorem}
\begin{proof}
Project the carrier of \autoref{def:cantor-ternary-set-carrier}. The row $I$ is the retained interval row inherited from the CantorSet sibling, and it is computed from the same finite triadic prefix and deleted-gap ledger as $W,D,V,G$.

By \autoref{thm:cantor-ternary-set-triadic-nonescape}, every real-facing read remains on the route $W,D,V,G,I\longmapsto R\longmapsto E$. Thus the CantorSet retained-prefix and deleted-gap row is consumed before the regular-rational readback $R$ and Real seal $E$ are available.

The remaining rows $R,E,H,C,P,N$ use the RegSeqRat, StreamName, and DyadicRatCore surfaces named in the carrier provenance. None of those rows supplies compactness, measure, host powersets, or ambient real completeness, so the dependency is exactly the finite sibling consumption stated above.
\end{proof}

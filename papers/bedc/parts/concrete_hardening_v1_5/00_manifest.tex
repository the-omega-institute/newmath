\chapter{Concrete Instance Hardening Manifest v1.5}
\label{ch:v15-manifest}

This increment is deliberately small. It does not add new derived objects and it does not rewrite the frozen core. It hardens the v1.4 concrete-instance layer by replacing three remaining meta-level conveniences with explicit proof obligations.

\begin{enumerate}
  \item Domain membership is internalized as a judgment \(\InDom(D,h)\), replacing the expository notation \(h\in D\) in concrete gap statements.
  \item Signature packages are generated by a token-introduction judgment \(\TokIntro(\Pi,s,p)\), rather than by a function-like notation \(\Tok_{\Pi}(s)\).
  \item Package reflection is made explicit: package-sameness in the concrete signature instance reflects history-sameness of the signatures that generated the packages.
  \item The \(\AddUp\) certificate is strengthened by adding a commutativity obligation for unary continuation. Until that obligation is supplied, the conservative live name is ``unary continuation monoid-like interface.''
\end{enumerate}

\paragraph{Non-goal.}
This part does not expand the real-like, circle-like, fold-like, or completion-like layers. It only hardens the first concrete finite-kernel instance.

\paragraph{Live correction rule.}
For active development, the v1.5 statements in this part refine the v1.4 concrete-instance modules. The v1.4 modules remain as historical paper constructions, but the current live obligations are those stated here.

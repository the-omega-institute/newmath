\chapter{Machine-Checkable Milestones}
\label{ch:hardening-machine-checkable-milestones}

This chapter converts the current proof-sprint plan into a milestone schedule. It is implementation-facing, but it is written without depending on any particular proof assistant syntax.

\section{Milestone A: history kernel}

\begin{protocol}[Milestone A]
Milestone A is complete when the following facts have been checked:
\begin{enumerate}
  \item \(\MSame\) is reflexive, symmetric, and transitive.
  \item raw marks satisfy no-confusion: \(\bzero\) and \(\bone\) are not cross-identified.
  \item \(\HSame\) is reflexive, symmetric, and transitive.
  \item history constructors satisfy no-confusion.
  \item \(\Ext\) is deterministic up to \(\HSame\).
  \item \(\Cont\) is deterministic up to \(\HSame\).
  \item \(\Cont\) has left and right units.
  \item \(\Cont\) is associative up to \(\HSame\).
\end{enumerate}
\end{protocol}

\section{Milestone B: signature kernel}

\begin{protocol}[Milestone B]
Milestone B is complete when the following facts have been checked:
\begin{enumerate}
  \item bundles are generated by \(\Bnil\) and \(\Bcons\);
  \item signature events are generated by the empty and cons signature rules;
  \item under \(\AskPol(\Pi,D)\), signature generation is deterministic up to \(\HSame\);
  \item under \(\AskPol(\Pi,D)\), \(\SameSig\) is reflexive, symmetric, and transitive on \(D\).
\end{enumerate}
\end{protocol}

\section{Milestone C: package and gap kernel}

\begin{protocol}[Milestone C]
Milestone C is complete when the following facts have been checked:
\begin{enumerate}
  \item package policies provide existence, extensionality, and grounding;
  \item gap policies provide coverage, separation, and signature determination;
  \item packages classify by signatures;
  \item gap ledgers are determined by signatures and package tokens;
  \item composite gap ledgers satisfy composite coverage.
\end{enumerate}
\end{protocol}

\section{Milestone D: first derived interface seeds}

\begin{protocol}[Milestone D]
Milestone D is complete when the following certificates have been constructed:
\begin{enumerate}
  \item the unary repetition certificate for \(\NatUp\);
  \item the continuation certificate for \(\AddUp\);
  \item a descent-certificate template for function-like derived interfaces.
\end{enumerate}
\end{protocol}

\section{Settled-kernel criterion}

\begin{definition}[Settled-kernel criterion]
The finite kernel becomes a settled basis for further development once Milestones A, B, and C have been checked in a concrete formal environment and the statements of Milestone D are internally consistent with the checked kernel.
\end{definition}

\begin{remark}
Until then, large derived modules such as real-like interfaces, phase geometry, fold normalization, and arithmetic beyond addition should remain non-blocking appendices.
\end{remark}

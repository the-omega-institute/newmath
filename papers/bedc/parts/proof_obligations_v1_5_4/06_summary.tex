
\chapter{v1.5.4 Summary}
\label{ch:v154-summary}

The v1.5.4 increment is a minimal verification contract. Its purpose is to make the next checker sprint unambiguous.

\begin{enumerate}
  \item The active concrete package-sameness relation is \(\psame_{\Pi}^{\mathrm{base}}\), a one-constructor relation.
  \item \(\psame_{\Pi}^{\mathrm{sig}}\) is read as an alias for \(\psame_{\Pi}^{\mathrm{base}}\) in this release.
  \item Any equivalence closure must be written separately as \(\psame_{\Pi}^{\mathrm{eq}}\) and cannot inherit reflection without \(\ClosureReflect(\Pi)\).
  \item Token uniqueness remains an active paper policy, while canonical-constructor uniqueness is recorded as an implementation target.
  \item Exact concrete Globalize is exported only for the base relation.
  \item The additive name \(\AddUp\) remains conditional until unary continuation commutativity is checked.
\end{enumerate}

The next useful release should be a checked proof report, not another broad theory extension.

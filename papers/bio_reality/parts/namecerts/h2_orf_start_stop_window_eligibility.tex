\subsection{Start--Stop Window Eligibility as a Finite Boundary Test}
\label{sec:h2-orf-start-stop-window-eligibility}

\paragraph{Claim.}
The record named \texttt{h2.orf.start\_stop.window\_eligibility} concerns only the finite eligibility question obtained by enumerating start and stop codons inside an externally supplied nucleotide window. Its biological object is open-reading-frame eligibility from start/stop codon enumeration, and its layer is \texttt{orf\_eligibility}. The verified run \texttt{c26a0d6656f7} loaded one external window from a curated finite nucleotide-window manifest dated 2026-05-25. The run reports \texttt{external\_window\_dataset\_loaded}=1, \texttt{window\_count}=1, and classified all supplied windows by start/stop enumeration:
$$
\begin{aligned}
\text{input windows} &= 1,\\
\text{classified windows} &= 1.
\end{aligned}
$$
The scope is exactly \texttt{external\_window\_start\_stop\_enumeration\_only}. This means that the computation is a finite boundary classifier over already supplied nucleotide-window records, not a derivation of an observed gene from codon geometry alone.

The ORF layer therefore treats geometry as a classifier input only after a separate biological interval has been supplied.

\paragraph{Local boundary.}
The verified window record explicitly says \texttt{no\_geometry\_to\_translation\_promotion}=\texttt{orf\_eligibility\_only}. The self-check \texttt{0f47d2f0cf41} records that no codon geometry is promoted across biological layers and that three gates were checked.

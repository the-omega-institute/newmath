\subsection{A punctured-cube row in the standard-code display}\label{sec:bioreality-h0-r-punctured-cube}\origin{ai}

\paragraph{Finite codon object.} The packet starts with the ordinary six-bit codon display of the standard genetic code, read only as a table of codon labels.  The distinguished row is the thirteen-codon set
$$\begin{aligned}
R=\{&AAA,AGA,AGG,AUA,CUA,CUC,CUG,CUU,\\
&UAA,UAG,UCA,UGA,UUA\}.
\end{aligned}$$
The independent cardinality check gives $|R|=13$, and the same computation returns the expected list of codons with no extra or missing row entries.  In the local cube complex determined by the window-six coordinates, this row has the face count
$$\begin{aligned}
f(R)=(f_0,f_1,f_2)=(13,16,4).
\end{aligned}$$
The term \emph{punctured cube} is used here in this finite combinatorial sense: the row occupies thirteen vertices, supports sixteen local edges, and carries four square faces in the induced tile.  This is a code-read observation, not a statement about biochemical realization.

\paragraph{Scope discipline.} The contact used here is a curated standard-code table assigning codons to amino-acid or stop labels.  That contact is sufficient for codon-label readback, local row membership, local face counting, null-model comparison, and leave-one tests at the code layer.  It is not a contact with translation dynamics, folded structure, physical admissibility, biochemical function, a universal mechanism, or organism-level necessity.  Therefore no claim of translation, folding, structural order, physical viability, function, mechanism, or universality follows from this packet unless a separate empirical contact supplies that layer.  In particular, amino-acid label readback is not protein structure, and the local punctured-cube geometry is not a biological law.

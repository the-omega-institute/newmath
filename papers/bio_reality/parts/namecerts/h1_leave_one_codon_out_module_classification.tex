\subsection{Window-Six Module Classification for Leave-One-Codon Recovery}
\label{sec:codon-window-six-module-classification}

\paragraph{Finite codon display.}
The record concerns the standard-code codon table after it has been placed in a six-coordinate codon display.  The biological contact used here is only the curated assignment table from codons to amino-acid or stop labels.  The mathematical carrier is the finite codon cube on the $64$ codons, while the tested row is the exceptional boundary set selected by the window-six construction.  In the verified run, this set has cardinality $13$ and is exactly
$$
\begin{aligned}
R=\{&AAA, AGA, AGG, AUA, CUA, CUC, CUG, CUU,\\
&UAA, UAG, UCA, UGA, UUA\}.
\end{aligned}
$$
The same run identifies $R$ as a punctured cubical tile with $f$-vector $(13,16,4)$.  Thus the object under discussion is not an unstructured list of codons: it is a finite boundary-shaped row in the six-coordinate display, and the label table is read against that row rather than inferred from it.

\paragraph{Interpretation at the code layer.}
The result should be read as a code-table statement.  The contact supports codon-to-label readback and local split tests for the standard genetic code.  It does not observe translation dynamics, ribosomal implementation, molecular folding, energetic admissibility, organismal function, or a universal generative mechanism for biology.  An amino-acid label sequence obtained from the table is still only a label sequence; it is not a folded protein, not a physical structure, and not evidence of function.  Any claim crossing those layers would require an additional contact that measures the relevant higher-level object.  Within the present data, the supported conclusion is narrower and finite: the window-six codon coordinates expose a local boundary-tile organization of the curated standard-code table, and the leave-one-codon module classification covers the entire $13$-codon boundary row without surplus or omission.

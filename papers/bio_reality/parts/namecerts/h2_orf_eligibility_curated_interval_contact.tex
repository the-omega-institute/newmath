\subsection{Curated interval contact for ORF eligibility}
\label{sec:h2-orf-eligibility-curated-interval-contact}

\paragraph{Claim and biological scope.}
This record concerns open-reading-frame eligibility as a finite start--stop interval question.  The biological object is not a translated peptide, not a folded molecule, and not a cellular function.  It is the narrower assertion that a curated nucleotide interval can be tested against a fixed start/stop-window classifier without allowing the classifier to learn from translation, structure, biophysics, phenotype, or mechanism.  The packet therefore has one contact-bearing layer: interval eligibility against a curated annotation source.  Every stronger biological reading remains outside the result unless a different observation supplies it.

\paragraph{Frozen protocol.}
The protocol was fixed before the curated intervals were observed.  The rules for the classifier named $\mathrm{orf\_start\_stop\_window\_eligibility}$ were not modified; the baseline comparisons, input classes, and held-out splits were already declared.  The protocol timestamp was $2026$-$05$-$29\,19{:}11{:}13+00{:}00$, while the external observation timestamp was $2026$-$05$-$30\,10{:}29{:}31+00{:}00$.  The analysis records explicitly set translation and structure inputs to absent.  Thus the finite test separates a geometry-only start/stop eligibility calculation from any downstream biological realization.

\paragraph{Curated finite table.}
The interval table was loaded from \texttt{tools/bio\_reality/data/curated\_orf\_interval\_contact\_dataset.json}.  It contains $168$ rows, sampled from NCBI RefSeq GenBank XML CDS intervals for \emph{Escherichia coli} K--12 MG1655 complete genome accession NC\_000913.3, snapshot date $2026$-$05$-$30$.  The required row classes are exactly present: boundary cases, matched non-ORF decoys, and positive curated ORFs.  The test uses $72$ positive curated intervals, $72$ matched decoys, and $24$ boundary cases.  These numbers are the finite support of the claim; there is no extrapolation to unlisted genomic windows.

\paragraph{Start/stop eligibility contact.}
The predecessor window-enumeration contact loaded one external window on snapshot date $2026$-$05$-$25$ and classified all input windows by start/stop enumeration.  Its scope was recorded as external-window enumeration only.  The present curated-interval contact enlarges the finite biological table to $168$ rows while preserving the same boundary: the classifier may accept or reject interval eligibility, but it may not promote that classification to translation, protein existence, fold formation, physical admissibility, function, mechanism, or a universal genomic law.

\paragraph{Boundary cases.}
The $24$ boundary cases were not promoted: the promoted count is $0$ and the promoted fraction is $0.0$.  This matters because the boundary rows test whether the pipeline silently turns ambiguous start/stop geometry into an ORF-level biological assertion.  It does not do so on this finite table.  The local rule is therefore: a curated interval may be counted for ORF eligibility only when the curated annotation contact is present; otherwise the record remains a boundary probe rather than an interval claim.
